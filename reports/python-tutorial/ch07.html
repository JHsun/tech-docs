CHAPTER 07 · PART 2

函式與函數式編程

在 Python 中，函式是「一等公民」—— 可以被賦值、當參數傳遞、當回傳值、儲存在資料結構中。本章從參數系統開始，走到 Python 最具代表性的語法糖 裝飾器（decorator），並帶你進入函數式編程的思維世界。

Python 3.14 參數 · 閉包 · 裝飾器 預估閱讀 30 分鐘

§0章節導論

Python 函式系統的強大之處在於它的統一與一致：

  • 任何值（包括函式本身）都是物件，可以任意傳遞
  • 五種參數類型（位置、關鍵字、預設、可變、關鍵字可變）組合出靈活的 API
  • 作用域遵循 LEGB 規則，錯誤理解會造成最多的 bug
  • 閉包、裝飾器、高階函式 —— 函數式編程的完整工具箱

§7.1函式基礎

python 
# 最基本的函式
def greet(name):
    return f"Hello, {name}!"

# 呼叫
msg = greet("Alice")

# 沒有 return = 隱式回傳 None
def print_hi():
    print("Hi!")
result = print_hi()   # result = None

# 多值回傳 — 實際上是回傳 tuple
def min_max(lst):
    return min(lst), max(lst)

lo, hi = min_max([3, 1, 4])   # 解包

# Docstring — 函式的說明文件
def divide(a, b):
    """兩數相除。

    Args:
        a: 被除數
        b: 除數（不得為 0）
    Returns:
        a / b 的結果
    Raises:
        ZeroDivisionError: 當 b = 0
    """
    return a / b

help(divide)     # 顯示 docstring
print(divide.__doc__)

函式即物件

python 
def greet(name): return f"Hi {name}"

# 函式是物件，可以被賦值
f = greet
print(f("Alice"))       # 'Hi Alice'

# 可當作引數傳遞
def apply(fn, x):
    return fn(x)

apply(greet, "Bob")      # 'Hi Bob'

# 可放進資料結構
operations = [str.upper, str.lower, len]
for op in operations:
    print(op("Hello"))

§7.2參數系統

位置參數與關鍵字參數

python 
def point(x, y):
    return (x, y)

point(3, 4)          # 位置傳遞
point(y=4, x=3)      # 關鍵字傳遞（順序可變）
point(3, y=4)        # 混合（位置必須在前）
point(x=3, 4)        # ❌ SyntaxError

預設值

python 
def greet(name, greeting="Hello", punct="!"):
    return f"{greeting}, {name}{punct}"

greet("Alice")                        # 'Hello, Alice!'
greet("Bob", "Hi")                    # 'Hi, Bob!'
greet("Chloe", punct=".")              # 'Hello, Chloe.'
⚠️ 預設值的可變物件陷阱 —— Python 最著名的坑！
# ❌ 錯誤：預設值在函式定義時「只創建一次」
def append_to(item, lst=[]):    # 💀 這個 [] 是共享的！
    lst.append(item)
    return lst

append_to(1)    # [1]
append_to(2)    # [1, 2] ← 不是 [2]！
append_to(3)    # [1, 2, 3]

# ✅ 正確做法：用 None 作為 sentinel
def append_to(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst

§7.3*args 與 **kwargs

處理「不定數量參數」的兩個武器。名字不是保留字，但慣例必須遵守。

python 
# *args — 收集多餘的「位置參數」為 tuple
def sum_all(*args):
    return sum(args)

sum_all(1, 2, 3)                # 6
sum_all(1, 2, 3, 4, 5)          # 15

# **kwargs — 收集多餘的「關鍵字參數」為 dict
def show(**kwargs):
    for k, v in kwargs.items():
        print(f"{k}={v}")

show(name="Alice", age=30, city="Taipei")

# 混用
def full(a, b, *args, key="x", **kwargs):
    print(a, b, args, key, kwargs)

full(1, 2, 3, 4, 5, key="y", z=10)
# 1  2  (3, 4, 5)  y  {'z': 10}

呼叫時反向解包

python 
def point(x, y, z):
    return (x, y, z)

coords = [1, 2, 3]
point(*coords)              # 等於 point(1, 2, 3)

kwargs = {"x": 1, "y": 2, "z": 3}
point(**kwargs)             # 等於 point(x=1, y=2, z=3)

# 轉發參數給另一個函式（很常用的模式）
def logged(*args, **kwargs):
    print("called")
    return original(*args, **kwargs)

§7.4位置限定 / 關鍵字限定參數

Python 3.8+ 提供 / 和 * 兩個特殊符號，精準控制參數傳遞方式。

python 
# / 之前的參數：只能位置傳遞
# * 之後的參數：只能關鍵字傳遞
def func(a, b, /, c, d, *, e, f):
    return (a, b, c, d, e, f)

# a, b 只能位置傳
# c, d 可位置或關鍵字
# e, f 只能關鍵字傳

func(1, 2, 3, 4, e=5, f=6)      # ✅
func(1, 2, c=3, d=4, e=5, f=6)  # ✅
func(a=1, b=2, ...)                # ❌ TypeError
func(1, 2, 3, 4, 5, 6)           # ❌ TypeError（e, f 必須關鍵字）

實務應用

python 
# 強制用關鍵字，讓呼叫端更易讀
def create_user(name, *, admin=False, active=True):
    ...

create_user("Alice", admin=True)     # ✅ 意圖清楚
create_user("Alice", True)           # ❌ 強制你寫 admin=True

# 禁止關鍵字傳遞，保護 API 命名彈性
def distance(p1, p2, /):
    return ((p1[0] - p2[0])**2 + (p1[1] - p2[1])**2) ** 0.5

§7.5作用域與 LEGB 規則

變數查找順序：Local → Enclosing → Global → Built-in。

python 
x = "global"

def outer():
    x = "enclosing"
    def inner():
        x = "local"
        print(x)   # 'local'
    inner()
    print(x)       # 'enclosing'

outer()
print(x)           # 'global'

global 與 nonlocal

python 
count = 0

def increment():
    global count      # 宣告要修改全域變數
    count += 1

# 改巢狀函式外層（非全域）的變數用 nonlocal
def counter():
    n = 0
    def tick():
        nonlocal n    # 修改 outer() 的 n
        n += 1
        return n
    return tick

c = counter()
print(c(), c(), c())    # 1 2 3
⚠️ 只讀不寫可省略 global/nonlocal

讀取外層變數不需要宣告，但只要有賦值，Python 就把該變數當作 local（除非宣告）。這規則常讓新手困惑。

§7.6Lambda 匿名函式

Lambda 是單一運算式的匿名函式。語法：lambda 參數: 運算式。

python 
square = lambda x: x ** 2
square(5)     # 25

lambda x, y: x + y      # 多參數
lambda: 42                # 無參數
lambda x=1: x * 2         # 帶預設值

# 真正的用途：當作「拋棄式」參數
sorted(users, key=lambda u: u.age)
list(filter(lambda x: x > 0, nums))
list(map(lambda x: x*2, nums))
💡 什麼時候用 lambda？

只有在：(1) 函式極短（一個運算式）、(2) 只用一次、(3) 放在 sorted/map/filter/key 等參數位置。
若要命名、加 docstring、超過一行，一律用 def。PEP 8 甚至禁止「把 lambda 指派給變數」。

§7.7高階函式 Higher-Order Functions

python 
# map — 對每個元素套函式
list(map(str, [1, 2, 3]))          # ['1', '2', '3']
list(map(lambda x: x*2, [1, 2, 3])) # [2, 4, 6]

# filter — 依條件過濾
list(filter(lambda x: x>0, [-1, 2, -3, 4]))  # [2, 4]
list(filter(None, [0, 1, None, "", "x"]))  # [1, 'x']（只留 truthy）

# functools.reduce — 累積運算
from functools import reduce
reduce(lambda a, b: a * b, [1, 2, 3, 4])   # 24 (階乘)
reduce(lambda a, b: a + b, [1, 2, 3], 100)   # 106 (初始值 100)

# sorted 的 key 函式
users = [{"name": "Alice", "age": 30},
         {"name": "Bob", "age": 25}]
sorted(users, key=lambda u: u["age"])
📘 推導式通常比 map/filter 更 Pythonic
# 相對於
list(map(lambda x: x*2, filter(lambda x: x > 0, nums)))
# 推導式更清晰：
[x*2 for x in nums if x > 0]

§7.8閉包 Closure

閉包 = 函式 + 它能「記住」的外層變數。Python 的內層函式可以引用外層函式的變數，即使外層函式早已結束。

python 
def make_multiplier(factor):
    def multiply(x):
        return x * factor       # 引用外層的 factor
    return multiply

double = make_multiplier(2)
triple = make_multiplier(3)
double(5)    # 10
triple(5)    # 15

# 閉包保留狀態 — 計數器
def counter(start=0):
    n = start
    def tick():
        nonlocal n
        n += 1
        return n
    return tick

c = counter()
c(); c(); print(c())   # 3
⚠️ 迴圈中建立閉包的經典陷阱
# ❌ 所有 lambda 共享同一個 i
funcs = [lambda: i for i in range(3)]
print([f() for f in funcs])    # [2, 2, 2]（全都是最後的 i）

# ✅ 用預設參數「凍結」當下的值
funcs = [lambda i=i: i for i in range(3)]
print([f() for f in funcs])    # [0, 1, 2]

§7.9裝飾器 Decorator（Python 核心特性）

裝飾器是「接收函式、回傳函式」的函式，用 @decorator 語法套用。它讓你能在不修改原函式的前提下，為它加功能。

最簡單的裝飾器

python 
def log(func):
    def wrapper(*args, **kwargs):
        print(f"呼叫 {func.__name__}")
        result = func(*args, **kwargs)
        print(f"完成 {func.__name__}")
        return result
    return wrapper

@log
def greet(name):
    print(f"Hello {name}")

greet("Alice")
# 呼叫 greet
# Hello Alice
# 完成 greet

# @log 語法等同於：
# greet = log(greet)

保留原函式元資訊：functools.wraps

python 
from functools import wraps

def log(func):
    @wraps(func)         # ← 關鍵！保留原函式的 __name__、__doc__ 等
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@log
def greet(name):
    """打招呼"""
    return f"Hi {name}"

print(greet.__name__)    # 'greet'（沒加 @wraps 會是 'wrapper'）
print(greet.__doc__)     # '打招呼'

帶參數的裝飾器

python 
def retry(times=3):
    """帶參數的裝飾器：三層嵌套函式"""
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for i in range(times):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    print(f"第 {i+1} 次失敗: {e}")
            raise Exception("全部失敗")
        return wrapper
    return decorator

@retry(times=5)
def fetch_data():
    import random
    if random.random() < 0.7:
        raise ConnectionError("timeout")
    return "data"

多重裝飾器

python 
@outer
@middle
@inner
def f(): ...

# 等同於：
# f = outer(middle(inner(f)))

# 執行順序：最靠近 def 的先套用，呼叫時最外層先執行

實用裝飾器範例

python 
# 計時器
import time
from functools import wraps

def timer(func):
    @wraps(func)
    def wrapper(*a, **kw):
        t0 = time.perf_counter()
        r = func(*a, **kw)
        print(f"{func.__name__}: {time.perf_counter()-t0:.3f}s")
        return r
    return wrapper

# 權限檢查
def require_admin(func):
    @wraps(func)
    def wrapper(user, *a, **kw):
        if not user.is_admin:
            raise PermissionError("需要 admin 權限")
        return func(user, *a, **kw)
    return wrapper

§7.10functools 模組精華

python 
from functools import wraps, cache, lru_cache, partial, reduce

# @cache — 無上限快取（3.9+）
@cache
def fib(n):
    return n if n < 2 else fib(n-1) + fib(n-2)

fib(100)    # 瞬間完成（原本會超慢）

# @lru_cache — 有上限的 LRU 快取
@lru_cache(maxsize=128)
def slow_query(key): ...

# partial — 固定部分參數
from functools import partial

def power(base, exp):
    return base ** exp

square = partial(power, exp=2)
square(5)        # 25
cube = partial(power, exp=3)
cube(5)          # 125

# @singledispatch — 根據第一個引數型別分派
from functools import singledispatch

@singledispatch
def process(data):
    raise NotImplementedError

@process.register
def _(data: int):
    return f"int: {data * 2}"

@process.register
def _(data: str):
    return f"str: {data.upper()}"

process(5)      # 'int: 10'
process("hi")   # 'str: HI'

🎁 裝飾器樣板
from functools import wraps

# 不帶參數
def deco(func):
    @wraps(func)
    def wrapper(*a, **kw):
        # 前處理
        r = func(*a, **kw)
        # 後處理
        return r
    return wrapper

# 帶參數（三層）
def deco_with_args(arg):
    def decorator(func):
        @wraps(func)
        def wrapper(*a, **kw):
            return func(*a, **kw)
        return wrapper
    return decorator

@deco            # f = deco(f)
@deco_with_args(5)    # f = deco_with_args(5)(f)
🛠 functools
@cache             # 無限快取
@lru_cache(maxsize=128)
@wraps(func)       # 裝飾器必備
@singledispatch    # 依型別分派

partial(f, x=1)    # 固定參數

⚠️ 經典陷阱

  • 忘記 @wraps 導致 debug 痛苦
