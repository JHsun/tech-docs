CHAPTER 26 · PART 8

資料處理三巨頭

Python 在資料科學的地位來自三大套件：NumPy（數值計算基石）、pandas（DataFrame 老王）、Polars（Rust 驅動的 2025-2026 新寵，速度 5-20x 於 pandas）。本章讓你同時掌握三者，並知道何時選用哪一個。

Python 3.14 NumPy · pandas · Polars 預估閱讀 30 分鐘

§0章節導論

工具定位適用場景
NumPy底層數值陣列數學計算、科學運算、ML 前處理、大多數庫的基礎
pandas表格資料操作中小型資料探索、生態系最廣（matplotlib、scikit-learn 深整合）
Polars高效能 DataFrame大資料、ETL、查詢優化、串流處理（超過記憶體的資料）
🆕 2026 年的資料處理生態

Polars 已從「新興工具」升級為「一線選擇」：NumPy 2.3+ 完整支援 Free-threaded Python；pandas 3.0 規劃以 PyArrow 為預設後端。如果你現在才開始學資料處理，直接學 Polars 會更划算。但 pandas 的市場慣性仍大，理解兩者才能讀懂大多數現有程式碼。

§26.1NumPy —— 數值運算基石

bash

python 
import numpy as np

# 建立陣列
a = np.array([1, 2, 3, 4])           # 1D
b = np.array([[1, 2], [3, 4]])       # 2D
np.zeros((3, 4))                      # 3×4 全 0
np.ones((2, 3))                       # 2×3 全 1
np.arange(0, 10, 0.5)                  # 0 ~ 9.5 步 0.5
np.linspace(0, 1, 5)                    # 0~1 等距 5 個點
np.random.rand(3, 4)                  # 隨機 0~1
np.random.randn(100)                   # 常態分佈

# 屬性
a.shape             # (4,)
a.ndim              # 1
a.dtype             # int64
a.size              # 4

# 型別指定
np.array([1, 2, 3], dtype=np.float32)

# 索引與切片（與 list 類似，但更強）
a[0], a[-1], a[1:3]
b[0, 1]                        # 二維索引（不是 b[0][1]）
b[:, 0]                         # 第 0 欄
b[1, :]                         # 第 1 列

# 布林索引（超好用）
a[a > 2]                      # array([3, 4])
a[(a > 1) & (a < 4)]            # & 不是 and！

向量化運算（關鍵）

python 
# ❌ 慢 — Python 迴圈
result = [x**2 for x in range(1_000_000)]

# ✅ 快 50-100 倍 — NumPy 向量化
a = np.arange(1_000_000)
result = a ** 2

# 陣列四則運算（逐元素）
a + b          # 相加
a * 2          # 每元素 × 2
np.sqrt(a)     # 每元素開根
np.exp(a)      # e^x
np.sin(a)      # sin

# 廣播（broadcasting）— 自動對齊形狀
a = np.array([[1, 2, 3], [4, 5, 6]])    # 2×3
b = np.array([10, 20, 30])             # 3
a + b      # b 自動被「廣播」到每列
# [[11, 22, 33],
#  [14, 25, 36]]

# groupby — 資料分析核心
df.groupby("city")["age"].mean()           # 每城市平均年齡
df.groupby("city").agg({
    "age": ["mean", "min", "max", "count"],
    "salary": "sum",
})

df.pivot_table(
    index="city",
    columns="age_group",
    values="salary",

)

§26.6🆕 Polars —— Rust 驅動新星

🆕 為什麼 Polars 很重要
  • Rust 核心 + 多核心並行：對大資料比 pandas 快 5-20 倍
  • Apache Arrow 記憶體格式：欄位式儲存，適合分析
  • LazyFrame 查詢優化：類似資料庫的查詢計劃器
  • 串流處理：可處理超過記憶體的資料（pandas 做不到）
  • 表達式 DSL：一致、可組合、易優化的 API
bash 
uv add polars
python 
import polars as pl

# 建立 DataFrame（與 pandas 類似）
df = pl.DataFrame({
    "name": ["Alice", "Bob", "Charlie"],
    "age": [30, 25, 35],
    "city": ["Taipei", "Tokyo", "NYC"],
})

# 讀檔（介面類似 pandas）
df = pl.read_csv("data.csv")
df = pl.read_parquet("data.parquet")
df = pl.read_excel("data.xlsx")

# 存檔
df.write_csv("out.csv")
df.write_parquet("out.parquet")

Polars 核心：表達式（Expression）

python 
# Polars 用「表達式」取代 pandas 的 lambda
# pl.col("x") 代表「x 欄」，可組合出複雜運算

# select — 選擇與轉換欄位
df.select(
    pl.col("name"),
    pl.col("age") + 1,                       # age + 1
    (pl.col("age") * 2).alias("double_age"),
)

# filter
df.filter(pl.col("age") > 25)
df.filter((pl.col("age") > 25) & (pl.col("city") == "Taipei"))

# with_columns — 新增/修改欄位
df.with_columns(
    (pl.col("age") * 2).alias("age_x2"),
    pl.col("name").str.to_uppercase().alias("name_upper"),
    age_group=pl.when(pl.col("age") >= 30)
               .then(pl.lit("senior"))
               .otherwise(pl.lit("junior")),
)

# 排序
df.sort("age", descending=True)

# group_by（注意底線！）
df.group_by("city").agg(
    pl.col("age").mean().alias("avg_age"),
    pl.col("salary").sum().alias("total_salary"),
    pl.len().alias("count"),
)

# 合併
df1.join(df2, on="id", how="inner")

# 和 pandas 互轉
df_pandas = df.to_pandas()
df_polars = pl.from_pandas(df_pandas)

§26.7🆕 LazyFrame —— 查詢優化

Polars 最強的殺手鐧：LazyFrame。不立即執行，而是建立查詢計劃，執行前自動優化（類似 SQL 資料庫）。

python 
import polars as pl

# scan_* 建立 LazyFrame（不載入資料！）
lf = pl.scan_csv("huge_data.csv")    # 即使是 30GB 的檔案
lf = pl.scan_parquet("huge.parquet")

# 建立複雜查詢（不實際執行）
result = (
    pl.scan_csv("sales.csv")
    .filter(pl.col("date") >= "2025-01-01")
    .filter(pl.col("amount") > 100)
    .group_by("region")
    .agg(
        pl.col("amount").sum().alias("total"),
        pl.col("order_id").count().alias("orders"),
    )
    .sort("total", descending=True)
    .head(10)
    .collect()                          # ← 呼叫 collect() 才會執行
)

# 檢視查詢計劃（優化前後）
lf.explain()                            # 文字形式
lf.show_graph()                         # 視覺化 DAG

為什麼這麼快？

Polars 自動執行的優化包括：

  • Predicate Pushdown（條件下推）：filter 條件會被推到讀檔階段，只讀需要的列
  • Projection Pushdown（欄位下推）：只讀真正會用到的欄位，省 I/O
  • Common Subexpression Elimination：重複計算合併為一次
  • 平行處理：自動用所有 CPU 核心
  • Streaming：不一次載入全部資料，分塊處理

超過記憶體的資料

python 
# pandas 處理 30GB CSV → 記憶體爆掉
# Polars 用 streaming 可處理超過 RAM 的資料

result = (
    pl.scan_csv("30gb_file.csv")
    .filter(pl.col("year") == 2025)
    .group_by("category")
    .agg(pl.col("value").sum())
    .collect(streaming=True)      # 🔑 串流模式
)

# 或直接寫出，不要整個結果載入記憶體
(pl.scan_csv("huge.csv")
    .filter(pl.col("valid"))
    .sink_parquet("filtered.parquet"))   # 直接寫出

§26.8選擇決策

情境推薦
純數值計算、矩陣運算NumPy
小資料（< 100 萬列）、需要豐富生態pandas
中大資料（100 萬列以上）、ETLPolars
超過記憶體的資料Polars + streaming
要與 scikit-learn 整合pandas（或 Polars 最後 .to_pandas()）
查詢優化、資料管線Polars LazyFrame
純探索式分析（Jupyter）pandas（即時回饋）
新專案Polars（建議）

API 語法差異速看

python · pandas vs polars 對照 
# 讀 CSV 並篩選聚合

# pandas
df = pd.read_csv("sales.csv")
result = (df[df["amount"] > 100]
          .groupby("region")
          .agg(total=("amount", "sum"))
          .sort_values("total", ascending=False))

# Polars（eager）
df = pl.read_csv("sales.csv")
result = (df.filter(pl.col("amount") > 100)
          .group_by("region")
          .agg(pl.col("amount").sum().alias("total"))
          .sort("total", descending=True))

# Polars（lazy，最快）
result = (pl.scan_csv("sales.csv")
          .filter(pl.col("amount") > 100)
          .group_by("region")
          .agg(pl.col("amount").sum().alias("total"))
          .sort("total", descending=True)
          .collect())
