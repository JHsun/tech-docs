CHAPTER 01 · PART 1

環境建置與開發工具

學 Python 的第一步不是寫 Hello World，而是把「環境」搞對。本章將帶你安裝 Python 3.14、建立虛擬環境、認識現代 Python 工具鏈的靈魂 —— uv，以及設定舒適的編輯器環境。

Python 3.14.4 跨平台（macOS / Linux / Windows） 預估閱讀 15 分鐘

§0章節導論

在開始寫 Python 之前，你需要一個能穩定執行程式的環境。本章聚焦三件事：

  • 安裝正確的 Python 版本（目前最新穩定版為 3.14.4，2026 年 4 月發布）
  • 隔離專案依賴，使用虛擬環境避免「在我電腦可以跑」的悲劇
  • 掌握現代工具鏈：從古老的 pip 到極速的 uv，並了解 REPL、編輯器整合
🆕 Python 3.14 的重點更新

Windows 版本改用全新的 Python Install Manager（取代舊版 installer）、官方 macOS/Windows 二進位包預設內建實驗性 JIT 編譯器、REPL 加入語法高亮與匯入自動補全、自 3.14 起改用 Sigstore 簽章取代 PGP。

§1.1安裝 Python 3.14

依照你的作業系統選擇最順手的安裝方式。本書推薦使用 uv 管理 Python 版本（見 §1.5），但先了解傳統方式也有好處。

方式 A：官方安裝檔

前往 python.org/downloads 下載 3.14.4。

  • Windows：下載 Python Install Manager（推薦）或傳統 .exe installer。勾選「Add python.exe to PATH」。
  • macOS：下載 .pkg，安裝後可用 python3 --version 驗證。
  • Linux：建議使用發行版套件管理器或 pyenv/uv，避免動到系統 Python。

方式 B：用 uv 安裝（最推薦）

bash 
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows (PowerShell)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# 安裝 Python 3.14（並列多版本）
uv python install 3.14
uv python install 3.12 3.13 3.14  # 一次安裝三個

# 查看已安裝版本
uv python list

驗證安裝

bash 
python --version      # 應顯示 Python 3.14.4
python3 --version     # Linux/macOS 通常用 python3
python -c "import sys; print(sys.version)"
⚠️ 常見陷阱

Windows 上若 python 打開 Microsoft Store，表示 PATH 未正確設定。重新安裝時請勾選「Add to PATH」，或改用 Install Manager。

§1.2Windows 全新 Install Manager

Python 3.14 時代，Windows 版本引進了全新的 Python Install Manager（類似 Node 的 nvm），可以管理多個 Python 版本共存。

PowerShell 
py install 3.14         # 安裝 Python 3.14
py install 3.13         # 安裝 3.13 並存
py list                 # 列出已安裝版本
py -3.14                # 啟動 3.14 的 REPL
py -3.13 script.py      # 用 3.13 執行特定腳本
py uninstall 3.13       # 移除某版本

Install Manager 可從 Microsoft Store 取得，也可從 python.org 下載獨立安裝檔。它會取代舊版 py.exe 啟動器。

§1.3虛擬環境 venv

虛擬環境（virtual environment）是一個獨立的 Python 工作區，讓不同專案能使用不同版本的套件而不互相干擾。這是 Python 工程的絕對基本功。

bash 
# 建立虛擬環境（資料夾名習慣用 .venv）
python -m venv .venv

# 啟動（macOS / Linux）
source .venv/bin/activate

# 啟動（Windows PowerShell）
.venv\Scripts\Activate.ps1

# 啟動成功後，shell 提示符會出現 (.venv)
# 在此環境內 pip install 的套件只存在於此專案

deactivate              # 離開虛擬環境
💡 最佳實踐

每個新專案一開始就建立 .venv，並加入 .gitignore。現代工具如 uv 會自動處理虛擬環境，你幾乎不用手動管理。

§1.4pip 傳統套件管理

pip 是 Python 內建的套件管理器，用於從 PyPI（Python Package Index）安裝第三方套件。

bash 
pip install requests              # 安裝套件
pip install "django>=5.0"            # 指定版本
pip install -U requests           # 升級
pip uninstall requests            # 移除
pip list                          # 列出已安裝套件
pip show requests                 # 顯示套件資訊
pip freeze > requirements.txt     # 匯出依賴清單
pip install -r requirements.txt   # 從清單批次安裝

requirements.txt 範例

requirements.txt 
requests>=2.32
fastapi>=0.110
pydantic>=2.5,<3.0
pytest>=8.0

§1.5uv — 現代 Python 的瑞士刀

uv 是由 Astral 公司（Ruff 作者）用 Rust 寫成的極速 Python 工具，一口氣取代 pip、pip-tools、pipx、poetry、pyenv、virtualenv 這六個工具。安裝套件比 pip 快 10–100 倍，CI 流程可省下大量時間。

核心命令

bash 
# 建立新專案（自動生成 pyproject.toml、.venv）
uv init my-app
cd my-app

# 新增依賴（自動裝進專案虛擬環境）
uv add requests fastapi

# 新增開發依賴
uv add --dev pytest ruff

# 移除依賴
uv remove requests

# 執行腳本（自動啟動 venv）
uv run main.py

# 同步鎖定檔
uv sync

# 臨時執行工具（像 npx）
uvx ruff check .
uvx black script.py

pip 與 uv 命令對照

任務pip 版本uv 版本
建立虛擬環境python -m venv .venvuv venv
安裝套件pip install requestsuv add requests
全域工具pipx install ruffuv tool install ruff
一次性執行—uvx ruff check
從 requirements 安裝pip install -r req.txtuv pip install -r req.txt
鎖定依賴pip freezeuv lock
📘 為什麼要用 uv？

傳統工作流需要 pyenv 管版本 + venv 建環境 + pip 裝套件 + pip-tools 鎖版本，mental model 複雜。uv 一個工具搞定，pyproject.toml 統一配置，跨專案體驗一致。本書後續章節預設使用 uv。

§1.6REPL 互動介面（3.14 大改版）

Python REPL（Read-Eval-Print Loop）是最快的實驗工具。輸入 python 即可進入。

🆕 Python 3.14 REPL 新功能
  • 語法高亮：關鍵字、字串、數字、註解皆有彩色顯示
  • import 自動補全：打 import ma 按 Tab → 補齊 math
  • 錯誤訊息智慧建議：improt math 會提示 Did you mean 'import'?
  • pdb 除錯器：語法高亮 + 遠端附加到執行中的程序
python REPL 
$ python
Python 3.14.4 (main, Apr  7 2026, 12:00:00)
>>> 1 + 2
3
>>> import math
>>> math.pi
3.141592653589793
>>> help(str.upper)     # 查看內建說明
>>> exit()              # 或 Ctrl+D 離開

REPL 小技巧

  • _ 代表上一次運算結果
  • 上箭頭回顧歷史命令
  • dir(obj) 列出物件所有屬性方法
  • help(obj) 顯示 docstring

兩大主流選擇：VS Code（免費、輕量、生態強）與 PyCharm（重量級 IDE、除錯與重構王者）。

VS Code 推薦擴充套件

§1.8執行第一個程式

hello.py 
# hello.py — 你的第一支 Python 程式
def greet(name: str) -> str:
    return f"你好，{name}！歡迎來到 Python 3.14"

if __name__ == "__main__":
    print(greet("世界"))

三種執行方式

bash 
python hello.py              # 傳統
uv run hello.py              # 用 uv（自動處理 venv）
python -m hello              # 作為模組執行

PEP 723 行內腳本依賴（3.12+）

如果腳本需要外部套件，可在檔案頂端宣告依賴，配合 uv run 使用：

fetch.py 
# /// script
# requires-python = ">=3.14"
# dependencies = ["requests"]
# ///
import requests
print(requests.get("https://api.github.com").status_code)

# 執行：uv run fetch.py — uv 會自動建立臨時環境並安裝 requests

§1.9最佳實踐

  • 絕不動系統 Python：Linux/macOS 的系統 Python 是給作業系統用的，破壞它會讓系統工具失靈
  • 一個專案一個 venv：依賴絕對不要裝到全域
  • 用 uv 取代 pip + venv：速度、體驗、可重現性三面向勝出
  • 鎖定依賴版本：uv lock 產生 uv.lock，確保團隊與 CI 環境一致
  • .gitignore 基本款：.venv/、__pycache__/、*.pyc、.pytest_cache/、.ruff_cache/
  • 保持 Python 更新：每年 10 月會發布新的主版本（3.15 預計 2026.10）
