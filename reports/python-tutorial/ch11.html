進階類別特性

現代 Python 提供了一批「類別語法糖」讓你寫出簡潔、正確、可維護的程式碼：@dataclass 消除樣板、Enum 表達狀態、Protocol 定義介面、__slots__ 節省記憶體、描述器則是 Python 許多神奇特性的底層基礎。

Python 3.14 dataclass / Enum / Protocol 預估閱讀 25 分鐘

§0章節導論

Ch09 介紹的是類別語法基礎，本章進入實戰常用的高階特性：

  • dataclass：自動產生 __init__、__repr__、__eq__，寫資料類別不再痛苦
  • Enum：代替魔法常數，讓程式碼自我文件化
  • ABC 與 Protocol：兩種不同哲學的介面定義
  • __slots__：限制屬性、加速存取、省記憶體
  • Descriptor：理解 @property、@staticmethod、__get__ 的秘密

§11.1dataclass — 資料類別

以前要寫一個簡單的資料類別，需要 __init__、__repr__、__eq__ 三個樣板方法。@dataclass 自動幫你生成。

python 
from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float
    label: str = "origin"      # 預設值

# 等同於手寫：
# class Point:
#     def __init__(self, x: float, y: float, label: str = "origin"):
#         self.x, self.y, self.label = x, y, label
#     def __repr__(self):
#         return f"Point(x={self.x!r}, y={self.y!r}, label={self.label!r})"
#     def __eq__(self, other):
#         return (self.x, self.y, self.label) == (other.x, other.y, other.label)

p = Point(3, 4)
print(p)              # Point(x=3, y=4, label='origin')
Point(1, 2) == Point(1, 2)   # True
⚠️ 可變預設值要用 field()
from dataclasses import dataclass, field

@dataclass
class Bad:
    items: list = []             # ❌ ValueError！禁止可變預設值

@dataclass
class Good:
    items: list = field(default_factory=list)  # ✅ 每個實例獨立的 list
    tags: set = field(default_factory=set)
    counter: int = 0

§11.2dataclass 進階選項

python 
from dataclasses import dataclass, field

@dataclass(
    frozen=True,      # 不可變（像 tuple）
    order=True,       # 自動產生 < <= > >= 比較
    slots=True,       # 使用 __slots__ 優化（3.10+）
    kw_only=True,     # 所有欄位強制用關鍵字傳（3.10+）
)
class User:
    name: str
    age: int
    tags: list[str] = field(default_factory=list)
    password: str = field(default="", repr=False)  # repr 中隱藏
    internal_id: int = field(default=0, compare=False)  # 不參與 == 比較

u = User(name="Alice", age=30)
print(u)                  # User(name='Alice', age=30, tags=[])
u.age = 31               # ❌ FrozenInstanceError（因為 frozen=True）

__post_init__ 後處理

python 
@dataclass
class Circle:
    radius: float
    area: float = field(init=False)   # 不在 __init__ 參數中

    def __post_init__(self):
        """__init__ 結束後自動呼叫，做衍生計算或驗證"""
        if self.radius < 0:
            raise ValueError("radius must be >= 0")
        self.area = 3.14159 * self.radius ** 2

c = Circle(5)
print(c)   # Circle(radius=5, area=78.53975)

轉換工具 asdict / astuple

python 
from dataclasses import asdict, astuple, replace

p = Point(3, 4)
asdict(p)              # {'x': 3, 'y': 4, 'label': 'origin'}
astuple(p)             # (3, 4, 'origin')

# replace — 複製並修改某欄位（適用 frozen）
p2 = replace(p, x=10)   # Point(x=10, y=4, label='origin')

§11.3Enum — 列舉

Enum 讓你用具名常數取代「魔法數字」/「魔法字串」，使程式碼自我文件化。

python 
from enum import Enum, IntEnum, StrEnum, auto

class Color(Enum):
    RED = 1
    GREEN = 2
    BLUE = 3

# 存取
Color.RED                   # <Color.RED: 1>
Color.RED.name              # 'RED'
Color.RED.value             # 1
Color(1)                    # <Color.RED: 1> (依值查詢)
Color["RED"]                # <Color.RED: 1> (依名查詢)

# 比較
Color.RED == Color.RED     # True
Color.RED is Color.RED     # True（Enum 成員都是單例）

# 走訪
for c in Color:
    print(c.name, c.value)

list(Color)                 # [Color.RED, Color.GREEN, Color.BLUE]
len(Color)                  # 3

Enum 變體

python 
# auto() 自動編號
class Status(Enum):
    PENDING = auto()          # 1
    ACTIVE = auto()           # 2
    DONE = auto()             # 3

# IntEnum — 成員同時是 int
class HTTPStatus(IntEnum):
    OK = 200
    NOT_FOUND = 404
    ERROR = 500

HTTPStatus.OK == 200         # True！直接與 int 比較
HTTPStatus.OK + 1           # 201

# StrEnum — 成員同時是 str（3.11+）
class Priority(StrEnum):
    LOW = "low"
    HIGH = "high"

Priority.LOW == "low"        # True

# Flag — 位元旗標
from enum import Flag
class Perm(Flag):
    READ = auto()
    WRITE = auto()
    EXEC = auto()

rw = Perm.READ | Perm.WRITE
Perm.READ in rw             # True
💡 Enum 的典型場景
  • 訂單狀態、權限等級、卡片花色 —— 有限集合
  • HTTP 狀態碼、錯誤代碼 —— 用 IntEnum
  • API 參數 —— 用 StrEnum 讓序列化成字串
  • 權限組合 —— 用 Flag 支援位元運算

§11.4ABC — 抽象基底類別

當你想強制子類必須實作某些方法時，用 ABC（Abstract Base Class）+ @abstractmethod。

python 
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self) -> float:
        """子類必須實作"""

    @abstractmethod
    def perimeter(self) -> float:
        ...

    def describe(self):   # 可混入具體方法
        return f"Area={self.area()}, Perimeter={self.perimeter()}"

# 不能直接實例化
Shape()   # ❌ TypeError: Can't instantiate abstract class

class Rectangle(Shape):
    def __init__(self, w, h):
        self.w, self.h = w, h
    def area(self):
        return self.w * self.h
    def perimeter(self):
        return 2 * (self.w + self.h)

r = Rectangle(3, 4)
print(r.describe())   # Area=12, Perimeter=14

§11.5Protocol — 結構型別（Structural Typing）

ABC 是「nominal typing」—— 必須明確繼承。Protocol（PEP 544）是「structural typing」—— 只要「長得對」就符合，不需要繼承。這讓鴨子型別可以被靜態檢查器檢查。

python 
from typing import Protocol, runtime_checkable

class Drawable(Protocol):
    def draw(self) -> None: ...

# 任何類別只要有 draw() 方法就符合，不需繼承
class Circle:
    def draw(self):
        print("○")

class Square:
    def draw(self):
        print("□")

def render(shape: Drawable):
    shape.draw()    # 型別檢查器會確認有 draw 方法

render(Circle())   # ✅
render(Square())   # ✅（沒繼承 Drawable！）

# 若要執行階段用 isinstance 檢查 Protocol
@runtime_checkable
class Sized(Protocol):
    def __len__(self) -> int: ...

isinstance([1, 2], Sized)   # True（list 有 __len__）
📘 ABC vs Protocol 選擇
  • ABC：你控制類別設計，想強制繼承關係
  • Protocol：處理別人（或內建）類別，只關心「有沒有某方法」
  • 現代 Python 編碼傾向優先用 Protocol（更符合鴨子型別哲學）

§11.6__slots__ —— 記憶體優化

Python 類別預設用 __dict__ 存屬性，很彈性但占記憶體。用 __slots__ 限定屬性清單，可節省 30~50% 記憶體，並加快存取速度。

python 
class Point:
    __slots__ = ("x", "y")        # 預先宣告可能的屬性

    def __init__(self, x, y):
        self.x, self.y = x, y

p = Point(1, 2)
p.x, p.y   # OK
p.z = 3      # ❌ AttributeError！不在 __slots__

# 百萬個點：一般類別可能占 500 MB，有 __slots__ 只要 200 MB

# 也可結合 dataclass（3.10+）
from dataclasses import dataclass
@dataclass(slots=True)
class Point:
    x: float
    y: float
⚠️ 使用限制
  • 無法動態新增不在 __slots__ 的屬性
  • 多重繼承時會複雜（所有父類也要有 __slots__）
  • 不相容 weakref（除非加 '__weakref__'）
  • 通常用在大量實例的場景（如科學計算、遊戲實體）

§11.7描述器 Descriptor（進階）

描述器是 Python 屬性存取的「黑魔法」：@property、@staticmethod、@classmethod 全部都是描述器實作的。實作 __get__、__set__、__delete__ 即可。

python 
class Validated:
    """帶驗證的屬性描述器"""
    def __init__(self, min_val=None, max_val=None):
        self.min_val = min_val
        self.max_val = max_val

    def __set_name__(self, owner, name):
        """3.6+ 自動知道屬性名"""
        self.name = f"_{name}"

    def __get__(self, obj, owner=None):
        if obj is None:
            return self
        return getattr(obj, self.name)

    def __set__(self, obj, value):
        if self.min_val is not None and value < self.min_val:
            raise ValueError(f"{value} < {self.min_val}")
        if self.max_val is not None and value > self.max_val:
            raise ValueError(f"{value} > {self.max_val}")
        setattr(obj, self.name, value)

class Person:
    age = Validated(min_val=0, max_val=150)
    height = Validated(min_val=0)

    def __init__(self, age, height):
        self.age = age
        self.height = height

p = Person(30, 170)
p.age = -5         # ❌ ValueError
p.height = 180       # ✅
📘 描述器的真實應用
  • ORM：SQLAlchemy、Django ORM 的欄位類別都是描述器
  • 型別驗證框架：Pydantic、marshmallow
  • Python 內建：@property、方法綁定、@classmethod 都靠描述器
