CHAPTER 17 · PART 5

非同步程式設計

asyncio 不是讓程式變快的魔法，而是在「等待」時讓出控制權的協作式並發模型。學會 async/await 後，你可以在單執行緒裡同時處理上萬個網路連線。本章帶你建立正確的心智模型，掌握 TaskGroup、取消、超時等現代用法。

Python 3.14 async / await / TaskGroup 預估閱讀 25 分鐘

§0章節導論

asyncio 的適用場景：

  • ✅ 高並發網路 I/O（爬蟲、API client、websocket、gRPC）
  • ✅ Web 後端（FastAPI、aiohttp、Sanic）
  • ✅ 即時訊息系統、長連線應用
  • ❌ CPU 密集型任務（應用 threading/multiprocessing）
  • ❌ 純同步 I/O 腳本（用 threading 更簡單）

§17.1思維模型：協作式並發

📘 threading vs asyncio
  • threading：OS 負責切換，隨時可能中斷（preemptive）
  • asyncio：你寫的 await 才是切換點（cooperative）
  • asyncio 只有一個執行緒，但能同時「進行」上萬個協程
  • 核心概念：I/O 等待時讓出控制權，而不是阻塞
python 
# ❌ 同步：10 個請求要等 10×1s = 10s
import time
for _ in range(10):
    time.sleep(1)          # 阻塞整個程式

# ✅ asyncio：10 個請求共 ~1s
import asyncio
async def task():
    await asyncio.sleep(1)    # 讓出控制，等待時可切到其他任務

async def main():
    await asyncio.gather(*[task() for _ in range(10)])

asyncio.run(main())   # 總共 ~1s

§17.2協程 async / await

python 
# async def 定義協程函式
async def fetch(url):
    print(f"fetching {url}")
    await asyncio.sleep(1)      # 等待，但不阻塞 event loop
    return f"data from {url}"

# 呼叫協程函式 → 得到「協程物件」（還沒執行！）
coro = fetch("https://...")
print(coro)               # <coroutine object fetch at 0x...>

# await 才是真正執行 — 但 await 只能在 async 函式內用
async def main():
    result = await fetch("https://a.com")
    print(result)
⚠️ await 只能在 async 函式內

在一般函式裡寫 await 是 SyntaxError。要從同步世界進入 asyncio 世界，必須透過 asyncio.run()。

§17.3asyncio.run 啟動 event loop

python 
import asyncio

async def main():
    print("hello")
    await asyncio.sleep(1)
    print("world")

# 同步程式的入口，啟動 event loop、執行 main()、結束時關閉
asyncio.run(main())

# ⚠️ 一個程式通常只有一個 asyncio.run()，在最頂層
# 不要在 async 函式內呼叫 asyncio.run()

§17.4gather — 並行執行多個協程

python 
async def fetch(url, delay):
    await asyncio.sleep(delay)
    return f"{url} done"

async def main():
    # ❌ 序列執行：總共 1+2+3 = 6s
    a = await fetch("A", 1)
    b = await fetch("B", 2)
    c = await fetch("C", 3)

    # ✅ 並行執行：總共 3s（取最長的）
    a, b, c = await asyncio.gather(
        fetch("A", 1),
        fetch("B", 2),
        fetch("C", 3),
    )

    # 動態數量
    urls = ["A", "B", "C"]
    results = await asyncio.gather(*[fetch(u, 1) for u in urls])

    # 收集例外而非立刻拋出
    results = await asyncio.gather(
        fetch("A", 1), fetch("B", 2),
        return_exceptions=True
    )
    # 如果某個失敗，results 中會包含 Exception 實例而非正常值

§17.5TaskGroup —— 現代首選（3.11+）

TaskGroup 是 Python 3.11 引入的結構化並發（Structured Concurrency）工具，取代 gather 的最佳實踐：更安全的錯誤處理、更清楚的生命週期、保證子任務不會脫逃。

python 
async def main():
    async with asyncio.TaskGroup() as tg:
        t1 = tg.create_task(fetch("A"))
        t2 = tg.create_task(fetch("B"))
        t3 = tg.create_task(fetch("C"))
    # 離開 with 時，自動等所有 task 完成

    # 取得結果
    print(t1.result(), t2.result(), t3.result())

TaskGroup 的錯誤處理

python 
# TaskGroup 的關鍵特性：
# 1. 任一 task 拋例外，其他 task 會被自動 cancel
# 2. 所有例外會包成 ExceptionGroup 一起拋出
# 3. 離開 with 之前不會逃出 block（結構化）

async def main():
    try:
        async with asyncio.TaskGroup() as tg:
            tg.create_task(fetch("A"))
            tg.create_task(may_fail())      # 可能拋 ValueError
            tg.create_task(fetch("B"))
    except* ValueError as eg:
        print(f"值錯誤：{eg.exceptions}")
    except* ConnectionError as eg:
        print(f"連線錯誤：{eg.exceptions}")
💡 用 TaskGroup 取代 gather

新寫的程式優先用 TaskGroup。gather 仍然存在，但它的「錯誤時不取消其他 task」行為容易造成資源洩漏。TaskGroup + except*（Ch13）是結構化並發在 Python 的完整答案。

§17.6超時與取消

python 
# asyncio.timeout — 3.11+ 推薦
async def main():
    try:
        async with asyncio.timeout(5):           # 5 秒超時
            await slow_operation()
    except TimeoutError:
        print("超時了")

# 也可動態延長／縮短
async with asyncio.timeout(5) as cm:
    if need_more_time:
        cm.reschedule(asyncio.get_event_loop().time() + 10)
    await op()

# 舊 API：wait_for
try:
    result = await asyncio.wait_for(slow_op(), timeout=5)
except TimeoutError: ...

# 手動取消 task
task = asyncio.create_task(long_work())
await asyncio.sleep(1)
task.cancel()
try:
    await task
except asyncio.CancelledError:
    print("已取消")
⚠️ 不要吞掉 CancelledError
async def work():
    try:
        await some_io()
    except asyncio.CancelledError:
        await cleanup()
        raise         # ✅ 一定要 re-raise！否則取消機制失效
    except Exception as e:
        handle(e)

§17.7async context manager / iterator

python 
# async with — 非同步 context manager（__aenter__ / __aexit__）
import aiohttp

async def fetch(url):
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as resp:
            return await resp.text()

# 自訂 async context manager
from contextlib import asynccontextmanager

@asynccontextmanager
async def connection(host):
    conn = await connect(host)
    try:
        yield conn
    finally:
        await conn.close()

async with connection("db.example.com") as conn:
    await conn.query("...")

# async for — 非同步迭代器
async def stream_data():
    async for chunk in some_async_source:
        process(chunk)

# async generator — 用 yield
async def counter(n):
    for i in range(n):
        await asyncio.sleep(0.1)
        yield i

async def main():
    async for n in counter(5):
        print(n)

§17.8同步函式橋接

在 async 程式中呼叫慢的同步函式（如 requests、pandas、DB 驅動）會阻塞整個 event loop —— 必須丟到執行緒池。

python 
import asyncio
import requests

async def fetch_sync(url):
    loop = asyncio.get_running_loop()
    # ❌ 直接 requests.get 會阻塞 event loop
    # ✅ 丟到執行緒池執行
    return await loop.run_in_executor(
        None,                    # 預設 ThreadPoolExecutor
        requests.get, url       # 位置參數
    )

# Python 3.9+ 更簡潔的 asyncio.to_thread
async def fetch_sync(url):
    return await asyncio.to_thread(requests.get, url)

# 反向：從同步 code 執行 async
# 最外層用 asyncio.run（一個程式只能有一個）
asyncio.run(main())

§17.9常見模式

限制並發數（Semaphore）

python 
import asyncio

sem = asyncio.Semaphore(10)   # 最多 10 個同時進行

async def fetch(url):
    async with sem:             # 未釋放前阻塞
        async with session.get(url) as r:
            return await r.text()

# 即使開 1000 個 task，同時只會有 10 個在跑
async with asyncio.TaskGroup() as tg:
    for url in urls_1000:
        tg.create_task(fetch(url))

Queue 生產者／消費者

python 
async def producer(q):
    for i in range(10):
        await q.put(i)
    await q.put(None)

async def consumer(q):
    while True:
        item = await q.get()
        if item is None: break
        print(item)
        q.task_done()

async def main():
    q = asyncio.Queue(maxsize=5)
    async with asyncio.TaskGroup() as tg:
        tg.create_task(producer(q))
        tg.create_task(consumer(q))

重試（retry with backoff）

python 
async def fetch_with_retry(url, retries=3):
    for attempt in range(retries):
        try:
            return await fetch(url)
        except ConnectionError:
            if attempt == retries - 1:
                raise
            await asyncio.sleep(2 ** attempt)   # 指數退避 1s, 2s, 4s...

完整的爬蟲骨架

python 
import asyncio
import aiohttp

async def fetch(session, url, sem):
    async with sem:
        async with asyncio.timeout(10):
            async with session.get(url) as resp:
                resp.raise_for_status()
                return url, len(await resp.text())

async def main(urls):
    sem = asyncio.Semaphore(20)
    async with aiohttp.ClientSession() as session:
        async with asyncio.TaskGroup() as tg:
            tasks = [tg.create_task(fetch(session, url, sem))
                     for url in urls]
    for t in tasks:
        print(t.result())

asyncio.run(main(urls))

⚠️ 常見陷阱
  • 忘加 await → coroutine warning
  • 在 async 內用 time.sleep
  • 在 async 內用同步 requests
  • 多個 asyncio.run()
  • 吞掉 CancelledError
  • 當 CPU bound 用 asyncio
