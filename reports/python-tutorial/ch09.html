CHAPTER 09 · PART 3

類別與物件

「Python 中萬物皆物件」—— 即使 function、module 甚至 class 本身都是物件。理解 Python 的物件模型是成為進階開發者的分水嶺。本章從類別語法開始，走到繼承、MRO、多型，最後討論「組合勝於繼承」的現代設計哲學。

Python 3.14 OOP 核心 預估閱讀 25 分鐘

§0章節導論

Python 的 OOP 特色：

  • 語法極簡：沒有 public/private 強制關鍵字，只靠命名慣例
  • 所有屬性預設可訪問，但有 _、__ 慣例作為弱保護
  • 鴨子型別：不檢查類別繼承，只看「能做什麼」
  • 多重繼承支援（MRO 以 C3 linearization 解決鑽石問題）

§9.1類別基礎

python 
class Dog:
    """表示一隻狗"""

    def __init__(self, name, age):
        # 初始化方法 — 建立實例時執行
        self.name = name    # 實例屬性
        self.age = age

    def bark(self):
        print(f"{self.name}: 汪！")

# 建立實例
d = Dog("Rex", 3)       # 呼叫 __init__
d.bark()                 # Rex: 汪！
print(d.name, d.age)      # Rex 3
print(type(d))            # <class '__main__.Dog'>
# isinstance 檢查
isinstance(d, Dog)       # True
📘 關於 self

self 不是關鍵字，只是「第一個參數」的約定俗成名字（也可以叫 this、me，但別這麼做）。呼叫 d.bark() 時，Python 會自動把 d 作為 self 傳入，等同 Dog.bark(d)。

__init__ 與 __new__ 的差別

python 
class Foo:
    def __new__(cls, *args):
        print("1. 建立實例")
        return super().__new__(cls)

    def __init__(self, value):
        print("2. 初始化實例")
        self.value = value

f = Foo(42)
# 1. 建立實例
# 2. 初始化實例

99% 的情況只需實作 __init__。__new__ 只在需要「控制實例建立」時使用（如單例模式、不可變物件）。

§9.2實例屬性 vs 類別屬性

python 
class Dog:
    species = "Canis familiaris"   # ← 類別屬性（所有實例共享）
    count = 0

    def __init__(self, name):
        self.name = name                # ← 實例屬性（每個實例獨立）
        Dog.count += 1

rex = Dog("Rex")
max = Dog("Max")

print(rex.species)       # 'Canis familiaris'
print(Dog.species)       # 'Canis familiaris'
print(Dog.count)         # 2
⚠️ 類別屬性的可變物件陷阱
class Bucket:
    items = []          # ❌ 所有實例共享這個 list！

    def add(self, x):
        self.items.append(x)

a, b = Bucket(), Bucket()
a.add("x")
print(b.items)    # ['x'] ← 悲劇！

# ✅ 可變型別一律放 __init__ 中
class Bucket:
    def __init__(self):
        self.items = []

§9.3三種方法類型

類型裝飾器第一個參數用途
實例方法—self（實例）操作實例屬性
類別方法@classmethodcls（類別）替代建構子、類別層級操作
靜態方法@staticmethod無僅邏輯屬於此類別，但不需 self/cls
python 
class Person:
    population = 0

    def __init__(self, name, age):
        self.name = name
        self.age = age
        Person.population += 1

    # 實例方法 — 操作實例
    def greet(self):
        return f"Hi, I'm {self.name}"

    # 類別方法 — 常作為替代建構子
    @classmethod
    def from_dict(cls, data):
        return cls(data["name"], data["age"])

    @classmethod
    def count(cls):
        return cls.population

    # 靜態方法 — 純工具函式
    @staticmethod
    def is_adult(age):
        return age >= 18

# 使用
alice = Person("Alice", 30)
bob = Person.from_dict({"name": "Bob", "age": 25})

print(alice.greet())         # Hi, I'm Alice
print(Person.count())        # 2
print(Person.is_adult(20))    # True

§9.4property — 屬性封裝

Python 不使用 getter/setter 方法風格，而是用 @property 讓你寫起來像屬性存取，實際執行方法邏輯。

python 
class Circle:
    def __init__(self, radius):
        self._radius = radius   # 慣例：底線表示「內部」

    @property
    def radius(self):
        return self._radius

    @radius.setter
    def radius(self, value):
        if value < 0:
            raise ValueError("radius cannot be negative")
        self._radius = value

    @property
    def area(self):
        """計算屬性 — 每次存取都重算"""
        return 3.14159 * self._radius ** 2

c = Circle(5)
print(c.radius)       # 5（像屬性一樣存取）
print(c.area)         # 78.54（自動計算）
c.radius = 10          # 呼叫 setter，觸發驗證
c.radius = -1          # ❌ ValueError

只讀屬性與 cached_property

python 
from functools import cached_property

class Dataset:
    def __init__(self, rows):
        self.rows = rows

    @cached_property       # 第一次存取後會快取，之後不重算
    def stats(self):
        print("計算統計...")
        return {"sum": sum(self.rows), "len": len(self.rows)}

d = Dataset([1, 2, 3])
print(d.stats)        # 計算統計... {'sum': 6, 'len': 3}
print(d.stats)        # {'sum': 6, 'len': 3}（不重算）

§9.5「私有」成員：命名慣例

Python 沒有真正的 private，只有三種命名慣例：

前綴意義範例
無公開 APIname、calculate()
_（單底線）慣例「內部用」，仍可存取（弱提示）_internal
__（雙底線）觸發 name mangling（改名避免子類覆蓋）__private
__x__（雙底線兩側）Dunder，Python 內建約定保留__init__
python 
class Account:
    def __init__(self):
        self.owner = "Alice"      # 公開
        self._balance = 100          # 「內部用」(仍可從外部存取)
        self.__pin = "1234"          # name mangling → 變成 _Account__pin

a = Account()
print(a.owner)          # 'Alice'
print(a._balance)       # 100（仍可存取，但 IDE 會警告）
print(a.__pin)          # ❌ AttributeError
print(a._Account__pin)  # '1234'（其實還是拿得到，只是比較醜）
📘 Python 的哲學

"We are all consenting adults here." —— 不用強制封裝，而是透過慣例表達意圖。如果你呼叫別人的 _foo，就得自己承擔被改壞的風險。

§9.6繼承與 super()

python 
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return "..."

class Dog(Animal):              # ← 繼承自 Animal
    def __init__(self, name, breed):
        super().__init__(name)      # 呼叫父類 __init__
        self.breed = breed

    def speak(self):               # 覆寫方法
        return "汪！"

class Poodle(Dog):
    def speak(self):
        base = super().speak()         # 取父類的 "汪！"
        return f"小狗 {base}"

p = Poodle("Toby", "Toy")
print(p.name)     # 'Toby'（繼承自 Animal）
print(p.breed)    # 'Toy'（繼承自 Dog）
print(p.speak())  # '小狗 汪！'

# isinstance / issubclass
isinstance(p, Poodle)    # True
isinstance(p, Dog)       # True
isinstance(p, Animal)    # True
issubclass(Dog, Animal)  # True

§9.7多重繼承與 MRO

Python 支援多重繼承。當多個父類有同名方法時，用 MRO（Method Resolution Order）決定呼叫順序，演算法為 C3 linearization。

python 
class A:
    def greet(self): print("A")

class B(A):
    def greet(self): print("B")

class C(A):
    def greet(self): print("C")

class D(B, C):            # 鑽石繼承
    pass

d = D()
d.greet()              # 'B'（MRO: D → B → C → A → object）

print(D.mro())
# [<class 'D'>, <class 'B'>, <class 'C'>, <class 'A'>, <class 'object'>]

# 或
print(D.__mro__)

Mixin 模式

python 
# Mixin = 只提供方法、沒有狀態的「可混入」類別
class JSONMixin:
    def to_json(self):
        import json
        return json.dumps(self.__dict__)

class ReprMixin:
    def __repr__(self):
        fields = ", ".join(f"{k}={v!r}" for k, v in self.__dict__.items())
        return f"{type(self).__name__}({fields})"

class User(JSONMixin, ReprMixin):
    def __init__(self, name, age):
        self.name = name
        self.age = age

u = User("Alice", 30)
print(u)             # User(name='Alice', age=30)
print(u.to_json())   # {"name": "Alice", "age": 30}

§9.8多型與鴨子型別

Python 不要求繼承關係來達成多型 —— 只要「長得像、叫得像」就可以互換使用。這就是 Duck Typing。

python 
class Dog:
    def speak(self): return "汪"

class Cat:
    def speak(self): return "喵"

class Robot:
    def speak(self): return "嗶嗶"

# 不需要共同父類，只要有 speak() 方法都可以
for thing in [Dog(), Cat(), Robot()]:
    print(thing.speak())

# "If it walks like a duck and quacks like a duck, it's a duck."
💡 EAFP vs LBYL

Python 文化傾向 EAFP（Easier to Ask Forgiveness than Permission）：先嘗試使用，出錯再處理。相對於 LBYL（Look Before You Leap）：先檢查再使用。鴨子型別讓 EAFP 風格變得自然。

# ❌ LBYL 風格
if hasattr(obj, "speak"):
    obj.speak()

# ✅ EAFP 風格（更 Pythonic）
try:
    obj.speak()
except AttributeError:
    ...

§9.9組合優於繼承

「繼承」表示 is-a 關係（Poodle is a Dog）。「組合」表示 has-a 關係（Car has an Engine）。多數情況組合更靈活、耦合更低。

python 
# ❌ 過度繼承
class Car:
    def start_engine(self): ...
    def stop_engine(self): ...
    def fill_fuel(self): ...

class ElectricCar(Car):         # 繼承了不需要的 fill_fuel！
    pass

# ✅ 組合
class GasEngine:
    def start(self): print("燃油引擎啟動")

class ElectricEngine:
    def start(self): print("電機啟動")

class Car:
    def __init__(self, engine):
        self.engine = engine     # ← 組合

    def start(self):
        self.engine.start()

car = Car(ElectricEngine())   # 換引擎就像換零件
car.start()                    # 電機啟動
📘 何時用繼承、何時用組合？
  • 繼承：子類真的「is-a」父類，且共享大量行為（如 DatabaseConnection → PostgresConnection）
  • 組合：邏輯上是「has-a」或「uses-a」關係
  • 共享小塊功能 → Mixin
  • 看到繼承超過 3 層 → 通常設計出問題了
