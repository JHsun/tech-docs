結構化模式匹配

Python 3.10 引入的 match/case 不是 switch，它是結構化模式匹配（structural pattern matching）—— 一種以「資料形狀」為判斷依據的控制流。借鑑自 Haskell、Rust、Scala，在處理 AST、狀態機、API 回應時，讓程式碼從「一大坨 if-elif」變成宣告式的資料形狀描述。

Python 3.10+ · PEP 634 match / case 預估閱讀 22 分鐘

§0章節導論

一般 switch 只能比較值。Python 的 match 可以：

  • 匹配字面值（如 200, "GET", None）
  • 匹配型別與結構（list of 3 elements, dict with specific keys）
  • 綁定變數名字到匹配到的部分（自動解包）
  • 用 OR 模式合併多個 case
  • 用 guard 加額外條件
  • 匹配類別與具名屬性

最基本的例子

python 
def http_error(status):
    match status:
        case 400:
            return "Bad Request"
        case 401 | 403 | 407:        # OR 模式
            return "Unauthorized"
        case 404:
            return "Not Found"
        case 418:
            return "I'm a teapot"
        case _:                    # 萬用（wildcard）
            return "Something's wrong"
📘 match 與 case 是「軟關鍵字」

match 和 case 在 3.10+ 仍不是保留字，只在 match 陳述句的位置才具特殊意義。所以 match = 5 仍然合法（但不建議這樣取變數名，會混淆讀者）。

§6.1字面模式（Literal Pattern）

最簡單的模式：與字面值比較。支援數字、字串、True、False、None。

python 
match command:
    case "help":
        show_help()
    case "quit":
        exit()
    case 0:
        print("零")
    case True:
        print("真")
    case None:
        print("無")

具名常數：必須用點號

python · ⚠️ 關鍵陷阱 
from enum import Enum

class Color(Enum):
    RED = 1
    GREEN = 2

color = Color.GREEN

# ✅ 正確：帶點號的名稱被視為「值模式」
match color:
    case Color.RED:
        print("紅")
    case Color.GREEN:
        print("綠")

# ❌ 錯誤：裸名稱會被當作「捕獲模式」
RED = Color.RED
match color:
    case RED:           # 💀 RED 會被重新綁定為 color 的值
        print("紅")     # 永遠印「紅」— 因為任何值都會被「捕獲」
⚠️ 模式匹配中的重大陷阱

在 case 中：裸小寫名稱一律是「捕獲變數」（任何值都符合並綁定），而非比較常數。要比較常數請用：SomeEnum.VALUE、module.CONST，或任何包含 . 的路徑。

§6.2捕獲模式與萬用模式

python 
match x:
    case 0:
        print("zero")
    case n:                      # 捕獲：任何值都符合，並綁定到 n
        print(f"got {n}")

# _ 是萬用：匹配但不綁定（慣例用作 default）
match x:
    case 0:
        ...
    case _:                      # 類似 switch 的 default
        print("default")

§6.3OR 模式

用 | 合併多個模式，任一符合即可。

python 
match command.split():
    case ["go", "north"] | ["north"]:
        move("N")
    case ["get", item] | ["pick", "up", item] | ["pick", item, "up"]:
        pick_up(item)

# 匹配數字範圍（用 OR）
match status:
    case 200 | 201 | 204:
        return "OK"
    case 301 | 302:
        return "Redirect"
    case 400 | 401 | 403 | 404:
        return "Client Error"
    case 500 | 502 | 503:
        return "Server Error"

§6.4序列模式（Sequence Pattern）

匹配 list、tuple、或任何序列。不匹配字串與 bytes（特意設計如此）。

python 
match point:
    case []:
        print("empty")
    case [x]:
        print(f"one: {x}")
    case [x, y]:
        print(f"two: {x}, {y}")
    case [x, y, z]:
        print(f"three: {x}, {y}, {z}")

# 星號捕獲剩餘（可 0 個或多個）
match items:
    case [first, *rest]:
        print(f"head={first}, rest={rest}")
    case [*init, last]:
        print(f"init={init}, last={last}")
    case [first, *mid, last]:
        print(f"first={first}, mid={mid}, last={last}")

# * 忽略不要的部分
match record:
    case ["user", name, *_]:      # 只關心第二個，其餘丟掉
        print(name)

# 特定字面 + 捕獲
match action:
    case ["INSERT", table, values]:
        do_insert(table, values)
    case ["DELETE", table, "WHERE", cond]:
        do_delete(table, cond)
    case ["SELECT", *cols, "FROM", table]:
        do_select(cols, table)

巢狀序列

python 
# 2D 座標 list，逐個解構
match points:
    case [(0, 0)]:
        print("只有原點")
    case [(x, y), (0, 0)]:
        print(f"第一個是 ({x},{y})，第二個是原點")
    case [*points]:
        print(f"{len(points)} 個點")

§6.5映射模式（Mapping Pattern）

匹配 dict。關鍵：只需要「包含」指定的 key，其他 key 被忽略。

python 
match response:
    case {"status": 200, "data": data}:
        process(data)
    case {"status": 404}:
        print("Not found")
    case {"status": status, "error": msg}:
        print(f"Error {status}: {msg}")

# 用 ** 捕獲剩餘 key/value（3.10+）
match config:
    case {"name": name, **rest}:
        print(f"name={name}, others={rest}")

# 注意：dict 模式不要求完全相等！
data = {"a": 1, "b": 2, "c": 3}
match data:
    case {"a": 1}:       # ✅ 符合（b, c 被忽略）
        pass

典型應用：解析 JSON API

python 
def handle_message(msg: dict):
    match msg:
        case {"type": "text", "content": content}:
            show_text(content)

        case {"type": "image", "url": url, "width": w, "height": h}:
            show_image(url, w, h)

        case {"type": "video", "url": url, **opts}:
            play_video(url, opts)

        case {"type": unknown}:
            raise ValueError(f"Unknown type: {unknown}")

        case _:
            raise ValueError("Invalid message format")

§6.6類別模式（Class Pattern）

匹配指定型別的實例，並可解構其屬性。這是模式匹配最強大的部分。

python 
from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float

@dataclass
class Circle:
    center: Point
    radius: float

shape = Circle(Point(0, 0), 5)

match shape:
    # 型別匹配 + 關鍵字解構
    case Point(x=0, y=0):
        print("原點")
    case Point(x=x, y=0):
        print(f"在 x 軸上：x={x}")
    case Circle(center=Point(x=0, y=0), radius=r):
        print(f"以原點為圓心，半徑 {r}")
    case Circle(radius=r) if r > 10:
        print(f"大圓：r={r}")

位置參數匹配（__match_args__）

python 
# dataclass 自動產生 __match_args__，可用位置參數匹配

match point:
    case Point(0, 0):              # 等同 Point(x=0, y=0)
        print("原點")
    case Point(x, y):              # 等同 Point(x=x, y=y)
        print(f"({x}, {y})")

# 自訂類別需手動定義 __match_args__
class Vector:
    __match_args__ = ("x", "y", "z")
    def __init__(self, x, y, z):
        self.x, self.y, self.z = x, y, z

match v:
    case Vector(0, 0, 0):
        print("零向量")
    case Vector(x, y, z):
        print(f"({x}, {y}, {z})")

內建型別快速檢查

python 
def describe(value):
    match value:
        case None:
            return "None"
        case bool(b):
            return f"bool: {b}"
        case int(n):
            return f"int: {n}"
        case str(s) if len(s) > 10:
            return f"long str ({len(s)})"
        case str(s):
            return f"str: {s!r}"
        case list() as lst:
            return f"list of {len(lst)}"
        case dict() as d if not d:
            return "empty dict"
        case _:
            return f"其他: {type(value).__name__}"

§6.7守衛條件（Guard）

在 case 後加 if <condition>，必須同時滿足模式與條件才匹配。

python 
def categorize(n):
    match n:
        case int(x) if x < 0:
            return "negative"
        case 0:
            return "zero"
        case int(x) if x % 2 == 0:
            return "positive even"
        case int():
            return "positive odd"
        case float(x) if x.is_integer():
            return "integer-like float"
        case float():
            return "float"
        case _:
            return "not a number"

# 對序列套條件
match point:
    case [x, y] if x == y:
        print("在 y=x 直線上")
    case [x, y]:
        print(f"任意點 ({x},{y})")

§6.8AS 綁定與巢狀模式

用 as 把匹配到的子模式額外取一個名字，方便後續使用。

python 
match shape:
    case Point(x=0, y=0) as origin:
        print(f"原點：{origin}")    # origin 綁定整個 Point 實例
    case (Point() | Circle()) as shape_obj:
        process(shape_obj)

# 在序列模式中用 as
match data:
    case [int() as first, *rest]:
        print(f"first int: {first}")

綜合範例：簡易直譯器

python 
from dataclasses import dataclass

@dataclass
class Num: value: float
@dataclass
class Add: l: "Expr"; r: "Expr"
@dataclass
class Mul: l: "Expr"; r: "Expr"
@dataclass
class Neg: e: "Expr"

def eval_expr(e):
    match e:
        case Num(v):           return v
        case Add(l, r):        return eval_expr(l) + eval_expr(r)
        case Mul(l, r):        return eval_expr(l) * eval_expr(r)
        case Neg(e):           return -eval_expr(e)
        case _:               raise ValueError("unknown expr")

# (3 + 4) * -2
expr = Mul(Add(Num(3), Num(4)), Neg(Num(2)))
print(eval_expr(expr))   # -14

§6.9常見陷阱與最佳實踐

陷阱 1：裸名稱被當成捕獲

python · ❌ 
STATUS_OK = 200

match code:
    case STATUS_OK:       # ❌ 任何值都符合！STATUS_OK 被重新綁定
        print("OK")

# ✅ 解法：用 module.CONST 或 Enum
class Status:
    OK = 200

match code:
    case Status.OK:       # ✅ 有點號 → 值模式
        print("OK")

陷阱 2：case 順序很重要

match 從上往下匹配，第一個符合就停。specific 的 case 必須在 general 之前。

python 
# ❌ 錯誤順序：_ 吃掉所有 case
match x:
    case _:
        print("anything")
    case 0:              # 永遠不會執行
        print("zero")

# ✅ 具體在前，通用在後
match x:
    case 0:
        print("zero")
    case _:
        print("anything")

陷阱 3：字串不是 sequence pattern

python 
# 字串與 bytes 特意設計為不匹配 sequence pattern
match "abc":
    case [a, b, c]:     # ❌ 不會匹配
        pass
    case "abc":         # ✅ 用字面
        pass

# 想拆字元先轉 list
match list("abc"):
    case [a, b, c]:
        print(a, b, c)

最佳實踐

  • 什麼時候用 match？當你在處理「資料的形狀」（不同的 JSON 結構、AST 節點、狀態機狀態）
  • 什麼時候用 if-elif？當條件是單純的布林判斷（x > 10 and y < 5）
  • 常數要比較 → 一律放到 Enum 或 Class 中，用點號存取
  • 有 _ 萬用 case 時要放最後，且最好真的用到它（否則遺漏匹配會悄悄通過）
  • 複雜模式善用 dataclass + 型別模式，程式碼會非常乾淨
