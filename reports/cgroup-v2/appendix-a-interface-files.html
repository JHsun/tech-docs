Appendix A · Reference

介面檔案完整速查表

cgroup v2 所有 controller 介面檔案的速查索引——按 controller 分類、含讀寫屬性、簡短說明、範例與相關章節跳轉。 適合在 troubleshoot 或寫應用時快速查找正確介面名稱與用法。

本附錄整理 cgroup v2 所有使用者可見的介面檔案——按 controller 分類，每個檔案包含：

  • 檔案名：相對於 cgroup 目錄的路徑
  • 讀寫屬性：R=唯讀、RW=可讀寫、W=唯寫（罕見）
  • 說明與範例值：常見格式與含意
  • 相關章節：跳轉到本書講解該介面的章節

使用注意：所有路徑都相對於 cgroup 目錄。例如 root cgroup 的 cpu.max 完整路徑是 /sys/fs/cgroup/cpu.max；某 Pod cgroup 則是 /sys/fs/cgroup/kubepods.slice/.../cpu.max。介面在每個 cgroup 都存在（如果該 controller 已 enable）。

讀寫圖例： R 唯讀（kernel 寫入、user 讀） · RW 可讀寫（user 設限制） · W 唯寫（特殊命令）

A.1 · Core

cgroup.* 核心介面

所有 cgroup 都有的核心檔案——管理 process / thread 成員、controller 啟用、freezer、events 等。 不屬於任何特定 controller，cgroup core 提供。

檔案 R/W 說明 參考
cgroup.procs RW cgroup 內所有 process 的 PID 列表（讀）；寫入 PID 將 process 移入此 cgroup。
$ cat cgroup.procs
1234
5678		 Ch.4
cgroup.threads RW thread-mode cgroup 的 thread (TID) 列表。只在 threaded type cgroup 有效。 Ch.6
cgroup.controllers R 當前 cgroup 可用的 controllers（從父繼承）。
$ cat cgroup.controllers
cpuset cpu io memory hugetlb pids rdma misc		 Ch.4
cgroup.subtree_control RW 傳播給子 cgroup 的 controllers。寫入 +name 啟用、-name 禁用。
$ echo "+memory +cpu" > cgroup.subtree_control		 Ch.4
cgroup.type RW cgroup 類型：domain（預設）/ threaded / domain threaded / domain invalid。 Ch.6
cgroup.events R 事件計數，可用 inotify 監聽：
populated（0/1）：是否有 process
frozen（0/1）：是否被 freeze
populated 1
frozen 0		 Ch.12
cgroup.freeze RW 0=正常運行；1=凍結整個 cgroup（所有 process SIGSTOP）。取代 v1 freezer.state。 Ch.11
cgroup.kill W 寫 1 殺死 cgroup 內所有 process（kernel 5.14+）。比 freeze + kill -9 簡潔。
$ echo 1 > cgroup.kill		 Ch.11
cgroup.max.depth RW 子樹最大深度（防 fork bomb 創 cgroup）。預設 max。
$ echo 5 > cgroup.max.depth		 Ch.11
cgroup.max.descendants RW 子樹最大子 cgroup 數量。預設 max。 Ch.11
cgroup.stat R cgroup 統計：
nr_descendants：子 cgroup 數
nr_dying_descendants：已釋放但仍有 reference 的		 Ch.12
cpu.pressure RW PSI——CPU 壓力。寫入可調 trigger。詳 A.9。 Ch.12
memory.pressure RW PSI——記憶體壓力。詳 A.9。 Ch.12
io.pressure RW PSI——IO 壓力。詳 A.9。 Ch.12
A.2 · CPU controller

cpu.* 介面

CPU controller 提供權重（weight） 與頻寬限制（bandwidth） 兩種模式。詳細語義參見 Ch.7。

檔案 R/W 說明 參考
cpu.weight RW CPU 權重（公平分享）。範圍 1–10000，預設 100。對應 systemd CPUWeight=。
$ echo 200 > cpu.weight # 雙倍權重		 Ch.7
cpu.weight.nice RW 用 nice 值（-20 到 19）表達 weight。等同 cpu.weight 的另一種寫法。 Ch.7
cpu.max RW CPU 頻寬硬限：$MAX $PERIOD（μs）。max 表不限。
$ cat cpu.max
200000 100000 # 2 CPU
$ echo "max 100000" > cpu.max # 解除限制		 Ch.7
cpu.max.burst RW 允許短期 burst 超過 cpu.max 的累積額度（μs）。kernel 5.14+。 Ch.7
cpu.idle RW 1=SCHED_IDLE 模式（極低優先級，類似 nice 19 但更激進）。kernel 5.15+。 Ch.7
cpu.stat R CPU 使用統計：
usage_usec：總 CPU 使用時間
user_usec / system_usec
nr_periods：頻寬週期數
nr_throttled：被 throttle 次數
throttled_usec：累計 throttle 時間
nr_bursts / burst_usec：burst 統計		 Ch.7
cpu.stat.local R 當前 cgroup 自身（不含子）統計。kernel 6.6+。 Ch.7
cpu.uclamp.min RW util-clamp 最小值（0–100%）。EAS scheduler 用，提示「至少這麼快」。手機/嵌入式用。 Ch.11
cpu.uclamp.max RW util-clamp 最大值。提示「不需要更快」省電。 Ch.11
cpu.pressure RW PSI CPU pressure（some 單行）。詳 A.9。 Ch.12
A.3 · cpuset

cpuset.* 介面

cpuset controller 提供 CPU/NUMA node 綁定（pinning）。K8s CPU Manager static policy、HPC 低延遲應用的核心。

檔案 R/W 說明 參考
cpuset.cpus RW 允許執行的 CPU set。空字串繼承父。
$ echo "0-3,8" > cpuset.cpus		 Ch.9
cpuset.cpus.effective R 實際生效的 CPU set（考慮 hot-unplug、parent 限制等）。 Ch.9
cpuset.mems RW 允許分配記憶體的 NUMA node。
$ echo "0,1" > cpuset.mems # NUMA 0,1		 Ch.9
cpuset.mems.effective R 實際生效的 NUMA node set。 Ch.9
cpuset.cpus.partition RW partition 模式：member（預設） / root（獨佔，不被其他 cgroup 使用）/ isolated（root + 隔離 IRQ）/ root invalid / isolated invalid。 Ch.9
cpuset.cpus.exclusive RW 標記哪些 CPU 在子樹中為「獨佔」。kernel 6.7+。 Ch.9
cpuset.cpus.exclusive.effective R 實際獨佔的 CPU set。 Ch.9
A.4 · Memory controller

memory.* 介面

Memory controller 提供四層保護：min（保證）/ low（軟保護）/ high（軟限+throttle）/ max（硬限）。 v2 大幅優化 v1 的 OOM 行為——詳 Ch.8。

檔案 R/W 說明 參考
記憶體限制
memory.min RW 絕對保證 不被 reclaim 的記憶體量。即使其他 cgroup 飢渴也不釋放。對應 systemd MemoryMin=。 Ch.8
memory.low RW 軟保護，best-effort 不被 reclaim。其他 cgroup 嚴重飢渴時仍可能釋放。對應 MemoryLow=。 Ch.8
memory.high RW 軟限制 + throttle。超過時 kernel 強制 reclaim、必要時 throttle process（不 OOM）。對應 MemoryHigh=。Memory QoS 核心。 Ch.8
memory.max RW 硬限。超過 → OOM。對應 MemoryMax=，取代 v1 的 memory.limit_in_bytes。
$ echo 1G > memory.max		 Ch.8
當前用量與統計
memory.current R 當前記憶體用量（含 page cache、anon、kmem 等）。 Ch.8
memory.peak RW 歷史峰值記憶體用量。寫入任何值會 reset 計數（kernel 6.5+）。 Ch.8
memory.stat R 詳細記憶體統計。重要欄位：
anon：匿名記憶體
file：page cache
kernel_stack、slab：kernel 結構
workingset_refault：working set thrashing 訊號		 Ch.8
memory.events R 事件計數，inotify 可監聽：
low：超過 memory.low 次數
high：超過 memory.high 次數（throttle）
max：嘗試超過 memory.max 次數
oom：OOM 觸發次數
oom_kill：OOM kill 次數
oom_group_kill：整 group kill 次數		 Ch.12
memory.events.local R 當前 cgroup 自身（不含子）的 events。 Ch.12
Swap
memory.swap.current R 當前 swap 使用量。v2 將 swap 與 memory 分開計算（v1 是合計）。 Ch.8
memory.swap.max RW swap 使用上限（獨立於 memory.max）。對應 MemorySwapMax=。 Ch.8
memory.swap.high RW swap 軟限。kernel 5.19+。 Ch.8
memory.swap.events R swap 事件：high、max、fail。 Ch.8
memory.zswap.current R 當前 zswap（壓縮記憶體 swap）用量。kernel 5.19+。 Ch.8
memory.zswap.max RW zswap 使用上限。 Ch.8
OOM 與雜項
memory.oom.group RW 1=OOM 時殺整 group（一次清空），0=只殺最大 process（預設）。 Ch.8
memory.reclaim W 主動觸發 reclaim：echo "1G" > memory.reclaim 嘗試釋放 1G。kernel 5.19+。proactive reclaim 工具用。 Ch.8
memory.numa_stat R per-NUMA-node 記憶體統計。NUMA-aware tuning 用。 Ch.9
memory.pressure RW PSI memory pressure。詳 A.9。 Ch.12
A.5 · IO controller

io.* 介面

IO controller 提供 weight（公平）、max（硬限）、cost-based（blk-iocost）三種模式。per-device 設定，需 device major:minor。詳 Ch.10。

檔案 R/W 說明 參考
io.weight RW IO 權重（1–10000，預設 100）。需要 blk-iocost enabled 才生效。對應 systemd IOWeight=。
$ echo "default 200" > io.weight # 全部 device
$ echo "8:0 500" > io.weight # 特定 device		 Ch.10
io.max RW per-device 頻寬硬限。格式：major:minor [rbps=N] [wbps=N] [riops=N] [wiops=N]
$ echo "8:0 rbps=10485760 wbps=10485760" > io.max		 Ch.10
io.latency RW per-device latency 目標（μs）。低於目標時不限，超過時 throttle 較低優先 cgroup。
$ echo "8:0 target=10000" > io.latency		 Ch.10
io.stat R per-device IO 統計：rbytes、wbytes、rios、wios、dbytes（discard）、dios。 Ch.10
io.cost.qos RW blk-iocost 的 QoS 設定（per-device）。複雜參數含 ctrl mode、min/max rpct 等。 Ch.10
io.cost.model RW blk-iocost 的 device model（rbps/rseqiops/rrandiops/wbps...）。需要先 benchmark 設定。 Ch.10
io.prio.class RW IO priority class：no-change / none / rt / be / idle / promote-to-rt（kernel 6.0+）。 Ch.10
io.bfq.weight RW BFQ scheduler 專用 weight（與 io.weight 互斥）。需 BFQ scheduler 啟用。 Ch.10
io.pressure RW PSI IO pressure。詳 A.9。 Ch.12

pids.* 介面

PID controller 防止 fork bomb。計入 thread——多 thread 應用要記得拉高。

檔案 R/W 說明 參考
pids.max RW process + thread 總數上限。max = 不限。對應 systemd TasksMax=。
$ echo 1000 > pids.max		 Ch.9
pids.current R 當前 process+thread 數。 Ch.9
pids.peak R 歷史峰值 PID 數。kernel 6.1+。 Ch.9
pids.events R 事件：max（達上限被拒次數）。 Ch.9
pids.events.local R 當前 cgroup 自身（不含子）events。 Ch.9
A.7 · HugeTLB controller

hugetlb.* 介面

限制 HugeTLB（透明大頁的「顯式預留」版本）使用。每個 page size（2M、1G）獨立。DB / HPC 場景常用。

檔案 R/W 說明 參考
hugetlb.<size>.max RW 該 page size 的 HugeTLB 使用上限。
$ echo 1G > hugetlb.2MB.max
$ echo 4 > hugetlb.1GB.max # 4 個 1G page		 Ch.11
hugetlb.<size>.current R 當前該 page size 的使用量。 Ch.11
hugetlb.<size>.events R 事件：max（達上限次數）。 Ch.11
hugetlb.<size>.events.local R 本 cgroup 自身 events。 Ch.11
hugetlb.<size>.numa_stat R per-NUMA-node HugeTLB 統計。kernel 6.1+。 Ch.9
A.8 · Misc & RDMA controllers

misc.* 與 rdma.*

Misc controller（kernel 5.13+）為 keyed resource 提供通用限制——當前主要用於 SEV/SEV-ES（AMD memory encryption）。 RDMA controller 限制 RDMA 資源（HCA handle、object）。

檔案 R/W 說明 參考
Misc
misc.max RW per-resource 上限（如 SEV ASID slot）。
$ echo "sev 100" > misc.max		 Ch.11
misc.current R 當前各 resource 用量。 Ch.11
misc.capacity R 系統總 capacity（root 才有）。 Ch.11
misc.events R 事件：max（達上限次數）。 Ch.11
RDMA
rdma.max RW per-device RDMA 資源上限：hca_handle、hca_object。
$ echo "mlx5_0 hca_handle=10 hca_object=1000" > rdma.max		 Ch.11
rdma.current R 當前 per-device RDMA 用量。 Ch.11
A.9 · PSI

PSI 介面細節

Pressure Stall Information（PSI）是 cgroup v2 最重要的觀測介面之一。cpu.pressure / memory.pressure / io.pressure 各自有相同格式。詳 Ch.12。

概念 說明
some 至少一個 task stall 等資源的時間百分比。avg10 / avg60 / avg300（10s/60s/300s 平均）。
some avg10=2.34 avg60=1.50 avg300=0.85 total=12345678
full 所有 task 都 stall 等資源的時間百分比。memory.pressure 與 io.pressure 有，cpu.pressure 沒有（CPU 永遠至少有一個 task 跑）。
觸發條件 寫入 cpu.pressure / memory.pressure / io.pressure 可設 trigger，poll(2) 等待。
格式：type window threshold
$ echo "some 1000000 100000" > memory.pressure # 1s 窗口內 stall 100ms 觸發
監控 alert 閾值 實務常用：
• cpu.pressure some avg10 > 5%：CPU 擁擠
• memory.pressure full avg10 > 5%：嚴重 memory pressure（Netdata 文件指 imminent OOM 訊號）
• io.pressure full avg10 > 10%：IO 嚴重 stall
