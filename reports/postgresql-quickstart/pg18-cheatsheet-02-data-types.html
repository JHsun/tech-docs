🔢 數值型別 (Numeric)
型別儲存範圍別名
smallint2 B-32768 ~ 32767int2
integer4 B-2³¹ ~ 2³¹-1int, int4
bigint8 B-2⁶³ ~ 2⁶³-1int8
numeric(p,s)可變131072位整數+16383位小數decimal
real4 B≈6位精度float4
double precision8 B≈15位精度float8
smallserial2 B1~32767自增int2
serial4 B1~2³¹-1自增int4
bigserial8 B1~2⁶³-1自增int8
numeric 特殊值
'NaN'::numeric  'Infinity'::numeric  '-Infinity'::numeric
-- NaN 排序大於所有數字；NaN = NaN → true
建議：金額→numeric，一般整數→integer，自增主鍵→GENERATED AS IDENTITY 取代 serial。
💰 貨幣型別
型別儲存說明
money8 B±92兆，受 lc_monetary locale 影響格式
注意：跨 locale 遷移時 money 可能出錯，建議用 numeric 儲存金額。
📝 字元型別
型別說明最大
char(n)固定長度，空白填充n 字元
varchar(n)可變長度，有上限n 字元
text可變長度，無上限~1 GB
效能提示：PostgreSQL 中 text ≈ varchar 效能相同。char(n) 因空白填充反而更慢。建議直接用 text，需長度限制用 CHECK(length(col)<=n)。
-- COLLATION 控制排序
SELECT * FROM t ORDER BY name COLLATE "zh-TW-x-icu";
-- "C" collation 用於 LIKE 索引加速
CREATE INDEX ON t (name COLLATE "C");

bytea可變長度二進位字串，最大 ~1 GB
SET bytea_output = 'hex';  -- 預設輸出格式
SELECT '\xDEADBEEF'::bytea;

📅 日期 / 時間型別
型別儲存範圍精度
date4 B4713 BC ~ 5874897 AD1 日
time [(p)]8 B00:00:00~24:00:001 μs
timetz [(p)]12 B同上+時區1 μs
timestamp [(p)]8 B4713 BC~294276 AD1 μs
timestamptz [(p)]8 B同上+自動時區轉換1 μs
interval [(f)] [(p)]16 B±178000000 年1 μs
輸入格式與特殊值
'2024-03-15'::date
'2024-03-15 14:30:00+08'::timestamptz
'1 year 2 months 3 days'::interval
'P1Y2M3DT4H'::interval  -- ISO 8601
-- 特殊: 'epoch' 'infinity' '-infinity' 'now'
--       'today' 'yesterday' 'tomorrow'
永遠用 timestamptz：它以 UTC 儲存、按 session 時區顯示，避免時區混亂。timetz SQL 標準不建議使用。

📋 列舉型別 (Enum)
CREATE TYPE status AS ENUM ('draft','published','archived');
-- 排序按定義順序（非字母序），4 bytes 儲存

ALTER TYPE status ADD VALUE 'review' BEFORE 'published';
ALTER TYPE status RENAME VALUE 'draft' TO 'initial';
SELECT enum_range(NULL::status); -- 列所有值
限制：無法刪除已有值；不可在同一交易中 ADD VALUE 後立即使用新值。
🆔 UUID
-- 128 bits, 16 bytes 儲存
-- 格式: a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11

SELECT gen_random_uuid();  -- 原生 UUIDv4 (PG14+)
SELECT uuidv7();           -- 時間排序 UUIDv7 (PG18)

-- 用作主鍵
id uuid DEFAULT uuidv7() PRIMARY KEY
UUIDv7 (PG18新增)：基於時間排序的 UUID，B-tree 寫入更友好，比 v4 更適合做分散式主鍵。
📄 JSON / JSONB
型別儲存特點
json原始文字保留空白/順序/重複鍵，每次存取重解析
jsonb二進位去空白/排序鍵/去重複，支援索引，效能佳
jsonb 運算子
->取元素 (json)->>取元素 (text)
#>路徑取值#>>路徑取text
@>包含<@被包含
?鍵存在?| ?&任一/全部鍵
||合併-刪鍵
#-路徑刪除@? @@JSONPath
常用函式

jsonb 索引策略
-- 通用 GIN（支援 @>, ?, ?|, ?&）
CREATE INDEX ON t USING gin(data);
-- jsonb_path_ops（僅 @>，更小更快）
CREATE INDEX ON t USING gin(data jsonb_path_ops);
-- 特定欄位 btree
CREATE INDEX ON t ((data->>'email'));
📚 陣列型別 (Array)

-- 輸入
'{1,2,3}'::int[]    ARRAY[1,2,3]

-- 存取 (1-based!)
arr[1]  arr[2:4]  arr[1][2]
運算子與函式
= && @> <@ ||相等/重疊/包含/串接
ANY(arr) ALL(arr)任一/全部匹配
array_length(a,dim) cardinality(a) unnest(a)
array_append(a,v)   array_remove(a,v)
array_position(a,v) array_agg(expr)
array_to_string(a,',') string_to_array(s,',')
GIN 索引：CREATE INDEX ON t USING gin(tags); 加速 @> <@ &&。

🔍 全文搜尋型別
tsvector正規化詞位列表（含位置/權重）
tsquery搜尋條件（AND/OR/NOT/FOLLOWED BY）
to_tsvector('english','Fat Cats sat')
→ 'cat':2 'fat':1 'sat':3

to_tsquery('english','fat & cat')
plainto_tsquery('english','fat cat')
phraseto_tsquery('english','fat cat')
websearch_to_tsquery('english','"fat cat" -dog')

-- 匹配: tsvector @@ tsquery
-- 排序: ts_rank(vector, query)

-- 效能：存儲欄+GIN索引
tsv tsvector GENERATED ALWAYS AS
  (to_tsvector('english',body)) STORED;
CREATE INDEX ON docs USING gin(tsv);
