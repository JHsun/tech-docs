01

快速上手 Quick Start

psql（PostgreSQL interactive terminal）是一個終端機前端， 讓你可以：互動式輸入 SQL、從檔案批次執行、以 \d 之類 的 meta-command（反斜線命令）快速查看資料庫物件，並將結果匯出為多種格式。

三分鐘最小工作流程

# 1. 連線（預設使用作業系統使用者名稱當 DB user）
psql -h localhost -U postgres -d mydb

# 2. 列出所有資料庫
mydb=> \l

# 3. 列出目前 schema 下的 tables
mydb=> \dt

# 4. 看某張表的結構（含索引、constraint）
mydb=> \d users

# 5. 執行一個 SQL
mydb=> SELECT id, name FROM users LIMIT 5;

# 6. 離開
mydb=> \q
Tip｜關鍵心智模型 以 ; 結尾的是 SQL（送到 server 執行）； 以 \ 開頭的是 meta-command（由 psql 本身處理、不用加分號）。 兩者可以在同一行混用，以 \\ 分隔。

你需要知道的四組指令

Help
\? \h \? variables

\? 列出所有 meta-command；\h 列出所有 SQL 指令；\h CREATE TABLE 顯示特定語法。

Navigate
\l \c \dn \dt

看 資料庫 → schema → 資料表 的階層結構，四個命令走天下。

Inspect
\d \d+ \sf \sv

查看物件定義：\d 看欄位，\sf 看 function 原始碼，\sv 看 view 定義。

Control
\x \timing \pset \watch

切換展開模式、計時、微調輸出格式，以及每隔 N 秒重跑查詢。

02

連線與登入

基本連線語法

# 位置式參數
psql [OPTION...] [DBNAME [USERNAME]]

# 最常見寫法
psql -h db.example.com -p 5432 -U alice -d shop

# 使用連線字串 (conninfo)
psql "host=db.example.com port=5432 dbname=shop user=alice sslmode=require"

# 使用 URI 格式
psql postgresql://alice@db.example.com:5432/shop?sslmode=require

# 使用 service 名稱 (設定於 ~/.pg_service.conf)
psql service=prod-readonly

環境變數（免打字的捷徑）

變數說明對應命令列參數
PGHOST伺服器主機名稱，若開頭為 / 則視為 Unix socket 目錄-h
PGPORT連線埠，預設 5432-p
PGUSER登入使用者名稱，預設為 OS 使用者-U
PGDATABASE預設要連線的資料庫名稱-d
PGPASSWORD密碼（不建議；安全性較差，建議改用 .pgpass）-W
PGPASSFILE自訂 .pgpass 檔案位置—
PGSERVICE預設 service 名稱service=
PGSSLMODESSL 模式：disable / allow / prefer / require / verify-ca / verify-fullsslmode=
PGCLIENTENCODING用戶端編碼（例如 UTF8、BIG5）\encoding
PGOPTIONS啟動時要送出的額外 server 參數options=

~/.pgpass 密碼檔

避免在命令列或環境變數中放明碼密碼。建立 ~/.pgpass（Windows：%APPDATA%\postgresql\pgpass.conf）、權限 chmod 600，每行一筆： 

# hostname:port:database:username:password
db.example.com:5432:shop:alice:s3cret!
# 欄位可用 * 通配
*:*:*:postgres:postmaster-password
Warning｜權限 .pgpass 權限需為 0600，否則 libpq 會忽略它。Windows 則忽略權限設定。

檢查連線狀態

postgres=# \conninfo

切換連線（不離開 psql）

# 連到同 server 的另一個 DB，會繼承 user/host/port
\c other_db

# 完整切換 (db, user, host, port)
\c shop alice db2.example.com 5433

# 用 conninfo 字串
\c "host=localhost dbname=shop sslmode=require"

# 只調整 sslmode，其餘沿用前一連線
\c -reuse-previous=on sslmode=require
03

命令列選項 完整表

以下為 psql 18 全部命令列選項的完整對照表，節錄自官方 app-psql.html。 凡是可以用 meta-command 做的事，幾乎都有對應的 CLI flag，很適合寫進 shell 腳本。

一般 / 執行選項

選項長格式說明
-c CMD--command=CMD執行單一 SQL 或一個 backslash 命令後結束。可重複指定並與 -f 混用；每個 -c 各自為一個 transaction。
-f FILE--file=FILE從檔案讀取命令並執行。- 代表 stdin。錯誤訊息會附上行號，優於 shell redirect。
-l--list列出所有資料庫後結束（類似 \l）。
-v NAME=VAL--set / --variable設定 psql 變數（等同 \set）。
-X--no-psqlrc啟動時不讀取 ~/.psqlrc 與系統 psqlrc。
-1--single-transaction搭配 -c/-f：整批包在單一 transaction 內，任一失敗就 ROLLBACK（需 ON_ERROR_STOP）。
-s--single-step每條 SQL 傳送前先提示確認，用於 debug 腳本。
-S--single-line把換行視為命令結尾（取代分號語意）。
-n--no-readline停用 readline（無 tab 補全、無歷史），貼上大量文字時很有用。
-q--quiet安靜模式：不顯示歡迎訊息與額外資訊。
-V--version印出 psql 版本。
-?--help[=TOPIC]顯示說明。TOPIC 可選 options、commands、variables。

連線選項

選項長格式說明
-d DBNAME--dbname=DBNAME要連線的資料庫名稱；也可以是 conninfo 字串或 URI。
-h HOST--host=HOST伺服器主機；開頭 / 視為 Unix socket 目錄。
-p PORT--port=PORTTCP 埠，預設 5432。
-U USER--username=USER登入使用者。
-w--no-password不提示密碼，適合 batch 使用，無密碼時直接失敗。
-W--password強制提示輸入密碼（即使不需要）。

輸出 / 輸入選項

選項長格式說明
-a--echo-all回顯所有非空輸入行（ECHO=all）。
-b--echo-errors僅回顯失敗的 SQL（ECHO=errors）。
-e--echo-queries回顯真正送到 server 的 SQL（ECHO=queries）。
-E--echo-hidden回顯 \d 之類內部產生的 SQL（除錯/學習用）。
-A--no-align切為 unaligned 模式（最適合管線串接）。
-H--htmlHTML 表格模式輸出。
--csv—CSV 模式輸出（等同 \pset format csv）。
-t--tuples-only只輸出資料列，不含欄位名稱與列數註腳。
-x--expanded啟用「垂直展開」模式（適合欄位多或長字串）。
-F SEP--field-separator=SEP設欄位分隔符號（unaligned 模式用）。
-R SEP--record-separator=SEP設列分隔符號（unaligned 模式用）。
-z--field-separator-zero欄位分隔改為 NUL，適合 xargs -0。
-0--record-separator-zero列分隔改為 NUL。
-P NAME=VAL--pset=NAME=VAL對應 \pset，例如 -P format=latex。
-T OPTS--table-attr=OPTSHTML 表格 <table> 標籤屬性。
-o FILE--output=FILE查詢輸出導向檔案（等同 \o）。
-L FILE--log-file=FILE額外把所有輸出寫進一份 log 檔。

Exit Code 對照

Exit代表意義
0正常結束
1psql 自身的致命錯誤（out of memory、找不到檔案等）
2非互動式工作階段中連線斷線
3腳本錯誤且 ON_ERROR_STOP 為 on

實用組合範例

# 一次性跑查詢、CSV 輸出到檔案
psql -h db -U alice -d shop --csv \
     -c "SELECT id, name FROM users WHERE active" \
     -o active_users.csv

# 跑腳本，遇錯立刻停，整體一個 transaction
psql -v ON_ERROR_STOP=1 -1 -f migration.sql

# 取得單一數值（純文字無裝飾）
count=$(psql -tA -c "SELECT count(*) FROM orders")

# 從 shell 變數注入 SQL 變數
psql -v uid=42 -c "SELECT * FROM users WHERE id = :uid"
Tip｜-tA 組合技 -t -A（tuples-only + no-align）是腳本最常用的組合， 回傳純粹的資料行、沒有欄名、沒有格線，方便直接 parse。
04

Meta-Commands 入門

所有以 未加引號的反斜線開頭的輸入，都是 psql 的 meta-command，由 psql 用戶端本身處理。 它們不需要分號結尾，參數之間用空白分隔，要夾入空白需以單引號包起來。

語法規則速記

Rule 01
以反斜線開頭、換行結束

一個 meta-command 的作用範圍到該行結束。例外：\copy、\!、\ef、\ev 等會吃掉整行剩餘文字。

Rule 02
\\\\ 可中斷參數

想在同一行接著寫 SQL？用 \\（雙反斜線）當分隔。例：\x \\ SELECT 1;

Rule 03
附加 x 可展開

多數 \d… 命令支援附加 x：例如 \dtx 以展開模式顯示表格清單。

Rule 04
附加 + 顯示更多

\d+ users、\dn+、\df+：加號會帶出額外資訊（大小、描述、權限、replica identity…）。

Rule 05
附加 S 包含系統物件

\dtS、\dnS+：S 代表 System，預設只列使用者物件。

Rule 06
Pattern 支援萬用字元

大部分命令可吃 pattern：*（任意字元）、?（單一字元）、"…" 保留大小寫、. 區分 schema 與物件名。

Pattern 範例

shop=> \dt public.*          -- public schema 下所有資料表
shop=> \dt *.order_*         -- 任何 schema 中，名稱以 order_ 開頭的 table
shop=> \df +user_*           -- 名稱以 user_ 開頭的 function，含詳細資訊
shop=> \d "MixedCaseTable"  -- 雙引號保留大小寫

內建說明系統

shop=> \?                    -- 所有 meta-command 列表（約 100+ 條）
shop=> \? variables          -- 所有 psql 變數的說明
shop=> \? options            -- 所有命令列參數的說明
shop=> \h                    -- 所有 SQL 指令清單
shop=> \h CREATE INDEX       -- 特定 SQL 的完整語法
05

物件檢視 \d 系列全覽

\d 家族是 psql 最常用的命令。記住首字母的對應： t=table、v=view、i=index、 s=sequence、m=materialized view、 E=foreign table、n=schema (namespace)、 f=function、T=type、 u/g=role、x=extension。

資料庫全景

命令用途
\l[x+] [PATTERN]列出所有 database；+ 顯示大小、表空間、描述。
\dn[Sx+] [PATTERN]列出 schema（namespace）。
\db[x+] [PATTERN]列出 tablespace。
\dx[x+] [PATTERN]列出已安裝 extension；+ 列出該 extension 所屬物件。
\dconfig[x+] [PATTERN]列出 server 設定參數。無參數時只列非預設值；用 * 顯示全部。
\dy[x+] [PATTERN]列出 event trigger。
\drg[Sx] [PATTERN]列出 role 之間的 GRANT 關係（ADMIN、INHERIT、SET 選項）。

表、檢視、索引、序列

命令用途
\d [PATTERN]對應到 table/view/mv/sequence/index/foreign table 都能用；顯示欄位、索引、constraint、trigger。無 pattern 時等同 \dtvmsE。
\d+ [PATTERN]前者加上：欄位描述、OID、view 定義、replica identity、access method 名稱。
\dt[Sx+] [PATTERN]只列 table；+ 顯示大小、永續性（permanent/temp/unlogged）、描述。
\dv[Sx+] [PATTERN]只列 view。
\dm[Sx+] [PATTERN]只列 materialized view。
\di[Sx+] [PATTERN]只列 index。
\ds[Sx+] [PATTERN]只列 sequence。
\dE[Sx+] [PATTERN]只列 foreign table（外部資料表）。
\dP[itnx+] [PATTERN]列出 partitioned relation；t/i 限制 table/index；n 包含 nested sub-partition；+ 顯示分割區總大小與 access method（PG 18 新增）。

Function、Type、Operator、Cast

命令用途
\df[anptwSx+] [PAT [ARG…]]列 function；a=aggregate、n=normal、p=procedure、t=trigger、w=window；+ 顯示 volatility、並行安全、owner、security、leakproof、權限、語言、描述。
\da[Sx] [PATTERN]只列 aggregate 函式。
\sf FUNCNAME顯示 function/procedure 的原始碼（CREATE OR REPLACE 形式）。
\sv VIEWNAME顯示 view 的原始定義。
\dT[Sx+] [PATTERN]列 data type；+ 顯示內部名稱、大小、enum 值、權限。
\dD[Sx+] [PATTERN]列 domain。
\do[Sx+] [PAT [L [R]]]列運算子（operator），附加 arg pattern 可過濾左右運算元的型別。
\dO[Sx+] [PATTERN]列 collation（排序規則）。
\dC[x+] [PATTERN]列型別轉換（cast）；+（PG 18）顯示 leakproof。
\dc[Sx+] [PATTERN]列字元編碼轉換（conversion）。

權限、角色、註解

命令用途
\dp[Sx] [PATTERN]列 table/view/sequence 的存取權限（等同 \z）。
\z [PATTERN]\dp 的別名。
\ddp[x] [PATTERN]列 ALTER DEFAULT PRIVILEGES 設定。
\du[Sx+] [PATTERN]列 role（使用者）；+ 顯示 comment。
\dg[Sx+] [PATTERN]列 role；由於 role 統一了 user 與 group，此命令與 \du 等價。
\dd[Sx] [PATTERN]顯示物件的註解（僅 constraint、operator class/family、rule、trigger）。
\drds[x] [R-PAT [DB-PAT]]列 per-role、per-database 的參數設定（ALTER ROLE SET）。

存取方法、文字檢索、Foreign Data、Replication、統計

命令用途
\dA[x+] [PATTERN]列 access method（索引類型，如 btree、gist）。
\dAc[x+] [AM [TYPE]]列 operator class。
\dAf[x+] [AM [TYPE]]列 operator family。
\dAo[x+] [AM [OPF]]列 operator family 下的 operator；+（PG 18）顯示 leakproof。
\dAp[x+] [AM [OPF]]列 operator family 的 support function。
\dF[x+] [PATTERN]列文字搜尋 configuration。
\dFd[x+] [PATTERN]列文字搜尋 dictionary。
\dFp[x+] [PATTERN]列文字搜尋 parser。
\dFt[x+] [PATTERN]列文字搜尋 template。
\des[x+] [PATTERN]列 foreign server。
\det[x+] [PATTERN]列 foreign table。
\deu[x+] [PATTERN]列 user mapping（+ 可能洩漏密碼，小心！）
\dew[x+] [PATTERN]列 foreign-data wrapper。
\dRp[x+] [PATTERN]列 logical replication publication。
\dRs[x+] [PATTERN]列 logical replication subscription。
\dL[Sx+] [PATTERN]列 procedural language。
\dl[x+]列 large object（等同 \lo_list）。
\dX[x] [PATTERN]列 extended statistics 物件。
Tip｜偷看 \d 的 SQL 想知道 \d 實際送了什麼 SQL？啟動時加 -E 或在 psql 中 \set ECHO_HIDDEN on，就能看到底層查詢 pg_catalog 的 SQL。對學習系統 catalog 很有幫助。
06

查詢執行控制

這一組 meta-command 處理「如何送 query」：query buffer 就是你輸入、但還沒送到 server 的 SQL。 \g 家族負責送出；\e/\r 負責編輯與重置。

送出、重送、導向

命令用途
\g [(OPT=VAL...)] [FILE|\|CMD]送出 query buffer；可將本次結果導向檔案或管線到 shell 命令；參數可臨時覆寫 \pset。若 buffer 為空則重跑上一個 query。
\gx [(OPT=VAL...)] [FILE|\|CMD]與 \g 相同，但強制用展開模式 (expanded=on) 顯示。
\gdesc顯示下一個 query 結果集的欄位與型別，但不真正執行（檢查語法的好方法）。
\gexec把 query 結果的每一格視為另一條 SQL 並逐一執行。元程式設計神器。
\gset [PREFIX]將 query 結果（必須恰好一列）的各欄位存成同名 psql 變數。
\crosstabview [V [H [D [S]]]]以 crosstab（樞紐表）呈現結果；V=垂直、H=水平、D=資料、S=水平排序欄。
\watch [SEC|INT=n [...]]每隔 N 秒（預設 2）重跑最後一個 query；Ctrl+C 停止。
\;若在一行中多個分號，代表將多個 SQL 組成單一 request 送出（multi-statement）。

編輯與 buffer 操作

命令用途
\e / \edit [FILE [LINE]]開啟 $EDITOR 編輯目前 query buffer 或指定檔案，儲存後回到 psql。
\ef [FUNC [LINE]]編輯 function/procedure 的定義（帶出 CREATE OR REPLACE 樣板）。
\ev [VIEW [LINE]]編輯 view 定義。
\p / \print印出目前 query buffer 內容。
\r / \reset清空 query buffer。
\w FILE把目前 query buffer 寫入檔案。
\s [FILE]顯示或儲存命令歷史。

實戰範例：\gexec 與 \gset

-- \gexec：動態對每個欄位建索引
shop=> SELECT format('CREATE INDEX ON my_table(%I)', attname)
shop->   FROM pg_attribute
shop->   WHERE attrelid = 'my_table'::regclass AND attnum > 0
shop-> \gexec
CREATE INDEX
CREATE INDEX
CREATE INDEX

-- \gset：把查詢結果當變數用
shop=> SELECT count(*) AS row_count FROM orders \gset
shop=> \echo orders total: :row_count
orders total: 12843

-- \gdesc：檢查欄位型別不執行
shop=> SELECT id, created_at, amount FROM orders \gdesc
   Column   |           Type
────────────┼─────────────────────────
 id         │ integer
 created_at │ timestamp with time zone
 amount     │ numeric

實戰範例：\watch 監控

-- 每 1 秒更新一次連線統計
postgres=> SELECT state, count(*)
postgres->   FROM pg_stat_activity
postgres->   GROUP BY state;
postgres=> \watch 1

-- PG 16+ 可用具名選項：只跑 5 次、間隔 2 秒
postgres=> \watch interval=2 count=5

計時與執行資訊

命令用途
\timing [on|off]切換或設定「顯示每條 SQL 執行時間」。
\errverbose以最大 verbosity 重印最後一次 server 錯誤訊息。
\copyright顯示 PostgreSQL 版權與授權。
07

輸出格式化 \pset 全覽

\pset NAME [VALUE] 是控制輸出樣式的主命令。以下命令都是它的常用捷徑： \a（切換 aligned）、\x（展開）、\t（僅資料列）、 \H（HTML）、\f（欄位分隔符）、\C（標題）。

\pset 全部可用選項

選項值說明
formataligned / unaligned / wrapped / html / asciidoc / latex / latex-longtable / troff-ms / csv整體輸出格式。wrapped 會自動折行以符合螢幕寬度。
border0 / 1 / 2 / 3表格框線層級（0 無、1 基本、2 外框+分隔、3 完整）。
expanded (x)on / off / auto展開模式：每列轉成 "欄名｜值" 的兩欄顯示。auto 由 psql 依寬度自動判斷。
null'任意字串'NULL 顯示為何，例如 '(null)'。預設為空字串，無法與空值區分。
fieldsep'字元'unaligned 模式的欄位分隔（預設 |）。
fieldsep_zero—欄位分隔設為 NUL byte。
recordsep'字串'unaligned 模式的列分隔符號（預設 \n）。
recordsep_zero—列分隔改為 NUL byte。
tuples_only (t)on / off只印資料列，省略欄名與列數註腳。
title'標題文字'表格上方標題（也可用 \C）。
tableattr (T)HTML attrsHTML 表格 <table> 的屬性。

變數、條件與 Prompt

psql 具備類似 shell 的變數系統與簡單的條件控制流程，讓腳本更動態。 變數以 :name 代入；若想代入為 SQL 字串則用 :'name'，代入為 SQL 識別字則用 :"name"。

變數基本操作

命令用途
\set [NAME [VALUE…]]設定變數；無參數時列出所有變數。
\unset NAME刪除變數。
\echo [TEXT…]輸出到 stdout，自動解析 :var；-n 取消換行。
\qecho [TEXT…]同上，但輸出到 \o 指定的目的地。
\warn [TEXT…]輸出到 stderr。
\getenv PSQL_VAR ENV_VAR把環境變數讀進 psql 變數。
\prompt [TEXT] NAME互動提示使用者輸入值，儲存到變數。
\password [USER]安全地為指定 role 設定新密碼（以 SCRAM 加密後傳送）。

變數代入的三種形式

shop=> \set uid 42
shop=> \set tbl users
shop=> \set email 'alice@example.com'

-- 1) :name  直接原樣代入（注意注入風險）
shop=> SELECT * FROM :tbl WHERE id = :uid;

-- 2) :'name'  代入為 SQL 字串常數（自動加引號與跳脫）
shop=> SELECT * FROM users WHERE email = :'email';
-- 展開為：WHERE email = 'alice@example.com'

-- 3) :"name"  代入為 SQL 識別字（自動加雙引號）
shop=> SELECT * FROM :"tbl";
-- 展開為：FROM "users"

-- 檢查變數是否定義
shop=> \if :{?uid}
shop@>   \echo uid is set
shop@> \endif

腳本與自動化

psql 的腳本能力介於 SQL 批次執行與 shell 之間，很適合用來寫 migration、備份腳本、報表產生器。 關鍵工具：\i、\ir、\!、\cd。

檔案、shell、目錄

命令用途
\i FILENAME從指定檔案讀取並執行 SQL/meta-commands。
\ir FILENAME同 \i，但路徑相對於當前腳本所在目錄（適合專案內相對引用）。
\include FILENAME\i 的完整名稱。
\include_relative FILENAME\ir 的完整名稱。
\! [CMD]執行 shell 命令；不帶參數則進入子 shell（離開後回到 psql）。
\cd [DIR]切換 psql 的工作目錄。不帶參數切回家目錄。
\quit / \q離開 psql。

錯誤處理的最佳實踐

-- 建議在所有 migration 腳本頂部加入以下兩行
\set ON_ERROR_STOP on
\set VERBOSITY verbose

BEGIN;

\ir 001_create_tables.sql
\ir 002_seed_data.sql
\ir 003_create_indexes.sql

-- 驗證
SELECT count(*) FROM users;
SELECT count(*) FROM orders;

COMMIT;
Tip｜-1 與 -v ON_ERROR_STOP 的差別 -v ON_ERROR_STOP=1 只是遇錯停止； -1 / --single-transaction 則額外把整個 -c/-f 批次包入單一 transaction。 兩者常常搭配使用。

互動式 prompt 範例

\prompt '確認刪除？輸入 YES: ' confirm

\if :'confirm' = 'YES'
    DROP SCHEMA :"target_schema" CASCADE;
    \echo 'schema dropped'
\else
    \echo '取消'
\endif

在 shell 與 psql 之間傳遞資料

匯入匯出 \copy

SQL COPY vs \copy 的本質差異： SQL COPY 由 server 端讀寫檔案，需要 superuser 權限並使用 server 本機路徑； \copy 則由 psql 用戶端讀寫本地檔案，再透過連線傳輸資料，無須特權，是日常匯入匯出的主力。

\copy 語法

-- 從檔案 / 命令 / stdin 匯入
\copy TABLE [ (COL, ...) ]
      FROM {'FILENAME' | PROGRAM 'CMD' | STDIN | PSTDIN}
      [ [ WITH ] (OPTION [, ...]) ]
      [ WHERE CONDITION ]

-- 匯出到檔案 / 命令 / stdout
\copy { TABLE [ (COL, ...) ] | (QUERY) }
      TO {'FILENAME' | PROGRAM 'CMD' | STDOUT | PSTDOUT}
      [ [ WITH ] (OPTION [, ...]) ]

常用 WITH 選項

選項預設說明
FORMATtexttext / csv / binary
DELIMITERtab / ','欄位分隔符
NULL'\\N' / ''代表 NULL 的字串
HEADERfalseCSV：含標題行；讀取時跳過第一列
QUOTE'"'CSV 引號字元
ESCAPE同 QUOTECSV 引號的跳脫字元
FORCE_QUOTE—強制為指定欄位加引號
FORCE_NOT_NULL—匯入時指定欄位空字串視為空字串而非 NULL
FORCE_NULL—匯入時指定欄位加引號的空字串也視為 NULL
ENCODING—檔案編碼
FREEZEfalse匯入時直接 freeze（減少後續 VACUUM 負擔）
ON_ERRORstopPG 17+：stop/ignore，忽略無法轉型的列

實戰範例

-- 匯出 CSV（含標題）
shop=> \copy users TO 'users.csv' WITH (FORMAT csv, HEADER);

-- 匯出查詢結果
shop=> \copy (SELECT id, name FROM users WHERE active) TO 'active.csv' CSV HEADER;

-- 匯入 CSV（跳過第一行標題）
shop=> \copy users FROM 'users.csv' WITH (FORMAT csv, HEADER);

-- 匯入時只載入符合條件的列
shop=> \copy users FROM 'users.csv' CSV HEADER WHERE created_at >= '2026-01-01';

-- 透過 PROGRAM 壓縮管線
shop=> \copy big_table TO PROGRAM 'gzip > big.csv.gz' CSV HEADER;
shop=> \copy big_table FROM PROGRAM 'gunzip -c big.csv.gz' CSV HEADER;

-- 在 .sql 腳本中 inline 匯入資料（STDIN）
\copy countries (code, name) FROM STDIN CSV
TW,Taiwan
JP,Japan
US,United States
\.
Warning｜PostgreSQL 18 的 \copy 變更 \. 在 PG 18 開始必須單獨一行（不可有其他字元）才會被當成 CSV STDIN 的結束符號。 若舊腳本在資料行尾巴接 \.，可能需要調整。

PostgreSQL 18 新功能

PostgreSQL 18（首次發佈於 2025 年 9 月，目前最新為 18.3）為 psql 帶來多項革命性更新， 主要圍繞兩大主題：Pipeline 模式與Prepared Statement 原生支援。

Pipeline 模式

在高延遲網路下，傳統的「送一條、等一條」會浪費大量 RTT。Pipeline 讓 psql 在不等待回應的狀況下， 連續把多條 query 推送給 server，顯著提升吞吐量。以下七個新命令都是為此而生：

命令用途
\startpipeline進入 pipeline 模式。Prompt 會出現 * 或 %P 顯示狀態。進入後會隱含一個 transaction。
\endpipeline離開 pipeline 模式，送出所有剩餘命令並取得結果。
\syncpipeline在 pipeline 中建立同步點；本點之前失敗會 rollback 到上一個 sync，不影響之後。
\sendpipeline把目前 query buffer 送入 pipeline，但不等待結果。
\flushrequest告訴 server 立即回傳目前已完成的結果（不等待更多 request）。
\flush強制把本地緩衝的命令實際送出到 server。
\getresults [N]抓取 pipeline 中已完成的結果；可指定抓幾個。
 
-- 典型的 pipeline：批次寫入高延遲 DB
BEGIN;
PREPARE ins AS INSERT INTO events(device, ts, value)
          VALUES($1, $2, $3);

\startpipeline
EXECUTE ins('dev-001', now(), 12.3);
EXECUTE ins('dev-002', now(), 45.6);
EXECUTE ins('dev-003', now(), 78.9);
\syncpipeline                -- 建立同步點
EXECUTE ins('dev-004', now(), 10.1);
\endpipeline
COMMIT;
New｜Pipeline 效能收益 官方示範：在 500ms 人工延遲下，5 條 query 從 10 秒降到約 6 秒； 高延遲環境下的 ETL/Migration 腳本會明顯感覺到差異。
⚠️ Pipeline 模式不支援 COPY、simple query protocol 專用語法，也不支援同一 request 中多條 SQL。

Prepared Statement 原生命令

PG 18 引入三個低階命令，讓你可以在 psql 直接測試 extended query protocol， 而不用透過 SQL 的 PREPARE/EXECUTE。

命令用途
\parse STATEMENT_NAME把目前 query buffer 解析（parse）為 prepared statement，以 STATEMENT_NAME 為名儲存在連線中。空字串代表 unnamed statement。
\bind [PARAM ...]為下一個要執行的 query 綁定參數（對應 $1、$2…）。會觸發使用 extended query protocol。
\bind_named NAME [PARAM ...]綁定到已命名的 prepared statement。
\close_prepared NAME釋放指定 prepared statement；空字串代表 unnamed。
 
-- 方法一：直接綁參數送一次性查詢（自動使用 extended protocol）
shop=> SELECT * FROM users WHERE id = $1 AND name = $2
shop-> \bind 42 'alice' \g

-- 方法二：預先 parse，多次綁定執行
shop=> SELECT * FROM users WHERE id = $1 \parse get_user
shop=> \bind_named get_user 42 \g
shop=> \bind_named get_user 99 \g

-- 用完釋放
shop=> \close_prepared get_user

其他 psql 18 亮點

PG 18
\conninfo 改版

改為表格式輸出並增加欄位：SSL 協定、protocol 版本、Session User 與 Current User（差異在於 SET ROLE）、Host Address 等。

PG 18
\d+x 新後綴

所有 list 命令都支援加 x（expanded）後綴，無需事先 \x。例：\dtx、\dfx。

PG 18
Leakproof 指標

\df+、\do+、\dAo+、\dC+ 新增 leakproof 欄，對 row-level security 相關審計很有幫助。

PG 18
\dP+ 分割區細節

分割表顯示各分割區的 access method；\dP+n 額外顯示 nested 分割總大小。

PG 18
Prompt %P、%s

%P 顯示 pipeline 狀態；%s 顯示 service 名稱。有 service 檔 + 多環境切換時很實用。

PG 18
Service 變數

新增 SERVICE 這個唯讀變數，讓腳本能根據目前連的 service 做判斷。

Server 端相關亮點（摘錄）

雖然這份文件聚焦 psql，但值得一提 PG 18 的 server 端重大改變： 非同步 I/O（AIO）、Skip Scan 多欄位 B-tree 索引、 uuidv7() 產生時序 UUID、virtual generated columns（現為預設）、 OAuth 認證、RETURNING 支援 OLD/NEW、 PRIMARY KEY / UNIQUE / FOREIGN KEY 支援 temporal constraints。

.psqlrc 個人化

psql 啟動時會自動讀取 ~/.psqlrc（Windows 為 %APPDATA%\postgresql\psqlrc.conf）， 並可再讀取 ~/.psqlrc-<version> 做版本專屬設定。 加上 -X 啟動則會略過讀取，適合寫進自動化腳本避免受影響。

實用 .psqlrc 範本

\set QUIET on

\set ON_ERROR_ROLLBACK interactive

\pset expanded auto

-- NULL 顯示為 (null)，避免與空字串混淆
\pset null '(null)'

-- 每 100 列分頁，數字加千分位
\pset pager_min_lines 100
\pset numericlocale on

-- 補全關鍵字自動大寫
\set COMP_KEYWORD_CASE upper

-- 歷史：忽略重複，最多 5000 筆，每個 DB 分開儲存
\set HISTCONTROL ignoredups
\set HISTSIZE 5000
\set HISTFILE ~/.psql_history- :DBNAME

-- 彩色 prompt：黃色使用者 @ 青色 DB : port  [tx 狀態] [pipeline 狀態] #
\set PROMPT1 '%[%033[1;33m%]%n%[%033[0m%]@%[%033[1;36m%]%/%[%033[0m%]:%> %x[pipeline:%P] %# '
\set PROMPT2 '%[%033[2m%]... %R %# %[%033[0m%]'

-- 自訂捷徑：重要查詢給個短別名
\set activity 'SELECT pid, state, wait_event, query_start, left(query, 60) FROM pg_stat_activity WHERE pid <> pg_backend_pid() ORDER BY query_start;'

\set blocking 'SELECT blocked_locks.pid AS blocked_pid, blocking_locks.pid AS blocking_pid, blocked_activity.query AS blocked_query, blocking_activity.query AS blocking_query FROM pg_catalog.pg_locks blocked_locks JOIN pg_catalog.pg_stat_activity blocked_activity ON blocked_activity.pid = blocked_locks.pid JOIN pg_catalog.pg_locks blocking_locks ON blocking_locks.locktype = blocked_locks.locktype AND blocking_locks.database IS NOT DISTINCT FROM blocked_locks.database AND blocking_locks.relation IS NOT DISTINCT FROM blocked_locks.relation AND blocking_locks.pid <> blocked_locks.pid JOIN pg_catalog.pg_stat_activity blocking_activity ON blocking_activity.pid = blocking_locks.pid WHERE NOT blocked_locks.granted;'

\set dbsize 'SELECT datname, pg_size_pretty(pg_database_size(datname)) FROM pg_database ORDER BY pg_database_size(datname) DESC;'

\set tblsize 'SELECT schemaname, relname, pg_size_pretty(pg_total_relation_size(relid)) FROM pg_stat_user_tables ORDER BY pg_total_relation_size(relid) DESC LIMIT 20;'

-- 使用方式：輸入 :activity 然後 Enter

\unset QUIET
Tip｜版本專屬 psqlrc 若同時維護多個 PG 版本的伺服器，可建立 ~/.psqlrc-18、~/.psqlrc-17， psql 會先讀 ~/.psqlrc、再讀版本專屬檔（覆寫設定）。

⑫ 五個你一定要記的絕招

Trick 01
用 -E 學系統 catalog

psql -E 會顯示 \d 內部產生的 SQL，是學習 pg_catalog 最快的方法。

Trick 02
展開模式 auto

\pset expanded auto：輸出寬度不夠才自動展開，兼顧緊湊與可讀性。

Trick 03
FETCH_COUNT 處理大結果

\set FETCH_COUNT 1000：psql 改為用 cursor 分批讀，不會一次灌爆記憶體。

Trick 04
-tA 拿純值

shell 腳本裡：psql -tA -c 'SELECT now()' 得到乾淨的單一數值或資料列。

Trick 05
\watch 秒速監控

任何查詢 + \watch 1 = 一個即時儀表板，用 Ctrl+C 停。

Trick 06
\gexec 元程式設計

用 SQL 產生 SQL，再用 \gexec 執行。批次重命名、建立索引、清理物件都靠它。
