🌐叢集啟動 — 三種方式

方式一：靜態配置

TOKEN=token-01; CLUSTER_STATE=new
CLUSTER=m1=http://10.0.0.1:2380,m2=http://10.0.0.2:2380,m3=http://10.0.0.3:2380
etcd --name m1 \
  --initial-advertise-peer-urls http://10.0.0.1:2380 \
  --listen-peer-urls http://10.0.0.1:2380 \
  --advertise-client-urls http://10.0.0.1:2379 \
  --listen-client-urls http://10.0.0.1:2379 \
  --initial-cluster ${CLUSTER} \
  --initial-cluster-state ${CLUSTER_STATE} \
  --initial-cluster-token ${TOKEN}

方式二：etcd Discovery

curl https://discovery.etcd.io/new?size=3
etcd --name m1 --discovery https://discovery.etcd.io/<token> ...

方式三：DNS SRV

_etcd-server-ssl._tcp.example.com. 300 IN SRV 0 0 2380 m1.example.com.
etcd --name m1 --discovery-srv example.com ...
--initial-cluster 啟動後忽略。每叢集唯一 --initial-cluster-token。

🐳容器 / K8s 部署

Docker

docker run -p 2379:2379 -p 2380:2380 \
  --volume=${DATA_DIR}:/etcd-data \
  quay.io/coreos/etcd:v3.5.21 /usr/local/bin/etcd \
  --data-dir=/etcd-data --name node1 \
  --listen-client-urls http://0.0.0.0:2379 \
  --advertise-client-urls http://${NODE1}:2379

K8s StatefulSet 要點

  • Headless Service (2379/2380/8080)
  • 3 Replicas + podManagementPolicy: Parallel
  • Readiness: /readyz:8080, Liveness: /livez:8080
  • Anti-affinity + VolumeClaimTemplate
  • TLS: cert-manager + ClusterIssuer

🔧成員管理

etcdctl member list -w table
etcdctl member add node4 --peer-urls=http://10.0.1.13:2380
etcdctl member add node4 --peer-urls=http://10.0.1.13:2380 --learner
etcdctl member promote <ID>
etcdctl member update <ID> --peer-urls=http://10.0.1.10:2380
etcdctl member remove <ID>
新成員以 --initial-cluster-state=existing 啟動。

🔒TLS 傳輸安全

客戶端 HTTPS

etcd --cert-file=/path/server.crt --key-file=/path/server.key \
  --advertise-client-urls=https://127.0.0.1:2379 \
  --listen-client-urls=https://127.0.0.1:2379

客戶端憑證認證

etcd --client-cert-auth --trusted-ca-file=/path/ca.crt \
  --cert-file=/path/server.crt --key-file=/path/server.key

Peer TLS

etcd --peer-client-cert-auth --peer-trusted-ca-file=/path/ca.crt \
  --peer-cert-file=/path/m1.crt --peer-key-file=/path/m1.key

自簽 TLS

etcd --auto-tls --peer-auto-tls
Flag預設說明
--cert-file''Client TLS 憑證
--key-file''Client TLS 私鑰
--client-cert-authfalse啟用憑證驗證
--auto-tlsfalse自簽憑證
--peer-auto-tlsfalsePeer 自簽
--tls-min-versionTLS1.2最低版本
v3.2.0+ TLS 憑證每次連線重新載入。

👤RBAC 認證授權

使用者管理

etcdctl user add root                              # 建立 root
etcdctl user add root --new-user-password=pw       # 行內密碼
etcdctl user add myuser --no-password               # TLS CN 認證
etcdctl user passwd myuser                           # 改密碼
etcdctl user list / user get / user delete
etcdctl user grant-role myuser myrole
etcdctl user revoke-role myuser myrole

角色管理

etcdctl role add myrole
etcdctl role grant-permission myrole read key1              # 單 key
etcdctl role grant-permission myrole readwrite key1 key5  # 範圍
etcdctl role grant-permission myrole --prefix read /p/    # 前綴
etcdctl role list / role get / role delete

啟用認證

etcdctl auth enable     # 需先有 root user + root role
etcdctl --user root:pw put foo bar
Flag預設說明
--auth-tokensimplesimple 或 jwt
--bcrypt-cost104–31
--auth-token-ttl300秒
