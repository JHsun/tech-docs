⚡etcd 是什麼？

etcd 是強一致性的分散式鍵值儲存系統，基於 Raft 共識演算法，用於儲存分散式系統中最關鍵的資料。

核心特性

  • 強一致性 — 讀寫預設線性化 (Linearizable)
  • MVCC — 多版本並行控制，支援歷史查詢
  • Watch — 可靠即時變更通知串流
  • 原子事務 — compare-and-swap 事務
  • Lease 租約 — TTL 自動過期刪除
  • RBAC — 角色基礎存取控制

預設端口

2379客戶端通訊 (Client)
2380節點間通訊 (Peer)

📦安裝方式

預編譯二進位檔（推薦）

# GitHub Releases 下載
ETCD_VER=v3.5.21
curl -L https://github.com/etcd-io/etcd/releases/download/${ETCD_VER}/etcd-${ETCD_VER}-linux-amd64.tar.gz -o etcd.tar.gz
tar xzf etcd.tar.gz
sudo mv etcd-${ETCD_VER}-linux-amd64/etcd* /usr/local/bin/

macOS Homebrew

brew install etcd

原始碼（Go 1.16+）

git clone -b v3.5.21 https://github.com/etcd-io/etcd.git
cd etcd && ./build.sh
export PATH="$PATH:$(pwd)/bin"

Docker

docker run -p 2379:2379 -p 2380:2380 \
  quay.io/coreos/etcd:v3.5.21

驗證

etcd --version
etcdctl version
etcdutl version

🚀快速入門

啟動單節點

# 終端機 1：啟動
$ etcd
# 終端機 2：測試
$ etcdctl put greeting "Hello, etcd"
OK
$ etcdctl get greeting
greeting
Hello, etcd
預設資料目錄 ${name}.etcd，名稱預設 default

輸出格式 -w

格式用途
simple預設文字
json含 revision / raft_term
table表格，人類友善
protobuf二進位
fields逐行欄位

⚠系統限制

項目預設 / 建議
單次請求大小1.5 MiB --max-request-bytes
儲存空間配額建議 ≤ 8 GiB --quota-backend-bytes
Key 大小無硬限制，建議精簡
Value 大小≤ 1.5 MiB（受請求限制）
Lease TTL秒為單位
etcd 定位為中繼資料儲存，不適合大量資料。

✏️寫入 — Put

etcdctl put foo "Hello World"
etcdctl put foo "new" --prev-kv          # 回傳舊值
etcdctl put foo "tmp" --lease=ID       # 綁定租約
etcdctl put foo "" --ignore-value       # 僅更新 lease
etcdctl put foo "bar" --ignore-lease    # 僅更新值

🔍讀取 — Get / Range

etcdctl get foo                              # 單鍵
etcdctl get foo --print-value-only          # 僅值
etcdctl get /cfg/ --prefix                  # 前綴
etcdctl get key1 key5                       # 範圍
etcdctl get "" --prefix --limit=10        # 限筆
etcdctl get "" --prefix                      # 全部
etcdctl get foo --rev=4                     # 歷史
etcdctl get "" --prefix --order=ASCEND --sort-by=KEY
etcdctl get "" --prefix --keys-only         # 僅鍵
etcdctl get "" --prefix --count-only        # 筆數
etcdctl get foo --consistency=s             # 序列化讀
--sort-by：KEY / VERSION / CREATE / MOD / VALUE

🗑刪除 — Del

etcdctl del foo                       # 單鍵
etcdctl del /tmp/ --prefix            # 前綴
etcdctl del key1 key5                # 範圍
etcdctl del foo --prev-kv            # 回傳被刪值

🔄事務 — Txn (原子 CAS)

etcdctl txn --interactive
# compares:
value("/users/123/email") = "old@test.com"
# success requests:
put /users/123/email "new@test.com"
# failure requests:
get /users/123/email

比較目標

value(key)比較值
version(key)版本號
create(key)建立 revision
mod(key)修改 revision

👁監聽 — Watch

etcdctl watch foo                     # 單鍵
etcdctl watch /cfg/ --prefix          # 前綴
etcdctl watch foo --rev=2             # 從 rev 開始
etcdctl watch foo --prev-kv          # 含舊值
etcdctl watch -i                      # 互動模式

Watch 保證

  • 有序：按 revision 排
  • 可靠：不遺漏連續事件
  • 原子：同 revision 一起送出

⏱租約 — Lease

etcdctl lease grant 300               # 授予 300s TTL
etcdctl put s1 "d" --lease=ID       # 綁定
etcdctl lease keep-alive ID           # 心跳續約
etcdctl lease timetolive ID           # 剩餘 TTL
etcdctl lease timetolive ID --keys   # 含關聯 key
etcdctl lease revoke ID               # 撤銷(刪關聯key)
etcdctl lease list                     # 列出全部
租約不保證互斥，不可單獨做分散式鎖。

🔒Lock & Election

分散式鎖

etcdctl lock my-lock                  # 阻塞取鎖
etcdctl lock my-lock -- echo "done"   # 取鎖後執行

Leader 選舉

etcdctl elect my-election node1      # 參選
gRPC RPC說明
Lock / Unlock取得 / 釋放鎖
Campaign參與選舉
Proclaim更新 Leader 值
Resign放棄領導權
Observe串流選舉結果

🧹日常維護速查

etcdctl compact $REV                  # 壓縮舊版本
etcdctl defrag                         # 重組磁碟
etcdctl alarm list                     # 查看告警
etcdctl alarm disarm                   # 解除告警
etcdctl move-leader ID                # 移轉 Leader
etcdctl snapshot save backup.db       # 備份快照
etcdutl snapshot status backup.db -w table

NOSPACE 告警恢復

rev=$(etcdctl endpoint status -w json | grep -o '"revision":[0-9]*' | grep -o '[0-9]*')
etcdctl compact $rev
etcdctl defrag
etcdctl alarm disarm
