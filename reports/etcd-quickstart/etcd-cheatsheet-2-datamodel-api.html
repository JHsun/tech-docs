🧬MVCC 資料模型

etcd 使用多版本並行控制 (MVCC) 的持久化鍵值儲存，後端為 bbolt（B+ Tree）。每次異動產生新版本，不覆蓋舊資料。

核心概念

概念說明
Key Space扁平鍵空間，按位元組字串字典序排列
Revision64-bit 全域單調遞增邏輯時鐘。每次原子異動 +1。初始 = 1
Version每個 key 的修改計數器。修改 +1，刪除歸零
Generationkey 從建立到刪除的生命週期。刪除產生 tombstone
Compaction移除指定 revision 之前的歷史版本（最新值保留）

物理儲存結構

B+ Tree key 為三元組 (major, sub, type)：major = 儲存 revision，sub = 同 revision 內區分子，type = 選用後綴（如 t 代表 tombstone）。另有記憶體內 B-Tree 索引將使用者 key 映射到 revision 指標。

📋KeyValue Protobuf 結構

message KeyValue {
  bytes  key             = 1;  // 鍵（不可為空）
  int64  create_revision = 2;  // 最後建立的 revision
  int64  mod_revision    = 3;  // 最後修改的 revision
  int64  version         = 4;  // 當前 generation 的版本
  bytes  value           = 5;  // 值
  int64  lease           = 6;  // 租約 ID（0=無）
}

Response Header（所有回應共有）

message ResponseHeader {
  uint64 cluster_id = 1;
  uint64 member_id  = 2;
  int64  revision   = 3;  // 當前 KV revision
  uint64 raft_term  = 4;
}

📡gRPC Gateway (REST API)

v3.5+ 使用 /v3/* 端點。Key/Value 需 base64 編碼。

# Put
curl -X POST http://127.0.0.1:2379/v3/kv/put \
  -d '{"key":"Zm9v","value":"YmFy"}'
# Get
curl -X POST http://127.0.0.1:2379/v3/kv/range \
  -d '{"key":"Zm9v"}'
# Watch
curl -N -X POST http://127.0.0.1:2379/v3/watch \
  -d '{"create_request":{"key":"Zm9v"}}'
# 認證取 Token
curl -X POST http://127.0.0.1:2379/v3/auth/authenticate \
  -d '{"name":"root","password":"pw"}'
# 後續請求: -H "Authorization: <token>"
gRPC Gateway 不支援 TLS Common Name 認證。

🔑RangeRequest 關鍵參數

參數說明
key查詢起始鍵
range_end範圍結束鍵（不含）
limit最大回傳筆數
revision查詢指定歷史版本
sort_orderNONE / ASCEND / DESCEND
sort_targetKEY / VERSION / CREATE / MOD / VALUE
serializable允許非最新讀（低延遲）
keys_only僅回傳鍵
count_only僅回傳筆數

🛡️API 一致性保證

保證說明
原子性所有 API 呼叫皆原子
持久性已完成操作在故障後持久
線性化讀預設所有讀寫經 Raft 共識
序列化讀低延遲但可能讀到舊資料

Watch 保證

  • 有序 — 按 revision 排列
  • 可靠 — 不遺漏連續事件
  • 原子 — 同 revision 完整送出
  • 可書籤 — Progress notification
Lease 不保證互斥。

🎓Learner 設計

Learner = 非投票 Raft 成員

etcdctl member add node4 --peer-urls=http://10.0.1.13:2380 --learner
etcdctl member promote <memberID>
  • 不能被選為 Leader
  • 拒絕客戶端讀寫（序列化讀除外）
  • 叢集同時只允許一個 Learner

🔐認證設計 (v3 Auth)

  • 認證元資料透過 Raft 共識同步
  • 密碼雜湊：bcrypt（成本 4–31，預設 10）
  • Token：Simple 或 JWT

權限模型

  • 精確 key 或範圍 [start, end)
  • 類型：read / write / readwrite

⚖️gRPC Naming & Discovery

import (
    resolver "go.etcd.io/etcd/client/v3/naming/resolver"

r, _ := resolver.NewBuilder(etcdClient)
