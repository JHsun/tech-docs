🛟災難復原

快照備份

etcdctl --endpoints=http://10.0.0.1:2379 snapshot save snapshot.db
etcdutl snapshot status snapshot.db -w table

快照還原

etcdutl snapshot restore snapshot.db \
  --name m1 \
  --initial-cluster m1=http://h1:2380,m2=http://h2:2380,m3=http://h3:2380 \
  --initial-cluster-token etcd-cluster-1 \
  --initial-advertise-peer-urls http://h1:2380
# K8s 專用
etcdutl snapshot restore snapshot.db \
  --bump-revision 1000000000 --mark-compacted --data-dir /restore
--bump-revision 覆蓋過期 watch。所有成員從同一份快照還原。

🚪etcd Gateway (L4)

無狀態 TCP 代理，不終結 TLS。

etcd gateway start --endpoints=infra0:2379,infra1:2379
etcd gateway start --discovery-srv=example.com
# 預設 127.0.0.1:23790

🔀gRPC Proxy (L7)

Watch 合併、Lease 合併、命名空間隔離、TLS 終結。

etcd grpc-proxy start --endpoints=localhost:2379 \
  --listen-addr=127.0.0.1:23790
# 命名空間
  --namespace=my-prefix/
# TLS 終結
  --cert c.crt --key c.key --cacert=ca.crt
# 端點註冊
  --advertise-client-url=127.0.0.1:23790 \
  --resolver-prefix="___grpc_proxy_endpoint" --resolver-ttl=60

🔧資料損壞處理

etcd --experimental-initial-corrupt-check=true
etcd --experimental-corrupt-check-time=2h
  • 移出 snap 子目錄重啟
  • member remove → member add
  • 全叢集快照還原
etcdctl alarm disarm
