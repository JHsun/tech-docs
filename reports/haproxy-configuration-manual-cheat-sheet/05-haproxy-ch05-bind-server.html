5.1

Bind 選項（Frontend 監聽）

基本語法

bind [<address>]:<port>[,<port>...] [param1] [param2] ...

位址格式

格式說明範例
IPv4指定 IP 與 portbind 192.168.1.1:80
IPv4 萬用所有介面bind *:80 或 bind :80
IPv6須用方括號或 ipv6@bind ipv6@:80 或 bind [::]:80
Unix Socket本機通訊bind unix@/var/run/haproxy.sock
fd@N繼承的 file descriptorbind fd@${FD_APP}
多 port逗號分隔bind :80,:443

一般 Bind 參數

參數說明
name <n>為此 listener 命名（顯示於 stats）
maxconn <n>此 listener 最大併發連線
backlog <n>TCP listen backlog 佇列大小
interface <if>綁定至特定網路介面（Linux）
namespace <ns>Linux network namespace
transparent啟用透明代理（需 ip_nonlocal_bind）
defer-accept延遲 accept 直到有資料到達（Linux TCP_DEFER_ACCEPT）
mss <n>設定 TCP MSS
tfo啟用 TCP Fast Open
shards <n|by-thread|by-group>將 listener 分片到多個執行緒

Bind SSL/TLS 參數

參數說明
ssl啟用 SSL 終止（必須搭配 crt）
crt <path>憑證檔路徑（PEM 格式含 cert + key）；可為目錄（載入全部）
crt-list <file>憑證對照清單（SNI → cert 映射）
ca-file <path>Client CA 驗證用憑證
verify <none|optional|required>Client 憑證驗證模式
crl-file <path>憑證吊銷清單
alpn <list>ALPN 協商清單
ciphers <list>TLS ≤ 1.2 cipher suite
ciphersuites <list>TLS 1.3 cipher suite
ssl-min-ver <ver>最低 TLS 版本：TLSv1.2、TLSv1.3
ssl-max-ver <ver>最高 TLS 版本
prefer-client-ciphers優先使用 client 的 cipher 順序
strict-sniSNI 不匹配任何憑證時拒絕連線
generate-certificates動態產生憑證（SSL 攔截用）
no-tls-tickets停用 TLS session ticket
allow-0rtt允許 TLS 1.3 0-RTT early data 安全風險
ocsp-update on自動更新 OCSP stapling
 
# HTTPS 前端典型設定
frontend https-in
    bind :443 ssl \
        crt /etc/ssl/certs/site.pem \
        alpn h2,http/1.1 \
        ssl-min-ver TLSv1.2 \
        ciphers ECDHE+AESGCM:ECDHE+CHACHA20 \
        ciphersuites TLS_AES_128_GCM_SHA256:TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256

# SNI 多憑證 — crt-list
frontend multi-ssl
    bind :443 ssl crt-list /etc/haproxy/crt-list.cfg

# crt-list.cfg 內容：
# /etc/ssl/app-a.pem [alpn h2,http/1.1] app-a.example.com
# /etc/ssl/app-b.pem                     app-b.example.com *.app-b.example.com

# Client mTLS
frontend mtls-in
    bind :8443 ssl crt /etc/ssl/server.pem ca-file /etc/ssl/client-ca.pem verify required

PROXY Protocol 參數

參數說明
accept-proxy接受 PROXY Protocol v1/v2 header（從上游 LB 取得真實 IP）
accept-netscaler-cip接受 NetScaler CIP header
proxy-v2-options <opts>指定接受的 PPv2 TLV：authority、ssl、cert-cn…

QUIC / HTTP/3

# HTTP/3 支援（HAProxy 3.0 實驗性）
frontend h3-in
    bind quic4@:443 ssl crt /etc/ssl/site.pem alpn h3
    bind :443 ssl crt /etc/ssl/site.pem alpn h2,http/1.1
⚠ QUIC 注意事項使用 quic4@ 或 quic6@ 前綴。需搭配 limited-quic 或編譯時啟用完整 QUIC 支援。連線遷移目前不支援。
5.2

Server & default-server 選項

基本語法

server <name> <address>[:port] [param1] [param2] ...
default-server [param1] [param2] ...

常用一般參數

參數說明
weight <0-256>權重（0 = 不接收新連線；預設 1）
maxconn <n>此 server 最大併發連線
minconn <n>搭配 fullconn 使用的最小連線數
maxqueue <n>此 server 佇列上限
backup標記為備用 server（僅所有主 server 全 DOWN 時啟用）
disabled初始狀態為停用
cookie <value>用於 cookie-based persistence 的 server ID
redir <url>將所有請求重導至此 URL
slowstart <time>恢復後漸進增加流量的時間
on-error <act>連續錯誤時的行為：fastinter、fail-check、sudden-death、mark-down
error-limit <n>觸發 on-error 的錯誤次數
on-marked-down <act>標記 DOWN 時：shutdown-sessions
on-marked-up <act>標記 UP 時：shutdown-backup-sessions
send-proxy向 server 發送 PROXY Protocol v1
send-proxy-v2向 server 發送 PROXY Protocol v2
send-proxy-v2-sslPPv2 含 SSL 資訊
send-proxy-v2-ssl-cnPPv2 含 Client CN
proto <name>強制使用的協定 mux（如 h2）
socks4 <addr>:<port>透過 SOCKS4 proxy 連線

Server SSL/TLS 參數（後端加密）

參數說明
ssl啟用 SSL 連線至 server
verify <none|required>是否驗證 server 憑證 生產用 required
ca-file <path>驗證 server 憑證的 CA
crt <path>Client 憑證（mTLS 對 server 端）
sni req.hdr(host)動態設定 SNI
alpn <list>ALPN 協商
check-sslHealth check 也走 SSL
check-sni <host>Check 使用的 SNI
check-alpn <list>Check 使用的 ALPN
ssl-reuse重用 SSL session（預設啟用）
ciphers / ciphersuites同 bind 選項

Server Health Check 參數

參數說明預設
check啟用 health check—
inter <time>check 間隔2s
fastinter <time>狀態轉換中的快速 check 間隔—
downinter <time>DOWN 狀態時的 check 間隔—
fall <n>連續失敗 n 次 → DOWN3
rise <n>連續成功 n 次 → UP2
addr <ip>check 使用的目的 IP（不同於資料流）—
port <n>check 使用的 port—
observe <layer4|layer7>額外觀察實際流量判斷健康—
agent-check啟用 agent check（外部程式回報狀態）—
agent-addr / agent-portAgent 位址與 port—
agent-inter <time>Agent check 間隔—
agent-send <string>向 agent 發送的字串—

Server DNS 解析參數

參數說明
resolvers <id>使用的 resolvers section
resolve-prefer <ipv4|ipv6>DNS 解析偏好
resolve-net <cidr>[,...]偏好的子網段
resolve-opts <opts>如 allow-dup-ip、prevent-dup-ip
init-addr <methods>初始化時的位址解析方法順序：last、libc、none
5.3

DNS Resolvers Section

定義 DNS 伺服器與解析策略，讓 server 位址可依 DNS 動態更新。

關鍵字說明
nameserver <id> <addr>:<port>定義 DNS 伺服器
parse-resolv-conf自動讀取 /etc/resolv.conf
resolve_retries <n>查詢失敗重試次數（預設 3）
hold valid <time>成功解析結果的保留時間
hold obsolete <time>過期解析結果的保留時間
hold nx <time>NXDOMAIN 結果的保留時間
hold timeout <time>查詢逾時結果的保留時間
hold refused <time>REFUSED 結果的保留時間
hold other <time>其他錯誤的保留時間
accepted_payload_size <n>DNS 回應最大大小（預設 512）
timeout retry <time>重試間隔（預設 1s）
 
# DNS 動態解析完整範例
resolvers mydns
    nameserver dns1 8.8.8.8:53
    nameserver dns2 8.8.4.4:53
    parse-resolv-conf           # 也讀取系統 /etc/resolv.conf
    resolve_retries 3
    timeout retry 1s
    hold valid 30s
    hold nx 10s
    accepted_payload_size 8192  # 支援較大的 DNS 回應

backend dynamic-app
    balance roundrobin

    # 單台 server — FQDN 自動解析更新
    server app1 app.service.consul:8080 check resolvers mydns resolve-prefer ipv4 init-addr last,libc,none

    # server-template — 自動依 DNS SRV/A 記錄建立多台
    server-template app_ 10 _http._tcp.app.service.consul:8080 check resolvers mydns resolve-prefer ipv4 init-addr none
💡 server-template 與服務發現 server-template <prefix> <count> <fqdn>:<port> 會預先建立 count 個 server slot，隨 DNS 解析結果自動填充。搭配 Consul、Kubernetes CoreDNS 等實現動態服務發現。
★

完整生產範例 — Bind & Server

resolvers k8s_dns
    nameserver coredns 10.96.0.10:53
    accepted_payload_size 8192
    hold valid 10s
    hold nx 5s

frontend https-in
    # ── Bind ──
    bind :80
    bind :443 ssl \
        crt /etc/ssl/certs/wildcard.pem \
        crt-list /etc/haproxy/crt-list.cfg \
        alpn h2,http/1.1 \
        ssl-min-ver TLSv1.2

    # ── Behind AWS NLB — 接受 PROXY Protocol ──
    # bind :443 ssl crt ... accept-proxy

    redirect scheme https if !{ ssl_fc }
    default_backend app

backend app
    balance roundrobin

    # ── default-server 統一設定 ──
    default-server inter 3s fall 3 rise 2 maxconn 500 \
                   ssl verify required ca-file /etc/ssl/certs/ca-bundle.crt \
                   check check-ssl check-sni app.internal \
                   resolvers k8s_dns resolve-prefer ipv4 init-addr last,libc,none

    # ── 動態 DNS 解析 server ──
    server-template app_ 5 app.default.svc.cluster.local:8443

    # ── 靜態 server ──
    server app-legacy 10.0.5.10:8443 weight 50 cookie legacy
    server app-backup 10.0.5.99:8443 backup cookie backup

backend ws
    balance source
    timeout tunnel 1h

    default-server check inter 5s send-proxy-v2

    server ws1 10.0.6.1:8080
    server ws2 10.0.6.2:8080
