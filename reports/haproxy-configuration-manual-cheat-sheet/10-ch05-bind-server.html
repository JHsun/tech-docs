Bind 與 Server 選項

前端監聽設定、後端伺服器參數、SSL/TLS 與 DNS 解析

5.1Bind 選項（前端監聽）

# 基本語法
bind [<addr>]:<port> [param...]

# 範例
bind  *:80                                          # 所有介面 TCP 80
bind  *:443 ssl crt /etc/ssl/site.pem              # HTTPS
bind  *:443 ssl crt /etc/ssl/certs/               # 目錄下所有憑證（SNI）
bind  unix@/var/run/haproxy.sock mode 660          # Unix socket
bind  quic4@:443 ssl crt /etc/ssl/site.pem        # QUIC/HTTP3
bind  *:8443 ssl crt /etc/ssl/site.pem alpn h2,http/1.1  # 指定 ALPN

常用 Bind 參數

參數說明
ssl啟用 SSL/TLS 終止
crt <path>憑證（PEM 檔或目錄）。PEM 含 cert + key [+ chain]
crt-list <file>憑證清單檔（支援 SNI 對應不同憑證）
ca-file <path>CA 憑證（用於驗證客戶端憑證 mTLS）
verify [none|optional|required]客戶端憑證驗證模式
alpn <list>ALPN 協商清單，如 h2,http/1.1
ssl-min-ver <ver>最低 TLS 版本：TLSv1.2, TLSv1.3
ssl-max-ver <ver>最高 TLS 版本
ciphers <list>TLS 1.2 加密套件
ciphersuites <list>TLS 1.3 加密套件
maxconn <n>此 listener 的最大連線數
accept-proxy接受 PROXY Protocol v1/v2
accept-netscaler-cip接受 NetScaler Client IP 插入
defer-acceptLinux TCP_DEFER_ACCEPT（等到有資料才 accept）
backlog <n>listen backlog 大小
tfo啟用 TCP Fast Open
interface <name>綁定到特定網卡
namespace <name>Linux network namespace
shards <n>socket 分片數（搭配 SO_REUSEPORT）
no-tls-tickets停用 TLS session tickets
generate-certificates動態產生 SSL 憑證（SSL bumping）
strict-sni未匹配 SNI 的連線直接拒絕

mTLS 雙向認證範例

frontend fe_mtls
    bind *:443 ssl \
        crt     /etc/ssl/server.pem \
        ca-file /etc/ssl/ca.pem \
        verify  required \
        crl-file /etc/ssl/crl.pem

    # 取得客戶端憑證資訊
    http-request set-header X-SSL-Client-DN     %[ssl_c_s_dn]
    http-request set-header X-SSL-Client-Verify %[ssl_c_verify]

5.2Server 與 default-server 選項

# 基本語法
server <name> <addr>:<port> [param...]
default-server [param...]       # 設定此 backend 所有 server 的預設值
server-template <prefix> <num> <fqdn>:<port> [param...]  # 動態 server 模板

常用 Server 參數

參數說明
check啟用健康檢查
inter <t>檢查間隔（預設 2s）
fastinter <t>狀態變化中的加速檢查間隔
downinter <t>server DOWN 時的檢查間隔
fall <n>連續失敗 n 次標記為 DOWN（預設 3）
rise <n>連續成功 n 次標記為 UP（預設 2）
weight <0-256>負載權重（預設 1）。0 = 不接收新連線
maxconn <n>此 server 的最大連線數
maxqueue <n>佇列中等待此 server 的最大請求數
minconn <n>搭配 fullconn，動態調整的最小連線數
backup備援伺服器（僅當主 server 全部 DOWN 才啟用）
disabled初始為停用狀態
slowstart <t>恢復後逐漸增加權重的時間
on-error <action>檢查失敗時的動作：fastinter / fail-check / sudden-death / mark-down
on-marked-down <action>標記 DOWN 時的動作：shutdown-sessions
error-limit <n>連續錯誤數上限，觸發 on-error
observe <layer>被動觀測：layer4（連線失敗）/ layer7（5xx 回應）

Server SSL 連線（後端加密）

參數說明
ssl以 SSL/TLS 連接後端
verify [none|required]驗證後端憑證（預設 none）
ca-file <path>驗證後端憑證的 CA
sni <expr>動態設定 SNI，如 sni req.hdr(Host)
alpn <list>ALPN 協商
check-ssl健康檢查也走 SSL
check-sni <name>健康檢查的 SNI
crt <path>客戶端憑證（mTLS 到後端）
backend secure_upstream
    default-server ssl verify required ca-file /etc/ssl/ca-bundle.crt \
                   check check-ssl inter 5s
    server  s1  backend1.example.com:443 sni str(backend1.example.com)
    server  s2  backend2.example.com:443 sni str(backend2.example.com)

PROXY Protocol（傳遞真實 IP）

參數說明
send-proxy傳送 PROXY Protocol v1 到後端
send-proxy-v2傳送 PROXY Protocol v2（二進位格式）
send-proxy-v2-sslv2 含 SSL 連線資訊
send-proxy-v2-ssl-cnv2 含客戶端憑證 CN

5.3Server DNS 解析

HAProxy 可動態解析 server 的 FQDN，實現服務發現和自動更新後端 IP：

resolvers mydns
    nameserver  dns1  8.8.8.8:53
    nameserver  dns2  8.8.4.4:53
    resolve_retries  3
    timeout  resolve  1s
    timeout  retry    1s
    hold     valid    10s     # 解析結果有效期
    hold     other    30s
    hold     refused  30s
    hold     nx       30s
    hold     timeout  30s
    hold     obsolete 30s

backend dynamic_backend
    balance  roundrobin
    server-template srv  5  _http._tcp.myservice.consul  resolvers mydns \
        resolve-prefer ipv4 check init-addr none
Server 解析參數說明
resolvers <name>指定 DNS resolver 區塊
resolve-prefer <ipv4|ipv6>偏好的 IP 版本
resolve-net <cidr>只接受特定網段的解析結果
resolve-opts <opts>如 allow-dup-ip, prevent-dup-ip
init-addr <methods>啟動時 IP 初始化方式：last, libc, none
Consul / SRV 服務發現
使用 server-template + SRV 記錄可實現自動化服務發現。server-template srv 5 表示最多 5 個動態 server 實例，DNS 回應幾筆就啟用幾台。
