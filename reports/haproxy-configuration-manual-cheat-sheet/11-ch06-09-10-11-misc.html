6Cache（內建 HTTP 快取）

HAProxy 3.0 內建輕量級 HTTP 快取，可在記憶體中快取靜態回應，減少後端負擔。

限制
快取僅支援 GET/HEAD 且回應包含有效 Cache-Control / Expires 的內容。不支援 Vary 以外的條件快取（3.0 支援 Vary + process-vary）。最大物件大小預設 256 KB。 
# Step 1: 定義 cache 區塊
cache my_cache
    total-max-size  100          # MB（共享記憶體）
    max-object-size 1048576     # 單一物件上限 bytes (1MB)
    max-age         3600        # 預設最大快取秒數
    max-secondary-entries 10   # Vary 變體數上限
    process-vary    on          # 啟用 Vary header 處理

# Step 2: 在 proxy 中啟用
frontend fe_main
    bind *:80
    http-request cache-use my_cache        # 請求階段查快取
    http-response cache-store my_cache     # 回應階段存快取
    default_backend web
快取命中判斷
可用 fetch res.cache_hit（0/1）和 res.cache_name 來判斷是否命中快取，加入 log 或 Header 以便除錯。

9Supported Filters（過濾器）

9.1 Trace

除錯用，追蹤 proxy 內部事件：

filter trace name my_trace random-parsing random-forwarding hexdump

9.2 HTTP Compression（壓縮）

defaults
    compression algo  gzip           # 或 deflate, raw-deflate, identity
    compression type  text/html text/css text/javascript application/json application/javascript
    compression offload               # 移除後端的 Accept-Encoding，由 HAProxy 壓縮
指令說明
compression algo <list>壓縮演算法：gzip, deflate, raw-deflate, identity
compression type <list>要壓縮的 MIME 類型清單
compression offloadHAProxy 負責壓縮，後端送原始資料
compression direction <req|res|both>壓縮方向（3.0 新增 request 壓縮支援）
效能注意
壓縮會消耗 CPU。可用 maxcompcpuusage <%>（global）限制壓縮的 CPU 使用上限；maxcomprate <kbps> 限制壓縮速率。高流量場景建議由後端或 CDN 處理壓縮。

9.3 SPOE（Stream Processing Offload Engine）

將請求/回應處理外包給外部 agent（如 ModSecurity、自訂反詐騙引擎）：

frontend fe_main
    filter spoe engine modsecurity config /etc/haproxy/spoe-modsecurity.conf
    # SPOE agent 會收到請求資訊並回傳決策變數

9.7 Bandwidth Limitation（頻寬限制）

frontend fe_main
    filter bwlim-in  limit_upload   default-limit 1m default-period 1s
    filter bwlim-out limit_download default-limit 10m default-period 1s

    http-request set-bandwidth-limit limit_upload
    http-response set-bandwidth-limit limit_download
Filter用途
filter trace除錯追蹤
filter compressionHTTP 壓縮（通常由 compression 指令隱式啟用）
filter spoe外部處理引擎整合
filter cache快取（通常由 cache-use/cache-store 隱式啟用）
filter fcgi-appFastCGI 應用
filter opentracing分散式追蹤（OpenTracing / Jaeger）
filter bwlim-in/out上傳/下載頻寬限制

10FastCGI 應用

HAProxy 可直接做為 FastCGI gateway，無需 Nginx 中繼。適合 PHP-FPM 等場景：

fcgi-app php-fpm
    docroot      /var/www/html
    path-info    ^(/.+\.php)(/.*)$
    index        index.php
    pass-header  Authorization

    acl  is_php  path_end .php
    set-param  SCRIPT_FILENAME  %[req.hdr(Host),lower,map_str(/etc/haproxy/docroot.map,/var/www/html)]%[path]

backend php_backend
    mode  http
    use-fcgi-app  php-fpm
    server  fpm1  127.0.0.1:9000  proto fcgi check
FastCGI 限制
HAProxy 的 FastCGI 僅支援 responder 角色（不支援 authorizer/filter）。不支援 FastCGI multiplexing（每連線單一 request）。Server 需指定 proto fcgi。

11位址格式

11.1 位址族前綴 (Address Family)

前綴說明範例
ipv4@強制 IPv4ipv4@0.0.0.0:80
ipv6@強制 IPv6ipv6@:::80
unix@Unix domain socketunix@/var/run/app.sock
abns@抽象 namespace socket（Linux）abns@haproxy-stats
fd@繼承的 file descriptorfd@${FD_NUM}
sockpair@socketpair（master-worker 通訊）
quic4@QUIC over IPv4quic4@:443
quic6@QUIC over IPv6quic6@:443

11.2 Socket 類型前綴

前綴說明
（預設）TCP stream socket
dgram@UDP datagram（僅用於 log-forward 等場景）

11.3 Protocol 前綴

前綴說明
（預設）raw TCP / HTTP（依 mode）
proto fcgiFastCGI 協議（於 server 設定）
# 實用位址範例
bind  *:80                           # 所有介面 IPv4+IPv6 TCP 80
bind  ipv4@*:80                      # 僅 IPv4
bind  :::80                           # IPv6 任意位址
bind  unix@/var/run/haproxy.sock      # Unix socket
bind  quic4@:443 ssl crt cert.pem  # HTTP/3 QUIC
server s1 unix@/var/run/app.sock      # 後端用 Unix socket
