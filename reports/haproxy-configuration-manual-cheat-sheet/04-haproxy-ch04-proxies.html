4.0

Proxy 類型總覽

Section角色可用指令
defaults [name]為後續 proxy 設定預設值；可出現多次mode, timeout, option, log…
frontend <name>面向 client 的監聽端；定義 bind、ACL、路由bind, acl, use_backend, http-request…
backend <name>面向 server 的後端；定義 server、balanceserver, balance, cookie, stick-table…
listen <name>合併 frontend + backend以上全部
💡 defaults 的範圍一個 defaults 段的設定持續生效直到下一個 defaults 段出現。命名的 defaults myname 可被 proxy 用 from myname 明確引用。
核心

mode / bind / server / default_backend

關鍵字適用說明
mode http|tcpD F B Lhttp：完整 HTTP 解析（L7）
tcp：純 TCP 轉發（L4）
bind <addr>:<port> [opts]F L監聽位址。可加 ssl crt … 等選項
server <name> <addr>:<port> [opts]B L後端 server 定義
server-template <prefix> <n> <fqdn>:<port> [opts]B L根據 DNS 動態產生 n 台 server
default_backend <name>F無 ACL 匹配時的預設後端
default-server [opts]B L為該段所有 server 設定預設參數
use_backend <name> if|unless <cond>F L條件式路由到指定 backend
log globalD F B L繼承 global 段的 log 設定
maxconn <n>F L此 frontend 的最大併發連線

D=defaults F=frontend B=backend L=listen

TIM

Timeout 完整速查

Timeout適用說明建議值
timeout connectD B連線到 server 的最大等待時間5s
timeout clientD Fclient 端閒置逾時30s – 60s
timeout serverD Bserver 端閒置逾時30s – 60s
timeout http-requestD F等待完整 HTTP request 的最大時間（防慢速攻擊）5s – 10s
timeout http-keep-aliveD Fkeep-alive 連線上等待下一筆 request5s
timeout queueD Brequest 在佇列中的最大等待時間= connect
timeout tunnelD F BWebSocket / CONNECT 隧道的閒置逾時1h
timeout client-finD Fclient 半關閉後的逾時5s
timeout server-finD Bserver 半關閉後的逾時5s
timeout tarpitD Ftarpit 延遲持續時間—
timeout checkD Bhealth check 的額外逾時—
⚠ 必須設定的 TimeoutHAProxy 啟動時若未設定 timeout connect、timeout client、timeout server 會發出警告。TCP 模式建議 client 與 server 設為相同值。
BAL

Balance 負載均衡演算法

演算法說明適用場景
roundrobin加權輪詢（權重可動態調整）預設通用；支援 slowstart
static-rr靜態輪詢（不支援動態權重）不需 slowstart 時效能更高
leastconn最少連線數優先（加權）長連線（DB、LDAP、SSH tunnel）
first按 server ID 順序，填滿一台再下一台省電、容器自動擴縮
source來源 IP 雜湊（同 IP 固定到同 server）無 cookie 的簡易 session persistence
uriURI 或 URI 前半段雜湊快取代理（同 URI 路由到同節點）
url_param <p>URL 參數值雜湊依 session ID 參數分流
hdr(<name>)HTTP header 值雜湊依 Host / User-Agent 分流
rdp-cookie(<n>)RDP cookie 雜湊RDP Gateway 環境
random隨機加權選擇大量 server 時比 roundrobin 更均勻
random(<draws>)隨機 + 取最少連線（Power of Two）兼顧隨機性與連線均衡
 
backend app
    balance leastconn
    server s1 10.0.0.1:80 weight 100 check
    server s2 10.0.0.2:80 weight 50 check   # 權重為 s1 的一半
ACL

ACL 路由速查

# ACL 定義語法
acl <name> <criterion> [flags] [operator] <value> ...

# 常用 criteria
acl is_api     path_beg /api/
acl is_static  path_end .css .js .png .jpg
acl is_host_a  hdr(host) -i app-a.example.com
acl is_post    method POST
acl is_local   src 192.168.0.0/16 10.0.0.0/8

# 路由
use_backend api_servers   if is_api
use_backend static_cache  if is_static
use_backend app_a         if is_host_a
default_backend            app_default

常用 ACL 比對旗標

旗標說明
-i不區分大小寫
-f <file>從檔案載入值清單
-m <type>強制匹配方法：str、beg、end、sub、reg、found、ip
-u對 ACL 值做 unique 去重

RES

http-response Actions 速查

與 http-request 類似但作用於 server 回應。常用：

http-response set-header X-Frame-Options SAMEORIGIN
http-response set-header Strict-Transport-Security "max-age=31536000; includeSubDomains"
http-response del-header Server
http-response del-header X-Powered-By
http-response add-header X-Cache-Status %[res.cache_hit]
http-response deny status 502 if bad_response      # 阻擋不良回應
http-response replace-header Set-Cookie "(.*)" "\1; Secure; HttpOnly"

redirect 速查

形式說明範例
redirect scheme httpsHTTP → HTTPS 重導redirect scheme https if !{ ssl_fc }

STK

Stick Tables（會話持續 & 計數器）

backend app
    # 定義 stick-table
    stick-table type ip size 200k expire 30m \
        store conn_cur,conn_rate(10s),http_req_rate(10s),http_err_rate(10s)

    # session persistence — 來源 IP
    stick on src

    # Rate limiting — 阻擋超過 50 req/10s 的 IP
    http-request track-sc0 src
    http-request deny deny_status 429 if { sc_http_req_rate(0) gt 50 }

stick-table type 可用鍵值

ip、ipv6、integer、string [len <n>]、binary [len <n>]

可儲存的計數器 (store)

conn_cnt、conn_cur、conn_rate(<period>)、sess_cnt、sess_rate、http_req_cnt、http_req_rate、http_err_cnt、http_err_rate、bytes_in_rate、bytes_out_rate、gpc0、gpc0_rate、gpc1、gpt0

CHK

Health Check 速查

關鍵字說明
server … check啟用 TCP 連線 check
server … check inter <t>check 間隔（預設 2s）
server … check fall <n>連續失敗 n 次後標為 DOWN（預設 3）
server … check rise <n>連續成功 n 次後恢復 UP（預設 2）
option httpchk [method uri ver]啟用 HTTP Layer 7 check
http-check send meth GET uri /healthz新式 HTTP check 語法（3.0 推薦）
http-check expect status 200預期回應狀態碼
http-check expect string "OK"預期回應 body 含指定字串
option mysql-checkMySQL 協定 check
option pgsql-checkPostgreSQL 協定 check
option redis-checkRedis 協定 check
option ldap-checkLDAP 協定 check
 
backend api
    option httpchk
    http-check send meth GET uri /healthz hdr Host api.internal
    http-check expect status 200

    default-server inter 3s fall 3 rise 2
    server api1 10.0.1.1:8080 check
    server api2 10.0.1.2:8080 check

PER

Cookie-based Session Persistence

backend app
    balance roundrobin
    cookie SRVID insert indirect nocache httponly secure
    server s1 10.0.0.1:80 check cookie s1
    server s2 10.0.0.2:80 check cookie s2
💡 cookie 選項 insert：HAProxy 插入 cookie。 indirect：不將 cookie 傳給 server。 nocache：阻止上游 cache 儲存含 cookie 的回應。 httponly / secure：增強安全性標記。

★

生產級完整設定範例

defaults web_defaults
    mode http
    log global
    option httplog
    option dontlognull
    option http-server-close
    option forwardfor except 127.0.0.0/8
    timeout connect 5s
    timeout client  30s
    timeout server  30s
    timeout http-request 10s
    timeout http-keep-alive 5s
    timeout queue 25s
    compression algo gzip
    compression type text/html text/css application/json application/javascript

# ── HTTPS Frontend ──
frontend https-in
    bind :443 ssl crt /etc/ssl/certs/site.pem alpn h2,http/1.1
    bind :80
    redirect scheme https if !{ ssl_fc }

    # Security headers
    http-response set-header Strict-Transport-Security "max-age=63072000"
    http-response set-header X-Frame-Options DENY
    http-response del-header Server

    # ACL routing
    acl is_api     path_beg /api/
    acl is_ws      hdr(Upgrade) -i websocket
    acl is_static  path_end .css .js .png .jpg .woff2

    use_backend ws_backend     if is_ws
    use_backend api_backend    if is_api
    use_backend static_cache   if is_static
    default_backend            app_backend

# ── API Backend ──
backend api_backend
    balance leastconn
    option httpchk
    http-check send meth GET uri /healthz
    http-check expect status 200

    # Rate limiting
    stick-table type ip size 100k expire 30s store http_req_rate(10s)
    http-request track-sc0 src
    http-request deny deny_status 429 if { sc_http_req_rate(0) gt 100 }

    default-server inter 3s fall 3 rise 2 maxconn 200
    server api1 10.0.1.1:8080 check
    server api2 10.0.1.2:8080 check

# ── App Backend (session persistence) ──
backend app_backend
    balance roundrobin
    cookie SRVID insert indirect nocache httponly secure
    default-server inter 5s fall 3 rise 2
    server app1 10.0.2.1:8080 check cookie app1
    server app2 10.0.2.2:8080 check cookie app2
    server app3 10.0.2.3:8080 check cookie app3 backup

# ── WebSocket Backend ──
backend ws_backend
    balance source
    timeout tunnel 1h
    server ws1 10.0.3.1:8080 check
    server ws2 10.0.3.2:8080 check

# ── Static Cache ──
backend static_cache
    balance roundrobin
    server static1 10.0.4.1:80 check

# ── Stats ──
listen stats
    bind :8404
    mode http
    stats enable
    stats uri /stats
    stats auth admin:SecureP@ss
    stats refresh 10s
    stats admin if TRUE
