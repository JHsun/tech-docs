3.1行程管理與安全

daemon
以背景行程（daemon）執行。生產必備。
master-worker
啟用 master-worker 模式。master 監控 worker，支援不中斷重載 (reload)。推薦搭配 systemd。
nbthread <n>
工作執行緒數。預設自動偵測 CPU 數量。建議設為可用 CPU 核心數。
maxconn <n>
全域最大同時連線數。每條連線消耗約 17 KB（含 buffer），請依記憶體估算。
user <name> / group <name>
指定 worker 行程降權後的執行身分。搭配 chroot 使用。
chroot <path>
將 worker chroot 到指定目錄，限制檔案系統存取。
log <addr> [len] <facility> [level]
全域日誌輸出目標。可指定多條。
stats socket <path> [mode] [level]
Runtime API 的 Unix socket 路徑。level admin 允許全功能管理。
stats timeout <time>
stats socket 連線閒置超時。預設 10s。
pidfile <path>
PID 檔路徑，搭配 init system 使用。
hard-stop-after <time>
軟重啟後，舊 worker 存活的最長時間。超時強制終止。
ssl-default-bind-ciphers / ssl-default-bind-ciphersuites
所有 bind 的 TLS 1.2 / 1.3 預設加密套件。
ssl-default-bind-options
如 no-sslv3 no-tlsv10 no-tlsv11。統一設定所有前端 SSL。
ssl-dh-param-file <path>
自訂 DH 參數檔路徑（至少 2048 bit）。
cluster-secret <secret>
叢集間共享密鑰，用於 stick-table 同步和 PROXY Protocol v2 TLV 驗證。
global
    daemon
    master-worker
    nbthread          4
    maxconn           50000
    chroot            /var/lib/haproxy
    user              haproxy
    group             haproxy
    log               /dev/log  local0  info
    log               /dev/log  local1  notice
    stats socket      /run/haproxy/admin.sock mode 660 level admin
    stats timeout     30s
    pidfile           /run/haproxy.pid
    hard-stop-after   60s

    # SSL 全域預設
    ssl-default-bind-ciphers      ECDHE+AESGCM:DHE+AESGCM
    ssl-default-bind-ciphersuites TLS_AES_128_GCM_SHA256:TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256
    ssl-default-bind-options      prefer-client-ciphers no-sslv3 no-tlsv10 no-tlsv11

3.2效能調校

參數預設說明
maxconn2000全域最大連線數
maxconnrate不限每秒最大新建連線數
maxsessrate不限每秒最大新建 session 數
maxsslconn不限同時 SSL 連線上限
maxsslrate不限每秒最大 SSL 握手數
maxpipes自動splice 使用的 pipe 上限
spread-checks <0-50>0健康檢查時間隨機散佈百分比
max-spread-checks <time>無檢查散佈的最大延遲
tune.bufsize16384每個 buffer 大小（bytes）。影響大 header 處理
tune.http.logurilen1024log 中 URI 最大長度
tune.http.maxhdr101每筆請求/回應最大 header 數
tune.maxrewrite1024header 重寫預留空間
tune.ssl.default-dh-param2048未指定 DH 檔時的 DH 位元數
tune.ssl.maxrecord0(自動)SSL record 最大大小
tune.ssl.cachesize20000SSL session cache 條目數
記憶體估算公式
每條連線 ≈ 2 × tune.bufsize + 固定開銷 ≈ 34 KB（預設 buffer 16 KB）
例如 maxconn 50000 ≈ 50000 × 34 KB ≈ 1.6 GB 記憶體

3.3除錯

debug
前台執行並輸出所有連線/斷線資訊。僅限除錯，勿用於生產。
quiet
啟動時抑制大部分輸出訊息。
profiling.tasks <auto|on|off>
任務層級的 CPU 耗時分析。可透過 Runtime API show profiling 查看。
profiling.memory <on|off>
記憶體配置追蹤。
set-dumpable
允許產生 core dump（需搭配 ulimit -c）。

3.4Userlists（使用者清單）

用於 HTTP Basic/Digest 認證：

userlist myusers
    group  admins  users alice,bob
    group  devs    users charlie

    user   alice   insecure-password secret123  groups admins
    user   bob     password $6$rounds=5000$salt$hash...  groups admins
    user   charlie insecure-password dev456

# 在 frontend/backend 中使用
frontend fe_admin
    acl  auth_ok  http_auth(myusers)
    http-request auth  unless auth_ok
安全提示
insecure-password 明文儲存密碼，僅適用於測試。生產環境請使用 password + SHA-256/SHA-512 雜湊。可用 mkpasswd -m sha-512 產生。

3.5Peers（對等節點）

用於多台 HAProxy 之間同步 stick-table 資料（如 session 黏著、速率限制計數器）：

peers mycluster
    bind  *:10000
    server  haproxy1  10.0.0.1:10000
    server  haproxy2  10.0.0.2:10000

backend app
    stick-table type ip size 200k expire 30m peers mycluster
    stick on src

3.6Mailers（郵件通知）

mailers alert_mailers
    mailer  smtp1  smtp.example.com:587

backend app
    email-alert mailers  alert_mailers
    email-alert from     haproxy@example.com
    email-alert to       ops@example.com
    email-alert level    alert

3.7–3.12其他全域子區塊

區塊用途關鍵指令
program 由 master process 管理的外部程式 command, user, group
http-errors 集中定義自訂錯誤頁面，供多個 proxy 引用 errorfile <code> <file>
ring 記憶體環形 buffer，用於日誌的本地快取 size, backing-file, server（轉發目標）
log-forward 接收 syslog 並轉發到其他目標（如 ring） bind, log, dgram-bind
httpclient.* HAProxy 內部 HTTP client 的調校（如 OCSP、Lua） httpclient.ssl.ca-file, httpclient.timeout.connect
crt-store 3.0 新增：集中管理憑證的載入路徑與選項 crt-base, key-base, load + crt 
# http-errors 範例：集中定義錯誤頁
http-errors custom_errors
    errorfile 400  /etc/haproxy/errors/400.http
    errorfile 403  /etc/haproxy/errors/403.http
    errorfile 500  /etc/haproxy/errors/500.http
    errorfile 502  /etc/haproxy/errors/502.http
    errorfile 503  /etc/haproxy/errors/503.http

defaults
    errorfiles  custom_errors  # 引用上方的錯誤頁集合
Ring 日誌架構
ring 可搭配 log-forward 實現：接收 syslog → 暫存 ring buffer → 轉發到遠端 log server。適合高流量環境減少日誌丟失。
