Chapter 8

Logging 日誌

日誌等級、格式、時間欄位、連線狀態 與 自訂 log

8.1Log 等級

等級數值說明
emerg0系統不可用
alert1需立即處理
crit2嚴重狀況
err3錯誤
warning4警告
notice5正常但值得注意
info6資訊性訊息（預設 access log 等級）
debug7除錯訊息
global
    log  /dev/log  local0  info       # facility=local0, 最高記錄到 info
    log  /dev/log  local1  notice     # 第二組日誌

defaults
    log  global                         # 繼承 global 的日誌設定

8.2日誌格式

預設格式

格式啟用方式適用
Default（不設定 option）最簡單，僅基本連線資訊
TCP logoption tcplogL4 模式，含時間、來源、server 資訊
HTTP logoption httplogL7 模式，含 method、URI、status
HTTPS logoption httpslog同 HTTP log 再加 SSL 資訊
Error log自動（錯誤時）連線錯誤的詳細資訊
Customlog-format <string>完全自訂

8.5連線終止狀態 (Termination State)

日誌中的 4 字元旗標，例如 ---- 或 cD--：

位置含義常見值
第 1 字元 第一個事件（誰發起結束） C = 客戶端中止, S = 伺服器中止, P = HAProxy 主動關閉, R = 資源不足, I = 內部錯誤, D = 因安全規則拒絕, - = 正常
第 2 字元 TCP/會話層狀態 R = 請求階段中止, Q = 佇列中中止, C = 連線階段中止, H = 回應 Header 階段中止, D = 資料傳輸中中止, L = 傳輸完最後資料後中止, T = 因 tarpit 中止, - = 正常完成
第 3 字元 Persistence cookie N = 無 cookie, I = 不合法, D = cookie 對應的 server DOWN, V = 有效, - = 不使用
第 4 字元 Persistence cookie 處理 N = 未設 cookie, I = 已插入, U = 已更新, P = 已加前綴, D = 已刪除, - = 無動作
常見組合
---- = 正常完成 | CD-- = 客戶端在資料傳輸中斷線 | cD-- = 客戶端提前關閉 | sH-- = server 在回應 Header 階段錯誤 | PR-- = HAProxy 在請求階段拒絕（如 ACL deny）| SC-- = server 連線失敗

8.2.6自訂 log-format

defaults
    log-format "%ci:%cp [%tr] %ft %b/%s %TR/%Tw/%Tc/%Tr/%Ta %ST %B %CC %CS %tsc %ac/%fc/%bc/%sc/%rc %sq/%bq %hr %hs %{+Q}r"

常用 log-format 變數

變數說明
%ci / %cp客戶端 IP / Port
%fi / %fp前端 IP / Port
%si / %sp後端 server IP / Port
%bi / %bp後端連線 IP / Port
%ftfrontend 名稱（含 ~ssl 標記）
%b / %sbackend / server 名稱
%t / %tr / %Taccept 時間 / 請求時間 / Unix timestamp
%TR請求 Header 接收時間 (ms)
%Tw佇列等待時間 (ms)
%Tc後端連線時間 (ms)
%Tr回應時間 (ms)
%Ta / %Tt活躍時間 / 總時間 (ms)
%STHTTP 狀態碼
%B / %U傳送 bytes / 接收 bytes
%CC / %CS捕獲的 request / response cookie
%tsc終止狀態碼（4 字元）
%ac / %fc / %bc / %sc / %rc活躍/前端/後端/server/重試 連線數
%sq / %bqserver / backend 佇列長度
%hr / %hs捕獲的 request / response Header
%r / %HM / %HP / %HVHTTP 請求行 / method / path / version
%{+Q}r帶引號的 HTTP 請求行
%IDunique-id
%sslc / %sslvSSL cipher / SSL version
%[var(txn.xxx)]自訂變數（任意 sample fetch 均可用）

JSON 日誌格式範例

log-format '{"timestamp":"%t","client":"%ci:%cp","frontend":"%ft","backend":"%b/%s","status":%ST,"bytes":%B,"timers":{"req":%TR,"queue":%Tw,"conn":%Tc,"resp":%Tr,"total":%Ta},"method":"%HM","path":"%HP","termination":"%tsc"}'

8.3進階日誌選項

選項說明
option dontlognull不記錄未送資料就斷開的連線（如健康檢查探測）
option dontlog-normal只記錄異常連線（重試、錯誤、超時等）
option logasap收到 server 回應 Header 後立即記錄（不等資料傳完）
option log-separate-errors將錯誤日誌提升等級到 err（分離到不同 facility）
option log-health-checks記錄健康檢查狀態變化
option http-ignore-probes完全忽略不像 HTTP 的連線探測（不記錄、不回應）
unique-id-format <fmt>定義 unique ID 格式
unique-id-header <name>將 unique ID 放入指定 Header
frontend fe_main
    unique-id-format  %{+X}o_%ci_%cp_%fi_%fp_%Ts_%rt:%pid
    unique-id-header  X-Request-ID
    capture request header  Host           len 40
    capture request header  User-Agent     len 100
    capture response header Content-Type   len 30

8.7–8.8捕獲 Cookie 與 Header

# 捕獲 Cookie（出現在 %CC / %CS）
capture cookie  JSESSIONID=  len 32

# 捕獲 Header（出現在 %hr / %hs）
capture request header   X-Forwarded-For  len 45
capture response header  X-Cache           len 10

# 以 declare capture + action 動態捕獲
declare capture request  len 64         # slot 0
http-request capture req.hdr(Authorization) id 0
日誌分析建議
配合 option httplog + 自訂 log-format 輸出 JSON，可直接對接 ELK / Loki / Splunk 等日誌平台。用 unique-id-header 實現跨服務的請求追蹤串聯。
