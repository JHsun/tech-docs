Chapter 7

ACL 與 Sample Fetching

條件匹配、資料擷取、轉換器 — HAProxy 路由與決策的核心

7.1ACL 基礎

# ACL 語法
acl <name> <criterion> [flags] [operator] <value> ...

# 範例
acl  is_api       path_beg      /api/
acl  is_post      method        POST
acl  host_blog   hdr(Host) -i blog.example.com
acl  rate_abuse  sc_http_req_rate(0) gt 100

匹配類型

類型匹配方法Flag / 運算子
Boolean 有值 = true / 無值 = false 直接使用，如 { ssl_fc }
Integer 數值比較 eq ne lt le gt ge，範圍 1:100
String 字串匹配 -i (不分大小寫), -f <file> (從檔案讀), -m <method>
Regex 正規表達式 -m reg，-i 可加
IP IP / CIDR 匹配 -f <file> 可載入 IP 清單

字串匹配方法 (-m)

方法說明對應速記 fetch
-m str完全匹配path
-m beg前綴匹配path_beg
-m end後綴匹配path_end
-m sub子字串匹配path_sub
-m dir目錄匹配（以 / 分隔）path_dir
-m dom域名匹配（以 . 分隔）hdr_dom
-m reg正規表達式path_reg
-m len字串長度
-m found是否存在

7.2ACL 條件組合

# 邏輯運算
use_backend api  if  is_api is_post        # AND（空格 = 且）
use_backend api  if  is_api || is_post    # OR
use_backend api  if  !is_static              # NOT
use_backend api  if  is_api !is_blocked || is_admin

# 匿名 (inline) ACL — 大括號語法
use_backend api  if  { path_beg /api/ } { method POST PUT }
http-request deny  if  { src 10.0.0.0/8 } !{ src 10.0.1.0/24 }
優先順序
!（NOT）最高 → 空格（AND）→ ||（OR）。建議用命名 ACL 保持可讀性，inline ACL 適合簡單條件。

7.3Sample Fetching（資料擷取）

Sample fetch 是 HAProxy 的資料擷取引擎，可用於 ACL、log-format、set-header、set-var 等幾乎所有需要動態值的地方。

7.3.1 Converters（轉換器）

轉換器以逗號串接在 fetch 後方，形成處理管線：

# 語法：fetch,converter1,converter2,...
%[req.hdr(Host),lower,map_str(/etc/haproxy/hosts.map,default_backend)]

# 常用轉換器範例
%[src,ipmask(24)]                          # IP 遮罩
%[date,ltime(%Y-%m-%d)]                    # 時間格式化
%[req.hdr(X-Token),sha1,hex]               # SHA1 + hex 編碼
%[path,regsub(^/old/,/new/)]               # 正規替換
轉換器說明
lower / upper轉小寫 / 大寫
base64 / b64decBase64 編碼 / 解碼
hex / hex2iHex 編碼 / hex→整數
jsonJSON 跳脫字串
sha1 / sha2(<bits>)SHA 雜湊
crc32 / djb2 / sdbm雜湊函式（回傳整數）
map_str(<file>[,<default>])字串→字串 map 查詢
map_ip / map_int / map_dom / map_beg / map_end / map_reg不同匹配方式的 map 查詢
regsub(<re>,<sub>[,<flags>])正規表達式替換
field(<delim>,<n>)以分隔符取第 n 段
word(<delim>,<n>)以分隔符取第 n 個字
concat(<pfx>,<sfx>,<delim>)串接前後綴
ipmask(<bits>)IP 遮罩運算
ltime(<fmt>) / utime(<fmt>)Unix timestamp → 本地/UTC 時間字串
add(<n>) / sub / mul / div / mod整數算術
and(<n>) / or(<n>) / neg位元運算 / 邏輯否定
nbsrv取得指定 backend 的活躍 server 數
set-var(<scope>.<name>)作為 converter 使用，存入變數
iif(<true>,<false>)三元運算（boolean → 字串）
strcmp(<str>)字串比較（回傳 -1/0/1）
digest(<algo>)指定演算法的 digest
hmac(<algo>,<key>)HMAC 計算
aes_gcm_enc / aes_gcm_decAES-GCM 加密/解密
protobuf(<field>)解析 gRPC/Protobuf 欄位
json_query(<path>)JSON 路徑查詢
jwt_header_query / jwt_payload_queryJWT 解析

實戰範例：多條件路由

frontend fe_main
    bind *:443 ssl crt /etc/ssl/certs/

    # 以 SNI / Host 路由
    acl  host_api    hdr(Host) -i  api.example.com
    acl  host_web    hdr(Host) -i  www.example.com

    # 以路徑路由
    acl  path_ws     path_beg  /ws/

    # 以 Header 路由
    acl  is_grpc     hdr(Content-Type) -i  application/grpc

    # 速率限制
    stick-table type ip size 100k expire 30s store http_req_rate(10s)
    http-request track-sc0 src
    acl  rate_limit  sc_http_req_rate(0) gt 50
    http-request deny deny_status 429 if rate_limit

    # Map-based 路由（動態，可 runtime 更新）
    use_backend  %[req.hdr(Host),lower,map_str(/etc/haproxy/hosts.map)]

    # 條件路由
    use_backend  be_websocket  if  path_ws
    use_backend  be_grpc       if  is_grpc
    use_backend  be_api        if  host_api
    default_backend  be_web
Map 檔案格式
/etc/haproxy/hosts.map：每行 key value（空格分隔）。可透過 Runtime API 動態更新：
echo "add map /etc/haproxy/hosts.map new.example.com be_new" | socat stdio /run/haproxy/admin.sock
