Proxies 代理設定

frontend / backend / listen / defaults — 核心關鍵字與 Actions 速查

4.0Proxy 區塊類型

區塊角色何時用
defaults為後續 proxy 設定預設值統一 timeout、mode、log 等
frontend <name>接收客戶端連線（bind）定義監聽位址、ACL 路由
backend <name>伺服器群、負載均衡定義 server、balance 策略
listen <name>frontend + backend 合一簡單場景如 stats、TCP proxy

4.1核心 Proxy 關鍵字速查

模式與路由

mode <http|tcp>
defaultsfrontendbackendlisten
運作模式。http = L7 完整 HTTP 解析；tcp = L4 純 TCP 轉發。
default_backend <name>
defaultsfrontend
未命中任何 use_backend 規則時的預設後端。
use_backend <name> [if|unless <cond>]
frontend
依條件路由到指定後端。規則由上而下匹配。
frontend http_in
    bind  *:80
    mode  http

    acl  is_api     path_beg /api/
    acl  is_static  path_end .css .js .png .jpg

    use_backend  api_servers     if is_api
    use_backend  static_servers  if is_static
    default_backend  web_servers

負載均衡 (balance)

演算法說明適用場景
roundrobin加權輪詢（動態權重，預設）通用，最常用
static-rr靜態輪詢（不支援動態權重調整）server 數極大時
leastconn最少連線優先長連線（DB、WebSocket）
source來源 IP 雜湊簡易 IP 黏著
uriURI 雜湊快取伺服器
url_param <p>URL 參數雜湊如 ?sessionid=xxx
hdr(<name>)指定 Header 雜湊如 hdr(X-Tenant-ID)
rdp-cookie(<n>)RDP cookie 雜湊遠端桌面
random隨機（加權）大量 server 均勻分佈
random(<draws>)power-of-N-choices 隨機兼顧均勻與低延遲
backend cache_servers
    balance  uri             # 同一 URI 固定到同一台
    hash-type  consistent    # 一致性雜湊，減少 server 增減的影響
    server  cache1  10.0.0.1:80  check
    server  cache2  10.0.0.2:80  check

連線管理與超時

關鍵字適用說明
timeout connect <t>B/D連到後端的 TCP 握手超時
timeout client <t>F/D前端連線閒置超時
timeout server <t>B/D後端連線閒置超時
timeout tunnel <t>B/DWebSocket / CONNECT 隧道超時
timeout http-request <t>F/D完整接收 HTTP 請求的超時（防 Slowloris）
timeout http-keep-alive <t>F/Dkeep-alive 間隙的閒置超時
timeout queue <t>B/D請求在佇列中等候 server 的最長時間
timeout check <t>B健康檢查的額外等待時間
retries <n>B/D連線失敗的重試次數（預設 3）
retry-on <conditions>B/DL7 重試條件，如 conn-failure empty-response 0rtt-rejected

F = frontend, B = backend, D = defaults

HTTP 連線模式

選項前端→後端說明
option http-keep-alivekeep-alive ↔ keep-alive預設。兩端都維持長連線
option http-server-closekeep-alive ↔ close後端連線用完即關。適合不支援 keep-alive 的後端
option httpcloseclose ↔ close兩端都用短連線
http-reuse <mode>safe（預設）/ always / never。控制後端連線重用策略

健康檢查 (Health Check)

backend web
    # HTTP 健康檢查
    option httpchk
    http-check send meth GET uri /health ver HTTP/1.1 hdr Host www.example.com
    http-check expect status 200

    server  s1  10.0.0.1:80  check inter 5s fall 3 rise 2
    # inter = 檢查間隔  fall = 連續失敗幾次標記 DOWN  rise = 連續成功幾次標記 UP
檢查類型關鍵字用途
TCP 預設check僅測試 TCP 連線是否成功
HTTPoption httpchk送 HTTP 請求判斷狀態碼/body
MySQLoption mysql-checkMySQL 協議握手
PostgreSQLoption pgsql-checkPgSQL 協議握手
Redisoption redis-checkRedis PING/PONG
LDAPoption ldap-checkLDAP bind
SMTPoption smtpchkSMTP EHLO
Agentagent-check外部 agent 回報狀態/權重

Session Persistence（黏著）

backend app
    balance  roundrobin

    # 方法 1：Cookie-based persistence
    cookie  SRVID  insert indirect nocache httponly secure
    server  s1  10.0.0.1:80  check cookie s1
    server  s2  10.0.0.2:80  check cookie s2

    # 方法 2：Stick-table persistence
    stick-table  type ip size 200k expire 30m
    stick on     src
Cookie 選項說明
insertHAProxy 自動插入 cookie
indirect回傳時移除 HAProxy 的 cookie（對應用透明）
nocache加 Cache-Control: private 避免快取
httponly設定 HttpOnly flag
secure設定 Secure flag（僅 HTTPS）
maxidle <t>cookie 閒置最長時間
maxlife <t>cookie 最大生命週期
dynamic動態 cookie（搭配 dynamic-cookie-key）

4.3Actions 動作速查

http-request actions（前端/後端請求階段）

Action說明範例
deny [status <n>]拒絕請求，回傳錯誤碼http-request deny status 403 if bad_ip
allow允許請求（停止後續 deny 規則）http-request allow if whitelist
redirectHTTP 重導向http-request redirect scheme https if !{ ssl_fc }
set-header <n> <v>設定/覆蓋 Headerhttp-request set-header X-Forwarded-Proto https
add-header <n> <v>新增 Header（不覆蓋）http-request add-header X-Request-ID %[unique-id]
del-header <n>刪除 Headerhttp-request del-header X-Powered-By
replace-header <n> <re> <v>正規表達式替換http-request replace-header Cookie foo=([^;]*) foo=xxx
set-path <v>改寫 URI pathhttp-request set-path /v2%[path]
set-uri <v>改寫整個 URI
set-query <v>改寫查詢字串
set-method <m>改寫 HTTP Method
set-var(<scope>.<n>) <expr>設定變數http-request set-var(txn.host) req.hdr(Host)
set-src <expr>覆寫來源 IP（如取自 X-Forwarded-For）http-request set-src hdr(X-Real-IP)
auth [realm <r>]要求 HTTP Basic Auth
tarpit [status <n>]延遲回應（消耗攻擊者資源）
silent-drop靜默丟棄（不回應，不送 RST）
return <status> [content-type <t>] [body <b>]自訂回應http-request return status 200 content-type "application/json" string '{"ok":true}'
do-resolve(<var>,<resolver>) <expr>動態 DNS 解析
track-sc<n> <key> [table <t>]追蹤 stick-table 計數器http-request track-sc0 src
sc-inc-gpc0(<n>)遞增 stick-table 通用計數器
use-service <svc>使用 Lua 或內建服務http-request use-service lua.myhandler

http-response actions（回應階段）

Action說明
set-header / add-header / del-header / replace-header同 request，但作用於回應 Header
set-status <n> [reason <r>]改寫回應狀態碼
allow / deny放行或阻擋回應
set-var / set-var-fmt設定變數供後續 log 或規則使用

http-after-response actions（回應完成後）

在回應 Header 即將送出前最後修改。適合加安全 Header：

frontend fe_main
    http-after-response set-header Strict-Transport-Security "max-age=31536000; includeSubDomains"
    http-after-response set-header X-Content-Type-Options    nosniff
    http-after-response set-header X-Frame-Options            DENY
    http-after-response del-header Server

redirect 完整語法

# 重導向到 HTTPS
redirect scheme https code 301 if !{ ssl_fc }

# 重導向到不同路徑
redirect prefix /new-site code 302

# 重導向到完整 URL
redirect location https://new.example.com/ code 301

# append-slash 範例
redirect code 301 prefix / if { path_reg ^/[^/.]+$ } append-slash

常用 Proxy 選項速查

選項說明
option forwardfor [header <n>]自動加入 X-Forwarded-For Header
option httplog使用 HTTP 格式記錄日誌（含 URI、狀態碼等）
option httpslog同上，額外記錄 SSL 資訊
option dontlognull不記錄未送資料就斷開的連線
option http-buffer-request完整緩衝請求 body 後才轉發（ACL 需要 body 時用）
option abortonclose客戶端斷線時，中止佇列中的等待請求
option redispatchsession persistence 失敗時，重新分配到其他 server
option log-health-checks記錄健康檢查狀態變化
option tcp-smart-connect延遲 SYN，直到有資料要送時才建立連線
compression algo gzip啟用 gzip 壓縮（搭配 compression type）

Stick-Table 速查

backend rate_limit
    stick-table type ip size 100k expire 30s store http_req_rate(10s),conn_cur,gpc0
    http-request track-sc0 src
    http-request deny deny_status 429 if { sc_http_req_rate(0) gt 100 }
table type說明
ipIPv4 位址
ipv6IPv6 位址
integer整數
string [len <n>]字串（可限長度）
binary [len <n>]二進位
store 計數器說明
conn_cnt連線總數
conn_cur當前連線數
conn_rate(<period>)連線速率
http_req_cntHTTP 請求總數
http_req_rate(<period>)HTTP 請求速率
http_err_cnt / http_err_rateHTTP 錯誤計數/速率
bytes_in_rate / bytes_out_rate流量速率
gpc0 / gpc1 / gpc_rate通用計數器
gpt0通用標記
速率限制實戰模板
上方範例實現了：追蹤每個來源 IP 在 10 秒內的 HTTP 請求數；超過 100 次即返回 429 Too Many Requests。結合 gpc0 可進一步實現黑名單機制。
