3.1

Process 管理與安全

關鍵字說明範例
daemon以背景 daemon 模式執行daemon
master-worker啟用 master-worker 模式：master 負責管理，worker 負責流量master-worker
nbthread <n>設定 worker 執行緒數（取代已廢棄的 nbproc）nbthread 4
user <name>啟動後切換至指定使用者（需以 root 啟動）user haproxy
group <name>啟動後切換至指定群組group haproxy
chroot <dir>chroot 至指定目錄（安全隔離）chroot /var/lib/haproxy
pidfile <path>PID 檔路徑pidfile /run/haproxy.pid
hard-stop-after <time>soft-stop (SIGUSR1) 後的最長存活時間hard-stop-after 30s
close-spread-time <time>soft-stop 時將連線關閉動作分散在此時間內close-spread-time 10s
mworker-max-reloads <n>master-worker 模式中舊 worker 最大存留 reload 次數mworker-max-reloads 3
node <name>設定節點名稱（HA 環境中辨識用）node lb-prod-01
description <text>描述文字（顯示於 stats）description "Production LB"

SSL / TLS 全域預設

關鍵字說明
ca-base <dir>CA 憑證搜尋基底路徑
crt-base <dir>Server 憑證搜尋基底路徑
ssl-default-bind-ciphers <list>前端預設 TLSv1.2 cipher
ssl-default-bind-ciphersuites <list>前端預設 TLSv1.3 ciphersuite
ssl-default-bind-options <opts>前端預設 SSL 選項（如 no-sslv3 no-tlsv10）
ssl-default-server-*同上，但套用於後端連線
ssl-dh-param-file <path>全域 DH 參數檔
ssl-security-level <n>OpenSSL security level (0–5)

Stats Socket（Runtime API）

關鍵字說明
stats socket <addr> [opts]啟用 Unix/TCP 管理 socket
stats timeout <time>管理連線逾時（預設 10s）
stats maxconn <n>管理 socket 最大併發連線（預設 10）
 
# 實務範例 — 生產環境 global 段
global
    daemon
    master-worker
    nbthread 4
    user haproxy
    group haproxy
    chroot /var/lib/haproxy
    pidfile /run/haproxy.pid
    hard-stop-after 60s

    # SSL
    ca-base /etc/ssl/certs
    crt-base /etc/ssl/private
    ssl-default-bind-ciphers ECDHE+AESGCM:ECDHE+CHACHA20
    ssl-default-bind-ciphersuites TLS_AES_128_GCM_SHA256:TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256
    ssl-default-bind-options prefer-client-ciphers no-sslv3 no-tlsv10 no-tlsv11

    # Runtime API
    stats socket /run/haproxy-admin.sock mode 660 level admin
    stats timeout 30s

    # Logging
    log /dev/log local0 info
    log /dev/log local1 notice
3.2

效能調校 (Performance Tuning)

關鍵字說明預設 / 建議
maxconn <n>全域最大併發連線數依記憶體計算；等同 -n 參數
maxconnrate <n>每秒最大新建連線數保護用，需搭配 frontend 限制
maxsessrate <n>每秒最大新 session 數—
maxsslconn <n>全域最大 SSL 連線數—
maxsslrate <n>每秒最大 SSL 握手數—
maxpipes <n>splice 用的 pipe 上限—
maxzlibmem <n>zlib 壓縮用的最大記憶體 (MB)—
maxcompcpuusage <n>壓縮使用的最大 CPU %100
spread-checks <0-50>分散 health check 的隨機變化百分比0
max-spread-checks <time>分散 check 的最大延遲上限—

tune.* 微調參數（常用）

關鍵字說明預設
tune.bufsize <n>緩衝區大小（bytes）— 影響 header 容量16384
tune.maxrewrite <n>保留給 header 重寫的空間bufsize / 2
tune.http.maxhdr <n>最大 HTTP header 數量101
tune.http.logurilen <n>日誌中 URI 的最大長度1024
tune.ssl.default-dh-param <n>DH 參數位元數2048
tune.ssl.cachesize <n>SSL session cache 條目數20000
tune.ssl.lifetime <time>SSL session cache 存活時間300s
tune.ssl.maxrecord <n>SSL record 最大大小16384
tune.comp.maxlevel <0-9>壓縮最高等級1
tune.idle-pool.shared <on|off>跨執行緒共享閒置連線on
 
# 效能調校範例
global
    maxconn 50000
    spread-checks 5

    tune.bufsize 32768         # 處理超長 header / cookie
    tune.ssl.default-dh-param 2048
    tune.http.maxhdr 200       # 微服務可能帶很多 header
3.3

除錯 (Debugging)

關鍵字說明
debug啟用除錯模式（前景、詳細輸出）勿用於生產
quiet安靜模式，抑制輸出
profiling.tasks <on|auto|off>啟用任務 profiling
profiling.memory <on|off>啟用記憶體使用分析
set-dumpable允許 core dump 產生
anonkey <key>匿名化 dump 用的 key（保護敏感資訊）
3.4

Userlists（使用者清單）

定義 HTTP Basic Auth 使用者帳號，供 http-request auth 或 stats 頁面認證使用。

userlist admin_users
    group admin users admin1,admin2
    user admin1 insecure-password MyP@ss1
    user admin2 password $6$rounds=5000$salt$hash...

# 在 frontend/backend 中引用
http-request auth unless { http_auth(admin_users) }

# Stats 認證
stats auth admin1:MyP@ss1
💡 密碼格式 insecure-password 為明文（測試用）；password 接受 crypt(3) 格式雜湊（生產用）。
3.5

Peers（跨節點同步）

在多台 HAProxy 之間同步 stick-table 狀態（如 session persistence、rate limiting 計數器），實現 HA 下的狀態共享。

peers mycluster
    peer lb1 192.168.1.10:1024
    peer lb2 192.168.1.11:1024

backend app
    stick-table type ip size 200k expire 30m peers mycluster
    stick on src
💡 要點 每台 HAProxy 的 peer 名稱必須與該機器的 localpeer（預設為 hostname）一致。可用 -L 命令列參數覆蓋。
3.6

Mailers（郵件通知）

定義 SMTP 伺服器，搭配 email-alert 於 server 狀態變更時發送通知信。

mailers smtp_servers
    mailer smtp1 smtp.example.com:587

backend app
    email-alert mailers smtp_servers
    email-alert from haproxy@example.com
    email-alert to ops@example.com
    email-alert level alert
    server s1 10.0.0.1:80 check
3.7

Programs（外部程式管理）

讓 HAProxy master process 管理外部程式的生命週期（啟動、監控、重啟），常用於 dataplane API 或 sidecar。

program dataplane-api
    command /usr/bin/dataplaneapi --host 0.0.0.0 --port 5555
    user haproxy
    group haproxy
3.8

HTTP-errors（錯誤頁面集）

定義可重用的錯誤頁面集合，供多個 proxy 引用。

http-errors custom_errors
    errorfile 400 /etc/haproxy/errors/400.http
    errorfile 403 /etc/haproxy/errors/403.http
    errorfile 503 /etc/haproxy/errors/503.http

frontend http-in
    errorfiles custom_errors
💡 errorfile 格式 檔案必須是完整的 HTTP response（含 status line + headers + body），以 .http 結尾。Content-Length 需正確。
3.9

Rings（記憶體環型緩衝區）

定義記憶體中的 ring buffer，用於存放 log 事件，可透過 CLI show events 讀取。適合除錯或低延遲日誌收集。

ring mylog
    description "In-memory log buffer"
    size 4m                    # 4 MB

global
    log ring@mylog local0 info # 將 log 寫入 ring

透過 Runtime API 讀取：show events mylog

3.10

Log Forwarding（日誌轉發）

HAProxy 可接收 syslog 訊息（UDP/TCP）並轉發至其他目的地，兼具 syslog relay 功能。

log-forward sylog-relay
    bind udp4@:1514          # 接收 UDP syslog
    log 10.0.0.5:514 local0  # 轉發目的地
    maxconn 1000
3.11

HTTPClient 調校

HAProxy 內部的 HTTP client 用於 OCSP 更新、Lua 發出的 HTTP 請求等。

關鍵字說明
httpclient.resolvers.id <id>指定 DNS resolver
httpclient.resolvers.prefer <ipv4|ipv6>DNS 解析偏好
httpclient.ssl.ca-file <path>CA 憑證路徑
httpclient.ssl.verify <none|required>是否驗證 server 憑證
httpclient.timeout.connect <time>連線逾時
httpclient.retries <n>重試次數
3.12

Certificate Storage（憑證管理）

HAProxy 3.0 引入了集中式憑證管理區段 crt-store，允許預先載入和組織 SSL 憑證。

3.12.1 Load Options

關鍵字說明
crt-base <dir>憑證路徑基底（在 crt-store 內）
key-base <dir>私鑰路徑基底
load crt <path> [opts]載入憑證（可指定 key、ocsp、issuer 等選項）
 
# crt-store 範例
crt-store my_certs
    crt-base /etc/ssl/certs
    key-base /etc/ssl/private
    load crt example.com.pem key example.com.key

# 在 bind 中引用
frontend https-in
    bind :443 ssl crt example.com.pem
