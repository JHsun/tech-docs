2.1

設定檔格式 (Configuration File Format)

HAProxy 設定來自三個來源（優先順序由高到低）：

#來源說明
1命令列參數永遠優先；如 -n、-N 會覆蓋設定檔值
2設定檔支援多檔載入：haproxy -f a.cfg -f b.cfg
3環境變數設定檔中以 "$VAR" 引用

檔案基本語法規則

規則說明
語句一行為一個語句（非空行且在 # 註解之前的部分）
斷詞以空格或 Tab 分隔 token（word）
第一個 word即為關鍵字；後續 word 為其參數
註解未被引號保護的 # 之後全為註解
縮排純粹美觀用途；常見做法：section 名稱靠左，內容縮排 2–4 空格

四大 Section 類型

Section用途
global全域參數：process 管理、安全、效能調校
defaults後續 proxy 的預設值；可出現多次，後者覆蓋前者
frontend面向 client 端的監聽設定
backend面向 server 端的後端設定
listen結合 frontend + backend 的快捷寫法
 
# 等價寫法
global                       global
    daemon                       daemon

frontend foo                 # section 關鍵字靠左
    mode http                # 內容縮排
2.2

引號與跳脫 (Quoting & Escaping)

HAProxy 設定支援三種字串保護機制：

① 反斜線跳脫（Escaping）

語法效果
\ （反斜線+空格）將空格視為值的一部分，非分隔符
\#將 # 視為字元，非註解
\\輸出一個反斜線
\'輸出一個單引號
\"輸出一個雙引號
\n換行字元 LF (0x0a)
\r歸位字元 CR (0x0d)
\tTab 字元 (0x09)
\xNN十六進位 ASCII 碼，如 \x0a

② 弱引號 — 雙引號 "…"

防止空格、單引號、# 被解析為特殊字元。允許環境變數 $VAR 展開。若需要字面 $，需寫成 \$。

③ 強引號 — 單引號 '…'

引號內完全不做解析（不展開變數、不解析跳脫）。適合包含正規表示式等特殊字元的場景。

# 跳脫範例
http-request set-header X-Msg hello\ world     # 空格跳脫
http-request set-header X-Msg "hello world"    # 弱引號
acl is_foo path_reg -i '^/foo[0-9]+'           # 強引號包 regex
⚠ 巢狀引號 — 兩層跳脫陷阱 某些關鍵字的參數在括號內（如 regsub()），此時括號內的 ) 需要額外跳脫。
內層使用與外層不同的引號型別，或明確反斜線跳脫，避免語法錯誤。
2.3

環境變數 (Environment Variables)

環境變數僅在雙引號內被展開，解析發生在設定檔載入時。

語法說明範例
"$VAR"基本展開bind "$BIND_ADDR":80
"${VAR}"大括號包裹（與 Shell 相同）"${HAPROXY_USER}"
"${VAR-default}"未設定時使用預設值"${PORT-8080}"
"${VAR[*]}"空格分隔值展開為多個參數"${SERVER_LIST[*]}"

HAProxy 內建環境變數

變數說明可用 / 可改
.FILE目前設定檔路徑✓ / ✗
.LINE目前行號✓ / ✗
.SECTION目前 section 名稱✓ / ✗
HAPROXY_LOCALPEER本機 peer 名稱✗ / ✗（初始化後設定）
HAPROXY_CFGFILES載入的設定檔清單✗ / ✗
HAPROXY_MWORKER是否為 master-worker 模式✗ / ✗
HAPROXY_CLICLI socket 位址✗ / ✗
HAPROXY_MASTER_CLIMaster CLI 位址✗ / ✗

Global 段相關指令

指令用途
setenv <VAR> <value>設定變數（已存在則報錯）
presetenv <VAR> <value>僅在變數不存在時設定
resetenv <VAR> [...]清除所有變數，僅保留列出的
unsetenv <VAR> [...]刪除指定變數
 
# 使用範例
global
    setenv MYAPP_PORT 8080

frontend http-in
    bind :"$MYAPP_PORT"
2.4

條件區塊 (Conditional Blocks)

類似 C 預處理器的條件編譯，可嵌套使用，在設定檔載入時求值。用於根據功能、版本、環境變數動態啟用/停用設定。

四個指令

指令行為
.if <condition>開啟新的巢狀層級
.elif <condition>同層級的替代條件（可連續多個）
.else同層級的否則區塊（每個 .if 最多一個）
.endif關閉當前層級

Condition 求值規則

值結果
空字串 ""→ false
整數 0→ false
非零整數 1→ true
predicate 函式見下方表格
! (NOT)邏輯否定
&& (AND)左到右短路求值；優先權高於 OR
|| (OR)右到左短路求值

Predicate 函式速查

Predicate說明
defined(<VAR>)環境變數是否存在（不論值）
feature(<NAME>)功能是否編譯啟用（對應 haproxy -vv 的 +NAME）
streq(<s1>,<s2>)兩字串相等
strneq(<s1>,<s2>)兩字串不相等
strstr(<s1>,<s2>)s1 是否包含 s2
version_atleast(<ver>)HAProxy 版本 ≥ ver
version_before(<ver>)HAProxy 版本 < ver
enabled(<opt>)執行期選項是否啟用（EPOLL, SPLICE, REUSEPORT…）

診斷輸出指令

指令層級行為
.diag "msg"DEBUG僅 -dD 診斷模式下輸出
.notice "msg"NOTICE一般通知
.warning "msg"WARNING啟用 zero-warning 時會導致啟動失敗
.alert "msg"ALERT永遠導致 fatal error
 
# 實務範例 — 條件啟用 SSL
.if feature(OPENSSL) && streq("$WITH_SSL",yes)
    bind :443 ssl crt /etc/ssl/cert.pem
.endif

.if !feature(OPENSSL)
    .alert "此設定需要 OpenSSL 支援"
.endif

# 版本相容性
.if version_atleast(3.0)
    # 3.0+ 新語法
.else
    # 舊版相容語法
.endif

# Master-Worker 條件
.if defined(HAPROXY_MWORKER)
    listen master_cli
        bind :9999
.endif
⚠ 空格陷阱 .if streq("$VAR", 1) — 逗號後的空格會成為第二個參數的一部分！
正確寫法：.if streq("$VAR",1)（不要在逗號後加空格）。
2.5

時間格式 (Time Format)

所有接受時間值的參數（如 timeout），預設單位為毫秒 (ms)，可在數值後加上單位後綴：

後綴單位換算
us微秒1 us = 0.001 ms
ms毫秒 預設—
s秒1s = 1,000 ms
m分鐘1m = 60s = 60,000 ms
h小時1h = 3,600s = 3,600,000 ms
d天1d = 86,400s = 86,400,000 ms
 
# 等價寫法
timeout client 50000       # 50000ms（隱含預設）
timeout client 50000ms     # 明確 ms
timeout client 50s         # 50 秒

# 實務建議：timeout 設為 3 秒的倍數 + 少許
timeout connect 5s         # 覆蓋 1~2 次 TCP 重傳
timeout server  30s
timeout client  30s
2.6

大小格式 (Size Format)

接受大小值的參數（如 tune.bufsize、max-object-size），可使用後綴：

後綴單位換算
（無）bytes 預設—
kkilobytes1k = 1,024 bytes
mmegabytes1m = 1,048,576 bytes
ggigabytes1g = 1,073,741,824 bytes
 
tune.bufsize 16384         # 16384 bytes
tune.bufsize 16k           # 等價
cache mycache
    total-max-size 100m    # 100 MB
2.7

Map & ACL 名稱格式

Map 檔與 ACL 檔可透過兩種方式引用：

方式語法說明
檔案路徑/etc/haproxy/whitelist.map使用檔案系統路徑（載入時讀取）
命名引用@myacl使用 @ 前綴的名稱，可透過 Runtime API 動態更新

同一個檔案若被多處引用，HAProxy 只會載入一次並共享。透過 Runtime API（show acl、add acl、del acl、show map 等）可在不重新載入設定的情況下動態管理。

2.8

最小設定範例

範例 ① — 分離式寫法（frontend + backend）

global
    daemon
    maxconn 256

defaults
    mode http
    timeout connect 5s
    timeout client  50s
    timeout server  50s

frontend http-in
    bind *:80
    default_backend servers

backend servers
    server server1 127.0.0.1:8000 maxconn 32

範例 ② — 合併式寫法（listen）

global
    daemon
    maxconn 256

defaults
    mode http
    timeout connect 5s
    timeout client  50s
    timeout server  50s

listen http-proxy
    bind *:80
    server server1 127.0.0.1:8000 maxconn 32
💡 兩種寫法完全等價 listen 適合簡單的單一用途場景；frontend + backend 分離寫法適合需要多個 frontend 共用 backend、或設定複雜路由規則的場景。

範例 ③ — 搭配條件區塊與環境變數

global
    daemon

defaults
    mode http
    timeout connect 5s
    timeout client  30s
    timeout server  30s

frontend http-in
    bind :"${PORT-80}"

    .if feature(OPENSSL) && defined(SSL_CERT)
    bind :"${SSL_PORT-443}" ssl crt "$SSL_CERT"
    .endif

    default_backend app

backend app
    server app1 "${APP_HOST-127.0.0.1}":"${APP_PORT-8080}" check
