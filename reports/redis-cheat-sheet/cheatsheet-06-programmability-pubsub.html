Lua 腳本 (EVAL)

命令說明
EVAL script numkeys [key ...] [arg ...]執行 Lua 腳本
EVALSHA sha1 numkeys [key ...] [arg ...]依 SHA1 執行已快取腳本
EVALRO / EVALSHA_RO唯讀模式執行腳本（Redis 7+）
SCRIPT LOAD script載入腳本並回傳 SHA1
SCRIPT EXISTS sha1 [...]檢查腳本是否已快取
SCRIPT FLUSH [ASYNC]清除腳本快取
SCRIPT KILL終止執行中的腳本
 
# 基本語法
EVAL "return redis.call('GET', KEYS[1])" 1 mykey

# 條件更新（原子 CAS）
EVAL "
  local val = redis.call('GET', KEYS[1])
  if val == ARGV[1] then
    redis.call('SET', KEYS[1], ARGV[2])
    return 1
  end
  return 0
" 1 mykey "old_val" "new_val"

# 安全釋放分散式鎖
EVAL "
  if redis.call('GET',KEYS[1]) == ARGV[1] then
    return redis.call('DEL',KEYS[1])
  else return 0 end
" 1 lock:resource $my_token

# Lua API
redis.call('CMD', ...)     -- 執行命令，錯誤時中止腳本
redis.pcall('CMD', ...)    -- 執行命令，錯誤時回傳錯誤物件
redis.log(redis.LOG_WARNING, 'msg')  -- 寫入日誌
redis.sha1hex('string')    -- 計算 SHA1
KEYS[n]                     -- 第 n 個 key 參數
ARGV[n]                     -- 第 n 個一般參數
⚠️
重要：Lua 腳本是原子執行的，會阻塞整個伺服器直到完成。預設 5 秒逾時（lua-time-limit）。腳本中所有 key 必須透過 KEYS 傳入，不可動態產生 key 名稱。
⚙️

Redis Functions

Redis 7.0+
命令說明
FUNCTION LOAD [REPLACE] code載入函式庫
FCALL func numkeys [key ...] [arg ...]呼叫函式
FCALL_RO func numkeys ...唯讀模式呼叫
FUNCTION LIST [LIBRARYNAME lib]列出已載入函式庫
FUNCTION DELETE library刪除函式庫
FUNCTION DUMP序列化所有函式庫
FUNCTION RESTORE data還原函式庫
FUNCTION FLUSH [ASYNC]清除所有函式庫
 
# 定義函式庫（Lua 語法）
FUNCTION LOAD "#!lua name=mylib

local function my_hset(keys, args)
  local hash = keys[1]
  local field = args[1]
  local value = args[2]
  local current = redis.call('HGET', hash, field)
  if current == false then
    return redis.call('HSET', hash, field, value)
  end
  return 0
end

redis.register_function('my_hset_nx', my_hset)
"

# 呼叫函式
FCALL my_hset_nx 1 user:1001 name "Alice"
✅
Functions vs EVAL：Functions 是伺服器的一等公民，持久化且隨複製傳播；EVAL 腳本是臨時的客戶端程式碼。新專案推薦使用 Functions。
📢

Pub/Sub（發布/訂閱）

命令說明
頻道模式
SUBSCRIBE ch [ch ...]訂閱指定頻道
UNSUBSCRIBE [ch ...]取消訂閱
PUBLISH ch message發布訊息至頻道（回傳接收者數）
模式匹配
PSUBSCRIBE pattern [...]依模式訂閱（*, ?, [ae]）
PUNSUBSCRIBE [pattern ...]取消模式訂閱
Sharded Pub/Sub（Redis 7+）
SSUBSCRIBE shardch [...]訂閱 shard 頻道（Cluster 友好）
SUNSUBSCRIBE [shardch ...]取消 shard 訂閱
SPUBLISH shardch message發布至 shard 頻道
管理
PUBSUB CHANNELS [pattern]列出活躍頻道
PUBSUB NUMSUB [ch ...]各頻道訂閱者數
PUBSUB NUMPAT模式訂閱數
 
# 終端 1：訂閱者
SUBSCRIBE news:tech news:sports

# 終端 2：發布者
PUBLISH news:tech "Redis 8.0 released!"

# 模式訂閱
PSUBSCRIBE news:*    # 匹配 news:tech, news:sports 等
PSUBSCRIBE order:*:status  # 匹配 order:123:status

# Python 範例
pubsub = r.pubsub()
pubsub.subscribe('events')
for msg in pubsub.listen():
    if msg['type'] == 'message':
        process(msg['data'])
⚠️
Pub/Sub 特性：Fire-and-forget（無持久化、無重試）。訂閱者離線時錯過的訊息會永久遺失。若需可靠訊息傳遞，請改用 Stream。
💡
Sharded Pub/Sub（Redis 7+）將頻道分配到特定 slot，僅在該 shard 內傳播，適合 Cluster 環境。
🔔

Keyspace 通知

# 啟用通知（預設關閉）
CONFIG SET notify-keyspace-events KEA
# K=keyspace事件, E=keyevent事件, A=所有命令
# g=通用命令, $=String, l=List, s=Set, z=Sorted Set
# h=Hash, x=過期事件, e=驅逐事件, m=miss事件

# 監聽 key 過期事件
PSUBSCRIBE "__keyevent@0__:expired"
# 收到通知：channel=__keyevent@0__:expired, message=expired_key_name

# 監聽特定 key 的所有事件
SUBSCRIBE "__keyspace@0__:mykey"
# SET mykey → 收到 "set"
# DEL mykey → 收到 "del"
💡
常見用途：延遲佇列（key 過期觸發處理）、快取失效通知、資料同步觸發。注意：通知也是 fire-and-forget。
⚖️

Pub/Sub vs Stream 比較

特性Pub/SubStream
持久化❌ 無✅ 有（隨 RDB/AOF）
離線消費❌ 錯過即遺失✅ 可從任意位置讀取
消費者群組❌✅ 支援多群組獨立消費
訊息確認❌✅ XACK 確認機制
歷史回放❌✅ XRANGE 任意回放
延遲最低（即時推送）極低（輪詢或 BLOCK）
適用場景即時通知、聊天室、廣播任務佇列、事件溯源、日誌
