Stream（串流）

訊息日誌
Redis 5.0+ append-only log 消費者群組 訊息確認 (ACK) 類似 Kafka 語義
命令說明
XADD key [MAXLEN|MINID] * field val ...新增訊息（* 自動產生 ID）
XREAD COUNT n BLOCK ms STREAMS key id讀取訊息（可阻塞）。$ 表示僅新訊息
XRANGE key start end [COUNT n]範圍查詢（- 到 + 表全部）
XREVRANGE key end start反向範圍查詢
XLEN key訊息總數
XDEL key id [id ...]刪除特定訊息
XTRIM key MAXLEN|MINID [~] n修剪串流長度
XINFO STREAM key串流資訊
消費者群組命令說明
XGROUP CREATE key group id建立消費者群組。$ 從最新，0 從頭
XREADGROUP GROUP g consumer COUNT n STREAMS key >以消費者身分讀取未消費訊息
XACK key group id [id ...]確認已處理訊息
XPENDING key group [- + count]查看未確認訊息
XCLAIM key group consumer min-idle id轉移逾時未確認訊息的所有權
XAUTOCLAIM key group consumer min-idle start自動認領閒置訊息
XINFO GROUPS key列出消費者群組
 
# 生產者：新增事件
XADD events:orders * action "create" order_id 1001 amount 599

# 建立消費者群組
XGROUP CREATE events:orders processor-group 0

# 消費者讀取（> 表示尚未投遞的訊息）
XREADGROUP GROUP processor-group worker-1 COUNT 5 BLOCK 2000 STREAMS events:orders >

# 處理完成後確認
XACK events:orders processor-group 1711600000000-0

JSON

Redis Stack
需 RedisJSON 模組 JSONPath 語法 支援巢狀結構
命令說明
JSON.SET key $ '{"k":"v"}'設定 JSON（$ 為根路徑）
JSON.GET key [path]取得 JSON（可指定路徑）
JSON.DEL key [path]刪除路徑
JSON.MGET k1 k2 path批次取得
JSON.NUMINCRBY key path n數值增加
JSON.ARRAPPEND key path val陣列追加元素
JSON.ARRINDEX key path val陣列搜尋
JSON.ARRLEN key path陣列長度
JSON.TYPE key [path]查看 JSON 值型別
JSON.MERGE key path val合併 JSON
 
JSON.SET user:1001 $ '{"name":"Alice","age":28,"tags":["dev"]}'
JSON.GET user:1001 $.name           # → ["Alice"]
JSON.NUMINCRBY user:1001 $.age 1 # → [29]
JSON.ARRAPPEND user:1001 $.tags '"redis"'
🔢

Bitmap & Bitfield

位元運算
基於 String 型別 極省記憶體 適合布林狀態追蹤
命令說明
SETBIT key offset 0|1設定位元
GETBIT key offset取得位元
BITCOUNT key [start end]計數為 1 的位元數
BITOP AND|OR|XOR|NOT dst k1 k2位元運算
BITPOS key 0|1 [start end]找到第一個 0 或 1 的位置
BITFIELD key GET type offset將字串視為整數陣列操作
BITFIELD key SET type offset val設定 bitfield 值
BITFIELD key INCRBY type offset n增加 bitfield 值（支援溢出策略）
 
# 每日簽到追蹤（offset = 用戶 ID）
SETBIT checkin:2024-03-28 1001 1
SETBIT checkin:2024-03-28 1002 1
BITCOUNT checkin:2024-03-28        # → 今日簽到人數

# 連續 7 天都簽到的用戶
BITOP AND result checkin:day1 checkin:day2 ... checkin:day7

HyperLogLog

基數估計
機率型資料結構 誤差 < 1% 固定 ~12 KB 記憶體
命令說明O()
PFADD key elem [elem ...]新增元素O(1)
PFCOUNT key [key ...]估計唯一元素數（多 key 時先合併）O(1)/O(N)
PFMERGE dst src [src ...]合併多個 HLLO(N)
 
# UV 統計（每天獨立訪客）
PFADD uv:2024-03-28 "user:1001" "user:1002" "user:1001"
PFCOUNT uv:2024-03-28              # → 2（去重）

# 合併多日統計
PFMERGE uv:2024-03-week uv:2024-03-25 uv:2024-03-26 uv:2024-03-27
✅
適合「不需精確但需低記憶體」的大規模基數統計，如獨立訪客數、搜尋去重等。1 億個唯一元素僅用 ~12 KB。
🌍

Geospatial（地理空間）

位置查詢
底層為 Sorted Set Geohash 編碼 半徑/矩形搜尋
命令說明
GEOADD key lng lat member [...]新增地理位置
GEOPOS key member [...]取得座標
GEODIST key m1 m2 [unit]計算兩點距離（m/km/mi/ft）
GEOSEARCH key FROMMEMBER|FROMLONLAT BYRADIUS|BYBOX ... ASC|DESC範圍搜尋（Redis 6.2+）
GEOSEARCHSTORE dst src ...搜尋結果存入新 key
GEOHASH key member [...]取得 Geohash 字串
 
# 新增餐廳位置
GEOADD restaurants 121.5654 25.0330 "鼎泰豐" 121.5275 25.0225 "欣葉"

# 搜尋半徑 2 公里內的餐廳
GEOSEARCH restaurants FROMLONLAT 121.55 25.03 BYRADIUS 2 km ASC WITHCOORD WITHDIST

# 兩點距離
GEODIST restaurants "鼎泰豐" "欣葉" km

機率型資料結構

Redis Stack
需 RedisBloom 模組 近似計算 超低記憶體
結構用途主要命令
Bloom Filter 元素是否可能存在（可能有假陽性，無假陰性） BF.ADD, BF.EXISTS, BF.MADD, BF.MEXISTS, BF.RESERVE
Cuckoo Filter 類似 Bloom，支援刪除、計數 CF.ADD, CF.EXISTS, CF.DEL, CF.COUNT
Count-Min Sketch 估計元素出現頻率 CMS.INITBYDIM, CMS.INCRBY, CMS.QUERY
Top-K 估計 Top-K 最頻繁元素 TOPK.RESERVE, TOPK.ADD, TOPK.LIST, TOPK.QUERY
t-digest 估計百分位數（p50、p99 等） TDIGEST.CREATE, TDIGEST.ADD, TDIGEST.QUANTILE, TDIGEST.CDF 
# Bloom Filter：防止快取穿透
BF.RESERVE user:emails 0.001 1000000  # 誤差率 0.1%，容量 100 萬
BF.ADD user:emails "alice@example.com"
BF.EXISTS user:emails "bob@example.com"    # → 0（確定不存在）

# Top-K：即時熱門商品追蹤
TOPK.RESERVE trending:products 10
TOPK.ADD trending:products "product:42" "product:7"
TOPK.LIST trending:products

Vector Set（向量集合）

Redis 8+
原生向量相似度搜尋 HNSW 演算法 餘弦相似度 適合 ML/RAG/語意搜尋
命令說明
VADD key [REDUCE dim] VALUES n v1 v2 ... elem新增向量元素
VSIM key [VALUES n v1 ...| ELE elem] [COUNT n] [WITHSCORES]向量相似度查詢
VDIM key取得向量維度
VCARD key元素數量
VREM key elem移除元素
VEMB key elem取得元素的向量
VSETATTR key elem '{"k":"v"}'設定元素 JSON 屬性
VGETATTR key elem取得元素屬性
VINFO key向量集合資訊
VLINKS key elem查看 HNSW 圖的鄰居
 
# 新增文件向量（3 維簡化範例）
VADD docs VALUES 3 0.1 0.8 0.3 "doc:1"
VADD docs VALUES 3 0.9 0.2 0.5 "doc:2"

# 搜尋最相似的 5 個文件
VSIM docs VALUES 3 0.15 0.75 0.35 COUNT 5 WITHSCORES

# 搭配屬性篩選
VSETATTR docs "doc:1" '{"category":"tech","lang":"zh"}'
VSIM docs ELE "doc:1" COUNT 10 WITHSCORES FILTER '.category == "tech"'

TimeSeries（時間序列）

Redis Stack
需 RedisTimeSeries 模組 自動降採樣 標籤篩選
命令說明
TS.CREATE key [RETENTION ms] [LABELS l v]建立時間序列
TS.ADD key timestamp value新增資料點（* 自動時間戳）
TS.MADD k1 t1 v1 k2 t2 v2 ...批次新增
TS.RANGE key fromTime toTime範圍查詢
TS.MRANGE from to FILTER label=val跨多 key 的範圍查詢
TS.GET key取得最新資料點
TS.CREATERULE src dst AGGREGATION avg 60000建立降採樣規則
TS.INFO key時間序列資訊
 
# IoT 感測器溫度監控
TS.CREATE temp:sensor:1 RETENTION 86400000 LABELS location taipei type temperature
TS.ADD temp:sensor:1 * 25.6
TS.RANGE temp:sensor:1 - + AGGREGATION avg 3600000  # 每小時平均

資料型別選擇指南

需求場景推薦型別原因
簡單快取、計數器String最基礎型別，支援原子遞增
物件/使用者資料Hash 或 JSONHash 適合扁平結構，JSON 適合巢狀
佇列/任務排程List 或 Stream簡單佇列用 List，需 ACK/群組用 Stream
標籤/唯一值/集合運算Set天然去重 + 交並差運算
排行榜/優先佇列Sorted Set自動排序 + 範圍查詢
獨立訪客數/大規模基數HyperLogLog固定 12 KB，<1% 誤差
地理位置搜尋Geospatial內建距離計算與半徑搜尋
布林狀態/簽到Bitmap極省記憶體的位元操作
事件串流/訊息系統Stream持久化日誌 + 消費者群組
元素存在性檢測Bloom Filter空間效率極高的成員檢測
向量相似度搜尋 / AIVector Set 或 Search原生 HNSW 或 FT.SEARCH
時間序列/監控指標TimeSeries自動降採樣 + 標籤篩選
