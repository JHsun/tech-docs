什麼是 Redis？

全名
Remote Dictionary Server
類型
開源、記憶體內（in-memory）資料結構存儲，可作為資料庫、快取、訊息中介
授權
Redis 7.4+ 採用 SSPLv1 + RSALv2 雙授權；之前版本為 BSD-3
預設埠號
6379（Sentinel：26379，Cluster Bus：16379）
核心特點
單執行緒事件迴圈、原子操作、亞毫秒延遲、多種資料結構、可選持久化
典型用途
Session 快取、排行榜、即時分析、速率限制、訊息佇列、Pub/Sub、全文搜尋、向量搜尋
🏗

架構概覽

單執行緒模型

主執行緒處理所有命令，避免鎖競爭。I/O 多工由 epoll/kqueue 驅動。Redis 6+ 支援多執行緒 I/O。

記憶體優先

所有資料存於記憶體，讀寫延遲低於 1ms。可搭配 RDB/AOF 持久化至磁碟。

主從複製

非同步複製，支援一主多從。Replica 可處理讀請求以分散負載。

Cluster 模式

資料自動分片至 16384 個 hash slots，水平擴展。每個 shard 可搭配 replica。

💡
Redis 使用事件驅動的 reactor 模式，所有命令依序執行，天生保證原子性，無需額外加鎖。
📦

Redis CLI 常用操作

# 連線（預設 localhost:6379）
redis-cli
redis-cli -h 192.168.1.100 -p 6380 -a password
redis-cli --tls -h myredis.example.com

# 使用 AUTH 認證（Redis 6+ ACL）
AUTH username password

# 選擇資料庫（預設 0，共 16 個：0-15）
SELECT 2

# 伺服器資訊
INFO                # 全部資訊
INFO server          # 伺服器資訊
INFO memory          # 記憶體使用
INFO replication     # 複製狀態
INFO keyspace        # 各 DB 的 key 統計

# 監控 & 除錯
MONITOR              # 即時觀察所有命令（除錯用，影響效能）
SLOWLOG GET 10       # 檢視最近 10 筆慢查詢
CLIENT LIST          # 列出所有連線中的客戶端
DBSIZE               # 當前 DB 的 key 數量

Key 管理與生命週期

命令說明複雜度
SET key value設定 key 值（可附加 EX/PX/NX/XX）O(1)
GET key取得 key 值O(1)
DEL key [key ...]同步刪除 keyO(N)
UNLINK key [key ...]非同步刪除（非阻塞）O(1)+背景
EXISTS key檢查 key 是否存在O(1)
TYPE key回傳 key 的資料型別O(1)
RENAME key newkey重新命名O(1)
KEYS pattern匹配 key（危險 勿用於生產）O(N)
SCAN cursor [MATCH] [COUNT]漸進式迭代 key（生產安全）O(1)/次
OBJECT ENCODING key查看底層編碼方式O(1)
過期命令說明
EXPIRE key seconds設定 TTL（秒）
PEXPIRE key ms設定 TTL（毫秒）
EXPIREAT key timestamp設定過期 Unix 時間戳
TTL key查詢剩餘 TTL（秒）
-1=無過期，-2=不存在
PTTL key查詢剩餘 TTL（毫秒）
PERSIST key移除過期設定
⚠️
EXPIRE 支援條件選項：NX 僅無 TTL 時設定、XX 僅有 TTL 時更新、GT 新值 > 舊值時更新、LT 新值 < 舊值時更新。
📝

Key 命名最佳實踐

規則
Key 是二進位安全字串，最大 512 MB。建議保持簡潔有意義。
命名慣例
使用 : 分隔命名空間，例：user:1001:session、order:2024:items
避免
過長 key（浪費記憶體）、過短 key（難以理解）、使用 KEYS * 於生產環境
# ✅ 好的命名範例
user:1001:profile
cache:api:products:page:1
session:abc123
queue:email:pending

# ❌ 避免的命名
u1001              # 太短，難以理解
this-is-a-very-very-long-key-name  # 過長
⚙️

基礎配置 (redis.conf)

# 網路
bind 127.0.0.1 -::1    # 綁定介面
port 6379               # 監聽埠號
protected-mode yes      # 保護模式（無密碼時僅接受本地連線）
timeout 0               # 閒置斷線秒數（0=不斷線）
tcp-keepalive 300       # TCP keepalive 間隔

# 記憶體
maxmemory 4gb           # 最大記憶體限制
maxmemory-policy allkeys-lru  # 逐出策略

# 持久化（RDB）
save 900 1              # 900秒內至少1次寫入則快照
save 300 10             # 300秒內至少10次寫入
save 60 10000           # 60秒內至少10000次寫入
dbfilename dump.rdb

# 持久化（AOF）
appendonly yes
appendfsync everysec    # always | everysec | no

# 安全
requirepass your_password
aclfile /etc/redis/users.acl
💡
執行時可用 CONFIG GET * 查看所有設定，CONFIG SET key value 動態修改（部分參數），CONFIG REWRITE 寫回檔案。

Redis 產品生態一覽

Redis Open Source

核心引擎，支援所有基本資料結構。社群版本，可自行部署與管理。

Redis Stack

Redis OSS + 額外模組（JSON、Search、TimeSeries、Bloom Filter 等），適合開發者快速上手。

Redis Cloud

全託管 DBaaS，支援 AWS / GCP / Azure。自動擴展、備份、高可用。提供免費方案。

Redis Software (Enterprise)

自建部署的企業版，支援 Active-Active 跨地域複製、RBAC、進階安全與合規。
