交易 & Pipeline

命令說明
MULTI開始交易（後續命令入隊）
EXEC執行所有入隊命令（原子執行）
DISCARD取消交易（清空命令佇列）
WATCH key [key ...]樂觀鎖：監視 key，若被修改則 EXEC 失敗
UNWATCH取消所有 WATCH
 
# 交易範例：原子轉帳
WATCH balance:alice balance:bob
MULTI
DECRBY balance:alice 100
INCRBY balance:bob 100
EXEC   # 若 WATCH 的 key 被修改，回傳 nil
💡
Pipeline 是客戶端功能，一次送出多個命令減少 RTT。與 MULTI 不同：Pipeline 不保證原子性，但效率更高。兩者可組合使用。
# Pipeline（Python redis-py 範例）
pipe = r.pipeline()
pipe.set("k1", "v1")
pipe.set("k2", "v2")
pipe.get("k1")
results = pipe.execute()  # 一次往返取得所有結果

# Pipeline + Transaction
pipe = r.pipeline(transaction=True)  # 內部自動 MULTI/EXEC

Search & Query（搜尋與查詢）

Redis Stack
命令說明
FT.CREATE idx ON HASH|JSON PREFIX 1 prefix SCHEMA ...建立索引（指定結構與欄位）
FT.SEARCH idx query [RETURN n field ...]全文搜尋/篩選查詢
FT.AGGREGATE idx query [GROUPBY ...] [REDUCE ...]聚合查詢（分組、統計）
FT.DROPINDEX idx [DD]刪除索引（DD=連同資料刪除）
FT.INFO idx索引資訊與統計
FT.TAGVALS idx field列出 TAG 欄位的所有值
FT._LIST列出所有索引
Schema 欄位型別說明
TEXT全文搜尋（分詞、模糊、前綴）
TAG精確匹配/多值標籤
NUMERIC數值範圍查詢
GEO地理位置查詢
VECTOR向量欄位（FLAT/HNSW）
GEOSHAPE幾何形狀查詢
 
# 建立產品索引
FT.CREATE idx:products ON JSON PREFIX 1 product: SCHEMA \
  $.name AS name TEXT WEIGHT 2.0 \
  $.price AS price NUMERIC SORTABLE \
  $.tags AS tags TAG SEPARATOR "," \
  $.location AS location GEO

# 搜尋：名稱含 "laptop" 且價格 500-2000
FT.SEARCH idx:products "@name:laptop @price:[500 2000]" SORTBY price ASC LIMIT 0 10

# TAG 篩選 + 全文搜尋
FT.SEARCH idx:products "@tags:{electronics} @name:(wireless keyboard)"

# 聚合：各標籤的平均價格
FT.AGGREGATE idx:products "*" GROUPBY 1 @tags REDUCE AVG 1 @price AS avg_price

生產環境危險命令

命令風險替代方案
KEYS *遍歷全部 key，阻塞伺服器SCAN 漸進式迭代
FLUSHDB / FLUSHALL清空所有資料ACL 限制 + 加 ASYNC 參數
MONITOR持續監聽所有命令，嚴重影響效能僅限除錯短時間使用
SAVE同步 RDB 快照，完全阻塞BGSAVE 背景執行
DEBUG *除錯命令，可能造成崩潰生產環境禁用
CONFIG SET動態修改配置可能導致問題ACL 限制管理者才能使用
SHUTDOWN關閉伺服器ACL 限制
⚠️
建議透過 rename-command 或 ACL 限制危險命令。例：rename-command FLUSHALL "" 完全禁用。

常用模式速查

# 分散式鎖（SET NX + EX）
SET lock:resource $token NX EX 30
# 釋放鎖（Lua 原子操作確保只刪自己的鎖）
EVAL "if redis.call('get',KEYS[1])==ARGV[1] then return redis.call('del',KEYS[1]) else return 0 end" 1 lock:resource $token

# 速率限制（滑動視窗 - Sorted Set）
ZADD ratelimit:$ip $now $request_id
ZREMRANGEBYSCORE ratelimit:$ip 0 $window_start
ZCARD ratelimit:$ip  # 若 > limit 則拒絕
EXPIRE ratelimit:$ip 60

# 快取模式（Cache-Aside）
data = GET cache:user:$id
if data == nil:
    data = db.query("SELECT ...")
    SET cache:user:$id data EX 300

# 排行榜實時更新
ZINCRBY leaderboard:daily 1 "player:42"
ZREVRANGE leaderboard:daily 0 9 WITHSCORES  # Top 10

# Session 管理
HSET session:$sid user_id 1001 role "admin" ip "10.0.0.1"
EXPIRE session:$sid 1800  # 30 分鐘過期

SCAN 家族 — 生產安全的迭代命令

命令對象語法說明
SCANkey 空間SCAN cursor [MATCH pattern] [COUNT hint] [TYPE type]漸進迭代所有 key
HSCANHash fieldHSCAN key cursor [MATCH] [COUNT]迭代 Hash 的 field-value
SSCANSet 成員SSCAN key cursor [MATCH] [COUNT]迭代 Set 的成員
ZSCANSorted SetZSCAN key cursor [MATCH] [COUNT]迭代 Sorted Set 的 member-score
 
# SCAN 使用：cursor 從 0 開始，回傳 0 時迭代完成
SCAN 0 MATCH "user:*" COUNT 100 TYPE hash
# 回傳: 1) "next_cursor"   2) [key1, key2, ...]
# 持續用 next_cursor 呼叫直到回傳 "0"

# Python 範例：安全迭代所有符合的 key
for key in r.scan_iter(match="cache:*", count=100):
    r.delete(key)
✅
COUNT 是提示而非精確數量。SCAN 保證最終一致性但可能回傳重複 key，客戶端需自行去重。
