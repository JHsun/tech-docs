Search & Query 進階用法

Redis Stack

查詢語法速查

語法說明
word全文搜尋（在所有 TEXT 欄位）
@field:word指定欄位搜尋
@field:(hello world)聯集（OR）搜尋
"exact phrase"精確詞組匹配
hel*前綴匹配
%%word%%模糊搜尋（Levenshtein 距離 2）
@tags:{redis|cache}TAG 欄位匹配（精確，多值用 |）
@price:[100 500]數值範圍（含邊界）
@price:[(100 +inf]數值範圍（不含下界）
@location:[lon lat radius unit]地理半徑搜尋
-@field:word排除條件（NOT）
*匹配所有文件

聚合 (FT.AGGREGATE) 管道

步驟說明
GROUPBY n @field依欄位分組
REDUCE func n @field AS alias聚合函式：COUNT, SUM, AVG, MIN, MAX, TOLIST, QUANTILE, STDDEV, FIRST_VALUE, RANDOM_SAMPLE
SORTBY n @field ASC|DESC排序
APPLY "expr" AS alias計算表達式（數學、字串函式）
FILTER "expr"聚合後篩選
LIMIT offset num分頁
LOAD n @field載入額外欄位
 
# 複雜聚合範例：各城市平均訂單金額 Top 5
FT.AGGREGATE idx:orders "*" \
  GROUPBY 1 @city \
    REDUCE AVG 1 @amount AS avg_amount \
    REDUCE COUNT 0 AS order_count \
  SORTBY 2 @avg_amount DESC \
  LIMIT 0 5

向量搜尋 (Vector Search)

方式一
FT.SEARCH + VECTOR 欄位（Redis Stack）— 在索引中定義 VECTOR 欄位，搭配 KNN 查詢
方式二
Vector Set（Redis 8+ 原生）— 專用 VADD/VSIM 命令，HNSW 演算法，更簡潔的 API
# 方式一：FT.SEARCH 向量搜尋

# 建立向量索引
FT.CREATE idx:docs ON HASH PREFIX 1 doc: SCHEMA \
  title TEXT \
  category TAG \
  embedding VECTOR HNSW 6 TYPE FLOAT32 DIM 768 DISTANCE_METRIC COSINE

# 存入文件（embedding 為二進位向量）
HSET doc:1 title "Redis 入門教學" category "tutorial" \
  embedding "<768-dim float32 binary>"

# KNN 搜尋最相似的 5 個文件
FT.SEARCH idx:docs "*=>[KNN 5 @embedding $vec AS score]" \
  PARAMS 2 vec "<query_vector_binary>" \
  SORTBY score RETURN 3 title category score

# 混合搜尋：向量 + 篩選
FT.SEARCH idx:docs "@category:{tutorial}=>[KNN 5 @embedding $vec AS score]" \
  PARAMS 2 vec "<query_vector>"
💡
HNSW vs FLAT：HNSW 建索引較慢但查詢快（適合大資料集），FLAT 暴力搜尋精確但慢（適合小資料集 <10K）。

RAG 模式與 AI 應用

RAG 是什麼
Retrieval-Augmented Generation — 先從向量資料庫檢索相關文件，再餵給 LLM 生成回答
Redis 角色
作為向量資料庫存儲 embeddings，提供低延遲的相似度檢索
# RAG 流程（偽碼）

# 1. 文件切片 + 向量化（離線）
chunks = split_document(doc)
for chunk in chunks:
    vec = embedding_model.encode(chunk.text)
    redis.hset(f"chunk:{chunk.id}",
        mapping={"text": chunk.text,
                 "source": chunk.source,
                 "embedding": vec.tobytes()})

# 2. 查詢時檢索（線上）
query_vec = embedding_model.encode(user_question)
results = redis.ft("idx:chunks").search(
    Query("*=>[KNN 5 @embedding $vec AS score]")
    .sort_by("score")
    .paging(0, 5)
    .dialect(2),
    query_params={"vec": query_vec.tobytes()})

# 3. 組裝 Prompt 並呼叫 LLM
context = "\n".join([r.text for r in results.docs])
answer = llm.generate(f"根據以下資料回答：\n{context}\n\n問題：{user_question}")

語意快取

快取 LLM 回答，相似問題直接命中快取，節省 API 成本。

Session 記憶

用 Redis 存儲對話歷史，為 LLM 提供上下文。TTL 自動清理過期對話。

即時推薦

用戶向量 + 商品向量的相似度計算，毫秒級回應推薦結果。

快速上手：Python 範例

# pip install redis
import redis

# 基本連線
r = redis.Redis(host='localhost', port=6379,
                decode_responses=True)

# String
r.set('key', 'value', ex=3600)
r.get('key')

# Hash
r.hset('user:1', mapping={'name': 'Alice', 'age': 28})
r.hgetall('user:1')

# Pipeline
pipe = r.pipeline()
pipe.set('a', 1).set('b', 2).get('a')
results = pipe.execute()

# JSON（需 Redis Stack）
r.json().set('doc:1', '$', {'title': 'Hello'})
r.json().get('doc:1', '$.title')

# Pub/Sub
p = r.pubsub()
p.subscribe('events')
r.publish('events', 'hello!')

快速上手：Node.js 範例

// npm install redis
import { createClient } from 'redis';

const client = createClient({ url: 'redis://localhost:6379' });
client.on('error', err => console.log(err));
await client.connect();

// String
await client.set('key', 'value', { EX: 3600 });
const val = await client.get('key');

// Hash
await client.hSet('user:1', { name: 'Alice', age: '28' });
const user = await client.hGetAll('user:1');

// Sorted Set（排行榜）
await client.zAdd('leaderboard', [
  { score: 100, value: 'Alice' },
  { score: 85, value: 'Bob' }
]);
const top = await client.zRangeWithScores(
  'leaderboard', 0, 4, { REV: true });

// JSON（需 Redis Stack）
await client.json.set('doc:1', '$',
  { title: 'Hello', tags: ['redis'] });

// Stream
await client.xAdd('events', '*',
  { action: 'login', userId: '1001' });

await client.disconnect();
