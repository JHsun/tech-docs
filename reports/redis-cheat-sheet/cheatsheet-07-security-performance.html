ACL 存取控制列表

Redis 6.0+
命令說明
ACL LIST列出所有使用者及其 ACL 規則
ACL SETUSER name [rules ...]建立或修改使用者
ACL DELUSER name [name ...]刪除使用者
ACL GETUSER name查看使用者詳情
ACL WHOAMI當前已認證使用者
ACL CAT [category]列出命令類別或類別內的命令
ACL LOG [count|RESET]查看 ACL 拒絕日誌
ACL SAVE / ACL LOAD存儲/載入 ACL 檔案
ACL DRYRUN user cmd [arg ...]模擬測試權限（Redis 7+）
 
# ACL 規則語法
ACL SETUSER app_reader on >SecurePass123 \
  ~cache:* ~session:*       # 可存取的 key 模式 \
  +@read -@dangerous         # 允許讀取，禁止危險命令 \
  &events:*                  # Pub/Sub 頻道權限

# 常用 ACL 規則元素
on / off              # 啟用/停用使用者
>password             # 添加密碼
nopass                # 無需密碼
~pattern              # key 模式（~* 所有 key）
%R~pattern            # 僅讀取權限的 key 模式（Redis 7+）
%W~pattern            # 僅寫入權限的 key 模式
&pattern              # Pub/Sub 頻道模式
+command              # 允許命令
-command              # 禁止命令
+@category            # 允許類別（@read, @write, @admin 等）
-@category            # 禁止類別
allcommands (+@all)   # 所有命令
allkeys (~*)          # 所有 key
allchannels (&*)      # 所有頻道

# 命令類別
@read @write @set @sortedset @list @hash @string
@bitmap @hyperloglog @geo @stream @pubsub
@admin @fast @slow @blocking @dangerous @connection

安全配置清單

# redis.conf — 安全設定

# 1. 綁定介面（勿用 0.0.0.0）
bind 127.0.0.1 10.0.0.5
protected-mode yes

# 2. TLS 加密
tls-port 6380
tls-cert-file /path/to/redis.crt
tls-key-file /path/to/redis.key
tls-ca-cert-file /path/to/ca.crt
tls-auth-clients optional    # yes/no/optional

# 3. ACL 檔案
aclfile /etc/redis/users.acl

# 4. 禁用或重命名危險命令
rename-command FLUSHALL ""
rename-command FLUSHDB ""
rename-command DEBUG ""
rename-command CONFIG "ADMIN_CONFIG_$(random)"

# 5. 其他
maxclients 10000
tcp-keepalive 300
✅
安全最佳實踐：①不暴露至公網 ②啟用 ACL ③使用 TLS ④禁用危險命令 ⑤以非 root 使用者執行 ⑥定期更新 Redis 版本 ⑦開啟 protected-mode

記憶體管理與逐出策略

逐出策略 (maxmemory-policy)說明
noeviction不逐出，寫入命令回傳錯誤（預設）
allkeys-lru所有 key 中移除最近最少使用的 ⭐推薦
allkeys-lfu所有 key 中移除使用頻率最低的
allkeys-random所有 key 中隨機移除
volatile-lru有 TTL 的 key 中移除 LRU
volatile-lfu有 TTL 的 key 中移除 LFU
volatile-random有 TTL 的 key 中隨機移除
volatile-ttl有 TTL 的 key 中移除最快過期的
💡
快取場景推薦 allkeys-lru。混合用途（快取+持久資料）推薦 volatile-lru，確保持久資料不被逐出。
# 記憶體配置
maxmemory 4gb
maxmemory-policy allkeys-lru
maxmemory-samples 10  # LRU/LFU 取樣數（越大越精確）

# 記憶體分析命令
INFO memory               # 記憶體使用摘要
MEMORY USAGE key          # 單一 key 的記憶體使用量（bytes）
MEMORY DOCTOR             # 記憶體問題診斷
MEMORY STATS              # 詳細記憶體統計
MEMORY PURGE              # 請求分配器釋放記憶體

# 重要 INFO memory 指標
used_memory           # Redis 已分配的記憶體
used_memory_rss       # OS 層面的記憶體（含碎片）
used_memory_peak      # 歷史尖峰
mem_fragmentation_ratio # >1.5 表示碎片嚴重
maxmemory             # 設定的上限

效能優化技巧

策略說明
Pipeline批次發送命令減少網路往返（RTT），可提升數倍至數十倍吞吐量
連線池重用連線避免反覆建立 TCP。設定合理的 pool size（通常 CPU 核數 × 2-4）
批次操作用 MSET/MGET、HMSET 代替逐一 SET/GET。減少命令數量
避免大 KeyString < 10KB，Hash/Set/List < 5000 元素。大 key 影響刪除、遷移、序列化
避免熱點 Key讀取熱點可用本地快取 + 隨機加後綴分散。寫入熱點考慮聚合後批次寫入
Lazy Free開啟 lazyfree-lazy-eviction 等，讓刪除在背景執行不阻塞
多執行緒 I/ORedis 6+：io-threads 4、io-threads-do-reads yes
適當資料結構小 Hash 用 listpack 比多個 String 省記憶體 30%+
 
# redis.conf — 效能相關配置
io-threads 4              # 多執行緒 I/O（Redis 6+）
io-threads-do-reads yes
lazyfree-lazy-eviction yes
lazyfree-lazy-expire yes
lazyfree-lazy-server-del yes
lazyfree-lazy-user-del yes  # DEL 行為同 UNLINK

# 編碼閾值（小值用壓縮編碼更省記憶體）
hash-max-listpack-entries 128
hash-max-listpack-value 64
list-max-listpack-size -2
set-max-listpack-entries 128
zset-max-listpack-entries 128
zset-max-listpack-value 64

監控與故障排除

# 關鍵監控指標
INFO stats
  instantaneous_ops_per_sec  # 每秒操作數 (QPS)
  total_commands_processed   # 總處理命令數
  keyspace_hits / keyspace_misses  # 命中率
  rejected_connections       # 被拒連線（maxclients 不足）
  evicted_keys               # 已逐出 key 數

INFO memory
  used_memory_human          # 當前記憶體使用
  mem_fragmentation_ratio    # 碎片率（>1.5 需注意）

INFO clients
  connected_clients          # 當前連線數
  blocked_clients            # 阻塞中的客戶端

INFO replication
  master_link_status         # 複製連線狀態
  master_repl_offset         # 複製偏移量

# 慢查詢排查
slowlog-log-slower-than 10000  # 記錄 >10ms 的命令
slowlog-max-len 128
SLOWLOG GET 25                  # 查看最近 25 筆

# 延遲診斷
redis-cli --latency              # 持續測量延遲
redis-cli --latency-history      # 延遲歷史（每 15 秒）
redis-cli --latency-dist         # 延遲分布
redis-cli --bigkeys              # 掃描大 key
redis-cli --memkeys              # 掃描記憶體大戶
redis-cli --hotkeys              # 熱點 key（需開啟 LFU）

常見問題與解決方案

問題症狀解決方案
記憶體碎片mem_fragmentation_ratio > 1.5開啟 activedefrag yes；或計畫重啟
大 Key 阻塞刪除大 Hash/Set 時延遲飆升用 UNLINK 替代 DEL；開啟 lazyfree
連線數耗盡rejected_connections 增加使用連線池；調高 maxclients；檢查連線洩漏

快取雪崩大量 key 同時過期TTL 加隨機偏移量；預熱快取；熔斷降級
快取擊穿熱點 key 過期瞬間大量 DB 查詢互斥鎖（SETNX）；邏輯過期；永不過期+背景更新
