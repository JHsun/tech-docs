§1 — §4 · Introduction

範圍、符合性與語言概覽

Scope, Conformance & Overview

規範的第一章定義了「ECMAScript 是什麼」以及「什麼樣的實作才算符合規範」。它不是語法教學,而是給實作者(瀏覽器、Node.js、Deno 等)的合規契約。理解這一章,才能正確解讀後續所有規範條文。

  1. ECMAScript 是一般用途的程式語言,並非僅為 Web 設計。
  2. 符合實作必須支援規範中定義的所有型別、值、物件、屬性與語法。
  3. Unicode 與 ISO/IEC 10646 是原始碼編碼的唯一基準。
  4. ECMA-402 提供國際化 API,是與本規範互補的姊妹標準。
  5. 規範允許實作擴充,但不可違反既有語意。
Cheat Sheet · §1 KEY TERMS
Term 術語Meaning 意義
ECMAScript由 Ecma International TC39 維護的程式語言標準;JavaScript 是其最知名實作。
Conforming Implementation符合實作 — 必須支援所有規範定義的型別、值、物件、函式、語法與語意。
Host Environment宿主環境 — 瀏覽器、Node.js、Deno 等提供 globalThis 擴充的執行環境。
Normative / Informative規範性(必須遵循) vs. 資訊性(僅供參考)。Annex B 為規範性但僅限於瀏覽器。

ECMA-404JSON 資料交換格式規範。

Stage 0 — 4提案流程階段。僅 Stage 4 完成提案會進入下一版規範。

§5 · Notational Conventions

標記慣例

Notational Conventions

這是規範的「元語言」章節 — 教你如何閱讀規範本身。BNF 文法、演算法步驟、抽象操作呼叫、完成記錄 (Completion Record) 的表示法,全都在這裡定義。沒讀懂這一章,看後面的章節會像看密碼。

  1. Context-Free Grammar (CFG) 採用擴充 BNF 描述語法。
  2. Lexical Grammar、Syntactic Grammar、RegExp Grammar 三者分離。
  3. 演算法步驟 (Algorithm Steps) 用編號步驟取代程式碼。
  4. Completion Record 是控制流的基本單位(normal / break / continue / return / throw)。
  5. ? 與 ! 是規範專用的簡寫語法糖。

§6 · Data Types and Values

資料型別與值

ECMAScript Data Types & Values

ECMAScript 的型別系統看似簡單,實則細節繁多。原始型別 (Primitive) 有七種,加上 Object 型別共八種。這一章也定義了屬性描述符 (Property Descriptor)、規範型別 (Reference Record、Environment Record 等) — 這些是理解引擎行為的基礎。

  1. Language Types 共 8 種:Undefined、Null、Boolean、String、Symbol、Number、BigInt、Object。
  2. String 是 UTF-16 code unit 序列,不是字元序列。
  3. Number 遵循 IEEE 754 double precision,但多出 -0 與 NaN 的特殊處理。
  4. Spec Types 包含 Reference Record、Property Descriptor、Environment Record、PrivateName。
  5. Well-Known Intrinsics 與 Well-Known Symbols 清單必須熟記其用途。
Cheat Sheet · §6 TYPE SYSTEM
Type 型別Notes 備註
Undefined僅一個值 undefined。未初始化變數、無回傳值的預設。
Null僅一個值 null。表示「刻意空值」。typeof null === "object" 是歷史錯誤。
Booleantrue / false。
StringUTF-16 code unit 的有限有序序列。長度上限 2^53 − 1。
Symbol唯一不可變識別子。用於屬性鍵與 well-known hooks。
Number64-bit IEEE 754 雙精度浮點。包含 NaN、±0、±Infinity。
BigInt任意精度整數。與 Number 不可混用算術,需明確轉換。
Object屬性的集合。每個屬性都是資料屬性或存取器屬性。
Property Descriptor{ value, writable, enumerable, configurable } 或 { get, set, enumerable, configurable }。
Well-Known Symbol@@iterator、@@asyncIterator、@@toPrimitive、@@hasInstance 等。

§7 · Abstract Operations

抽象操作

Abstract Operations

抽象操作 (AO) 是規範定義演算法的基本單位。型別轉換、比較、物件操作、迭代器操作都在這裡。它們是看不見的「幕後規則」 — 你寫 a + b,引擎就會呼叫 ToPrimitive、ToNumber、ToString 這些 AO。

  1. Type Conversion:ToPrimitive、ToBoolean、ToNumber、ToString、ToObject、ToPropertyKey。
  2. Testing and Comparison:IsCallable、IsConstructor、SameValue、IsStrictlyEqual、IsLooselyEqual。
  3. Operations on Objects:Get、Set、CreateDataProperty、HasProperty、Invoke。
  4. Iterator Operations:GetIterator、IteratorNext、IteratorClose、IteratorStep。
  5. 讀懂常見 AO,就能在腦中模擬引擎執行任何表達式。
Cheat Sheet · §7 CORE OPERATIONS
Operation 操作Purpose 用途
ToPrimitive(v, hint)將物件轉為原始值。呼叫 Symbol.toPrimitive 或 valueOf/toString。
ToBoolean(v)判斷 truthy/falsy。七個 falsy 值:undefined null 0 -0 NaN "" 0n。
ToNumber / ToNumeric轉 Number 或保留 BigInt。字串需要 trim 與解析。
ToString(v)轉字串。物件會觸發 ToPrimitive(v, "string")。
ToPropertyKey(v)轉成合法屬性鍵(String 或 Symbol)。
SameValue(a, b)Object.is 的底層。NaN 等於 NaN,+0 不等於 -0。
SameValueZero用於 Map/Set。NaN 等於 NaN,但 +0 等於 -0。
IsStrictlyEqual(a, b)=== 的規範定義。
IsLooselyEqual(a, b)== 的規範定義。允許型別強制轉換。
GetIterator(obj, hint)取得同步或非同步迭代器。
IteratorStep(iter)呼叫 .next() 並判斷是否 done。

§9 · Executable Code & Execution Contexts

執行上下文、Realms 與 Agents

Execution Contexts, Realms & Agents

「ECMAScript 程式碼是如何被執行的?」這一章給出答案。Environment Record 管理變數範疇、Realm 管理內建物件、Agent 管理並行與共享記憶體。理解這三層結構,你會徹底明白為什麼 this、closure、Worker 是這樣運作的。

  1. Environment Record 四類:Declarative、Object、Function、Global、Module。
  2. Execution Context 包含 LexicalEnvironment、VariableEnvironment、PrivateEnvironment、Realm。
  3. Realm 是一整套內建物件(全域物件 + 內建建構子 + 原型鏈),跨 Realm 的 Array 不是 Array。
  4. Agent 是並行執行單位,含 Event Loop 與 Job Queue(Promise 微任務)。
  5. Agent Cluster 共享 SharedArrayBuffer,涉及 Memory Model。

§10 · Ordinary & Exotic Objects Behaviours

一般物件與奇異物件行為

Ordinary & Exotic Objects

所有 JS 物件都透過 11 個 Internal Methods([[Get]]、[[Set]]、[[Delete]]...)運作。Ordinary Object 是「預設行為」;Exotic Object(Array、String、Arguments、Proxy、Module Namespace 等)覆寫這些方法來實現特殊語意。理解這一章就能解釋「為什麼 Array.length 這麼奇怪」。

  1. Essential Internal Methods 共 11 個,每個 Exotic Object 都覆寫部分方法。
  2. OrdinaryGet、OrdinarySet、OrdinaryDefineOwnProperty 是預設實作。
  3. Array 是 exotic,因為 length 會隨索引屬性變動。
  4. Proxy 是「完全可自訂」的 exotic object,透過 trap 攔截所有方法。
  5. Module Namespace Object 不可變,是 import * as ns 回傳的物件。

[[DefineOwnProperty]]定義自有屬性。對應 Object.defineProperty。

§11 — §12 · Source Code & Lexical Grammar

原始碼與詞法文法

Source Code & Lexical Grammar

原始碼在進入語法解析之前,要先被切成 Token。這一章定義 Unicode 處理、保留字、識別字規則、自動分號插入 (ASI)、Template Literal 的詞法 — 全部都是語法錯誤的常見根源。

  1. 原始碼是 Unicode code point 序列,內部表示可為 UTF-16。
  2. 識別字遵循 UAX #31,支援 $、_ 與 Unicode 字母。
  3. 保留字分為:Reserved Words、Contextual Keywords、Future Reserved Words。
  4. Automatic Semicolon Insertion (ASI) 有三條規則,且有「限制產生式」(Restricted Productions)。
  5. Number Literal 支援 0b、0o、0x 與數值分隔符 1_000_000。
Cheat Sheet · §11 — §12 TOKENS
Token 類別Examples 範例
Reserved Wordsbreak case class const do else export extends for if import in let new return super switch this throw try typeof var void while with yield 等。
Contextual Keywordsasync of from as using await — 僅在特定語境為關鍵字。
Identifier需以 ID_Start 字元開頭,後接 ID_Continue。$、_ 視為合法。
Numeric Literal123、0xFF、0b1010、0o777、1_000_000、1n(BigInt)。
String Literal單引號或雙引號。支援 \u{1F600} code point escape。
Template Literal反引號字串,含 ${expr};tagged template 可接收 raw 字串。

ASI Rule 2程式結尾,若缺分號則插入。
ASI Rule 3限制產生式(如 return 後直接換行)強制插入分號。

§13 · Expressions

運算式

ECMAScript Language: Expressions

運算式是產生值的語法結構。從最基本的 this、識別字參照、字面值,到物件字面值、箭頭函式、解構、擴充、可選鏈,乃至於所有二元運算子的優先級與結合律 — 這一章是日常寫程式最常翻的一頁。

  1. Primary Expression 包括 this、識別字、字面值、陣列/物件字面值、正則、模板字串。
  2. Member Access 包含點運算子、[]、可選鏈 ?.、new.target、import.meta。
  3. Destructuring Assignment Pattern 是語法糖但有獨立產生式。
  4. 邏輯運算子 &&、||、?? 是短路求值;??=、&&=、||= 是邏輯賦值。
  5. 運算子優先級由文法層級隱含決定,不依靠額外表格。

§14 · Statements & Declarations

陳述式與宣告

Statements & Declarations

陳述式控制流程,宣告引入繫結。這一章涵蓋變數宣告、迴圈、條件判斷、例外處理,以及 TDZ(暫時性死區)的產生原因。特別注意 let/const 與 var 的範疇差異、for-of 與 for-in 的語意區別。

  1. Declaration 包括 var、let、const、function、class。

  3. Iteration:for、while、do-while、for-in、for-of、for await-of。

§15 · Functions & Classes

函式與類別

Functions & Classes

函式是 ECMAScript 的一等公民。這一章定義函式宣告、箭頭函式、generator、async function、async generator,以及類別的所有細節:繼承、static、private field、static initialization block。私有欄位 #x 使用 PrivateName,是真正的封裝而非慣例。

  1. 函式類型:FunctionDeclaration、FunctionExpression、ArrowFunction、GeneratorFunction、AsyncFunction、AsyncGeneratorFunction、MethodDefinition。
  2. Arrow function 無自己的 this、arguments、super、new.target。
  3. Class body 為隱式嚴格模式;class 宣告不提升。
  4. Private fields #name 使用獨立 PrivateEnvironment 管理。
  5. Static initialization block static { ... } 在類別定義時執行一次。

§16 · Scripts & Modules ES2025 重點

腳本與模組

Scripts & Modules

Script 與 Module 是 ECMAScript 的兩種頂層語法單位。Module 是靜態分析、自動嚴格模式、每模組一個 Module Environment Record。ES2025 在這一章引入了最重要的兩項新特性:Import Attributes 與 JSON Modules,讓 import 語句可攜帶中繼資料。

ES2025 新增:Import Attributes & JSON Modules

// 靜態 import 附加屬性 import config from "./config.json" with { type: "json" }; // 動態 import 附加屬性 const data = await import("./data.json", { with: { type: "json" } }); // re-export 時也可使用 export { default as settings } from "./s.json" with { type: "json" };
  1. Script 可有 var、未宣告變數會成全域;Module 必嚴格模式、頂層 this 為 undefined。
  2. Module 有 Module Record(三種:Cyclic、Source Text、Synthetic)。
  3. Import 宣告是靜態分析的;動態 import() 回傳 Promise。
  4. import with { type: "json" } 明確指定模組類型,由宿主負責驗證。
  5. Top-Level Await (TLA) 讓模組成為 async;會影響模組依賴順序。

§18 — §20 · Fundamental & Numeric Objects

全域、基礎與數值物件

Global, Fundamental & Numeric Objects

從 globalThis 開始,經 Object、Function、Boolean、Symbol、Error,一路到 Number、BigInt、Math、Date。這些是所有 JS 程式的共同基底。重點是各原型方法的完整列表與 Math.f16round 等 ES2025 新成員。

  1. globalThis 統一各環境的全域物件參照。
  2. Object 靜態方法:assign、create、defineProperty、entries、fromEntries、freeze、hasOwn、groupBy。
  3. Symbol.for/Symbol.keyFor 使用全域註冊表;Symbol() 則唯一。

  5. Date 遵循 simplified ISO 8601;時區處理常見坑點。

§21 — §23 · Text, Indexed & Keyed Collections ES2025 重點

文字、索引與鍵集合

Text, Indexed & Keyed Collections

字串、正則、陣列、TypedArray、Map、Set、WeakMap、WeakSet。ES2025 在這裡有兩大重點:RegExp 的 重複命名捕獲群組(Duplicate Named Capture Groups)與 RegExp.escape(),以及 Set 的七個新方法(intersection, union, difference, symmetricDifference, isSubsetOf, isSupersetOf, isDisjointFrom)。TypedArray 家族也多了 Float16Array。

ES2025:Set 的新方法

const a = new Set([1, 2, 3, 4]); const b = new Set([3, 4, 5, 6]); a.union(b); // Set {1,2,3,4,5,6} a.intersection(b); // Set {3,4} a.difference(b); // Set {1,2} a.symmetricDifference(b); // Set {1,2,5,6} a.isSubsetOf(b); // false a.isSupersetOf(b); // false a.isDisjointFrom(b); // false

ES2025:RegExp.escape & 重複命名捕獲

// 過去需手動跳脫特殊字元 const input = "a.b*c"; const safe = RegExp.escape(input); // "a\\.b\\*c" new RegExp(safe); // 重複命名捕獲 (不同分支同名) const re = /(?<year>\d{4})-\d\d|\d\d-(?<year>\d{4})/;
  1. 字串 UTF-16;String.prototype.at、codePointAt、normalize 處理 Unicode。
  2. Array 方法:不可變 toReversed, toSorted, toSpliced, with(ES2023)配合 ES2025 仍為主力。
  3. TypedArray 家族:Int8, Uint8, Uint8Clamped, Int16, Uint16, Int32, Uint32, Float16, Float32, Float64, BigInt64, BigUint64。
  4. Map / Set 使用 SameValueZero 比較,鍵可為任意值。
  5. WeakMap / WeakSet 鍵為弱引用,可被 GC。

§24 — §25 · Structured Data & Memory

結構化資料與記憶體管理

Structured Data & Memory Management

ArrayBuffer 與 SharedArrayBuffer 是底層二進位緩衝區;DataView 提供任意位元組順序的存取;Atomics 提供跨 Agent 的原子操作;JSON 是資料交換標準;WeakRef 與 FinalizationRegistry 用於自訂弱引用語意。

§26 · Control Abstraction Objects ES2025 重點

控制抽象物件

Control Abstraction Objects

Iterator、AsyncIterator、Generator、Promise — ECMAScript 中處理非同步與惰性求值的核心。ES2025 的重頭戲就在這裡:全新的 Iterator Helpers 讓迭代器擁有類似陣列的鏈式操作,以及 Promise.try() 統一同步錯誤與非同步錯誤的處理。

ES2025:Iterator Helpers

// 同步 Iterator Helpers const result = naturals() .filter(n => n % 2 === 0) .map(n => n * n) .take(5) .toArray(); // [4, 16, 36, 64, 100] // 從任何 iterable 建構 Iterator.from(someIterable) .drop(10) .take(5) .forEach(console.log); // Async 版本 await AsyncIterator.from(stream) .map(async x => await process(x)) .toArray();

ES2025:Promise.try

// 統一包裝同步/非同步函式,讓錯誤都走 Promise 鏈 Promise.try(() => { if (!user) throw new Error("未登入"); return fetchProfile(user.id); // 同步 throw 或 async }).then(render).catch(handleError);
  1. Iterator Protocol:{ next(), return?(), throw?() },回傳 { value, done }。
  2. Iterable Protocol:物件具 [Symbol.iterator]() 方法。
  3. Generator 同時實作 Iterator + Iterable;yield* 委派給另一個 iterable。
  4. Iterator.from() + Helpers 讓迭代器也能鏈式操作。
  5. Promise 狀態:pending → fulfilled / rejected;一旦轉變不可逆。

§27 — §28 & Annexes · Reflection, Proxy & Intl

反射、Proxy 與國際化

Reflection, Proxy & Internationalization

Proxy 是元程式設計的旗艦;Reflect 提供內部方法的函式形式;Intl(由 ECMA-402 定義)負責地區感知的格式化。最後規範還有一系列 Annex,包括 Annex B(瀏覽器相容性,規範性但有限適用)與 Annex E(歷年修訂紀錄)。

  1. Proxy handler 可定義 13 個 trap,對應 11 個 internal methods + apply/construct。
  2. Proxy invariants 必須遵守,否則拋 TypeError(如回傳 non-configurable 屬性的假值)。
  3. Reflect 方法名與 internal method 一一對應,回傳值取代 throw。
  4. Intl:Collator, NumberFormat, DateTimeFormat, RelativeTimeFormat, PluralRules, ListFormat, Locale, Segmenter, DurationFormat。
  5. Annex B 僅瀏覽器實作需遵循;包含 escape/unescape、String.prototype.substr、HTML wrappers 等歷史遺留。
