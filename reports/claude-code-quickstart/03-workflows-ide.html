💻 VS Code 整合

安裝 & 啟動

在 Extensions Marketplace 搜尋「Claude Code」安裝。安裝後可透過 Activity Bar 的 Spark 圖示、Command Palette (Cmd+Shift+P)、或底部狀態列的 ✱ 圖示開啟。

面板位置
Secondary Sidebar右側邊欄（推薦，coding 同時看 Claude）
Primary Sidebar左側邊欄
Editor Tab作為分頁（適合 side task）
快捷鍵
Cmd+Esc切換 Editor ↔ Claude 焦點
Alt+K插入 @file#lines 引用
Shift+Enter多行輸入
核心功能
▸
@ 引用 — 輸入 @ 後 fuzzy 搜尋檔案。支援 @file#5-10 引用特定行。
▸
選取文字 — 選取程式碼後 Claude 自動看到。可切換 eye icon 隱藏/顯示。
▸
Inline Diff — Claude 的修改以 diff viewer 顯示，可逐一接受/拒絕。
▸
多對話 — Open in New Tab / New Window 開啟多個獨立對話。
▸
Chrome 整合 — 附件選單可開啟瀏覽器工具。Claude 共享你的登入狀態。
▸
Session 歷史 — 頂部下拉選單可搜尋 & 瀏覽過去對話。
狀態指示：tab 上的彩色點 = 🔵 需核准權限、🟠 Claude 已完成但 tab 被隱藏。

📝 Code Review（GitHub PR 自動審查）

功能概覽

多個專門子代理在完整 codebase 的 context 下審查 PR，尋找邏輯錯誤、安全漏洞、邊界案例和回歸問題。發現以 inline comment 形式標註在程式碼上，附帶嚴重程度標籤。

不會 approve 或 block PR — 不干擾既有的 review 流程。

Review 觸發行為
PR 建立時PR 開啟或標為 ready 時自動 review 一次
每次 Push每次推送到 PR 分支都重新 review，自動解決已修復的 issue
手動在 PR 上留言 @claude review 觸發
成本注意：每次 push 觸發花費最高。管理介面顯示每個 repo 的平均 review 成本。
設定方式
1
安裝 Claude GitHub App，授權 repo 存取
2
在 admin 設定中選擇要啟用的 repo
3
為每個 repo 設定 Review Behavior
4
用 CLAUDE.md 或 REVIEW.md 自訂 review 重點
替代方案

GitHub Actions / GitLab CI/CD：在自己的 CI 基礎設施中執行 Claude，適合需要自託管 & 自訂自動化的場景。

🌳 Git Worktrees

Claude 可在獨立的 Git worktree 中工作，實現平行開發。

# Claude 自動建立 worktree claude --worktree # 手動建立後啟動 git worktree add ../feature-a -b feature-a cd ../feature-a && claude

結束後 Claude 會詢問保留或移除 worktree。加 .claude/worktrees/ 到 .gitignore。

🔗 LSP 整合（程式碼智慧）

LSP (Language Server Protocol) 讓 Claude 像 IDE 一樣「看見」你的程式碼。

Go-to-definition跳到符號定義處
Find references列出所有使用位置
Hover docs取得型別資訊和文件
啟用方式（settings.json）
{ "enabledPlugins": { "typescript-lsp@claude-plugins-official": true, "pyright-lsp@claude-plugins-official": true } }

支援 TypeScript、Python、Go、Rust 等有 LSP 的語言。
