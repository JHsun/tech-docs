🔌 MCP（Model Context Protocol）— 外部工具連線深入

概念

MCP 是連接 Claude 到外部系統的標準協定。沒有 MCP，Claude 無法查詢資料庫或發 Slack。MCP 提供能力（tools），Skills 提供如何有效使用這些能力的知識。

兩種傳輸類型
stdio本地 process（最常用）。啟動一個子程序並透過 stdin/stdout 通訊。
url遠端 HTTP/SSE 端點。適合雲端服務。
設定範圍 & 優先順序
.mcp.json專案 — commit 到 git 讓團隊共享
~/.claude/.mcp.json個人 — 所有專案可用

覆寫：local > project > user

.mcp.json 完整範例
{ "mcpServers": { "github": { "type": "stdio", "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"], "env": { "GITHUB_TOKEN": "${GITHUB_TOKEN}" } }, "slack": { "type": "url", "url": "https://mcp.slack.com/sse" }, "jira": { "type": "stdio", "command": "npx", "args": ["-y", "@anthropic/mcp-jira"], "env": { "JIRA_HOST": "${JIRA_HOST}", "JIRA_EMAIL": "${JIRA_EMAIL}", "JIRA_API_TOKEN": "${JIRA_API_TOKEN}" } } } }
CLI 管理命令
# 新增 MCP 伺服器 claude mcp add github -- npx -y \ @modelcontextprotocol/server-github # 指定範圍 (user scope) claude mcp add github -s user \ -e GITHUB_TOKEN=xxx -- npx -y \ @modelcontextprotocol/server-github # 列出 / 移除 claude mcp list claude mcp remove github # Session 內查看狀態 & token 成本 /mcp
常見 MCP 伺服器
GitHubPR、Issues、Repos
JIRA / LinearTicket 工作流
Slack通知 & 搜尋
PostgreSQL直接查詢 DB
Playwright瀏覽器自動化
Filesystem受限範圍檔案存取
MCP Prompts：MCP 伺服器可暴露 prompts 作為命令，格式為 /mcp__server__prompt。

4 種 Handler 類型
command
執行 shell 指令（95% 使用場景）
http 2026.2 新增
POST 到外部端點。非阻擋性錯誤（失敗時繼續執行）。
prompt
單輪 LLM 評估。用 $ARGUMENTS 接收事件資料。適合語意判斷。
agent
產生子代理做深度驗證。可存取 Read、Grep、Glob 工具。
Exit Code 語意
0允許操作繼續
2阻擋操作（最強控制）
其他非阻擋性錯誤，顯示 stderr
PreToolUse 決策控制
allow繞過權限系統
deny阻擋工具，告知原因
ask提示使用者確認
updatedInput修改工具輸入參數
additionalContext為 Claude 注入額外 context
常見 Hook 模式

自動格式化（最常用）

"PostToolUse": [{ "matcher": "Write|Edit|MultiEdit", "hooks": [{ "type": "command", "command": "npx prettier --write \"$CLAUDE_TOOL_INPUT_FILE_PATH\"" }] }]

阻擋危險命令

"PreToolUse": [{ "matcher": "Bash", "hooks": [{ "type": "command", "command": "echo \"$CLAUDE_TOOL_INPUT\" | grep -qE 'rm -rf|DROP TABLE' && exit 2 || exit 0" }] }]

Session 啟動注入 context

"SessionStart": [{ "hooks": [{ "type": "command", "command": "echo '{\"additionalContext\": \"Branch: '$(git branch --show-current)'\"}'" }] }]
Hooks > CLAUDE.md：CLAUDE.md 的安全規則約 70% 遵守率。Hook 的 exit code 2 是 100% 確定性阻擋。安全規則務必用 Hook。Hooks 在所有層級合併執行（不覆寫）。

⚙ Settings 系統 — 設定檔層級與完整結構

設定檔層級
檔案說明Git
Managed policy企業管理員強制規則—
~/.claude/settings.json個人全域設定—
.claude/settings.json專案設定（團隊共享）✓ commit
.claude/settings.local.json個人覆寫（不共享）✗ gitignore
完整 settings.json 結構
{ "permissions": { "allow": [ "Bash(git *)", "Bash(npm run *)", "Read", "mcp__github__*" ], "deny": [ "Read(.env*)", "Write(production.config.*)" ] }, "hooks": { ... }, "env": { "MAX_THINKING_TOKENS": "10000", "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "50" }, "model": "claude-sonnet-4-6", "enabledPlugins": { "typescript-lsp@claude-plugins-official": true } }
Permission 語法規則

工具名稱：Read、Write、Edit、Bash、Glob、Grep、WebSearch、WebFetch、NotebookEdit、LSP

精細控制範例：

// Bash 只允許特定前綴 "Bash(git *)" // 允許 git 命令 "Bash(npm run *)" // 允許 npm scripts // 檔案路徑限制 "Read(.env*)" // 阻擋讀取 .env "Write(src/**)" // 允許寫入 src/ // MCP 工具 "mcp__github__create_issue" "mcp__slack__post_message"
環境變數
MAX_THINKING_TOKENS思考 token 上限
CLAUDE_AUTOCOMPACT_PCT_OVERRIDE自動壓縮 context 百分比
ANTHROPIC_API_KEYAPI key（headless 用）
CLAUDE_CODE_USE_BEDROCK使用 Amazon Bedrock
CLAUDE_CODE_USE_VERTEX使用 Google Vertex AI
企業管理：管理員可用 allowManagedHooksOnly 阻擋 user/project/plugin hooks，強制只使用企業管理的 hooks。

🔧 故障排除

常見問題
▸
command not found
開新終端視窗，或檢查 PATH。用 claude doctor 診斷。
▸
權限錯誤
不要用 sudo。用 nvm 安裝 Node.js，或切換到 native binary。
▸
Hook 不觸發
檢查 matcher regex。用 Ctrl+O 開啟 verbose 查看 hook 日誌。
▸
Stop hook 無限迴圈
Stop hook exit 2 會強制 Claude 繼續，要避免無條件觸發。
▸
context 太滿
用 /compact 壓縮，或把 reference 移到 skills。
診斷指令：claude doctor 檢查安裝、/debug 檢查 session、/mcp 檢查 MCP 狀態。
