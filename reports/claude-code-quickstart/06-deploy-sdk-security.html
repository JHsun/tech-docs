⚙ GitHub Actions 整合

在 GitHub Actions workflow 中執行 Claude Code，實現 CI/CD 自動化。

# .github/workflows/claude.yml - name: Run Claude Code run: | claude -p "run all tests and fix any failures" \ --allowedTools "Read,Write,Bash" \ --output-format stream-json \ --dangerously-skip-permissions env: ANTHROPIC_API_KEY: ${{ secrets.KEY }}
▸
可自動修復 lint 錯誤、更新文件、跑 code review
▸
搭配 CLAUDE.md 確保一致的程式碼風格
安全提醒：--dangerously-skip-permissions 只在隔離的 CI 容器中使用。Hook 仍可阻擋危險操作。

🛠 Agent SDK — 程式化使用 Claude Code

概念

Agent SDK 讓你以程式庫方式使用 Claude Code。給定任務後，SDK 執行代理迴圈，包含內建的檔案、shell、web 工具。適合建構自訂工具、自動化 pipeline、或將 Claude 嵌入你的應用。

支援語言
TypeScript@anthropic-ai/claude-code（完整支援）
Pythonanthropic-claude-code（完整支援）
TypeScript 範例
import { ClaudeCode } from "@anthropic-ai/claude-code"; const cc = new ClaudeCode({ permissionMode: "auto", allowedTools: ["Read", "Write", "Bash"], systemPrompt: "你是 CI 助手...", }); const result = await cc.run( "修復所有 TypeScript 型別錯誤" ); console.log(result.output);
關鍵能力
▸
Permission 控制 — allowedTools、disallowedTools、permissionMode
▸
Hooks — PreToolUse、PostToolUse 等回呼
▸
Streaming — 即時接收工具呼叫和回應
▸
Subagents — 在 SDK 中委派子任務
▸
Human-in-the-loop — 自訂核准 UI（--permission-prompt-tool）
適合場景：CI/CD pipeline 自動化、自訂 IDE 整合、建構基於 Claude 的產品、批量程式碼遷移。

🔒 安全模型與最佳實踐

安全架構
▸
預設唯讀 — Claude 預設只能讀取，修改需核准
▸
Permission 層級 — Deny > Ask > Allow（deny 優先）
▸
Hooks 強制執行 — PreToolUse hooks 可 100% 阻擋危險操作
▸
工具隔離 — Subagent 可限制工具存取範圍
▸
Sandbox — WSL 2 和容器支援沙箱隔離
安全最佳實踐
▸
秘密用環境變數（${VAR}），永遠不硬編碼
▸
用 deny 規則阻擋 .env 和敏感檔案的讀取
▸
用 PreToolUse hook 阻擋 rm -rf、DROP TABLE 等
▸
--dangerously-skip-permissions 只在隔離容器用
Data Usage & 隱私
▸
對話資料 — 收集 feedback（使用數據、接受/拒絕、/bug 回報）
▸
本地儲存 — Session 對話存在本地
▸
Zero Data Retention — 企業可啟用 ZDR（Code Review 不可用於 ZDR）
▸
Binary 驗證 — 可透過 SHA256 checksums 和 code signatures 驗證
企業安全管控
▸
Managed Policies — 管理員可強制設定，覆蓋所有層級
▸
allowManagedHooksOnly — 只允許企業管理的 hooks
▸
SSO & IAM — Teams/Enterprise 支援 SSO 和存取控制
▸
Analytics API — 追蹤整個組織的 Claude Code 使用狀況
▸
Healthcare (BAA) — 提供 BAA 合約支援
