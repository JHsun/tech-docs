01
📱 手機 App 寫程式
用 iOS/Android 隨時開啟 Claude Code Session

Boris 透露他有很大一部分程式碼是在 iPhone 上撰寫的。Claude Code 已內建於 Claude 的行動 App 中，讓你不用打開筆電就能啟動開發任務。

如何使用：

// 步驟 1. 下載 Claude App（iOS 或 Android） 2. 開啟 App，在左側選單找到 Code 分頁 3. 點擊即可開始一個完整的 Claude Code Session
⚠️
實際限制：手機端最適合輕量編輯和任務派發。複雜的多檔案重構還是建議在電腦上操作。Boris 的做法是早上在手機上啟動幾個 session，稍後再回電腦檢查結果。
02
🔀 /teleport 與 /remote-control — 跨裝置無縫切換
在手機、網頁、終端機之間移轉完整 Session

這是 Boris 多裝置工作流的核心。/teleport 可以把雲端 session 拉到本地終端機繼續，完整保留所有 context。/remote-control 則讓你從另一台裝置控制正在運行的 session，而不需要移轉它。

終端機 # 把雲端 session 拉到本地終端機 claude --teleport # 或在 session 內執行 /teleport # 從手機/網頁控制本地正在跑的 session /remote-control
功能說明使用場景
/teleport將雲端 session 完整下載到本地終端通勤時在手機開始，到辦公室接手
/remote-control遠端控制另一台裝置上的 session（不移轉）人在外面，遠端操控辦公室電腦上的 Claude
💡
Boris 的設定：在 /config 中啟用「Enable Remote Control for all sessions」，這樣所有 session 預設都可被遠端控制。目前 Remote Control 仍在 research preview，一次只能控制一個 session。
03
🔄 /loop 與 /schedule — 排程自動化
Boris 稱之為「Claude Code 中最強大的兩個功能」

這兩個命令讓 Claude 在設定的間隔自動重複執行任務，最長可達一週。差別在於 /loop 在本地執行（最多 3 天），/schedule 在 Anthropic 雲端執行（即使電腦關機也會繼續運作）。

Boris 實際運行的自動化排程：

Boris 的排程 # 每 5 分鐘：自動處理 code review、rebase、推送 PR 到 production /loop 5m /babysit # 每 30 分鐘：檢查 Slack feedback，自動建立 PR 來修正 /loop 30m /slack-feedback # 每次 merge 後：掃描遺漏的 review comments 並修復 /loop /post-merge-sweeper # 每小時：關閉過時不再需要的 PR /loop 1h /pr-pruner # 雲端排程（電腦關機也能跑，最長一週） /schedule 依照 cron 語法設定
功能執行位置最長運行並行上限
/loop本地機器3 天50 個並行 loop
/scheduleAnthropic 雲端7 天依方案而定
🚨
重要警告：像 /babysit 這樣會自動推送到 production 的排程，需要有完善的測試覆蓋率。如果你的專案測試很薄弱，不要直接照搬這個模式。Boris 的建議是：先把工作流寫成 skill，再用 loop 去排程執行它。
04
⚡ Hooks — 在代理生命週期中注入確定性邏輯
不是「建議」而是「保證執行」的自動化

Hooks 讓你在 Claude 的工作流程中的特定時間點自動執行自訂邏輯。跟寫在 CLAUDE.md 裡的指令不同（約 70% 遵守率），Hooks 是 100% 確定性執行。

Boris 提到的 Hook 使用範例：

SessionStart
每次啟動 Claude 時動態載入 context（如當前分支、環境變數）
PreToolUse
記錄每一個 bash 命令，或阻擋危險操作
PermissionRequest
把權限請求路由到 WhatsApp，讓你在手機上核准/拒絕
Stop
每當 Claude 停下來時，用 hook 戳它繼續工作
settings.json 範例 { "hooks": { "PreToolUse": [{ "matcher": "Bash", "hooks": [{ "type": "command", "command": "echo \"$CLAUDE_TOOL_INPUT\" | grep -qE 'rm -rf' && exit 2 || exit 0" }] }] } }
05
🖥️ Cowork Dispatch — 遠端桌面控制
不寫程式時，用 Dispatch 處理一切

Dispatch 是 Claude Desktop App 的安全遠端控制功能。它可以操作你的 MCP 伺服器、瀏覽器、檔案系統，比 Remote Control（僅控制終端 session）更強大。

Boris 每天用 Dispatch 來追 Slack、回 email、管理檔案 — 即使人不在電腦旁。他的說法是：「不寫程式的時候，我就在用 Dispatch。」

ℹ️
Dispatch vs Remote Control：Remote Control 控制的是終端機裡的 Claude Code session。Dispatch 控制的是 Claude Desktop App，可以操作瀏覽器和整個電腦。Dispatch 目前只支援 macOS。
06
🌐 Chrome 擴充功能 — 讓 Claude 看見瀏覽器
Boris 做前端開發時的必備工具

Boris 最重要的建議之一：讓 Claude 有辦法驗證自己的產出。Chrome 擴充功能讓 Claude 可以直接看到網頁渲染結果，然後自己迭代改進。就像讓工程師邊寫網站邊開瀏覽器看效果 — 品質會大幅提升。

✅
Boris 的原則：「使用 Claude Code 最重要的技巧就是：給 Claude 一個方法來驗證自己的輸出。一旦做到這點，Claude 就會自己迭代直到結果很好。」Chrome 擴充功能就是實現這個原則最直接的方式。
07
💻 Desktop App — 內建瀏覽器與伺服器
自動啟動 web server 並在內建瀏覽器中測試

Desktop App 打包了一個內建瀏覽器，Claude 可以自動啟動你的 web server 並在裡面測試，形成一個完整的「寫 → 跑 → 看 → 改」迴圈，不需要你手動切換視窗。

在 CLI 或 VS Code 中，你可以用 Chrome 擴充功能達到類似效果；或直接用 Desktop App 獲得開箱即用的體驗。

08
🌿 /branch — 分叉 Session 做實驗
像 Git 分支，但用在 AI 對話上

想嘗試不同的方案但不想弄亂現有對話？/branch 可以複製當前 session 到一個分支，兩個 session 獨立運行。原始對話完全不受影響。

# 在 session 內分叉 /branch # 從 CLI 分叉特定 session claude --resume <session-id> --fork-session # 回到原始 session claude -r <original-session-id>

Boris 經常用這個功能：在代理工作時分出一個分支回答快速問題，完成後回到主線。

09
💬 /btw — 不中斷代理的側面提問
Boris 說他「一直在用」的功能

當 Claude 正在執行一個大任務時，你突然想問個不相關的問題。/btw 讓你插入一個側面提問，Claude 回答完後會無縫回到原本的任務，不會遺失任何 context。

# Claude 正在重構一個大模組... # 你突然想知道一個拼寫 /btw dachshund 怎麼拼？ # Claude 回答後自動回到重構工作 # 完全不會打斷主任務的 context
🎯
類比：就像在演講者報告時輕聲問他一個問題 — 他小聲回答你後繼續報告，完全不會忘記講到哪裡。
10
🌳 Git Worktrees — 數十個代理平行工作
Boris 同時跑「數十個 Claude」的基礎設施

這是 Boris 整個工作流的基礎設施。Git worktrees 讓多個 Claude 代理在同一個 repo 中平行工作，每個代理有自己隔離的程式碼副本，不會互相衝突。Boris 說他「隨時都有數十個 Claude 在運行」。

# 在新的 worktree 中啟動 Claude session claude -w # 或在 Desktop App 中勾選 "worktree" 選項 # 批次在多個 worktree 中展開工作 /batch # → 見功能 11 # 非 Git VCS 使用者 # 用 WorktreeCreate hook 自訂 worktree 建立邏輯
💡
類比：想像 20 個包商在翻新你的房子，但每個人有自己的房子副本可以施工。完成後再把改動合併回來。這就是 worktrees 的概念。
11
⚡ /batch — 批次展開到數百個代理
需要大規模修改時的終極武器

/batch 會先訪談你了解需求，然後自動把工作分配到任意數量的 worktree 代理中（可以是數十、數百、甚至數千個）。適合大規模的 codebase 修改。

# 啟動批次模式 /batch # Claude 會： # 1. 訪談你了解需求 # 2. 規劃如何拆分工作 # 3. 建立多個 worktree # 4. 分配代理平行執行 # 5. 匯整結果
12
⚡ --bare — 啟動速度快 10 倍
跳過自動載入，適合 SDK 和 CI 場景

--bare 旗標跳過自動載入 CLAUDE.md、settings、MCP、plugins 等，讓啟動速度提升最多 10 倍。在腳本化或 CI/CD pipeline 中，每一秒都很重要。

claude --bare -p "summarize this codebase" \ --output-format=stream-json \ --verbose
ℹ️
未來預告：Boris 提到未來版本可能會將 --bare 設為預設行為。目前需要手動加上旗標。
13
📂 --add-dir — 跨多個 Repo 工作
讓 Claude 同時看見並操作多個程式碼庫

Boris 通常在一個 repo 中啟動 Claude，再用 --add-dir 讓 Claude 看到另一個 repo。這不僅告訴 Claude 另一個 repo 的存在，也賦予它在該 repo 中操作的權限。

# 啟動時加入額外目錄 claude --add-dir ../shared-library # Session 內加入 /add-dir ../another-repo # 團隊設定：在 settings.json 中永久加入 { "additionalDirectories": ["../shared-lib"] }
14
🤖 --agent — 自訂代理
一個常被忽略的強大原語

Boris 特別點出自訂代理是一個「常被忽略的強大原語（primitive）」。在 .claude/agents/ 中定義一個 Markdown 檔案，就可以創建具有特定角色、工具限制、模型選擇的專門代理。

.claude/agents/security-reviewer.md --- name: security-reviewer description: 安全審查程式碼變更 tools: Read, Grep, Glob model: opus permissionMode: plan --- 你是資深安全工程師。檢查所有變更中的： - OWASP Top 10 漏洞 - 硬編碼的秘密和憑證 - SQL injection 風險 報告發現並附帶嚴重程度和修復建議。
# 使用自訂代理 claude --agent security-reviewer # 或在 session 中 /agents
15
🎤 /voice — 語音輸入
Boris 大部分的程式碼都是「說」出來的

Boris 透露一個令人驚訝的事實：他大部分的程式碼是透過語音輸入完成的，而不是打字。/voice 在不同平台有不同的啟用方式：

平台啟用方式
CLI 終端機執行 /voice，然後按住空白鍵說話
Desktop App按下語音按鈕
iOS App在 iOS 設定中啟用 Dictation
💡
適合場景：描述需求、解釋 bug、口述架構設計等自然語言任務。搭配 Claude 的理解能力，語音輸入比打字更快更自然。
