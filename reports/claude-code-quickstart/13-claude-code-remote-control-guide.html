§ 01 — Overview

這不是雲端遷移，而是同步層。

What it is

Remote Control 在你正在運行的本機 session 和 Claude 手機 App 或瀏覽器之間建立一條加密通道。Session 繼續在你的機器上執行——你的 filesystem、MCP servers、工具、專案配置全部都還在。

它解決一個具體的工作場景：你啟動了一個長時間的 refactoring、測試跑批、或 dependency upgrade 任務，但不想被綁在桌子前面。關上筆電蓋子太可惜，開著又走不開。Remote Control 讓你把筆電放進包裡，用手機繼續監督、核准、給指令——直到工作完成。

與 Claude Code on the web 不同的是，web sessions 跑在 Anthropic 託管的雲端基礎設施上，每個 session 都是乾淨的起點；Remote Control 則保留了你本機的一切——CLAUDE.md、自訂 skills、MCP 整合、檔案權限都原封不動。程式碼不會離開你的機器。

架構要點

你的本機 Claude Code process 只對外發送 HTTPS 請求，不會開啟任何 inbound port。它向 Anthropic API 註冊後輪詢指令。你從手機送出的訊息會先到 API，再透過既有的 streaming 連線轉發回本機。手機本身不執行程式碼、不直接存取檔案。

§ 02 — Prerequisites

開始前的三件事。

版本檢查

Remote Control 需要 Claude Code v2.1.51 或更新版本。

claude --version

訂閱與認證

  • 訂閱方案：Pro、Max、Team、Enterprise 都可用。API keys 不支援，必須用 claude.ai OAuth 登入。
  • Team / Enterprise：預設關閉，需要管理員在 Claude Code admin settings 開啟 Remote Control 開關。
  • 環境變數：如果環境裡有 ANTHROPIC_API_KEY，先 unset 掉，否則會走 API 認證導致失敗。

登入與工作區信任

# 1. 透過 claude.ai 登入
claude
# 然後在 session 裡執行：
/login

# 2. 至少在專案目錄下跑一次 claude，接受 workspace trust 對話框
cd ~/your-project
claude
§ 03 — Launch Modes

三種啟動方式，各有各的用途。

Remote Control 沒有「唯一正確」的啟動方式——要看你現在在哪裡。是正要開新工作？還是已經聊到一半想把它帶走？

A · Server Mode

純 server，等待遠端連線

claude remote-control

Process 停留在終端機以 server mode 運行，不進入互動介面。顯示 session URL，按空白鍵切換 QR code。適合打算完全從遠端操作的情境。

B · Interactive

本機互動 ＋ 遠端同時可用

claude --remote-control

開啟完整的終端機互動 session，同時讓 claude.ai 和手機 App 可以控制。你可以本機打字，也可以用手機打字，對話會同步。

C · Mid-Session ★

把現有 session 轉成可遠端

/remote-control

已經在 Claude Code session 裡、不想丟掉對話歷史？在 prompt 輸入 /rc 就好。完整歷史帶過去，session URL 與 QR code 隨即出現。

命名小技巧

每種方式都可以帶名稱參數，例如 claude --remote-control "My Project" 或 /remote-control My Project，這個名稱會顯示在 claude.ai/code 的 session list 裡。沒設的話會自動產生像 myhost-graceful-unicorn 的名字。

§ 04 — Step by Step

從零到手機遠端控制。

確認版本與登入狀態

執行 claude --version 確認 ≥ v2.1.51。如果沒登入或用的是 API key，先 claude auth login 並選 claude.ai 選項。

在專案目錄下啟動 Remote Control

挑一個適合你當下狀況的啟動方式（參見 § 03）。最常見的是 claude --remote-control，給你本機與遠端雙介面。

cd ~/my-project
claude --remote-control "Refactor auth module"

取得 session URL 或 QR code

終端機會印出 session URL；在 server mode 下按空白鍵可以切換 QR code 顯示。QR code 用來直接在手機 App 裡開啟 session。

從另一個裝置連線

有三條路：瀏覽器打開 session URL、手機掃 QR code、或到 claude.ai/code / Claude App 的 session list 裡找。在線的 Remote Control sessions 會顯示一顆電腦圖示加綠點。

交替使用、隨時切換

對話在所有已連線裝置間同步。你可以終端機打幾句，手機回幾句，瀏覽器補一句——都是同一個 session。如果筆電進入睡眠或網路斷線，session 會在機器恢復後自動重連。

（可選）設為所有 session 預設

在 Claude Code 裡執行 /config，把 Enable Remote Control for all sessions 設為 true。之後每個互動 session 都會自動註冊一個 remote session。

§ 06 — Permission Modes

關於 dangerously-skip-permissions 的真相。

如果你平常愛用 --dangerously-skip-permissions（也就是 bypass mode）來避免被權限提示打斷，這裡有個壞消息要告訴你：Remote Control 目前不支援 bypass 模式。

官方文件明講

Claude Code 的 Permission Modes 文件寫得很清楚：在 Remote Control 場景下，Auto 與 Bypass permissions 都不可用。你能選的最寬鬆模式是 acceptEdits。

實測與 Bug 回報

  • 直接組合會報錯：claude remote-control --dangerously-skip-permissions 會回 Unknown argument。反過來 claude --dangerously-skip-permissions remote-control 也不會啟動 remote control。
  • 中途用 /rc 轉換也沒用：GitHub Issue #29214 記錄，以 bypass 模式啟動 Claude Code 後用 /rc 開遠端，手機 App 仍會對每個檔案編輯和 bash 指令顯示權限提示。本機不會被打斷，但遠端介面還是要一個一個核准。
  • 社群需求已提出：GitHub Issue #31908 請求未來支援這個組合（給 Docker sandbox 裡的 agents 用），但目前還沒實作。
安全設計的考量

這個限制其實是合理的——如果允許跳過所有權限確認、再加上遠端控制，那麼任何能登入你 claude.ai 帳號的裝置就能對你本機做任意操作。攻擊面會被放大到沒辦法承受。

折衷方案：acceptEdits

想要遠端操作又不要一直被打斷，目前官方認可的最佳選擇是：

claude remote-control --permission-mode acceptEdits

acceptEdits 會自動核准工作目錄內的檔案編輯，以及常見的 filesystem bash 指令（mkdir、touch、rm、rmdir、mv、cp、sed）。其他 bash 指令、工作目錄外的寫入、受保護路徑仍會提示。這是 Remote Control 下最接近 bypass 體驗的安全選擇。

§ 08 — Limitations

值得先知道的邊界條件。

  • 一個 process 一個 remote session：除非你用 server mode，否則每個 Claude Code instance 同時只支援一個 remote session。
  • 本機 process 必須持續運行：Remote Control 是本機 process。關閉終端機、退出 VS Code、或停掉 claude process，session 就結束。
  • 網路中斷超過約 10 分鐘：如果機器是醒著的但網路連不出去超過約十分鐘，session 會 timeout、process 退出，要重跑 claude remote-control。
  • Ultraplan 會斷線：啟動 ultraplan session 會中斷目前的 Remote Control，因為兩者共用 claude.ai/code 介面，一次只能連一個。
  • Bypass permissions 不可用：見 § 06。目前只能用到 acceptEdits。
§ 09 — Mobile Push

讓 Claude 主動通知你。

Remote Control 啟用時，Claude 可以在長任務跑完、或需要你做決策時主動推 push notification 到手機。需要 Claude Code v2.1.110 或以上。

  • 下載 Claude 手機 App（iOS 或 Android），用同一個帳號登入。
  • 接受系統的 notification 權限請求。
  • 在終端機執行 /config，啟用 Push when Claude decides。
  • 也可以在 prompt 裡直接說：「notify me when the tests finish」。
如果收不到推播

iOS 的 Focus 模式與通知摘要可能延遲或壓抑通知；Android 上要把 Claude App 從電池優化白名單裡加進去。/config 顯示「No mobile registered」時，打開手機 App 讓它刷新 push token 即可。

§ 10 — Troubleshooting

常見錯誤的對應方式。

「Remote Control requires a claude.ai subscription」

沒用 claude.ai 帳號登入。執行 claude auth login 選 claude.ai 選項。環境裡有 ANTHROPIC_API_KEY 要先 unset。

「Remote Control requires a full-scope login token」

你用的是 claude setup-token 或 CLAUDE_CODE_OAUTH_TOKEN 的長效 token，這類 token 只能做 inference，不能開 Remote Control session。改用 claude auth login 取得 full-scope session token。

「Remote Control is not yet enabled for your account」

檢查環境變數：CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC、DISABLE_TELEMETRY 要 unset；CLAUDE_CODE_USE_BEDROCK／VERTEX／FOUNDRY 會讓 Remote Control 不可用（它只支援 claude.ai 認證）。都沒有的話，/logout 再 /login 刷新。

「Remote Control is disabled by your organization's policy」

先用 /status 看目前登入方式。三種可能原因：（1）用 API key 或 Console 帳號登入、（2）Team/Enterprise 管理員尚未啟用、（3）組織的 data retention 或 compliance 設定不相容，此時需要聯絡 Anthropic 支援。

「Remote credentials fetch failed」

加 --verbose 看完整錯誤：

claude remote-control --verbose

常見原因是沒登入、firewall 擋掉對外 HTTPS 443、或訂閱失效。

¶

重點整理：一張紙可以記住的事。

  • 啟動：新開用 claude --remote-control，中途用 /rc，想全開在 /config 打勾。
  • 連線：session URL、QR code、或 claude.ai/code session list 任選。
  • 權限：bypass 不能用，想少被打斷選 --permission-mode acceptEdits。
  • 界線：本機 process 要活著；Ultraplan 會中斷連線；一個 process 一個 remote session。
  • 安全：只有 outbound HTTPS，不開任何 inbound port，所有流量經 Anthropic API 以 TLS 轉發。
