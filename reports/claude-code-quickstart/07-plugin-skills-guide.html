什麼是 Skill？ Skill 是 Claude Code 的按需載入知識模組，透過 /plugin 安裝後自動啟用。當你的對話內容匹配 Skill 的觸發描述時，Claude 會自動調用對應 Skill 來增強回應品質。你也可以直接用 /skill-name 手動觸發。

安裝方式： 執行 /plugin 開啟插件管理介面，勾選需要的插件安裝。執行 /reload-plugins 重新載入已安裝的插件。

⚡ Superpowers — 核心開發工作流 (14 Skills)

最重要的插件，涵蓋從規劃到交付的完整開發生命週期。
規劃與設計
brainstorming /brainstorm
創意發想與需求探索。在任何創造性工作之前必須使用 — 建立功能、元件、新行為前，先探索使用者意圖、需求與設計方案，避免直接跳入實作。
建立功能、新增元件、修改行為前自動觸發
writing-plans
撰寫實作計畫。當你有規格或需求要進行多步驟任務時使用。產出結構化的分步計畫，包含檔案清單、依賴關係、風險評估。在動手寫程式碼之前先對齊方向。
有 spec 或多步驟任務需求時
executing-plans
執行實作計畫。在獨立 session 中按計畫逐步執行，設有 review checkpoint。適合將大型任務拆分到隔離環境中完成。
已有完成的實作計畫，準備執行時
開發與測試
test-driven-development
測試驅動開發（TDD）。在實作任何功能或修 bug 前，先撰寫測試案例，再寫實作程式碼。確保程式碼品質從一開始就有測試保障。
實作功能或修 bug 前
systematic-debugging
系統化除錯。遇到任何 bug、測試失敗或非預期行為時使用。不猜測原因，而是有系統地定位根因：重現問題 → 縮小範圍 → 驗證假設 → 修復。
遇到 bug、測試失敗、非預期行為時
subagent-driven-development
子代理驅動開發。在當前 session 中執行有獨立任務的實作計畫，將各任務分派給子代理並行處理。
計畫中有多個可並行的獨立任務時
dispatching-parallel-agents
平行代理調度。面對 2 個以上彼此獨立、不需共享狀態或循序執行的任務時，同時啟動多個代理並行處理。
2+ 獨立任務需要並行處理時
Git 與整合
using-git-worktrees
Git Worktree 隔離開發。需要隔離開發時建立 worktree，包含智慧目錄選擇和安全性驗證。適合執行實作計畫前建立獨立工作環境。
需要與目前工作區隔離的功能開發時
finishing-a-development-branch
完成開發分支。實作完成且測試通過後，引導你選擇整合方式：合併、建立 PR 或清理。提供結構化選項避免遺漏步驟。
實作完成、測試通過，準備整合時
Code Review 與驗證
requesting-code-review
發起 Code Review。完成任務、實作主要功能或合併前使用，驗證工作成果是否符合需求。
receiving-code-review
接收 Code Review 回饋。收到 review 意見時使用，特別是回饋不清楚或技術上有疑問時。要求技術嚴謹性，不盲目接受或敷衍同意。
verification-before-completion
完成前驗證。在宣稱工作完成、修復成功或測試通過之前，必須先執行驗證命令並確認輸出。「證據先於斷言」。
準備提交或建立 PR 前自動觸發
writing-skills
撰寫 Skill。建立新 Skill、編輯現有 Skill 或部署前驗證 Skill 是否正常運作。
using-superpowers
Superpowers 入口。開始任何對話時使用，建立如何找到和使用 Skills 的方式。確保在回應前先查找合適的 Skill。

🔀 Commit Commands — Git 工作流

commit /commit
智慧 Git 提交。自動分析變更內容、產生符合慣例的 commit message，處理 staging 和提交流程。
commit-push-pr /commit-push-pr
一鍵提交、推送、開 PR。串連 commit → push → create PR 的完整流程，自動產生 PR 標題和描述。
clean_gone /clean_gone
清理已刪除的遠端分支。清除所有標記為 [gone] 的本地分支（遠端已刪除但本地殘留的分支），包括相關的 worktree。

✅ 程式碼品質 & 文件

code-review /code-review
自動化 Code Review。對 Pull Request 進行全面的程式碼審查，檢查邏輯、風格、安全性與效能問題。
simplify /simplify
程式碼簡化。審查已變更的程式碼，檢查可重用性、品質與效率，自動修復發現的問題。
revise-claude-md
更新 CLAUDE.md。將本次 session 中學到的經驗更新到 CLAUDE.md，保持專案記憶最新。
claude-md-improver
審計 & 改善 CLAUDE.md。掃描所有 CLAUDE.md 檔案，評估品質、輸出報告，並進行針對性的更新改善。
使用者要求檢查、改善 CLAUDE.md 時

🔌 Plugin Dev — 插件開發工具組 (7 Skills)

完整的插件開發 SDK，從腳手架到上線一站式支援。
plugin-structure
插件腳手架。建立新插件、理解插件結構、設定 plugin.json、組織 commands/agents/skills/hooks。
「建立插件」「插件結構」等關鍵字
command-development
斜線命令開發。建立自訂斜線命令、定義命令參數、使用 frontmatter、組織命令檔案結構。
「建立斜線命令」「自訂命令」等
skill-development
Skill 開發。建立 Skill、新增到插件、組織 Skill 內容、設定漸進式揭露（progressive disclosure）。
「建立 skill」「新增技能」等
agent-development
子代理開發。建立代理定義檔、設定 frontmatter（工具限制、顏色、自動觸發）、agent 範例。
「建立 agent」「子代理」等
hook-development
Hook 開發。建立 PreToolUse/PostToolUse/Stop 等 hook、實作 prompt-based hook、使用 ${CLAUDE_PLUGIN_ROOT}。
「建立 hook」「驗證工具使用」等
mcp-integration
MCP 整合。在插件中加入 MCP 伺服器、設定 .mcp.json、連接外部服務（Model Context Protocol）。
「整合 MCP」「設定 MCP」等
plugin-settings
插件設定。儲存插件組態、建立使用者可配置的插件、使用 .local.md 和 YAML frontmatter。
「插件設定」「plugin config」等

🌐 Chrome DevTools MCP (4 Skills)

透過 MCP 連接 Chrome DevTools，實現瀏覽器自動化除錯。
chrome-devtools
瀏覽器除錯與自動化。除錯網頁、自動化瀏覽器互動、分析效能、檢查網路請求。
a11y-debugging
無障礙（Accessibility）審計。基於 web.dev 指南，測試語意 HTML、ARIA 標籤、焦點狀態、鍵盤導航、點擊目標、色彩對比。
debug-optimize-lcp
LCP 效能最佳化。使用 DevTools 工具分析 Largest Contentful Paint，找出慢載入原因並最佳化 Core Web Vitals。
troubleshooting
連線問題排除。當 list_pages、new_page 或 navigate_page 失敗時，排除 Chrome DevTools MCP 連線和目標問題。

🛠️ MCP Server Dev (3 Skills)

設計和建構 MCP 伺服器的完整工具組。
build-mcp-server
建構 MCP 伺服器。為 Claude 包裝 API、暴露工具（tools）、實作 Model Context Protocol。支援 Node.js 和 Python。
「建立 MCP」「包裝 API」等
build-mcpb
打包 MCP 伺服器。將 MCP 伺服器打包成 .mcpb 檔案，內含 Node 或 Python runtime，方便發佈和安裝。
「打包 MCP」「bundle MCP」等
build-mcp-app
建構 MCP App。加入互動式 UI、widgets 到 MCP 伺服器，在聊天中渲染表單、選擇器、儀表板等元件。
「MCP app」「MCP UI」等

frontend-design
高品質前端設計。建立獨特的、生產級的前端介面。產出有創意、打磨過的程式碼，避免通用的 AI 美學風格。適用於 web 元件、頁面或完整應用程式。
使用者要求建構 web 元件、頁面或應用程式時

⚙️ 工具 & 設定

update-config
設定 Claude Code。透過 settings.json 配置自動化行為。所有「每當 X 就 Y」的自動化都需要透過 hooks 設定。
keybindings-help
鍵盤快捷鍵自訂。重新綁定按鍵、加入組合鍵（chord bindings）、修改 ~/.claude/keybindings.json。
「自訂快捷鍵」「rebind keys」等
loop /loop
循環執行。定時重複執行 prompt 或斜線命令。例如 /loop 5m /foo 每 5 分鐘執行一次（預設 10 分鐘）。
schedule /schedule
排程遠端代理。建立、更新、列出或執行按 cron 排程執行的遠端代理（triggers）。

Claude API 開發。使用 Claude API 或 Anthropic SDK 建構應用程式。自動在程式碼 import anthropic 或 @anthropic-ai/sdk 時觸發。
程式碼 import anthropic SDK 時自動觸發
new-sdk-app
新建 Agent SDK 應用。建立和設定全新的 Claude Agent SDK 應用程式，包含專案初始化和基本架構。
skill-creator /skill-creator
Skill 建立器。從零建立 Skill、修改和最佳化現有 Skill、執行 eval 測試、基準效能分析、最佳化 Skill 的觸發描述準確度。
claude-automation-recommender
自動化建議。分析 codebase 並推薦適合的 Claude Code 自動化方案（hooks、subagents、skills、plugins、MCP servers）。
「推薦自動化」「設定 Claude Code」等

💡 使用技巧與最佳實踐

安裝與管理
安裝插件
/plugin → 選擇插件 → 安裝

重載插件
/reload-plugins 讓新安裝的插件生效

查看已安裝
/plugin 可查看已安裝和可用的插件列表

插件位置
~/.claude/plugins/cache/
觸發方式
自動觸發
對話內容匹配 Skill 描述時自動載入

手動觸發
直接輸入斜線命令如 /commit

上下文觸發
某些 Skill 在特定情境自動啟用（如 import SDK 時觸發 claude-api）

組合使用
多個 Skill 可以在同一對話中協作
推薦工作流
① 規劃階段
brainstorming → writing-plans

② 開發階段
using-git-worktrees → test-driven-development → executing-plans

③ 驗證階段
verification-before-completion → requesting-code-review

④ 交付階段
finishing-a-development-branch → commit-push-pr
