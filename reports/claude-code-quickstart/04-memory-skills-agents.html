📝 CLAUDE.md — 專案記憶系統（深入）

載入層級（全部累加）
~/.claude/CLAUDE.md全域 — 適用所有專案
~/project/CLAUDE.md專案根目錄 — 團隊共享（commit 到 git）
~/project/src/CLAUDE.md子目錄 — 模組特定指令
父目錄 CLAUDE.mdMonorepo — 從工作目錄向上搜尋到根目錄

全部累加：所有層級的內容同時載入 context。子目錄的 CLAUDE.md 在 Claude 存取到該目錄時才被發現。

建立方式
▸
/init — 自動掃描專案並產生初始 CLAUDE.md
▸
# 鍵 — 在對話中快速新增指令到 CLAUDE.md
▸
手動編輯 — 直接修改檔案
建議內容
▸
建構/測試命令 — npm run dev、npm test 等
▸
程式碼慣例 — 命名規則、import 風格、錯誤處理
▸
架構 — 目錄結構、模組關係、關鍵抽象層
▸
禁止事項 — "永遠不要直接 commit 到 main"
▸
工作流 — 部署流程、PR 流程、測試要求
最佳實踐：保持在 ~500 行以內。超過時，把參考資料移到 Skills，把條件性規則移到 .claude/rules/。

🧠 Auto Memory

Claude 自動儲存工作過程中學到的模式和偏好。

運作方式
▸
自動存到 ~/.claude/MEMORY.md
▸
每次 session 開始載入前 200 行
▸
記錄專案模式、你的偏好、修正過的錯誤
管理
▸
/memory — 查看 & 管理
▸
定期清理過時內容
CLAUDE.md vs Memory vs Rules
CLAUDE.md每 session 必載的穩定指令
MEMORY.md自動累積的學習（跨 session）
.claude/rules/條件性規則（用 paths 限定生效範圍）

📏 Rules 檔案

將 CLAUDE.md 拆分，保持主檔精簡。放在 .claude/rules/ 下。

# .claude/rules/typescript.md --- paths: - "src/**/*.ts" - "src/**/*.tsx" --- ## TypeScript 規則 - 使用 strict mode - 優先用 interface 而非 type - 禁止 any，用 unknown 替代

paths frontmatter：只在 Claude 處理匹配的檔案時才載入，節省 context。

🎯 Skills — 按需載入的知識 & 工作流（深入）

什麼是 Skill

Skill 是 Markdown 檔案 + 腳本 + 參考資料的組合。可透過斜線命令觸發，或 Claude 根據描述自動匹配。

兩種類型
▸
Reference Skill — 提供知識（API 文件、風格指南）
▸
Action Skill — 觸發特定工作流（/deploy）
載入時機

Session 開始時只載入描述（frontmatter）。完整內容在使用時才載入，實現漸進式揭露。

目錄結構
.claude/skills/ └── deploy/ ├── SKILL.md # 主指令 ├── scripts/ │ └── deploy.sh # 可執行腳本 ├── references/ │ ├── AWS.md # 按需參考 │ └── ROLLBACK.md └── assets/ └── template.yml
SKILL.md Frontmatter
--- name: deploy description: 執行部署流程 allowed-tools: Bash, Read, Write model: claude-sonnet-4-6 --- ## 部署流程 1. 執行測試套件 2. 建構 Docker image 3. 推送到 registry 4. 更新 k8s deployment
放置位置 & 優先順序
~/.claude/skills/個人 — 所有專案可用
.claude/skills/專案 — 團隊共享
Plugin skills透過 plugin 安裝（有命名空間）

同名覆寫：managed > user > project

何時用 Skill vs CLAUDE.md
▸
CLAUDE.md — 每次都需要知道的
▸
Skill — 有時需要的參考或工作流
▸
經驗法則：同樣指令給了第三次 → 做成 Skill
2026 統一：Commands 和 Skills 已整合。.claude/commands/ 仍可運作。

🤖 Subagents — 隔離式子代理（深入）

內建子代理
ExploreHaiku、唯讀、快速搜尋定位
Plan繼承模型、唯讀、研究規劃
General-purpose完整工具存取、複雜委派
claude-code-guide查找官方文件
Context 隔離

子代理有自己的 context window，不繼承父對話歷史。父 session 只收到摘要。

自訂子代理
# .claude/agents/security-reviewer.md --- name: security-reviewer description: 安全審查程式碼變更 tools: Read, Grep, Glob, Bash model: opus permissionMode: plan --- 你是資深安全工程師。檢查 OWASP Top 10、硬編碼秘密、SQL injection。
Frontmatter 可設定項
name代理名稱
description描述（觸發判斷）
tools可用工具清單
model使用的模型
permissionModeplan / default / bypass
skills預載入的 skills
hooks代理級別 hooks
前景 vs 背景
▸
背景 — Ctrl+B 送到背景
▸
管理 — /agents 建立/編輯/刪除

👥 Agent Teams (實驗性)

多個獨立 session 協作，支援共享任務 & peer-to-peer 訊息。

▸
每個代理有獨立 context window
▸
適合大型任務平行處理
vs Subagent：Subagent 在 session 內。Agent Team 是完全獨立的 session 互相協調。

📦 Plugins — 封裝 & 分發

什麼是 Plugin

把 skills + hooks + agents + MCP 打包成可安裝單元。

安裝 & 使用
# 本地安裝 claude plugin add --path ./my-plugin # marketplace claude plugin install my-plugin # 帶命名空間的 skill /my-plugin:review
Plugin 結構
my-plugin/ ├── .claude-plugin/ │ └── plugin.json ├── commands/ ├── agents/ ├── skills/ ├── hooks/ ├── .mcp.json └── README.md
優先順序

enterprise > user > project > plugin
