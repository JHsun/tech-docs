第 1 章 · 入門與環境

Shell 是介於使用者與作業系統核心之間的命令解譯器。當你在終端機輸入 ls、按下 Enter，把這串文字翻譯成系統呼叫的就是 shell。Shell Script，則是把一連串 shell 指令寫進檔案、讓電腦自動依序執行的程式。

這個語言看起來樸素、甚至有點老派，但它的份量在 2026 年不減反增 ── 每一台 Linux 伺服器、每一個 Docker 容器、每一條 CI/CD 流水線、幾乎每一個 macOS 系統，都預裝了 Bash。當你寫一支 shell 腳本，它在哪裡都跑得起來，不需要安裝任何東西。這份「無所不在」就是 shell 最大的價值。

1.1 Shell 家族總覽 ¶

Shell 不只一種。理解它們的差異，能幫你決定腳本要用哪種方言來寫：

Shell特性適用場景
sh (POSIX)標準化、最小語法、最大相容性跨系統部署腳本、Alpine 容器、嵌入式
bashGNU 擴充、陣列、關聯陣列、進階展開、Bash 5.3 新語法Linux 主流、生產環境腳本（最常用）
dashDebian/Ubuntu 的 /bin/sh，POSIX 嚴格系統開機腳本、需要極快啟動的場景
zsh互動體驗強、補全功能豐富個人 shell（macOS Catalina+ 預設）
fish使用者友好、語法不完全相容 POSIX互動使用，不適合寫部署腳本
💡 寫腳本要選哪個？

本手冊建議：Bash 為主、POSIX 為輔。Bash 在所有主流 Linux、macOS、WSL 都可用，且功能完整；當你需要在 Alpine（預設 ash）或極端輕量環境跑時，再退回 POSIX 子集。每個範例我們都會以徽章標註相容性。

1.2 Bash 5.3 — 你應該知道的最新語法 ¶

Bash 5.3（2025 年 7 月發布，距 5.2 三年）是這份手冊撰寫時的最新穩定版。它最重要的新功能是 「不 fork 的命令替換」，能在當前 shell 上下文執行命令、避免子行程開銷。

bash-5.3-new-substitution.sh Bash 5.3+ 
# 舊式：${ ... } 會 fork 一個子 shell，read 設的變數消失
result=$(read -r line < file.txt; echo "$line")

# Bash 5.3 新式 1：捕捉 stdout，但不 fork（速度快很多）
result=${ read -r line < file.txt; printf '%s' "$line"; }

# Bash 5.3 新式 2：用 REPLY 變數帶回結果
${| read -r line < file.txt; REPLY="$line"; }
echo "$REPLY"

其他值得注意的 5.3 新增：

  • GLOBSORT 變數可控制 * 展開的排序方式（依 name/size/mtime/ctime…，可升降冪）
  • compgen 新增 -V 選項，把補全結果寫入變數而非 stdout
  • read -E：用 readline 互動輸入並支援可程式化補全
  • source -p PATH：明確指定搜尋路徑，不依賴 $PATH
  • $EPOCHREALTIME：微秒級時間戳，方便計時
⚠️ 相容性提醒：macOS 的 Bash 困境

Bash 5.3 的新語法在 macOS 內建 Bash 完全不可用。Apple 自 2007 年的 Bash 3.2.57 之後就停止升級，原因是 Bash 4+ 改採 GPLv3 授權，Apple 不願受其條款約束。截至最新的 macOS Tahoe 26（2025 年 9 月發布、本手冊撰寫時最新為 2026/4 的 26.4.1），/bin/bash 仍停在 3.2.57，且 Catalina（2019）起預設 shell 已改為 zsh。

若需要在 macOS 上跑現代 Bash 腳本，三種解法：(1) Homebrew 安裝 brew install bash 後改 shebang 為 #!/usr/bin/env bash；(2) 限制腳本使用 Bash 3.2 通用語法（不能用關聯陣列、${var^^}、Bash 5.3 新語法等）；(3) 改用 zsh 或 POSIX sh 寫。

1.3 你的第一支腳本 ¶

把以下內容存成 hello.sh，然後一步步試：

hello.sh Bash 
#!/usr/bin/env bash
# 一支腳本最重要的開頭三行 ── 你會在後續每一支腳本看到
set -euo pipefail
IFS=$'\n\t'

readonly name="${1:-世界}"

echo "你好，${name}！"
echo "今天是 $(date '+%Y-%m-%d %H:%M:%S')"
echo "我是 PID $$，跑在 ${BASH_VERSION:-未知 shell}"

執行它：

terminal Bash 
# 方法 1：透過 bash 解譯器執行（不需要執行權限）
$ bash hello.sh
$ bash hello.sh Alice

# 方法 2：給予執行權限，直接呼叫
$ chmod +x hello.sh
$ ./hello.sh
$ ./hello.sh Alice

# 方法 3：在當前 shell 載入（不 fork 子行程，變數會留下）
$ source hello.sh
$ . hello.sh        # . 是 source 的 POSIX 寫法

1.4 Shebang 完整剖析 ¶

檔案第一行的 #! 叫 shebang（也叫 hashbang），告訴作業系統用哪個解譯器執行這支腳本。

寫法意義建議度
#!/bin/bash用絕對路徑找 bash⭐⭐⭐ 在 Linux 可，在 macOS（bash 3.2 老舊）會出問題
#!/usr/bin/env bash透過 env 從 PATH 找 bash⭐⭐⭐⭐⭐ 最推薦，跨平台最佳
#!/bin/sh用系統的 POSIX shell僅在你確定只用 POSIX 語法時使用
#!/usr/bin/env -S bash -euo pipefailshebang 直接帶選項（需 GNU env）進階用法，CI 環境慎用
⚠️ 反模式：沒有 shebang

不寫 shebang 時，腳本是用 呼叫者的 shell 執行 ── 同一支腳本，被 bash 呼叫和被 dash 呼叫行為可能完全不同。永遠加上 shebang。

1.5 執行權限與 chmod ¶

Unix 檔案有三組權限（owner / group / others），每組有 read（4）、write（2）、execute（1）三個位元。腳本要能直接呼叫，必須有 execute 權限。

chmod 範例 POSIX 
# 給自己執行權限（最常用）
chmod u+x hello.sh

# 給所有人讀和執行，只有自己能寫（推薦的部署腳本權限）
chmod 755 hello.sh
# 等同 chmod u=rwx,go=rx hello.sh

# 機敏腳本（含 API key 之類）：只有 owner 可讀寫執行
chmod 700 secret-deploy.sh

# 看權限（八進位＋字母兩種顯示）
stat -c '%a %A %n' hello.sh    # Linux
stat -f '%Lp %Sp %N' hello.sh  # macOS / BSD

1.6 開發環境建議 ¶

2026 年寫 shell 腳本，這四樣工具是「不裝就吃虧」：

  • ShellCheck ── 靜態分析器，能在你跑腳本前就抓出 90% 的常見錯誤。brew install shellcheck 或 apt install shellcheck。
  • shfmt ── 格式化工具，類似 Bash 界的 Prettier。go install mvdan.cc/sh/v3/cmd/shfmt@latest。
  • bats-core ── 單元測試框架，後面第 16 章會詳述。
  • 編輯器整合 ── VS Code 安裝 Bash IDE + shellcheck 擴充，或 Vim 用 ALE，能在打字時即時提示錯誤。
✅ 章節重點

Shell 是命令解譯器，腳本是命令的批次。Bash 是 2026 年最通用的選擇，5.3 帶來不 fork 的命令替換等新功能。寫腳本永遠加 shebang（首選 #!/usr/bin/env bash），用 chmod +x 賦予執行權，並在第一天就把 ShellCheck 裝起來。

第 2 章 · 變數與引號

變數和引號看似最基礎，卻是 shell 最大的地雷區。一個沒加引號的變數，可能讓你的腳本在遇到含空格的檔名時當場壞掉；一個誤用單引號的字串，可能讓變數展開失敗。引號規則的掌握程度，幾乎等同於 shell 程度本身。

2.1 變數的宣告與使用 ¶

變數基礎 POSIX 
# 宣告：變數名 = 值，等號兩邊「絕對」不能有空格
name="Alice"
age=30
city='Taipei'

# 使用：用 $ 取值，建議永遠用大括號 ${} 包起來
echo "$name"        # Alice
echo "${name}"      # 推薦寫法
echo "${name}_log"  # 必須用 {} 才不會被當成 $name_log

# 反例：等號旁邊有空格 ── 這是「執行 name 命令、傳入 = 和 Alice 兩個參數」
name = "Alice"   # ❌ 錯！
⚠️ 常見地雷：等號空格

幾乎每個初學者都會踩這個雷：name = "Alice" 是錯的，shell 會把 name 當成命令名執行。賦值時等號兩側絕對不能有空格。

變數的特殊宣告方式（Bash）

declare 與屬性 Bash 
# readonly：唯讀，宣告後不能改也不能 unset
readonly APP_NAME="myapp"
readonly -a SUPPORTED_LANGS=("zh" "en" "ja")  # 唯讀陣列

# declare：明確宣告類型與屬性
declare -i count=0           # integer 整數，自動算術
declare -l lower="HELLO"     # lowercase 自動轉小寫 → hello
declare -u upper="hello"     # uppercase 自動轉大寫 → HELLO
declare -r CONST="immut"     # readonly 等同 readonly CONST
declare -x EXPORTED="x"      # 匯出到子行程，等同 export

# export：讓變數被子行程繼承
export PATH="$HOME/bin:$PATH"
export EDITOR="vim"

# local：函式內的區域變數（必須在函式裡）
my_func() {
    local tmp_var="僅限此函式"
    local -i counter=0
}

# unset：刪除變數
unset EXPORTED

命名慣例

類型慣例範例
環境變數 / 全域常數UPPER_SNAKE_CASEDATABASE_URL、MAX_RETRIES
區域變數lower_snake_casefile_count、tmp_dir
避免單字母（除迴圈計數器外）、特殊符號❌ x、tmp1

2.2 引號的三種形式 ¶

Shell 有三種「引號」，行為完全不同。記住這張表能幫你解決 80% 的引號問題：

形式變數展開 $var命令替換 $(cmd)跳脫字元 \n萬用字元 *
雙引號 "..."✅ 展開✅ 展開✅ 處理❌ 不展開
單引號 '...'❌ 字面值❌ 字面值❌ 字面值❌ 字面值
無引號✅ 展開＋word splitting✅ 展開＋word splitting視情況✅ 展開（globbing）
引號實例 Bash 
name="World"

# 雙引號：展開變數
echo "Hello, $name"          # Hello, World

# 單引號：原樣輸出
echo 'Hello, $name'          # Hello, $name

# 無引號：當變數含空格時會災難
files="my file.txt"
ls $files                 # ❌ ls my file.txt → 錯誤地執行 ls 'my' 'file.txt'
ls "$files"               # ✅ ls 'my file.txt'

# 混用：把單引號塞進雙引號
echo "It's $name"           # It's World
echo 'It'\''s '"$name"      # 想在單引號裡放單引號的醜陋寫法

# Bash 額外提供 $'...'：C-style 字串，會處理跳脫字元
echo $'第一行\n第二行'      # 真的會換行
echo $'\t TAB \t'          # 真的有 tab
🚨 不引號 = 大部分 shell 災難的根源

沒加引號的變數會經歷 word splitting（依 IFS 分詞）和 globbing（萬用字元展開）。當變數值含空格、tab、換行、星號時會出意外。規則：對人類給的、外部來的、可能含空白的所有變數，永遠加雙引號。

2.3 IFS — 你應該認識的隱形變數 ¶

IFS（Internal Field Separator）控制 shell 怎麼把字串切成「字」。預設是「空格、tab、換行」三個字元，這也是 word splitting 的根源。

IFS 行為 Bash 
# 預設 IFS：空格、tab、換行
data="apple banana cherry"
for fruit in $data; do echo "[$fruit]"; done
# [apple]
# [banana]
# [cherry]

# 改用逗號當分隔符
csv="alice,bob,carol"
IFS=',' read -ra people <<< "$csv"
for p in "${people[@]}"; do echo "$p"; done

# 嚴格模式建議：限制 IFS 只用 \n 和 \t（不含空格）
IFS=$'\n\t'
# 這樣含空格的字串就不會被意外切割

2.4 特殊變數速查 ¶

變數意義範例
$0腳本檔名（含路徑）./deploy.sh
$1, $2, ... $9第 1～9 個參數傳給腳本的引數
${10} 起第 10 個之後必須用 {}$10 ≠ ${10}
$#參數個數用來檢查是否有傳參
$@所有參數（雙引號內視為獨立詞）幾乎永遠用 "$@"
$*所有參數（雙引號內合併成一個字串，以 IFS 第一字元相連）少用
$?上一個命令的退出碼（0 成功）錯誤處理核心
$$當前 shell 的 PID產生唯一暫存檔名
$!最近背景命令的 PID背景任務管理
$_上一個命令的最後一個引數互動式較常用
$-當前 shell 的選項旗標偵測是否互動模式
BASH_VERSIONBash 版本字串判斷 5.3+ 功能可用否
EPOCHSECONDSUnix 時戳（秒）Bash 5.0+
EPOCHREALTIMEUnix 時戳（微秒）Bash 5.0+，計時用
💡 "$@" vs "$*" 一次搞懂

假設腳本被呼叫為 ./script.sh "a b" "c d"：
"$@" 展開為兩個獨立的引數："a b" 和 "c d"
"$*" 展開為單一字串："a b c d"
當你要把所有參數原樣傳給另一個命令時，永遠用 "$@"。

2.5 變數作用域 ¶

scope.sh Bash 
#!/usr/bin/env bash
set -euo pipefail

global_var="我是全域"

demo() {
    local local_var="我只在 demo 函式裡"
    global_var="被函式修改"      # 仍會影響外部 ── 沒加 local
    echo "函式內：$local_var"
}

demo
echo "函式外的 global_var：$global_var"   # 「被函式修改」
echo "函式外的 local_var：${local_var:-不存在}"  # 「不存在」

# 子 shell 的變數修改不會影響父 shell
(
    global_var="子 shell 內修改"
    echo "子 shell 內：$global_var"
)
echo "回到父 shell：$global_var"  # 仍是「被函式修改」

第 3 章 · 參數展開

「參數展開（Parameter Expansion）」是 Bash 最強大、也最常被忽略的功能之一。它能直接在 ${...} 裡完成預設值、字串切割、替換、長度計算──不用呼叫任何外部程式（不需 sed、cut、basename）。掌握它能讓你的腳本快上數倍且更可讀。

3.1 預設值與替代值 ¶

語法意義當 var 未設或空時
${var:-default}取值，無則用預設（不修改原變數）傳回 default
${var:=default}取值，無則賦予並用預設賦予 var=default 並傳回
${var:?error msg}取值，無則印錯誤並退出印 error msg 到 stderr 並退出非 0
${var:+other}有值時用 other，無則空字串傳回空

注意：含 : 時「未設或空字串」皆觸發；不含 : 時僅「未設」觸發。

預設值實戰 BashPOSIX 
# 取環境變數，沒設就用預設
readonly LOG_DIR="${LOG_DIR:-/var/log/myapp}"
readonly PORT="${PORT:-8080}"

# 必填參數檢查 ── 沒設就直接死掉
readonly API_TOKEN="${API_TOKEN:?環境變數 API_TOKEN 必填}"

# 條件性附加：有 DEBUG 才加 -x flag
extra_flags="${DEBUG:+-x}"
bash $extra_flags sub-script.sh

# 第一次呼叫會初始化，後續呼叫沿用
echo "${TMPDIR:=/tmp}"   # 同時設定變數

3.2 字串長度與子字串 ¶

字串切片 Bash 
str="Hello, World!"

# 長度
echo "${#str}"            # 13

# 子字串：${var:offset:length}（offset 從 0 起）
echo "${str:7}"           # World!
echo "${str:7:5}"         # World
echo "${str:0:5}"         # Hello

# 負偏移要在前面加空格或 ()，避免與預設值語法 ${var:-...} 衝突
echo "${str: -1}"         # !
echo "${str:(-6)}"        # World!

3.3 移除前綴 / 後綴 ¶

語法意義
${var#pattern}從左移除最短匹配
${var##pattern}從左移除最長匹配
${var%pattern}從右移除最短匹配
${var%%pattern}從右移除最長匹配
前後綴移除實戰 BashPOSIX 
path="/home/alice/projects/myapp/main.sh"

# 取檔名（取代 basename）
echo "${path##*/}"        # main.sh ── 從左移除最長的 */

# 取目錄（取代 dirname）
echo "${path%/*}"         # /home/alice/projects/myapp

# 取副檔名
file="${path##*/}"         # main.sh
echo "${file##*.}"        # sh

# 去掉副檔名
echo "${file%.*}"         # main

# 實戰：把 .log 改 .log.gz
log="server.log"
echo "${log%.log}.log.gz" # server.log.gz

# URL 解析
url="https://api.example.com/v1/users"
echo "${url#*://}"        # api.example.com/v1/users
echo "${url%%/*}"         # https: ── 注意這個結果！
host="${url#*://}"
echo "${host%%/*}"        # api.example.com
💡 一個記法

# 在鍵盤左邊（移除左邊），% 在鍵盤右邊（移除右邊）。雙寫（## 或 %%）= 貪婪最長匹配，單寫 = 最短匹配。

3.4 字串替換 ¶

替換語法 Bash 
text="hello world hello bash"

# 替換第一次出現
echo "${text/hello/HI}"     # HI world hello bash

# 替換所有出現（兩個 //）
echo "${text//hello/HI}"    # HI world HI bash

# 只替換開頭（# 像前綴）
echo "${text/#hello/HI}"    # HI world hello bash

# 只替換結尾（% 像後綴）
echo "${text/%bash/BASH}"   # hello world hello BASH

# 刪除：替換成空字串
echo "${text// /_}"         # hello_world_hello_bash
echo "${text// /}"          # helloworldhellobash

# 大小寫轉換（Bash 4+）
name="alice"
echo "${name^}"             # Alice  ── 首字母大寫
echo "${name^^}"            # ALICE  ── 全部大寫
echo "${name~}"             # 切換首字母大小寫
echo "${name~~}"            # 切換全部大小寫

NAME="ALICE"
echo "${NAME,}"             # aLICE  ── 首字母小寫
echo "${NAME,,}"            # alice  ── 全部小寫

3.5 間接引用與展開列表 ¶

進階展開 Bash 
# 間接引用：用變數的值當變數名
real_var="hello"
ptr="real_var"
echo "${!ptr}"              # hello

# 列出所有 PREFIX_* 開頭的變數名
PREFIX_NAME="alice"
PREFIX_AGE=30
PREFIX_CITY="Taipei"
echo "${!PREFIX_@}"         # PREFIX_AGE PREFIX_CITY PREFIX_NAME

# 大括號展開（Brace Expansion，不是參數展開但常一起用）
echo {1..5}                # 1 2 3 4 5
echo {a..e}                # a b c d e
echo {01..10}              # 01 02 03 ... 10（補零）
echo {1..10..2}            # 1 3 5 7 9（步進）
echo file_{a,b,c}.txt      # file_a.txt file_b.txt file_c.txt
mkdir -p project/{src,test,docs,build}/{2025,2026}

3.6 參數展開速查綜合表 ¶

語法速記POSIX？
${#var}長度✅
${var:-x}沒值用 x✅
${var:=x}沒值用 x 並賦予✅
${var:?msg}沒值就死✅
${var:+x}有值才用 x✅
${var:offset}從 offset 開始❌ Bash
${var:offset:len}子字串❌ Bash
${var#p} / ##p砍前綴（短/長）✅
${var%p} / %%p砍後綴（短/長）✅
${var/old/new}替換首次❌ Bash
${var//old/new}替換全部❌ Bash
${var^} / ${var,,}大寫 / 小寫❌ Bash 4+
${!var}間接引用❌ Bash
${!prefix*}列出同前綴變數名❌ Bash

第 4 章 · 條件判斷

Shell 的條件判斷有三種語法：[ ... ]（POSIX，最古老）、[[ ... ]]（Bash 擴充，最強大）、(( ... ))（算術，數值專用）。三者場景不同，別搞混。

4.1 三種測試構造的差異 ¶

構造用途POSIX支援
[ ... ]傳統 test，字串/檔案/數值✅所有 shell
[[ ... ]]Bash 增強：正規表達式、無需引號、邏輯運算更安全❌Bash, ksh, zsh
(( ... ))純算術運算與比較❌Bash, ksh, zsh
test ...[ ] 的命令版本✅所有 shell
三種比較風格 Bash 
num=42
str="hello"
file="/etc/passwd"

# [ ] ── POSIX，需引號保護
if [ "$str" = "hello" ]; then echo "match"; fi
if [ "$num" -gt 10 ]; then echo "big"; fi
if [ -f "$file" ]; then echo "exists"; fi

# [[ ]] ── Bash，不需引號（但建議加），支援 =~ 正規
if [[ $str == "hello" ]]; then echo "match"; fi
if [[ $str =~ ^hel ]]; then echo "starts with hel"; fi
if [[ $str == h* ]]; then echo "glob match"; fi

# (( )) ── 算術比較，可用 < > == 等數學符號
if (( num > 10 )); then echo "big"; fi
if (( num % 2 == 0 )); then echo "even"; fi
⚠️ 用 [ ] 時的危險陷阱

[ $a = $b ] 在 $a 為空時會變成 [ = $b ]，語法錯誤！永遠寫成 [ "$a" = "$b" ]。
在 Bash 環境下優先用 [[ ]]，它對未引號變數安全許多，且支援更多功能。

4.2 字串比較 ¶

運算子意義[ ][[ ]]
= 或 ==相等=兩者皆可
!=不相等✅✅
< / >字典序比較需跳脫 \<直接用
-z str長度為 0（空字串）✅✅
-n str長度非 0✅✅
str =~ regex正規表達式匹配❌✅ Bash 限定
str == pattern萬用字元匹配（pattern 不加引號）❌✅ Bash 限定

4.3 數值比較 ¶

在 [ ] 和 [[ ]] 裡，數值比較用字母（-eq、-lt…），不能用 <、==（會被當字串）。在 (( )) 裡才能用數學符號。

運算子意義
-eq等於 (equal)
-ne不等於 (not equal)
-lt小於 (less than)
-le小於等於
-gt大於 (greater than)
-ge大於等於

4.4 檔案測試 ¶

運算子意義
-e file檔案存在（任何類型）
-f file是普通檔案（不是目錄/符號連結）
-d file是目錄
-L file是符號連結
-s file檔案存在且不為空
-r / -w / -x當前使用者可讀 / 寫 / 執行
-O file檔案屬於當前使用者
file1 -nt file2file1 比 file2 新（newer than）
file1 -ot file2file1 比 file2 舊
file1 -ef file2同一個檔案（同 inode）
檔案測試實戰 POSIX 
readonly CONFIG="/etc/myapp/config.yml"
readonly CACHE="/var/cache/myapp"

# 設定檔不存在就用預設
if [ ! -f "$CONFIG" ]; then
    echo "找不到設定檔，使用預設" >&2
    CONFIG="/etc/myapp/default.yml"
fi

# 確保快取目錄存在且可寫
if [ ! -d "$CACHE" ] || [ ! -w "$CACHE" ]; then
    mkdir -p "$CACHE" || {
        echo "無法建立快取目錄" >&2
        exit 1
    }
fi

# 檢查兩個檔案哪個新
if [ src.tar.gz -nt build/last_build ]; then
    echo "原始碼有變動，重新建置"
    rebuild
fi

4.5 邏輯運算與短路求值 ¶

邏輯組合 Bash 
# [[ ]] 內用 && 和 || ── 推薦
if [[ -f "$file" && -r "$file" ]]; then
    echo "可讀檔案"
fi

if [[ $status == "ok" || $status == "ready" ]]; then
    proceed
fi

# [ ] 內用 -a 和 -o（不建議，行為不穩定）
if [ -f "$file" -a -r "$file" ]; then ...; fi

# 推薦：[ ] 之間用 shell 的 && ||（每個 [ ] 是獨立命令）
if [ -f "$file" ] && [ -r "$file" ]; then ...; fi

# 短路求值：常見慣用法
[[ -d backup ]] || mkdir backup       # 沒有就建
command -v jq >/dev/null || { echo "需要 jq" >&2; exit 1; }

# 反值：!
if [[ ! -f "$file" ]]; then echo "缺檔"; fi

4.6 if / elif / else 完整結構 ¶

完整 if 結構 BashPOSIX 
# 注意：if 後面接的是「命令」，命令的退出碼決定走哪條
# [ ] 和 [[ ]] 也只是命令，退出 0 表 true，非 0 表 false

if command_a; then
    echo "a 成功"
elif command_b; then
    echo "a 失敗，但 b 成功"
else
    echo "都失敗"
fi

# 直接判斷命令成敗（不需要 [[ ]]）
if grep -q "ERROR" /var/log/app.log; then
    notify_admin
fi

# 多條件鏈
if [[ $status == "ok" ]] && ping -c1 -W1 example.com >/dev/null; then
    echo "全部正常"
fi

4.7 case 模式匹配 ¶

當你要根據一個變數值分多種情況處理，case 比一連串 elif 漂亮太多。

case 模式 BashPOSIX 
action="${1:-help}"

case "$action" in
    start)
        echo "啟動服務"
        ;;
    stop|halt|kill)             # | 是 OR
        echo "停止服務"
        ;;
    restart|reload)
        echo "重啟服務"
        ;;
    status|stat)
        echo "查詢狀態"
        ;;
    -v|--verbose)
        VERBOSE=1
        ;;
    *.txt|*.log)                # glob 模式
        echo "處理文字檔"
        ;;
    [0-9]*)                     # 字元類別：以數字開頭
        echo "看起來像數字: $action"
        ;;
    "")
        echo "什麼都沒傳"
        ;;
    *)                          # 萬用 default
        echo "未知動作: $action" >&2
        exit 1
        ;;
esac

# Bash 4+：用 ;;& 繼續匹配下一個（fall through）
case "$1" in
    *.tar*) echo "tar 系列" ;;&
    *.gz)   echo "gzip 壓縮" ;;
    *.bz2)  echo "bzip2 壓縮" ;;
esac
# input file.tar.gz → 印 "tar 系列" 和 "gzip 壓縮"
💡 case 比 if 好用的時機

當你有 3 個以上的分支、或要做 glob/萬用匹配時，用 case。它的可讀性遠勝串接的 elif，且效能也更好（不需要每次都呼叫 [[ ]]）。

第 5 章 · 迴圈

Shell 提供四種迴圈：for（遍歷列表）、while（條件成立時執行）、until（條件不成立時執行）、C-style for（傳統三段式）。每種有自己最適合的情境。

5.1 for 迴圈 ¶

for 各式用法 BashPOSIX 
# 1. 遍歷字面值列表
for color in red green blue; do
    echo "顏色：$color"
done

# 2. 遍歷檔案（用 glob）── 注意要加引號處理空白檔名
for file in *.txt; do
    [[ -f "$file" ]] || continue   # 沒有 *.txt 時 glob 會原樣保留
    echo "處理：$file"
    wc -l "$file"
done

# 設定 nullglob 讓沒有匹配時 glob 變成空（更安全）
shopt -s nullglob
for file in *.txt; do ...; done
shopt -u nullglob

# 3. 遍歷腳本所有參數 ── 標準慣用法
for arg in "$@"; do
    echo "[$arg]"
done

# 4. 數列：seq（POSIX）vs Brace Expansion（Bash）
for i in $(seq 1 5); do echo "$i"; done      # POSIX
for i in {1..5}; do echo "$i"; done            # Bash，更快

# 5. C-style for（Bash 限定，最像其他語言）
for ((i=0; i<10; i++)); do
    echo "i = $i"
done

for ((i=10; i>0; i--)); do
    echo "倒數 $i..."
    sleep 0.1
done
⚠️ 經典反模式：for f in $(ls *.txt)

用 $(ls) 取得檔名清單，遇到含空格的檔名會崩潰（被 word splitting 切碎）。正確做法：直接用 glob：for f in *.txt。
若要遞迴尋找，用 find ... -print0 | while IFS= read -r -d '' file（見 5.2）。

5.2 while 迴圈 ¶

while 經典用法 BashPOSIX 
# 1. 經典：條件迴圈
count=0
while [ $count -lt 5 ]; do
    echo "$count"
    count=$((count + 1))
done

# 2. 逐行讀檔 ── 標準慣用法（最重要！）
while IFS= read -r line; do
    echo "讀到: $line"
done < input.txt

# 3. 逐行讀並同時取得行號
n=0
while IFS= read -r line; do
    n=$((n + 1))
    printf '%4d  %s\n' "$n" "$line"
done < input.txt

# 4. 讀 CSV：用 IFS 切欄位
while IFS=',' read -r name age city; do
    echo "$name 住在 $city，$age 歲"
done < people.csv

# 5. 從命令輸出讀（用 process substitution，避免 subshell 陷阱）
while IFS= read -r file; do
    echo "處理 $file"
done < <(find . -name '*.log')

# 6. 處理含空白的檔名 ── find -print0 + read -d ''
while IFS= read -r -d '' file; do
    echo "安全處理: $file"
done < <(find . -name '*.log' -print0)

# 7. 無窮迴圈
while true; do
    check_health || break
    sleep 30
done
🚨 Subshell 陷阱：迴圈內變數消失

cmd | while read line; do count=$((count+1)); done 跑完後 $count 不會改變！因為 pipe 右側的 while 是在 subshell 裡執行的，變數修改隨 subshell 結束消失。
解法：用 process substitution while ... done < <(cmd)，或 Bash 5.3 的不 fork 命令替換。

5.3 until 迴圈 ¶

until 是 while 的反向 ── 條件為「假」時持續，「真」時跳出。實務上大多數場景兩者可互換，while 比較常見。

until 範例 BashPOSIX 
# 等待服務啟動
until curl -sf http://localhost:8080/health >/dev/null; do
    echo "等待服務啟動..."
    sleep 2
done
echo "服務已就緒"

# 等檔案出現
until [ -f /tmp/done.flag ]; do
    sleep 1
done

5.4 break、continue 與 break N ¶

流程控制 BashPOSIX 
# break：跳出迴圈
for i in {1..10}; do
    [[ $i -gt 5 ]] && break
    echo "$i"
done
# 印 1 2 3 4 5

# continue：跳過本次，進入下一次
for i in {1..10}; do
    (( i % 2 == 0 )) && continue
    echo "$i"
done
# 印 1 3 5 7 9

# break N / continue N：跳出 / 跳過 N 層
for outer in a b c; do
    for inner in 1 2 3; do
        if [[ $inner == 2 && $outer == b ]]; then
            break 2          # 直接跳出兩層
        fi
        echo "$outer-$inner"
    done
done

5.5 select：互動選單 ¶

select 是 Bash 一個常被忽略但很實用的功能 ── 自動產生編號選單，使用者輸入數字就選定。

select 選單 Bash 
#!/usr/bin/env bash
PS3="請選擇環境（輸入數字）："

select env in dev staging prod quit; do
    case "$env" in
        dev|staging|prod)
            echo "你選了 $env"
            deploy "$env"
            break
            ;;
        quit)
            echo "再見"
            break
            ;;
        *)
            echo "選項無效，請重試"
            ;;
    esac
done

# 執行畫面：
# 1) dev
# 2) staging
# 3) prod
# 4) quit
# 請選擇環境（輸入數字）：_

5.6 迴圈設計實戰：批次處理 ¶

batch-rename.sh Bash 
#!/usr/bin/env bash
# 把當前目錄所有 .jpeg 改名成 .jpg，並加上時間戳前綴
set -euo pipefail
shopt -s nullglob

readonly ts="$(date +%Y%m%d)"
count=0
skipped=0

for file in *.jpeg; do
    new="${ts}_${file%.jpeg}.jpg"

    if [[ -e "$new" ]]; then
        echo "略過 (已存在): $new" >&2
        skipped=$((skipped + 1))
        continue
    fi

    mv -- "$file" "$new"
    count=$((count + 1))
    echo "✓ $file → $new"
done

echo "完成：重命名 $count 個檔案，略過 $skipped 個"
