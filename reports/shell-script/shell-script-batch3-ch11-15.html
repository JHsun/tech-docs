第 11 章 · 命令列參數解析

一個專業的 shell 腳本，使用者應該能像呼叫 tar -xzf file.tar.gz 那樣傳入旗標。本章涵蓋三種解析方法：手動 shift、Bash 內建 getopts（短選項）、以及手刻長選項 case 迴圈。

11.1 三種風格速覽 ¶

方法支援短選項支援長選項可移植性使用建議
手動 shift✅✅✅ POSIX選項少且簡單時
內建 getopts✅❌（需手刻）✅ POSIX短選項為主，2026 年首選
外部 getopt✅✅⚠️ GNU vs BSD 不同跨平台請避免
手刻 case 迴圈✅✅✅ POSIX需要長選項時的最佳解
⚠️ 為什麼不推薦外部 getopt？

BSD 與 GNU 的 getopt 行為不同、互不相容，BSD 版本連空白引數都處理不了。Greg's Wiki（最權威的 Bash FAQ 之一）明確建議避免使用外部 getopt。要長選項，請用「手刻 case 迴圈」（見 11.4），這也是 2026 年大多數新腳本的選擇。

11.2 手動 shift（最簡單） ¶

manual-shift.sh POSIX 
#!/usr/bin/env bash
set -euo pipefail

# 適合：選項極少、固定順序（如「兩個位置參數」）
if [[ $# -lt 2 ]]; then
    echo "Usage: $0 SOURCE DEST" >&2
    exit 1
fi

readonly source=$1
readonly dest=$2
shift 2

# 剩下的當額外選項
extra_args=("$@")
echo "Source: $source"
echo "Dest: $dest"
echo "Extras: ${extra_args[@]}"

11.3 getopts：標準短選項解析 ¶

getopts 是 Bash 內建的命令，1986 年問世（SVR3 的 Bourne shell），1995 年進入 POSIX 標準。它專處理短選項（-v、-f file），不處理長選項。寫法熟悉後極為穩定。

getopts-template.sh BashPOSIX 
#!/usr/bin/env bash
set -euo pipefail

#
# getopts 規則速記：
#   "abc"   ── 三個選項 -a -b -c，都不帶值
#   "a:b"   ── -a 帶值（用 OPTARG 取），-b 不帶值
#   ":abc"  ── 開頭的冒號 = 「靜默模式」（自己處理錯誤訊息）
#

usage() {
    cat <<EOF
Usage: $0 [OPTIONS] FILE...

Options:
  -v          詳細模式
  -q          安靜模式
  -o FILE     輸出檔（預設 stdout）
  -n COUNT    處理數量（預設 10）
  -h          顯示說明

Examples:
  $0 -v -o out.txt -n 100 *.log
EOF
}

# 預設值
verbose=0
quiet=0
output=""
count=10

# 開頭加冒號 = 靜默模式，由我們自己印錯誤
while getopts ":vqo:n:h" opt; do
    case "$opt" in
        v) verbose=1 ;;
        q) quiet=1 ;;
        o) output="$OPTARG" ;;
        n) count="$OPTARG" ;;
        h) usage; exit 0 ;;
        \?) echo "未知選項: -$OPTARG" >&2; usage >&2; exit 2 ;;
        :)  echo "選項 -$OPTARG 需要參數" >&2; exit 2 ;;
    esac
done

# 移除已處理的選項，剩下的是位置參數
shift $((OPTIND - 1))

# 衝突檢查
if [[ $verbose -eq 1 && $quiet -eq 1 ]]; then
    echo "錯誤：-v 與 -q 不能同時使用" >&2
    exit 2
fi

# 必填位置參數
if [[ $# -eq 0 ]]; then
    echo "錯誤：至少需要一個 FILE" >&2
    usage >&2
    exit 2
fi

# 主邏輯
echo "Verbose: $verbose, Output: ${output:-stdout}, Count: $count"
for file in "$@"; do
    echo "處理: $file"
done
💡 OPTIND 與 OPTARG 兩個內建變數

OPTIND：getopts 內部維護的「下一個要處理的引數索引」，迴圈結束後做 shift $((OPTIND-1)) 把選項移除。
OPTARG：當前選項帶的值（只在 opt: 形式時有效）。
在函式內用 getopts 時記得 local OPTIND=1，否則第二次呼叫會異常。

11.4 手刻 case 迴圈：支援長選項 ¶

當你需要 --help、--output=file.txt、--verbose 這類長選項，手刻 case 迴圈是 2026 年的推薦做法──比外部 getopt 安全可移植，比庫函式輕量。

long-options-template.sh Bash 
#!/usr/bin/env bash
set -euo pipefail

usage() {
    cat <<EOF
Usage: $0 [OPTIONS] FILE...

Options:
  -v, --verbose         詳細模式
  -q, --quiet           安靜模式
  -o, --output FILE     輸出檔
  -n, --count N         處理數量（預設 10）
      --dry-run         不實際執行
  -h, --help            顯示說明

Long options also support --output=FILE form.
EOF
}

verbose=0
quiet=0
output=""
count=10
dry_run=0
positional=()

while [[ $# -gt 0 ]]; do
    case "$1" in
        -v|--verbose)
            verbose=1; shift ;;
        -q|--quiet)
            quiet=1; shift ;;
        --dry-run)
            dry_run=1; shift ;;

        # 帶參數的長選項：兩種形式都支援
        -o|--output)
            output="${2:?--output 需要參數}"
            shift 2 ;;
        --output=*)
            output="${1#*=}"
            shift ;;
        -n|--count)
            count="${2:?--count 需要參數}"
            shift 2 ;;
        --count=*)
            count="${1#*=}"
            shift ;;

        -h|--help)
            usage; exit 0 ;;

        # -- 表示後面全是位置參數，不再解析
        --)
            shift
            positional+=("$@")
            break ;;

        # 不認得的長選項
        --*)
            echo "未知選項: $1" >&2
            usage >&2
            exit 2 ;;

        # 不認得的短選項
        -*)
            echo "未知選項: $1" >&2
            usage >&2
            exit 2 ;;

        # 位置參數
        *)
            positional+=("$1")
            shift ;;
    esac
done

# 還原位置參數到 $@
set -- "${positional[@]}"

# 必填檢查
[[ $# -gt 0 ]] || { echo "至少需要一個檔案" >&2; exit 2; }

echo "verbose=$verbose output=${output:-stdout} count=$count dry=$dry_run"
echo "Files: $@"

第 12 章 · 錯誤處理

Shell 腳本的「預設行為」是極度危險的──任何指令失敗，後續仍會繼續執行。一個 cd /tmp/mydir 失敗後緊接 rm -rf *，可能就是一場災難。本章教你把腳本變成「有錯就停、會清理、能除錯」的可靠程式。

12.1 嚴格模式：set -euo pipefail ¶

2026 年的 shell 腳本，第一行（在 shebang 之後）幾乎都應該寫：

strict-mode-header.sh Bash 
#!/usr/bin/env bash
set -euo pipefail
IFS=$'\n\t'

這四個東西的意義：

選項行為解決什麼問題
-e (errexit)任何命令退出非 0 → 立即終止避免錯誤後仍繼續執行
-u (nounset)引用未定義變數 → 立即終止抓出 typo（如 $nme vs $name）
-o pipefailpipe 中任一命令失敗 → 整條 pipe 視為失敗抓出 cmd_might_fail \| sort 中的失敗
IFS=$'\n\t'分詞時不切空格，只切 tab/換行處理含空格的檔名與字串

沒有 pipefail 時的危險示範：

why-pipefail.sh Bash 
# 預設行為：pipe 的退出碼只看最後一個命令
grep "ERROR" /not/exist | sort
echo "$?"     # 0 ── grep 失敗了，但 sort 成功，整體被視為成功

set -o pipefail
grep "ERROR" /not/exist | sort
echo "$?"     # 2 ── 現在反映 grep 的失敗

12.2 嚴格模式的陷阱（必讀） ¶

嚴格模式不是萬靈丹，它有許多意外行為，不知道會被坑慘：

陷阱 1：((var++)) 在 set -e 下會死

trap-postincrement.sh Bash陷阱 
#!/usr/bin/env bash
set -e
i=0
((i++))             # ❌ 這行會讓腳本退出！
# 原因：(()) 回傳「結果值」當退出碼，i++ 的「結果」是 0（後綴自增的舊值）
#       退出碼 0 是 false，所以整個 (()) 失敗 → set -e 觸發
echo "永遠到不了這裡"

# 修正方法：用前綴 ++ 或 += 1
((++i))             # ✅
((i += 1))          # ✅
i=$((i + 1))        # ✅
((i++)) || true     # ✅ 也行（明確忽略失敗）

陷阱 2：set -e 在子 shell 中行為微妙

trap-subshell.sh Bash陷阱 
#!/usr/bin/env bash
set -e

# 子 shell 內的失敗，預設不會傳到父 shell
result=$(false; echo "after_false")
echo "$result"   # after_false ── false 沒有讓 $() 退出

# 修正：Bash 4.4+ 的 inherit_errexit
shopt -s inherit_errexit
result=$(false; echo "after_false")
# 現在 false 會讓 $() 子 shell 立即退出，而 set -e 也會傳到父 shell

陷阱 3：函式中的失敗，呼叫位置決定行為

trap-function.sh Bash陷阱 
set -e

risky() {
    false       # 在 if 條件下不會觸發 set -e
    echo "在函式內仍執行"
}

# 場景 A：函式直接呼叫 → false 後 set -e 生效，函式立刻死
risky                  # ❌ 中斷

# 場景 B：在 if 條件下 → set -e 不會觸發（這是 errexit 的特例）
if risky; then ...; fi     # ✅ 不中斷
risky && do_x                  # ✅ 不中斷
risky || do_y                  # ✅ 不中斷
⚠️ 嚴格模式不能取代主動錯誤檢查

嚴格模式是「安全網」，不是「正確程式碼」。重要邏輯仍要明確檢查：cmd || die "...訊息"。Aaron Maxwell 原文（"Unofficial Bash Strict Mode"）也強調：嚴格模式是預設保護，不是消除人為謹慎的理由。

12.3 trap：清理與訊號處理 ¶

trap 註冊「當收到某種訊號時要執行的命令」。最重要的兩個訊號：

訊號觸發時機典型用途
EXIT (0)腳本以任何方式結束清理暫存檔（最常用）
ERR命令以非 0 退出（同 set -e）記錄錯誤位置與細節
INT (2)使用者按 Ctrl+C提示中斷、收尾
TERM (15)kill 預設訊號正常終止前的清理
HUP (1)終端關閉背景任務的 reload

標準清理範本

trap-cleanup.sh Bash 
#!/usr/bin/env bash
set -euo pipefail
IFS=$'\n\t'

# 1. 建立暫存資源（在 trap 之前）
readonly tmp_dir=$(mktemp -d)
readonly lock_file="/var/run/myapp.lock"

# 2. 清理函式
cleanup() {
    local exit_code=$?
    if [[ $exit_code -eq 0 ]]; then
        echo "✓ 完成清理"
    else
        echo "✗ 失敗清理（exit $exit_code）" >&2
    fi

    rm -rf "$tmp_dir"
    rm -f "$lock_file"

    return "$exit_code"  # 維持原始退出碼
}

# 3. 註冊 trap ── 永遠把 EXIT 一起捕捉
trap cleanup EXIT INT TERM HUP

# 4. 主邏輯（不必擔心暫存檔）
echo "暫存目錄: $tmp_dir"
touch "$lock_file"
do_work_with "$tmp_dir"

進階：含 line + command 的 ERR trap

2026 年最佳實踐：用 trap ... ERR 印出失敗的精確位置與命令，比 set -e 的「靜默死亡」好用得多。

err-trap.sh Bash 
#!/usr/bin/env bash
set -Eeuo pipefail
# -E：讓 ERR trap 在子 shell / 函式內也生效（重要）

on_error() {
    local exit_code=$?
    local line_no=$1
    echo "─────────────────────────────" >&2
    echo "💥 ERROR in $0"           >&2
    echo "   Line:    $line_no"        >&2
    echo "   Command: $BASH_COMMAND"   >&2
    echo "   Exit:    $exit_code"       >&2
    echo "─────────────────────────────" >&2
    exit "$exit_code"
}

trap 'on_error $LINENO' ERR

# 故意失敗試試：
ls /this/does/not/exist
# 輸出：
# ─────────────────────────────
# 💥 ERROR in ./script.sh
#    Line:    18
#    Command: ls /this/does/not/exist
#    Exit:    2
# ─────────────────────────────

12.4 防呆 die 函式 ¶

die-helper.sh BashPOSIX 
# 標準慣用法：把 stderr + exit 1 包起來
die() {
    echo "Error: $*" >&2
    exit 1
}

# 進階：可選的退出碼
die() {
    local code=1
    [[ "${1:-}" =~ ^[0-9]+$ ]] && { code=$1; shift; }
    echo "Error: $*" >&2
    exit "$code"
}

# 使用
[[ -f "$config" ]] || die "找不到設定檔: $config"
[[ -r "$config" ]] || die 13 "設定檔不可讀（權限不足）"
command -v jq >/dev/null || die "需要 jq 工具"

12.5 除錯利器 ¶

debugging.sh Bash 
# 命令列方式
bash -n script.sh         # 語法檢查（不執行）
bash -x script.sh         # trace mode：印出每行
bash -v script.sh         # verbose：印出讀入的每行（含註解）
bash -xv script.sh        # 兩者皆有

# 在腳本內動態切換
set -x   # 開啟 trace
risky_section
set +x   # 關閉

# 美化 trace 輸出（PS4 控制 -x 的前綴）
export PS4='+ ${BASH_SOURCE##*/}:${LINENO}:${FUNCNAME[0]:-main} > '
set -x

# 條件式除錯：環境變數控制
[[ "${DEBUG:-0}" == "1" ]] && set -x

# 然後執行：
DEBUG=1 ./script.sh

第 13 章 · 進階主題

13.1 子 shell vs 命令群組 ¶

Shell 有兩種「把多個命令當一組」的語法：(...) 和 { ...; }。差別在前者建立子 shell，後者不建。

subshell-vs-group.sh BashPOSIX 
# (...) 子 shell：fork 一個新行程，變數隔離
x=1
(
    x=99
    cd /tmp        # 不影響父 shell 的 PWD
    echo "子: x=$x, pwd=$(pwd)"
)
echo "父: x=$x, pwd=$(pwd)"
# 子: x=99, pwd=/tmp
# 父: x=1, pwd=/原來的目錄

# {...} 群組：仍在當前 shell（不 fork），變數會留下
# 注意：{ 和 } 兩側必須有空格，且最後一條命令需有 ; 或換行
y=1
{
    y=99
    cd /tmp
    echo "群組: y=$y"
}
echo "外: y=$y"      # 99 ── 修改保留
pwd                # /tmp ── PWD 也變了

# 實用場景 1：群組共用重導向
{
    echo "=== Report ==="
    date
    df -h
} > report.txt 2>&1

# 實用場景 2：子 shell 隔離環境變數
(
    export DATABASE_URL="postgres://test/test"
    export DEBUG=1
    run_tests
)
# 出來後 DATABASE_URL 與 DEBUG 都還原

13.2 背景任務與 wait ¶

background-jobs.sh BashPOSIX 
# 用 & 把命令送到背景
sleep 5 &
pid=$!                          # 取得最後背景任務的 PID
echo "背景任務 PID: $pid"

# wait：等待任務結束（取得退出碼）
wait $pid
echo "任務退出碼: $?"

# 平行下載 + 等全部
download() {
    local url=$1
    curl -fsS "$url" > "out_$(basename "$url")"
}

declare -a pids
for url in "${urls[@]}"; do
    download "$url" &
    pids+=($!)
done

# 等全部完成
fail=0
for pid in "${pids[@]}"; do
    wait "$pid" || fail=$((fail + 1))
done
echo "失敗: $fail / ${#pids[@]}"

# jobs 列出當前 shell 的背景任務
jobs                            # [1]+ Running   sleep 5 &
jobs -p                         # 只印 PID

# 限制並行度（Bash 5.1+ 的 wait -n）
max_jobs=4
for task in "${tasks[@]}"; do
    # 若已達上限，等任一個完成
    while (( $(jobs -rp | wc -l) >= max_jobs )); do
        wait -n
    done
    process "$task" &
done
wait

13.3 訊號模型 ¶

每個訊號都有編號。kill -l 列出所有訊號。常用：

編號名稱意義可被 trap？
1HUP終端中斷（hangup）✅
2INT使用者中斷（Ctrl+C）✅
3QUITCtrl+\，產生 core dump✅
9KILL無條件終止❌
15TERMkill 預設訊號✅
17/19STOP暫停（Ctrl+Z 是 TSTP）❌（STOP 不可，TSTP 可）
18CONT恢復執行✅
—EXITshell 內部偽訊號（任何退出）✅（最常用）
💡 退出碼慣例

當訊號終止行程時，退出碼是 128 + signal_number。所以：
Ctrl+C 中斷 → exit code 130（128+2）
被 kill -9 → exit code 137（128+9）
被 kill 預設 → exit code 143（128+15）
腳本內可用這個慣例：exit 130 表示「使用者中斷」。

13.4 lock file：避免重覆執行 ¶

lock-file.sh Bash 
#!/usr/bin/env bash
set -euo pipefail

readonly LOCK_FILE="/var/lock/myapp.lock"

acquire_lock() {
    # flock：核心級檔案鎖（util-linux 提供）
    exec 200>"$LOCK_FILE"
    flock -n 200 || {
        echo "另一個實例正在執行" >&2
        exit 1
    }
    # 寫入 PID 方便除錯
    echo $$ >&200
}

acquire_lock
trap 'rm -f "$LOCK_FILE"' EXIT

# 主邏輯（保證只有一個實例執行）
echo "做事中..."
sleep 30

第 14 章 · 正規表達式

正規表達式（regex）是 shell 工具鏈的共通語言。同樣的模式語法，grep、sed、awk、Bash 的 [[ =~ ]] 都看得懂──但細節有微妙差異，本章把這些差異講清楚。

14.1 三種 regex 方言 ¶

方言全名使用場景
BREBasic Regular Expression傳統 grep、sed 預設
EREExtended Regular Expressiongrep -E、egrep、awk、Bash [[ =~ ]]
PCREPerl-Compatible Regexgrep -P、Perl、Python

關鍵差異：BRE 中許多元字元需要跳脫（\(\)、\+、\?、\{\}），ERE 不用。本手冊全用 ERE 為基準。

14.2 ERE 完整速查 ¶

類別語法意義
定位^行首
$行尾
\b單字邊界（PCRE）
\<單字開頭（GNU）
\>單字結尾（GNU）
字元類.任意字元（除換行）
[abc]a 或 b 或 c
[a-z]小寫字母
[^abc]除了 a/b/c
[[:alpha:]]POSIX 字符類（含中文等本地字母）
\d \w \s數字 / 單字字元 / 空白（PCRE）
量詞*0 或多次
+1 或多次（ERE）
?0 或 1 次（ERE）
{n}正好 n 次
{n,}至少 n 次
{n,m}n 到 m 次
分組(...)分組（ERE）
(?:...)非捕獲分組（PCRE）
\1 \2反向參考
選擇a\|b / a\|b\|ca 或 b（ERE）
前後查詢(?=...)positive lookahead（PCRE）
(?<=...)positive lookbehind（PCRE）

14.4 Bash 的 [[ =~ ]] 與 BASH_REMATCH ¶

Bash 的 [[ string =~ regex ]] 用 ERE 進行匹配，並把捕獲組存入 BASH_REMATCH 陣列。這是 shell 內進行字串解析的最強武器，不用呼叫 grep/sed。

bash-regex.sh Bash 
# 基本用法
str="version 2.10.5-rc1"
if [[ "$str" =~ ^version[[:space:]]+([0-9]+)\.([0-9]+)\.([0-9]+) ]]; then
    major=${BASH_REMATCH[1]}
    minor=${BASH_REMATCH[2]}
    patch=${BASH_REMATCH[3]}
    echo "Major=$major Minor=$minor Patch=$patch"
fi

# BASH_REMATCH[0] 是整段匹配，[1]+ 是各捕獲組
echo "全部匹配: ${BASH_REMATCH[0]}"   # version 2.10.5

# 重要：regex 不要加引號（會被當字面值）
if [[ "$file" =~ \.tar\.gz$ ]]; then     # ✅ 對
[[ "$file" =~ "\.tar\.gz$" ]]            # ❌ 變字面值

# 把 regex 存變數可避免引號問題
pattern='^[A-Z][a-z]+$'
[[ "$word" =~ $pattern ]] && echo "PascalCase"

# 實戰：解析 IP 位址
ip="192.168.1.100"
if [[ "$ip" =~ ^([0-9]{1,3})\.([0-9]{1,3})\.([0-9]{1,3})\.([0-9]{1,3})$ ]]; then
    for i in 1 2 3 4; do
        octet=${BASH_REMATCH[$i]}
        (( octet >= 0 && octet <= 255 )) || { echo "無效 IP"; exit 1; }
    done
    echo "有效 IP"
fi

# 實戰：從 log 抽取結構化資料
log='[2026-04-29 10:23:45] ERROR Failed to connect: timeout'
if [[ "$log" =~ ^\[([0-9-]+\ [0-9:]+)\]\ ([A-Z]+)\ (.+)$ ]]; then
    timestamp=${BASH_REMATCH[1]}
    level=${BASH_REMATCH[2]}
    message=${BASH_REMATCH[3]}
    echo "Time: $timestamp, Level: $level, Msg: $message"
fi

#

readonly VERSION="1.0.0"

die()  { echo "[$(date '+%H:%M:%S')] ERROR: $*" >&2; exit 1; }

usage()() {
    cat <<EOF
$SCRIPT_NAME v$VERSION - 一句話說明

Options:
  -v, --verbose   詳細輸出
  -o, --output    輸出檔
  -h, --help      顯示說明

# ─── 清理 trap ─────────────────────────────
cleanup() {
    local rc=$?
    [[ -n "${tmp_dir:-}" && -d "$tmp_dir" ]] && rm -rf "$tmp_dir"
    return "$rc"
}

on_error() {
    local rc=$? line=$1
    warn "Failed at line $line: $BASH_COMMAND"
    exit "$rc"
}

trap cleanup EXIT
trap 'on_error $LINENO' ERR

# ─── 依賴檢查 ──────────────────────────────

    command -v "$1" >/dev/null || die "需要 '$1' 但找不到"
}

# ─── 解析參數 ──────────────────────────────
verbose=0
output=""
positional=()

while [[ $# -gt 0 ]]; do
    case "$1" in
        -v|--verbose) verbose=1; shift ;;
        -o|--output)  output=${2:?--output 需要值}; shift 2 ;;
        --output=*)   output=${1#*=}; shift ;;
        -h|--help)    usage; exit 0 ;;
        --version)    echo "$SCRIPT_NAME v$VERSION"; exit 0 ;;
        --)           shift; positional+=("$@"); break ;;
        -*)           die "未知選項: $1" ;;
        *)            positional+=("$1"); shift ;;
    esac
done
set -- "${positional[@]}"

[[ $# -gt 0 ]] || { usage >&2; die "至少需要一個檔案"; }

main() {
    require_cmd jq

    tmp_dir=$(mktemp -d)
    log "暫存目錄: $tmp_dir"

    for file in "$@"; do
        [[ -f "$file" ]] || { warn "略過不存在的檔案: $file"; continue; }
        log "處理 $file"
        # ... 真正的工作 ...
    done

    log "完成"
}

#!/usr/bin/env bash
# 將 SOURCE 備份到 BACKUP_DIR，保留 N 天
set -Eeuo pipefail

readonly SOURCE="${1:?用法: $0 SOURCE BACKUP_DIR [KEEP_DAYS]}"
readonly BACKUP_DIR="${2:?用法: ...}"

mkdir -p "$BACKUP_DIR"

    --exclude='node_modules' \
    --exclude='.git' \
    --exclude='*.log' \
    --exclude='__pycache__' \
    -C "$(dirname "$SOURCE")" "$(basename "$SOURCE")"

echo "[$(date)] 備份完成: $(du -h "$archive" | cut -f1)"

# 清理舊備份（依 mtime）
find "$BACKUP_DIR" -name 'backup-*.tar.gz' -mtime +$KEEP_DAYS -print -delete

echo "[$(date)] 清理 $KEEP_DAYS 天前的備份完成"

15.3 範本 C：服務健康檢查 + 通知 ¶

health-check.sh Bash 
#!/usr/bin/env bash
set -Eeuo pipefail

readonly URL=${1:?用法: $0 URL [TIMEOUT_SEC]}
readonly TIMEOUT=${2:-5}
readonly SLACK_WEBHOOK="${SLACK_WEBHOOK:-}"

notify() {
    local emoji=$1 message=$2
    local hostname; hostname=$(hostname)
    local payload
    payload=$(printf '{"text": "%s [%s] %s"}' "$emoji" "$hostname" "$message")

    [[ -z "$SLACK_WEBHOOK" ]] && { echo "$emoji $message"; return; }
    curl -fsS -X POST -H 'Content-Type: application/json' \
        -d "$payload" "$SLACK_WEBHOOK" >/dev/null
}

# curl 拿到 status code 與耗時
output=$(curl -sS -o /dev/null \
    -w '%{http_code} %{time_total}\n' \
    --max-time "$TIMEOUT" \
    "$URL" 2>&1) || {
    notify "🔴" "$URL 連線失敗（超時 ${TIMEOUT}s）"
    exit 1
}

read -r status duration <<< "$output"

if [[ "$status" =~ ^2 ]]; then
    echo "✅ $URL OK ($status, ${duration}s)"
else
    notify "🔴" "$URL 異常狀態碼 $status"
    exit 1
fi

15.4 範本 D：批次重命名（dry-run + 互動確認） ¶

batch-rename.sh Bash 
#!/usr/bin/env bash
# 用 sed 規則批次重命名檔案，預設 dry-run
set -Eeuo pipefail
shopt -s nullglob

usage() {
    cat <<EOF
Usage: $0 [-y] PATTERN REPLACEMENT GLOB

Options:
  -y    跳過確認，直接執行（CI 用）

Examples:
  $0 'IMG_' 'photo_' '*.jpg'              # dry-run
  $0 -y 's/[ ]+/_/g' '' '*.txt'           # 直接執行
EOF
}

yes_mode=0
while getopts ":yh" opt; do
    case "$opt" in
        y) yes_mode=1 ;;
        h) usage; exit 0 ;;
        *) usage >&2; exit 2 ;;
    esac
done
shift $((OPTIND - 1))

[[ $# -eq 3 ]] || { usage >&2; exit 2; }
pattern=$1 replacement=$2 glob=$3

# 收集要重命名的檔案
declare -a renames
for file in $glob; do
    [[ -f "$file" ]] || continue
    new=$(echo "$file" | sed -E "s/$pattern/$replacement/g")
    [[ "$file" != "$new" ]] && renames+=("$file|$new")
done

[[ ${#renames[@]} -eq 0 ]] && { echo "沒有檔案需要重命名"; exit 0; }

# 預覽
echo "── 預覽 (${#renames[@]} 個檔案) ──"
for r in "${renames[@]}"; do
    echo "  ${r%|*}  →  ${r#*|}"
done

# 互動確認
if [[ $yes_mode -eq 0 ]]; then
    read -r -p "確定執行？[y/N] " ans
    [[ "$ans" =~ ^[Yy]$ ]] || { echo "已取消"; exit 0; }
fi

# 執行
for r in "${renames[@]}"; do
    src=${r%|*} dst=${r#*|}
    [[ -e "$dst" ]] && { echo "略過 (已存在): $dst" >&2; continue; }
    mv -- "$src" "$dst"
    echo "✓ $src → $dst"
done
