第 6 章 · 陣列

當你需要儲存多個相關的值──一串檔名、一份伺服器列表、一個設定字典──就要用陣列。Bash 提供兩種陣列：索引陣列（indexed array，用整數索引）和 關聯陣列（associative array，用字串當索引）。POSIX sh 完全沒有陣列功能，這是 Bash 比純 sh 強很多的關鍵。

6.1 索引陣列：宣告與基本操作 ¶

indexed-array-basics.sh Bash 
# 三種宣告方式
fruits=("apple" "banana" "cherry")             # 字面值
declare -a colors=("red" "green" "blue")        # 顯式宣告
servers[0]="web1"; servers[1]="web2"           # 逐一賦值

# 取單一元素
echo "${fruits[0]}"          # apple
echo "${fruits[1]}"          # banana

# 取「全部」── 兩種寫法的差異是一切的關鍵
echo "${fruits[@]}"          # apple banana cherry（每個元素獨立）
echo "${fruits[*]}"          # 在無引號時相同

# 在雙引號內，差異很大！
for f in "${fruits[@]}"; do echo "[$f]"; done
# [apple] [banana] [cherry]  ── 三個獨立元素，正確

for f in "${fruits[*]}"; do echo "[$f]"; done
# [apple banana cherry]  ── 整個合併成一個字串！

# 元素個數
echo "${#fruits[@]}"         # 3

# 單一元素的長度
echo "${#fruits[0]}"         # 5（"apple"）

# 所有索引
echo "${!fruits[@]}"         # 0 1 2
⚠️ 黃金規則：永遠用 "${array[@]}"

遍歷或傳遞陣列時，用 "${array[@]}"（雙引號 + @）。"${array[*]}" 會把所有元素合併成一個字串；無引號的 ${array[@]} 會做 word splitting，含空格的元素會被切碎。記住這條一輩子受用。

6.2 索引陣列：進階操作 ¶

array-operations.sh Bash 
arr=(a b c d e)

# 切片：${arr[@]:offset:length}
echo "${arr[@]:1:3}"          # b c d
echo "${arr[@]:2}"            # c d e（從 index 2 到結尾）

# 追加
arr+=("f" "g")
echo "${arr[@]}"              # a b c d e f g

# 插入到開頭（用陣列展開）
arr=("0" "${arr[@]}")
echo "${arr[@]}"              # 0 a b c d e f g

# 修改
arr[2]="B"

# 刪除單一元素 ── 注意：unset 後該位置的索引會「跳號」
unset 'arr[3]'                  # 必須用引號（單引號避免 glob）
echo "${!arr[@]}"              # 0 1 2 4 5 6 7（少了 3）

# 重新打包索引（去除空洞）
arr=("${arr[@]}")
echo "${!arr[@]}"              # 0 1 2 3 4 5 6（連續）

# 反向遍歷
for ((i=${#arr[@]}-1; i>=0; i--)); do
    echo "${arr[i]}"
done

# 將命令輸出讀入陣列（每行一個元素）── Bash 4+ 推薦做法
mapfile -t lines < input.txt
# 等同 readarray -t lines < input.txt
echo "讀到 ${#lines[@]} 行"

# 從命令輸出讀
mapfile -t users < <(cut -d: -f1 /etc/passwd)
💡 mapfile vs read 迴圈

讀檔成陣列時，mapfile -t（Bash 4+）比傳統的 while read 迴圈快數倍，且程式碼更簡潔。-t 會去掉每行尾的換行符。2026 年寫 Bash 4+ 腳本，這是首選。

6.3 關聯陣列（dictionary / hashmap） ¶

關聯陣列是 Bash 4 才加入的功能，可以用字串當 key。寫腳本時需要「以 ID 查名稱」「以名稱對應設定」這類映射，就用它。

associative-array.sh Bash 
# 必須用 declare -A 宣告，這是「associative」的 A
declare -A user_info

# 賦值
user_info["name"]="Alice"
user_info["age"]=30
user_info["role"]="admin"

# 一次設多個
declare -A status_color=(
    [pending]="yellow"
    [success]="green"
    [failed]="red"
)

# 取值
echo "${user_info[name]}"        # Alice
echo "${status_color[failed]}"   # red

# 所有 keys / values
echo "${!user_info[@]}"          # name age role（順序不保證）
echo "${user_info[@]}"           # Alice 30 admin

# 元素個數
echo "${#user_info[@]}"          # 3

# 檢查 key 是否存在（注意：不能用 [[ -n ${a[k]} ]]，那只判斷值非空）
if [[ -v "user_info[email]" ]]; then
    echo "有 email"
else
    echo "沒 email"
fi

# 遍歷
for key in "${!user_info[@]}"; do
    echo "$key = ${user_info[$key]}"
done

# 刪除
unset 'user_info[role]'
⚠️ 關聯陣列的限制

(1) 順序不保證，遍歷的順序可能與插入順序不同。
(2) 不能直接「巢狀」，即陣列裡不能放陣列。要放複雜結構需用 nameref（見 6.4）。
(3) macOS 內建 Bash 3.2 完全不支援，需 Homebrew 安裝 Bash 4+。

6.4 Nameref：陣列傳給函式的現代方法 ¶

陣列怎麼傳給函式？傳統做法是 "${arr[@]}" 攤平成參數，但無法傳遞「修改後的結果」回去。Bash 4.3 起的 declare -n（nameref，名稱參考）是現代解法。

array-nameref.sh Bash 
#!/usr/bin/env bash
set -euo pipefail

# 函式接收一個陣列名稱，並能直接修改它
add_prefix() {
    local -n arr_ref=$1     # nameref 到呼叫者傳入的陣列名
    local prefix=$2
    local i
    for i in "${!arr_ref[@]}"; do
        arr_ref[i]="${prefix}${arr_ref[i]}"
    done
}

names=("alice" "bob" "carol")
add_prefix names "user_"
echo "${names[@]}"            # user_alice user_bob user_carol

# 用於關聯陣列也一樣
print_dict() {
    local -n dict=$1
    local k
    for k in "${!dict[@]}"; do
        printf '%-12s = %s\n' "$k" "${dict[$k]}"
    done
}

declare -A config=(
    [host]="api.example.com"
    [port]="8080"
    [protocol]="https"
)
print_dict config
✅ Nameref 是現代寫法

2026 年寫 Bash 4.3+ 腳本傳遞陣列，nameref 應該是首選。它取代了舊式的 ${!var} 間接引用，語法更清晰、語意更明確、配合 local 也不會污染呼叫者命名空間。

第 7 章 · 函式

函式（function）是把一段腳本邏輯封裝起來、以名稱呼叫、可重複使用的單元。Bash 函式有它的怪癖（最重要的：不能像其他語言一樣 return "字串"），但只要掌握那幾條規則，函式就是組織複雜腳本的關鍵。

7.1 定義與呼叫 ¶

function-basics.sh BashPOSIX 
# 定義函式：兩種語法

# 寫法 1：function 關鍵字（Bash 風格，POSIX 不支援）
function greet() {
    echo "Hello, $1!"
}

# 寫法 2：name() {} （POSIX 標準，推薦）
greet() {
    echo "Hello, $1!"
}

# 呼叫：直接寫函式名，不加括號（不像 C/Python）
greet "World"
greet "Alice"

# 取消函式定義
unset -f greet

7.2 參數傳遞與位置參數 ¶

函式內的 $1 $2 ... $@ $# 與腳本層級意義完全相同──但作用域是函式自己的，呼叫結束就還原。

function-params.sh BashPOSIX 
show_args() {
    echo "我收到 $# 個參數"
    echo "第 1 個: $1"
    echo "第 2 個: $2"
    echo "全部: $@"
    echo "腳本名稱仍是: $0"
}

show_args alice "bob smith" carol
# 我收到 3 個參數
# 第 1 個: alice
# 第 2 個: bob smith   ← 注意有引號保護
# 第 3 個: carol
# $0 不會變成 "show_args"，仍是腳本名

# 慣用法：把參數命名（讓函式內易讀）
deploy() {
    local env=${1:?缺少環境名稱}
    local version=${2:?缺少版本號}
    local dry_run=${3:-false}

    echo "部署 v$version 到 $env（dry_run=$dry_run）"
}
deploy staging 2.1.0
deploy prod 2.1.0 true

7.3 區域變數：local 是必修 ¶

Bash 函式內預設變數仍是全域的──和 C、Python 完全相反。沒加 local 的變數會污染呼叫者的命名空間，這是 shell 腳本最大的副作用陷阱。

local-vs-global.sh Bash 
count=0   # 全域

leaky() {
    count=99       # ❌ 修改全域，副作用！
    tmp="hello"    # ❌ 創建全域變數，污染！
}

clean() {
    local count=99       # ✅ 區域，函式結束就消失
    local tmp="hello"    # ✅ 不影響呼叫者
    local -i n=10         # ✅ 區域整數
    local -a items=(a b c) # ✅ 區域陣列
}

leaky
echo "leaky 後 count = $count"  # 99 ── 被改了
echo "leaky 後 tmp = $tmp"      # hello ── 多出來

count=0
unset tmp
clean
echo "clean 後 count = $count"  # 0 ── 安全
echo "clean 後 tmp = ${tmp:-未設}"  # 未設 ── 安全
🚨 鐵律：函式內所有變數都加 local

除非你明確要修改全域變數，否則函式內每個變數宣告都該加 local。這是寫可維護 shell 腳本的第一條規則，也是 ShellCheck 會強烈提示的事。

7.4 回傳值：return 不是你想的那樣 ¶

Bash 的 return 只能回傳 0–255 的整數作為退出碼，不能回傳字串、陣列或任意值。要回傳資料有四種模式，按推薦順序：

模式 1：echo 到 stdout，呼叫者用命令替換接

return-via-stdout.sh BashPOSIX 
get_full_name() {
    local first=$1 last=$2
    echo "$first $last"        # 「回傳」就是 print
}

name=$(get_full_name "Alice" "Wang")
echo "$name"   # Alice Wang

缺點：每次呼叫都會 fork 子 shell（$(...) 的開銷），高頻迴圈中效能不佳。

模式 2：Nameref（Bash 4.3+，推薦）

return-via-nameref.sh Bash 
get_full_name() {
    local -n result_ref=$1   # 第 1 參數是「要寫入的變數名」
    local first=$2 last=$3
    result_ref="$first $last"
}

get_full_name name "Alice" "Wang"
echo "$name"   # Alice Wang ── 不用 fork

# 也能回傳陣列
get_unique_words() {
    local -n out_ref=$1
    shift
    local -A seen
    out_ref=()
    for w in "$@"; do
        [[ -v "seen[$w]" ]] || { out_ref+=("$w"); seen[$w]=1; }
    done
}

declare -a unique
get_unique_words unique apple banana apple cherry banana
echo "${unique[@]}"    # apple banana cherry

模式 3：return 退出碼（給 true/false 場景）

return-status.sh BashPOSIX 
is_user_admin() {
    local user=$1
    [[ "$user" == "root" || "$user" == "admin" ]] && return 0
    return 1
}

# 函式變成像 [[ ]] 一樣可被 if 判斷
if is_user_admin "alice"; then
    echo "admin"
else
    echo "regular"
fi

# 自訂錯誤碼（0=成功, 非0=不同錯誤類型）
fetch_data() {
    [[ -z "$1" ]] && return 2     # 缺參數
    [[ -f "$1" ]] || return 3     # 檔案不存在
    cat "$1" || return 4         # 讀檔失敗
    return 0
}

模式 4：全域變數（盡量避免，但有時最簡單）

return-via-global.sh Bash慎用 
# 約定：全大寫 + 函式前綴，明示這是「函式輸出」
RESULT_NAME=""
RESULT_AGE=0

parse_user() {
    RESULT_NAME=$(echo "$1" | cut -d, -f1)
    RESULT_AGE=$(echo "$1" | cut -d, -f2)
}

parse_user "Alice,30"
echo "$RESULT_NAME: $RESULT_AGE"
💡 怎麼選？

需要 true/false 判斷 → 模式 3（return 0/1）
回傳單一值且呼叫頻率不高 → 模式 1（echo + $(...)）
回傳陣列或關聯陣列、需要高效能、Bash 4.3+ → 模式 2（nameref）
不得不用 → 模式 4（全域），但全域名稱要明顯標示

7.5 函式與遞迴 ¶

recursive.sh Bash 
#!/usr/bin/env bash

# 階乘（注意：local 在遞迴中至關重要）
factorial() {
    local -i n=$1
    (( n <= 1 )) && { echo 1; return; }
    local -i prev=$(factorial $((n - 1)))
    echo $((n * prev))
}

factorial 5    # 120
factorial 10   # 3628800

# 遞迴目錄遍歷（雖然 find 通常更好）
walk() {
    local dir=$1
    local entry
    for entry in "$dir"/*; do
        [[ -e "$entry" ]] || continue
        echo "$entry"
        [[ -d "$entry" ]] && walk "$entry"
    done
}
walk /etc/cron.d
⚠️ 遞迴在 shell 中很慢

每層遞迴都是函式呼叫，加上 $(...) 的 fork 成本，shell 遞迴比迴圈慢數十倍。能用迴圈或 find、tree 等工具就用，遞迴留給結構真的需要的場合（例如 JSON tree、parser）。

第 8 章 · 輸入輸出與重導向

Unix 哲學的核心：每個程式都讀 stdin、寫 stdout、錯誤寫 stderr。Shell 的重導向就是把這三條流引導到檔案、其他命令、或彼此之間。把這套流體系統摸熟，你就能用 shell 組裝出非常強大的資料處理流水線。

8.1 三條標準流 ¶

名稱編號（FD）用途
stdin0標準輸入（鍵盤、上一條 pipe 的輸出）
stdout1標準輸出（正常結果）
stderr2標準錯誤（錯誤訊息、診斷資訊）

8.2 重導向運算子完整表 ¶

運算子意義範例
> filestdout 寫入檔案（覆蓋）ls > out.txt
>> filestdout 附加到檔案echo log >> app.log
< file從檔案讀入 stdinwc -l < data.txt
2> filestderr 寫入檔案cmd 2> errors.log
2>> filestderr 附加cmd 2>> errors.log
&> filestdout + stderr 都寫入（Bash）cmd &> all.log
> file 2>&1同上，POSIX 寫法cmd > all.log 2>&1
2>&1stderr 重導到 stdout 的目的地合併兩條流
>/dev/null丟棄輸出cmd >/dev/null 2>&1
<<EOFhere-document（多行字串輸入）見 8.4
<<<here-string（單行字串輸入）grep foo <<< "$var"
|pipe，前者 stdout 接後者 stdinls \| sort
\|&pipe stdout + stderr（Bash 4+）cmd \|& less

8.3 重導向常見實戰 ¶

redirect-recipes.sh BashPOSIX 
# 1. 把錯誤訊息寫到 stderr（腳本內常用）
echo "Error: 找不到設定檔" >&2

# 2. 同時寫入檔案 + 顯示在螢幕上 ── tee
make 2>&1 | tee build.log

# 3. 只要 stdout，stderr 丟棄
find / -name '*.conf' 2>/dev/null

# 4. 完全靜默（連退出碼都檢查）
if command -v jq >/dev/null 2>&1; then
    echo "jq is available"
fi

# 5. 把 stderr 接到 pipe（用於 grep 過濾錯誤訊息）
make 2>&1 | grep -i error

# 6. 順序重要！下面兩條結果不同
cmd > out.log 2>&1     # ✅ stderr 也寫進 out.log
cmd 2>&1 > out.log     # ❌ stderr 仍在原處（已先複製）

# 7. 暫存原本的 stdout，做 swap
exec 3>&1            # 把當前 stdout 備份到 fd 3
exec 1> /tmp/log     # stdout 改寫到檔案
echo "這行進檔案"
exec 1>&3 3>&-      # 還原並關閉 fd 3
echo "這行回到螢幕"

# 8. 整段 block 共用一個重導向
{
    echo "header"
    date
    df -h
} > report.txt

8.4 here-document 與 here-string ¶

heredoc-herestring.sh BashPOSIX 
# here-document：多行字串當成 stdin
cat <<EOF
這是第一行
這是第二行
變數會展開：HOME = $HOME
EOF

# 加引號 'EOF' ── 變數不展開（純字面值）
cat <<'EOF'
$HOME 不會被展開
$(date) 也不會
EOF

# 用於產生設定檔
cat <<CONFIG > /tmp/myapp.conf
[server]
host = api.example.com
port = ${PORT:-8080}
log_dir = ${LOG_DIR}
CONFIG

# <<-EOF：移除每行開頭的「tab」（不是空格）── 方便縮排
if [[ -z "$name" ]]; then
    cat <<-EOF
        Usage: $0 NAME
        錯誤：缺少 NAME 參數
    EOF
fi

# here-string：單行字串當 stdin（Bash）
read -r name age <<< "Alice 30"
echo "$name is $age"

# 常見用法：把變數內容餵給 grep
data="apple
banana
cherry"
grep -c '^a' <<< "$data"   # 1（一個 a 開頭）

8.5 行程替換（Process Substitution） ¶

<(cmd) 與 >(cmd) 是 Bash 一個極為強大的功能：把命令的輸入/輸出當成「檔案」來用，這讓那些「只接受檔名參數」的工具（如 diff、comm）可以直接吃命令的輸出。

process-substitution.sh Bash 
# 比較兩個命令的輸出，不需要建中介檔
diff <(ls /etc/cron.d) <(ls /etc/cron.daily)

# 比較兩個 sorted file 的差集
comm -23 <(sort file1.txt) <(sort file2.txt)

# 取得「只在 dir1 不在 dir2」的檔案
comm -23 \
    <(cd dir1 && find . -type f | sort) \
    <(cd dir2 && find . -type f | sort)

# 解 subshell 陷阱（第 5 章提過）── 用 <(cmd) 而非 cmd | while
total=0
while read -r size; do
    total=$((total + size))
done < <(find . -type f -printf '%s\n')
echo "總大小: $total"   # 在當前 shell，total 有效

# >(cmd)：寫入端 ── 一份輸出送到多個處理
make 2>&1 | tee \
    >(grep -i error > errors.log) \
    >(grep -i warning > warnings.log) \
    > full.log
💡 行程替換的本質

<(cmd) 在背後其實會建立一個 named pipe（FIFO 或 /dev/fd/N），把它的路徑當參數傳給命令。所以 diff <(ls A) <(ls B) 實際展開成類似 diff /dev/fd/63 /dev/fd/62。

第 9 章 · 命令替換與算術

9.1 命令替換的兩種寫法 ¶

command-substitution.sh BashPOSIX 
# 寫法 1：$(...)  ── 推薦
today=$(date '+%Y-%m-%d')
files=$(ls *.txt)

# 寫法 2：` ... `  ── 反引號，舊式，不推薦
today=`date '+%Y-%m-%d'`

# 為什麼推薦 $()：
# (1) 易讀，括號比反引號明顯
# (2) 可以巢狀：$(cmd1 $(cmd2))
# (3) 不會跟字串中的反引號搞混

# 巢狀範例
latest_log=$(ls -t $(find /var/log -name '*.log') | head -1)

# 注意：命令替換會「去掉」結尾的所有換行符！
result=$(printf 'line1\nline2\n\n\n')
echo "[$result]"
# [line1
# line2]   ← 結尾的 \n\n\n 被去掉了

9.2 Bash 5.3 不 fork 的命令替換 ¶

Batch 1 提過：傳統 $(cmd) 會 fork 一個子 shell，效能成本不小。Bash 5.3（2025/7 發布）引入 ${ cmd; } 與 ${| cmd; } 兩種「在當前 shell 執行」的新語法。

no-fork-substitution.sh Bash 5.3+ 
# 傳統：fork → 變數修改不會留下
result=$(x=99; echo "$x")
echo "${x:-未設}"        # 未設（subshell 裡的 x=99 沒留下）

# Bash 5.3 新語法 1：${ cmd; } ── 捕獲 stdout，但不 fork
result=${ x=99; printf '%s' "$x"; }
echo "$x"                # 99（變數修改保留）

# Bash 5.3 新語法 2：${| cmd; } ── 透過 REPLY 變數帶值
${| x=99; REPLY="value:$x"; }
echo "$REPLY"           # value:99
echo "$x"                # 99

# 實戰：高頻迴圈中可顯著提速
# 舊：1000 次 fork（慢）
for i in {1..1000}; do
    line=$(head -n1 "$file")
done

# 新：0 次 fork（快數倍）
for i in {1..1000}; do
    line=${ read -r line < "$file"; printf '%s' "$line"; }
done

9.3 算術展開 ¶

arithmetic.sh BashPOSIX 
# $(( ... ))：算術展開，回傳數值
n=$((5 + 3))           # 8
n=$((10 * 2 + 3))      # 23
n=$((10 / 3))          # 3（整數除法，shell 不支援浮點）
n=$((10 % 3))          # 1（取餘）
n=$((2 ** 10))         # 1024（次方）

# 變數可省略 $
x=10
y=$((x * 2 + 5))       # x 不需 $x

# 自增 / 自減（Bash 限定）
declare -i count=0
((count++))            # count=1
((++count))
((count += 5))         # count=7
((count *= 2))         # count=14

# (( )) 命令式：作為 if 條件
if ((count > 10 && count < 100)); then
    echo "two-digit"
fi

# 三元運算（Bash）
n=$(( count > 10 ? 1 : 0 ))

# 進位轉換
echo $((16#FF))        # 16 進位 FF → 255
echo $((2#1010))       # 2 進位 1010 → 10
echo $((8#777))        # 8 進位 777 → 511

# 浮點？用 bc 或 awk
echo "scale=4; 10/3" | bc       # 3.3333
awk 'BEGIN { printf "%.4f\n", 10/3 }'
⚠️ shell 不支援浮點運算

$((1.5 + 2.5)) 會直接報錯。需要浮點計算就用 bc、awk 或 printf '%.2f'。如果你需要做大量數值運算，這通常是該改用 Python/Go 的訊號。

第 10 章 · 文字三劍客 grep / sed / awk

grep（搜尋）、sed（編輯）、awk（分析）是 Unix 文字處理的三大支柱。它們存在 50 年仍未被取代──每個 Linux 發行版都有，組合起來能完成 80% 的日常文字處理任務。本章不教完整語法（那需要一整本書），而是聚焦「shell 腳本中最該會的 30 個慣用法」。

10.1 grep — 搜尋字串與正規表達式 ¶

選項意義
-i忽略大小寫
-v反向（不匹配的）
-n顯示行號
-c只算數量
-l只列出有匹配的檔名
-L列出沒匹配的檔名
-r / -R遞迴目錄
-w整字匹配（不要部分匹配）
-q安靜模式（只看 exit code）
-E使用 ERE（延伸正規），等同 egrep
-F固定字串（不解讀正規），等同 fgrep，最快
-P使用 PCRE（GNU grep 限定）
-A N顯示匹配後 N 行
-B N顯示匹配前 N 行
-C N顯示前後各 N 行
--color=auto高亮匹配部分
grep-recipes.sh POSIX 
# 1. 基本搜尋
grep "ERROR" app.log
grep -i "error" app.log              # 不分大小寫

# 2. 計算匹配行數
grep -c "ERROR" app.log

# 3. 反向：找出「沒有」匹配的行
grep -v "^#" config.ini              # 略過註解
grep -v "^$" config.ini              # 略過空行
grep -v -e "^#" -e "^$" config.ini   # 兩者皆略

# 4. 遞迴搜尋整個目錄
grep -rn "TODO" src/                 # 顯示行號
grep -rln "deprecated" .             # 只列檔名

# 5. 只當 if 條件用（不要輸出，只看有沒有）
if grep -q "ERROR" app.log; then
    notify_admin
fi

# 6. 顯示匹配行的前後文
grep -B 2 -A 2 "Exception" stack.log
grep -C 3 "Exception" stack.log      # 等同 -B3 -A3

# 7. 用 ERE 正規（更強大）
grep -E "ERROR|FATAL|CRITICAL" app.log

# 8. 從 stdin 過濾
ps aux | grep -v grep | grep nginx   # 排除 grep 自己
# 更乾淨的寫法：
pgrep -fl nginx

# 9. 整字匹配 ── 找變數名 user 不誤中 username
grep -w "user" config.txt

# 10. PCRE：用 lookahead 找「後面接 = 但不要捕捉 =」
grep -P 'name(?==)' file.txt

10.2 sed — 流式編輯器 ¶

sed 最常見的用途其實只有一個：用正規找替換。其他功能（刪除、插入、跳行）都是衍生。

sed-recipes.sh POSIX 
# 1. 替換每行第一個匹配
sed 's/foo/bar/' file.txt

# 2. 替換每行所有匹配
sed 's/foo/bar/g' file.txt

# 3. 不分大小寫（GNU sed）
sed 's/foo/bar/gi' file.txt

# 4. 用 # 當分隔符（路徑替換時不用跳脫 /）
sed 's#/old/path#/new/path#g' file.txt

# 5. 直接修改檔案（in-place）
sed -i 's/foo/bar/g' file.txt        # GNU/Linux
sed -i '' 's/foo/bar/g' file.txt     # macOS（BSD sed 需空字串備份名）
sed -i.bak 's/foo/bar/g' file.txt    # 兩者皆通用：留 .bak 備份

# 6. 只處理特定行
sed '3s/foo/bar/' file.txt           # 只第 3 行
sed '1,5s/foo/bar/' file.txt         # 1-5 行
sed '/^#/!s/foo/bar/' file.txt       # 不在 # 開頭的行

# 7. 刪除行
sed '5d' file.txt                    # 刪第 5 行
sed '/pattern/d' file.txt            # 刪含 pattern 的行
sed '/^$/d' file.txt                 # 刪空行
sed '/^#/d' file.txt                 # 刪註解

# 8. 只印特定行（配合 -n 抑制預設輸出）
sed -n '5,10p' file.txt              # 印第 5-10 行
sed -n '/start/,/end/p' file.txt     # 印兩個 pattern 之間

# 9. 反向參考（capture group）
sed -E 's/(\w+) (\w+)/\2 \1/' file.txt   # 交換前兩個單字

# 10. 多個指令（用 -e 或 ;）
sed -e 's/foo/bar/g' -e 's/baz/qux/g' file.txt
sed 's/foo/bar/g; s/baz/qux/g' file.txt
⚠️ macOS / BSD sed 與 GNU sed 不一致

sed -i 在兩家 BSD/macOS 與 GNU 行為不同：BSD 必須帶備份檔字串（即使是空字串 ''），GNU 則可省略。最可移植的寫法：sed -i.bak '...'，兩家都接受、留下 .bak 備份。寫好後再用 rm *.bak 清掉。

10.3 awk — 資料的程式語言 ¶

awk 是一個小型程式語言，專為「逐行讀資料、按欄位處理」而設計。它預設把每行切成欄位（$1 $2 $3），再用條件 + 動作的語法處理。

awk 三大實作

實作特性常見發行版
gawk（GNU awk）功能最全、擴充最多、記憶體用量最高RHEL、Fedora、Arch
mawk（Mike's awk）速度最快（用 bytecode）、功能精簡Debian、Ubuntu 預設
nawk（One True Awk）原作者 Brian Kernighan 維護、2025 年仍在更新、記憶體最省BSD、macOS 預設

2026 年的基準測試結果：mawk 處理速度最快、nawk 記憶體最省、gawk 功能最豐富但最耗資源。寫腳本時用 awk 即可（系統會連到對應實作），需要 GNU 擴充才寫 gawk。

awk 基本語法

awk-basics.sh POSIX 
# 結構：awk 'pattern { action }'
# pattern 省略 → 對所有行；action 省略 → print $0

# 1. 印第 N 欄
awk '{ print $1 }' data.txt          # 第 1 欄
awk '{ print $NF }' data.txt         # 最後一欄
awk '{ print $(NF-1) }' data.txt     # 倒數第 2 欄

# 2. 內建變數
#   $0   整行
#   $1, $2, ...  各欄
#   NF   number of fields（本行欄數）
#   NR   number of records（當前行號）
#   FS   field separator（輸入分隔符，預設空白）
#   OFS  output field separator
#   FILENAME  當前檔案名

awk '{ print NR ": " $0 }' file.txt  # 加行號（類似 cat -n）
awk 'END { print NR }' file.txt      # 總行數（類似 wc -l）

# 3. 改變分隔符
awk -F: '{ print $1 }' /etc/passwd        # 用 : 分割
awk -F'\t' '{ print $2 }' data.tsv        # tab
awk -F'[,:]' '{ print $3 }' mixed.txt     # 多種分隔符（regex）

# 4. 條件 + 動作
awk '/ERROR/ { print $0 }' app.log
awk '$3 > 100 { print $1, $3 }' sales.txt
awk 'NR > 1 { print }' csv.txt            # 跳過 header
awk 'NR == 1 || /pattern/ { print }' f    # 第 1 行 or 匹配的

# 5. BEGIN 與 END 區塊
awk 'BEGIN { print "Start" }
     { print NR ": " $0 }
     END { print "Total: " NR " lines" }' file.txt

awk 實戰

awk-recipes.sh POSIX 
# 1. 加總一欄
awk '{ sum += $3 } END { print sum }' sales.txt

# 2. 算平均
awk '{ sum += $3; n++ } END { printf "%.2f\n", sum/n }' sales.txt

# 3. 找最大值（含對應行）
awk '$3 > max { max=$3; line=$0 } END { print line }' sales.txt

# 4. 唯一值計數（替代 sort | uniq -c）
awk '{ count[$1]++ } END { for (k in count) print k, count[k] }' data.txt

# 5. 重新格式化（CSV → TSV，加引號等）
awk -F, 'BEGIN { OFS="\t" } { print $1, $2, $3 }' data.csv

# 6. 條件式修改
awk -F: '$3 >= 1000 { print $1 }' /etc/passwd   # UID >= 1000 的使用者

# 7. 多檔合併處理（FNR：當前檔案的行號）
awk 'FNR == 1 { print "=== " FILENAME " ===" } { print }' f1 f2 f3

# 8. 取出 access log 的 top 10 IP
awk '{ ips[$1]++ } END { for (ip in ips) print ips[ip], ip }' access.log \
    | sort -rn | head -10

# 9. 簡單的 grouping（依第 1 欄群組、加總第 3 欄）
awk '{ sum[$1] += $3 } END { for (k in sum) print k, sum[k] }' data.txt

# 10. printf 格式化輸出
awk '{ printf "%-20s %5d %8.2f\n", $1, $2, $3 }' data.txt

10.4 配角群：cut / tr / sort / uniq / xargs ¶

supporting-tools.sh POSIX 
# cut：簡單的欄位提取（比 awk 快、語法簡單）
cut -d: -f1 /etc/passwd              # 第 1 欄（: 分隔）
cut -d, -f2,4 data.csv               # 第 2、4 欄
cut -c1-10 data.txt                  # 每行的前 10 個字元

# tr：字元層級的轉換（不認得正規）
echo "Hello" | tr 'a-z' 'A-Z'        # HELLO（轉大寫）
echo "Hello" | tr -d 'l'             # Heo（刪除 l）
echo "a   b   c" | tr -s ' '         # a b c（壓縮重複空格）
tr '\r' '' < win.txt > unix.txt      # 移除 CR（Windows 換行）

# sort：排序
sort file.txt                        # 字典序
sort -n numbers.txt                  # 數值排序
sort -rn numbers.txt                 # 反向
sort -u file.txt                     # 排序＋去重
sort -k2 data.txt                    # 依第 2 欄
sort -k2,2 -k3,3n data.txt           # 主鍵第 2 欄字典序、副鍵第 3 欄數值
sort -t: -k3 -n /etc/passwd          # 用 : 為分隔符，依第 3 欄數值

# uniq：去重（必須先 sort）
sort file.txt | uniq                 # 去重
sort file.txt | uniq -c              # 加上計數
sort file.txt | uniq -d              # 只印重複的
sort file.txt | uniq -u              # 只印唯一的

# xargs：把 stdin 變成命令的參數
find . -name '*.bak' | xargs rm                 # 危險：含空白檔名會壞
find . -name '*.bak' -print0 | xargs -0 rm      # 安全：用 NUL 分隔
find . -name '*.bak' -delete                    # 更安全：完全不用 xargs

# xargs 平行處理（-P）
ls *.jpg | xargs -P 4 -I{} convert {} {}.thumb.jpg

# 經典組合：top 10 最大檔案
du -ah . | sort -rh | head -10

# 計算唯一 IP 數
awk '{print $1}' access.log | sort -u | wc -l
