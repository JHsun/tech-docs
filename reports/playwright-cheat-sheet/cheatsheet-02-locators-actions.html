🎯 推薦定位器方法（Locators）— 優先順序由高至低

方法Node.jsPython說明
⭐ 角色getByRole('button', {name:'送出'})get_by_role("button", name="送出")ARIA 角色定位（最優先）
文字getByText('歡迎', {exact:true})get_by_text("歡迎", exact=True)文字內容（非互動元素）
標籤getByLabel('使用者名稱')get_by_label("使用者名稱")表單 label 定位
PlaceholdergetByPlaceholder('搜尋...')get_by_placeholder("搜尋...")placeholder 屬性
Alt 文字getByAltText('Logo')get_by_alt_text("Logo")img alt 屬性
TitlegetByTitle('關閉')get_by_title("關閉")title 屬性
Test IDgetByTestId('submit-btn')get_by_test_id("submit-btn")data-testid 屬性
CSSlocator('div.card > h2')locator("div.card > h2")CSS 選擇器（可穿透 Shadow DOM）
XPathlocator('xpath=//button')locator("xpath=//button")XPath（不穿透 Shadow DOM）
⚠️ 最佳實踐：優先使用 getByRole → getByText → getByTestId。避免依賴 CSS class 或 XPath，因為這些容易隨 UI 重構改變。

🔍 getByRole 詳細選項

Node.js
page.getByRole('button', {
  name: '送出',        // 無障礙名稱（字串或正則）
  exact: true,         // 精確匹配（預設 false）
  checked: true,       // checkbox/radio 狀態
  disabled: false,     // 停用狀態
  expanded: true,      // 展開狀態
  level: 2,            // heading level（1-6）
  pressed: true,       // toggle button 按下狀態
  selected: true,      // option 選取狀態
});

🔗 篩選與組合定位器

篩選（filter）

Node.js
// 依文字篩選
page.getByRole('listitem').filter({ hasText: 'Product 2' })
page.getByRole('listitem').filter({ hasNotText: 'Out of stock' })

// 依子元素篩選
page.getByRole('listitem').filter({
  has: page.getByRole('heading', { name: 'Product 2' })
})
page.getByRole('listitem').filter({
  hasNot: page.getByText('Sold out')
})

// 可見性篩選（v1.51+）
page.getByRole('button').filter({ visible: true })

Python
page.get_by_role("listitem").filter(has_text="Product 2")
page.get_by_role("listitem").filter(has_not_text="Out of stock")
page.get_by_role("listitem").filter(
  has=page.get_by_role("heading", name="Product 2")
)

組合運算

Node.js
// 同時符合（AND）
locator.and(page.getByRole('button'))
// 任一符合（OR）
locator.or(page.getByRole('link'))

Python  ⚠️ 避免與關鍵字衝突
locator.and_(page.get_by_role("button"))
locator.or_(page.get_by_role("link"))

串接（Chaining）

// 先範圍定位，再深入
page.getByRole('article').getByRole('button', { name: '購買' })

📋 清單操作與計數

Node.js
const items = page.getByRole('listitem');

items.first()           // 第一個
items.last()            // 最後一個
items.nth(2)            // 第三個（0-based）
await items.count()     // 數量

// 遍歷全部
for (const item of await items.all()) {
  console.log(await item.textContent());
}

Python  ⚠️ first/last 是屬性非方法
items = page.get_by_role("listitem")

items.first              # 第一個（屬性）
items.last               # 最後一個（屬性）
items.nth(2)             # 第三個
items.count()            # 數量

for item in items.all():
    print(item.text_content())
💡 nth() 是最後手段。優先用 filter() 篩選出唯一元素。

📁 檔案上傳與下載

上傳

Node.js
// 單檔上傳
await page.getByLabel('Upload').setInputFiles('file.pdf');
// 多檔上傳
await page.getByLabel('Upload').setInputFiles(['f1.png', 'f2.png']);
// 清除選擇
await page.getByLabel('Upload').setInputFiles([]);
// 從記憶體上傳
await page.getByLabel('Upload').setInputFiles({
  name: 'test.txt',
  mimeType: 'text/plain',
  buffer: Buffer.from('content')
});

Python
page.get_by_label("Upload").set_input_files("file.pdf")
page.get_by_label("Upload").set_input_files([])

下載

Node.js
const downloadPromise = page.waitForEvent('download');
await page.getByText('Download').click();
const download = await downloadPromise;
await download.saveAs('/path/file.pdf');

Python
with page.expect_download() as download_info:
    page.get_by_text("Download").click()
download = download_info.value
download.save_as("/path/file.pdf")

🔄 自動等待（Auto-waiting）可操作性檢查

檢查項目說明適用操作
Visible非空邊界框 + 非 visibility:hidden（opacity:0 仍算可見）click · fill · hover · check · select
Stable連續兩個動畫幀邊界框相同（元素停止移動）click · hover · check · drag
Enabled非 disabled 屬性click · fill · check · select · tap
Editable非 readonly 屬性fill · clear
Receives Events為操作目標點的命中目標（hit target）click · hover · check
💡 重要機制：所有操作自動重試直到元素滿足條件或逾時。元素分離時自動從 DOM 根重新查詢。全部步驟超時則拋出 TimeoutError。

📤 select & 對話框

下拉選單

Node.js
// 依 value
await page.selectOption('select#color', 'blue');
// 依 label
await page.selectOption('select#color', { label: '藍色' });
// 多選
await page.selectOption('select#size', ['s', 'm', 'l']);

Python
page.select_option("select#color", "blue")
page.select_option("select#color", label="藍色")
page.select_option("select#size", ["s", "m", "l"])

對話框（Dialogs）

Node.js
page.on('dialog', dialog => dialog.accept());     // 接受
page.on('dialog', dialog => dialog.dismiss());    // 取消
page.on('dialog', dialog => {
  console.log(dialog.message());
  dialog.accept('輸入值');   // prompt 輸入
});

Python
page.on("dialog", lambda d: d.accept())
page.on("dialog", lambda d: d.dismiss())
⚠️ 必須 accept 或 dismiss 對話框，否則頁面會凍結！未註冊監聽器時自動 dismiss。

🆕 v1.59 定位器增強

locator.normalize() — 轉換為最佳實踐定位器
// 將脆弱的 CSS/XPath 定位器自動轉換為
// getByRole / getByTestId 等建議寫法
const normalized = await page.locator('.submit-btn').normalize();
console.log(normalized);
// → page.getByRole('button', { name: '送出' })

page.pickLocator() — 互動式選取（Node.js）
// 進入互動模式，hover 元素顯示對應定位器
const loc = await page.pickLocator();   // 點擊元素後回傳 Locator
await page.cancelPickLocator();          // 取消
page.ariaSnapshot() — 頁面層級快照（v1.59）
// 相當於 page.locator('body').ariaSnapshot()
const snapshot = await page.ariaSnapshot();

locator.ariaSnapshot() 新增選項
await page.locator('main').ariaSnapshot({
  depth: 3,            // 限制遞迴深度
  mode: 'raw',         // 'raw' | 'minimal' | 'expect'
});
💡 normalize() 協助 Agent 自動修復脆弱定位器；pickLocator() 讓你在測試中暫停並互動式挑選元素。

🌐 自訂 Test ID

Node.js — playwright.config.ts
use: { testIdAttribute: 'data-pw' }

Python — conftest.py
from playwright.sync_api import Playwright
def setup(playwright: Playwright):
    playwright.selectors.set_test_id_attribute("data-pw")
HTML
<button data-pw="submit-btn">送出</button>

測試
page.getByTestId('submit-btn').click()   // Node.js
page.get_by_test_id("submit-btn").click()  # Python
