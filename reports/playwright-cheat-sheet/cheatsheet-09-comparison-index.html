Python 底線規則（避免關鍵字衝突）
⚠重要！Python 中以下方法加了底線 _ 以避免與 Python 關鍵字衝突：
Node.js 方法Python 方法原因
locator.and(other)locator.and_(other)and 是 Python 關鍵字
locator.or(other)locator.or_(other)or 是 Python 關鍵字
route.continue()route.continue_()continue 是 Python 關鍵字
📝Python 的 first、last、content_frame 是屬性（不加括號），Node.js 是方法（加括號）。否定斷言用 not_to_ 前綴取代 .not. 鏈式。

常見陷阱速查
#陷阱正確做法
1用 expect(await loc.isVisible()).toBe(true)用 await expect(loc).toBeVisible()（Web-first，自動重試）
2用 page.waitForTimeout(5000)用 web-first 斷言等待特定 UI 狀態
3用 waitForLoadState('networkidle')用 await expect(loc).toBeVisible() 等待具體元素
4Python locator.first()（加括號）Python 用 locator.first（屬性，不加括號）
5Python route.continue()Python 用 route.continue_()（底線避免關鍵字）
6Python expect(loc).not.to_be_visible()Python 用 expect(loc).not_to_be_visible()
7忘記 await（Node.js）全部 Playwright 操作加 await，用 ESLint no-floating-promises
8測試間共享狀態每個測試獨立上下文，不依賴其他測試結果
9CI 用太多 workersCI 建議 workers: 1，本機用預設（50% CPU）
10用 CSS 選擇器作為主要定位方式優先用 getByRole、getByText、getByLabel
