實戰指南
最佳實踐（Best Practices）
✅ 推薦做法 
// ✅ 使用推薦的定位器
page.getByRole('button', { name: '送出' })
page.getByLabel('帳號')
page.getByTestId('submit')

// ✅ Web-first 斷言（自動等待重試）
await expect(locator).toBeVisible()
await expect(locator).toHaveText('Done')

// ✅ 每個測試完全隔離
// 不依賴其他測試的狀態

// ✅ Mock 第三方依賴
await page.route('**/api/payment/**',
  route => route.fulfill({ json: { ok: true } }))

// ✅ 善用軟斷言
await expect.soft(el1).toHaveText('A')
await expect.soft(el2).toHaveText('B')

// ✅ 在 CI 上每次 commit 跑測試
❌ 避免做法 
// ❌ 避免 CSS/XPath 選擇器
page.locator('.btn-submit')
page.locator('#form > div:nth-child(3)')

// ❌ 不要用手動斷言（不會重試！）
expect(await locator.isVisible()).toBe(true)
expect(await locator.textContent()).toBe('X')

// ❌ 不要測試之間有依賴
// test1 建立資料 → test2 讀取

// ❌ 不要直接打第三方 API
// 測試結果不穩定、速度慢

// ❌ 不要用 waitForTimeout
await page.waitForTimeout(5000) // 永遠不要！

// ❌ 不要用 waitForNetworkIdle
await page.waitForLoadState('networkidle')
📏定位器優先順序：getByRole（最推薦）→ getByText → getByLabel → getByPlaceholder → getByTestId（最後手段）→ CSS/XPath（避免）。
CLI 完整參考
⌨ Node.js CLI 旗標完整對照
旗標簡寫預設說明
--headedheadless有頭模式（看到瀏覽器）
--debug—Inspector + headed + 1 worker + timeout=0
--ui—UI 互動模式
--workers <N>-j50% CPU並行 Worker 數（數字或百分比）
--retries <N>0重試次數
--grep <regex>-g.*依標題/標籤篩選
--grep-invert <regex>—排除匹配的測試
--project <name>全部指定專案（支援 glob *）
--shard <x/y>—分片執行（x 第幾片 / y 共幾片）
--trace <mode>—on | off | on-first-retry | retain-on-failure
--timeout <ms>30000測試逾時
--max-failures <N>-x0（∞）達到 N 次失敗後停止
--reporter <type>listlist | line | dot | html | json | junit
--update-snapshotsmissing更新快照（all | missing | changed | none）
--fully-parallelfalse所有測試平行
--last-failed—僅重跑上次失敗
--repeat-each <N>1每個測試重複 N 次
--output <dir>test-results產出目錄
--config <file>playwright.config指定設定檔
--forbid-onlyfalse禁止 .only（CI 用）
--global-timeout <ms>0全部測試總逾時
--list—僅列出測試，不執行
--pass-with-no-testsfalse無測試時視為通過

旗標預設說明

Agents（AI 代理人）& 遷移指南
🤖 Agents（僅 Node.js）
npx playwright init-agents --loop=vscode
代理人功能
🎭 planner探索應用 → 產生 Markdown 測試計畫
🎭 generator將計畫轉換為測試檔案
🎭 healer執行測試 → 自動修復失敗
# 支援的 AI 循環
--loop=vscode     # VS Code Copilot
--loop=claude     # Claude Code
--loop=opencode   # OpenCode
🆕Agents 是 v1.58+ 新功能，使用 LLM 自動生成和修復測試。

Docker & CI 快速參考
Docker 映像： 
# Node.js
docker pull mcr.microsoft.com/playwright:v1.59.1-noble
docker run -it --rm --ipc=host --init \
  mcr.microsoft.com/playwright:v1.59.1-noble /bin/bash

# Python
docker pull mcr.microsoft.com/playwright/python:v1.58.0-noble

# 遠端 Server
docker run -p 3000:3000 --rm --init \
  -it --workdir /home/pwuser --user pwuser \
  mcr.microsoft.com/playwright:v1.59.1-noble \
  /bin/sh -c "npx -y playwright@1.59.1 \
  run-server --port 3000 --host 0.0.0.0"

# 連接遠端
PW_TEST_CONNECT_WS_ENDPOINT=ws://127.0.0.1:3000/ \
  npx playwright test
分片 CI（GitHub Actions 範例）： 
strategy:
  fail-fast: false
  matrix:
    shardIndex: [1, 2, 3, 4]
    shardTotal: [4]
steps:
  - uses: actions/checkout@v4
  - uses: actions/setup-node@v4
    with: { node-version: lts/* }
  - run: npm ci
  - run: npx playwright install --with-deps
  - run: npx playwright test \
      --shard=${{ matrix.shardIndex }}/${{ matrix.shardTotal }}
  - uses: actions/upload-artifact@v4
    with:
      name: blob-${{ matrix.shardIndex }}
      path: blob-report/

# 合併報告
merge:
  needs: [test]
  steps:
    - uses: actions/download-artifact@v4
    - run: npx playwright merge-reports \
        --reporter html ./all-blob-reports
