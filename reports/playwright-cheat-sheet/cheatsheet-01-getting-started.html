✍️ 撰寫第一個測試

TypeScript — example.spec.ts
import { test, expect } from '@playwright/test';

test('首頁有正確標題', async ({ page }) => {
  await page.goto('https://playwright.dev/');
  await expect(page).toHaveTitle(/Playwright/);
});

test('點擊入門連結', async ({ page }) => {
  await page.goto('https://playwright.dev/');
  await page.getByRole('link', { name: 'Get started' }).click();
  await expect(page.getByRole('heading',
    { name: 'Installation' })).toBeVisible();
});
Python — test_example.py
import re
from playwright.sync_api import Page, expect

def test_has_title(page: Page):
    page.goto("https://playwright.dev/")
    expect(page).to_have_title(re.compile("Playwright"))

def test_get_started_link(page: Page):
    page.goto("https://playwright.dev/")
    page.get_by_role("link", name="Get started").click()
    expect(page.get_by_role("heading",
      name="Installation")).to_be_visible()
💡 測試隔離：每個測試自動獲得獨立 BrowserContext（cookies/localStorage/sessionStorage 完全隔離），無需手動清理。

🪝 測試鉤子（Hooks）

TypeScript
test.beforeAll(async () => { /* Worker 啟動 */ });
test.afterAll(async () =>  { /* Worker 結束 */ });
test.beforeEach(async ({ page }) => {
  await page.goto('https://example.com');
});
test.afterEach(async ({ page }) => { /* 清理 */ });

test.describe('功能群組', () => {
  test.beforeEach(async ({ page }) => { });
  test('測試一', async ({ page }) => { });
});
Python — 使用 pytest fixtures
@pytest.fixture(autouse=True)
def before_each(page):
    page.goto("https://example.com")
    yield           # ← 測試在此執行
    # yield 後 = afterEach 等效

@pytest.fixture(scope="session", autouse=True)
def setup_session():
    yield           # session scope ≈ beforeAll

▶️ 執行測試命令

Node.js

npx playwright test              # 全部
npx playwright test --headed     # 有頭模式
npx playwright test --ui         # UI 互動模式
npx playwright test --debug      # Inspector 除錯
npx playwright test landing.spec.ts     # 特定檔案
npx playwright test -g "add todo"       # 依標題
npx playwright test --project=webkit    # 指定瀏覽器
npx playwright test --grep @fast        # 依標籤
npx playwright test --last-failed       # 重跑失敗

Python

pytest                      pytest --headed
pytest --browser webkit     pytest -k "test_add_todo"
PWDEBUG=1 pytest -s         pytest --slowmo 500

🎬 程式碼產生器（Codegen）

基本  npx playwright codegen example.com     # Node.js
       playwright codegen example.com          # Python

模擬  npx playwright codegen --device="iPhone 13" site.com

選項  npx playwright codegen --color-scheme=dark \
         --timezone="Asia/Taipei" --viewport-size=1280,720 site.com

驗證  npx playwright codegen --save-storage=auth.json   # 儲存
       npx playwright codegen --load-storage=auth.json   # 載入

🔍 追蹤檢視器（Trace Viewer）

模式行為
'on'每次錄製
'off'不錄製
'on-first-retry' ⭐首次重試時錄製（推薦）
'retain-on-failure'全部錄，僅失敗保留
'retain-on-failure-and-retries' v1.59每次嘗試獨立保留，便於比對
Node.js  use: { trace: 'on-first-retry' }
npx playwright show-report
Python  pytest --tracing on
playwright show-trace trace.zip
手動
context.tracing.start({ screenshots: true, snapshots: true, sources: true });
context.tracing.stop({ path: 'trace.zip' });

🆕 v1.59 重點更新（2026-04）

新功能

  • 🎬 Screencast API：page.screencast.start/stop()、showActions()、showChapter()、showOverlay()，支援即時逐幀串流與 Agent 驗證影片
  • 🔗 browser.bind()：將瀏覽器暴露給 playwright-cli、@playwright/mcp 或其他客戶端共用
  • 📊 Dashboard：playwright-cli show 檢視所有綁定瀏覽器
  • 🐛 --debug=cli：Agent 用命令列除錯測試
  • 📋 npx playwright trace：命令列分析 trace.zip
  • ♻️ await using：自動釋放 route / init script / page
  • 🔍 Locator 增強：locator.normalize()、page.pickLocator()、page.ariaSnapshot()
  • 🗄️ Storage/Console：context.setStorageState()、page.clearConsoleMessages()、page.clearPageErrors()
