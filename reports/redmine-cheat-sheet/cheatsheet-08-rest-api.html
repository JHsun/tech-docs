REST API 總覽
Rest_api

基本資訊

  • 基底 URL：https://redmine.example.com/
  • 格式：JSON 或 XML（副檔名 .json / .xml）
  • 啟用方式：管理 → 設定 → API → 「啟用 REST web service」
  • 所有 POST/PUT 請求必須指定 Content-Type
  • 回應包含 HTTP 狀態碼表示成功或失敗

HTTP 方法對照

方法用途典型回應碼
GET讀取資源列表或單一資源200 OK
POST建立新資源201 Created
PUT更新既有資源200 OK
DEL刪除資源200 OK

認證方式 Authentication
Rest_api §Authentication

方式 1：API Key（查詢參數）

GET /issues.json?key=abcdef1234567890

API Key 可在「我的帳號」頁面取得

方式 2：API Key（HTTP Header）

GET /issues.json
X-Redmine-API-Key: abcdef1234567890

推薦此方式，避免 key 出現在 URL 日誌中

方式 3：HTTP Basic Auth

curl -u username:password \
  https://redmine.example.com\
  /issues.json

使用帳號密碼，需 HTTPS

使用者模擬（Impersonation）：管理員可在 Header 中加入 X-Redmine-Switch-User: login_name，以指定使用者身份執行操作。適用於整合系統代替使用者執行 API 呼叫。

分頁・關聯資料・自訂欄位
Rest_api §Pagination §Associated data §Custom fields

分頁 Pagination

列表型 API 回應包含分頁資訊：

{
  "issues": [...],
  "total_count": 2674,
  "offset": 0,
  "limit": 25
}

參數：?offset=25&limit=100
limit 最大值由管理員設定
（預設上限 100）

關聯資料 Include

使用 include 參數載入關聯資源：

# 議題含子任務和關聯
GET /issues/1.json?
  include=children,relations,
  attachments,changesets,
  journals,watchers

# 專案含追蹤器和分類
GET /projects/1.json?
  include=trackers,
  issue_categories,
  enabled_modules,
  time_entry_activities

自訂欄位讀寫

自訂欄位以 custom_fields 陣列呈現：

// 讀取回應中
"custom_fields": [
  {"id":1, "name":"客戶",
   "value":"Acme"}
]

// 寫入時（POST/PUT）
"issue": {
  "custom_fields": [
    {"id":1, "value":"Acme"},
    {"id":2, "value":["a","b"]}
  ]
}

附件上傳 API（兩步驟）
Rest_api §Attaching files

步驟 1：上傳檔案取得 Token

POST /uploads.json
Content-Type: application/octet-stream

# 將檔案內容作為 request body 傳送
curl -X POST \
  -H "Content-Type: application/octet-stream" \
  -H "X-Redmine-API-Key: xxx" \
  --data-binary @file.pdf \
  https://redmine.example.com/uploads.json

# 回應：
{"upload": {"token": "7167..."}}

步驟 2：將 Token 關聯到資源

# 建立議題時附加檔案
POST /issues.json
{
  "issue": {
    "project_id": "myproject",
    "subject": "新議題",
    "uploads": [
      {
        "token": "7167...",
        "filename": "file.pdf",
        "content_type": "application/pdf",
        "description": "說明文字"
      }
    ]
  }
}

Issues API（議題）
Rest_Issues
方法端點說明重要參數
GET /issues.json 列出議題 project_id tracker_id status_id assigned_to_id sort cf_X（自訂欄位篩選）
GET /issues/{id}.json 取得單一議題 include=children,attachments,relations,changesets,journals,watchers
POST /issues.json 建立議題 project_id（必填）tracker_id subject description status_id priority_id assigned_to_id category_id fixed_version_id parent_issue_id watcher_user_ids is_private
PUT /issues/{id}.json 更新議題 同上 + notes（備註）private_notes
DEL /issues/{id}.json 刪除議題 —
POST /issues/{id}/watchers.json 新增關注者 user_id
DEL /issues/{id}/watchers/{user_id}.json 移除關注者 —
篩選技巧：status_id=open（所有開啟）、status_id=closed（所有已關閉）、status_id=*（全部）、assigned_to_id=me（指派給自己）。自訂欄位用 cf_欄位ID=值 篩選。

Projects API（專案）
Rest_Projects
方法端點說明備註
GET/projects.json列出所有專案include=trackers,issue_categories,enabled_modules,time_entry_activities
GET/projects/{id}.json取得專案詳情id 可為數字 ID 或識別碼
POST/projects.json建立專案name identifier（必填）description is_public parent_id inherit_members tracker_ids enabled_module_names
PUT/projects/{id}.json更新專案同上（identifier 不可改）
PUT/projects/{id}/archive.json封存專案—
PUT/projects/{id}/unarchive.json解除封存—
DEL/projects/{id}.json刪除專案不可逆！

Users API & Memberships API
Rest_Users · Rest_Memberships
方法端點說明
GET/users.json列出使用者（status=0 已註冊 / 1 啟用 / 2 已鎖定，name 搜尋，group_id 篩選）
GET/users/{id}.json取得使用者（include=memberships,groups）；/users/current.json = 當前使用者
POST/users.json建立使用者（login password firstname lastname mail 必填）
PUT/users/{id}.json更新使用者
DEL/users/{id}.json刪除使用者
方法端點說明
GET/projects/{id}/memberships.json列出專案成員
POST/projects/{id}/memberships.json新增成員（user_id + role_ids）
PUT/memberships/{id}.json更新成員角色
DEL/memberships/{id}.json移除成員

Time Entries API（工時記錄）
Rest_TimeEntries
方法端點說明參數
GET/time_entries.json列出工時project_id user_id spent_on（篩選日期，支援 >=2024-01-01 >=2024-12-31）
GET/time_entries/{id}.json取得工時詳情—
POST/time_entries.json建立工時issue_id 或 project_id（擇一必填）spent_on hours activity_id comments
PUT/time_entries/{id}.json更新工時同上
DEL/time_entries/{id}.json刪除工時—

Wiki Pages API
Rest_WikiPages
方法端點說明
GET/projects/{id}/wiki/index.json列出 Wiki 所有頁面
GET/projects/{id}/wiki/{title}.json取得 Wiki 頁面（含 include=attachments）
GET/projects/{id}/wiki/{title}/{version}.json取得特定版本的 Wiki 頁面
PUT/projects/{id}/wiki/{title}.json建立或更新 Wiki 頁面（text comments parent_title）
DEL/projects/{id}/wiki/{title}.json刪除 Wiki 頁面

錯誤處理 & HTTP 狀態碼
Rest_api §Validation errors

常見 HTTP 狀態碼

狀態碼意義
200成功（GET/PUT/DELETE）
201建立成功（POST）
401未認證 — API key 無效或未提供
403禁止存取 — 權限不足
404資源不存在
422驗證失敗 — 含錯誤訊息陣列
500伺服器內部錯誤

驗證錯誤回應格式

// HTTP 422 回應範例
{
  "errors": [
    "Subject cannot be blank",
    "Tracker cannot be blank"
  ]
}

// XML 格式
<errors>
  <error>Subject cannot be blank</error>
  <error>Tracker cannot be blank</error>
</errors>

各語言/工具使用範例
Rest_api_with_*

cURL

# 建立議題
curl -X POST \
  -H "Content-Type: application/json" \
  -H "X-Redmine-API-Key: xxx" \
  -d '{"issue":{"project_id":"myprj",
    "subject":"API建立的議題",
    "tracker_id":1,"priority_id":2}}' \
  https://redmine.example.com/issues.json

# 更新議題狀態
curl -X PUT \
  -H "Content-Type: application/json" \
  -H "X-Redmine-API-Key: xxx" \
  -d '{"issue":{"status_id":3,
    "notes":"已修復"}}' \
  https://redmine.example.com/issues/123.json

Python

import requests

API_URL = "https://redmine.example.com"
API_KEY = "your-api-key"
headers = {
  "Content-Type": "application/json",
  "X-Redmine-API-Key": API_KEY
}

# 列出議題
r = requests.get(
  f"{API_URL}/issues.json",
  headers=headers,
  params={"project_id": "myprj",
          "status_id": "open"})
issues = r.json()["issues"]

# 建立議題
r = requests.post(
  f"{API_URL}/issues.json",
  headers=headers,
  json={"issue": {
    "project_id": "myprj",
    "subject": "API 建立",
    "tracker_id": 1}})
