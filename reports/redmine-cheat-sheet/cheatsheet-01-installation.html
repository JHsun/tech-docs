系統需求 Requirements
RedmineInstall §Requirements

🖥 作業系統

支援所有可執行 Ruby 的作業系統：Linux（推薦）、macOS、Windows、FreeBSD、Solaris 等。

建議：生產環境首選 Linux（Debian/Ubuntu 或 CentOS/RHEL）

💎 Ruby 版本

RedmineRuby 版本
6.03.1, 3.2, 3.3
5.13.0, 3.1, 3.2
5.02.7, 3.0, 3.1

🗄 資料庫

  • MySQL 5.7 以上（推薦）
  • PostgreSQL 9.2 以上
  • SQLite 3（僅限開發/測試）
  • SQL Server（需額外 adapter）

📦 必要元件

  • Bundler — Ruby 相依套件管理
  • 對應資料庫的開發庫（如 libmysqlclient-dev）
  • ImageMagick（選用，處理圖片附件與甘特圖匯出）

🔧 選用元件

  • RMagick — 支援甘特圖匯出為 PNG 及縮圖生成
  • MiniMagick — 輕量替代 RMagick
  • 版本控制工具：Git, SVN, Mercurial, CVS, Bazaar
  • Gemfile.local — 可放額外 gem 不影響升級

完整安裝步驟（10 步）
RedmineInstall §Installation
1

取得 Redmine 原始碼

從官網下載穩定版 tarball 或使用 SVN checkout

# 下載穩定版
wget https://www.redmine.org/releases/redmine-6.0.0.tar.gz
tar xzf redmine-6.0.0.tar.gz
cd redmine-6.0.0

# 或 SVN checkout
svn co https://svn.redmine.org/redmine/branches/6.0-stable redmine
2

建立空資料庫與使用者

為 Redmine 建立專用資料庫及帳號，依所選資料庫類型操作：

-- MySQL --
CREATE DATABASE redmine CHARACTER SET utf8mb4;
CREATE USER 'redmine'@'localhost' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON redmine.* TO 'redmine'@'localhost';

-- PostgreSQL --
CREATE ROLE redmine LOGIN ENCRYPTED PASSWORD 'password' NOINHERIT VALID UNTIL 'infinity';
CREATE DATABASE redmine WITH ENCODING='UTF8' OWNER=redmine;
3

配置資料庫連線

複製範本並編輯 config/database.yml

cp config/database.yml.example config/database.yml
vi config/database.yml

→ 詳細格式見下方「資料庫配置」區塊

4

安裝 Ruby 相依套件

bundle install --without development test
# 如需 RMagick：移除 --without 中的 rmagick
# 可透過 Gemfile.local 加入額外 gems
5

生成 Session 密鑰

bundle exec rake generate_secret_token
注意：此密鑰存於 config/initializers/secret_token.rb，需妥善保管。遺失將導致所有現有 session 失效。
6

建立資料庫結構

RAILS_ENV=production bundle exec rake db:migrate
7

載入預設資料集

RAILS_ENV=production REDMINE_LANG=zh-TW bundle exec rake redmine:load_default_data

可選語言：en, zh-TW, ja, fr, de 等。此步驟載入預設追蹤器、狀態、角色、工作流程。

8

設定檔案系統權限

確保以下目錄可由 Web 伺服器程序寫入：

mkdir -p tmp tmp/pdf public/plugin_assets
chown -R redmine:redmine files log tmp public/plugin_assets
chmod -R 755 files log tmp public/plugin_assets
files/ log/ tmp/ tmp/pdf/ public/plugin_assets/
9

測試安裝

bundle exec rails server -e production

開啟瀏覽器訪問 http://localhost:3000，應可看到 Redmine 首頁

10

首次登入

使用預設管理員帳號登入，並立即變更密碼：

帳號：admin 密碼：admin
安全提醒：務必在首次登入後立即修改預設 admin 密碼！Redmine 管理資訊頁面會持續提醒直到變更完成。
🗄
資料庫配置 database.yml
RedmineInstall §Step 3

MySQL 配置

production:
  adapter: mysql2
  database: redmine
  host: localhost
  username: redmine
  password: "your_password"
  encoding: utf8mb4
  # port: 3306

PostgreSQL 配置

production:
  adapter: postgresql
  database: redmine
  host: localhost
  username: redmine
  password: "your_password"
  encoding: utf8
  # schema_search_path: public

SQLite 配置

production:
  adapter: sqlite3
  database: db/redmine.sqlite3
限制：SQLite 僅適用於開發/測試環境，不建議用於生產。

SQL Server 配置

production:
  adapter: sqlserver
  database: redmine
  host: localhost\\SQLEXPRESS
  username: redmine
  password: "your_password"
🐬
MySQL 專項配置
MySQL_configuration

database.yml 調整

確保 Redmine 的 database.yml 中 adapter 使用 mysql2（非 mysql），且 encoding 使用 utf8mb4 以支援完整 Unicode（含 Emoji）。

adapter: mysql2
encoding: utf8mb4

MySQL 伺服器設定 my.cnf

修改 MySQL 的 transaction_isolation 以確保相容性：

[mysqld]
transaction_isolation = READ-COMMITTED
說明：Redmine 建議使用 READ-COMMITTED 隔離級別，而非 MySQL 預設的 REPEATABLE-READ，以避免特定場景下的鎖定問題。

Email / SMTP 設定
EmailConfiguration

設定檔位置：config/configuration.yml（從 configuration.yml.example 複製）。
所有設定位於 production: 區塊下的 email_delivery: 鍵。

主要配置指令
指令 說明 可選值
delivery_method 郵件傳送方式 :smtp、:sendmail、:async_smtp、:async_sendmail
smtp_settings.address SMTP 伺服器位址 如 smtp.gmail.com
smtp_settings.port SMTP 連接埠 25、465（SSL）、587（TLS）
smtp_settings.domain HELO 網域 你的網域名稱
smtp_settings.authentication 認證方式 :login、:plain、:cram_md5、nil
smtp_settings.enable_starttls_auto 自動啟用 STARTTLS true / false
smtp_settings.ssl 使用 SSL 連線 true / false
smtp_settings.user_name SMTP 登入帳號 —
smtp_settings.password SMTP 登入密碼 —
非同步郵件：使用 :async_smtp 或 :async_sendmail 可避免郵件發送時阻塞 Web 請求，特別建議在通知頻繁的環境中使用。
📋
Email 設定範例 configuration.yml
EmailConfiguration §Examples

Gmail / Google Workspace

production:
  email_delivery:
    delivery_method: :smtp
    smtp_settings:
      address: "smtp.gmail.com"
      port: 587
      domain: "smtp.gmail.com"
      authentication: :login
      user_name: "your@gmail.com"
      password: "app-password"
      enable_starttls_auto: true
提示：Gmail 需使用「應用程式密碼」而非帳號密碼（需啟用兩步驟驗證後生成）。

Office 365 / Exchange Online

production:
  email_delivery:
    delivery_method: :smtp
    smtp_settings:
      address: "smtp.office365.com"
      port: 587
      domain: "your-domain.com"
      authentication: :login
      user_name: "user@your-domain.com"
      password: "password"
      enable_starttls_auto: true

簡單 Login 認證

production:
  email_delivery:
    delivery_method: :smtp
    smtp_settings:
      address: "mail.example.com"
      port: 25
      authentication: :login
      user_name: "redmine@example.com"
      password: "password"

使用 sendmail（無需 SMTP 伺服器）

production:
  email_delivery:
    delivery_method: :sendmail
    # sendmail_settings:
    #   location: /usr/sbin/sendmail
    #   arguments: '-i'

適用於伺服器已安裝 sendmail 或 postfix 且配置完成的場景。

常見疑難排解
EmailConfiguration §Troubleshooting
問題 原因 解決方式
hostname was not match with the server certificate SMTP 伺服器的 SSL 證書主機名與設定的 address 不一致 確認 address 與證書的 CN/SAN 相符，或設定 openssl_verify_mode: 'none'（不建議用於生產）
Mail failure - no recipient addresses 郵件收件人欄位為空 確認使用者帳號有設定有效的 Email 地址
Timeout::Error Port 不正確或被防火牆阻擋；或應使用 SSL 但設定為 STARTTLS 確認 port 與加密方式的搭配：
465 → ssl: true
587 → enable_starttls_auto: true
Net::ReadTimeout SMTP 伺服器回應過慢或連線品質差 增加 read_timeout、open_timeout 設定值；或考慮使用 :async_smtp
安裝時 bundle install 失敗 缺少系統開發庫 安裝必要套件：
apt install libmysqlclient-dev（MySQL）
apt install libpq-dev（PostgreSQL）
apt install imagemagick libmagickwand-dev（RMagick）
啟動時 secret_token 錯誤 未執行步驟 5 或檔案遺失 重新執行 bundle exec rake generate_secret_token

安裝後重要設定與生產部署
RedmineInstall §Configuration / §Notes

📁 附件存儲路徑

預設存於 files/ 目錄。可在 configuration.yml 中修改：

production:
  attachments_storage_path: /var/data/redmine/files

🔧 SCM 執行檔路徑

若版本控制工具不在系統 PATH 中，需手動指定：

production:
  scm_subversion_command: /usr/bin/svn
  scm_git_command: /usr/bin/git
  scm_mercurial_command: /usr/bin/hg

📝 日誌配置

預設日誌位於 log/production.log，可配置級別：

# config/additional_environment.rb
config.log_level = :warn
# 可用級別: :debug :info :warn :error :fatal

🌐 生產環境 Web 伺服器

不建議使用內建 WEBrick 作為生產伺服器。推薦方案：

  • Passenger + Apache/Nginx（最常用）
  • Puma + Nginx（反向代理）
  • Unicorn + Nginx
  • Thin / Mongrel（較少用）

🐧 Linux 注意事項

  • 建議建立專用系統使用者 redmine
  • 使用 systemd service 管理
  • 設定 logrotate 管理日誌

🪟 Windows 注意事項

  • 使用 RubyInstaller 安裝 Ruby
  • DEVKIT 為必要元件
  • 路徑中避免空白與中文

📦 替代安裝方式

  • Bitnami 一鍵安裝包
  • Docker 映像檔
  • 各 Linux 發行版套件
  • Cloudron / 雲端托管
測試郵件功能：安裝完成後可用 Rake 任務測試郵件發送：
RAILS_ENV=production bundle exec rake redmine:email:test[user@example.com]
