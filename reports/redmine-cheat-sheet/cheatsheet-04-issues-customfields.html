追蹤器 Trackers
RedmineIssueTrackingSetup §Trackers

追蹤器是議題的分類型態，決定了議題可用的欄位和工作流程。Redmine 預設提供三個追蹤器，可自由新增。

預設追蹤器

名稱典型用途
Bug程式缺陷回報與修復追蹤
Feature新功能需求與規劃
Support支援請求與技術諮詢

常見自訂追蹤器範例：Task、Epic、User Story、Improvement、Documentation、Test Case

追蹤器設定項目

  • 名稱 — 顯示名稱
  • 預設狀態 — 新建議題的初始狀態
  • 描述 — 追蹤器說明文字
  • 標準欄位 — 選擇此追蹤器要顯示哪些標準欄位（指派、分類、目標版本、開始日期、到期日、預估工時、完成百分比等）
  • 專案關聯 — 哪些專案可使用此追蹤器
  • 自訂欄位 — 此追蹤器可用的自訂欄位
最佳實務：不要建太多追蹤器。每個追蹤器都需要獨立配置工作流程，數量過多會大幅增加維護成本。

議題狀態 Issue Statuses
RedmineIssueTrackingSetup §Statuses

預設狀態

狀態已關閉？說明
New否新建立的議題
In Progress否正在處理中
Resolved否已解決，待驗證
Feedback否需要更多資訊
Closed是已關閉完成
Rejected是已拒絕不處理

狀態屬性

  • 名稱 — 狀態顯示名稱
  • 已關閉（Is closed）— 標記為「已關閉」的狀態表示議題已完結。在統計和篩選中會歸類為已關閉議題
  • 排序 — 狀態在下拉選單中的排序

% 完成度與狀態的關係

在「管理 → 設定 → 議題追蹤」中可設定完成度的計算方式：

  • 使用議題欄位 — 手動填寫 0~100%
  • 使用議題狀態 — 每個狀態對應固定的完成百分比（如 New=0%, In Progress=50%, Closed=100%）
典型狀態流轉範例：
New → In Progress → Resolved → Closed
↳ 可穿插 Feedback → In Progress （退回重做）

工作流程 — 狀態轉換 Workflow Transitions
RedmineIssueTrackingSetup §Workflow

工作流程定義了哪個角色在哪個追蹤器中，可以將議題從哪個狀態轉換到哪個狀態。這是 Redmine 議題管理的核心機制。

設定介面

在「管理 → 工作流程」頁面，選擇：

  • 角色（如 Manager, Developer）
  • 追蹤器（如 Bug, Feature）

然後在矩陣表格中勾選允許的狀態轉換。行 = 目前狀態，列 = 目標狀態。打勾 = 允許此角色執行此轉換。

重要：工作流程是按「角色 × 追蹤器」的組合配置的。如果有 5 個角色和 4 個追蹤器，就需要配置 20 個工作流程矩陣。

工作流程矩陣範例（Developer × Bug）

目前 ↓ ╲ 目標 → New In Progress Resolved Feedback Closed Rejected
New—✔✔
In Progress—✔✔
Resolved✔—✔
Feedback✔—
Closed✔—

✔ = 允許轉換，空白 = 不允許

快速複製：配置完一個工作流程後，可使用頁面頂部的「複製」功能，將設定複製到其他角色或追蹤器，再微調差異。

工作流程 — 欄位權限 Field Permissions
RedmineIssueTrackingSetup §Fields permissions

除了狀態轉換，工作流程還可以控制每個狀態下各欄位的讀寫權限，實現更精細的流程控制。

欄位權限類型

權限說明
可讀寫該角色在此狀態下可修改此欄位（預設）
唯讀欄位可見但不可修改
必填此狀態下必須填寫此欄位才能儲存

使用場景範例

  • 狀態為「Closed」時，將「指派給」和「到期日」設為唯讀
  • 狀態為「Resolved」時，要求必填「完成百分比」為 100%
  • Reporter 角色不能修改「指派給」欄位
  • 轉為「Feedback」時必須填寫「備註」

自訂欄位 Custom Fields
RedmineCustomFields

自訂欄位讓管理員擴展 Redmine 的資料模型，為各種對象新增額外的欄位。自訂欄位可以設為必填、可篩選、可搜尋。

自訂欄位通用屬性
屬性說明
格式 Format欄位資料類型（見下方格式表）
名稱欄位顯示名稱
說明欄位提示文字
必填是否必須填寫
用於所有專案或僅限指定專案
用於所有追蹤器或僅限指定追蹤器
可搜尋是否出現在搜尋結果中
用作篩選是否可在議題列表中作為篩選條件
預設值新建時自動填入的值
可多值是否允許選擇多個值（限列表類型）
文字格式化長文字欄位可啟用 Textile/Markdown
連結至 URL自動將值轉為超連結（可用 %value% 佔位符）

自訂欄位格式 Custom Field Formats
RedmineCustomFields §Format
格式說明驗證 / 特殊功能
Boolean是/否 勾選框—
Date日期選擇器—
File檔案上傳欄位—
Float浮點數可設最小/最大值
Integer整數可設最小/最大值
Key/value list鍵值對列表內部存鍵、顯示值；支援多值

列舉值 Enumerations
RedmineEnumerations

列舉值定義了 Redmine 中幾個關鍵下拉選單的選項。在「管理 → 列舉」中統一管理。

📄 文件分類 Document categories

用於文件模組中分類文件，例如：

  • User documentation
  • Technical documentation

可自訂新增如：設計文件、會議記錄、規格書等

🔥 議題優先權 Issue priorities

議題的優先程度，預設值：

  • Low
  • Normal（預設）
  • High
  • Urgent
  • Immediate

可設定哪個為預設值；可新增/重新排序

⏱ 時間追蹤活動 Activities

記錄工時時選擇的活動類型，例如：

  • Design — 設計
  • Development — 開發
  • Testing — 測試
  • Documentation — 文件撰寫
  • Meeting — 會議

各專案可覆蓋全域活動（新增或停用）

刪除列舉值：如果某個列舉值已被使用，刪除時系統會要求你選擇一個替代值來取代現有資料。確保選擇合適的替代值以避免資料丟失。
專案覆蓋：時間追蹤活動可在專案設定 → 活動（時間追蹤）中針對該專案停用某些活動或新增專案專屬活動，而不影響其他專案。
