快速入門流程（5 步上手）
Getting_Started

① 建立專案

管理員建立專案、啟用模組（議題、Wiki、版本庫等）、選擇追蹤器。設定專案資訊與描述。

② 加入使用者

管理員建立帳號或啟用自行註冊。將使用者加入專案並指派角色（Manager / Developer / Reporter）。

③ 設定工作流程

配置「角色 × 追蹤器」的狀態轉換矩陣。決定誰可以將議題從哪個狀態移到哪個狀態。

提示：Resolved ≠ Closed。Resolved 表示開發者認為已修好，Closed 表示經驗證確認完成。

④ 建立議題 & 開始工作

團隊成員建立議題（Bug / Feature / Support），填寫描述、指派給對應人員、設定優先權和目標版本。指派者更新狀態和完成度。

⑤ 關閉議題

驗證後將狀態改為 Closed。使用路線圖追蹤版本進度。使用甘特圖和日曆視覺化時程。

我的帳號 My Account
RedmineAccounts

個人資訊 Information

  • 名字 / 姓氏 — 顯示名稱
  • Email — 主要信箱，用於通知和 Gravatar
  • 額外 Email — 可新增多個（接收通知用）
  • 語言 — 介面語言偏好
  • 變更密碼 — 需輸入目前密碼

偏好設定 Preferences

  • 隱藏 Email — 不在個人檔案中公開信箱
  • 時區 — 設定個人時區
  • 首頁 — 登入後導向哪個頁面
  • 通知選項 — 選擇接收哪些事件通知
  • 不要傳送自己的異動通知 — 自己操作不通知自己
  • 顯示評論 — 從新到舊或從舊到新
  • 關注自己編輯的議題 — 自動加入 Watcher
API 存取金鑰：在「我的帳號」頁面右側可查看 API 金鑰，用於 REST API 認證。可隨時重新產生（舊金鑰立即失效）。
🔐
登入 & 註冊
RedmineLogin · RedmineRegister

登入 Login

  • 輸入帳號（或 Email）與密碼
  • 記住我 — 使用 cookie 維持登入狀態（管理員可設定天數或停用）
  • 忘記密碼 — 透過註冊 Email 發送密碼重設連結（管理員可停用此功能）
  • 支援 LDAP 和 OpenID 登入（依系統設定）

自行註冊 Register

管理員可在設定中選擇三種帳號啟用方式：

方式流程
Email 驗證使用者註冊 → 收到驗證信 → 點擊連結啟用
管理員審核使用者註冊 → 管理員在後台手動啟用
自動啟用使用者註冊後立即可登入

也可完全停用自行註冊功能，僅由管理員建立帳號。

搜尋功能 Search
RedmineSearch

搜尋範圍

  • 全域搜尋 — 在頁面右上角搜尋框輸入關鍵字
  • 專案內搜尋 — 進入專案後，搜尋僅限該專案
  • 可勾選搜尋的範圍類型：議題、新聞、文件、Wiki 頁面、Changesets、論壇訊息

搜尋選項

  • 僅搜尋標題 — 只搜尋議題主旨和名稱
  • 搜尋自訂欄位 — 可選擇是否包含自訂欄位內容
  • 開啟/已關閉 — 篩選議題的開啟/關閉狀態
  • 快捷搜尋 — 輸入 #123 直接跳轉到議題 #123

專案概覽 & 活動
RedmineProjectOverview · RedmineProjectActivity

專案概覽頁面

進入專案後的首頁，顯示以下資訊：

  • 專案描述 — 格式化文字
  • 議題追蹤區 — 各追蹤器的開啟/已關閉議題數量統計
  • 成員區 — 專案成員清單及角色
  • 最新新聞 — 最近發佈的新聞項目
  • 自訂欄位 — 專案層級的自訂欄位值

專案活動頁面

以時間線方式顯示專案中所有操作的歷史紀錄。可篩選的活動類型：

  • 議題建立 / 狀態變更 / 關閉
  • 備註新增
  • 版本庫 Changeset 提交
  • 新聞發佈
  • 論壇訊息 / 回覆
  • Wiki 頁面編輯
  • 檔案 / 文件新增
  • 工時記錄

顯示天數範圍由管理員在全域設定中控制（預設 30 天）

議題操作 Issues
RedmineIssues

新增議題

  • 選擇追蹤器類型
  • 填寫主旨（Subject）和描述
  • 設定狀態、優先權、指派對象
  • 選擇目標版本、分類
  • 設定開始/到期日期
  • 設定預估工時
  • 加入關注者（Watcher）
  • 上傳附件
  • 設定父任務（子任務功能）
  • 填寫自訂欄位

更新議題

  • 新增備註 — 在議題下方留言，支援格式化
  • 變更屬性 — 狀態、指派、優先權等
  • 記錄工時 — 可在更新時同時記錄花費時間
  • 上傳附件 — 追加檔案
  • 私密備註 — 僅有權限的角色可見
快速更新：更新議題時可同時變更多個屬性並新增備註，全部操作在一次提交中完成。

編輯 vs 更新

  • 更新（Update）— 新增備註 + 變更屬性，會產生歷史紀錄
  • 編輯（Edit）— 修改議題的「主旨」或「描述」內容本身（需權限），修改也會記錄在歷史中

子任務 Subtasks

  • 任何議題都可以有子任務
  • 子任務自動顯示在父任務頁面
  • 父任務的到期日、完成度可自動從子任務推算（依系統設定）
  • 支援跨專案子任務（依設定）

議題標準欄位一覽
RedmineIssues §Fields
欄位說明可選/必填
追蹤器 Tracker議題類型（Bug / Feature / Support 等）必填
主旨 Subject議題標題，簡明描述問題或需求必填
描述 Description詳細說明，支援 Textile/Markdown 格式化選填
狀態 Status目前進度狀態（受工作流程控制）必填
優先權 Priority緊急程度（Low / Normal / High / Urgent / Immediate）必填
指派給 Assignee負責處理的使用者或群組選填
分類 Category專案內的議題分類選填
目標版本 Target version預計在哪個版本中處理選填
父任務 Parent task上層議題（建立子任務關係）選填
開始日期 Start date議題開始處理的日期選填
到期日 Due date預計完成期限選填
預估工時 Estimated time預估所需小時數選填
完成百分比 % Done0~100% 進度（或由狀態自動計算）選填
私密 Private僅特定角色可見的私密議題選填

議題關聯類型 Issue Relations
RedmineIssues §Related issues
關聯類型說明對排程的影響
related to
相關		兩個議題有關聯但無方向性無
duplicates
重複		此議題與另一議題重複關閉原始議題時可自動關閉重複議題
duplicated by
被重複		此議題被另一議題標記為重複同上（反向）
blocks
阻擋		此議題完成前，另一議題無法開始被阻擋的議題開始日期不可早於此議題的到期日
blocked by
被阻擋		此議題被另一議題阻擋同上（反向）
precedes
前置		此議題應在另一議題之前完成後續議題開始日期自動設為此議題到期日 + 延遲天數
follows
後續		此議題跟隨在另一議題之後同上（反向）
copied to / copied from
複製		議題複製時自動建立的關聯無
甘特圖整合：precedes/follows 和 blocks/blocked by 關聯會影響甘特圖的排程計算和依賴線顯示。

議題列表 Issue List
RedmineIssueList

篩選器 Filters

議題列表支援強大的多條件篩選：

  • 依狀態（開啟/已關閉/指定狀態）
  • 依追蹤器、優先權、指派對象
  • 依版本、分類、作者
  • 依日期範圍（建立/更新/開始/到期）
  • 依自訂欄位值
  • 依關注者、子專案

篩選條件支援AND 邏輯組合。

自訂查詢 Custom Queries

篩選條件可儲存為自訂查詢：

  • 個人查詢 — 僅自己可見
  • 公開查詢 — 全專案成員可見（需權限）
  • 可設定查詢的預設欄位顯示
  • 可設定預設排序
  • 可設定群組依據（依狀態、指派者等群組顯示）
  • 可設定欄位加總（如預估工時加總）

右鍵快捷選單

在議題列表中對議題按右鍵可快速：

  • 編輯 / 複製 / 刪除
  • 變更狀態 / 優先權 / 指派
  • 變更追蹤器 / 版本
  • 新增備註
  • 加入/移除關注

批次編輯 Bulk Edit

勾選多個議題後可一次性修改：

  • 批次變更狀態、指派、優先權
  • 批次設定版本、分類
  • 批次移動到其他專案
  • 批次複製
  • 批次刪除
操作範圍：批次編輯時留空的欄位不會被修改，僅有選擇了新值的欄位才會更新。
匯出格式：議題列表可匯出為 CSV 和 PDF 格式。匯出量受管理員設定的上限限制。側邊欄也提供 Atom feed 訂閱連結。

議題摘要報告 Issue Summary
RedmineIssueSummary

摘要總覽

以矩陣方式顯示各追蹤器的議題統計：

  • 各追蹤器的開啟/已關閉議題數
  • 開啟議題佔比
  • 點擊數字可直接跳轉到對應的篩選議題列表

詳細報告 Detail Reports

可依不同維度交叉分析議題分佈：

  • 依追蹤器分組
  • 依優先權分組
  • 依指派對象分組
  • 依作者分組
  • 依版本分組
  • 依分類分組
  • 依自訂欄位分組

路線圖 Roadmap & 版本概覽 Version
RedmineRoadmap · RedmineVersion

路線圖 Roadmap

以版本（Version / Milestone）為中心展示開發進度：

  • 每個版本的完成進度條（基於已關閉議題比例）
  • 版本的開啟/已關閉議題數量
  • 版本到期日與剩餘天數
  • 各版本包含的議題列表
  • 側邊欄可篩選顯示哪些追蹤器
  • 可選擇是否顯示已完成的版本

版本概覽 Version

點擊特定版本進入詳細頁面：

  • 議題列表與進度條 — 顯示此版本的所有議題及完成率
  • Wiki 頁面內容 — 版本可關聯一個 Wiki 頁面，自動顯示
  • 時間追蹤統計 — 此版本的總預估工時與已花費工時
  • 議題統計分組 — 依指派對象、作者、分類、優先權、追蹤器、狀態等維度統計
版本狀態：版本可設為 Open（開發中）、Locked（已凍結不接受新議題）、Closed（已發佈完成）。路線圖預設僅顯示 Open 版本。
