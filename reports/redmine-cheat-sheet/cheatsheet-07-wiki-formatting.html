時間追蹤 Time Tracking
RedmineTimeTracking

記錄工時的方式

  • 議題頁面 — 更新議題時可同時記錄花費時間
  • 專案層級 — 「時間追蹤」→「記錄工時」，可手動指定議題編號（選填）
  • Commit 訊息 — 在 commit message 中使用特殊語法自動記錄
Commit 工時語法：
refs #123 @2h30m — 為議題 #123 記錄 2.5 小時
fixes #456 @1.5h — 關閉議題 #456 並記錄 1.5 小時

工時記錄欄位

欄位說明
議題關聯的議題編號（選填）
日期工時發生日期
時數花費時間（支援多種格式）
活動活動類型（開發/設計/測試等）
備註工時說明文字

時間格式

1.5 → 1h30m 1h30 → 1h30m 1:30 → 1h30m 1h → 1h 30m → 30m 1h30m → 1h30m

工時明細 & 報告
RedmineTimelogDetails · RedmineTimelogReport

工時明細 Details

逐筆顯示所有工時記錄，支援：

  • 依日期範圍篩選
  • 依使用者、活動、議題篩選
  • 依自訂欄位篩選
  • 結果可匯出 CSV
  • 顯示每筆記錄的使用者、日期、活動、議題、備註

工時報告 Report

以彙總方式顯示工時統計：

  • 橫軸：年 / 月 / 週 / 日
  • 縱軸：依專案 / 使用者 / 追蹤器 / 活動 / 議題 / 分類 / 版本分組
  • 交叉表格顯示工時加總
  • 結果可匯出 CSV

甘特圖 & 日曆
RedmineGantt · RedmineCalendar

甘特圖 Gantt

  • 圖形化顯示議題的開始日期 → 到期日時間線
  • 依版本和議題雙層結構組織
  • 顯示完成百分比進度條
  • 顯示議題間的前置/後續依賴線
  • 可調整顯示月份範圍
  • 支援篩選器（與議題列表相同）
  • 可匯出為 PNG 圖片和 PDF
效能提示：甘特圖有最大項目數限制（管理員設定），議題太多時建議使用篩選器縮小範圍。

日曆 Calendar

  • 月曆視圖顯示議題的開始日期和到期日
  • 議題以色塊標示在對應的日期上
  • 可切換月份導覽
  • 支援篩選器控制顯示內容
  • 需在專案中啟用 Calendar 模組
  • 權限控制：需有「瀏覽日曆」權限

新聞・文件・檔案
RedmineNews · RedmineDocuments · RedmineFiles

📰 新聞 News

  • 發佈專案相關公告
  • 支援標題、摘要、正文
  • 正文支援格式化語法
  • 使用者可留言評論
  • 新聞會顯示在專案概覽頁面
  • 新增新聞會觸發郵件通知
  • 提供 Atom feed 訂閱

📄 文件 Documents

  • 上傳和管理專案文件
  • 依文件分類組織（由列舉值定義）
  • 每份文件可含多個附件
  • 支援文件標題和描述
  • 適合存放規格書、設計文件等

📦 檔案 Files

  • 專案的檔案發佈區（如軟體版本發佈）
  • 檔案可關聯到版本
  • 記錄上傳日期和大小
  • 提供下載連結
  • 適合發佈可交付物、安裝包
💬
論壇 Forums
RedmineForums

功能概覽

  • 每個專案可建立多個論壇看板
  • 支援主題和回覆的討論串結構
  • 回覆支援格式化語法與附件
  • 主題可設為置頂（Sticky）
  • 主題可被鎖定（不再接受回覆）
  • 回覆新訊息會觸發郵件通知（給關注者）

權限控制

  • 管理論壇 — 建立/編輯/刪除看板
  • 發佈訊息 — 建立新主題和回覆
  • 編輯訊息 — 編輯所有人的訊息
  • 編輯自己的訊息 — 僅能編輯自己的
  • 刪除訊息 — 刪除所有人的訊息
  • 刪除自己的訊息 — 僅能刪除自己的

Wiki
RedmineWikis

Wiki 功能

  • 每個專案有一個 Wiki
  • 支援無限層級的頁面父子關係
  • 使用 Textile 或 Markdown 語法
  • 完整的版本歷史和差異比較
  • 支援附件上傳
  • 提供側邊欄（建立名為 Sidebar 的頁面即自動顯示）
  • 可將 Wiki 頁面匯出為 HTML 或 TXT

Wiki 頁面操作

操作說明
建立新頁面在現有頁面中使用 [[PageName]] 語法建立連結，點擊紅色連結建立新頁面
建立子頁面編輯頁面時設定 Parent page
重新命名/移動更改頁面名稱或重新指定父頁面
保護/鎖定防止非授權使用者編輯
刪除刪除頁面（可選擇是否刪除子頁面）
歷史/差異查看版本歷史和版本間差異
關注關注頁面變更通知

版本庫瀏覽 Repository
RedmineRepository · RedmineRepositoryStatistics

瀏覽功能

  • 檔案瀏覽器 — 樹狀目錄結構，可展開瀏覽
  • 程式碼檢視 — 支援語法高亮顯示
  • 修訂歷史 — 每個檔案/目錄的修訂歷史
  • 差異比較 — 檢視任意兩個版本間的差異（inline / side-by-side）
  • Changeset 詳情 — 查看每次提交的完整變更
  • 議題引用 — commit message 中引用的議題自動建立雙向連結

版本庫統計

  • 提交活動圖 — 按月/日顯示提交頻率
  • 貢獻者統計 — 各作者的提交數量與佔比

Commit 訊息引用語法

在 commit message 中使用特定關鍵字觸發議題操作：

關鍵字效果
refs #123引用議題 #123（建立連結）
fixes #123引用並關閉議題 #123
closes #123同 fixes（關閉議題）

關鍵字可在管理員設定中自訂

Redmine 專用連結語法
RedmineTextFormatting §Redmine links

Redmine 擴展了格式化語法，加入了特殊的內部連結功能（Textile 和 Markdown 通用）：

語法連結到範例
#123議題 #123請參考 #123
##123議題 #123（含主旨文字）##123 → Bug #123: 修正登入問題
r456修訂版本 456已在 r456 修正
commit:abcd123Git commit hash—
[[Wiki頁面]]同專案 Wiki 頁面詳見 [[安裝說明]]
[[Wiki頁面|顯示文字]]指定顯示文字的 Wiki 連結[[FAQ|常見問題]]
[[專案識別碼:Wiki頁面]]跨專案 Wiki 連結[[backend:API文件]]
news:標題新聞項目—
version:1.0.0版本—
attachment:file.zip當前資源的附件下載 attachment:patch.diff
source:path/file.rb版本庫檔案—
source:path/file@123特定修訂版的檔案—
user:john使用者個人頁面—
常用巨集：{{toc}} 自動目錄、{{child_pages}} 子頁面列表、{{include(Wiki頁面)}} 嵌入其他 Wiki 頁面內容、{{collapse(標題) ... }} 可摺疊區塊。

鍵盤快捷鍵 Accesskeys
RedmineKeyboardNavigation

Redmine 提供 Accesskey 快捷鍵。按鍵組合依瀏覽器不同：Chrome/Edge = Alt+Shift+[key]，Firefox = Alt+Shift+[key]，Safari/macOS = Ctrl+Option+[key]

快捷鍵功能
Accesskey + 1首頁
Accesskey + 2我的頁面
Accesskey + 3專案列表
Accesskey + 4搜尋
Accesskey + 7新增議題
快捷鍵功能
Accesskey + e切換到編輯器（編輯/更新頁面時）
Accesskey + p預覽（編輯模式中）
Accesskey + f聚焦搜尋框
P前一頁（議題列表分頁）
N下一頁（議題列表分頁）
