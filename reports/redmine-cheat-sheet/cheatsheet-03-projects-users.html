專案管理 Projects
RedmineProjects

Redmine 核心以「專案」為中心組織所有資源。每個專案可擁有自己的議題、Wiki、版本庫、論壇、文件等模組。專案支援無限層級的父子關係（子專案）。

新增/編輯專案欄位

欄位說明
名稱專案顯示名稱
識別碼URL 用英數字串（建立後不可改）
說明支援 Textile/Markdown 格式化
首頁可指定 URL 作為專案首頁
公開勾選後非會員也可瀏覽
父專案設定上層專案（子專案功能）
繼承成員是否繼承父專案成員
追蹤器選擇此專案可用的追蹤器
自訂欄位選擇要啟用的自訂欄位

專案生命週期操作
RedmineProjects §Operations
操作效果可逆？注意事項
關閉
Close		 專案變為唯讀，所有資料保留可瀏覽，但不可新增或修改 ✔ 可重新開啟 適合已完成的專案，成員仍可檢視歷史資料
封存
Archive		 專案從所有列表消失（含子專案），無法存取 ✔ 可解除封存 子專案會一併封存；需管理員權限才能解除
複製
Copy		 建立專案副本，可選擇複製哪些資料 — 可複製：成員、版本、分類、議題、Wiki、版本庫設定、論壇、時間記錄、查詢
刪除
Delete		 永久刪除專案及所有相關資料 ✘ 不可逆 包含所有議題、Wiki、附件、時間記錄等全部銷毀。需輸入專案識別碼確認
最佳實務：考慮先「封存」而非直接刪除。封存的專案不佔搜尋資源但資料完整保留。

認證方式

Redmine 支援多種使用者認證方式：

  • 內建（Internal）— 密碼存於 Redmine 資料庫
  • LDAP — 連接 Active Directory / OpenLDAP
  • LDAP + 即時建立— 首次 LDAP 登入自動建立帳號
  • OpenID（需在設定中啟用）

郵件通知選項

  • 所有我的專案中的事件
  • 僅我關注或參與的事件
  • 僅指派給我的事件
  • 僅我擁有的事件
  • 無通知

使用者也可以在個人設定中覆蓋此選項

使用者帳號狀態
RedmineUsers §Status
狀態圖示說明可登入？管理員操作
啟用 Active ● 正常使用中的帳號 ✔ 可鎖定或刪除
已註冊 Registered ● 已自行註冊但尚未通過管理員審核或 Email 驗證 ✘ 可啟用或刪除
已鎖定 Locked ● 被管理員鎖定的帳號，無法登入但資料保留 ✘ 可解鎖或刪除
離職處理建議：使用「鎖定」而非「刪除」。鎖定後使用者無法登入但議題歷史紀錄（指派、報告者）完整保留。刪除使用者會將相關資料改為「匿名」。
自行註冊設定：管理員可在「設定 → 認證」中控制自行註冊模式：帳號啟用方式可為「Email 驗證」、「管理員手動啟用」或「自動啟用」。

群組管理 Groups
RedmineGroups

群組概念

群組是管理多個使用者的便捷方式。可以用群組：

  • 批次加入專案成員 — 一次將整個群組加入專案並分配角色
  • 指派議題 — 議題可以指派給群組（需在設定中啟用）
  • 簡化權限管理 — 修改群組角色即影響所有成員

群組操作

操作說明
建立群組指定群組名稱（顯示名稱）
加入成員將使用者加入群組；一人可屬於多個群組
關聯專案將群組加入專案並指定角色
自訂欄位群組也可有自己的自訂欄位
刪除群組移除群組（成員帳號不受影響）
注意：修改群組的專案角色時，群組內所有成員的權限會立即同步更新，無需逐一調整。

角色定義 Roles
RedmineRoles

角色定義了使用者在專案中可執行的操作。同一使用者在不同專案中可有不同角色。Redmine 預設安裝含 Manager、Developer、Reporter 三個角色。

角色屬性

屬性說明
名稱角色顯示名稱
議題可指派給此角色勾選後此角色的成員會出現在議題「指派給」下拉選單中
議題可見範圍全部 / 僅公開議題 / 自己建立或指派的
使用者可見範圍全部啟用使用者 / 僅同專案成員
時間記錄可見範圍全部 / 僅自己的
權限勾選各項功能權限（見下方總覽）

預設角色範例

角色典型權限範圍
Manager專案全功能管理（成員、模組、版本、議題、Wiki、論壇等）
Developer議題操作（查看、新增、編輯）、時間記錄、Wiki 編輯、版本庫瀏覽
Reporter議題查看與回報、基本 Wiki 瀏覽
自訂角色：可任意新增角色，例如「QA」、「外部客戶」、「只讀觀察者」等，依實際需求配置權限。

系統角色（Non-member & Anonymous）
RedmineRoles §System roles

非成員 Non-member

適用於：已登入但不是該專案成員的使用者存取公開專案時的權限。

  • 不可刪除此角色
  • 可配置允許的權限（如瀏覽議題、瀏覽 Wiki 等）
  • 典型設定：僅允許瀏覽公開資訊

匿名 Anonymous

適用於：未登入的訪客存取公開專案時的權限。

  • 不可刪除此角色
  • 權限應比 Non-member 更加限制
  • 若「設定 → 認證」中勾選「需要認證」，此角色無效
  • 典型設定：僅允許瀏覽議題清單
安全建議：生產環境中建議在「管理 → 設定 → 認證」中勾選「需要認證」，確保所有頁面都需登入才可存取。若需對外公開特定專案，則謹慎配置 Anonymous 角色權限。
