外掛開發教學 Plugin Tutorial
Plugin_Tutorial
1

建立外掛骨架

bundle exec rails generate redmine_plugin MyPlugin
# 建立於 plugins/my_plugin/ 目錄

自動生成 init.rb、基本目錄結構。

2

建立 Model（資料模型）

bundle exec rails generate redmine_plugin_model MyPlugin poll title:string description:text
# 生成 migration 和 model 檔案

RAILS_ENV=production bundle exec rake redmine:plugins:migrate
# 執行資料庫遷移
3

建立 Controller（控制器）

bundle exec rails generate redmine_plugin_controller MyPlugin polls index show
# 生成 controller 和 views 檔案

在 config/routes.rb 中定義路由：

# plugins/my_plugin/config/routes.rb
resources :polls, only: [:index, :show]
4

國際化（I18n）

# plugins/my_plugin/config/locales/en.yml
en:
  label_poll_plural: "Polls"
  label_poll_new: "New Poll"

# plugins/my_plugin/config/locales/zh-TW.yml
zh-TW:
  label_poll_plural: "投票"
  label_poll_new: "新增投票"
5

擴充選單與權限

# plugins/my_plugin/init.rb
Redmine::Plugin.register :my_plugin do
  name 'My Plugin'
  author 'Your Name'
  version '1.0.0'

  # 加入專案選單
  menu :project_menu, :polls,
    { controller: 'polls', action: 'index' },
    caption: :label_poll_plural, after: :activity,
    param: :project_id

  # 定義權限
  permission :view_polls, polls: [:index, :show]
  permission :manage_polls, polls: [:new, :create, :edit]

  # 定義專案模組
  project_module :polls do
    permission :view_polls, polls: [:index, :show]
    permission :manage_polls, polls: [:new, :create]
  end
end

init.rb 完整範例

Redmine::Plugin.register :my_plugin do
  name        'My Plugin'
  author      'Your Name'
  description 'Plugin description'
  version     '1.0.0'
  url         'https://github.com/...'
  author_url  'https://yoursite.com'

  # 要求 Redmine 最低版本
  requires_redmine version_or_higher: '5.0'

  # 要求其他外掛
  requires_redmine_plugin :other_plugin,
    version_or_higher: '0.5'

  # 設定頁面
  settings default: {'option' => 'value'},
    partial: 'settings/my_plugin'

  # 選單擴充
  menu :top_menu, :my_plugin,
    { controller: '...', action: '...' }
  menu :project_menu, :my_plugin,
    { ... }, after: :wiki
  menu :application_menu, :my_plugin,
    { ... }

  # 專案模組與權限
  project_module :my_module do
    permission :view_stuff, ...
    permission :manage_stuff, ...
  end
end

外掛進階功能
Plugin_Tutorial §Hooks §Stylesheets §Settings

新增樣式表

在 assets/stylesheets/ 放置 CSS，在 init.rb 中註冊，或使用 View hook 注入 <link> 標籤。

執行 rake redmine:plugins:assets 複製到 public/。

設定頁面

在 init.rb 使用 settings 指令，建立 app/views/settings/_my_plugin.html.erb 視圖。外掛設定會自動出現在「管理 → 外掛」的設定按鈕中。

測試外掛

# 執行外掛測試
RAILS_ENV=test \
bundle exec rake \
  redmine:plugins:test \
  NAME=my_plugin

測試檔案位於 test/ 目錄，使用標準 Rails 測試框架。

Hooks API（鉤子系統）
Hooks

Hooks 允許外掛在 Redmine 核心的特定位置注入程式碼或 HTML，而不需要直接修改核心檔案。三種 Hook 類型：

View Hooks（視圖鉤子）

在頁面特定位置注入 HTML 內容。

# lib/my_plugin/hooks.rb
class MyHookListener
  < Redmine::Hook::ViewListener

  # 使用 render_on 指定模板
  render_on :view_issues_show_details_bottom,
    partial: 'issues/my_extra_fields'

  # 或使用方法直接輸出
  def view_layouts_base_html_head(ctx={})
    stylesheet_link_tag('my_styles',
      plugin: 'my_plugin')
  end
end

Controller Hooks（控制器鉤子）

在控制器動作的特定時機執行自訂邏輯。

class MyHookListener
  < Redmine::Hook::Listener

  def controller_issues_new_after_save(ctx={})
    issue = ctx[:issue]
    # 議題建立後的自訂邏輯
    MyNotifier.notify(issue)
  end
end

Model Hooks（模型鉤子）

在模型操作的特定時機觸發。

class MyHookListener
  < Redmine::Hook::Listener

  def model_project_copy_before_save(ctx={})
    source = ctx[:source_project]
    dest = ctx[:destination_project]
    # 複製專案前的處理
  end
end

外掛進階技巧 Plugin Internals
Plugin_Internals

覆寫核心 Controller / Model

使用 prepend 或 include 注入自訂行為：

# lib/my_plugin/issue_patch.rb
module MyPlugin::IssuePatch
  def assignable_users
    # 自訂可指派使用者邏輯
    users = super
    users.select { |u| u.active? }
  end
end

# 在 init.rb 或 lib/ 中
Rails.configuration.to_prepare do
  Issue.prepend MyPlugin::IssuePatch
end

覆寫核心 View

在外掛的 app/views/ 中建立同路徑同檔名的視圖，即可完全覆寫核心視圖（不推薦，升級時容易衝突）。

建議：優先使用 Hooks 注入內容而非覆寫視圖。覆寫視圖在 Redmine 升級時極易產生相容性問題。

使用 Rails Callbacks

可在外掛中使用標準 Rails callbacks（before_save、after_create 等）來為核心模型增加行為。

外掛開發 FAQ
Plugin_FAQ
問題解法
如何判斷某模組是否在專案中啟用？project.module_enabled?(:module_name)
如何修改頁面佈局？使用 View Hooks 注入 HTML/CSS/JS，避免直接覆寫視圖
如何讀取 Redmine 設定值？Setting.app_title、Setting.host_name 等
如何讀取外掛自己的設定？Setting.plugin_my_plugin['option_name']
如何限定 Redmine 最低版本？在 init.rb 中使用 requires_redmine version_or_higher: '5.0'
如何限定依賴其他外掛？requires_redmine_plugin :other_plugin, version_or_higher: '1.0'
如何移除外掛？先回退遷移：rake redmine:plugins:migrate NAME=my_plugin VERSION=0，再刪除目錄

主題開發 Theme Development
HowTo_create_a_custom_Redmine_theme

主題目錄結構

public/themes/my_theme/
├── stylesheets/
│   └── application.css — 主要樣式
├── javascripts/
│   └── theme.js — 自訂 JS
├── images/ — 圖片資源
└── favicon/
     └── favicon.ico — 自訂圖示

套用主題

管理 → 設定 → 顯示 → 佈景主題 → 選擇你的主題名稱

開發重點

  • application.css 是主題入口，會在核心 CSS 之後載入（可覆寫）
  • 使用瀏覽器開發者工具檢查核心 CSS 類名與 ID
  • 主題 CSS 可覆蓋的常見選擇器：#header、#main、#sidebar、#content、#top-menu、#main-menu、#footer
  • 可在 javascripts/theme.js 中加入自訂 JavaScript
  • 圖片用 url(../images/my-image.png) 引用
快速起步：複製核心的 public/stylesheets/application.css 到你的主題目錄作為起始範本，然後逐步修改。

社群貢獻指南
Contribute

程式碼貢獻

  • Patch 提交 — 建立 diff/patch 附加到 redmine.org 議題上
  • 程式碼規範 — 遵循 Redmine 的 Coding Standards
  • 測試 — 提交的 Patch 需附帶對應的測試
  • 文件 — 程式碼變更需更新相關文件
  • 流程：Fork → 開發 → 測試 → 建立 Issue → 附加 Patch → 等待審核

非程式碼貢獻

  • 使用者支援 — 在論壇幫助其他使用者
  • 翻譯 — 協助 Redmine 介面多語系翻譯
  • 議題分類 — 協助確認和分類 Bug 報告
  • 文件改進 — 改善 Wiki 文件內容
  • UI 設計 — 提供介面設計與使用者體驗建議
