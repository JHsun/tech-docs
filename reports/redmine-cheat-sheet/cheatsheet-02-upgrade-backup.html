版本升級概覽
RedmineUpgrade

升級路徑原則

  • Redmine 支援跨版本升級（如 4.x → 6.x），但建議依序升級大版本
  • 每次升級前務必確認 Ruby 版本相容性
  • 外掛需個別檢查是否相容新版
  • 自訂主題可能需要調整

兩種升級方式

方式適用場景
下載穩定版大多數正式環境（推薦）
SVN 更新原本就使用 SVN checkout 的環境
黃金法則：升級前務必完成完整備份（資料庫 + 附件 + 設定檔）。沒有備份就不要開始升級！
▶
升級完整步驟（5 步）
RedmineUpgrade §Step 1-5
1

檢查需求

確認目標版本對 Ruby、Rails、資料庫的版本要求。查閱 Gemfile 與官方 Release Notes。

ruby -v          # 確認 Ruby 版本
rails -v         # 確認 Rails 版本
mysql --version  # 確認資料庫版本
2

完整備份

備份資料庫、附件目錄、設定檔（詳見下方「備份策略」區塊）

資料庫 dump files/ 目錄 config/ 設定檔 外掛目錄
3

執行升級

方式 A：穩定版下載

# 解壓新版到新目錄
tar xzf redmine-6.0.0.tar.gz

# 複製舊版設定
cp old/config/database.yml new/config/
cp old/config/configuration.yml new/config/
cp old/config/initializers/secret_token.rb \
   new/config/initializers/

# 複製附件與外掛
cp -r old/files new/
cp -r old/plugins/* new/plugins/
cp -r old/public/themes/* new/public/themes/

# 如有 Gemfile.local
cp old/Gemfile.local new/

方式 B：SVN 更新

cd /path/to/redmine

# 切換到目標版本分支
svn switch \
  ^/branches/6.0-stable .

# 或更新到最新修訂
svn update

# SVN 方式會保留你的設定檔
# 和 files/ 目錄不受影響
4

更新資料庫與依賴

cd /path/to/new-redmine

# 安裝新版依賴
bundle install --without development test

# 執行資料庫遷移
RAILS_ENV=production bundle exec rake db:migrate

# 遷移外掛資料庫
RAILS_ENV=production bundle exec rake redmine:plugins:migrate
5

清理與重啟

# 清除快取與 session
RAILS_ENV=production bundle exec rake tmp:cache:clear
RAILS_ENV=production bundle exec rake tmp:sessions:clear

# 重啟 Web 伺服器
sudo systemctl restart apache2   # 或 nginx
# 或 touch tmp/restart.txt（Passenger）

升級常見問題
RedmineUpgrade §Common issues
問題原因解決方式
版本庫管理功能異常 SCM adapter 設定需更新 重新配置 configuration.yml 中的 SCM 路徑，確認 SCM 工具已安裝
secret_token 相關錯誤 升級後缺少原有 token 檔案 從備份恢復 config/initializers/secret_token.rb；若確定遺失需重新生成（所有既有 session 將失效）
外掛相容性錯誤 外掛未支援新版 Redmine 暫時移除有問題的外掛目錄，等待外掛更新。可用 bundle exec rake redmine:plugins 列出已載入外掛
db:migrate 失敗 資料庫結構衝突或版本跳躍過大 嘗試依序升級中間版本；檢查 schema_migrations 表
CSS/頁面樣式異常 瀏覽器快取或主題不相容 清除瀏覽器快取、清除 tmp/cache、檢查主題相容性

從其他系統遷移至 Redmine
RedmineMigrate

從 Trac 遷移

Redmine 內建 Trac 遷移腳本，可匯入：

  • 議題（Tickets → Issues）
  • Wiki 內容
  • 附件
  • 使用者帳號
  • 版本（Milestones）
RAILS_ENV=production \
bundle exec rake \
  redmine:migrate_from_trac

腳本會互動式詢問 Trac 資料庫位置與設定。

從 Mantis 遷移

Redmine 內建 Mantis 遷移腳本，可匯入：

  • 所有 Bug 報告
  • 使用者與分類
  • 附件與註記
  • 專案結構
RAILS_ENV=production \
bundle exec rake \
  redmine:migrate_from_mantis

其他系統 / 第三方工具

非內建支援的系統可透過：

  • REST API — 程式化匯入
  • CSV Import — 議題 CSV 匯入功能
  • 社群第三方腳本：
    • Jira → Redmine
    • Bugzilla → Redmine
    • GitHub Issues → Redmine
    • Assembla → Redmine
建議：社群腳本品質不一，使用前務必在測試環境先行驗證。

備份策略與指令
RedmineBackupRestore §Backup

完整的 Redmine 備份必須包含兩個部分：資料庫 dump + 附件檔案。建議另外備份設定檔。

資料庫備份指令

MySQL

mysqldump -u redmine \
  -p --single-transaction \
  redmine \
  > redmine_$(date +%F).sql

--single-transaction 確保一致性快照而不鎖表。

PostgreSQL

pg_dump -U redmine \
  -Fc redmine \
  > redmine_$(date +%F).dump

# -Fc = custom format（可壓縮）
# 或用 -Fp 輸出純 SQL

SQLite

# 直接複製 DB 檔案
cp db/redmine.sqlite3 \
   backup/redmine_$(date +%F).db

# 或使用 sqlite3 dump
sqlite3 db/redmine.sqlite3 \
  .dump > backup.sql
附件與設定備份

📎 附件目錄

# 預設路徑
tar czf files_$(date +%F).tar.gz \
  /path/to/redmine/files/

所有上傳的附件、文件、檔案都存在此目錄中。

⚙ 設定檔（建議一併備份）

tar czf config_$(date +%F).tar.gz \
  config/database.yml \
  config/configuration.yml \
  config/initializers/secret_token.rb \
  Gemfile.local  # 如果有的話
自動備份腳本範例
#!/bin/bash
# Redmine 完整備份腳本
BACKUP_DIR="/backup/redmine/$(date +%F)"
REDMINE_DIR="/var/www/redmine"
DB_NAME="redmine"
DB_USER="redmine"

mkdir -p $BACKUP_DIR

# 1. 資料庫備份
mysqldump -u $DB_USER -p --single-transaction $DB_NAME \
  | gzip > $BACKUP_DIR/db.sql.gz

# 2. 附件備份
tar czf $BACKUP_DIR/files.tar.gz -C $REDMINE_DIR files/

# 3. 設定備份
tar czf $BACKUP_DIR/config.tar.gz -C $REDMINE_DIR \
  config/database.yml config/configuration.yml \
  config/initializers/secret_token.rb

# 4. 保留最近 30 天備份
find /backup/redmine/ -maxdepth 1 -mtime +30 -exec rm -rf {} \;

echo "備份完成：$BACKUP_DIR"
排程建議：使用 crontab 設定每日自動執行：
0 2 * * * /usr/local/bin/redmine_backup.sh >> /var/log/redmine_backup.log 2>&1

還原流程
RedmineBackupRestore §Restore

MySQL 還原

# 從 SQL 檔還原
mysql -u redmine -p \
  redmine < redmine_backup.sql

# 從壓縮檔還原
gunzip < db.sql.gz | \
  mysql -u redmine -p redmine

PostgreSQL 還原

# custom format（-Fc）
pg_restore -U redmine \
  -d redmine redmine.dump

# 純 SQL format（-Fp）
psql -U redmine \
  redmine < redmine_backup.sql

SQLite 還原

# 直接複製回原位
cp backup/redmine.db \
   db/redmine.sqlite3

# 或從 dump 還原
sqlite3 db/redmine.sqlite3 \
  < backup.sql
完整還原流程
1

停止 Redmine 服務

確保還原期間無使用者存取

2

還原資料庫

使用上方對應資料庫類型的還原指令

3

還原附件

tar xzf files_backup.tar.gz -C /path/to/redmine/
chown -R redmine:redmine /path/to/redmine/files
4

還原設定檔（如需要）

將 database.yml、configuration.yml、secret_token.rb 放回原位

5

清除快取並重啟

RAILS_ENV=production bundle exec rake tmp:cache:clear
sudo systemctl restart apache2
