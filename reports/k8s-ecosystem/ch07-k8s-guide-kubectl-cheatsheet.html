CHAPTER 07 · kubectl 速查手冊

kubectl：完整指令地圖

前六章建立了概念，本章純工具。一份可搜尋、可快速檢索的 kubectl 指令百科—— 從日常查詢、除錯、到進階技巧如 JSONPath 與外掛生態，按主題分類組織。 遇到不記得的指令，頂部搜尋框輸入關鍵字就能過濾。

基礎與結構

kubectl 指令通用格式

指令結構 
kubectl [command] [TYPE] [NAME] [flags]

  command  →  動作  (get, apply, delete, logs, exec, scale...)
  TYPE     →  資源  (pod, deploy, svc, ingress, node, cm, secret...)
  NAME     →  名稱  (web-7d9f8c-abc, 可省略代表全部)
  flags    →  選項  (-n default, -o yaml, --watch...)

# 範例
kubectl get pods web -n production -o yaml

Bash · 看所有可用資源 
# 列所有資源型別與縮寫
kubectl api-resources

# 過濾：只看有 namespace 的
kubectl api-resources --namespaced=true

# 查詢某個資源欄位說明
kubectl explain pod.spec.containers
kubectl explain deployment.spec --recursive | less

查詢資源

用途指令
列當前 ns 所有 Podkubectl get pods
列所有 ns 所有 Podkubectl get pods -A 或 --all-namespaces
加上 Pod IP 與 Nodekubectl get pods -o wide
持續 watch 變化kubectl get pods -w
依 label 過濾kubectl get pods -l app=web,env=prod
依欄位過濾kubectl get pods --field-selector status.phase=Running
排序kubectl get pods --sort-by=.metadata.creationTimestamp
只顯示名稱kubectl get pods -o name
輸出 YAML / JSONkubectl get pod web -o yaml / -o json
顯示 label 欄位kubectl get pods --show-labels
自訂欄位kubectl get pods -o custom-columns=NAME:.metadata.name,NODE:.spec.nodeName
詳細資訊（含 Events）kubectl describe pod <name>
多類資源一次看kubectl get all -n production（含 pod/svc/deploy/rs）
看所有 nskubectl get ns
看 contextkubectl config current-context
Cluster 資訊kubectl cluster-info
版本kubectl version --short
💡 -o wide 的價值
-o wide 多顯示 IP、Node、Nominated Node 等欄位。除錯時不確定 Pod 跑在哪台 Node？這個 flag 是最快答案。

建立與更新

apply vs create：宣告式 vs 命令式

kubectl apply -f

宣告式。資源不存在就建、已存在則更新差異欄位。會在 annotation 記錄上次設定供 diff。生產首選。

kubectl create

命令式。已存在會報錯。適合快速建立、產生 YAML 骨架（--dry-run）。

用途指令
套用單一檔案kubectl apply -f deploy.yaml
套用整個目錄kubectl apply -f ./manifests/
遞迴套用子目錄kubectl apply -f ./manifests/ -R
從網址套用kubectl apply -f https://example.com/deploy.yaml
從 stdin 套用cat deploy.yaml | kubectl apply -f -
dry-run（不實際套用）kubectl apply -f x.yaml --dry-run=client
Server 端驗證kubectl apply -f x.yaml --dry-run=server
看差異kubectl diff -f x.yaml
快速建 deploymentkubectl create deployment web --image=nginx:1.27 --replicas=3
快速建 servicekubectl expose deployment web --port=80 --type=ClusterIP
快速建 configmapkubectl create cm app-cfg --from-literal=KEY=value
快速建 secretkubectl create secret generic db --from-literal=password=xxx
建 namespacekubectl create ns staging
跑一次性 Pod（互動）kubectl run tmp --image=busybox --rm -it --restart=Never -- sh
手動觸發 CronJob 一次kubectl create job --from=cronjob/backup manual-run

產生 YAML 骨架（超實用）

Bash · --dry-run 生 YAML 
# 記不得 Deployment YAML 長怎樣？用這個！
kubectl create deployment web --image=nginx:1.27 --replicas=3 \
  --dry-run=client -o yaml > deploy.yaml

# Service 骨架
kubectl create service clusterip web --tcp=80:8080 \
  --dry-run=client -o yaml

# 產 Pod 骨架並加上環境變數
kubectl run app --image=myapp --env="LOG=info" --port=8080 \
  --dry-run=client -o yaml

編輯與 Patch

用途指令
互動編輯（開 $EDITOR）kubectl edit deployment web
更新映像版本kubectl set image deployment/web web=nginx:1.28
設定環境變數kubectl set env deployment/web LOG_LEVEL=debug
從 ConfigMap 注入 envkubectl set env deployment/web --from=configmap/app-cfg
設 resourceskubectl set resources deployment/web --limits=cpu=500m,memory=512Mi
加 labelkubectl label pod web tier=frontend
覆寫已有 labelkubectl label pod web tier=backend --overwrite
移除 labelkubectl label pod web tier-
加 annotationkubectl annotate deploy web description="v2"
Strategic Merge Patchkubectl patch deploy web -p '{"spec":{"replicas":5}}'
JSON Patchkubectl patch deploy web --type=json -p='[{"op":"replace","path":"/spec/replicas","value":5}]'

三種 patch 策略

type特點適合
strategic（預設）K8s 專用，陣列可依欄位智慧合併90% 情境
mergeRFC 7396 JSON Merge Patch覆寫整塊物件
jsonRFC 6902 JSON Patch，op-based精準控制 add/remove/replace

刪除

用途指令
刪除特定資源kubectl delete pod web
刪 YAML 檔所有資源kubectl delete -f deploy.yaml
依 label 批次刪kubectl delete pods -l app=web
刪整個 namespace（含內容物）kubectl delete ns dev
強制刪除（Pod 卡住時）kubectl delete pod web --grace-period=0 --force
批次刪 namespace 內所有 deploykubectl delete deploy --all -n dev
依 field-selector 刪kubectl delete pods --field-selector=status.phase=Failed
⚠️ 強制刪除的代價
--force --grace-period=0 跳過 Node 的確認直接從 etcd 移除 Pod 紀錄。若容器實際還在 Node 上跑就變成「殭屍」——尤其對 StatefulSet 有 split-brain 風險。先確認 Node 真的失聯或 Pod 真卡死再用。

日誌與容器執行

用途指令
看 Pod 日誌kubectl logs web
持續追蹤（tail -f）kubectl logs -f web
上次重啟前日誌kubectl logs web --previous
多容器指定kubectl logs web -c nginx
全部容器kubectl logs web --all-containers=true
最近 N 行kubectl logs web --tail=100
最近時段kubectl logs web --since=10m
帶時間戳kubectl logs web --timestamps
Deployment 所有 Pod 日誌kubectl logs -l app=web --tail=50
進入容器 shellkubectl exec -it web -- sh
多容器指定kubectl exec -it web -c nginx -- sh
執行單一指令kubectl exec web -- env
複製檔案到容器kubectl cp ./local.txt web:/tmp/
從容器複製出來kubectl cp web:/var/log/app.log ./app.log
本機 port-forwardkubectl port-forward pod/web 8080:80
port-forward 到 Servicekubectl port-forward svc/web 8080:80
Pod 資源使用率kubectl top pods -A --sort-by=cpu
Node 資源使用率kubectl top nodes

Rollout 與擴縮

用途指令
看更新進度kubectl rollout status deployment/web
看歷史版本kubectl rollout history deployment/web
看某版本詳情kubectl rollout history deployment/web --revision=3
回滾上個版本kubectl rollout undo deployment/web
回滾指定版本kubectl rollout undo deployment/web --to-revision=2
暫停滾動kubectl rollout pause deployment/web
恢復滾動kubectl rollout resume deployment/web
重啟所有 Podkubectl rollout restart deployment/web
手動擴縮kubectl scale deploy/web --replicas=5
條件式擴縮kubectl scale deploy/web --replicas=5 --current-replicas=3
建 HPAkubectl autoscale deploy web --min=2 --max=10 --cpu-percent=70
💡 rollout restart 的神奇用法
沒改 image 或 config 卻想重啟全部 Pod？kubectl rollout restart 會在 Pod 模板加 annotation 觸發滾動重啟——零停機。比 kubectl delete pods 安全太多。

除錯與網路

用途指令
看最近 eventskubectl get events --sort-by='.lastTimestamp'
只看 Warningkubectl get events --field-selector type=Warning -A
持續 watch eventskubectl get events -w
注入 debug 容器kubectl debug -it web --image=nicolaka/netshoot --target=app
複製 Pod 並換映像kubectl debug web --copy-to=debug --image=busybox -it
對 Node 開 debugkubectl debug node/worker-1 -it --image=ubuntu
跑臨時 DNS 測試 Podkubectl run dns -it --rm --image=busybox --restart=Never -- nslookup web

Node 管理

用途指令
列 Nodekubectl get nodes
Node 詳細（含 taint、allocatable）kubectl describe node worker-1
Cordon（禁排新 Pod）kubectl cordon worker-1
恢復 Cordonkubectl uncordon worker-1
Drain（排空 Node，升級前用）kubectl drain worker-1 --ignore-daemonsets --delete-emptydir-data
加 labelkubectl label nodes worker-1 disktype=ssd
加 Taintkubectl taint nodes worker-1 dedicated=gpu:NoSchedule
移除 Taintkubectl taint nodes worker-1 dedicated=gpu:NoSchedule-
看 Node 上所有 Podkubectl get pods -A --field-selector spec.nodeName=worker-1

權限檢查

JSONPath / go-template：精準取值

-o yaml 看全部很方便，但要在 script 裡取特定欄位就需要 JSONPath 或 go-template。這是 DevOps 自動化必備技能。

JSONPath 基本語法

用途指令
取單一欄位kubectl get pod web -o jsonpath='{.status.podIP}'
所有 Node 名稱kubectl get nodes -o jsonpath='{.items[*].metadata.name}'
每個 Pod + 換行kubectl get pods -o jsonpath='{range .items[*]}{.metadata.name}{"\n"}{end}'
名稱+Node 對照kubectl get pods -o jsonpath='{range .items[*]}{.metadata.name}{"\t"}{.spec.nodeName}{"\n"}{end}'
所有 container 映像kubectl get pods -o jsonpath='{.items[*].spec.containers[*].image}'
讀 Secret 並 base64 -dkubectl get secret db -o jsonpath='{.data.password}' | base64 -d
取 LoadBalancer 外部 IPkubectl get svc my-lb -o jsonpath='{.status.loadBalancer.ingress[0].ip}'

go-template：更彈性但語法較複雜

Bash · go-template 範例 
# 條件式輸出：只印 Running 的 Pod
kubectl get pods -o go-template='{{range .items}}{{if eq .status.phase "Running"}}{{.metadata.name}}{{"\n"}}{{end}}{{end}}'

# 用 template 檔
kubectl get pods -o go-template-file=template.tmpl

自訂欄位（最人性化）

Bash · custom-columns 
kubectl get pods -o custom-columns=\
NAME:.metadata.name,\
NODE:.spec.nodeName,\
STATUS:.status.phase,\
IP:.status.podIP

# 一次看所有 Pod 的映像版本
kubectl get pods -o custom-columns='NAME:.metadata.name,IMAGE:.spec.containers[*].image' -A

Context 與 Kubeconfig

~/.kube/config 可同時存多個 Cluster 的憑證。一個 Context = Cluster + User + Namespace 的組合。

用途指令
看全部 contextkubectl config get-contexts
看當前 contextkubectl config current-context
切換 contextkubectl config use-context prod-east
切換預設 namespacekubectl config set-context --current --namespace=production
看當前完整設定kubectl config view --minify
從 URL 匯入 kubeconfigexport KUBECONFIG=~/.kube/config:~/.kube/prod-config
用特定 config 檔kubectl --kubeconfig=./prod.yaml get pods

kubeconfig 結構

~/.kube/config 骨架 
apiVersion: v1
kind: Config
current-context: prod-east
clusters:
- name: prod-east
  cluster:
    server: https://prod.k8s.example.com
    certificate-authority-data: LS0tLS1CRUdJTi...
users:
- name: alice
  user:
    token: eyJhbGciOi...
contexts:
- name: prod-east
  context:
    cluster: prod-east
    user: alice
    namespace: production

Shell 別名建議

每天敲 kubectl 敲到手痛？這套別名讓你生產力翻倍：

~/.bashrc / ~/.zshrc 
# 基本別名
alias k='kubectl'
alias kg='kubectl get'
alias kd='kubectl describe'
alias kl='kubectl logs'
alias ke='kubectl exec -it'
alias ka='kubectl apply -f'
alias kdel='kubectl delete'

# 常用組合
alias kgp='kubectl get pods'
alias kgs='kubectl get svc'
alias kgd='kubectl get deploy'
alias kgi='kubectl get ingress'
alias kgn='kubectl get nodes'
alias kga='kubectl get all'
alias kgpw='kubectl get pods -o wide'
alias kgpa='kubectl get pods -A'

# 日誌
alias klf='kubectl logs -f'
alias klp='kubectl logs --previous'

# Context / Namespace 切換（配合 kubectx / kubens）
alias kx='kubectx'

# 啟用 kubectl 自動補完（配合 k 別名）
source <(kubectl completion bash)
complete -F __start_kubectl k

kubectx / kubens：切換神器

Bash · 安裝與使用 
# macOS
brew install kubectx

# 切換 context（互動選單）
kubectx

# 切換到上一個 context（像 cd -）
kubectx -

# 切換 namespace
kubens production
kubens -    # 回上個 namespace

kubectl 外掛生態

透過 Krew 可安裝 200+ 社群外掛，擴充 kubectl 能力。

Bash · 安裝 Krew 
# https://krew.sigs.k8s.io/docs/user-guide/setup/install/
(set -x; cd "$(mktemp -d)" &&
  curl -fsSLO "https://github.com/kubernetes-sigs/krew/releases/latest/download/krew-linux_amd64.tar.gz" &&
  tar zxvf krew-linux_amd64.tar.gz &&
  ./krew-linux_amd64 install krew)

# 加到 PATH
export PATH="${KREW_ROOT:-$HOME/.krew}/bin:$PATH"

# 搜尋與安裝
kubectl krew search logs
kubectl krew install tree

必裝十大外掛

外掛功能
ctx / nskubectx / kubens 的 plugin 版
tree樹狀顯示資源擁有關係（Deploy→RS→Pod）
neat把 get -o yaml 清掉 K8s 自動加的欄位，留純粹你寫的
stern多 Pod 日誌聚合追蹤，顯示 Pod 名稱 + 顏色
who-can誰有權限對某資源做某動作？RBAC 反查神器
access-matrix列出所有 Subject 對某資源的權限矩陣
outdated列出有新版可更新的映像
resource-capacity顯示 Node 資源使用狀況漂亮表格
deprecations掃描 YAML 中已棄用的 API 版本
view-secret直接解碼 secret 內容（免再手 base64 -d）

stern 用法示範

Bash · stern 多 Pod 日誌 
# 追蹤所有 app=web 的 Pod 日誌（每個 Pod 不同顏色）
kubectl stern -l app=web

# 過濾關鍵字
kubectl stern web --include "error|exception"

# 只看最近 5 分鐘
kubectl stern web --since 5m

進階技巧

1. 用 YAML 測試表達式

Bash · explain + grep 
# 完整欄位樹狀輸出 + 搜尋
kubectl explain pod --recursive | grep -A 1 "securityContext"

2. 批次跨 namespace 操作

Bash · 找所有失敗 Pod 
# 所有 ns 中 status != Running 的 Pod
kubectl get pods -A --field-selector=status.phase!=Running

# 一次刪所有 Failed Pod
kubectl delete pods -A --field-selector=status.phase=Failed

3. 即時觀察資源使用

Bash · watch 組合 
# 每 2 秒更新
watch kubectl top nodes

# 或用 kubectl 內建 -w（僅限部分命令）
kubectl get pods -w

4. 從 Pod 複製整個目錄出來

Bash · kubectl cp 
kubectl cp production/web-abc123:/app/logs ./local-logs

# 注意：容器必須有 tar 指令！distroless 映像會失敗

5. 把 YAML 改乾淨（neat）

Bash · 取得 clean YAML 
# 安裝 neat 外掛後
kubectl get deploy web -o yaml | kubectl neat > web-clean.yaml

# 或用 --export（已棄用，僅供參考）

6. Diff 檔案與 Cluster 的差異

Bash · kubectl diff 
# 看 YAML 檔與 cluster 當前狀態的差異（不實際套用）
kubectl diff -f deploy.yaml

# 可設定 diff 工具
export KUBECTL_EXTERNAL_DIFF="diff -u"
