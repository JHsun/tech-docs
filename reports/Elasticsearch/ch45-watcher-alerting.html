ES 的兩種告警系統

項目Watcher（API-first）Kibana Alerting（UI-first）
定義JSON / DSL，存 clusterKibana UI 建立 Rule
授權Gold+Basic 即可
學習曲線陡（5 大區塊）平（界面化）
整合限定 actions豐富 connectors（Slack、PagerDuty、Jira...）
適合進階自訂、複雜邏輯大多通用場景、團隊操作
未來方向維護中主力推進
💡 推薦：以 Kibana Alerting 為主 Elastic 目前主推 Kibana Alerting，新功能（如 ES|QL alerting、AI Assistant）都優先在這裡。Watcher 仍維護但不再強化。

Watcher 完整範例

PUT _watcher/watch/error-rate-alert
{
  "trigger": {
    "schedule": { "interval": "5m" }
  },

  "input": {
    "search": {
      "request": {
        "indices": [ "logs-app-*" ],
        "body": {
          "size": 0,
          "query": {
            "bool": {
              "filter": [
                { "term":  { "log.level": "error" } },
                { "range": { "@timestamp": { "gte": "now-5m" } } }
              ]
            }
          },
          "aggs": {
            "by_service": {
              "terms": { "field": "service.name", "size": 10 }
            }
          }
        }
      }
    }
  },

  "condition": {
    "compare": {
      "ctx.payload.hits.total.value": { "gt": 100 }
    }
  },

  "transform": {
    "script": "
      def services = ctx.payload.aggregations.by_service.buckets;
      def text = '';
      for (s in services) {
        text += s.key + ': ' + s.doc_count + ' errors\n';
      }
      return [ 'summary': text, 'total': ctx.payload.hits.total.value ];
    "
  },

  "actions": {
    "notify_slack": {
      "webhook": {
        "method": "POST",
        "url":    "https://hooks.slack.com/services/...",
        "headers": { "Content-Type": "application/json" },
        "body":   "{\"text\":\"🚨 Error spike: {{ctx.payload.total}} errors in last 5min\\n{{ctx.payload.summary}}\"}"
      }
    },
    "log_to_index": {
      "index": {
        "index": "alert-history",
        "doc_type": "_doc"
      }
    },
    "email_oncall": {
      "throttle_period": "30m",
      "email": {
        "to":      [ "ops-team@example.com" ],
        "subject": "[ALERT] High error rate ({{ctx.payload.total}})",
        "body":    {
          "html": "
Error spike detected
{{ctx.payload.summary}}
"
        }
      }
    }
  }
}

Trigger 類型

Schedule（最常用）

// interval：固定間隔
"trigger": { "schedule": { "interval": "10s" } }

// cron 表達式
"trigger": { "schedule": { "cron": "0 0/5 * * * ?" } }

// daily：每天固定時間
"trigger": { "schedule": { "daily": { "at": [ "08:00", "20:00" ] } } }

// weekly：每週特定日
"trigger": { "schedule": {
  "weekly": { "on": "monday", "at": "08:00" }
}}

// hourly
"trigger": { "schedule": { "hourly": { "minute": [ 0, 30 ] } } }

Input 類型

simple：固定值（測試用）

"input": { "simple": { "name": "Alice", "score": 100 } }

search：ES 查詢（最常用）

"input": {
  "search": {
    "request": {
      "indices": [ "logs-*" ],
      "body": { ... query DSL ... }
    },
    "extract": [ "hits.total.value", "aggregations.by_service.buckets" ]
  }
}

http：呼叫外部 API

"input": {
  "http": {
    "request": {
      "host":   "api.example.com",
      "port":   443,
      "scheme": "https",
      "method": "GET",
      "path":   "/api/health",
      "headers": { "X-API-Key": "..." }
    }
  }
}

chain：多 input 組合

"input": {
  "chain": {
    "inputs": [
      { "first":  { "search": { ... } } },
      { "second": { "search": { ... } } }
    ]
  }
}

Condition 類型

// always 
"condition": { "always": {} }
// never
"condition": { "never": {} }

// compare：簡單比較
"condition": {
  "compare": { "ctx.payload.hits.total.value": { "gt": 100 } }
}

// array_compare
"condition": {
  "array_compare": {
    "ctx.payload.aggregations.by_service.buckets": {
      "path": "doc_count",
      "gte": 50
    }
  }
}

// script（最彈性）
"condition": {
  "script": "
    def buckets = ctx.payload.aggregations.by_service.buckets;
    return buckets.any(b -> b.doc_count > 100);
  "
}

Email account 必須先在 elasticsearch.yml 設定：

xpack.notification.email.account:
  smtp1:
    profile: standard
    smtp:
      host: smtp.example.com
      port: 587
      auth: true
      starttls:
        enable: true
      user: alerts@example.com

密碼存 keystore：

elasticsearch-keystore add xpack.notification.email.account.smtp1.smtp.secure_password

範例：建立 Index threshold rule

  1. Stack Management → Rules → Create Rule
  2. Type: Index threshold
  3. Index: logs-*
  4. Time field: @timestamp
  5. WHEN: count() matches a query
  6. OF: log.level:error
  7. OVER: 5 minutes
  8. IS ABOVE: 100
  9. For each interval: 1 minute
  10. Actions: 選 connector + 訊息範本

ES|QL Alerting（9.x 新）

用 ES|QL 寫 alert 條件，比 index threshold 更靈活：

FROM logs-*
| WHERE @timestamp > NOW() - 5 MINUTES
| WHERE log.level == "error"
| STATS error_count = COUNT(*) BY service.name
| WHERE error_count > 100

有結果即觸發 alert，bucket key 變成 alert context。

Alert History

Watcher 執行紀錄存於 .watcher-history-* data stream。Kibana Alerting 在 .alerts-*。可查詢分析告警頻率：

# Watcher 歷史
curl -u elastic:xxx --cacert ca.crt \
  "https://localhost:9200/.watcher-history-*/_search?pretty&size=10&sort=trigger_event.triggered_time:desc"

最佳實踐

  • 新建告警一律用 Kibana Alerting，Watcher 只維護現有
  • 連接 PagerDuty / OpsGenie 走值班輪替，避免一個信箱地獄
  • Throttle 必設：避免相同告警每分鐘 spam
  • 分嚴重度 channel：critical → PagerDuty；warning → Slack
  • 告警內容含足夠 context：service、host、time range、Kibana 連結
  • 把 alert event 寫進 ES（index action / connector）做後續分析
  • 每月檢視告警雜訊：哪些 alert 從未需要 action？降閾值或刪除
  • 用 SLO + burn rate 取代靜態閾值（適合可用性監控）
  • 結合 ML AD：anomaly score > 75 觸發告警，不依賴硬閾值

本章重點回顧

  • ES 兩個告警系統：Watcher（API-first，Gold+）、Kibana Alerting（UI-first，Basic）
  • 新建用 Kibana Alerting，Watcher 仍維護但不再強化
  • Watcher 五區塊：trigger、input、condition、transform、actions
  • Trigger 主用 schedule（interval、cron、daily、weekly）
  • Input 主用 search（ES 查詢），也可用 http、chain
  • Condition 用 compare 簡單、script 彈性
  • Actions 內建 email / webhook / slack / pagerduty / index
  • Kibana Alerting 提供 15+ 種 rule type，含 ES|QL alerting（9.x 新）
  • 連接 connector 種類遠多於 Watcher（Jira、ServiceNow、Teams 等）
  • Throttle、嚴重度分流、SLO 是降低告警雜訊的關鍵
