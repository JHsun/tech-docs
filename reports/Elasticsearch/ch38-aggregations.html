圖 38-1：三大 aggregation 類型

基本語法

{
  "size": 0,                          // 不要 hits，只看 aggs
  "query": {
    "range": { "@timestamp": { "gte": "now-7d" } }
  },
  "aggs": {                            // 與 aggregations 等同
    "by_category": {
      "terms": { "field": "category", "size": 10 },
      "aggs": {                        // 巢狀（sub-aggs）
        "avg_price":   { "avg":   { "field": "price" } },
        "total_sales": { "sum":   { "field": "quantity" } }
      }
    }
  }
}

Bucket Aggregations

terms：依欄位值分組

{
  "aggs": {
    "by_brand": {
      "terms": {
        "field":      "brand",
        "size":       20,
        "order":      { "_count": "desc" },
        "min_doc_count": 1,
        "missing":    "(unknown)"
      }
    }
  }
}

輸出：

{
  "by_brand": {
    "doc_count_error_upper_bound": 0,
    "sum_other_doc_count": 142,
    "buckets": [
      { "key": "Apple",   "doc_count": 523 },
      { "key": "Samsung", "doc_count": 412 },
      ...
    ]
  }
}

terms 精度問題

⚠️ terms 不是 100% 準確 分散式環境下，每 shard 各回 top-N 後合併。可能漏掉「在每個 shard 都不在 top-N、但加總後該排第 1 的詞」。
  • doc_count_error_upper_bound：每個 bucket 最多誤差
  • sum_other_doc_count：被排除的其他 bucket 加總
  • 解法：shard_size: 50（每 shard 取多一點）；或用 composite 完整迭代

date_histogram：時間切片

{
  "aggs": {
    "by_day": {
      "date_histogram": {
        "field":          "@timestamp",
        "calendar_interval": "1d",         // 或 fixed_interval: "1h"
        "time_zone":      "+08:00",
        "format":         "yyyy-MM-dd",
        "min_doc_count":  0,                // 補空格子
        "extended_bounds": {                // 補上時間範圍外的空格
          "min": "2026-01-01",
          "max": "2026-04-15"
        }
      }
    }
  }
}

calendar_interval vs fixed_interval：

  • calendar_interval：1m、1h、1d、1w、1M、1q、1y（日曆對齊）
  • fixed_interval：固定毫秒長度（30s、5m、12h）

histogram：數值切片

{
  "aggs": {
    "price_distribution": {
      "histogram": {
        "field":    "price",
        "interval": 50,
        "extended_bounds": { "min": 0, "max": 1000 }
      }
    }
  }
}

range：自訂範圍

{
  "aggs": {
    "price_range": {
      "range": {
        "field": "price",
        "ranges": [
          { "to":   50,                      "key": "cheap" },
          { "from": 50,  "to": 200,          "key": "mid"   },
          { "from": 200,                     "key": "high"  }
        ]
      }
    }
  }
}

filters：條件分組

{
  "aggs": {
    "log_levels": {
      "filters": {
        "filters": {
          "errors":   { "match": { "log.level": "error" } },
          "warnings": { "match": { "log.level": "warn" } },
          "info":     { "match": { "log.level": "info" } }
        }
      }
    }
  }
}

nested aggregation

處理 nested 型別欄位：

{
  "aggs": {
    "comments": {
      "nested": { "path": "comments" },
      "aggs": {
        "by_author": { "terms": { "field": "comments.author" } }
      }
    }
  }
}

composite：完整迭代分頁

用於需要看「所有 bucket」（例如總共 100 萬個不同 user）：

{
  "size": 0,
  "aggs": {
    "all_users": {
      "composite": {
        "size": 1000,
        "sources": [
          { "user": { "terms": { "field": "user_id" } } },
          { "day":  { "date_histogram": { "field": "@timestamp", "calendar_interval": "1d" } } }
        ]
      }
    }
  }
}

第一頁有 after_key，下次 query 帶 after：

{
  "aggs": {
    "all_users": {
      "composite": {
        "size":   1000,
        "after":  { "user": "user_999", "day": 1735689600000 },
        "sources": [ ... ]
      }
    }
  }
}

Metric Aggregations

基本

{
  "aggs": {
    "avg_price":     { "avg":   { "field": "price" } },
    "max_price":     { "max":   { "field": "price" } },
    "min_price":     { "min":   { "field": "price" } },
    "sum_quantity":  { "sum":   { "field": "quantity" } },
    "count_orders":  { "value_count": { "field": "order_id" } },
    "stats":         { "stats": { "field": "price" } },     // avg+min+max+sum+count 一次
    "ext_stats":     { "extended_stats": { "field": "price" } }   // + variance, std_deviation
  }
}

cardinality：去重計數

{
  "aggs": {
    "unique_users": {
      "cardinality": {
        "field":              "user_id",
        "precision_threshold": 3000        // 100-40000，越大越準也越耗記憶體
      }
    }
  }
}

注意：cardinality 是近似值（HyperLogLog++）。precision_threshold 預設 3000，誤差約 1%。

percentiles：百分位

{
  "aggs": {
    "response_time_pcts": {
      "percentiles": {
        "field":    "response_time_ms",
        "percents": [ 50, 75, 90, 95, 99, 99.9 ]
      }
    }
  }
}

輸出：

{
  "response_time_pcts": {
    "values": {
      "50.0":   123.4,
      "75.0":   189.2,
      "95.0":   456.8,
      "99.0":   1234.5,
      "99.9":   3456.7
    }
  }
}

percentile_ranks：給定值在哪個百分位

{
  "aggs": {
    "rt_in_percentile": {
      "percentile_ranks": {
        "field":  "response_time_ms",
        "values": [ 100, 500, 1000 ]
      }
    }
  }
}
// 輸出：500ms 落在 89.2% 位置

top_hits：每組取代表文件

{
  "aggs": {
    "by_brand": {
      "terms": { "field": "brand", "size": 5 },
      "aggs": {
        "top_3_products": {
          "top_hits": {
            "size":     3,
            "sort":     [ { "popularity": "desc" } ],
            "_source":  [ "name", "price" ]
          }
        }
      }
    }
  }
}

類似 SQL 的「每個 group 取 top N」。

9.x 新功能：rate 與 boxplot

// rate：每單位時間的計數/sum（在 date_histogram 中用）
{
  "aggs": {
    "by_day": {
      "date_histogram": { "field": "@timestamp", "calendar_interval": "1d" },
      "aggs": {
        "events_per_minute": {
          "rate": { "unit": "minute" }
        }
      }
    }
  }
}

// boxplot：四分位數箱型圖
{
  "aggs": {
    "rt_box": { "boxplot": { "field": "response_time_ms" } }
  }
}

Pipeline Aggregations

Pipeline agg 對其他 agg 的結果再做計算。語法用 buckets_path 指向來源：

moving_fn：移動視窗

{
  "aggs": {
    "by_day": {
      "date_histogram": { "field": "@timestamp", "calendar_interval": "1d" },
      "aggs": {
        "daily_sales":   { "sum": { "field": "amount" } },
        "moving_avg_7d": {
          "moving_fn": {
            "buckets_path": "daily_sales",
            "window":       7,
            "script":       "MovingFunctions.unweightedAvg(values)"
          }
        }
      }
    }
  }
}

內建函式：min、max、sum、unweightedAvg、linearWeightedAvg、ewma、holt、holtWinters、stdDev。

cumulative_sum：累計

{
  "aggs": {
    "by_day": {
      "date_histogram": { "field": "@timestamp", "calendar_interval": "1d" },
      "aggs": {
        "daily_revenue":   { "sum": { "field": "amount" } },
        "cumulative_revenue": {
          "cumulative_sum": { "buckets_path": "daily_revenue" }
        }
      }
    }
  }
}

bucket_script：欄位間運算

{
  "aggs": {
    "by_day": {
      "date_histogram": { "field": "@timestamp", "calendar_interval": "1d" },
      "aggs": {
        "total_sales": { "sum": { "field": "amount" } },
        "total_cost":  { "sum": { "field": "cost" } },
        "profit": {
          "bucket_script": {
            "buckets_path": {
              "sales": "total_sales",
              "cost":  "total_cost"
            },
            "script": "params.sales - params.cost"
          }
        }
      }
    }
  }
}

bucket_selector：過濾 bucket

{
  "aggs": {
    "by_user": {
      "terms": { "field": "user_id", "size": 1000 },
      "aggs": {
        "total_spent": { "sum": { "field": "amount" } },
        "high_value": {
          "bucket_selector": {
            "buckets_path": { "spent": "total_spent" },
            "script":       "params.spent > 1000"
          }
        }
      }
    }
  }
}

只保留 total_spent > 1000 的 bucket。

derivative：差分

{
  "aggs": {
    "by_day": {
      "date_histogram": { "field": "@timestamp", "calendar_interval": "1d" },
      "aggs": {
        "users":            { "cardinality": { "field": "user_id" } },
        "users_change":     { "derivative":  { "buckets_path": "users" } },
        "users_change_pct": {
          "derivative": {
            "buckets_path": "users",
            "unit":         "1d",
            "format":       "0.00%"
          }
        }
      }
    }
  }
}

Aggregation 效能調校

並行化（9.x 預設）

9.x 對大多 aggregation 啟用 per-segment 並行，相同查詢可比 7.x 快數倍。無需特別設定。

execution_hint：terms 演算法選擇

{
  "terms": {
    "field":          "user_id",
    "execution_hint": "map"           // 預設 global_ordinals
  }
}
  • global_ordinals（預設）：用 keyword 的 global ordinals 加速。第一次查 build cost 高
  • map：用 hashmap，適合 cardinality 低或 query 範圍小

精度與成本權衡

Agg調校點建議
termssize + shard_sizeshard_size = size × 1.5（更準）
cardinalityprecision_threshold3000-10000；越大越準也越耗 RAM
percentilescompression預設 100；增大更準（耗 RAM）
histogrammin_doc_count1（避免大量空 bucket）

9.x TSDS 時序專用

對 TSDS（CH 29），9.x 提供新 TS aggregation（pipe syntax in ES|QL，CH 39）：

TS metrics-system
| WHERE TRANGE(1h)
| STATS avg(rate(requests, 10m)) BY TBUCKET(1m), host

支援：rate、increase、delta、idelta，並可指定 sliding window 減少 jitter。

9.x exponential_histogram 欄位

9.x 新欄位型別，原生支援 OpenTelemetry exponential histogram，可用 percentiles、avg、min、max、sum 聚合。比一般 histogram 更精確且空間更省。

Aggregation 最佳實踐

  • aggregation 必設 size: 0，省 hits 流量
  • 對 filter context 的查詢做 aggregation，效能 + cache 更好
  • terms agg 用 keyword 欄位（不要用 text，需 fielddata）
  • cardinality 是近似，要精確去重用 composite 完整迭代
  • composite agg 適合大量 bucket 場景，可分頁
  • top_hits 取每組代表，但 sub-aggregation 中量大時要小心記憶體
  • moving_fn / derivative 處理時序趨勢
  • bucket_selector 過濾 bucket，但無法縮減 query 階段的成本
  • 9.x 自動 parallelization，aggregation 比 7.x 快數倍
  • 用 profile API 找慢的 agg 環節

本章重點回顧

  • 三大 agg 類型：bucket（切組）、metric（算）、pipeline（對結果再算）
  • terms 是最常用 bucket agg，但有精度問題（用 shard_size 緩解）
  • composite agg 適合大量 bucket 完整迭代
  • cardinality / percentiles 都是近似值，可調精度
  • top_hits 取每組代表，類似 SQL 的「每組 top N」
  • Pipeline agg 用 buckets_path 指向其他 agg 結果
  • moving_fn / cumulative_sum / derivative / bucket_script / bucket_selector 是時序分析常用
  • 9.x parallelization 大幅提升 aggregation 效能
  • 9.x exponential_histogram 欄位原生支援 OTel
