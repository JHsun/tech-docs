圖 27-1：Analyzer 三層管線

Analyzer 由三部分組成：

  1. Character Filters（0..n 個）：先處理原始字元（移 HTML、字元替換等）
  2. Tokenizer（必須 1 個）：把字串切成 tokens
  3. Token Filters（0..n 個）：處理 tokens（小寫化、移停用詞、stemming、同義詞）

用 _analyze API 測試

測試 analyzer 的最快方式：

curl -X POST -u elastic:xxx --cacert ca.crt \
  -H 'Content-Type: application/json' \
  "https://localhost:9200/_analyze?pretty" -d '
{
  "analyzer": "standard",
  "text": "The Quick Brown FOXES"
}'

輸出：

{
  "tokens": [
    { "token": "the",   "start_offset": 0,  "end_offset": 3,  "type": "", "position": 0 },
    { "token": "quick", "start_offset": 4,  "end_offset": 9,  "type": "", "position": 1 },
    { "token": "brown", "start_offset": 10, "end_offset": 15, "type": "", "position": 2 },
    { "token": "foxes", "start_offset": 16, "end_offset": 21, "type": "", "position": 3 }
  ]
}

內建 Analyzer 速覽

Analyzer切詞行為適用
standard（預設）Unicode 文字邊界 + lowercase + 移除標點通用、多語言基礎
simple非字母切詞 + lowercase簡單英文
whitespace僅依空白切，不 lowercasecode、保留大小寫
stopsimple + 停用詞過濾英文簡單分析
keyword整字串當一個 token整字精確匹配（與 keyword type 不同！）
patternregex 切詞自訂分隔符
english / spanish / 等language-specific 含 stemmer + stopwords單語言全文搜尋
fingerprint切詞 + lowercase + 排序 + 去重 + concat「指紋」式去重

Character Filters

html_strip

POST _analyze
{
  "char_filter": [ "html_strip" ],
  "tokenizer":   "standard",
  "text": "
I'm so happy!
"
}
// → tokens: ["i'm", "so", "happy"]

mapping

{
  "char_filter": {
    "emoticon_normalize": {
      "type": "mapping",
      "mappings": [
        ":) => _happy_",
        ":( => _sad_"
      ]
    }
  }
}

pattern_replace

{
  "char_filter": {
    "phone_normalize": {
      "type": "pattern_replace",
      "pattern": "[\\s\\-]+",
      "replacement": ""
    }
  }
}
// "0912-345 678" → "0912345678"

Tokenizer 選擇

Tokenizer說明
standardUnicode UAX#29 文字邊界
letter非字母即切
lowercaseletter + lowercase 一起
whitespace僅空白切
uax_url_emailstandard 但保留 URL/email 完整
classic傳統英文，保留首字母縮寫、公司名稱
thai泰文（用 ICU）
ngram切成所有 n-gram（substring matching）
edge_ngram從詞首切 n-gram（autocomplete 用）
keyword整字串一個 token
patternregex 切
simple_pattern / simple_pattern_split有限 regex（更快）
path_hierarchy/a/b/c → /a, /a/b, /a/b/c

edge_ngram（autocomplete）

PUT autocomplete-test
{
  "settings": {
    "analysis": {
      "analyzer": {
        "edge_ngram_analyzer": {
          "type": "custom",
          "tokenizer": "edge_ngram_tokenizer",
          "filter": [ "lowercase" ]
        }
      },
      "tokenizer": {
        "edge_ngram_tokenizer": {
          "type": "edge_ngram",
          "min_gram": 2,
          "max_gram": 10,
          "token_chars": [ "letter", "digit" ]
        }
      }
    }
  },
  "mappings": {
    "properties": {
      "title": {
        "type": "text",
        "analyzer": "edge_ngram_analyzer",
        "search_analyzer": "standard"   // 重要：search 用 standard，不再 ngram
      }
    }
  }
}

「Elasticsearch」→ el, ela, elas, elast, elasti, ... 索引；查詢 elas 即可命中。

path_hierarchy

POST _analyze
{
  "tokenizer": "path_hierarchy",
  "text": "/usr/share/elasticsearch/data"
}
// → ["/usr", "/usr/share", "/usr/share/elasticsearch", "/usr/share/elasticsearch/data"]

用於檔案路徑、URL 樹狀查詢。

常用 Token Filter

Filter說明
lowercase / uppercase大小寫轉換
stop移停用詞（the, a, an...）
stemmer (snowball)取詞幹（running → run）
kstem輕量英文 stemmer，較不激進
synonym / synonym_graph同義詞展開
asciifolding變音符 → ASCII（café → cafe）
ngram / edge_ngramtoken 內再切 n-gram
length過濾過長/過短的 token
trim移前後空白
shingle把多個相鄰 token 合成 phrase
word_delimiter_graph智能切詞（PowerShot → power, shot）
pattern_captureregex 抓取多個 token
cjk_bigram中日韓字元雙字組合
icu_* (plugin)強化的多語言處理

同義詞範例

PUT my-shop
{
  "settings": {
    "analysis": {
      "analyzer": {
        "shop_analyzer": {
          "tokenizer": "standard",
          "filter": [ "lowercase", "shop_synonyms" ]
        }
      },
      "filter": {
        "shop_synonyms": {
          "type": "synonym_graph",
          "synonyms": [
            "tv, television, telly",
            "i-pod, ipod, mp3",
            "laptop, notebook"
          ]
        }
      }
    }
  }
}

實務上 synonyms 常存在檔案 $ES_PATH_CONF/analysis/synonyms.txt，再用 synonyms_path 指定。

9.x 用 Synonyms API

9.x 推薦用 Synonyms API（cluster 層級）取代每 index 一份檔案：

# 建立 synonyms set
curl -X PUT -u elastic:xxx --cacert ca.crt \
  -H 'Content-Type: application/json' \
  "https://localhost:9200/_synonyms/shop-synonyms" -d '
{
  "synonyms_set": [
    { "id": "tv-synonyms",     "synonyms": "tv, television, telly" },
    { "id": "laptop-synonyms", "synonyms": "laptop, notebook" }
  ]
}'

# 在 mapping 中參照
PUT my-shop2
{
  "settings": {
    "analysis": {
      "filter": {
        "shop_syn": {
          "type": "synonym_graph",
          "synonyms_set": "shop-synonyms",
          "updateable": true   # 可即時更新
        }
      }
    }
  }
}

建立 Custom Analyzer

PUT my-blog
{
  "settings": {
    "analysis": {
      "analyzer": {
        "blog_analyzer": {
          "type": "custom",
          "char_filter": [ "html_strip" ],
          "tokenizer":   "standard",
          "filter": [
            "lowercase",
            "english_stop",
            "english_stemmer",
            "asciifolding"
          ]
        }
      },
      "filter": {
        "english_stop": {
          "type": "stop",
          "stopwords": "_english_"
        },
        "english_stemmer": {
          "type": "stemmer",
          "language": "english"
        }
      }
    }
  },
  "mappings": {
    "properties": {
      "body": {
        "type": "text",
        "analyzer": "blog_analyzer"
      }
    }
  }
}

測試：

curl -X POST -u elastic:xxx --cacert ca.crt \
  -H 'Content-Type: application/json' \
  "https://localhost:9200/my-blog/_analyze?pretty" -d '
{
  "analyzer": "blog_analyzer",
  "text": "
The runners are running fast through the cafés.
"
}'
// → ["runner", "run", "fast", "cafe"]

Index-time vs Search-time Analyzer

同欄位可在索引時與查詢時用不同 analyzer：

{
  "title": {
    "type": "text",
    "analyzer":        "edge_ngram_analyzer",  // 索引時切 ngram
    "search_analyzer": "standard"               // 查詢時不切 ngram
  }
}

常見場景：

  • autocomplete：索引切 edge_ngram，查詢用 standard
  • 同義詞展開：索引時不展開（節省空間），查詢時展開（更彈性）
  • 變音符處理：索引時 asciifolding，查詢時也 asciifolding

Normalizer：給 keyword 用

Keyword 不能用 analyzer，但可用 normalizer（只允許 char filter + 部分 token filter，且輸出必須是單一 token）：

PUT users
{
  "settings": {
    "analysis": {
      "normalizer": {
        "case_insensitive": {
          "type": "custom",
          "filter": [ "lowercase", "asciifolding" ]
        }
      }
    }
  },
  "mappings": {
    "properties": {
      "email": {
        "type": "keyword",
        "normalizer": "case_insensitive"
      }
    }
  }
}

POST users/_doc
{ "email": "Alice@Example.COM" }

GET users/_search
{ "query": { "term": { "email": "alice@example.com" } } }   // 命中

中文分詞

standard analyzer 對中文是每個字一個 token（不理想）：

POST _analyze
{ "analyzer": "standard", "text": "我愛 Elasticsearch 全文搜尋" }
// → ["我", "愛", "elasticsearch", "全", "文", "搜", "尋"]

內建 cjk_analyzer（雙字 bigram）

POST _analyze
{ "analyzer": "cjk", "text": "我愛全文搜尋" }
// → ["我愛", "愛全", "全文", "文搜", "搜尋"]

適合 PoC，但因為是 bigram，搜尋精度有限。

IK Analyzer（社群外掛，常見）

# 安裝（每節點都要）
sudo /usr/share/elasticsearch/bin/elasticsearch-plugin install \
  https://release.infinilabs.com/analysis-ik/stable/elasticsearch-analysis-ik-9.3.4.zip

# 重啟 ES

提供兩個 analyzer：

  • ik_max_word：盡量多切（高 recall）
  • ik_smart：智能切詞（高 precision）
POST _analyze
{ "analyzer": "ik_smart", "text": "我愛全文搜尋" }
// → ["我", "愛", "全文", "搜尋"]

Jieba（社群）

Python 風格的結巴分詞 ES 移植版。功能與 IK 類似，社群維護程度依套件而定。

用 ICU plugin（官方）

sudo /usr/share/elasticsearch/bin/elasticsearch-plugin install analysis-icu

提供 icu_tokenizer、icu_normalizer、icu_folding 等。對中文用 dictionary-based 切詞比 cjk_bigram 好。

自訂中文 analyzer

PUT zh-blog
{
  "settings": {
    "analysis": {
      "analyzer": {
        "zh_analyzer": {
          "type": "custom",
          "char_filter": [ "html_strip" ],
          "tokenizer":   "ik_max_word",
          "filter": [ "lowercase" ]
        },
        "zh_search_analyzer": {
          "type": "custom",
          "tokenizer": "ik_smart",
          "filter": [ "lowercase" ]
        }
      }
    }
  },
  "mappings": {
    "properties": {
      "body": {
        "type": "text",
        "analyzer":        "zh_analyzer",         // 索引時 ik_max_word
        "search_analyzer": "zh_search_analyzer"   // 查詢時 ik_smart
      }
    }
  }
}

繁簡轉換 + 同義詞

用 analysis-stconvert 或 analysis-pinyin 等社群外掛。在台灣場景常見搭配：

  • 使用者搜「网络」也應命中索引中的「網路」
  • 搜「youhua」（拼音）也命中「優化」

查詢分析：_termvectors 與 explain

看某文件的 token 結構

curl -u elastic:xxx --cacert ca.crt \
  "https://localhost:9200/my-index/_termvectors/1?fields=title&pretty"

看為何命中（或不命中）

curl -X GET -u elastic:xxx --cacert ca.crt \
  -H 'Content-Type: application/json' \
  "https://localhost:9200/my-index/_explain/1?pretty" -d '
{
  "query": { "match": { "title": "elasticsearch" } }
}'

最佳實踐

  • 不要過度自訂：standard analyzer 在多數場景已夠用
  • 保留 .keyword multi-field：聚合與 sort 必備
  • autocomplete 用 search-as-you-type 型別（9.x 內建）取代手刻 edge_ngram
  • 同義詞用 9.x Synonyms API，可即時更新（updateable: true）
  • 中文：選一套 analyzer 全公司一致（IK / Jieba / ICU），避免不同 index 不一致
  • search_analyzer 配合 index analyzer，autocomplete 場景必設
  • analyzer 改變需要 reindex，先在 staging 充分測試
  • 用 _analyze API 驗證：每改 analyzer 都先用 sample text 測試

本章重點回顧

  • Analyzer 三層管線：char filter → tokenizer → token filter
  • 內建 analyzer：standard（預設）、simple、whitespace、keyword、english、cjk 等
  • Custom analyzer 可組合 char filter + tokenizer + token filter
  • Index-time 與 search-time 可用不同 analyzer（autocomplete 必備）
  • Normalizer 是 keyword 專用的「迷你 analyzer」（只能輸出單 token）
  • 中文必裝額外 analyzer：IK、Jieba、ICU plugin
  • 9.x Synonyms API 取代檔案，可即時更新（updateable: true）
  • 分析 issue：用 _analyze、_termvectors、_explain 三大 API
