資料 onboarding

Elastic Security 大量倚賴 ECS（Elastic Common Schema）標準化欄位。常見來源：

  • Windows：Sysmon、Security event、PowerShell
  • Linux：auditd、syslog、SSH
  • Endpoint Defend：process、file、network、DNS、registry
  • Network：Zeek/Bro、Suricata、Packetbeat、firewall
  • Cloud：AWS CloudTrail、VPC Flow、GuardDuty、GCP Audit、Azure Activity
  • SaaS：Okta、AzureAD、Microsoft 365、GitHub、Slack
  • Identity：Active Directory、LDAP
  • Web：CDN log、WAF、proxy

每個來源都有 Fleet integration（CH 55），開箱即用。

Detection Engine

Detection rule 監控 ES 資料找威脅指標，產生 alert。

六種 Rule Types

類型說明例
Custom queryKQL/Lucene 命中即 alert找特定 process 執行
Threshold命中數超閾值10 分鐘內 SSH 失敗 > 50 次
Indicator match欄位匹配 IOC（threat intel）IP 在已知 C2 list
EQLEvent Query Language（序列關聯）process A 後 30s 內 process B
ES|QL（9.x 新）用 ES|QL 寫條件跨 source 聚合
Machine learning關聯 ML AD job 結果網路流量異常
New terms欄位出現新值新 user / 新 host / 新 country

Prebuilt Rules（500+）

Elastic 維護的官方規則集合，涵蓋：

  • Brute force（SSH、RDP、Web login）
  • Privilege escalation
  • Lateral movement
  • Persistence（registry、scheduled task、service）
  • Defense evasion（log clearing、obfuscation）
  • Credential access（mimikatz、credential dumping）
  • Exfiltration
  • Cloud-specific（AWS S3 leak、IAM 異常）

每月由 Elastic Security Lab 更新。Kibana → Security → Rules 一鍵啟用。

自訂 Rule 範例

EQL rule：偵測「PowerShell encoded command 後跟 outbound 連線」：

sequence by host.id, user.name with maxspan=2m
  [process where event.action == "start" and
    process.name : "powershell.exe" and
    process.command_line : "*-EncodedCommand*"]
  [network where event.action == "connection_attempted" and
    destination.ip != "192.168.0.0/16" and
    destination.ip != "10.0.0.0/8"]

ES|QL rule：偵測短時間多失敗登入（9.x）：

FROM logs-*
| WHERE @timestamp > NOW() - 10 MINUTES
| WHERE event.category == "authentication"
| WHERE event.outcome    == "failure"
| STATS count = COUNT(*) BY user.name, source.ip
| WHERE count > 10

Rule Actions

Rule 觸發後的 actions：

  • 建立 alert（在 Security app）
  • 送 Slack / PagerDuty / Email / Jira / ServiceNow（CH 45）
  • 建立 case
  • 呼叫 webhook
  • 9.x：用 AI Connector 自動分析 alert（OpenAI / Bedrock / Anthropic）

Alerts 介面

Security → Alerts：

  • Trend chart：時間序列 alert 數
  • Group by：rule、severity、host、user
  • Status：open / acknowledged / closed
  • Bulk action：批次標記、調查、關閉
  • 關聯到 MITRE ATT&CK 戰術 / 技術

Timeline：互動式調查

Timeline 是 SOC 分析師的核心工具：

  • 把多個 event 拖到 timeline 看時序
  • 儲存為 case 證據
  • 支援 query 過濾（KQL / EQL）
  • 9.x 整合 AI Assistant：對 timeline 摘要 + 可疑點建議

Timeline Templates

對特定 alert type 預設打開的 timeline view，自動帶入相關欄位（如 host.id、user.name、event.action）。

Cases：事件管理

Cases 把多個 alert + timeline + observation 合併成 incident：

  • 指派 owner、tags、severity
  • 整合 Jira / ServiceNow ITSM 雙向同步
  • 留言 / 附檔
  • 看 alert 與 case 的關聯時序
  • 事後做 post-mortem

Elastic Defend：Endpoint Security

Defend 是 Elastic 自家 EDR，Elastic Agent integration。需 Platinum 訂閱。

主要功能

  • Malware prevention：靜態 ML 模型 + signature
  • Ransomware protection：行為偵測 + canary file
  • Memory threat protection：偵測 in-memory 注入、shellcode
  • Behavior protection：可疑行為（如 LSASS dump）
  • Process / file / network event：完整 telemetry
  • Host isolation：一鍵把感染主機從網路隔離
  • Response actions：kill process、delete file、execute command（9.x 強化）

Modes：Detect vs Prevent

每種威脅類別可獨立設：

  • Detect：只偵測 + alert，不阻擋
  • Prevent：阻擋 + alert
  • Off：關閉

導入期常先 detect 觀察 false positive，再切 prevent。

部署

Fleet → Integrations → Elastic Defend → Add to policy → 套用到 host。

支援 OS：Windows、macOS、Linux（Linux 需 BPF）。

MITRE ATT&CK 對應

每個 prebuilt rule 都標記對應 ATT&CK：

  • Tactic（戰術）：Initial Access、Execution、Persistence、...
  • Technique（技術）：T1059 - Command and Scripting Interpreter、...
  • Sub-technique：T1059.001 - PowerShell

Kibana → Security → Coverage → MITRE ATT&CK Coverage 看你目前部署的 rules 覆蓋了哪些技術，找出 detection gap。

Threat Intelligence Integration

Fleet integrations 內建多家 TI feed：

  • AbuseCH（MalwareBazaar、URLhaus、ThreatFox）
  • AlienVault OTX
  • Anomali ThreatStream
  • MISP
  • Mandiant
  • Recorded Future
  • VirusTotal

把 IOC（IP、domain、hash、URL）拉進 ES 後，Indicator match rule 自動比對。

Threat Hunting

主動搜尋未被 alert 覆蓋的威脅：

  • Discover：用 KQL / ES|QL 查 raw event
  • Timeline：時序關聯
  • OSquery（內建在 Defend）：對 host 即時下 SQL 查狀態

Osquery 範例

-- 在 Kibana → Security → Investigations → Osquery
SELECT
  pid, name, path, cmdline, parent
FROM processes
WHERE name LIKE '%powershell%'
  AND cmdline LIKE '%-Encoded%';

結果即時返回給 Kibana。可同時對多 host 執行。

Entity Analytics（9.x）

建立 user / host 的「risk score」：

  • 聚合該 entity 上所有 alert
  • 依嚴重度與時序加權
  • 排序高風險 entity
  • Dashboard 顯示「Top risky users / hosts」

適合大型 SOC 從萬筆 alert 中找重點。

Detection Engineering Workflow

把 detection rule 視為 code：

  1. Rule 定義在 git（YAML）
  2. CI/CD 自動 deploy 到 ES
  3. 用 elastic/detection-rules 框架測試
  4. 每 rule 有 unit test（Sigma format）
  5. Code review 走 PR 流程
# 範例 rule YAML
rule:
  name: "Suspicious PowerShell EncodedCommand"
  type: query
  query: |
    process.name:"powershell.exe" and process.args:"-EncodedCommand"
  language: kuery
  severity: high
  risk_score: 73
  threat:
    - framework: "MITRE ATT&CK"
      tactic:
        id: TA0002
        name: Execution
      technique:
        - id: T1059
          name: Command and Scripting Interpreter
          subtechnique:
            - id: T1059.001
              name: PowerShell

AI Assistant for Security（9.x）

  • 對 alert 自動產生調查摘要（依 ECS 欄位）
  • 建議下一步調查方向（基於 ATT&CK）
  • 翻譯告警為自然語言（給 non-analyst 看）
  • 產生 KQL / ES|QL 查詢
  • Custom knowledge sources：附加自家 runbook、過往 incident
  • Connector：OpenAI、Anthropic、Bedrock、Azure OpenAI、Gemini

SOAR：Security Orchestration & Automation

Connector 整合：

  • EDR：CrowdStrike、SentinelOne
  • Identity：Microsoft 365 Defender、Okta
  • SOAR：Tines、Torq、Swimlane
  • Ticketing：Jira、ServiceNow
  • Communication：Slack、Teams、PagerDuty

用 alert action 自動觸發：

  • 隔離主機（Defend 或 CrowdStrike）
  • 停用 user（Okta / AzureAD）
  • Block IP（防火牆）
  • 建立 ticket
  • 送 Slack 給 oncall

Compliance Dashboards

9.x 內建合規 dashboards：

  • PCI-DSS
  • HIPAA
  • NIST 800-53
  • CIS Benchmark
  • ISO 27001

對應 ECS 欄位映射標準控制項目，看哪些已涵蓋、哪些未涵蓋。

最佳實踐

  • 從 prebuilt rules 開始，再逐步加 custom
  • Detection rules 進 GitOps（detection-rules repo 範本）
  • 用 ECS 標準化欄位，rule 可跨 source 通用
  • 勿一次啟用全部 prebuilt rules：先看 false positive 調 exception
  • MITRE ATT&CK coverage 是 detection 成熟度指標
  • Defend 從 Detect 開始，再切 Prevent
  • Threat intel feed 至少接 1-2 個（AbuseCH 免費）
  • Entity Analytics 開啟，從 alert 找出重點 entity
  • Cases + Jira 雙向同步，閉環事件處理
  • AI Assistant 用 RAG 加自家 runbook，提升 SOC 效率
  • Compliance dashboard 定期 review，找控制項缺口
  • 每季 fire drill：模擬 incident 練 SOC SOP
