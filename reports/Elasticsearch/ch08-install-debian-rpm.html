CH 08 · Debian/RPM 套件安裝

適合對象 🔴 維運者
閱讀時間 約 20 分鐘
前置知識 CH 06 · 系統需求
本章目標 透過 APT/YUM 完成生產級安裝、自動 systemd 整合、apt-mark hold 鎖版本、套件升級流程

套件安裝 vs tar.gz

對於生產環境，套件（Debian/RPM）通常是首選，因為：

  • ✅ 自動建立 elasticsearch user 與目錄結構
  • ✅ 自動安裝 systemd unit（無需手動撰寫）
  • ✅ logrotate、tmpfile 整合
  • ✅ apt/yum 統一管理升級
  • ✅ 與 RPM/DEB 的 dependency 機制相容

缺點：

  • 需要 root 權限
  • 同一台機器無法輕易跑多版本
  • 路徑與 ownership 是固定的

套件安裝後的標準目錄

路徑用途
/usr/share/elasticsearchES_HOME（jdk、bin、lib、modules、plugins）
/etc/elasticsearch所有設定檔
/var/lib/elasticsearch資料目錄（path.data）
/var/log/elasticsearch日誌目錄（path.logs）
/etc/default/elasticsearch（Debian）
/etc/sysconfig/elasticsearch（RHEL）		環境變數設定
/usr/lib/systemd/system/elasticsearch.servicesystemd unit
/usr/lib/sysctl.d/elasticsearch.confsysctl 預設設定（含 vm.max_map_count）

Debian / Ubuntu（APT）安裝

Step 1：新增 Elastic APT repository

# 安裝必要工具
sudo apt update
sudo apt install -y apt-transport-https gnupg ca-certificates wget

# 匯入 Elastic 公鑰
wget -qO - https://artifacts.elastic.co/GPG-KEY-elasticsearch | \
  sudo gpg --dearmor -o /usr/share/keyrings/elasticsearch-keyring.gpg

# 新增 9.x repo
echo "deb [signed-by=/usr/share/keyrings/elasticsearch-keyring.gpg] https://artifacts.elastic.co/packages/9.x/apt stable main" | \
  sudo tee /etc/apt/sources.list.d/elastic-9.x.list

sudo apt update

Step 2：安裝

# 安裝最新版
sudo apt install -y elasticsearch

# 或指定版本
sudo apt install -y elasticsearch=9.3.4
✅ 首次安裝會自動產生密碼與憑證 注意 apt 安裝結尾的訊息！它會輸出 elastic 使用者的初始密碼，這個密碼只顯示一次。若沒記下也沒關係，後續可用 elasticsearch-reset-password 重設。

Step 3：啟用 systemd 服務

sudo systemctl daemon-reload
sudo systemctl enable elasticsearch.service
sudo systemctl start elasticsearch.service
sudo systemctl status elasticsearch.service

Step 4：驗證

# 重設密碼
sudo /usr/share/elasticsearch/bin/elasticsearch-reset-password -u elastic

# 測試
curl -X GET "https://localhost:9200" \
  -u elastic --cacert /etc/elasticsearch/certs/http_ca.crt

RHEL / CentOS Stream / Rocky / Alma（YUM/DNF）安裝

Step 1：新增 Elastic YUM repository

建立 /etc/yum.repos.d/elasticsearch.repo：

sudo tee /etc/yum.repos.d/elasticsearch.repo << 'EOF'
[elasticsearch]
name=Elasticsearch repository for 9.x packages
baseurl=https://artifacts.elastic.co/packages/9.x/yum
gpgcheck=1
gpgkey=https://artifacts.elastic.co/GPG-KEY-elasticsearch
enabled=0
autorefresh=1
type=rpm-md
EOF
⚠️ 注意 enabled=0 Elastic 預設 repo 為 enabled=0，避免後續 dnf update 不小心升級。每次需要安裝/升級時用 --enablerepo 旗標暫時啟用。

Step 2：安裝

sudo dnf install --enablerepo=elasticsearch elasticsearch
# 或指定版本
sudo dnf install --enablerepo=elasticsearch elasticsearch-9.3.4

RHEL 7/CentOS 7 用 yum 取代 dnf。

Step 3：啟動服務

sudo systemctl daemon-reload
sudo systemctl enable elasticsearch
sudo systemctl start elasticsearch
sudo systemctl status elasticsearch

直接下載 .deb / .rpm 安裝

適合 air-gapped 環境（CH 16 詳述）：

Debian 套件

ES_VERSION=9.3.4

# x86_64
wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-${ES_VERSION}-amd64.deb
wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-${ES_VERSION}-amd64.deb.sha512
sha512sum -c elasticsearch-${ES_VERSION}-amd64.deb.sha512

# 安裝
sudo dpkg -i elasticsearch-${ES_VERSION}-amd64.deb

# aarch64
# wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-${ES_VERSION}-arm64.deb

RPM 套件

ES_VERSION=9.3.4

wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-${ES_VERSION}-x86_64.rpm
wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-${ES_VERSION}-x86_64.rpm.sha512
sha512sum -c elasticsearch-${ES_VERSION}-x86_64.rpm.sha512

sudo rpm --install --verbose --hash elasticsearch-${ES_VERSION}-x86_64.rpm

鎖定版本（重要！）

避免 apt upgrade / dnf update 不小心一次升級多個版本。Elasticsearch 升級必須一個一個 minor 版本來（CH 54）。

Debian/Ubuntu：apt-mark hold

sudo apt-mark hold elasticsearch
# 確認
apt-mark showhold

# 解除
sudo apt-mark unhold elasticsearch

RHEL：dnf versionlock

sudo dnf install -y python3-dnf-plugin-versionlock
sudo dnf versionlock add elasticsearch
# 確認
sudo dnf versionlock list

# 解除
sudo dnf versionlock delete elasticsearch

環境變數設定（/etc/default 或 /etc/sysconfig）

套件安裝會建立預設環境變數檔。Debian 在 /etc/default/elasticsearch，RHEL 在 /etc/sysconfig/elasticsearch：

# /etc/default/elasticsearch  或  /etc/sysconfig/elasticsearch

# 自訂 ES_HOME（一般不需要）
# ES_HOME=/usr/share/elasticsearch

# 自訂設定路徑（一般不需要）
# ES_PATH_CONF=/etc/elasticsearch

# 自訂啟動參數
# ES_JAVA_OPTS="-XX:+UnlockExperimentalVMOptions"

# 使用自訂 JDK（不建議）
# ES_JAVA_HOME=/path/to/your/jdk

# 自訂 PID 檔位置
# PID_DIR=/var/run/elasticsearch

# 自訂啟動 timeout（systemd）
# RESTART_ON_UPGRADE=true

驗證安裝細節

檔案 ownership

ls -la /etc/elasticsearch/
# elasticsearch.yml 應為 root:elasticsearch 0660
# certs/ 應為 root:elasticsearch 0750
ls -la /var/lib/elasticsearch/
# 應為 elasticsearch:elasticsearch 0750

查看 systemd unit

systemctl cat elasticsearch.service

內容類似：

[Unit]
Description=Elasticsearch
Documentation=https://www.elastic.co
Wants=network-online.target
After=network-online.target

[Service]
Type=notify
RuntimeDirectory=elasticsearch
PrivateTmp=true
Environment=ES_HOME=/usr/share/elasticsearch
Environment=ES_PATH_CONF=/etc/elasticsearch
Environment=PID_DIR=/var/run/elasticsearch
Environment=ES_SD_NOTIFY=true
EnvironmentFile=-/etc/default/elasticsearch
WorkingDirectory=/usr/share/elasticsearch

User=elasticsearch
Group=elasticsearch

ExecStart=/usr/share/elasticsearch/bin/systemd-entrypoint -p ${PID_DIR}/elasticsearch.pid --quiet

# StandardOutput → journald
StandardOutput=journal
StandardError=inherit

# 已預設好的 limits
LimitNOFILE=65535
LimitNPROC=4096
LimitAS=infinity
LimitFSIZE=infinity
LimitMEMLOCK=infinity

TimeoutStopSec=0
KillSignal=SIGTERM
KillMode=process
SendSIGKILL=no
SuccessExitStatus=143
TimeoutStartSec=900

[Install]
WantedBy=multi-user.target

自訂 systemd override（不要直接修改 unit）

sudo systemctl edit elasticsearch

會開啟編輯器讓你寫 override 區段，例如：

[Service]
# 增加啟動 timeout 到 10 分鐘
TimeoutStartSec=600

# 自訂 JVM 參數
Environment="ES_JAVA_OPTS=-XX:+UnlockExperimentalVMOptions"

儲存後會建立 /etc/systemd/system/elasticsearch.service.d/override.conf，並自動 daemon-reload。

Log Rotation

套件已自動安裝 logrotate 設定。可調整 /etc/logrotate.d/elasticsearch：

/var/log/elasticsearch/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    copytruncate
    create 0640 elasticsearch elasticsearch
}

/var/log/elasticsearch/*_audit.json {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
    copytruncate
    create 0640 elasticsearch elasticsearch
}

升級流程

Rolling upgrade（生產環境）

每個節點逐一執行：

# 1. 停用 shard allocation（一次即可，在叢集中執行）
curl -X PUT "https://localhost:9200/_cluster/settings" \
  -u elastic --cacert /etc/elasticsearch/certs/http_ca.crt \
  -H 'Content-Type: application/json' \
  -d '{ "persistent": { "cluster.routing.allocation.enable": "primaries" } }'

# 2. 在當前節點：flush + 停止
curl -X POST "https://localhost:9200/_flush" \
  -u elastic --cacert /etc/elasticsearch/certs/http_ca.crt
sudo systemctl stop elasticsearch

# 3. 升級套件
sudo apt-mark unhold elasticsearch
sudo apt update && sudo apt install -y elasticsearch=9.3.5
sudo apt-mark hold elasticsearch

# 4. 啟動
sudo systemctl start elasticsearch

# 5. 等到節點 join 並 health 為 green/yellow，繼續下一節點

# 6. 全部升級後恢復 allocation
curl -X PUT "https://localhost:9200/_cluster/settings" \
  -u elastic --cacert /etc/elasticsearch/certs/http_ca.crt \
  -H 'Content-Type: application/json' \
  -d '{ "persistent": { "cluster.routing.allocation.enable": null } }'

升級時保留設定檔

套件升級會保留 /etc/elasticsearch/ 中的所有檔案。如果新版有新增的設定範本，會以 .dpkg-dist（Debian）或 .rpmnew（RHEL）出現，不會覆蓋現有設定。手動 diff 後合併即可：

# Debian
diff /etc/elasticsearch/elasticsearch.yml /etc/elasticsearch/elasticsearch.yml.dpkg-dist

# RHEL
diff /etc/elasticsearch/elasticsearch.yml /etc/elasticsearch/elasticsearch.yml.rpmnew

解除安裝

Debian/Ubuntu

sudo systemctl stop elasticsearch
sudo systemctl disable elasticsearch

# 移除套件，保留設定與資料
sudo apt remove elasticsearch

# 完全移除（包含 /etc/elasticsearch）
sudo apt purge elasticsearch

# 移除 repo
sudo rm /etc/apt/sources.list.d/elastic-9.x.list
sudo rm /usr/share/keyrings/elasticsearch-keyring.gpg

# 手動清除資料（重要：先備份！）
sudo rm -rf /var/lib/elasticsearch
sudo rm -rf /var/log/elasticsearch

RHEL

sudo systemctl stop elasticsearch
sudo systemctl disable elasticsearch

sudo dnf remove elasticsearch

# 手動清除
sudo rm -rf /var/lib/elasticsearch
sudo rm -rf /var/log/elasticsearch
sudo rm -rf /etc/elasticsearch
sudo rm /etc/yum.repos.d/elasticsearch.repo

套件 vs tar.gz：選擇建議

場景推薦
生產環境，標準 Linux 發行版套件（apt/dnf）
同機跑多版本（測試）tar.gz
非標準路徑（NFS、特殊 mount）tar.gz
Air-gapped 環境下載 .deb/.rpm 或 tar.gz 都可
容器化部署Docker（CH 10）
KubernetesECK（CH 11-14）
macOS 開發機tar.gz 或 Docker

本章重點回顧

  • 套件安裝會自動建立 user、systemd unit、目錄結構，是生產 Linux 的首選
  • Debian 用 APT，RHEL/Rocky 用 DNF；都從 https://artifacts.elastic.co/packages/9.x 取得
  • RHEL repo 預設 enabled=0，需用 --enablerepo 暫時啟用
  • 生產環境必須鎖定版本：apt-mark hold 或 dnf versionlock
  • 修改 systemd 設定用 systemctl edit，不要直接改 unit 檔
  • 升級保留 /etc/elasticsearch/，新範本以 .dpkg-dist/.rpmnew 出現
  • 解除安裝後資料目錄需手動清除（且應先備份）
