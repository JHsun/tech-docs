Connectors 是什麼？

Elastic Connectors 是把第三方系統的資料同步到 Elasticsearch 的官方框架。從 SaaS、檔案儲存、資料庫等系統把文件 indexing 到 ES，常用於：

  • 企業搜尋（搜內部 Confluence / SharePoint / Drive）
  • RAG 知識庫（CH 60）
  • 合規/稽核（把多源 log 集中到 ES）
⚠️ Connectors 取代舊 Workplace Search 的 content sources 9.x 不再用 Workplace Search 自家的 content source 機制，全部統一到 Connectors（CH 59）。

Native vs Self-managed

類型部署位置適合
Native (Cloud)Elastic Cloud 內部 host用 Cloud 服務、不想管 infrastructure
Self-managed自己 host（Docker / K8s）self-managed cluster、air-gapped、需要客製化

Self-managed connector 由 Elastic 開源於 elastic/connectors（Python 框架）。

部署 Self-managed Connector

Step 1：在 Kibana 建立 connector

Kibana → Search → Connectors → New connector → 選類型 → 取得 connector ID 與 service token。

Step 2：用 Docker 跑 connector service

docker run -d \
  --name connector-confluence \
  -v /path/to/config:/config:ro \
  docker.elastic.co/integrations/elastic-connectors:9.3.4 \
  /app/bin/elastic-ingest \
  --config /config/config.yml

config.yml：

elasticsearch:
  host: https://es:9200
  api_key: "..."
  ssl: true
  ca_certs: /config/ca.crt

connectors:
  - connector_id: "abc-123-confluence"
    service_type: "confluence"
    api_key: "..."           # connector-specific service token

Step 3：在 Kibana 填 source 認證

Kibana → Search → Connectors → 該 connector → Configuration：

  • Confluence URL
  • Username + API token
  • Spaces to sync（白名單）
  • 同步排程（每 30 分鐘）
  • 啟用 Document-Level Security

Step 4：觸發第一次同步

點 Sync → 看到資料進入 ES。同步狀態 / 錯誤在 Kibana UI 顯示。

同步類型

Sync 類型說明
Full整個 source 重抓
Incremental只抓變更（基於 timestamp / cursor / last sync）
Access control只同步 permission（不抓內容）

典型策略：每天 1 次 full sync + 每 30 分鐘 incremental + 每小時 access control。

Document-Level Security (DLS)

許多資料源原生有 ACL（誰能看哪個檔）。Connector 把 ACL 同步到 ES，做per-user 過濾：

原理

  1. Connector 同步資料時，連帶記錄 _allow_access_control 欄位（陣列：可讀的 user / group ID）
  2. 同步 source 的 user-group mapping 到 .search-acl-filter-{connector_id} index
  3. 建立 ES API Key 時帶 DLS query："query": { "terms": { "_allow_access_control": [ "user-id", "group-id" ] } }
  4. 用該 API Key 查詢自動套 filter

DLS API Key 範例

POST /_security/api_key
{
  "name": "alice-search-key",
  "role_descriptors": {
    "search": {
      "indices": [
        {
          "names":      [ "search-confluence" ],
          "privileges": [ "read" ],
          "query": {
            "template": {
              "params": {
                "access_control": [
                  "user.id:alice",
                  "group.id:engineering",
                  "group.id:platform-team"
                ]
              },
              "source": {
                "bool": {
                  "should": [
                    {
                      "terms": {
                        "_allow_access_control.enum_keyword": "{{access_control}}"
                      }
                    },
                    {
                      "bool": {
                        "must_not": [
                          { "exists": { "field": "_allow_access_control" } }
                        ]
                      }
                    }
                  ]
                }
              }
            }
          }
        }
      ]
    }
  }
}

Application 用 user 對應的 API key 查 ES，自動只看到 user 可訪問的文件。

常用 Connector 範例

Confluence

  • 同步 Spaces、Pages、Blogposts、Attachments
  • 支援 Cloud 與 Server
  • DLS：sync user / group permissions
  • 增量同步：基於 last_modified

SharePoint Online

  • 同步 Sites、Lists、Drives、Files、Groups
  • 用 OAuth 2.0 + Microsoft Graph API
  • DLS：sync Microsoft 365 group / user
  • 增量同步：用 delta query

GitHub

  • 同步 Repos、Issues、Pull Requests、Files、Commits
  • 用 GitHub App 或 PAT
  • 支援 Org-wide
  • DLS：repo-level 權限

S3

  • 同步 bucket 內檔案內容
  • 支援 PDF、Word、Excel、PPT、TXT、HTML（用 Tika 解析）
  • 用 IAM role（推薦 IRSA）或 access key
  • 過濾：bucket prefix、object size、副檔名

MySQL / PostgreSQL

  • 用 SQL query 抓資料
  • 增量：基於 timestamp 或 auto-increment ID
  • schema 自動 mapping 到 ES
  • 適合產品 catalog、user profile 等表

自訂 Connector

對未支援的源，可用 Python framework 自寫：

# elastic/connectors framework 範例
from connectors.source import BaseDataSource
from connectors.utils import iso_utc

class MyCustomDataSource(BaseDataSource):
    name = "My Custom Source"
    service_type = "my_custom"

    @classmethod
    def get_default_configuration(cls):
        return {
            "host": { "label": "Host", "value": "https://api.example.com" },
            "api_key": { "label": "API Key", "value": "", "sensitive": True },
        }

    async def ping(self):
        # 檢查連線
        pass

    async def get_docs(self, filtering=None):
        # 產生文件給 ES
        for item in await self.fetch_items():
            doc = {
                "_id": item["id"],
                "title": item["title"],
                "content": item["body"],
                "url": item["url"],
                "_timestamp": iso_utc(),
            }
            yield doc, None

把 plug-in 安裝到 connector service，註冊在 Kibana 即可使用。

與 RAG 結合（CH 60）

典型 enterprise RAG：

  1. 多個 Connector 同步到對應 search-* index
  2. 所有 index 加 semantic_text 欄位（自動 embed）
  3. Search Application（CH 59）統一 query 介面
  4. Application 用 user-specific API key（DLS filter）查詢
  5. Hybrid retriever + rerank 取 top-K
  6. 送 LLM 產生答案

使用者得到「我能存取的文件」+「最相關片段」+「LLM 整理過的答案」+「source citation」。

Connector 監控

Kibana → Search → Connectors → 各 connector 看：

  • Sync history（成功/失敗、文件數、耗時）
  • Last error
  • Permission sync 狀態
  • Throughput

進階用 watcher 監控失敗：

PUT _watcher/watch/connector-failure
{
  "trigger": { "schedule": { "interval": "10m" } },
  "input": {
    "search": {
      "request": {
        "indices": [ ".elastic-connectors-sync-jobs*" ],
        "body": {
          "query": {
            "bool": {
              "filter": [
                { "term":  { "status": "error" } },
                { "range": { "completed_at": { "gte": "now-10m" } } }
              ]
            }
          }
        }
      }
    }
  },
  "condition": { "compare": { "ctx.payload.hits.total.value": { "gt": 0 } } },
  "actions": { ... }
}

最佳實踐

  • 從關鍵 source 開始：先 1-2 個源做好再加
  • 啟用 DLS：避免 user 看到不該看的
  • incremental sync 排程：每 30 分鐘到 1 小時，視容忍度
  • Full sync 在離峰時段（每天 1 次即可）
  • API token 權限最小化：read-only、限定 space/folder
  • Connector service 高可用：K8s deployment 至少 2 replicas
  • 監控同步失敗：用 watcher / alerting 即時通知
  • 用 ILM 管理 search-* index 生命週期
  • Crawler 用於公開網站，但慎防 robots.txt 限制
  • 自訂 connector 進 GitOps：CI 構建 Docker image

本章重點回顧

  • Connectors 把第三方系統資料同步到 ES，用於 Enterprise Search / RAG
  • Native（Cloud）vs Self-managed（Docker / K8s）
  • 支援 30+ 資料源：Confluence、SharePoint、S3、GitHub、MySQL、Slack...
  • 三種 sync：full、incremental、access control（permission only）
  • Document-Level Security (DLS)：自動套 per-user filter
  • 用 API key + role template 實現 multi-tenant 搜尋
  • 9.x 統一 Connectors 取代舊 Workplace Search content sources
  • 自訂 connector 用 Python framework（elastic/connectors）
  • 典型 enterprise RAG：Connectors + semantic_text + Search Applications + DLS + LLM
