📦 A. Data Stream 基本操作

Data stream 是時序資料（logs / metrics / traces）的標準容器。對外是一個邏輯名稱，內部是多個 hidden backing indices（自動 rollover）。

⚠️ F. ILM 常見錯誤對照

錯誤訊息
原因
修復
action [shrink] is unauthorized
上次編輯 policy 的 user 沒權限
由 superuser 重新 PUT 一次 policy（會以該 user 身分執行）
shrink target node not found
tier_preference 與實際 node role 不符
檢查 
GET /_cat/nodeattrs?v
是否有對應 tier
force_merge failed (disk)
force_merge 需暫時 2× 空間
清空間或暫關 force_merge action
snapshot repository does not exist
searchable_snapshot 對應 repo 缺失
先建 repo（CS 10）
illegal_argument shrink shards
shrink 目標 shard 數無法整除原 shard 數
從 12 → 1 / 2 / 3 / 4 / 6 OK；12 → 5 不行
tier filter mismatch
index settings 中 tier preference 不存在 node 提供
確認 cluster 中有對應 data tier (data_warm / data_cold) 節點

🆕 G. Data Stream Lifecycle (DSL)

9.x 推出的簡化版 lifecycle，原本給 Serverless 用，現也可在 self-managed 使用：

情境
命令 / Code
說明
設 DSL（最簡）🆕 9+
PUT /_data_stream/logs-app/_lifecycle
{
  "data_retention": "30d"
}
只要保留 30 天；無 phases / 無 shrink / 無 cold / frozen
在 template 中設 DSL
PUT /_index_template/logs-app-template
{
  "index_patterns": ["logs-app-*"],
  "data_stream":    {},
  "template": {
    "lifecycle": {
      "data_retention": "7d"
    }
  }
}
建 template 時就指定 DSL 保留
看 data stream lifecycle
GET /_data_stream/logs-app/_lifecycle
看當前 retention
移除 lifecycle（永久保留）
DELETE /_data_stream/logs-app/_lifecycle
移除後 data 永不刪
看 lifecycle explain
GET /.ds-logs-app-2026.05-000001/_lifecycle/explain
看 DSL 的執行狀態

G2. ILM vs DSL 比較

方案
覆蓋範圍
適用
ILM (傳統)
5 phases / shrink / forcemerge / searchable_snapshot / cold / frozen
功能完整。複雜場景必選
DSL (Data Stream Lifecycle)🆕 9+
只有 retention（保留多久）+ 自動 rollover
簡單場景。無 phase / 無 tier。Serverless 主推
共存規則
若同一 backing index 兩者都套，ILM 優先
可用 index.lifecycle.prefer_ilm: false 反過來
💡 何時用 DSL？
  • 只需要「保留 N 天」的場景
  • 不需要 hot/warm/cold/frozen tier
  • 不需要 shrink / forcemerge
  • 新 data stream，無歷史包袱
否則繼續用 ILM。

🌡️ J. Data Tiers 設定

Tier
節點 role
典型用途
🔥 data_hot
node.roles: [ master, data_hot, ingest ]
SSD / NVMe；最近資料；高 IOPS
☀️ data_warm
node.roles: [ data_warm ]
HDD；read-only 居多；shrink 後

node.roles: [ data_content ]
非時序資料（產品 catalog 等）

{
  "index.routing.allocation.include._tier_preference":
    "data_warm,data_hot"
}

🚒 K. 常見場景 SOP

場景：磁碟快滿，要快速減少資料

  1. 找最大的 index：GET /_cat/indices?v&s=store.size:desc&bytes=gb
  2. 確認 ILM 是否生效：GET /_ilm/explain?only_errors=true
  3. 若卡住 → 修錯誤後 retry；或暫時縮短 retention
  4. 立即手動 rollover：POST /logs-app/_rollover
  5. 對舊 backing index 強制 delete（最後手段）

場景：從 ILM 遷移到 DSL

  1. 新 template 改用 "lifecycle": { "data_retention": "Nd" }
  2. 對既有 data stream：PUT /_data_stream/logs-app/_lifecycle 設 DSL
  3. 每個 backing index：PUT /backing-index/_settings { "index.lifecycle.prefer_ilm": false }
  4. 現有 ILM 管理的舊 backing index 會走完它的 ILM；新建的走 DSL

(function () {
  function init() {
    document.querySelectorAll('main pre').forEach(function (pre) {
      if (pre.parentElement.classList.contains('code-wrap')) return;
      const wrap = document.createElement('div');
      wrap.className = 'code-wrap';
      pre.parentNode.insertBefore(wrap, pre);
      wrap.appendChild(pre);
      const lang = pre.querySelector('code')?.className.match(/language-(\w+)/)?.[1];
      if (lang) {
        const tag = document.createElement('span');
        tag.className = 'code-lang';
        tag.textContent = lang;
        wrap.appendChild(tag);
      }
      const btn = document.createElement('button');
      btn.className = 'copy-btn';
      btn.type = 'button';
      btn.textContent = '📋 複製';
      btn.addEventListener('click', function () {
        const text = pre.innerText;
        if (navigator.clipboard?.writeText) {
          navigator.clipboard.writeText(text).then(success).catch(fallback);
        } else {
          fallback();
        }
        function success() {
          btn.textContent = '✓ 已複製';
          btn.classList.add('copied');
          setTimeout(function () { btn.textContent = '📋 複製'; btn.classList.remove('copied'); }, 1800);
        }
        function fallback() {
          const ta = document.createElement('textarea');
          ta.value = text;
          document.body.appendChild(ta);
          ta.select();
          try { document.execCommand('copy'); success(); } catch (_) {}
          document.body.removeChild(ta);
        }
      });
      wrap.appendChild(btn);
    });
  }
  if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
  else init();
})();

(function () {
  function init() {
    const toc = document.querySelector('.toc ul');
    if (!toc) return;
    const headings = document.querySelectorAll('main h2, main h3');
    headings.forEach(function (h) {
      if (!h.id) {
        h.id = h.textContent.trim().toLowerCase()
          .replace(/[^\w\u4e00-\u9fff\s-]/g, '')
          .replace(/\s+/g, '-');
      }
      const li = document.createElement('li');
      const a = document.createElement('a');
      a.href = '#' + h.id;
      a.textContent = h.textContent;
      a.className = 'toc-' + h.tagName.toLowerCase();
      li.appendChild(a);
      toc.appendChild(li);
    });
  }
  if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
  else init();
})();

    if (myCounter !== renderCounter) return;
    try {
      await mermaid.run({ querySelector: '.mermaid' });
    } catch (e) { console.warn('Mermaid render error', e); }
  }

  function init() {
    renderAll();
    window.addEventListener('jhsun:theme-changed', renderAll);
  }

  if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
  else init();
})();
