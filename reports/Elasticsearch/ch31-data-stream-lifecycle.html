CH 31 · Data Stream Lifecycle (DSL)

DSL 是什麼？

Data Stream Lifecycle（DSL）是 ES 8.x 推出、9.x 持續強化的輕量級資料生命週期管理機制，專為 data stream 設計。比 ILM 更簡單，但功能也較少。

💡 何時用 DSL？
  • 需求簡單：「保留 30 天就刪」
  • 所有資料都在 hot tier 即可
  • 不需要 searchable snapshot / cold / frozen
  • 不想學 ILM 5 phase 與各種 actions
何時用 ILM？
  • 需要 hot/warm/cold/frozen 分層省成本
  • 需要 forcemerge、shrink、searchable snapshot
  • 需要對 retention 邏輯做精細控制（例如刪前先確認 snapshot）

DSL 基本用法

在 index template 中設定 DSL

curl -X PUT -u elastic:xxx --cacert ca.crt \
  -H 'Content-Type: application/json' \
  "https://localhost:9200/_index_template/logs-app-dsl" -d '
{
  "index_patterns": [ "logs-app*" ],
  "data_stream": { },
  "priority": 200,
  "template": {
    "settings": {
      "number_of_shards":   2,
      "number_of_replicas": 1
    },
    "mappings": {
      "properties": {
        "@timestamp": { "type": "date" },
        "message":    { "type": "match_only_text" }
      }
    },
    "lifecycle": {
      "data_retention": "30d"
    }
  }
}'

新建的 data stream 會自動套用 DSL，30 天後自動刪除過期 backing indices。

對既有 data stream 設定 DSL

curl -X PUT -u elastic:xxx --cacert ca.crt \
  -H 'Content-Type: application/json' \
  "https://localhost:9200/_data_stream/logs-app/_lifecycle" -d '
{
  "data_retention": "60d"
}'

移除 DSL

curl -X DELETE -u elastic:xxx --cacert ca.crt \
  "https://localhost:9200/_data_stream/logs-app/_lifecycle"

DSL Rollover 預設條件

DSL 的 rollover 條件是 cluster 全域設定，不能 per-stream 覆寫：

curl -u elastic:xxx --cacert ca.crt \
  "https://localhost:9200/_cluster/settings?include_defaults=true&filter_path=*.cluster.lifecycle.default*&pretty"

預設值：

{
  "cluster.lifecycle.default.rollover": {
    "max_age":              "30d",
    "max_primary_shard_size": "50gb",
    "max_primary_shard_docs": "200000000",
    "min_docs":              "1"
  }
}

可改全域：

PUT _cluster/settings
{
  "persistent": {
    "cluster.lifecycle.default.rollover": "max_age=7d,max_primary_shard_size=20gb"
  }
}

自動 Tail Merge

DSL 對已 rollover 的 backing index 自動執行 tail merge：

  • 不像 forcemerge 把所有 segment 合成 1 個
  • 只合併「最尾端的小 segment」
  • 成本低，運作期間幾乎不影響查詢效能
  • 聚焦於最常被查的近期資料

無需設定，DSL 自動執行。

DSL + Downsampling（TSDS）

對 TSDS 可在 DSL 中設定 downsampling：

PUT _index_template/metrics-system-dsl
{
  "index_patterns": [ "metrics-system*" ],
  "data_stream": { },
  "template": {
    "settings": {
      "index.mode": "time_series",
      "index.routing_path": [ "host.name" ]
    },
    "mappings": { ... },
    "lifecycle": {
      "data_retention": "365d",
      "downsampling": [
        {
          "after": "1d",
          "fixed_interval": "10m"      # 1 天後降為 10 分鐘解析度
        },
        {
          "after": "30d",
          "fixed_interval": "1h"       # 30 天後降為 1 小時
        },
        {
          "after": "90d",
          "fixed_interval": "1d"       # 90 天後降為 1 天
        }
      ]
    }
  }
}

prefer_ilm 設定

當 backing index 同時被 ILM policy 與 DSL 配置時，index.lifecycle.prefer_ilm 決定優先：

  • true（預設）：ILM 優先（功能較完整）
  • false：DSL 優先

從 ILM 遷移到 DSL

# 1. 修改 index template，加 lifecycle，並設 prefer_ilm=false
PUT _index_template/logs-app-template
{
  "index_patterns": [ "logs-app*" ],
  "data_stream": { },
  "template": {
    "settings": {
      "index.lifecycle.name":      "logs-90d-policy",   # ILM 仍在
      "index.lifecycle.prefer_ilm": false                 # 但 DSL 優先
    },
    "lifecycle": {
      "data_retention": "30d"
    }
  }
}

# 2. 對既有 data stream 啟動 DSL
PUT _data_stream/logs-app/_lifecycle
{ "data_retention": "30d" }

結果：

  • 既有 backing index 仍被 ILM 管理（直到自然刪除）
  • 下個 rollover 後的新 backing index 由 DSL 管理
  • 逐步淘汰 ILM

PUT _cluster/settings
{
  "persistent": {
    "data_streams.lifecycle.poll_interval": "1m"   # 預設 5m
  }
}

Kibana Streams 介面（9.2+）

9.2+ Kibana 推出 Streams 頁面，提供統一介面管理 data stream 的 retention：

  • Stack Management → Streams
  • 顯示每個 data stream 的當前生命週期（DSL or ILM）
  • 可一鍵切換、修改 retention 或套用 ILM policy
  • 看到資料攝取量、儲存大小

最佳實踐

  • 新建簡單 use case 直接用 DSL：retention 是唯一需求
  • 需要 hot/warm/cold 分層 → 用 ILM
  • 遷移先設 prefer_ilm=false，逐步切換
  • TSDS 配 DSL downsampling，最簡單的時序資料管理
  • Rollover 條件用 cluster 預設，除非確有特殊需求
  • 用 Kibana Streams 介面管理（9.2+）
  • 定期 check _lifecycle/stats 看是否有 backing_indices_in_error

本章重點回顧

  • DSL 是專為 data stream 設計的輕量生命週期管理
  • 功能聚焦：rollover + retention + tail merge + downsampling，沒有 hot/warm/cold
  • 無 forcemerge、shrink、searchable snapshot 功能（這些要用 ILM）
  • DSL 與 ILM 可共存，由 index.lifecycle.prefer_ilm 決定優先
  • 常見遷移路徑：ILM → 設 prefer_ilm=false + DSL retention → 等舊 index 自然 expire → 純 DSL
  • 9.2+ Kibana Streams 提供 GUI 統一管理
  • 診斷 API：_lifecycle/explain、_lifecycle/stats
