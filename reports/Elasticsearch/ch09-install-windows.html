CH 09 · Windows 安裝

適合對象 🟢 初學者 🔴 維運者
閱讀時間 約 18 分鐘
前置知識 CH 06 · 系統需求
本章目標 在 Windows Server 上完成安裝、註冊為 Windows Service、PowerShell 操作要點、Active Directory 整合準備

Windows 支援版本

Elasticsearch 9.x 支援以下 Windows 版本（節錄自 Support Matrix）：

  • Windows Server 2019、2022、2025
  • x86_64 架構（Windows ARM64 自 9.0 起也支援）
⚠️ Windows 桌面版不適合生產 Windows 10/11 等桌面版本可用於開發測試，但生產環境應使用 Windows Server。EOL 的版本（Windows 7、Windows 8、Windows Server 2012）已不支援。

兩種安裝方式

方式適用說明
ZIP 解壓開發 / 測試 / 客製化部署類似 Linux tar.gz，需手動註冊 service
MSI 安裝程式生產 / 標準部署圖形化精靈，自動註冊 service、自動配置 Windows 防火牆

MSI 安裝（推薦）

Step 1：下載 MSI

到 官方下載頁取得 .msi 連結，或用 PowerShell：

# 以系統管理員身分執行 PowerShell
$ErrorActionPreference = 'Stop'
$ESVersion = '9.3.4'
$Url = "https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-$ESVersion.msi"

Invoke-WebRequest -Uri $Url -OutFile "$env:TEMP\elasticsearch-$ESVersion.msi"

# 驗證 SHA512
$Sha512Url = "$Url.sha512"
Invoke-WebRequest -Uri $Sha512Url -OutFile "$env:TEMP\elasticsearch-$ESVersion.msi.sha512"
$expected = (Get-Content "$env:TEMP\elasticsearch-$ESVersion.msi.sha512").Split(' ')[0]
$actual   = (Get-FileHash "$env:TEMP\elasticsearch-$ESVersion.msi" -Algorithm SHA512).Hash.ToLower()
if ($expected -ne $actual) { throw "SHA512 mismatch!" }
Write-Host "✅ SHA512 verified"

Step 2：互動式安裝

雙擊 .msi 啟動安裝精靈，依序：

  1. 歡迎畫面 → Next
  2. License Agreement（ELv2）→ I accept → Next
  3. 選擇安裝路徑（預設 C:\Program Files\Elasticsearch\Elasticsearch-9.3.4）
  4. 選擇 data path、logs path、config path
  5. 設定 cluster name、node name、network host
  6. 選擇是否註冊為 Windows Service
  7. 選擇 plugin（可後續再裝）
  8. 確認安裝

Step 3：無聲安裝（自動化部署）

適合大量部署或腳本化：

msiexec.exe /i "$env:TEMP\elasticsearch-9.3.4.msi" /qn `
  INSTALLDIR="C:\Program Files\Elasticsearch\Elasticsearch-9.3.4" `
  DATADIRECTORY="D:\elasticsearch\data" `
  LOGSDIRECTORY="D:\elasticsearch\logs" `
  CLUSTERNAME="production-cluster" `
  NODENAME="es-windows-1" `
  NETWORKHOST="0.0.0.0" `
  HTTPPORT=9200 `
  TRANSPORTTCPPORT=9300 `
  INSTALLASSERVICE=true `
  STARTAFTERINSTALL=true `
  STARTWHENWINDOWSSTARTS=true

常用 MSI 屬性：

屬性說明
INSTALLDIR安裝目錄
DATADIRECTORY資料目錄
LOGSDIRECTORY日誌目錄
CONFIGDIRECTORY設定目錄
CLUSTERNAME / NODENAME叢集與節點名稱
NETWORKHOST綁定 host（0.0.0.0、IP、_local_、_eth0_ 等）
HTTPPORT / TRANSPORTTCPPORTPort 設定
SEEDHOSTSdiscovery seed hosts，逗號分隔
INITIALMASTERNODEScluster.initial_master_nodes
INSTALLASSERVICE是否註冊為 Windows Service（true/false）
STARTAFTERINSTALL安裝後立即啟動
STARTWHENWINDOWSSTARTS開機自動啟動
USELOCALSYSTEM是否用 LocalSystem 帳號執行（建議 false 並設專屬 user）
USENETWORKSERVICE用 NetworkService 帳號（推薦給 AD 環境）
USEEXISTINGUSER / USER / PASSWORD用既有 AD 帳號執行 service

ZIP 解壓安裝

適合不能用 MSI 的特殊情境（如 portable 部署）：

$ESVersion = '9.3.4'
$Dest = 'C:\elastic'

# 下載
$ZipUrl = "https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-$ESVersion-windows-x86_64.zip"
Invoke-WebRequest -Uri $ZipUrl -OutFile "$env:TEMP\es.zip"

# 解壓
New-Item -ItemType Directory -Force -Path $Dest
Expand-Archive -Path "$env:TEMP\es.zip" -DestinationPath $Dest

# 進入目錄
$ESHome = "$Dest\elasticsearch-$ESVersion"
Set-Location $ESHome

# 前景啟動測試
.\bin\elasticsearch.bat

把 ZIP 解壓的 ES 註冊為 Windows Service

# 設定環境變數（持久化）
[Environment]::SetEnvironmentVariable('ES_HOME', $ESHome, 'Machine')
[Environment]::SetEnvironmentVariable('ES_PATH_CONF', "$ESHome\config", 'Machine')

# 註冊服務（系統管理員 PowerShell）
& "$ESHome\bin\elasticsearch-service.bat" install

# 啟動
& "$ESHome\bin\elasticsearch-service.bat" start

# 設定為自動啟動
Set-Service -Name 'elasticsearch-service-x64' -StartupType Automatic

elasticsearch-service.bat 子命令：

命令說明
install註冊 Windows service
remove移除 service
start / stop啟動 / 停止
manager開啟 GUI 管理介面（編輯 JVM、log、登入帳號）

設定 elasticsearch.yml

MSI 安裝後預設位置：C:\ProgramData\Elastic\Elasticsearch\config\elasticsearch.yml

ZIP 安裝：%ES_HOME%\config\elasticsearch.yml

cluster.name: windows-cluster
node.name: WIN-ES-01

path.data:  D:\elasticsearch\data
path.logs:  D:\elasticsearch\logs

network.host: 0.0.0.0
http.port: 9200
transport.port: 9300

discovery.seed_hosts: ["10.0.1.10", "10.0.1.11", "10.0.1.12"]
cluster.initial_master_nodes: ["WIN-ES-01", "WIN-ES-02", "WIN-ES-03"]   # 首次後移除

# Windows 不需 bootstrap.memory_lock（無 mlockall），改用：
bootstrap.memory_lock: false
⚠️ Windows 路徑分隔符 YAML 中的 Windows 路徑可用反斜線（D:\path），或正斜線（D:/path）。建議用正斜線避免 escape 問題。或用 quotation："D:\path"。

Windows 專屬 OS Tuning

關閉 Windows Search 索引（資料磁碟）

避免 Windows Search 試圖索引 ES 的資料目錄，影響 I/O：

# 停用整個 Windows Search 服務（若不需要）
Stop-Service WSearch
Set-Service WSearch -StartupType Disabled

# 或保留服務但排除特定目錄：
# 控制台 → 索引選項 → 修改 → 取消勾選 D:\elasticsearch

排除 Windows Defender 掃描

Real-time scan 會大幅拖慢 ES 寫入：

Add-MpPreference -ExclusionPath "D:\elasticsearch\data"
Add-MpPreference -ExclusionPath "D:\elasticsearch\logs"
Add-MpPreference -ExclusionPath "C:\Program Files\Elasticsearch"
Add-MpPreference -ExclusionProcess "elasticsearch.exe"
Add-MpPreference -ExclusionProcess "java.exe"

# 確認
Get-MpPreference | Select-Object ExclusionPath, ExclusionProcess

設為高效能電源計劃

powercfg -setactive 8c5e7fda-e8bf-4a96-9a85-a6e23a8c635c
# 8c5e7fda... 是 High Performance GUID

調整 Pagefile

Windows 的 swap 替代品。建議：

  • 不要完全關閉（Windows 部分功能需要）
  • 固定大小，避免動態調整
  • ES 啟用 bootstrap.memory_lock: true（虛擬記憶體鎖定）— Windows 上等同於 mlockall
# 設定為 8 GB 固定大小（範例）
$ComputerSystem = Get-WmiObject -Class Win32_ComputerSystem -EnableAllPrivileges
$ComputerSystem.AutomaticManagedPagefile = $false
$ComputerSystem.Put()

$Pagefile = Get-WmiObject -Class Win32_PageFileSetting -Filter "name='C:\\pagefile.sys'"
$Pagefile.InitialSize = 8192
$Pagefile.MaximumSize = 8192
$Pagefile.Put()

Windows Firewall 規則

MSI 會自動加入規則。手動：

# HTTP 9200
New-NetFirewallRule -DisplayName "Elasticsearch HTTP" `
  -Direction Inbound -Protocol TCP -LocalPort 9200 -Action Allow `
  -RemoteAddress 10.0.0.0/8

# Transport 9300
New-NetFirewallRule -DisplayName "Elasticsearch Transport" `
  -Direction Inbound -Protocol TCP -LocalPort 9300 -Action Allow `
  -RemoteAddress 10.0.0.0/8

管理 — Service 操作

# 啟動 / 停止 / 重啟
Start-Service Elasticsearch
Stop-Service Elasticsearch
Restart-Service Elasticsearch

# 狀態
Get-Service Elasticsearch

# 檢視 service 設定
Get-WmiObject win32_service -Filter "name='elasticsearch-service-x64'" | Format-List *

Active Directory 整合準備

若計畫使用 AD 認證（CH 21 詳述），需做以下前置作業：

建立專屬 service account

# 建立 AD 服務帳號（在 DC 上執行）
New-ADUser -Name "es_service" -SamAccountName "es_service" `
  -AccountPassword (Read-Host -AsSecureString) `
  -Enabled $true -PasswordNeverExpires $true

# 在 ES 機器上設定 service 用此帳號執行
sc.exe config elasticsearch-service-x64 obj= "DOMAIN\es_service" password= "xxx"

Kerberos keytab（若用 Kerberos 認證）

# 在 DC 上產生 keytab
ktpass.exe -princ HTTP/es-host.example.com@EXAMPLE.COM `
  -mapuser es_service@EXAMPLE.COM `
  -pass MyP@ssw0rd `
  -ptype KRB5_NT_PRINCIPAL `
  -out C:\es.keytab

# 將 keytab 複製到 ES 機器，放入 config 目錄
copy C:\es.keytab "C:\ProgramData\Elastic\Elasticsearch\config\es.keytab"

測試連線

# 重設 elastic 密碼
& "C:\Program Files\Elasticsearch\Elasticsearch-9.3.4\bin\elasticsearch-reset-password.bat" -u elastic -i

# 測試
$cred = Get-Credential -UserName 'elastic' -Message 'Enter elastic password'
Invoke-RestMethod -Uri 'https://localhost:9200/' `
  -Credential $cred `
  -SkipCertificateCheck
# 生產環境請用 -Certificate 指定 http_ca.crt 而非 SkipCertificateCheck

日誌與排錯

日誌位置

  • MSI 預設：C:\ProgramData\Elastic\Elasticsearch\logs\
  • ZIP：%ES_HOME%\logs\
  • Service 啟動失敗事件：Windows Event Viewer → Windows Logs → Application（來源：elasticsearch-service-x64）

即時觀察 log

Get-Content -Path "C:\ProgramData\Elastic\Elasticsearch\logs\windows-cluster.log" -Wait -Tail 50

Windows 常見問題

症狀原因處理
Service 無法啟動，Event Viewer 報 1067JVM 啟動失敗（OOM 或設定錯）看 ES log；檢查 jvm.options heap 設定
啟動極慢Windows Defender 即時掃描排除 ES 目錄與 process
磁碟 I/O 異常高Windows Search 索引 data 目錄關閉 search 或排除目錄
無法 bind port其他程式佔用netstat -ano | findstr 9200 找出 PID
權限不足寫 data dirservice 帳號權限不足在 data dir 給 service 帳號 Full Control

升級流程（Windows）

MSI 升級

MSI 自動處理舊版本移除、新版本安裝、設定保留：

# 下載新版本 MSI
$NewVersion = '9.3.5'
Invoke-WebRequest "https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-$NewVersion.msi" `
  -OutFile "$env:TEMP\elasticsearch-$NewVersion.msi"

# 升級（保留設定）
msiexec.exe /i "$env:TEMP\elasticsearch-$NewVersion.msi" /qn /norestart

ZIP 升級（手動）

# 1. 停止 service
Stop-Service Elasticsearch

# 2. 移除舊 service
& "$ESHome\bin\elasticsearch-service.bat" remove

# 3. 解壓新版本到並列目錄
$NewESHome = 'C:\elastic\elasticsearch-9.3.5'
Expand-Archive -Path 'elasticsearch-9.3.5-windows-x86_64.zip' -DestinationPath 'C:\elastic\'

# 4. 複製設定（如果你不在 default config 路徑）
Copy-Item -Recurse "$ESHome\config\*" "$NewESHome\config\" -Force

# 5. 註冊新 service 並啟動
& "$NewESHome\bin\elasticsearch-service.bat" install
Start-Service elasticsearch-service-x64

本章重點回顧

  • Windows 安裝有 MSI（推薦）與 ZIP 兩種方式；生產用 MSI
  • MSI 支援無聲安裝（msiexec /qn），適合自動化部署
  • ZIP 解壓後用 elasticsearch-service.bat install 註冊為 Windows Service
  • Windows 必做的調校：排除 Defender 掃描、關閉 Windows Search 索引 data 目錄、設高效能電源計劃
  • Windows Firewall 開放 9200 / 9300 port
  • 計畫整合 AD 時用 ktpass 產生 keytab 並用專屬 service account
  • 用 PowerShell Invoke-RestMethod 替代 curl，操作更原生
