前置知識 CH 03-06、Docker 基礎
本章目標 用 Docker 跑單節點 ES、用 Docker Compose 跑生產級 3 節點叢集、處理 vm.max_map_count 與 volume 持久化

官方 Docker Image

Elastic 維護 4 種 image，全部托管於 docker.elastic.co：

Image說明
docker.elastic.co/elasticsearch/elasticsearch標準完整版（含 ML、所有 modules）
docker.elastic.co/elasticsearch/elasticsearch-fipsFIPS 140 合規版
docker.elastic.co/wolfi/elasticsearch基於 Wolfi（極簡 Linux distro）的版本，攻擊面小
docker.elastic.co/elasticsearch/elasticsearch:9.3.4-arm64ARM64 變體（如 Apple Silicon、AWS Graviton）

Tags 標準格式：:9.3.4、:9.3（最新 9.3.x）、:9（最新 9.x.x）、:latest（不建議用，難追溯）。

🚨 生產絕不要用 :latest Pin 到具體版本（如 9.3.4）才能精確控制升級。:latest 會在不知情下更新，導致升級失敗或叢集版本不一致。

標準版 vs Wolfi 版

項目標準（基於 RHEL UBI）Wolfi 版
大小~ 850 MB~ 530 MB
Shellbash 完整無 bash（只有 sh）
除錯工具豐富（curl、ps、netstat 等）幾乎沒有
安全性標準更佳（攻擊面更小）
適用多數場景合規環境、CSPM 嚴格的部署

先決條件：vm.max_map_count

Docker 容器內無法修改 sysctl 參數（除非用 --privileged），必須在 host 上設定：

# Linux host
sudo sysctl -w vm.max_map_count=262144

# 永久
echo 'vm.max_map_count=262144' | sudo tee /etc/sysctl.d/99-elasticsearch.conf
sudo sysctl --system
💡 Docker Desktop on macOS / Windows Docker Desktop 4.x 起，預設 VM 已將 vm.max_map_count 設為足夠值。如果遇到問題，可在 Docker Desktop 設定 → Resources → Virtual Machine 調整。

單節點：Docker run

開發模式（localhost only）

# 拉 image
docker pull docker.elastic.co/elasticsearch/elasticsearch:9.3.4

# 建立 network
docker network create elastic

# 啟動單節點 ES
docker run -d \
  --name es-single \
  --net elastic \
  -p 127.0.0.1:9200:9200 \
  -e "discovery.type=single-node" \
  -e "xpack.security.enabled=false" \
  -e "ES_JAVA_OPTS=-Xms2g -Xmx2g" \
  --memory=4g \
  docker.elastic.co/elasticsearch/elasticsearch:9.3.4

等 30-60 秒後測試：

curl http://localhost:9200/

開發模式（啟用 security）

9.x 預設 security 是開的，會自動產生 elastic 密碼：

docker run -d \
  --name es-single-secure \
  --net elastic \
  -p 127.0.0.1:9200:9200 \
  -e "discovery.type=single-node" \
  -e "ES_JAVA_OPTS=-Xms2g -Xmx2g" \
  --memory=4g \
  docker.elastic.co/elasticsearch/elasticsearch:9.3.4

# 等啟動完成後重設密碼
docker exec -it es-single-secure /usr/share/elasticsearch/bin/elasticsearch-reset-password -u elastic

取得自動產生的 CA 憑證並測試：

docker cp es-single-secure:/usr/share/elasticsearch/config/certs/http_ca.crt ./http_ca.crt

curl -X GET "https://localhost:9200" \
  -u elastic --cacert ./http_ca.crt

生產級單節點（含 volume 持久化）

docker volume create esdata
docker volume create eslogs

docker run -d \
  --name elasticsearch \
  --net elastic \
  -p 9200:9200 \
  -p 9300:9300 \
  -e "node.name=es-prod-1" \
  -e "cluster.name=production" \
  -e "discovery.type=single-node" \
  -e "ES_JAVA_OPTS=-Xms16g -Xmx16g" \
  -e "bootstrap.memory_lock=true" \
  --ulimit memlock=-1:-1 \
  --ulimit nofile=65535:65535 \
  --memory=32g \
  -v esdata:/usr/share/elasticsearch/data \
  -v eslogs:/usr/share/elasticsearch/logs \
  --restart unless-stopped \
  docker.elastic.co/elasticsearch/elasticsearch:9.3.4

關鍵參數解釋

參數必要性說明
--ulimit memlock=-1:-1啟用 memory_lock 必要允許容器內 process mlockall
--ulimit nofile=65535:65535強烈建議file descriptor 上限
--memory=32g建議限制容器最大記憶體；應 ≥ 2 × heap
-e ES_JAVA_OPTS建議明確設 heap，不要讓自動 sizing 依容器看到的 host 全部 RAM 計算
-v esdata:/usr/share/elasticsearch/data必要持久化資料，否則容器刪除即資料消失
--restart unless-stopped建議意外掛掉自動重啟

Docker Compose：3 節點叢集

真正的生產通常用 ECK（K8s），但 Docker Compose 適合單機測試多節點行為。建立 docker-compose.yml：

version: "3.9"

services:
  setup:
    image: docker.elastic.co/elasticsearch/elasticsearch:9.3.4
    volumes:
      - certs:/usr/share/elasticsearch/config/certs
    user: "0"
    command: >
      bash -c '
        if [ ! -f config/certs/ca.zip ]; then
          echo "Creating CA";
          bin/elasticsearch-certutil ca --silent --pem -out config/certs/ca.zip;
          unzip config/certs/ca.zip -d config/certs;
        fi;
        if [ ! -f config/certs/certs.zip ]; then
          echo "Creating certs";
          echo -ne "instances:\n  - name: es01\n    dns: [es01,localhost]\n    ip: [127.0.0.1]\n  - name: es02\n    dns: [es02,localhost]\n    ip: [127.0.0.1]\n  - name: es03\n    dns: [es03,localhost]\n    ip: [127.0.0.1]\n" > config/certs/instances.yml;
          bin/elasticsearch-certutil cert --silent --pem -out config/certs/certs.zip --in config/certs/instances.yml --ca-cert config/certs/ca/ca.crt --ca-key config/certs/ca/ca.key;
          unzip config/certs/certs.zip -d config/certs;
        fi;
        chown -R 1000:0 config/certs;
        echo "Setup complete";
      '

  es01: &es-default
    depends_on:
      setup:
        condition: service_completed_successfully
    image: docker.elastic.co/elasticsearch/elasticsearch:9.3.4
    volumes:
      - certs:/usr/share/elasticsearch/config/certs
      - esdata01:/usr/share/elasticsearch/data
    ports:
      - "9200:9200"
    environment:
      - node.name=es01
      - cluster.name=docker-cluster
      - cluster.initial_master_nodes=es01,es02,es03
      - discovery.seed_hosts=es02,es03
      - ELASTIC_PASSWORD=${ELASTIC_PASSWORD}
      - bootstrap.memory_lock=true
      - xpack.security.enabled=true
      - xpack.security.http.ssl.enabled=true
      - xpack.security.http.ssl.key=certs/es01/es01.key
      - xpack.security.http.ssl.certificate=certs/es01/es01.crt
      - xpack.security.http.ssl.certificate_authorities=certs/ca/ca.crt
      - xpack.security.transport.ssl.enabled=true
      - xpack.security.transport.ssl.key=certs/es01/es01.key
      - xpack.security.transport.ssl.certificate=certs/es01/es01.crt
      - xpack.security.transport.ssl.certificate_authorities=certs/ca/ca.crt
      - xpack.security.transport.ssl.verification_mode=certificate
      - xpack.license.self_generated.type=basic
      - ES_JAVA_OPTS=-Xms2g -Xmx2g
    ulimits:
      memlock: { soft: -1, hard: -1 }
      nofile:  { soft: 65536, hard: 65536 }
    mem_limit: 4g
    healthcheck:
      test: ["CMD-SHELL", "curl -s --cacert config/certs/ca/ca.crt https://localhost:9200/_cluster/health | grep -E '\"status\":\"(green|yellow)\"' || exit 1"]
      interval: 10s
      timeout: 10s
      retries: 120

  es02:
    <<: *es-default
    volumes:
      - certs:/usr/share/elasticsearch/config/certs
      - esdata02:/usr/share/elasticsearch/data
    ports:
      - "9201:9200"
    environment:
      - node.name=es02
      - cluster.name=docker-cluster
      - cluster.initial_master_nodes=es01,es02,es03
      - discovery.seed_hosts=es01,es03
      - ELASTIC_PASSWORD=${ELASTIC_PASSWORD}
      - bootstrap.memory_lock=true
      - xpack.security.enabled=true
      - xpack.security.http.ssl.enabled=true
      - xpack.security.http.ssl.key=certs/es02/es02.key
      - xpack.security.http.ssl.certificate=certs/es02/es02.crt
      - xpack.security.http.ssl.certificate_authorities=certs/ca/ca.crt
      - xpack.security.transport.ssl.enabled=true
      - xpack.security.transport.ssl.key=certs/es02/es02.key
      - xpack.security.transport.ssl.certificate=certs/es02/es02.crt
      - xpack.security.transport.ssl.certificate_authorities=certs/ca/ca.crt
      - xpack.security.transport.ssl.verification_mode=certificate
      - xpack.license.self_generated.type=basic
      - ES_JAVA_OPTS=-Xms2g -Xmx2g

  es03:
    <<: *es-default
    volumes:
      - certs:/usr/share/elasticsearch/config/certs
      - esdata03:/usr/share/elasticsearch/data
    ports:
      - "9202:9200"
    environment:
      - node.name=es03
      - cluster.name=docker-cluster
      - cluster.initial_master_nodes=es01,es02,es03
      - discovery.seed_hosts=es01,es02
      - ELASTIC_PASSWORD=${ELASTIC_PASSWORD}
      - bootstrap.memory_lock=true
      - xpack.security.enabled=true
      - xpack.security.http.ssl.enabled=true
      - xpack.security.http.ssl.key=certs/es03/es03.key
      - xpack.security.http.ssl.certificate=certs/es03/es03.crt
      - xpack.security.http.ssl.certificate_authorities=certs/ca/ca.crt
      - xpack.security.transport.ssl.enabled=true
      - xpack.security.transport.ssl.key=certs/es03/es03.key
      - xpack.security.transport.ssl.certificate=certs/es03/es03.crt
      - xpack.security.transport.ssl.certificate_authorities=certs/ca/ca.crt
      - xpack.security.transport.ssl.verification_mode=certificate
      - xpack.license.self_generated.type=basic
      - ES_JAVA_OPTS=-Xms2g -Xmx2g

  kibana:
    depends_on:
      es01: { condition: service_healthy }
      es02: { condition: service_healthy }
      es03: { condition: service_healthy }
    image: docker.elastic.co/kibana/kibana:9.3.4
    volumes:
      - certs:/usr/share/kibana/config/certs
      - kibanadata:/usr/share/kibana/data
    ports:
      - "5601:5601"
    environment:
      - SERVERNAME=kibana
      - ELASTICSEARCH_HOSTS=https://es01:9200
      - ELASTICSEARCH_USERNAME=kibana_system
      - ELASTICSEARCH_PASSWORD=${KIBANA_PASSWORD}
      - ELASTICSEARCH_SSL_CERTIFICATEAUTHORITIES=config/certs/ca/ca.crt
    mem_limit: 1g

volumes:
  certs:      { driver: local }
  esdata01:   { driver: local }
  esdata02:   { driver: local }
  esdata03:   { driver: local }
  kibanadata: { driver: local }

建立 .env 檔

cat > .env << EOF
# 至少 6 個字元，建議 16+
ELASTIC_PASSWORD=ChangeMeStr0ngP@ssw0rd
KIBANA_PASSWORD=AnotherStr0ngP@ssw0rd
EOF

chmod 600 .env

啟動叢集

docker compose up -d
docker compose ps

# 觀察 log
docker compose logs -f es01

# 等 health 為 green
curl -k -u elastic:ChangeMeStr0ngP@ssw0rd https://localhost:9200/_cluster/health?pretty

設定 kibana_system 密碼

docker exec -it $(docker compose ps -q es01) \
  /usr/share/elasticsearch/bin/elasticsearch-reset-password \
  -u kibana_system \
  --batch \
  --password "AnotherStr0ngP@ssw0rd"

Docker 專屬注意事項

記憶體規劃

Elasticsearch 在容器中會自動偵測 cgroup memory limit 並計算 heap。但強烈建議手動設定 ES_JAVA_OPTS：

  • 容器 mem_limit = 2 × heap（給 OS cache 留空間）
  • 明確指定 -Xms = -Xmx（避免動態 resize）
  • 單節點記憶體上限：< 32 GB（compressed oops 限制）

儲存效能

避免使用 Docker 的 overlay layer 寫資料（效能差）。Volume 一律 mount，並考慮：

  • Bind mount：直接掛 host 目錄，效能最佳，可備份
  • Named volume：Docker 管理，方便但需注意位於 /var/lib/docker/volumes
  • NFS / EFS：❌ 不要
# 改用 bind mount（在 docker-compose.yml）
volumes:
  - /mnt/nvme/es/data01:/usr/share/elasticsearch/data

UID 與檔案權限

Image 內的 elasticsearch user 是 UID/GID 1000。Bind mount host 目錄時必須：

sudo chown -R 1000:0 /mnt/nvme/es/data01
sudo chmod -R 770 /mnt/nvme/es/data01

Logging driver

預設 json-file driver 會無限累積。建議：

services:
  es01:
    # ...
    logging:
      driver: json-file
      options:
        max-size: "100m"
        max-file: "5"

或集中送往外部：

    logging:
      driver: gelf
      options:
        gelf-address: "udp://logstash:12201"

升級流程

同 major rolling upgrade

逐節點操作：

# 修改 docker-compose.yml 的 image tag（如 9.3.4 → 9.3.5）

# 停用 allocation
curl -k -u elastic:xxx -X PUT "https://localhost:9200/_cluster/settings" \
  -H 'Content-Type: application/json' \
  -d '{ "persistent": { "cluster.routing.allocation.enable": "primaries" } }'

# 一個一個升
docker compose up -d --no-deps --force-recreate es01
# 等 es01 重新 join health 為 green/yellow

docker compose up -d --no-deps --force-recreate es02
# 等 ...

docker compose up -d --no-deps --force-recreate es03
# 等 ...

# 恢復 allocation
curl -k -u elastic:xxx -X PUT "https://localhost:9200/_cluster/settings" \
  -H 'Content-Type: application/json' \
  -d '{ "persistent": { "cluster.routing.allocation.enable": null } }'

開發者實用技巧

進入容器

docker exec -it es-single bash
# Wolfi image 沒有 bash，用 sh
docker exec -it es-single sh

線上修改設定（不重啟）

Cluster-level dynamic settings 用 API 即可：

curl -k -u elastic:xxx -X PUT "https://localhost:9200/_cluster/settings" \
  -H 'Content-Type: application/json' \
  -d '{ "persistent": { "cluster.routing.allocation.disk.threshold_enabled": false } }'

Podman 相容性

9.x 的 image 完全相容 Podman。把所有命令的 docker 換成 podman 即可。Docker Compose 可用 podman-compose 取代。

本章重點回顧

  • 使用官方 image docker.elastic.co/elasticsearch/elasticsearch:9.3.4，永遠 pin 具體版本
  • 容器無法改 sysctl，必須在 host 上設定 vm.max_map_count=262144
  • 啟用 bootstrap.memory_lock 必須加 --ulimit memlock=-1:-1
  • 用 named volume 或 bind mount 持久化 data，不要寫進 container 層
  • Bind mount host 目錄時 ownership 設為 UID 1000:0
  • Docker Compose 3 節點叢集需要 setup container 預先產生 TLS 憑證
  • 真正生產環境多節點建議用 ECK（K8s），Docker Compose 主要用於本機測試
