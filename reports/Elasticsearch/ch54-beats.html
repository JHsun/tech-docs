Filebeat（最常用）

安裝

sudo apt-get install filebeat
# 或
sudo yum install filebeat
# Docker
docker run -d --name filebeat \
  --user root \
  -v /var/lib/docker/containers:/var/lib/docker/containers:ro \
  -v /var/run/docker.sock:/var/run/docker.sock:ro \
  docker.elastic.co/beats/filebeat:9.3.4

基本設定

/etc/filebeat/filebeat.yml：

filebeat.inputs:
- type: filestream
  id: app-logs
  paths:
    - /var/log/app/*.log
  parsers:
    - multiline:
        type: pattern
        pattern: '^\d{4}-\d{2}-\d{2}'
        negate: true
        match: after

  fields:
    service.name: my-app
    environment:  production
  fields_under_root: true

- type: filestream
  id: nginx-access
  paths:
    - /var/log/nginx/access.log
  fields:
    service.name: nginx
  fields_under_root: true

processors:
  - add_host_metadata: {}
  - add_cloud_metadata: {}
  - add_docker_metadata: {}
  - add_kubernetes_metadata: {}
  - drop_event:
      when:
        contains:
          message: "DEBUG"

# 直接送 ES（推薦走 data stream）
output.elasticsearch:
  hosts:    [ "https://es:9200" ]
  api_key:  "${ES_API_KEY}"
  ssl.certificate_authorities: [ "/etc/filebeat/certs/ca.crt" ]
  pipeline: "filebeat-cleanup"      # 串接 ES ingest pipeline

# 或送 Logstash 做複雜處理
# output.logstash:
#   hosts: [ "logstash:5044" ]
#   ssl.certificate_authorities: [ ... ]

Filebeat Modules

內建 80+ module，自動配置 input + ingest pipeline + dashboard：

# 啟用 module
sudo filebeat modules enable nginx mysql apache system

# 看哪些啟用了
sudo filebeat modules list

# 設定 module（檔在 /etc/filebeat/modules.d/）
sudo vim /etc/filebeat/modules.d/nginx.yml

nginx.yml：

- module: nginx
  access:
    enabled: true
    var.paths: [ "/var/log/nginx/access.log*" ]
  error:
    enabled: true
    var.paths: [ "/var/log/nginx/error.log*" ]
  ingress_controller:
    enabled: false

啟用後自動：

  1. 建對應 ingest pipeline 在 ES
  2. 建 index template 與 ILM policy
  3. 建 Kibana dashboard
  4. 開始收集

Autodiscover（容器環境）

filebeat.autodiscover:
  providers:
    - type: docker
      hints.enabled: true
      templates:
        - condition:
            contains:
              docker.container.image: nginx
          config:
            - module: nginx
              access.input:
                type: container
                paths: [ "/var/lib/docker/containers/${data.docker.container.id}/*.log" ]

    - type: kubernetes
      hints.enabled: true
      templates:
        - condition:
            equals:
              kubernetes.labels.app: my-app
          config:
            - type: filestream
              paths: [ "/var/log/containers/*${data.kubernetes.container.id}.log" ]
              parsers: [ container: {} ]

支援 hints：在 docker / k8s annotations 加 co.elastic.logs/module: nginx，filebeat 自動套對應 module。

Metricbeat

類似 filebeat 但收 metrics：

sudo metricbeat modules enable system docker kubernetes nginx mysql

# 看 module 設定
ls /etc/metricbeat/modules.d/

System module 範例：

- module: system
  period: 10s
  metricsets:
    - cpu
    - load
    - memory
    - network
    - process
    - process_summary
    - socket_summary
    - filesystem
    - fsstat
    - diskio
  process.include_top_n:
    by_cpu: 5
    by_memory: 5

支援的 modules（部分）

  • System、Linux、Windows
  • Docker、Kubernetes、Containerd
  • AWS、GCP、Azure（cloud metrics）
  • MySQL、PostgreSQL、MongoDB、Redis、Cassandra
  • Kafka、RabbitMQ、ActiveMQ
  • Nginx、Apache、HAProxy
  • Elasticsearch、Logstash、Kibana（CH 46 stack monitoring）
  • Prometheus（pull 既有 exporter）
  • vSphere、Zookeeper、etcd

Auditbeat：稽核事件

auditbeat.modules:
- module: auditd
  audit_rules: |
    -w /etc/passwd -p wa -k passwd_changes
    -w /etc/shadow -p wa -k shadow_changes
    -a always,exit -F arch=b64 -S execve -k exec

- module: file_integrity
  paths:
    - /bin
    - /usr/bin
    - /sbin
    - /etc

- module: system
  datasets:
    - host
    - login
    - package
    - process
    - socket
    - user

常用於合規（PCI-DSS、HIPAA），與 SIEM（CH 58）整合。

Packetbeat：網路分析

packetbeat.interfaces.device: any

packetbeat.protocols:
- type: http
  ports: [ 80, 8080, 8000, 5000, 8002 ]

- type: tls
  ports: [ 443, 993, 995, 5223, 8443, 8883, 9243 ]

- type: dns
  ports: [ 53 ]
  include_authorities: true
  include_additionals: true

- type: mysql
  ports: [ 3306 ]

- type: redis
  ports: [ 6379 ]

產生 protocol-level 事件（HTTP request/response、DNS query、MySQL query 等）。

Heartbeat：uptime 監控

heartbeat.monitors:
- type: http
  id: my-website
  name: My Website
  schedule: '@every 30s'
  hosts: [ "https://example.com" ]
  check.response.status: [ 200 ]
  check.response.body: 'Welcome'
  ssl:
    verification_mode: full
    supported_protocols: [ "TLSv1.2", "TLSv1.3" ]

- type: tcp
  id: db-port
  name: Database Port
  schedule: '@every 1m'
  hosts: [ "db1.internal:5432", "db2.internal:5432" ]

- type: icmp
  id: ping-router
  name: Ping Router
  schedule: '@every 30s'
  hosts: [ "192.168.1.1" ]

透過 Kibana → Observability → Uptime / Synthetics 視覺化。

winlogbeat.event_logs:

Processors（所有 Beats 通用）

processors:
  - add_host_metadata:
      cache.ttl: 5m
      geo:
        name: ap-northeast-1a
  - add_cloud_metadata: {}
  - add_docker_metadata: {}
  - add_kubernetes_metadata:
      host: ${NODE_NAME}
      matchers:
      - logs_path:
          logs_path: '/var/log/containers/'

  - drop_event:
      when:
        regexp:
          message: '^DEBUG'

  - rename:
      fields:
        - { from: "old_name", to: "new_name" }

  - script:
      lang: javascript
      source: >
        function process(event) {
          var msg = event.Get("message");
          if (msg && msg.indexOf("ERROR") !== -1) {
            event.Put("severity", "high");
          }
        }

Output 選項

Output to Elasticsearch（直送）

output.elasticsearch:
  hosts: [ "https://es1:9200", "https://es2:9200" ]
  api_key: "${ES_API_KEY}"
  ssl.certificate_authorities: [ "/etc/filebeat/certs/ca.crt" ]
  pipeline: "filebeat-final"        # 串 ES ingest pipeline
  bulk_max_size: 1600
  worker: 1

Output to Logstash

output.logstash:
  hosts: [ "logstash:5044" ]
  ssl.certificate_authorities: [ "/etc/filebeat/certs/ca.crt" ]
  loadbalance: true

Output to Kafka（緩衝）

output.kafka:
  hosts: [ "kafka1:9092", "kafka2:9092" ]
  topic: "logs-%{[service][name]}"
  partition.round_robin:
    reachable_only: true
  required_acks: 1
  compression: snappy

Data Stream 模式

9.x Beats 預設用 data stream（取代舊 daily index 模式）：

setup.template.name:    "filebeat"
setup.template.pattern: "filebeat-*"
setup.ilm.enabled:      true
setup.ilm.policy_name:  "filebeat-90d"

會自動建：

  • data stream filebeat-{version}-{namespace}
  • index template + ILM policy
  • Kibana dashboards

Beats vs Elastic Agent

項目BeatsElastic Agent
Process 數每個 Beat 一個單一 process（內含多個 sub-collector）
設定每 Beat 各自 yml統一 policy via Fleet
升級個別套件升級Fleet 中央升級
整合Beat moduleFleet integrations（更多）
OTel 支援有限內建 EDOT collector
Endpoint Security需另裝 Elastic Endpoint內建 Defend
適合已部署 / 簡單場景 / 客製化新部署 / 多功能 / 需中央管理

最佳實踐

  • 新部署優先 Elastic Agent + Fleet（CH 55）
  • 既有 Beats 部署可繼續用，無需強制遷移
  • 用 module 比手刻 input 快又標準
  • 容器環境用 autodiscover，無需重啟即可發現新容器
  • 產生 API Key 給 Beats（CH 22），最小權限
  • output 走 data stream + ILM
  • Beat → Logstash 走 Kafka 緩衝 高峰場景
  • processors 在 Beat 端做輕量 filter，重 ETL 給 Logstash
  • Resource limit（容器中）：filebeat 100m CPU、512Mi RAM 起跳
  • Stack Monitoring 啟用 Beat 自我監控

本章重點回顧

  • Beats 家族：Filebeat（log）、Metricbeat（metrics）、Auditbeat、Packetbeat、Heartbeat、Winlogbeat
  • Modules 提供開箱即用的整合（80+ 個）：input + ingest pipeline + dashboard
  • Filebeat autodiscover 容器環境動態發現
  • Output 三選：直送 ES（推薦）、Logstash、Kafka 緩衝
  • 9.x Beats 預設 data stream 模式
  • Processors 通用：add_host_metadata、add_kubernetes_metadata、drop_event、script
  • Elastic Agent 是 9.x 主推取代多個 Beats
  • 選擇：新部署 Elastic Agent；既有 Beats 部署繼續用
