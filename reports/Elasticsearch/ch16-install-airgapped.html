適合對象 🔴 維運者 ⚡ 進階
閱讀時間 約 22 分鐘
前置知識 CH 06-15
本章目標 在無網際網路的環境部署完整 Elastic Stack；私有 registry mirror、離線授權、ECK air-gapped 模式

什麼是 Air-gapped 環境？

Air-gapped（氣隙）環境指的是網路與外部網際網路完全隔離的部署環境，常見於：

  • 政府、金融、軍工、醫療等高安全敏感場域
  • 合規要求（HIPAA、PCI-DSS、ISO 27001、ITAR）
  • OT/SCADA 工業控制網路
  • 金鑰管理區、機敏資料儲存區

挑戰：

  • 無法直接 apt install / dnf install / helm pull / docker pull
  • 授權檔需要離線發放（不能線上啟用）
  • Beats、Elastic Agent 的 integrations 需要離線同步
  • plugin、ML model、ingest pipeline 都需手動帶入

Step 1：建立離線資源清單

以下是一個典型 Stack（ES + Kibana + Beats + ECK）所需的全部 artifacts：

類型來源大小（估）
Elasticsearch tar.gz / deb / rpm / Docker imageartifacts.elastic.co~ 700 MB / image ~ 1.2 GB
Kibana 同上artifacts.elastic.co~ 1.5 GB（含內建 Chromium）
Logstash（如需）同上~ 600 MB
Beats / Elastic Agent同上~ 100 MB each
Plugins（如 analysis-icu、analysis-smartcn）artifacts.elastic.co~ 10-50 MB each
Elastic Package Registry imagedocker.elastic.co/package-registry~ 6-8 GB（含全部 integrations）
ECK Operator image + CRDs/operator.yamldocker.elastic.co/eck~ 200 MB
ML pre-trained models（如 ELSER、E5）需透過 Eland 工具離線匯入~ 500 MB - 2 GB each
Stack license file（自簽 basic 或購買的訂閱檔）basic 自動產生；訂閱由 Elastic 提供< 10 KB

Step 2：在連網區下載所有 artifacts

Elasticsearch / Kibana / Logstash

VERSION=9.3.4
mkdir -p offline-bundle
cd offline-bundle

# Linux x86_64 + ARM64
for ARCH in linux-x86_64 linux-aarch64; do
  for COMP in elasticsearch kibana logstash; do
    wget "https://artifacts.elastic.co/downloads/${COMP}/${COMP}-${VERSION}-${ARCH}.tar.gz"
    wget "https://artifacts.elastic.co/downloads/${COMP}/${COMP}-${VERSION}-${ARCH}.tar.gz.sha512"
  done
done

# Debian / RPM
for ARCH in amd64 arm64; do
  wget "https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-${VERSION}-${ARCH}.deb"
  wget "https://artifacts.elastic.co/downloads/kibana/kibana-${VERSION}-${ARCH}.deb"
done
for ARCH in x86_64 aarch64; do
  wget "https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-${VERSION}-${ARCH}.rpm"
  wget "https://artifacts.elastic.co/downloads/kibana/kibana-${VERSION}-${ARCH}.rpm"
done

Docker images export

cd offline-bundle && mkdir docker-images && cd docker-images

# 拉取所有需要的 image
for img in \
  "docker.elastic.co/elasticsearch/elasticsearch:${VERSION}" \
  "docker.elastic.co/kibana/kibana:${VERSION}" \
  "docker.elastic.co/logstash/logstash:${VERSION}" \
  "docker.elastic.co/beats/filebeat:${VERSION}" \
  "docker.elastic.co/beats/metricbeat:${VERSION}" \
  "docker.elastic.co/elastic-agent/elastic-agent:${VERSION}" \
  "docker.elastic.co/package-registry/distribution:${VERSION}" \
  "docker.elastic.co/eck/eck-operator:3.3.2"; do

  docker pull $img
  fname=$(echo $img | tr '/:' '_')
  docker save $img -o "${fname}.tar"
  gzip "${fname}.tar"
done

ls -lh

Plugins

mkdir -p offline-bundle/plugins
cd offline-bundle/plugins

for plugin in analysis-icu analysis-smartcn analysis-kuromoji repository-s3 repository-gcs repository-azure; do

ECK manifest 下載

Helm charts（如使用）

Step 3：傳輸到 Air-gapped 區

產生 SHA256 清單作為驗證：

cd offline-bundle
find . -type f -exec sha256sum {} \; > MANIFEST.sha256

# 打包
tar -czf elastic-airgapped-${VERSION}.tar.gz .
sha256sum elastic-airgapped-${VERSION}.tar.gz > elastic-airgapped-${VERSION}.tar.gz.sha256

透過資安規範允許的傳輸方式（USB、跨網閘檔案傳輸、刻錄光碟等）帶入 air-gapped 區。

Step 4：在內部建立 Mirror

Docker images 推到內部 Registry

假設內部已有 Harbor / Nexus / Artifactory：

INTERNAL_REGISTRY=registry.internal.example.com

cd offline-bundle/docker-images

for tarball in *.tar.gz; do
  gunzip -k $tarball
  tar_file="${tarball%.gz}"

  # 載入到本機 docker
  docker load -i $tar_file

  # 從檔案名解析原始 image name
  original=$(docker load -i $tar_file 2>&1 | grep "Loaded image" | awk '{print $3}')

  # 改 tag 指向內部 registry
  internal_tag="${INTERNAL_REGISTRY}/${original#docker.elastic.co/}"

  docker tag $original $internal_tag
  docker push $internal_tag

  echo "Pushed: $internal_tag"
done

套件 Repository Mirror

用 aptly（Debian）、createrepo（RHEL）或 JFrog Artifactory 建立內部 repo：

# Debian/Ubuntu：用 aptly
aptly repo create -distribution=stable -component=main elastic-9
aptly repo add elastic-9 offline-bundle/elasticsearch-9.3.4-amd64.deb
aptly repo add elastic-9 offline-bundle/kibana-9.3.4-amd64.deb
aptly publish repo elastic-9
# Aptly 會啟動 HTTP server 服務該 repo

# RHEL：用 createrepo_c
mkdir -p /var/www/html/elastic/9
cp offline-bundle/*.rpm /var/www/html/elastic/9/
createrepo_c /var/www/html/elastic/9/
# 設 nginx 對外服務

內網 client 改用內部 repo：

# Debian
echo "deb [trusted=yes] http://aptly.internal.example.com/ stable main" | \
  sudo tee /etc/apt/sources.list.d/elastic-internal.list
sudo apt update

# RHEL
sudo tee /etc/yum.repos.d/elastic-internal.repo << EOF
[elastic-internal]
name=Internal Elastic Mirror
baseurl=http://repo.internal.example.com/elastic/9/
enabled=1
gpgcheck=0
EOF

Step 5：Elasticsearch 離線安裝

後續流程與 CH 07-08 完全相同，只是改從內部來源取得：

# Debian 從內部 repo 安裝
sudo apt install -y elasticsearch=9.3.4

# 或直接安裝離線檔
sudo dpkg -i elasticsearch-9.3.4-amd64.deb

Step 6：ECK 在 Air-gapped 環境

改用內部 registry 的 image

ECK Operator 預設拉 docker.elastic.co/elasticsearch/elasticsearch:9.3.4。在 air-gapped 必須讓它從內部 registry 拉：

方式 1：修改 Operator Deployment

# operator.yaml 中找到 elastic-operator Deployment
spec:
  template:
    spec:
      containers:
        - name: manager
          image: registry.internal.example.com/eck/eck-operator:3.3.2
          args:
            - "manager"
            - "--container-registry=registry.internal.example.com"
            - "--container-suffix="

方式 2：Helm values

# eck-values.yaml
image:
  repository: registry.internal.example.com/eck/eck-operator
  tag: 3.3.2

config:
  containerRegistry: "registry.internal.example.com"

# 內部 registry 若需 imagePullSecret
imagePullSecrets:
  - name: registry-creds

helm install elastic-operator eck-operator-3.3.2.tgz \
  -n elastic-system --create-namespace \
  -f eck-values.yaml

Elasticsearch CR 也需指定內部 image

apiVersion: elasticsearch.k8s.elastic.co/v1
kind: Elasticsearch
metadata:
  name: production
spec:
  version: 9.3.4
  image: registry.internal.example.com/elasticsearch/elasticsearch:9.3.4
  nodeSets:
    - name: default
      count: 3
      # ...

或在 ECK Operator 的 --container-registry 已設好的情況下，會自動 prefix。

Step 7：離線授權檔安裝

Basic（免費）授權

9.x 啟動時自動產生 self-signed basic license，無需操作。Basic 已涵蓋大多數核心功能（搜尋、聚合、ILM、基本安全）。

Platinum / Enterprise 付費訂閱

從 Elastic 取得 license.json（透過 sales contact 或 support portal），離線安裝：

curl -k -u elastic:$PASSWORD \
  -X PUT "https://localhost:9200/_license" \
  -H 'Content-Type: application/json' \
  -d @license.json

# 確認
curl -k -u elastic:$PASSWORD https://localhost:9200/_license

Step 8：Elastic Package Registry（EPR）

Fleet / Elastic Agent 的 integrations（如 nginx、kafka、windows、aws...）由 EPR 提供。在 air-gapped 必須內部部署 EPR。

部署內部 EPR

docker run -d \
  --name epr \
  -p 8443:8080 \
  registry.internal.example.com/package-registry/distribution:9.3.4

K8s 用 ECK 的 PackageRegistry CR：

apiVersion: packageregistry.k8s.elastic.co/v1alpha1
kind: PackageRegistry
metadata:
  name: epr
  namespace: elastic
spec:
  version: 9.3.4
  image: registry.internal.example.com/package-registry/distribution:9.3.4
  count: 1

設定 Kibana 用內部 EPR

# kibana.yml
xpack.fleet.registryUrl: "https://epr.internal.example.com:8443"

# Kibana 9.x 也支援的 air-gapped flag
xpack.fleet.isAirGapped: true

Step 9：離線匯入 ML Models

ELSER（Elastic Learned Sparse EncodeR）、E5 等向量搜尋模型，預設由 Kibana 從 ml-models.elastic.co 下載。Air-gapped 需用 Eland 工具：

在連網區下載並匯出

pip install eland
docker pull docker.elastic.co/eland/eland

# 用 eland 直接從 HuggingFace 匯入到 ES（連網區）
eland_import_hub_model \
  --hub-model-id sentence-transformers/all-MiniLM-L6-v2 \
  --task-type text_embedding \
  --es-username elastic \
  --es-password "$PASSWORD" \
  --url https://es-online:9200 \
  --insecure

# 從 online ES 把 model 匯出（透過 reindex 或 snapshot）

離線匯入

另一個方式：在連網區用 eland 把 model 包成可直接 import 的 tar：

eland_export_model \
  --model-id sentence-transformers__all-minilm-l6-v2 \
  --output ./models/

# 把 ./models 目錄帶入 air-gapped 環境
# 在 air-gapped 環境用 eland 匯入
eland_import_hub_model \
  --model-path ./models/ \
  --es-username elastic \
  --es-password "$PASSWORD" \
  --url https://es-airgapped:9200

Step 10：離線 Fleet Server

Fleet Server 是 Elastic Agent 的中央管理元件。在 air-gapped 環境同樣不能連 elastic.co，必須：

  1. 使用內部 registry 的 elastic-agent image
  2. 設定 Kibana 的 fleet 為 air-gapped 模式（上節）
  3. Agent enrollment 時用內部 Fleet Server 位址
docker run -d \
  --name fleet-server \
  --net host \
  -e FLEET_SERVER_ENABLE=1 \
  -e FLEET_SERVER_ELASTICSEARCH_HOST=https://es-airgapped:9200 \
  -e FLEET_URL=https://fleet-server.internal.example.com:8220 \
  -e FLEET_SERVER_SERVICE_TOKEN=$SERVICE_TOKEN \
  registry.internal.example.com/elastic-agent/elastic-agent:9.3.4

Air-gapped 升級流程

每次升級都重複完整流程：

  1. 連網區下載新版本所有 artifacts
  2. 透過跨網閘傳輸到 air-gapped 區
  3. 推上內部 Mirror / Registry
  4. 逐一升級：先 ES → Kibana → Beats / Agent → 升級 ECK Operator（若用）
  5. 若有付費授權：確認新版本仍在訂閱有效期
⚠️ 規劃緩衝 Air-gapped 升級前，必須在連網的 staging 環境完整跑過一遍升級流程，確認沒有相容性問題後再進入 production。

常見問題

症狀處理
Kibana log 出現 connection refused 至 epr.elastic.co未設 xpack.fleet.registryUrl 或內部 EPR 未啟動
ECK Operator 拉 image 失敗沒設 --container-registry 或 imagePullSecret 未配
Plugin install 失敗 unknown plugin需用 install file:///path/to/plugin.zip 而非 plugin name
授權過期從 Elastic 取得新 license.json，PUT 到 /_license
Telemetry 嘗試外連kibana.yml 設 telemetry.enabled: false
ML model download 失敗用 Eland 離線匯入

本章重點回顧

  • Air-gapped 部署需要在連網區下載完整 artifacts，透過跨網閘傳輸後推上內部 Mirror
  • 內部 Mirror 包含三類：Docker registry（Harbor/Nexus）、套件 repo（aptly/createrepo）、ECK manifest
  • ECK Operator 的 --container-registry 旗標讓所有子資源自動使用內部 registry
  • 授權：basic 自動產生；付費訂閱用 PUT /_license 離線安裝
  • Elastic Package Registry 在 air-gapped 必須內部部署，並設 xpack.fleet.registryUrl
  • ML models 用 Eland 離線匯入
  • 每次升級都需重新走完整流程；建議有 staging 環境先驗證
  • 關閉 telemetry 避免外連嘗試
