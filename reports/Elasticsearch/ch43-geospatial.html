圖 43-1：geo_point vs geo_shape 用途對比
型別儲存查詢適用
geo_point單一座標（lat, lon）geo_distance、geo_bounding_boxPOI、user 位置、車輛追蹤

geo_point 設定

PUT cafes
{
  "mappings": {
    "properties": {
      "name":     { "type": "keyword" },
      "location": { "type": "geo_point" }
    }
  }
}

輸入格式（5 種）

POST cafes/_doc
{
  "name": "Cafe A",
  "location": { "lat": 25.033, "lon": 121.564 }    // 物件
}

POST cafes/_doc
{
  "name": "Cafe B",
  "location": [ 121.564, 25.033 ]                   // [lon, lat] 陣列（GeoJSON 順序！）
}

POST cafes/_doc
{
  "name": "Cafe C",
  "location": "25.033,121.564"                      // "lat,lon" 字串
}

POST cafes/_doc
{
  "name": "Cafe D",
  "location": "POINT (121.564 25.033)"              // WKT (lon lat)
}

POST cafes/_doc
{
  "name": "Cafe E",
  "location": "drm3btev3e86"                         // geohash
}
⚠️ 陣列格式是 [lon, lat] 不是 [lat, lon] GeoJSON 規範：經度在前、緯度在後。物件格式則用 lat / lon 鍵明確。新人容易踩這個坑。

Geo Queries

geo_distance：找半徑內

{
  "query": {
    "bool": {
      "filter": [
        {
          "geo_distance": {
            "distance": "10km",
            "location": { "lat": 25.033, "lon": 121.564 }
          }
        }
      ]
    }
  }
}

支援單位：km、mi、m、ft、nmi、yd、cm。

geo_bounding_box：方框內

{
  "query": {
    "geo_bounding_box": {
      "location": {
        "top_left":     { "lat": 25.10, "lon": 121.50 },
        "bottom_right": { "lat": 25.00, "lon": 121.60 }
      }
    }
  }
}

geo_polygon（已棄用，改用 geo_shape）

9.x 已棄用 geo_polygon query；改用 geo_shape with relation=intersects。

geo_shape Query

{
  "query": {
    "geo_shape": {
      "location": {
        "shape": {
          "type": "Polygon",
          "coordinates": [[
            [121.50, 25.00],
            [121.60, 25.00],
            [121.60, 25.10],
            [121.50, 25.10],
            [121.50, 25.00]
          ]]
        },
        "relation": "intersects"
      }
    }
  }
}

Shape Relations（geo_shape 欄位的查詢關係）

Relation含意
intersects（預設）有交集即 match
disjoint完全沒交集
within文件 shape 完全包含於查詢 shape
contains文件 shape 完全包含查詢 shape

Pre-indexed Shape：避免重傳大型多邊形

# 先存 shape 到單獨 index
PUT regions
{
  "mappings": { "properties": { "geometry": { "type": "geo_shape" } } }
}

POST regions/_doc/taipei-city
{
  "geometry": {
    "type": "Polygon",
    "coordinates": [ ... 大量座標 ... ]
  }
}

# 查詢時引用
GET cafes/_search
{
  "query": {
    "geo_shape": {
      "location": {
        "indexed_shape": {
          "index": "regions",
          "id":    "taipei-city",
          "path":  "geometry"
        },
        "relation": "intersects"
      }
    }
  }
}

geo_shape 詳設定

{
  "mappings": {
    "properties": {
      "service_area": {
        "type": "geo_shape",
        "orientation": "ccw",            // counter-clockwise（GeoJSON 標準）
        "ignore_malformed": false,
        "ignore_z_value": true,           // 忽略 3D Z 值
        "coerce": false
      }
    }
  }
}

支援的 Shape Types

  • Point
  • MultiPoint
  • LineString
  • MultiLineString
  • Polygon
  • MultiPolygon
  • GeometryCollection
  • Envelope（矩形，ES 自家擴充）
  • Circle（搜尋時用，索引時需轉為多邊形）

距離排序

{
  "query": { "match_all": {} },
  "sort": [
    {
      "_geo_distance": {
        "location":      { "lat": 25.033, "lon": 121.564 },
        "order":         "asc",
        "unit":          "km",
        "distance_type": "arc",        // 或 plane（更快但較不準）
        "mode":          "min"          // 多值欄位的取距方式
      }
    }
  ]
}

輸出每筆會帶 sort: [距離值]，可顯示給使用者。

取得距離當欄位

{
  "_source": { "includes": [ "name" ] },
  "runtime_mappings": {
    "distance_km": {
      "type": "double",
      "script": {
        "source": "
          if (doc['location'].size() == 0) { emit(0); return; }
          GeoPoint p = doc['location'].value;
          double d = GeoUtils.arcDistance(params.lat, params.lon, p.lat, p.lon);
          emit(d / 1000);
        ",
        "params": { "lat": 25.033, "lon": 121.564 }
      }
    }
  },
  "fields": [ "distance_km" ]
}

Geo Aggregations

geo_bounds：找邊界

{
  "size": 0,
  "aggs": {
    "viewport": {
      "geo_bounds": { "field": "location" }
    }
  }
}
// 輸出：top_left + bottom_right，可用作地圖視窗

geo_centroid：找中心

{
  "size": 0,
  "aggs": {
    "by_district": {
      "terms": { "field": "district" },
      "aggs": {
        "center": { "geo_centroid": { "field": "location" } }
      }
    }
  }
}

geo_distance：依距離分組

{
  "size": 0,
  "aggs": {
    "by_distance": {
      "geo_distance": {
        "field": "location",
        "origin": { "lat": 25.033, "lon": 121.564 },
        "unit":   "km",
        "ranges": [
          { "to":   1, "key": "near" },
          { "from": 1, "to": 5, "key": "medium" },
          { "from": 5,           "key": "far" }
        ]
      }
    }
  }
}

geohash_grid：熱點網格

{
  "size": 0,
  "aggs": {
    "heatmap": {
      "geohash_grid": {
        "field":     "location",
        "precision": 6,            // 1-12，越大越精細（也越多 bucket）
        "size":      1000
      }
    }
  }
}

用於 Kibana Maps 的 heatmap layer。precision 對應網格大小：

Precision網格邊長（赤道）
1~5000 km
3~156 km
5~4.9 km
7~153 m
9~4.8 m
12~3.7 cm

geotile_grid：依 Tile 分組

對應地圖 z/x/y tile 系統，與前端地圖庫天然相容：

{
  "aggs": {
    "tiles": {
      "geotile_grid": {
        "field":     "location",
        "precision": 11        // zoom level
      }
    }
  }
}
// 輸出 key 格式："11/1707/870"（z/x/y）

geohex_grid（9.x 強化）

用 H3 hexagon 系統分組，比方形 tile 更均勻：

{
  "aggs": {
    "hexes": {
      "geohex_grid": {
        "field":     "location",
        "precision": 7
      }
    }
  }
}

geo_line：軌跡聚合（9.x）

把同一 entity 的多個 geo_point 串成線（適合車輛軌跡、user trace）：

{
  "size": 0,
  "aggs": {
    "by_user": {
      "terms": { "field": "user_id" },
      "aggs": {
        "track": {
          "geo_line": {
            "point": { "field": "location" },
            "sort":  { "field": "@timestamp" },
            "size":  10000
          }
        }
      }
    }
  }
}

Vector Tile Search API（Kibana Maps 用）

9.x 提供 _mvt endpoint 直接回傳 Mapbox Vector Tile，前端地圖直接渲染：

GET cafes/_mvt/location/11/1707/870
{
  "query": {
    "term": { "category": "coffee" }
  },
  "grid_precision": 8,
  "exact_bounds":   true,
  "extent":         4096,
  "buffer":         5
}

回傳 binary protobuf，前端 Mapbox / MapLibre / Leaflet 可直接消化。

Kibana Maps 簡介

Kibana 內建地圖視覺化（Stack Management → Maps）：

  • 多 layer：base map + document + heatmap + cluster
  • 支援所有 ES geo aggregation
  • 可整合 ES|QL（9.x）
  • EMS（Elastic Maps Service）提供高品質 base map 與行政區邊界
  • 離線部署可自架 EMS

最佳實踐

  • POI / 行動資料用 geo_point，最快
  • 區域 / 行政邊界用 geo_shape
  • 多邊形大量重複用 indexed_shape，避免每次重傳
  • filter context + geo_distance 是最常見模式
  • distance_type=arc 短距離準；plane 大範圍掃略快
  • geo_polygon query 已棄用，改 geo_shape with relation=intersects
  • 地圖視覺化用 geohex_grid，比方形更美觀
  • 軌跡資料用 geo_line aggregation 而非自己串
  • Kibana Maps + _mvt endpoint 是高效互動地圖架構

本章重點回顧

  • geo_point（單點）vs geo_shape（複雜幾何）
  • geo_point 5 種輸入格式；陣列 [lon, lat] 是 GeoJSON 順序
  • geo_distance 半徑、geo_bounding_box 方框；geo_polygon query 已棄用
  • geo_shape 支援 4 種 relation：intersects（預設）、within、contains、disjoint
  • Pre-indexed shape 避免重傳大型多邊形
  • 距離排序用 _geo_distance sort
  • Geo aggregations：geo_bounds、geo_centroid、geo_distance、geohash_grid、geotile_grid、geohex_grid
  • 9.x geo_line aggregation 處理軌跡資料
  • _mvt API 直接回傳 Mapbox Vector Tile，前端地圖無痛整合
