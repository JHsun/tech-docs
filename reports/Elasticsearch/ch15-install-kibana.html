Kibana 簡介

Kibana 是 Elastic Stack 的視覺化與管理介面。9.x 提供：

  • Discover：互動式查詢與資料探索
  • Dashboard / Lens / Canvas：視覺化儀表板
  • Maps：地理空間分析
  • Stack Management：使用者、角色、ILM、ingest pipelines、index management
  • Dev Tools / Console：互動式 ES API 測試
  • Observability / Security / Search 解決方案 UI
  • ML / Alerting / Cases
💡 版本必須一致 Kibana 版本必須與 Elasticsearch 完全相符（major.minor.patch）。例如 ES 9.3.4 要配 Kibana 9.3.4。降級或跨小版號連線會失敗。

首次安裝：Enrollment Token 流程

從 Elasticsearch 8.x 起，Kibana 與 ES 的首次連線採 enrollment token 機制：

sequenceDiagram
    autonumber
    participant ES as Elasticsearch
    participant Op as 操作者
    participant KB as Kibana

    Note over ES: 首次啟動已自動產生 token
    Op->>ES: bin/elasticsearch-create-enrollment-token -s kibana
    ES-->>Op: 印出 token（30 分鐘有效）
    Op->>KB: 啟動 Kibana
    KB-->>Op: 顯示 enrollment URL & 6 位驗證碼
    Op->>KB: 在瀏覽器開啟 URL，貼上 token
    KB->>ES: 用 token 註冊自己
取得 kibana_system 帳號 + CA 憑證
    KB-->>Op: 提示輸入 6 位驗證碼確認
    Op->>KB: 輸入驗證碼
    KB->>ES: 連線成功

# 在 ES 機器上執行
sudo /usr/share/elasticsearch/bin/elasticsearch-create-enrollment-token -s kibana

安裝方式 1：Debian / Ubuntu APT

# 1. 確認已新增 Elastic APT repo（CH 08 已做的話跳過）
wget -qO - https://artifacts.elastic.co/GPG-KEY-elasticsearch | \
  sudo gpg --dearmor -o /usr/share/keyrings/elasticsearch-keyring.gpg

echo "deb [signed-by=/usr/share/keyrings/elasticsearch-keyring.gpg] https://artifacts.elastic.co/packages/9.x/apt stable main" | \
  sudo tee /etc/apt/sources.list.d/elastic-9.x.list

sudo apt update

# 2. 安裝
sudo apt install -y kibana=9.3.4

# 3. 鎖版本
sudo apt-mark hold kibana

# 4. 啟動
sudo systemctl daemon-reload
sudo systemctl enable kibana
sudo systemctl start kibana
sudo systemctl status kibana

安裝方式 2：RHEL / Rocky / Alma DNF

sudo dnf install --enablerepo=elasticsearch kibana-9.3.4
sudo dnf versionlock add kibana

sudo systemctl daemon-reload
sudo systemctl enable kibana
sudo systemctl start kibana

安裝方式 3：tar.gz（Linux/macOS）

KB_VERSION=9.3.4
ARCH=linux-x86_64   # 或 darwin-aarch64

cd /tmp
wget https://artifacts.elastic.co/downloads/kibana/kibana-${KB_VERSION}-${ARCH}.tar.gz
wget https://artifacts.elastic.co/downloads/kibana/kibana-${KB_VERSION}-${ARCH}.tar.gz.sha512
sha512sum -c kibana-${KB_VERSION}-${ARCH}.tar.gz.sha512

sudo mkdir -p /opt/elastic
sudo tar -xzf kibana-${KB_VERSION}-${ARCH}.tar.gz -C /opt/elastic
sudo ln -s /opt/elastic/kibana-${KB_VERSION} /opt/elastic/kibana
sudo useradd -r -s /sbin/nologin kibana
sudo chown -R kibana:kibana /opt/elastic/kibana-${KB_VERSION}

# 前景啟動
sudo -u kibana /opt/elastic/kibana/bin/kibana

systemd unit（搭配 tar.gz）

# /etc/systemd/system/kibana.service
[Unit]
Description=Kibana
Documentation=https://www.elastic.co
Wants=network-online.target
After=network-online.target

[Service]
Type=simple
User=kibana
Group=kibana
PrivateTmp=true
Environment=KBN_PATH_CONF=/etc/kibana
ExecStart=/opt/elastic/kibana/bin/kibana --silent
Restart=always
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target

安裝方式 6：ECK Kibana CRD

ECK 環境只需建立 Kibana CR：

apiVersion: kibana.k8s.elastic.co/v1
kind: Kibana
metadata:
  name: production
  namespace: elastic
spec:
  version: 9.3.4
  count: 2
  elasticsearchRef:
    name: production
    namespace: elastic
  http:
    tls:
      selfSignedCertificate:
        subjectAltNames:
          - dns: kibana.example.com
  config:
    server.publicBaseUrl: "https://kibana.example.com"
  podTemplate:
    spec:
      containers:
        - name: kibana
          resources:
            requests: { memory: 1Gi, cpu: 500m }
            limits:   { memory: 2Gi, cpu: 1 }

ECK 自動處理：

  • 建立 kibana_system 使用者並注入密碼
  • 取用 ES 的 CA 憑證並信任
  • 產生 self-signed HTTPS 憑證
  • 滾動升級時與 ES 版本同步

kibana.yml 重要設定

套件安裝後位置：/etc/kibana/kibana.yml
tar.gz：$KIBANA_HOME/config/kibana.yml

# ─── Server 設定 ───────────────────────
server.host: "0.0.0.0"
server.port: 5601
server.publicBaseUrl: "https://kibana.example.com"
server.name: "kibana-prod-1"
server.maxPayload: 1048576    # 1 MB；上傳大型 saved object 時調高

# ─── Elasticsearch 連線 ────────────────
elasticsearch.hosts: ["https://es-1:9200", "https://es-2:9200", "https://es-3:9200"]
elasticsearch.username: "kibana_system"
# 密碼建議用 keystore 而非明文
# elasticsearch.password: "..."   ← 不要這樣寫
elasticsearch.ssl.certificateAuthorities: ["/etc/kibana/elasticsearch-ca.pem"]
elasticsearch.ssl.verificationMode: certificate

# ─── HTTPS（強烈建議） ─────────────────
server.ssl.enabled: true
server.ssl.certificate: /etc/kibana/server.crt
server.ssl.key: /etc/kibana/server.key

# ─── Logging ───────────────────────────
logging.appenders.file.type: file
logging.appenders.file.fileName: /var/log/kibana/kibana.log
logging.appenders.file.layout.type: json
logging.root.level: info
logging.root.appenders: ["file", "default"]

# ─── Encryption Keys（生產必設） ───────
# 至少 32 字元的隨機字串，建議 keystore
# xpack.security.encryptionKey: "..."
# xpack.encryptedSavedObjects.encryptionKey: "..."
# xpack.reporting.encryptionKey: "..."

# ─── Telemetry ─────────────────────────
telemetry.enabled: false
telemetry.optIn: false

# ─── i18n ──────────────────────────────
i18n.locale: "zh-TW"   # 或 en, ja, fr 等

kibana-keystore：安全儲存設定

類似 elasticsearch-keystore，避免明文密碼：

sudo /usr/share/kibana/bin/kibana-keystore create

# 加入 ES 密碼
sudo /usr/share/kibana/bin/kibana-keystore add elasticsearch.password
# 系統提示輸入密碼

# 加入加密金鑰
sudo /usr/share/kibana/bin/kibana-keystore add xpack.security.encryptionKey
sudo /usr/share/kibana/bin/kibana-keystore add xpack.encryptedSavedObjects.encryptionKey
sudo /usr/share/kibana/bin/kibana-keystore add xpack.reporting.encryptionKey

# 列出
sudo /usr/share/kibana/bin/kibana-keystore list
🚨 加密金鑰一旦設定，不能改變 xpack.encryptedSavedObjects.encryptionKey 用於加密 saved object（如 alerting rules、connector credentials）。一旦改變，舊的 saved objects 就無法解密。生產環境一定要：
  1. 用 strong random 32+ 字元金鑰
  2. 備份金鑰到安全處（HashiCorp Vault、Secret Manager 等）
  3. 所有 Kibana 副本使用相同金鑰

kibana_system 內建使用者

Kibana 不應該用 elastic superuser 連 ES，而要用內建的 kibana_system：

# 重設 kibana_system 密碼
sudo /usr/share/elasticsearch/bin/elasticsearch-reset-password -u kibana_system

# 把密碼存進 keystore
sudo /usr/share/kibana/bin/kibana-keystore add elasticsearch.password

kibana_system 已被授予最小必要權限（管 .kibana 索引、讀 cluster state、讀 monitoring 資料等）。

設定 HTTPS（自簽）

用 elasticsearch-certutil 產生憑證

# 在 ES 機器上產生 server cert
sudo /usr/share/elasticsearch/bin/elasticsearch-certutil cert \
  --ca /etc/elasticsearch/certs/http_ca.crt \
  --ca-key /etc/elasticsearch/certs/http_ca.key \
  --pem \
  --name kibana \
  --dns kibana.example.com \
  --out /tmp/kibana-certs.zip

unzip /tmp/kibana-certs.zip -d /tmp/kibana-certs
sudo cp /tmp/kibana-certs/kibana/kibana.crt /etc/kibana/server.crt
sudo cp /tmp/kibana-certs/kibana/kibana.key /etc/kibana/server.key
sudo cp /etc/elasticsearch/certs/http_ca.crt /etc/kibana/elasticsearch-ca.pem
sudo chown kibana:kibana /etc/kibana/server.* /etc/kibana/elasticsearch-ca.pem
sudo chmod 0640 /etc/kibana/server.key

kibana.yml 啟用

server.ssl.enabled: true
server.ssl.certificate: /etc/kibana/server.crt
server.ssl.key: /etc/kibana/server.key

elasticsearch.ssl.certificateAuthorities: ["/etc/kibana/elasticsearch-ca.pem"]
elasticsearch.ssl.verificationMode: full

用 Reverse Proxy（Nginx）擋在 Kibana 前

常見模式：Nginx 處理 TLS 終止 + IP 限制 + Auth。Kibana 自身可關 TLS（信任 Nginx 與 Kibana 之間是 trusted network）：

upstream kibana {
    server 10.0.1.20:5601;
    server 10.0.1.21:5601;
    keepalive 32;
}

server {
    listen 443 ssl http2;
    server_name kibana.example.com;

    ssl_certificate     /etc/letsencrypt/live/kibana.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/kibana.example.com/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;

    # 限制 IP（選用）
    allow 10.0.0.0/8;
    deny all;

    location / {
        proxy_pass http://kibana;
        proxy_http_version 1.1;
        proxy_set_header Connection "Keep-Alive";
        proxy_set_header Proxy-Connection "Keep-Alive";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }
}

Kibana 端對應設定：

server.publicBaseUrl: "https://kibana.example.com"
server.rewriteBasePath: false

多副本高可用部署

Kibana 是 stateless（state 全部存在 ES 的 .kibana 索引），可隨意水平擴展。生產建議至少 2 副本搭配 Load Balancer：

所有 Kibana 副本必須：

  • 使用相同的 encryption keys（從 keystore 同步）
  • 連到同一個 ES 叢集
  • 版本完全一致

升級 Kibana

⚠️ 升級順序：先 ES 後 Kibana 始終先升級 Elasticsearch 至全部完成，再升級 Kibana。Kibana 不能比 ES 高版本。 
# 套件方式（生產）
sudo apt-mark unhold kibana
sudo systemctl stop kibana
sudo apt install kibana=9.3.5
sudo systemctl start kibana
sudo apt-mark hold kibana

# 多副本：依序升級每個節點，驗證可用後再下一個

常見問題排查

症狀處理
Kibana server is not ready yet等 30-60 秒；若持續，看 log 找 ES 連線錯誤
Unable to authenticate user確認 kibana_system 密碼正確；keystore 有設定

本章重點回顧

  • Kibana 必須與 Elasticsearch 版本完全一致
  • 首次連線採 enrollment token（30 分鐘有效）+ 6 位驗證碼
  • 用 kibana-keystore 存放 ES 密碼與 encryption keys，避免明文於 yml
  • xpack.encryptedSavedObjects.encryptionKey 一旦設定不可改，否則 saved objects 無法解密
  • 所有副本使用相同 encryption keys；建議至少 2 副本 + Load Balancer 高可用
  • 用 kibana_system 內建使用者連 ES，不要用 elastic superuser
  • 啟用 server.ssl，或在前面加 Nginx/ALB 處理 TLS
  • 升級順序：先升 ES 全部完成，再升 Kibana
