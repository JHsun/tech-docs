🚦 D. Cluster Health 顏色解讀

狀態
意義
處理
🟢 GREEN
所有 primary + replica 都已分配
正常運行；可放心執行任何操作
🟡 YELLOW
所有 primary 都好，部分 replica 缺
寫讀都正常，但容錯降低。最常見原因：節點數不夠、磁碟 watermark、單節點 cluster
🔴 RED
至少一個 primary unassigned
部分資料無法存取。立即跑 allocation/explain；嚴重時從 snapshot 還原

🔍 E. Allocation Explain（找 unassigned 原因）

當 shard unassigned 時，這個 API 是最重要的診斷工具。

情境
命令 / Code
說明
找任意 unassigned
GET /_cluster/allocation/explain?pretty
無參數時自動找一個
查特定 shard
POST /_cluster/allocation/explain?pretty
{
  "index":   "my-index",
  "shard":   0,
  "primary": true
}
指定 index + shard + primary/replica
含磁碟資訊
GET /_cluster/allocation/explain?include_disk_info=true&pretty
看每節點 disk 容量資料
💡 解讀回應：看 allocate_explanation（人話原因）+ node_allocation_decisions[].deciders[]（每節點被拒原因）。常見 decider：disk_threshold、same_shard、filter、awareness、max_retry、throttling。

⚠️ F. 常見 Unassigned 原因 + 修復

原因 (unassigned_info.reason)
意義
修復方式
NODE_LEFT
某節點離線了（網路/重啟）
預設等 1 分鐘 (index.unassigned.node_left.delayed_timeout) 後才重分配。等節點回來最佳
NO_VALID_SHARD_COPY
所有 shard copy 都壞 / 找不到
從 snapshot 還原；無 snapshot 時用 allocate_empty_primary（資料丟失）
ALLOCATION_FAILED
max retries 達到（預設 5）
修原因後跑 POST /_cluster/reroute?retry_failed=true
CLUSTER_RECOVERED
剛重啟，等 gateway recovery
等就好；確認 dangling indices
DISK_THRESHOLD
磁碟達 watermark
清空間 / 加節點 / 調 watermark（見下節）
FILTER
index 設了 routing.allocation.include/exclude 但無匹配 node
檢查 GET /my-index/_settings 的 index.routing.allocation，與 GET /_cat/nodeattrs?v 對照
AWARENESS
zone 數不滿足 awareness 設定
加足夠 zone 的節點，或暫關 awareness
SAME_HOST
單節點 cluster 仍要求 replica
設 PUT /my-index/_settings 的 "number_of_replicas": 0
INDEX_CREATED
剛建 index 但 mapping/setting 衝突
看 explain 的 deciders，常是 filter 不匹配

💧 G. Disk Watermark 設定

情境
命令 / Code
說明
當前 watermark
GET /_cluster/settings?include_defaults=true&flat_settings=true
  &filter_path=*.cluster.routing.allocation.disk*
看設定值與預設值
調整 watermark
PUT /_cluster/settings
{
  "persistent": {
    "cluster.routing.allocation.disk.watermark.low":         "92%",
    "cluster.routing.allocation.disk.watermark.high":        "95%",
    "cluster.routing.allocation.disk.watermark.flood_stage": "97%"

PUT /_all/_settings
{
  "index.blocks.read_only_allow_delete": null
}
清空間後解除 read-only block。否則 index 會持續無法寫

📊 H. Watermark 行為（重要！）

Watermark
觸發行為
影響
low (預設 85%)
不分配新 shard 到此節點
開始 YELLOW，新 index 分配受限
high (預設 90%)
嘗試移走此節點的 shard
shard 開始遷移，IO 增加
flood_stage (預設 95%)
對該節點上的 index 設 read_only_allow_delete
寫入失敗！必須清空間 + 主動解鎖

🔀 I. Reroute 指令（強制重分配）

情境
命令 / Code
說明
關閉 shard 分配
PUT /_cluster/settings
{
  "persistent": {
    "cluster.routing.allocation.enable": "primaries"
  }
}
滾動升級前關。值：all / primaries / new_primaries / none
恢復分配
PUT /_cluster/settings
{
  "persistent": {
    "cluster.routing.allocation.enable": null
  }
}
null 重置為預設 (all)
重試失敗 shard
POST /_cluster/reroute?retry_failed=true
達 max_retries 的 shard 再試一次
強制移動 shard
POST /_cluster/reroute
{
  "commands": [
    {
      "move": {
        "index":     "my-index",
        "shard":     0,
        "from_node": "node-1",
        "to_node":   "node-2"
      }
    }
  ]
}
把熱 shard 從擁擠 node 移走
放棄 primary（資料遺失！）
POST /_cluster/reroute
{
  "commands": [
    {
      "allocate_empty_primary": {
        "index":  "broken-index",
        "shard":  0,
        "node":   "node-3",
        "accept_data_loss": true
      }
    }
  ]
}
最後手段。執行前先 snapshot 嘗試保留
用 stale primary（資料舊）
POST /_cluster/reroute
{
  "commands": [
    {
      "allocate_stale_primary": {
        "index":  "my-index",
        "shard":  0,
        "node":   "node-3",
        "accept_data_loss": true
      }
    }
  ]
}
使用舊版本的 shard 副本（部分資料）
⚠️ accept_data_loss=true 會永久丟失該 shard 全部資料。 執行前先嘗試從 snapshot 還原（CS 10）。

💾 場景：磁碟快滿

  1. GET /_cat/allocation?v 找出哪個節點滿
  2. 用 GET /_cat/indices?v&s=store.size:desc&bytes=gb 找最大 index
  3. 判斷：可刪 → DELETE /old-index；不可刪 → 加節點 / 加 disk / 移 cold tier
  4. 若已觸發 flood_stage：PUT /_all/_settings { "index.blocks.read_only_allow_delete": null }
  5. 長期：設 ILM（CS 03）

🔄 場景：要做滾動重啟

  1. 關閉 shard 分配（避免 shard 飛來飛去）：
    PUT /_cluster/settings { "persistent": { "cluster.routing.allocation.enable": "primaries" } }
  2. flush translog：POST /_flush
  3. 停止節點 → 操作 → 啟動
  4. 等節點 join：GET /_cat/nodes?v
  5. 恢復分配：
    PUT /_cluster/settings { "persistent": { "cluster.routing.allocation.enable": null } }
  6. 等 GREEN：GET /_cluster/health?wait_for_status=green&timeout=30m
  7. 下一節點重複

(function () {
  function init() {
    document.querySelectorAll('main pre').forEach(function (pre) {
      if (pre.parentElement.classList.contains('code-wrap')) return;
      const wrap = document.createElement('div');
      wrap.className = 'code-wrap';
      pre.parentNode.insertBefore(wrap, pre);
      wrap.appendChild(pre);
      const lang = pre.querySelector('code')?.className.match(/language-(\w+)/)?.[1];
      if (lang) {
        const tag = document.createElement('span');
        tag.className = 'code-lang';
        tag.textContent = lang;
        wrap.appendChild(tag);
      }
      const btn = document.createElement('button');
      btn.className = 'copy-btn';
      btn.type = 'button';
      btn.textContent = '📋 複製';
      btn.addEventListener('click', function () {
        const text = pre.innerText;
        if (navigator.clipboard?.writeText) {
          navigator.clipboard.writeText(text).then(success).catch(fallback);
        } else {
          fallback();
        }
        function success() {
          btn.textContent = '✓ 已複製';
          btn.classList.add('copied');
          setTimeout(function () { btn.textContent = '📋 複製'; btn.classList.remove('copied'); }, 1800);
        }
        function fallback() {
          const ta = document.createElement('textarea');
          ta.value = text;
          document.body.appendChild(ta);
          ta.select();
          try { document.execCommand('copy'); success(); } catch (_) {}
          document.body.removeChild(ta);
        }
      });
      wrap.appendChild(btn);
    });
  }
  if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
  else init();
})();

// Auto-build TOC from h2/h3 in 
    
(function () {
  function init() {
    const toc = document.querySelector('.toc ul');
    if (!toc) return;
    const headings = document.querySelectorAll('main h2, main h3');
    headings.forEach(function (h) {
      if (!h.id) {
        h.id = h.textContent.trim().toLowerCase()
          .replace(/[^\w\u4e00-\u9fff\s-]/g, '')
          .replace(/\s+/g, '-');
      }
      const li = document.createElement('li');
      const a = document.createElement('a');
      a.href = '#' + h.id;
      a.textContent = h.textContent;
      a.className = 'toc-' + h.tagName.toLowerCase();
      li.appendChild(a);
      toc.appendChild(li);
    });
  }
  if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
  else init();
})();

        loopTextColor: isDark ? '#e6f2f8' : '#0f172a',
        noteBkgColor: isDark ? '#3a4f3a' : '#fef3c7',
        noteBorderColor: isDark ? '#86c598' : '#d97706',
        noteTextColor: isDark ? '#e6f2f8' : '#0f172a',
      },
      flowchart: { useMaxWidth: true, htmlLabels: true, curve: 'basis' },
      sequence: { useMaxWidth: true, showSequenceNumbers: false, mirrorActors: true, bottomMarginAdj: 10 },
    };
  }

  let renderCounter = 0;
  async function renderAll() {
    if (typeof mermaid === 'undefined') return;
    renderCounter++;
    const myCounter = renderCounter;
    mermaid.initialize(getMermaidConfig());
    const blocks = document.querySelectorAll('.mermaid');
    for (const el of blocks) {
      // restore source if already rendered
      if (el.dataset.source) el.innerHTML = el.dataset.source;
      else el.dataset.source = el.innerHTML;
      el.removeAttribute('data-processed');
    }
    if (myCounter !== renderCounter) return;
    try {
      await mermaid.run({ querySelector: '.mermaid' });
    } catch (e) { console.warn('Mermaid render error', e); }
  }

  function init() {
    renderAll();
    window.addEventListener('jhsun:theme-changed', renderAll);
  }

  if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
  else init();
})();
