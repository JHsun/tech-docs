CH 07 · Linux/macOS（tar.gz）安裝

適合對象 🟢 初學者 🔴 維運者
閱讀時間 約 25 分鐘
前置知識 CH 06 · 系統需求
本章目標 透過 tar.gz 完成首次安裝、目錄結構解析、systemd 整合、首次啟動與安全驗證

為什麼選 tar.gz 安裝？

Elastic 提供多種安裝方式（Debian/RPM 套件、Docker、ECK），但 tar.gz 有幾個獨特優勢：

  • 不需 root：可以解壓到任意目錄、用普通 user 啟動
  • 多版本並存：在同一台機器上跑不同版本（測試用）
  • 客製化路徑：自由指定 data、log、config 位置
  • 跨發行版相容：同一個 tar.gz 在 RHEL、Ubuntu、SLES 都能用
  • 適合 macOS（Apple Silicon 與 Intel）開發環境

缺點：需要自行管理 systemd unit、log rotation 等；生產環境通常用套件（CH 08）較方便維護。

Step 1：下載發行版

到 官方下載頁取得最新版本連結，或直接用以下命令（Linux x86_64）：

cd /tmp
ES_VERSION=9.3.4
ARCH=linux-x86_64

# 下載主檔與 sha512 checksum
wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-${ES_VERSION}-${ARCH}.tar.gz
wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-${ES_VERSION}-${ARCH}.tar.gz.sha512

# 驗證 checksum
sha512sum -c elasticsearch-${ES_VERSION}-${ARCH}.tar.gz.sha512
# 應顯示：elasticsearch-9.3.4-linux-x86_64.tar.gz: OK

不同平台變體

平台檔名
Linux x86_64elasticsearch-9.3.4-linux-x86_64.tar.gz
Linux aarch64（ARM64）elasticsearch-9.3.4-linux-aarch64.tar.gz
macOS x86_64（Intel）elasticsearch-9.3.4-darwin-x86_64.tar.gz
macOS aarch64（Apple Silicon）elasticsearch-9.3.4-darwin-aarch64.tar.gz

PGP 簽章驗證（建議）

# 匯入 Elastic 公鑰
wget https://artifacts.elastic.co/GPG-KEY-elasticsearch
gpg --import GPG-KEY-elasticsearch

# 下載簽章檔
wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-${ES_VERSION}-${ARCH}.tar.gz.asc

# 驗證
gpg --verify elasticsearch-${ES_VERSION}-${ARCH}.tar.gz.asc \
  elasticsearch-${ES_VERSION}-${ARCH}.tar.gz
# 應看到 "Good signature from "Elasticsearch (Elasticsearch Signing Key) ..."

Step 2：解壓並建立系統 user

# 建議的目錄佈局（可自行調整）
sudo mkdir -p /opt/elastic
sudo tar -xzf /tmp/elasticsearch-9.3.4-linux-x86_64.tar.gz -C /opt/elastic
sudo ln -s /opt/elastic/elasticsearch-9.3.4 /opt/elastic/elasticsearch

# 建立執行 user（不可登入）
sudo useradd -r -s /sbin/nologin -d /opt/elastic/elasticsearch elasticsearch

# 建立資料與日誌目錄
sudo mkdir -p /var/lib/elasticsearch /var/log/elasticsearch /etc/elasticsearch

# 將 config 移到 /etc 統一管理（推薦做法）
sudo mv /opt/elastic/elasticsearch/config/* /etc/elasticsearch/
sudo rmdir /opt/elastic/elasticsearch/config
sudo ln -s /etc/elasticsearch /opt/elastic/elasticsearch/config

# 設定 ownership
sudo chown -R elasticsearch:elasticsearch \
  /opt/elastic/elasticsearch-9.3.4 \
  /var/lib/elasticsearch \
  /var/log/elasticsearch \
  /etc/elasticsearch

Step 3：解析 Elasticsearch 目錄結構

解壓後的目錄結構：

/opt/elastic/elasticsearch-9.3.4/
├── bin/                  # 執行檔（elasticsearch、elasticsearch-plugin、elasticsearch-keystore...）
├── config/               # 設定檔（→ /etc/elasticsearch）
│   ├── elasticsearch.yml
│   ├── jvm.options
│   ├── log4j2.properties
│   ├── role_mapping.yml
│   ├── roles.yml
│   ├── users
│   ├── users_roles
│   └── jvm.options.d/
├── jdk/                  # 內建 OpenJDK
├── lib/                  # JAR 檔
├── modules/              # 內建模組（kibana、ingest-common、aggregations...）
├── plugins/              # 自訂 plugins（首次為空）
└── LICENSE.txt / NOTICE.txt / README.asciidoc

關鍵路徑變數

變數典型值說明
ES_HOME/opt/elastic/elasticsearch安裝根目錄
ES_PATH_CONF/etc/elasticsearch設定檔路徑
ES_TMPDIR/tmp/elasticsearch臨時目錄
path.data/var/lib/elasticsearch資料目錄（在 elasticsearch.yml 設定）
path.logs/var/log/elasticsearch日誌目錄

Step 4：撰寫 elasticsearch.yml

編輯 /etc/elasticsearch/elasticsearch.yml：

# ─── Cluster ────────────────────────────
cluster.name: my-cluster
node.name: ${HOSTNAME}

# ─── Paths ──────────────────────────────
path.data: /var/lib/elasticsearch
path.logs: /var/log/elasticsearch

# ─── Network ────────────────────────────
network.host: 0.0.0.0
http.port: 9200
transport.port: 9300

# ─── Discovery ──────────────────────────
# 單節點開發
discovery.type: single-node

# 或多節點（生產）：
# discovery.seed_hosts: ["node-1", "node-2", "node-3"]
# cluster.initial_master_nodes: ["node-1", "node-2", "node-3"]   # 首次啟動後移除

# ─── Memory ─────────────────────────────
bootstrap.memory_lock: true

Step 5：套用 OS 調整

套用 CH 06 的 OS tuning script。重點：

sudo sysctl -w vm.max_map_count=262144
echo 'vm.max_map_count=262144' | sudo tee /etc/sysctl.d/99-elasticsearch.conf

# 關閉 swap
sudo swapoff -a
sudo sed -i.bak '/ swap / s/^\(.*\)$/#\1/g' /etc/fstab

Step 6：建立 systemd unit

建立 /etc/systemd/system/elasticsearch.service：

[Unit]
Description=Elasticsearch
Documentation=https://www.elastic.co
Wants=network-online.target
After=network-online.target

[Service]
Type=notify
NotifyAccess=all
RuntimeDirectory=elasticsearch
PrivateTmp=true
Environment=ES_HOME=/opt/elastic/elasticsearch
Environment=ES_PATH_CONF=/etc/elasticsearch
Environment=PID_DIR=/var/run/elasticsearch
WorkingDirectory=/opt/elastic/elasticsearch

User=elasticsearch
Group=elasticsearch

ExecStart=/opt/elastic/elasticsearch/bin/elasticsearch -p ${PID_DIR}/elasticsearch.pid

# 必要 limits（產品環境）
LimitNOFILE=65535
LimitNPROC=4096
LimitMEMLOCK=infinity
LimitFSIZE=infinity
LimitAS=infinity

# 給足夠時間啟動（大叢集 shard recovery 可能花很久）
TimeoutStartSec=180

# 收到 SIGTERM 時優雅關閉
KillSignal=SIGTERM
KillMode=process
SendSIGKILL=no
SuccessExitStatus=143

[Install]
WantedBy=multi-user.target

sudo systemctl daemon-reload
sudo systemctl enable elasticsearch

Step 7：首次啟動

sudo systemctl start elasticsearch

# 觀察 log（首次啟動需 30-60 秒）
sudo journalctl -u elasticsearch -f
# 或直接看 ES 自己的 log
sudo tail -f /var/log/elasticsearch/my-cluster.log

Step 8：取得密碼並測試連線

若初始密碼忘了，可重設：

# 重設 elastic 密碼為自動產生
sudo /opt/elastic/elasticsearch/bin/elasticsearch-reset-password -u elastic
# 或互動式設定密碼
sudo /opt/elastic/elasticsearch/bin/elasticsearch-reset-password -u elastic -i

測試連線：

curl -X GET "https://localhost:9200" \
  -u elastic \
  --cacert /etc/elasticsearch/certs/http_ca.crt
# 系統會提示輸入密碼

macOS 專屬說明

Homebrew vs tar.gz

macOS 開發者常見三種選擇：

  1. Homebrew（最方便，但 Elastic 已不再官方維護 brew formula；社群版可能落後）
  2. tar.gz（推薦）— 同 Linux 流程，但無 systemd
  3. Docker（推薦給跨平台一致性）

macOS tar.gz 安裝

# Apple Silicon
ARCH=darwin-aarch64
# Intel
# ARCH=darwin-x86_64

curl -O https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-9.3.4-${ARCH}.tar.gz
tar -xzf elasticsearch-9.3.4-${ARCH}.tar.gz
cd elasticsearch-9.3.4

# 直接前景啟動（適合開發測試）
./bin/elasticsearch

macOS 不需要 vm.max_map_count 設定（不適用），其他 OS limit 一般也預設足夠。

macOS launchd 整合（背景執行）

建立 ~/Library/LaunchAgents/co.elastic.elasticsearch.plist：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>co.elastic.elasticsearch</string>
  <key>ProgramArguments</key>
  <array>
    <string>/Users/me/elasticsearch-9.3.4/bin/elasticsearch</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>/Users/me/elasticsearch.log</string>
  <key>StandardErrorPath</key>
  <string>/Users/me/elasticsearch.err</string>
</dict>
</plist>

launchctl load ~/Library/LaunchAgents/co.elastic.elasticsearch.plist
launchctl list | grep elastic

elasticsearch-keystore：安全存放敏感設定

對於 password、API token 等敏感字串，不要明文寫在 elasticsearch.yml。改用 keystore：

sudo /opt/elastic/elasticsearch/bin/elasticsearch-keystore create
sudo /opt/elastic/elasticsearch/bin/elasticsearch-keystore add s3.client.default.access_key
# 系統會提示輸入值
sudo /opt/elastic/elasticsearch/bin/elasticsearch-keystore list

Keystore 路徑：$ES_PATH_CONF/elasticsearch.keystore。它會被自動 reload，無需重啟。

升級與重啟

重啟單一節點

sudo systemctl restart elasticsearch
sudo systemctl status elasticsearch

升級到新 patch（例如 9.3.4 → 9.3.5）

滾動升級流程（CH 54 詳述）：

# 1. 停用 shard allocation
curl -X PUT "https://localhost:9200/_cluster/settings" \
  -u elastic --cacert /etc/elasticsearch/certs/http_ca.crt \
  -H 'Content-Type: application/json' \
  -d '{ "persistent": { "cluster.routing.allocation.enable": "primaries" } }'

# 2. flush
curl -X POST "https://localhost:9200/_flush" \
  -u elastic --cacert /etc/elasticsearch/certs/http_ca.crt

# 3. 停止節點
sudo systemctl stop elasticsearch

# 4. 解壓新版本到並列目錄
cd /tmp && wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-9.3.5-linux-x86_64.tar.gz
sudo tar -xzf elasticsearch-9.3.5-linux-x86_64.tar.gz -C /opt/elastic
sudo chown -R elasticsearch:elasticsearch /opt/elastic/elasticsearch-9.3.5

# 5. 切換 symlink
sudo ln -sfn /opt/elastic/elasticsearch-9.3.5 /opt/elastic/elasticsearch
sudo rm /opt/elastic/elasticsearch-9.3.5/config
sudo ln -s /etc/elasticsearch /opt/elastic/elasticsearch-9.3.5/config

# 6. 啟動
sudo systemctl start elasticsearch

# 7. 等節點 join 後恢復 allocation
curl -X PUT "https://localhost:9200/_cluster/settings" \
  -u elastic --cacert /etc/elasticsearch/certs/http_ca.crt \
  -H 'Content-Type: application/json' \
  -d '{ "persistent": { "cluster.routing.allocation.enable": null } }'

解除安裝

sudo systemctl stop elasticsearch
sudo systemctl disable elasticsearch
sudo rm /etc/systemd/system/elasticsearch.service
sudo systemctl daemon-reload

# 移除檔案（注意先備份重要資料！）
sudo rm -rf /opt/elastic
sudo rm -rf /var/lib/elasticsearch
sudo rm -rf /var/log/elasticsearch
sudo rm -rf /etc/elasticsearch

# 移除 user
sudo userdel elasticsearch

常見問題排查

問題 1：Permission denied / cannot write to data dir

sudo chown -R elasticsearch:elasticsearch /var/lib/elasticsearch /var/log/elasticsearch
ls -la /var/lib/elasticsearch

問題 2：bootstrap checks failed

仔細看 log，把每條失敗訊息對照 CH 06 的修正方式。最常見：

  • max file descriptors [4096] for elasticsearch process is too low → 編輯 systemd unit 加 LimitNOFILE=65535
  • max virtual memory areas vm.max_map_count [65530] is too low → sudo sysctl -w vm.max_map_count=262144
  • memory locking requested but cannot be locked → 加 LimitMEMLOCK=infinity

問題 3：Address already in use（9200 / 9300）

sudo ss -tnlp | grep -E '9200|9300'
# 找到佔用 process 後 kill 或改用其他 port

問題 4：OutOfMemoryError 啟動時

檢查 /etc/elasticsearch/jvm.options.d/，可能 heap 設太大超過實體 RAM。預設讓 ES 自動估算（不要寫 -Xms/-Xmx）。

本章重點回顧

  • tar.gz 安裝適合多版本並存與客製化路徑；生產環境通常仍用套件
  • 建立 dedicated elasticsearch user 並設定資料/日誌目錄 ownership
  • config 統一放 /etc/elasticsearch，data 放 /var/lib/elasticsearch
  • 撰寫 systemd unit，設定 LimitNOFILE / LimitMEMLOCK / TimeoutStartSec
  • 首次啟動會自動產生 TLS 憑證、密碼、enrollment tokens
  • 用 elasticsearch-keystore 安全存放敏感設定，不要明文於 yml
  • macOS 開發環境建議直接 tar.gz 或 Docker，不需 OS 調整
