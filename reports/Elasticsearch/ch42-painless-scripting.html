Painless 是什麼？

Painless 是 Elasticsearch 自家的沙盒化 Java-like 腳本語言。預設且唯一啟用的腳本語言（取代過去的 Groovy、JavaScript）：

  • 類 Java 語法，學習曲線低
  • 沙盒化（不能存取檔案系統、網路）
  • 編譯為 JVM bytecode，比 interpret 語言快
  • 有 caching：同一 script 第二次執行極快

Painless 基本語法

變數與型別

int count = 10;
double price = 99.99;
String name = "Alice";
boolean active = true;
def anything = 42;          // def = dynamic, 任意型別

// 集合

Map meta = ['key': 'value', 'count': 3];

條件與迴圈

// if/else
if (price > 100) {
  ctx.tier = "premium";
} else if (price > 50) {
  ctx.tier = "mid";
} else {
  ctx.tier = "low";
}

// 三元
ctx.tier = price > 100 ? "premium" : "regular";

// for
for (int i = 0; i < items.size(); i++) {
  // ...
}

// foreach

  // ...
}

// while

}

字串操作

String s = "Hello World";
s.length();                  // 11
s.toUpperCase();             // "HELLO WORLD"

s.trim();

// 格式化（手動，無 sprintf）

四種 Script Context

不同 context 有不同的可用變數與限制：

Context可用變數用途
Ingestctx.field_name轉換資料
Updatectx._source.field, ctx.op原地更新文件
Search/Query (filter)doc['field'].valueruntime field、條件、score
Reindexctx._source, ctx._id, ctx._index, ctx.opreindex 轉換

Ingest Context

PUT _ingest/pipeline/normalize
{
  "processors": [{
    "script": {
      "source": "
        // 標準化 email
        if (ctx.email != null) {
          ctx.email = ctx.email.toLowerCase().trim();
        }

        // 計算衍生欄位
        if (ctx.duration_ns != null) {
          ctx.duration_ms = ctx.duration_ns / 1000000.0;
        }

        // 條件 drop
        if (ctx.log?.level == 'DEBUG') {
          ctx.op = 'drop';
        }

        // tags 陣列
        if (ctx.tags == null) ctx.tags = new ArrayList();
        if (ctx.error_count > 0) ctx.tags.add('errored');
      "
    }
  }]
}

Ingest context 可：

  • 讀寫任何 ctx.field（即 _source）
  • 修改 ctx._index 動態路由
  • 設 ctx.op = 'drop' 丟棄文件

Update Context

POST products/_update/1
{
  "script": {
    "source": "
      ctx._source.view_count++;
      ctx._source.last_viewed = params.now;

      if (ctx._source.view_count >= 1000) {
        ctx._source.is_popular = true;
      }
    ",
    "params": { "now": "2026-04-15T10:00:00Z" }
  }
}

更新時可設 ctx.op：

  • 'index'（預設）：寫入
  • 'noop'：不變更（version 不增加）
  • 'delete'：刪除

Search Context

用 doc['field'].value 而非 ctx._source（doc values 比較快）：

Runtime field

{
  "runtime_mappings": {
    "price_with_tax": {
      "type": "double",
      "script": {
        "source": "emit(doc['price'].value * (1 + params.tax))",
        "params": { "tax": 0.05 }
      }
    },
    "day_of_week": {
      "type": "keyword",
      "script": {
        "source": "emit(doc['@timestamp'].value.dayOfWeek.toString())"
      }
    }
  },
  "query": { ... }
}

用 emit(...) 輸出值（runtime field 必須）。

Script Query（過濾）

{
  "query": {
    "script": {
      "script": {
        "source": "doc['price'].value * doc['quantity'].value > params.min",
        "params": { "min": 1000 }
      }
    }
  }
}

慢，盡量改寫為 runtime field + range query。

Function Score

{
  "query": {
    "function_score": {
      "query": { "match_all": {} },
      "script_score": {
        "script": {
          "source": "
            double recency = (params.now - doc['created_at'].value.toInstant().toEpochMilli()) / 86400000.0;
            return _score * Math.exp(-0.05 * recency);
          ",
          "params": { "now": 1735689600000 }
        }
      }
    }
  }
}

Reindex Context

POST _reindex
{
  "source": { "index": "old-orders" },
  "dest":   { "index": "new-orders" },
  "script": {
    "source": "
      ctx._source.amount_with_tax = ctx._source.amount * 1.05;

      // 動態路由
      if (ctx._source.region == 'asia') {
        ctx._index = 'asia-orders';
      }

      // 跳過 cancelled
      if (ctx._source.status == 'cancelled') {
        ctx.op = 'noop';
      }
    "
  }
}

Multi-value 欄位

// doc['tags'].value 會丟錯（多值）
// 用 .values 取陣列：
List tags = doc['tags'];
for (t in tags) { ... }
int n = doc['tags'].size();

// 或安全取第一個
String first = doc['tags'].size() > 0 ? doc['tags'].get(0) : null;

處理 missing 欄位

// 安全 navigation: ?.
if (ctx.user?.email != null) { ... }

// doc.containsKey
if (doc.containsKey('optional_field') && doc['optional_field'].size() > 0) {
  String val = doc['optional_field'].value;
}

// 用 ?: elvis
String email = ctx.user?.email ?: 'unknown';

Stored Scripts：可重用

把常用 script 存到 cluster，引用名字而非 inline：

# 註冊
PUT _scripts/calculate-tax
{
  "script": {
    "lang":   "painless",
    "source": "ctx._source.amount * (1 + params.rate)"
  }
}

# 使用
POST orders/_update/1
{
  "script": {
    "id":     "calculate-tax",
    "params": { "rate": 0.05 }
  }
}

# 在 ingest pipeline 中
PUT _ingest/pipeline/tax-pipeline
{
  "processors": [
    { "script": { "id": "calculate-tax", "params": { "rate": 0.05 } } }
  ]
}

好處：

  • script logic 集中管理
  • 更新一次就影響所有用到的地方
  • 同 source 只 compile 一次（cache）

效能與 Cache

編譯 cache

每個 script source 第一次執行會 compile 為 bytecode。預設 cache 大小：

script.cache.max_size: 200          # 預設
script.cache.expire:   "0s"

看 cache 狀況：

curl -u elastic:xxx --cacert ca.crt \
  "https://localhost:9200/_nodes/stats/script?human&pretty"

關注：compilations、cache_evictions、compilation_limit_triggered。

Compilation rate limit

同 source pattern 的不同 script 過多時觸發 limit（避免 cache thrashing）：

script.context.{ctx}.max_compilations_rate: 75/5m

解法：用 params 取代 inline 值：

// ❌ 每次不同 source（不能 cache）
"source": "ctx._source.price * 1.05"
"source": "ctx._source.price * 1.10"
"source": "ctx._source.price * 1.15"

// ✅ source 相同，只是 params 不同（cache hit）
"source": "ctx._source.price * params.rate"
"params": { "rate": 1.05 }

Debug 與測試

用 _explain 看 script_score

POST products/_explain/1
{
  "query": {
    "script_score": {
      "query": { "match_all": {} },
      "script": { "source": "doc['price'].value" }
    }
  }
}

在 script 中拋例外看值

// 開發時用：強制拋錯把值印出
Debug.explain(ctx._source);
throw new RuntimeException("price = " + ctx.price);

Kibana Painless Lab

Kibana → Dev Tools → Painless Lab：互動式編輯器，即時看執行結果。最佳開發工具。

沙盒安全性

Painless 嚴格限制：

  • 無檔案系統 / 網路 / process 啟動 API
  • 受限 reflection
  • 不能定義 class
  • 不能 import 任意 package
  • 有執行時間 timeout（預設 1s）

禁用 inline / stored script（極嚴格場景）：

script.allowed_types: stored        # 只允許 stored
script.allowed_contexts: search,update    # 限制 context

不該用 Script 的場景

  • 能用 query DSL/ESQL 達成：原生比 script 快
  • 大量 script_score 排序：每筆都跑，慢且不 cache
  • script_query 主過濾：用 runtime field + range 取代
  • 每個請求不同 source：cache miss，cluster 卡死

最佳實踐

  • 常數抽到 params，提升 cache 命中
  • 用 stored script 集中管理
  • doc['field'].value 而非 ctx._source.field（search context 中）
  • 避免 ?: 在熱路徑（會多分支判斷）
  • 用 Painless Lab 開發，先測試再 deploy
  • Runtime field 取代 script_query，可享 query optimizer
  • 監控 compilations rate，逼近 limit 即優化
  • Reindex 大量資料時，先用小 size 驗證 script 正確
  • 用 _simulate 測 ingest script

本章重點回顧

  • Painless = ES 內建沙盒化 Java-like 腳本語言，9.x 預設且唯一
  • 四大 context：ingest、update、search、reindex；變數與限制各異
  • Ingest 用 ctx.field；search 用 doc['field'].value（更快）
  • Runtime field 用 emit(...)，可在 mapping 或 search 動態定義
  • 常數一律抽到 params，避免 cache miss
  • Stored script 集中 logic，跨多處引用
  • 用 Kibana Painless Lab 開發測試
  • 能用 query DSL 就不用 script，效能 + cache 雙贏
  • 監控 _nodes/stats/script 的 compilations 數
