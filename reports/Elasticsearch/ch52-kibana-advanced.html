Discover：互動式探索

查 raw data 與快速分析的起點：

  • 選 data view（取代舊的 index pattern）
  • 用 KQL（Kibana Query Language）或 Lucene 查詢
  • 9.x 起可切換到 ES|QL 模式（CH 39）
  • 儲存查詢為 saved search
  • 欄位 sidebar：點欄位看 top values、histogram
  • 匯出 CSV（受 size 限制）

KQL 速查

// 欄位 = 值
service.name : "order-svc"

// 全文搜尋
"connection refused"

// 範圍
http.status_code >= 500
duration > 1000

// 多條件
service.name : "order-svc" and log.level : "error"
service.name : ("order-svc" or "auth-svc")

// 通配
host.name : prod-*

// 巢狀
user.address.city : "Taipei"

// exists / missing
service.name : *
not service.name : *

Data View（資料視圖）

對應一個或多個 index / data stream，含：

  • index pattern（如 logs-*）
  • 主要 time field（@timestamp）
  • scripted fields（runtime field 取代）
  • field formatters（顯示格式）
# API 建立 data view
POST kbn:/api/data_views/data_view
{
  "data_view": {
    "title":         "logs-*",
    "name":          "All Logs",
    "timeFieldName": "@timestamp"
  }
}

Lens：拖拉視覺化

Kibana 主推的視覺化工具，取代舊版 TSVB / Visualize：

  • 拖欄位到 X / Y / breakdown，自動推薦圖表類型
  • 支援：bar、line、area、pie、treemap、metric、gauge、table、heatmap、sankey
  • 多 layer：同一圖混合 metric / line / bar
  • 內建格式化、條件染色、annotations
  • 9.x 支援 ES|QL data source

Lens 技巧

  • 多 sub-bucket：把第二個欄位拖到 break down by
  • 百分比堆疊：bar chart → 100% stacking
  • Quick functions：count、unique count、sum、avg、median、percentile、cumulative_sum
  • Formula：自訂運算式，類似 Excel formulas（average(price) * 1.05）

Dashboard：組合與互動

把多個 visualization 組合成 dashboard。9.x 重點功能：

  • Filters：頂部加 filter，影響全 dashboard panel
  • Time picker：統一時間範圍
  • Drilldowns：點 panel 跳到 Discover / 其他 dashboard / URL
  • Controls：下拉選單、範圍滑桿，給使用者互動
  • Linked filters：cross-panel filtering（點 bar 過濾其他 panel）
  • 9.x ES|QL panel：直接寫 ES|QL 編輯
  • Embedded Lens：dashboard 中直接編輯 Lens chart

組織技巧

  • 分區：用 markdown panel 當 section header
  • 3-3-2 排版：每行 3 個 KPI、再 3 個趨勢圖、再 2 個 table
  • 金字塔：上方 high-level KPI，往下細節
  • 顏色一致：類似指標用相同調色

Canvas：自由排版報表

類 PowerPoint / Figma 的設計工具，用於精緻 KPI 牆、總監報表：

  • 自由排版：背景圖、標誌、形狀
  • 多頁支援（slideshow）
  • 用 SQL 或 ES|QL 取資料
  • 適合 NOC 大螢幕、CXO dashboard
  • 不像 dashboard 那樣強調互動，重視視覺呈現

Maps（CH 43 補充）

  • 多 layer 疊加：基底 + document layer + heatmap layer + cluster + line
  • 連到 ES geo_point / geo_shape
  • EMS（Elastic Maps Service）提供高品質 base map 與行政邊界
  • 支援 vector tile 從 ES _mvt 取資料
  • Geo containment alerting：實體進入區域觸發 alert

Spaces：多租戶分隔

把 Kibana 切成多個獨立 workspace，每個 space 各有自己的：

  • Saved objects（dashboards、views、saved searches）
  • Index patterns / data views
  • Settings
  • UI features 開關（隱藏不用的功能）

圖 52-2：Kibana Spaces 多租戶

建立 space

POST kbn:/api/spaces/space
{
  "id":           "marketing",
  "name":         "Marketing Team",
  "description":  "Marketing analytics",
  "color":        "#FF6B6B",
  "initials":     "MK",
  "disabledFeatures": [
    "fleet",
    "siem"
  ]
}

Role + Space 整合

PUT _security/role/marketing_user
{
  "indices": [
    {
      "names":      [ "marketing-*" ],
      "privileges": [ "read" ]
    }
  ],
  "applications": [
    {
      "application": "kibana-.kibana",
      "privileges":  [ "feature_discover.read", "feature_dashboard.all" ],
      "resources":   [ "space:marketing" ]
    }
  ]
}

使用者只能看到 marketing space。

Saved Objects 管理

所有 Kibana 物件（dashboards、visualizations、data views）都是 saved object。

匯出 / 匯入

# 匯出整個 dashboard（含所有相依物件）
curl -X POST -u elastic:xxx \
  -H "kbn-xsrf: true" \
  "https://kibana:5601/api/saved_objects/_export" -d '
{
  "type":               [ "dashboard" ],
  "objects":            [ { "type": "dashboard", "id": "abc123" } ],
  "includeReferencesDeep": true
}' > my-dashboard.ndjson

# 匯入
curl -X POST -u elastic:xxx \
  -H "kbn-xsrf: true" \
  -F file=@my-dashboard.ndjson \
  "https://kibana:5601/api/saved_objects/_import?overwrite=true"

把 Saved Objects 進 GitOps

把匯出的 ndjson 放 git，用 CI 自動部署到不同 cluster：

# GitHub Actions 範例
name: Deploy Kibana Objects
on:
  push:
    paths: [ 'kibana/**' ]
jobs:
  deploy:
    steps:
      - uses: actions/checkout@v4
      - run: |
          for f in kibana/*.ndjson; do
            curl -X POST -u $KB_USER:$KB_PWD \
              -H "kbn-xsrf: true" \
              -F file=@$f \
              "$KB_URL/api/saved_objects/_import?overwrite=true"
          done

Reporting：PDF / PNG / CSV

Kibana 內建 reporting：

  • PDF：dashboard 截圖（multi-page）
  • PNG：單一 panel 截圖
  • CSV：discover saved search 匯出
  • 排程：每日 / 每週 email PDF 給管理層
# 用 API 觸發
POST kbn:/api/reporting/generate/printablePdfV2
{
  "jobParams": "..."
}

9.x 推出 scheduled reports：UI 設定排程，自動 email。

Feature Controls（細粒度權限）

每 role 可控制：

  • Discover：read / all
  • Dashboard：read / all
  • Visualize / Lens：read / all
  • Canvas、Maps、Stack Monitoring 等各功能獨立
  • Dev Tools：限制給 power user
  • Stack Management：通常只 admin

Kibana 效能調校

Headless Chromium 設定

xpack.reporting.capture.browser.chromium.disableSandbox: false
xpack.reporting.queue.timeout: 5m
xpack.reporting.csv.maxSizeBytes: 50000000   # 50MB CSV 上限

高負載部署

  • 多 Kibana 實例 + load balancer
  • 每實例 2-4 vCPU + 4-8 GB RAM
  • 用 Redis cache for shared sessions（9.x 內建）
  • Reporting 走獨立 worker（pre-built browser image）

Kibana 自我監控

# Kibana status
curl -u elastic:xxx https://kibana:5601/api/status

# 9.x kibana metrics 透過 Stack Monitoring（CH 46）

關鍵指標：

  • response time（API endpoint）
  • concurrent connection 數
  • saved object 操作速率
  • reporting queue length

Dev Tools

Kibana 內建工具：

  • Console：寫 ES API（自動補全！必用）
  • Search Profiler：query / aggregation 視覺化效能分析
  • Painless Lab：互動式 Painless 編輯器（CH 42）
  • Grok Debugger：測試 grok pattern

最佳實踐

  • 用 Spaces 隔離團隊，配合 RBAC（CH 21）
  • Saved objects 進 GitOps，避免單點手刻
  • 每團隊一個 dashboard 模板，可 fork 個別客製
  • 用 Lens 而非 Visualize（後者已棄用）
  • 大 dashboard 拆成多個，用 drilldown 串連
  • data view + runtime field 取代舊 scripted field
  • Time picker 預設 15 分鐘，避免 user 不小心查太大
  • 關不用的 features（在 space 設定）：少干擾
  • 用 controls panel 給互動性，比 raw filter 易用
  • 大量 user 用多 Kibana 實例 + LB

本章重點回顧

  • Kibana 三大 analytics：Discover（探索）、Lens（視覺化）、Dashboard（組合）
  • Lens 已取代舊版 TSVB / Visualize；9.x 支援 ES|QL data source
  • Canvas：自由排版（適合 NOC、CXO 報表）
  • Spaces：多租戶分隔，配合 role 細粒度授權
  • Saved Objects 全部可 export / import，進 GitOps
  • Reporting：PDF / PNG / CSV，9.x 支援 scheduled reports
  • Feature Controls：細粒度功能權限
  • Dev Tools 必備：Console、Search Profiler、Painless Lab、Grok Debugger
  • 大規模部署：多 Kibana 實例 + LB
