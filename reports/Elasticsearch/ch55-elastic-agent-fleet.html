Elastic Agent 是什麼？

Elastic Agent 是單一統一的 agent，取代過去 Filebeat、Metricbeat、Auditbeat、Endpoint Security 等多個 process，並可由 Fleet 中央管理。9.x 是主推方向。

部署 Fleet Server

Fleet Server 是 Elastic Agent 的 control plane（用 Elastic Agent 自己跑）。

Step 1：在 ES 與 Kibana 啟用前置條件

# elasticsearch.yml
xpack.security.authc.api_key.enabled: true

# kibana.yml
xpack.fleet.agents.enabled: true
xpack.encryptedSavedObjects.encryptionKey: "至少 32 字元的隨機 key"

Step 2：在 Kibana 設定 Fleet

Fleet → Settings → 設 Fleet Server 的 host 與 ES output。

Step 3：產生 service token

curl -X POST -u elastic:xxx --cacert ca.crt \
  "https://localhost:9200/_security/service/elastic/fleet-server/credential/token/token1?pretty"

Step 4：安裝 Fleet Server（用 Elastic Agent）

# 在 Fleet Server 機器上
curl -L -O https://artifacts.elastic.co/downloads/beats/elastic-agent/elastic-agent-9.3.4-linux-x86_64.tar.gz
tar xzvf elastic-agent-9.3.4-linux-x86_64.tar.gz
cd elastic-agent-9.3.4-linux-x86_64

sudo ./elastic-agent install \
  --fleet-server-es=https://es:9200 \
  --fleet-server-service-token=AAEAAW... \
  --fleet-server-policy=fleet-server-policy \
  --fleet-server-port=8220 \
  --certificate-authorities=/path/to/ca.crt \
  --fleet-server-cert=/path/to/server.crt \
  --fleet-server-cert-key=/path/to/server.key

產生 url（如 https://fleet-server.example.com:8220）給其他 agent enroll。

Enroll Agent

Step 1：在 Fleet UI 取得 enrollment token

Fleet → Agents → Add agent → 選 policy → 複製命令

Step 2：在 host 上安裝

# Linux
curl -L -O https://artifacts.elastic.co/downloads/beats/elastic-agent/elastic-agent-9.3.4-linux-x86_64.tar.gz
tar xzvf elastic-agent-9.3.4-linux-x86_64.tar.gz
cd elastic-agent-9.3.4-linux-x86_64

sudo ./elastic-agent install \
  --url=https://fleet-server.example.com:8220 \
  --enrollment-token=AAEAAW... \
  --certificate-authorities=/path/to/ca.crt

Step 3：檢查狀態

sudo elastic-agent status
sudo elastic-agent diagnostics

Kibana → Fleet → Agents 應該看到該 host 出現。

Agent Policy

Policy = 一組 integrations + outputs + monitoring 設定。多個 agent 可共用同一 policy。

建立 Policy

Fleet → Agent policies → Create agent policy：

  • Name：如 web-servers、kubernetes-prod
  • Description
  • Default namespace：對應 data stream 的 namespace
  • Output：選 ES 或 Logstash output
  • Monitoring：log + metric 自我監控
  • Agent privileges：unprivileged 模式（9.x 強化）

新增 Integration

對 policy 加 integration（取代 Beat module）：

  1. Fleet → Integrations → 搜尋（如 nginx、kubernetes、mysql）
  2. 點 Add integration
  3. 選 policy
  4. 填 settings（log 路徑、port 等）
  5. Save

Fleet 自動：

  • 把 integration 加進 policy
  • 產生對應 ingest pipeline、index template、ILM policy
  • Push policy 給所有用此 policy 的 agent
  • 建立 Kibana dashboard

Managed vs Standalone

模式設定方式升級適合
Fleet-managedKibana UIFleet 中央升級多 agent / 中央管控
Standalone本機 yml個別升級少量、客製化、air-gapped

Standalone 設定

sudo ./elastic-agent install --develop
# 編輯 /etc/elastic-agent/elastic-agent.yml

outputs:
  default:
    type: elasticsearch
    hosts: [ "https://es:9200" ]
    api_key: "..."
    ssl.certificate_authorities: [ "/etc/elastic-agent/ca.crt" ]

inputs:
  - id: filestream-app
    type: filestream
    paths:
      - /var/log/app/*.log
    data_stream:
      dataset:   "app"
      namespace: "production"

  - id: system-metrics
    type: system/metrics
    data_stream:
      dataset:   "system.cpu"
      namespace: "production"
    streams:
      - metricsets: [ cpu, load, memory ]
        period: 10s

Kubernetes 部署

用官方 Helm chart 或 manifests：

helm repo add elastic https://helm.elastic.co
helm install elastic-agent elastic/elastic-agent \
  --set kubernetes.enabled=true \
  --set agent.fleet.enabled=true \
  --set agent.fleet.url=https://fleet-server:8220 \
  --set agent.fleet.token=AAEAAW...

會 deploy：

  • DaemonSet：每節點一個 agent（收 host metrics + log）
  • Deployment：1 個 agent（收 cluster-level metrics from kube-state-metrics）

升級

Fleet-managed

Fleet → Agents → 選 agent → Upgrade agents → 選版本。

支援批次升級、回退（downgrade）、排程。9.x Fleet 提供 maintenance window 控制升級時段。

Standalone

sudo elastic-agent upgrade 9.3.4

Output 選項

9.x Fleet 支援 multi-output：不同 integration 可送到不同 ES cluster：

  • Elasticsearch：直送 ES（最常用）
  • Logstash：經 Logstash 處理
  • Kafka：緩衝
  • Remote Elasticsearch：跨 cluster 送
  • EDOT/OTel：以 OTLP 送（CH 64）

Elastic Defend（Endpoint Security）

Elastic Agent 內建 Defend integration：EDR、惡意軟體偵測、行為分析。需 Platinum 訂閱。

啟用：Fleet → Integrations → Elastic Defend → Add to policy。包含：

  • Malware prevention（NGAV）
  • Ransomware protection
  • Memory threat protection
  • Behavior protection
  • Process / file / network event collection

事件直接送 ES，與 SIEM（CH 58）整合。

EDOT：Elastic Distribution for OpenTelemetry

9.x Elastic Agent 內含 EDOT Collector——OpenTelemetry collector 的 Elastic 發行版：

  • 支援 OTLP、Prometheus、Jaeger、Zipkin 等 receiver
  • 內建 Elastic-specific processor（如 trace processor）
  • 取代傳統 APM Server
  • 送 ES 走 OTLP 端點（9.x ES 內建）

典型配置（fleet integration）：選 OpenTelemetry → 設定 receivers / pipelines。

Agent Providers（動態變數）

Provider 提供動態元資料給 input config：

inputs:
  - type: filestream
    condition: ${kubernetes.namespace} == "production"
    paths: [ "/var/log/${kubernetes.pod.name}/*.log" ]

常用 providers：

  • kubernetes：節點 / pod / container 資訊
  • docker：container 資訊
  • host：主機資訊
  • env：環境變數
  • local_dynamic：本機動態檔案

Agent 自我監控

Policy 中啟用 monitoring → agent 自動收集：

  • Agent log（agent.log、各 input 的 log）
  • Agent metric（CPU、RAM、events/sec）
  • Output stats（events sent、failures）

透過 Stack Monitoring（CH 46）查看。

常見問題

Enroll 失敗

  • 檢查 host 能不能到 Fleet Server（curl https://fleet-server:8220/api/status）
  • CA cert 是否正確
  • enrollment token 過期？重新產生

Policy 沒套用

sudo elastic-agent inspect | head -50
sudo journalctl -u elastic-agent -f

Agent 顯示 offline

  • 檢查 elastic-agent process 是否還在
  • 網路到 Fleet Server / ES 的連線
  • API Key / token 沒過期

最佳實踐

  • 新部署用 Elastic Agent + Fleet，取代 Beats
  • Fleet Server 高可用：至少 2 個，背後 Load Balancer
  • 用多個 policy 分組：web-servers、k8s、databases，便於批次管理
  • Integrations 用官方版本，不要客製太多（升級會痛）
  • 啟用 monitoring，提早發現 agent 故障
  • 升級分批：先 1 個試，再 10%，再全部
  • Air-gapped 環境用 self-hosted Elastic Package Registry
  • Defend 啟用 取代第三方 EDR（合規較簡單）
  • EDOT collector 取代 APM Server（CH 56）
  • K8s 環境用 DaemonSet + Deployment，配合 hints autodiscover

本章重點回顧

  • Elastic Agent 是單一 process取代多個 Beats + Endpoint
  • Fleet 提供中央管理：policy、enrollment、升級
  • Fleet Server 是 control plane（用 Elastic Agent 自己跑），預設 8220 port
  • Policy = integrations + outputs + monitoring 配置
  • Integration 取代舊版 Beat module，由官方維護
  • Managed（Fleet）vs Standalone（本機 yml），新部署選前者
  • 9.x 內建 EDOT（OpenTelemetry distribution），可取代 APM Server
  • Elastic Defend 提供 EDR / 惡意軟體防護（Platinum）
  • Agent providers 提供 K8s / Docker 動態變數
