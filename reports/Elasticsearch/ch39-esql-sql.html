ES|QL 是什麼？

Elasticsearch Query Language (ES|QL) 是 Elastic 推出的管線式（piped）查詢語言，靈感來自 Splunk SPL、Kibana KQL、Linux pipe。9.1 起全 GA。

第一個 ES|QL Query

curl -X POST -u elastic:xxx --cacert ca.crt \
  -H 'Content-Type: application/json' \
  "https://localhost:9200/_query?pretty" -d '
{
  "query": "
    FROM logs-*
    | WHERE @timestamp > NOW() - 1 HOUR
    | WHERE log.level == \"error\"
    | STATS count = COUNT(*) BY service.name
    | SORT count DESC
    | LIMIT 10
  "
}'

輸出（columnar 形式）：

{
  "took": 42,
  "columns": [
    { "name": "count",         "type": "long" },
    { "name": "service.name",  "type": "keyword" }
  ],
  "values": [
    [ 4523, "order-service" ],
    [ 3421, "auth-service" ],
    ...
  ]
}

ES|QL 核心命令

FROM：source 命令

FROM logs-app-2026.04
FROM logs-*, metrics-*           // 多個 patterns
FROM logs-app | METADATA _id, _index, _score   // 帶 metadata 欄位

WHERE / EVAL：過濾與計算

FROM logs-*
| WHERE log.level IN (\"error\", \"warn\")
| EVAL hour_of_day = DATE_EXTRACT(\"HOUR_OF_DAY\", @timestamp)
| EVAL response_seconds = response_ms / 1000.0
| WHERE response_seconds > 1

STATS：聚合

FROM logs-*
| STATS
    error_count = COUNT(*),
    avg_rt      = AVG(response_ms),
    p95_rt      = PERCENTILE(response_ms, 95),
    unique_users = COUNT_DISTINCT(user.id)
  BY service.name, log.level

支援聚合函式：COUNT、SUM、AVG、MIN、MAX、MEDIAN、PERCENTILE、COUNT_DISTINCT、VALUES、TOP、STD_DEV 等。

SORT / LIMIT

| SORT @timestamp DESC, service.name ASC
| LIMIT 100

KEEP / DROP / RENAME

| KEEP @timestamp, service.name, response_ms     // 只留這些欄位
| DROP host.architecture, debug_*                 // 刪除欄位（支援 wildcard）
| RENAME service.name AS svc, response_ms AS rt

DISSECT / GROK：字串解析

FROM raw-logs
| DISSECT message \"%{ts} [%{level}] %{msg}\"
| EVAL ts = TO_DATETIME(ts)
| KEEP ts, level, msg

// GROK 用 named patterns
FROM nginx-logs
| GROK message \"%{IP:client_ip} - %{USER:user} \\[%{HTTPDATE:ts}\\] \\\"%{WORD:method} %{URIPATHPARAM:url}\\\" %{INT:status} %{INT:size}\"

ENRICH：以 enrich policy 補資料

// 先建 enrich policy（一次性）
PUT /_enrich/policy/ip-to-geo
{
  "match": {
    "indices":     "ip-geo-db",
    "match_field": "ip",
    "enrich_fields": [ "country", "city", "asn" ]
  }
}
POST /_enrich/policy/ip-to-geo/_execute

// ES|QL 查詢中使用
FROM logs-*
| ENRICH ip-to-geo ON client_ip
| STATS count = COUNT(*) BY country

LOOKUP JOIN（9.1 GA）

查詢時動態 join 另一個 index：

FROM logs-*
| WHERE @timestamp > NOW() - 1 HOUR
| LOOKUP JOIN service-registry ON service.name
| STATS error_rate = COUNT(*) BY service.name, service.owner
| SORT error_rate DESC
| LIMIT 20

service-registry 必須建為 lookup mode index：

PUT service-registry
{
  "settings": {
    "index.mode": "lookup"
  },
  "mappings": {
    "properties": {
      "service.name":  { "type": "keyword" },
      "service.owner": { "type": "keyword" }
    }
  }
}

Lookup mode 的 index 會 broadcast 到所有節點，便於 join 時就地查詢。

常用 Functions

字串

| EVAL upper_msg = TO_UPPER(msg)
| EVAL trimmed   = TRIM(msg)
| EVAL contains  = CONTAINS(msg, \"error\")
| EVAL parts     = SPLIT(url, \"/\")
| EVAL prefix    = SUBSTRING(msg, 0, 50)
| EVAL replaced  = REPLACE(msg, \"foo\", \"bar\")
| EVAL concat    = CONCAT(host, \":\", TO_STRING(port))

日期

| EVAL hour_of_day = DATE_EXTRACT(\"HOUR_OF_DAY\", @timestamp)
| EVAL day_bucket  = DATE_TRUNC(1 day, @timestamp)
| EVAL formatted   = DATE_FORMAT(\"yyyy-MM-dd\", @timestamp)
| EVAL parsed      = DATE_PARSE(\"yyyy-MM-dd HH:mm:ss\", str_field)
| EVAL diff_hours  = DATE_DIFF(\"hours\", start, end)

數值

| EVAL rounded = ROUND(value, 2)
| EVAL absolute = ABS(value)
| EVAL ceiling  = CEIL(value)
| EVAL pi       = PI()

條件

| EVAL category = CASE(
    response_ms < 100, \"fast\",
    response_ms < 500, \"medium\",
    \"slow\"
  )

| EVAL is_error = response_status >= 500
| EVAL valid    = COALESCE(field1, field2, \"default\")

ES|QL Vector Search（9.x 新）

KNN search

FROM products
| WHERE category == \"electronics\"
| KNN product_embedding QUERY VECTOR [0.1, 0.2, ...] K 10
| SORT _score DESC
| LIMIT 10

Vector similarity functions

FROM products
| EVAL similarity = V_COSINE(product_embedding, [0.1, 0.2, ...])
| WHERE similarity > 0.7
| SORT similarity DESC
| LIMIT 20

支援：V_COSINE、V_DOT_PRODUCT、V_L2_NORM、V_L1_NORM、V_HAMMING。

TEXT_EMBEDDING（9.3+）

讓 ES 自動產生 embedding 再做 KNN：

FROM products
| WHERE in_stock == true
| KNN product_embedding QUERY TEXT_EMBEDDING(\"wireless headphones\", \"my-inference-endpoint\") K 10

需先建 inference endpoint（CH 60 RAG）。

Cross-cluster Query（9.1 GA）

FROM europe:logs-*, asia:logs-*, logs-*
| WHERE @timestamp > NOW() - 1 HOUR
| STATS count = COUNT(*) BY _cluster, service.name
| SORT count DESC

用 {cluster}:{index} 指定 remote cluster（CH 37）。

ES|QL 時序聚合（9.x）

對 TSDS 用 TS source command：

TS metrics-system
| WHERE TRANGE(1h)
| STATS
    avg_cpu = AVG(rate(cpu.user, 5m)),
    p99_rt  = PERCENTILE(rate(http_requests, 5m), 99)
  BY TBUCKET(1m), host.name
| SORT TBUCKET DESC
| LIMIT 60

支援：rate、increase、delta、idelta，可指定 sliding window：rate(metric, 10m)。

Async ES|QL Query

長查詢用 async：

POST /_query/async
{
  "query": "FROM logs-* | STATS count = COUNT(*) BY service.name",
  "wait_for_completion_timeout": "1s",
  "keep_alive":                  "5m"
}

# 取結果
GET /_query/async/{id}

# 取消
DELETE /_query/async/{id}

在 Kibana 中用 ES|QL

  • Discover：直接在 Discover 切換到 ES|QL 模式
  • Lens：建立視覺化時可用 ES|QL（9.x 強化中）
  • Dashboard：8.12+ 支援在 dashboard 直接編輯 ES|QL（無需切換到 Discover）
  • Alerting：alerting 條件可用 ES|QL

ES|QL vs ES SQL

ES 仍支援標準 SQL（_sql endpoint），但 ES|QL 是新主力。

項目ES SQLES|QL
語法標準 SQLPiped 自有語法
JDBC/ODBC✅❌（用 REST）
聚合GROUP BYSTATS BY
JOIN有限✅ LOOKUP JOIN
Pipeline 思維❌✅
Vector search❌✅
新功能投資少大

ES SQL 範例（傳統 SQL 玩家友善）

POST /_sql?format=txt
{
  "query": "
    SELECT service_name, COUNT(*) as cnt, AVG(response_ms) as avg_rt
    FROM \"logs-*\"
    WHERE log_level = 'error' AND \"@timestamp\" > NOW() - INTERVAL 1 HOUR
    GROUP BY service_name
    ORDER BY cnt DESC
    LIMIT 10
  "
}

支援格式：txt（人類友善）、csv、tsv、json、yaml、cbor、smile。

SQL 轉 Query DSL

POST /_sql/translate
{
  "query": "SELECT * FROM \"logs\" WHERE level = 'error'"
}

會回傳對應的 Query DSL JSON，是學習 mapping 的好工具。

ES|QL 最佳實踐

  • 新 dashboard / analytics 用 ES|QL，比 Lens 拖拉更靈活
  • FROM 後立即 WHERE，filter pushdown 大幅減少資料
  • STATS 前 DROP 不要的欄位，減少傳輸
  • 用 LOOKUP JOIN 取代 application 層 join
  • 用 ENRICH 補靜態資料（IP-to-geo、user info）
  • 長查詢用 /_query/async
  • 分頁用 LIMIT；大量匯出用 async + LIMIT 分批
  • 用 EXPLAIN（9.x 新）看執行計畫：EXPLAIN { FROM logs | STATS ... }
  • 9.x 對某些 query shape filter pushdown 達 86x 加速
  • Application 全文搜尋仍用 Query DSL（_score、highlight 等）

本章重點回顧

  • ES|QL 是 9.1 GA 的管線式查詢語言，靈感來自 SPL
  • 核心命令：FROM、WHERE、EVAL、STATS、SORT、LIMIT、KEEP/DROP/RENAME、DISSECT/GROK、ENRICH、LOOKUP JOIN
  • LOOKUP JOIN（9.1 GA）是 application-side join 的解放
  • ENRICH 用 enrich policy 補靜態資料（geo、user metadata）
  • 支援vector search（KNN、V_COSINE、TEXT_EMBEDDING in 9.3）
  • 跨叢集（CCS）9.1 GA：cluster:index
  • TSDS 用 TS source command + rate/increase 函式
  • ES SQL 仍可用，但 ES|QL 是新功能投資焦點
  • Application 搜尋仍以 Query DSL 為主；analytics、dashboard 改用 ES|QL
