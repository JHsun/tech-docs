ML Node 配置

# 專用 ML 節點
node.roles: [ "ml", "remote_cluster_client" ]

# Heap：跟 Data node 不同，ML node 用較少 heap，更多給 native processes
# 自動 sizing 已最佳化（CH 18）

9.x 預設讓 ML node 自動算 heap。需要記憶體最多的工作（如 BERT 推論）建議至少 16-32 GB RAM。

Anomaly Detection（AD）

ES 自家的無監督異常偵測，自動學習時序資料的「正常基準」，找出偏離的點。常用於：

  • 系統指標異常（CPU 突增、磁碟異常 IO）
  • 業務指標異常（訂單量、登入失敗率）
  • 登入異常行為（不尋常 IP 來源）

三種 Job 類型

類型說明例子
Single metric單欄位整體 CPU 使用率
Multi-metric多欄位 / 多 partition各 host 各自的 CPU
Population群體分析各 user 與其他 user 比較行為
Advanced / Geo / Categorization進階（geo 距離、log 分類等）geo_distance 異常移動

建立 AD Job（API）

# 建立 anomaly detection job
PUT _ml/anomaly_detectors/cpu-usage-job
{
  "description": "CPU usage anomaly per host",
  "analysis_config": {
    "bucket_span": "5m",
    "detectors": [
      {
        "function":      "high_mean",
        "field_name":    "system.cpu.user.pct",
        "partition_field_name": "host.name"
      }
    ],
    "influencers": [ "host.name", "process.name" ]
  },
  "data_description": {
    "time_field":  "@timestamp"
  },
  "model_plot_config": { "enabled": true }
}

# 建立 datafeed（從 ES index 餵資料給 job）
PUT _ml/datafeeds/datafeed-cpu-usage
{
  "job_id":   "cpu-usage-job",
  "indices":  [ "metrics-system.cpu-default" ],
  "query": { "match_all": {} }
}

# 啟動 job + datafeed
POST _ml/anomaly_detectors/cpu-usage-job/_open
POST _ml/datafeeds/datafeed-cpu-usage/_start

Detector Functions

Function偵測什麼
count / high_count / low_count事件數異常
mean / high_mean / low_mean平均值異常
min / max極值
sum / high_sum / low_sum總和異常
distinct_countcardinality 異常
rare / freq_rare少見值出現
info_content資訊量異常（log 多樣性）
geo_distance_lat_long地理位置異常
time_of_day / time_of_week時段異常

看異常結果

三種方式：

  1. Kibana Anomaly Explorer：圖形化最易懂
  2. Single Metric Viewer：時序圖含基準線
  3. API：
# 查最高分的異常
curl -u elastic:xxx --cacert ca.crt \
  -H 'Content-Type: application/json' \
  "https://localhost:9200/_ml/anomaly_detectors/cpu-usage-job/results/records?from=0&size=10&sort=record_score&desc=true&pretty" -d '
{
  "start": "now-1d",
  "end":   "now"
}'

Forecasting：預測未來

對 single metric job 可預測未來值：

POST _ml/anomaly_detectors/cpu-usage-job/_forecast
{
  "duration":           "7d",
  "expires_in":         "30d"
}

結果可用 API 或 Kibana ML 查看。預測常用於：

  • 容量規劃（disk usage 何時會滿）
  • 業務預測
  • 產生未來 anomaly threshold

Data Frame Analytics（DFA）

有監督 / 無監督的批次 ML，輸出 trained model：

類型用途例
Classification有監督多分類客戶流失預測、log 分級
Regression有監督回歸房價預測、order amount 預估
Outlier detection無監督找離群異常 user 識別

分類範例

PUT _ml/data_frame/analytics/customer-churn
{
  "source": {
    "index":   "customers",
    "query":   { "match_all": {} }
  },
  "dest": {
    "index":   "customers-churn-prediction"
  },
  "analysis": {
    "classification": {
      "dependent_variable": "churned",
      "training_percent":   80,
      "num_top_classes":    2
    }
  },
  "analyzed_fields": {
    "includes": [ "tenure_months", "monthly_spend", "support_tickets", "country", "churned" ]
  },
  "model_memory_limit": "1gb"
}

POST _ml/data_frame/analytics/customer-churn/_start

訓練完成後，customers-churn-prediction index 會包含每筆原始資料 + 預測欄位（ml.churned_prediction、ml.prediction_probability）。

評估模型

POST _ml/data_frame/_evaluate
{
  "index": "customers-churn-prediction",
  "query": { "term": { "ml.is_training": false } },
  "evaluation": {
    "classification": {
      "actual_field":     "churned",
      "predicted_field":  "ml.churned_prediction",
      "metrics": {
        "accuracy":             { },
        "auc_roc":              { "include_curve": true },
        "multiclass_confusion_matrix": { },
        "recall":               { },
        "precision":            { }
      }
    }
  }
}

Trained Models

模型可：

  • 由 ES Data Frame Analytics 訓練（上面範例）
  • 從 Hugging Face 等外部匯入（NLP 模型）
  • 使用 ES 內建（如 ELSER、E5）

列出 deployed models

curl -u elastic:xxx --cacert ca.crt \
  "https://localhost:9200/_ml/trained_models?pretty"

從 Hugging Face 匯入（用 eland）

# 安裝 eland Python client
pip install eland

# 匯入 NER 模型
eland_import_hub_model \
  --url https://es.example.com:9200 \
  --hub-model-id "dslim/bert-base-NER" \
  --task-type ner \
  --es-username elastic \
  --es-password ...

# 部署
POST _ml/trained_models/dslim__bert-base-ner/deployment/_start
{
  "number_of_allocations": 1,
  "threads_per_allocation": 1
}

支援的 NLP Tasks

  • fill_mask：填空預測
  • ner：命名實體辨識
  • question_answering：問答
  • text_classification：文本分類（含情緒分析）
  • text_embedding：產生 dense_vector
  • text_similarity：句子相似度
  • zero_shot_classification：未見過分類
  • pass_through：直接 pass

Inference Processor：套到資料

PUT _ingest/pipeline/ner-pipeline
{
  "processors": [
    {
      "inference": {
        "model_id":    "dslim__bert-base-ner",
        "input_output": [
          {
            "input_field":  "comment",
            "output_field": "ml.entities"
          }
        ]
      }
    }
  ]
}

# 索引時自動套
POST reviews/_doc?pipeline=ner-pipeline
{
  "comment": "Apple Watch from California works great"
}
// 結果含 ml.entities = [ { entity: "Apple Watch", class: "ORG" }, ... ]

Inference API（9.x 統一介面）

9.x 推出統一 inference endpoint API（CH 40），支援多家 provider。對 trained model 的調用也走這條路：

# 建 inference endpoint 對應某個 deployed model
PUT _inference/text_embedding/my-e5-small
{
  "service": "elasticsearch",
  "service_settings": {
    "model_id":         ".multilingual-e5-small",
    "num_allocations":  1,
    "num_threads":      1
  }
}

# 直接呼叫 inference
POST _inference/text_embedding/my-e5-small
{
  "input": "Hello world!"
}

ML 與 AD 最佳實踐

Anomaly Detection

  • 從簡單 detector 開始：先 high_mean，再加 partition
  • bucket_span 配合資料粒度：1m 級指標用 1-5m，每天的 KPI 用 1d
  • 用 influencers 找根因（host、user、url 等）
  • 讓 model 訓練 14-30 天再評估，避免基準不穩
  • 結合 Watcher（CH 45）：score > 75 觸發告警

Data Frame Analytics

  • feature engineering 在 ES 端做：用 transform job（CH 51 進階）建寬表
  • training_percent 一般 75-85%
  • model_memory_limit 開大避免 OOM
  • 用 _explain API 看 feature importance
  • Outlier detection：對 query 結果做特徵越多越準（但越慢）

Trained Models 部署

  • num_allocations 控制平行 model instance
  • threads_per_allocation 控制單 inference thread 數
  • 記憶體：每 allocation 約等於 model 大小（BERT-base ~ 400 MB）
  • 大模型用 GPU node（9.x 部分支援）
  • 用 deployment _stats API 看 inference latency / queue

本章重點回顧

  • ES ML 三大功能：Anomaly Detection、Data Frame Analytics、Inference
  • 需 Platinum 訂閱（Inference 部分功能 Basic 即可）
  • Anomaly Detection：無監督，自動學基準找異常
  • Detector functions：count、mean、rare、info_content、geo_distance 等
  • Anomaly score 0-100；> 75 為 critical
  • 可 forecast 未來值（容量規劃）
  • DFA：classification、regression、outlier detection 三選一
  • Trained models 可從 Hugging Face 匯入（用 eland）
  • NLP tasks：NER、問答、文本分類、embedding、similarity 等
  • Inference 透過 ingest pipeline 套到資料、或新 inference API endpoint
  • 9.x 統一 inference API 整合多家 provider
