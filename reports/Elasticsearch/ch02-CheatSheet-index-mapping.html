📦 A. Index CRUD（建立 / 修改 / 刪除）

情境

說明

🔀 C. Multi-fields（同欄位多用法）

最常見場景：欄位既要全文搜尋（text）又要排序聚合（keyword）——用 multi-fields 解決。

情境
命令 / Code
說明
同欄位多用法
同時要全文搜 + 排序聚合
{
  "title": {
    "type": "text",
    "fields": {
      "keyword": {
        "type": "keyword",
        "ignore_above": 256
      }
    }
  }
}
查詢：title（全文）
排序：title.keyword
多 analyzer
{
  "content": {
    "type": "text",
    "fields": {
      "english":   { "type": "text", "analyzer": "english" },
      "ngram":     { "type": "text", "analyzer": "ngram_analyzer" }
    }
  }

同一文字以多種 analyzer 索引；查詢時選對應子欄位
completion + text

  "city": {
    "type": "text",

    }
  }

🔤 D. Analyzer（分詞器）

情境
命令 / Code
說明
看內建 analyzers
GET /_analyze
{
  "analyzer": "standard",
  "text":     "Hello World 123"
}
測試 analyzer。內建：standard / simple / whitespace / stop / keyword / pattern
各國語言 analyzer
{

自訂 analyzer
PUT /my-index
{
  "settings": {
    "analysis": {
      "analyzer": {
        "my_analyzer": {
          "type":      "custom",
          "tokenizer": "standard",
          "filter":    ["lowercase", "asciifolding", "my_synonyms"]
        }
      },
      "filter": {
        "my_synonyms": {
          "type":     "synonym_graph",
          "synonyms": ["tv,television", "phone,mobile"]
        }
      }
    }
  }
}
自訂 tokenizer + filter chain
動態 Synonyms API 🆕 9+
PUT /_synonyms/products-synonyms
{
  "synonyms_set": [
    { "id": "1", "synonyms": "tv, television" },
    { "id": "2", "synonyms": "phone, mobile, smartphone" }
  ]
}
9.x 提供 dedicated API；不需 reindex 即可增刪
用 synonyms set in mapping
"filter": {
  "synonym_filter": {
    "type":         "synonym_graph",
    "synonyms_set": "products-synonyms",
    "updateable":   true
  }
}
updateable: true 讓 reload 即生效（但 search-time only）
第三方 analyzer
# 安裝 plugin（自管 ES）
bin/elasticsearch-plugin install analysis-icu
bin/elasticsearch-plugin install analysis-smartcn   # 中文
常用：analysis-icu (Unicode)、analysis-smartcn (中文)、IK plugin
💡 中文搜尋三選：
  • analysis-icu（內建選用）：Unicode 標準切詞，輕量
  • analysis-smartcn（內建選用）：簡體中文偏向
  • IK plugin（社群）：可自訂詞典，繁中常用

✏️ E. Mapping 動態與更新

情境
命令 / Code
說明
新增欄位
PUT /my-index/_mapping
{
  "properties": {
    "new_field": { "type": "keyword" }
  }
}
可加新欄位；無法改現有欄位類型
禁止 dynamic mapping
PUT /my-index/_mapping
{
  "dynamic": "strict"
}
值：true（預設，自動建）、false（不索引未知欄位）、strict（拒絕）、runtime（建為 runtime）
Dynamic templates
PUT /my-index
{
  "mappings": {
    "dynamic_templates": [
      {
        "strings_as_keyword": {
          "match_mapping_type": "string",
          "mapping": { "type": "keyword" }
        }
      }
    ]
  }
}
對符合條件的新欄位套規則。常見：string → keyword 而非 text
Runtime field（動態查詢）
PUT /my-index/_mapping
{
  "runtime": {
    "day_of_week": {
      "type":   "keyword",
      "script": {
        "source": "emit(doc['@timestamp'].value.dayOfWeekEnum.toString())"
      }
    }
  }
}
查詢時計算（不索引）；適合臨時新增欄位
欄位 alias
"user_name": {
  "type": "alias",
  "path": "user.name"
}
查詢用 user_name → 等同於 user.name（不存資料）
關閉 _source（謹慎）
"_source": { "enabled": false }
省空間但無法 reindex / update / highlight；通常不建議
Synthetic _source 🆕 9+
"_source": { "mode": "synthetic" }
不存原始 _source，從欄位重建。省 50% 空間
⚠️ 無法改現有欄位類型！ 例如想把 text 改 keyword → 必須 reindex 到新 index（H 段）。

📐 F. Index Templates（composable + component）

9.x 推薦 composable templates（傳統 templates 在 7.8 已棄用）。

情境
命令 / Code
說明
建立 component template
PUT /_component_template/logs-mappings
{
  "template": {
    "mappings": {
      "properties": {
        "@timestamp": { "type": "date" },
        "message":    { "type": "text" },
        "level":      { "type": "keyword" }
      }
    }
  }
}
可重用的 mapping 模組；多 template 可組合
建立 component（settings）
PUT /_component_template/logs-settings
{
  "template": {
    "settings": {
      "number_of_shards":   1,
      "index.lifecycle.name": "logs-policy"
    }
  }
}
把 settings 與 mappings 拆成獨立 component
組合 index template
PUT /_index_template/logs-template
{
  "index_patterns": ["logs-*"],
  "priority":       200,
  "composed_of":    ["logs-mappings", "logs-settings"],
  "template": {
    "aliases": {
      "logs-search": {}
    }
  }
}
高 priority 勝出。composed_of 從前到後合併（後者覆蓋）
data stream template
PUT /_index_template/logs-app-template
{
  "index_patterns": ["logs-app-*"],
  "priority":       500,
  "data_stream":    {},
  "template": {
    "mappings": {
      "properties": {
        "@timestamp": { "type": "date" }
      }
    },
    "settings": {
      "index.lifecycle.name": "logs-policy"
    }
  }
}
加 "data_stream": {} 即建立 data stream（必須有 @timestamp）
看 templates
GET /_index_template/logs-*
GET /_component_template/logs-*
列出符合 pattern 的 templates
模擬 template 結果
POST /_index_template/_simulate_index/logs-app-2026.01.01
{
  "index_template": "logs-app-template"
}
看建立 index 時最終會套用什麼 settings/mappings
刪除 template
DELETE /_index_template/logs-template
不影響已建 index；只影響後續

🏷️ G. Aliases（別名與切換）

情境
命令 / Code
說明
建立 alias
POST /_aliases
{
  "actions": [
    { "add": { "index": "my-index-2026.01", "alias": "current-data" } }
  ]
}
一個 alias 可指多 index
原子切換 alias
POST /_aliases
{
  "actions": [
    { "remove": { "index": "old-data", "alias": "current-data" } },
    { "add":    { "index": "new-data", "alias": "current-data" } }
  ]
}
Reindex 後切換的關鍵。原子操作！
Filter alias
POST /_aliases
{
  "actions": [
    { "add": {
      "index":  "logs-*",
      "alias":  "errors",
      "filter": { "term": { "level": "ERROR" } }
    }}
  ]
}
查 alias 自動加 filter；類似 view
寫入 alias（is_write_index）
POST /_aliases
{
  "actions": [
    { "add": {
      "index":          "logs-2026.01",
      "alias":          "logs",
      "is_write_index": true
    }}
  ]
}
寫入到 alias 時，自動寫到標記為 write_index 的那個 index
看 alias
GET /_alias
GET /_alias/current-data
GET /my-index/_alias
多種查法
刪除 alias
DELETE /my-index/_alias/current-data
只刪 alias，不刪 index
💡 經典 reindex 切換流程：
  1. 建 new-index（新 mapping）
  2. Reindex old → new
  3. 原子切 alias：移除 old、新增 new（一個 _aliases 請求）
  4. 確認應用正常後刪 old-index

🔄 H. Reindex / Update by Query / Delete by Query

情境
命令 / Code
說明
Reindex（最常用）
POST /_reindex?wait_for_completion=false
{
  "source": { "index": "old-index" },
  "dest":   { "index": "new-index" }
}
改 mapping 唯一辦法。wait_for_completion=false 異步執行，回 task ID
Reindex 帶查詢
POST /_reindex
{
  "source": {
    "index": "old-index",
    "query": { "term": { "status": "active" } }
  },
  "dest": { "index": "active-only" }
}
只搬部分資料
Reindex from remote
POST /_reindex
{
  "source": {
    "remote": {
      "host":     "https://other-cluster:9200",
      "username": "u",
      "password": "p"
    },
    "index": "src-*"
  },
  "dest": { "index": "destination" }
}
跨 cluster 遷移；遠端 host 須在 reindex.remote.whitelist 設定
Reindex 加 script
POST /_reindex
{
  "source": { "index": "old" },
  "dest":   { "index": "new" },
  "script": {
    "source": "ctx._source.year = ctx._source.year + 1"
  }
}
搬資料同時改欄位；用 Painless（CH 42）
Update by query
POST /my-index/_update_by_query
{
  "script": {
    "source": "ctx._source.tag = 'updated'"
  },
  "query": { "term": { "category": "old" } }
}
原地批次更新（耗 IO）
Delete by query
POST /my-index/_delete_by_query
{
  "query": {
    "range": { "@timestamp": { "lt": "now-30d" } }
  }
}
批次刪除（不可逆！執行前先看 count）
查 task 進度
GET /_tasks/<task_id>
Reindex 等異步任務進度查詢
取消 task
POST /_tasks/<task_id>/_cancel
中斷長跑的 reindex / delete_by_query
⚠️ 大資料量 reindex 注意：
  • 用 ?wait_for_completion=false 異步
  • 暫停 ILM：POST /_ilm/stop
  • 新 index 暫關 replica + 拉長 refresh_interval（CH 49）
  • 用 slice 平行："slices": "auto"

情境
命令 / Code
說明
新增（自動 ID）
POST /my-index/_doc
{
  "title": "Hello",

自動產生 ID

POST /my-index/_update/abc-123
{
  "doc": { "title": "Updated" }

只改部分欄位

POST /my-index/_update/abc-123
{
  "doc":           { "views": 1 },
  "doc_as_upsert": true
}
存在則 update；不存在則 create
Update with script
POST /my-index/_update/abc-123
{
  "script": {
    "source": "ctx._source.views += params.n",
    "params": { "n": 1 }
  }
}
原子計數器，避免 race condition
刪除
DELETE /my-index/_doc/abc-123
單筆刪

📦 J. Bulk API（批次操作）

情境
命令 / Code
說明
Bulk 操作
POST /_bulk
{ "index":  { "_index": "my-index", "_id": "1" } }
{ "title":  "One" }
{ "create": { "_index": "my-index", "_id": "2" } }
{ "title":  "Two" }
{ "update": { "_index": "my-index", "_id": "1" } }
{ "doc": { "title": "Updated" } }
{ "delete": { "_index": "my-index", "_id": "3" } }
每行都要 \n 結尾，包含最後一行。一次多筆操作
建議 bulk size：5-15 MB / batch；單筆過大時分批
Production：用 _bulk endpoint，client 端維護 retry + back-pressure
⚠️ Bulk 格式坑：
  • 每行 JSON 物件後必須 \n，包括最後一行
  • 不能用一般 JSON pretty-print（因為換行被吃）
  • Content-Type 必須是 application/x-ndjson
  • 用 curl 加 --data-binary（不是 -d）

(function () {
  function init() {
    document.querySelectorAll('main pre').forEach(function (pre) {
      if (pre.parentElement.classList.contains('code-wrap')) return;
      const wrap = document.createElement('div');
      wrap.className = 'code-wrap';
      pre.parentNode.insertBefore(wrap, pre);
      wrap.appendChild(pre);
      const lang = pre.querySelector('code')?.className.match(/language-(\w+)/)?.[1];
      if (lang) {
        const tag = document.createElement('span');
        tag.className = 'code-lang';
        tag.textContent = lang;
        wrap.appendChild(tag);
      }
      const btn = document.createElement('button');
      btn.className = 'copy-btn';
      btn.type = 'button';
      btn.textContent = '📋 複製';
      btn.addEventListener('click', function () {
        const text = pre.innerText;
        if (navigator.clipboard?.writeText) {
          navigator.clipboard.writeText(text).then(success).catch(fallback);
        } else {
          fallback();
        }
        function success() {
          btn.textContent = '✓ 已複製';
          btn.classList.add('copied');
          setTimeout(function () { btn.textContent = '📋 複製'; btn.classList.remove('copied'); }, 1800);
        }
        function fallback() {
          const ta = document.createElement('textarea');
          ta.value = text;
          document.body.appendChild(ta);
          ta.select();
          try { document.execCommand('copy'); success(); } catch (_) {}
          document.body.removeChild(ta);
        }
      });
      wrap.appendChild(btn);
    });
  }
  if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
  else init();
})();

// Auto-build TOC from h2/h3 in 
    
(function () {
  function init() {
    const toc = document.querySelector('.toc ul');
    if (!toc) return;
    const headings = document.querySelectorAll('main h2, main h3');
    headings.forEach(function (h) {
      if (!h.id) {
        h.id = h.textContent.trim().toLowerCase()
          .replace(/[^\w\u4e00-\u9fff\s-]/g, '')
          .replace(/\s+/g, '-');
      }
      const li = document.createElement('li');
      const a = document.createElement('a');
      a.href = '#' + h.id;
      a.textContent = h.textContent;
      a.className = 'toc-' + h.tagName.toLowerCase();
      li.appendChild(a);
      toc.appendChild(li);
    });
  }
  if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
  else init();
})();

  let renderCounter = 0;
  async function renderAll() {
    if (typeof mermaid === 'undefined') return;
    renderCounter++;
    const myCounter = renderCounter;
    mermaid.initialize(getMermaidConfig());
    const blocks = document.querySelectorAll('.mermaid');
    for (const el of blocks) {
      // restore source if already rendered
      if (el.dataset.source) el.innerHTML = el.dataset.source;
      else el.dataset.source = el.innerHTML;
      el.removeAttribute('data-processed');
    }
    if (myCounter !== renderCounter) return;
    try {
      await mermaid.run({ querySelector: '.mermaid' });
    } catch (e) { console.warn('Mermaid render error', e); }
  }

  function init() {
    renderAll();
    window.addEventListener('jhsun:theme-changed', renderAll);
