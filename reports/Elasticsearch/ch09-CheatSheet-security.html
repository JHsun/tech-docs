🔐 A. TLS 憑證（elasticsearch-certutil）

9.x 預設 xpack.security.enabled=true，必須設好 transport TLS 才能跑 production cluster。

產 transport cert（每節點）
bin/elasticsearch-certutil cert \
  --ca config/elastic-stack-ca.p12 \
  --ca-pass mySecret \
  --out config/elastic-certificates.p12 \
  --pass nodePass

transport TLS（port 9300）每個 node 用同一份 .p12

# 互動式：選 generate CSR 或自簽
# 輸出 elasticsearch-ssl-http.zip 含 elasticsearch/http.p12 + kibana/elasticsearch-ca.pem

🎭 C. Role 定義

情境
命令 / Code
說明
建 role（簡單）
PUT /_security/role/logs_viewer
{
  "cluster": [ "monitor" ],
  "indices": [
    {
      "names":      [ "logs-*" ],
      "privileges": [ "read", "view_index_metadata" ]
    }
  ]
}
cluster + indices 權限基本
含 DLS（文件級）🆕 9+
PUT /_security/role/team_data
{
  "indices": [
    {
      "names":      [ "events-*" ],
      "privileges": [ "read" ],
      "query": {
        "template": {
          "source": """{
            "term": { "team": "{{_user.metadata.team}}" }
          }"""
        }
      }
    }
  ]
}
Platinum 授權必要。query template 用 user metadata 自動過濾
含 FLS（欄位級）🆕 9+
PUT /_security/role/no_pii
{
  "indices": [
    {
      "names":      [ "users" ],
      "privileges": [ "read" ],
      "field_security": {
        "grant":  [ "*" ],
        "except": [ "ssn", "credit_card", "address" ]
      }
    }
  ]
}
隱藏敏感欄位。Platinum 授權
多 namespace 規則
"indices": [
  {
    "names":      [ "logs-prod-*" ],
    "privileges": [ "read", "write" ]
  },
  {
    "names":      [ "logs-dev-*" ],
    "privileges": [ "all" ]
  }
]
一 role 多 index pattern；不同 pattern 不同權限
Application privileges (Kibana)
"applications": [
  {
    "application": "kibana-.kibana",
    "privileges":  [ "feature_dashboard.read" ],
    "resources":   [ "space:default" ]
  }
]
控制 Kibana feature 訪問
列 roles
GET /_security/role
GET /_security/role/logs_viewer
看所有 / 單一
刪 role
DELETE /_security/role/logs_viewer
使用此 role 的 user 會失去這部分權限
💡 Cluster privileges 速查： all / monitor（看狀態） / manage（改 cluster 設定） / manage_security（改 user/role） / manage_index_templates / manage_pipeline / manage_ilm / manage_ml / read_ccr / manage_ccr
💡 Index privileges 速查： all / read（搜尋 + get） / read_cross_cluster（CCS 用） / view_index_metadata（看 mapping） / write（index/update/delete docs） / create_index / create_doc / delete_index / manage（settings/mapping） / manage_follow_index（CCR）

🔑 E. API Key（程式接 ES 用）

情境
命令 / Code
說明
建 API key（最簡）
POST /_security/api_key
{
  "name":       "client-app",
  "expiration": "30d"
}
回 id + api_key（一次性顯示！）；未限制權限 = 繼承 user 權限
建 API key with role descriptors
POST /_security/api_key
{
  "name":       "logstash-writer",
  "expiration": "90d",
  "role_descriptors": {
    "logstash_role": {
      "cluster": [ "monitor" ],
      "indices": [
        {
          "names":      [ "logstash-*" ],
          "privileges": [ "create_index", "write" ]
        }
      ]
    }
  },
  "metadata": {
    "app":         "logstash",
    "environment": "prod"
  }
}
建議做法：用 role_descriptors 限制權限到必要最小
建 API key with DLS query template
POST /_security/api_key
{
  "name": "tenant-a-key",
  "role_descriptors": {
    "tenant_role": {
      "indices": [
        {
          "names":      [ "events-*" ],
          "privileges": [ "read" ],
          "query":      """{ "term": { "tenant_id": "tenant-a" } }"""
        }
      ]
    }
  }
}
多租戶必備：每 tenant 一把 key，DLS 自動過濾
API key 使用
# 用 base64(id:api_key) 當認證
curl -H "Authorization: ApiKey <base64-id-colon-key>" \
  https://es:9200/_search

# Python
from elasticsearch import Elasticsearch
es = Elasticsearch(
  "https://es:9200",
  api_key=("<id>", "<api_key>")  # 或 base64 string
)
Authorization: ApiKey ... header
看自己的 keys
GET /_security/api_key?owner=true
只看自己擁有的 keys
查 keys（admin）
GET /_security/api_key?username=alice&realm_name=native
GET /_security/api_key?id=<key_id>
管理員查任意 key
更新 API key
PUT /_security/api_key/<key_id>
{
  "role_descriptors": { ... },
  "metadata":          { ... }
}
更新 role / metadata；不能改名
批量更新 keys
POST /_security/api_key/_bulk_update
{
  "ids":       [ "key1", "key2", "key3" ],
  "role_descriptors": { ... }
}
一次更新多把
撤銷 key（單一）
DELETE /_security/api_key
{ "ids": [ "<key_id>" ] }
立即失效
撤銷某 user 全部 keys
DELETE /_security/api_key
{ "username": "alice" }
離職時必跑！
撤銷某 realm 全部
DELETE /_security/api_key
{ "realm_name": "native" }
清空所有 native realm 的 key
⚠️ API Key 安全建議
  • API key 只顯示一次！產生後立即儲存到 secret manager（Vault / AWS Secrets Manager）
  • 永遠加 expiration（30d / 90d / 365d），定期輪替
  • 用 role_descriptors 限制權限，不要繼承 user 全部權限
  • 離職 / 服務退休時記得 DELETE /_security/api_key 撤銷
  • 多租戶用 query template + DLS 自動過濾

🌐 F. Cross-cluster API Key（CCS / CCR）

9.x 推 API key 模型取代傳統 TLS cert 模型（後者已 deprecated）。新模型用 port 9443（不是 9300）。

情境
命令 / Code
說明
建 cross-cluster API key🆕 9+
POST /_security/cross_cluster/api_key
{
  "name": "us-to-eu-ccr-key",
  "access": {
    "search": [
      {
        "names":  [ "logs-*", "metrics-*" ],
        "query":  { "term": { "region": "us" } }
      }
    ],
    "replication": [
      { "names": [ "logs-app-*" ] }
    ]
  },
  "expiration": "365d"
}
在 remote cluster 上建；給 local cluster 連用
local cluster 設定遠端連線
PUT /_cluster/settings
{
  "persistent": {
    "cluster.remote.eu_cluster.mode":  "proxy",
    "cluster.remote.eu_cluster.proxy_address":
      "eu.example.com:9443"
  }
}

# 把 cross-cluster API key 放 keystore
bin/elasticsearch-keystore add \
  cluster.remote.eu_cluster.credentials
# 互動貼上 base64(id:key)
API key 模型用 9443 port（不是 9300）
看 remote cluster 狀態
GET /_remote/info
看連線狀態、健康度
更新 cross-cluster key
PUT /_security/cross_cluster/api_key/<key_id>
{ "access": { ... } }

🔒 G. DLS / FLS（文件級 / 欄位級安全）

Platinum 授權專用功能。把資料的可見性下推到 ES 層。

類型
限制
範例
DLS Document-Level Security
"query": {
  "term": { "tenant_id": "{{_user.metadata.tenant}}" }
}
每筆 doc 是否可見
FLS Field-Level Security
"field_security": {
  "grant":  [ "*" ],
  "except": [ "ssn", "credit_card" ]
}
哪些欄位可見
⚠️ 授權不相容警告：當 cluster 授權降級時，DLS/FLS API key fail closed（throw error）而不是 fail open。 使用 DLS/FLS 前必須確保是 Platinum 或更高授權。

G2. 多租戶範本

# 1. 建 base role：每 user 看自己 tenant 的資料
PUT /_security/role/tenant_user
{
  "indices": [
    {
      "names":      [ "events-*" ],
      "privileges": [ "read" ],
      "query": {
        "template": {
          "source": """{ "term": { "tenant_id": "{{_user.metadata.tenant_id}}" } }"""
        }
      },
      "field_security": {
        "grant":  [ "*" ],
        "except": [ "internal_*" ]
      }
    }
  ]
}

# 2. user metadata 帶 tenant_id

📜 I. Audit Log（合規必備）

情境
命令 / Code
說明
啟用 audit log
xpack.security.audit.enabled: true
xpack.security.audit.logfile.events.include:
  - access_denied
  - access_granted
  - authentication_failed
  - connection_denied
  - tampered_request
  - run_as_denied
  - run_as_granted
xpack.security.audit.logfile.events.exclude:
  - access_granted
寫到 logs/<cluster>_audit.json
過濾 system actions
xpack.security.audit.logfile.events.emit_request_body: false
xpack.security.audit.logfile.events.ignore_filters:
  system_user:
    users: [ "_xpack_security", "_xpack" ]
減少噪音；勿開 emit_request_body 在 prod（會洩漏 PII）
查 audit log
# 直接看 file
tail -f /var/log/elasticsearch/<cluster>_audit.json | jq

# 或 ingest 到自己 cluster 用 Filebeat / Agent 看 dashboard
ndjson 格式；每 event 一行
典型 event 欄位
@timestamp / event.action / user.name / request.id / indices / request.body
用於合規（GDPR / HIPAA / SOX）
💡 重要 audit events：
  • access_denied：權限不足（防內賊）
  • authentication_failed：登入失敗（防爆破）
  • connection_denied：IP 黑名單擋下
  • tampered_request：請求被竄改
  • run_as_granted：跨身分執行（特權）

🚒 J. 安全 SOP（場景）

場景：API key 外洩

  1. 立即撤銷：DELETE /_security/api_key { "ids": [ "leaked_id" ] }
  2. 查 audit log 看 leaked 期間的所有 access：grep authentication.api_key.id == "leaked_id"
  3. 若 leaked key 有寫權限：對寫過的 indices 做 snapshot 比對
  4. 產生新 key（限制更嚴的 role_descriptors）
  5. 確認應用切換完畢
  6. 事後分析：為何洩漏？git push 帶 secret？env var 寫日誌？

場景：Employee 離職

  1. 停用 user：PUT /_security/user/alice/_disable
  2. 撤銷該 user 全部 API keys：DELETE /_security/api_key { "username": "alice" }
  3. 檢查 SSO（IdP）已撤銷：對方 group 應已自動 remove
  4. 查 audit log 過去 30 天的 activity（合規 / 審計）
  5. 確認無 service / batch job 依賴此 user 的 cred
  6. 數週後可永久刪除：DELETE /_security/user/alice

場景：TLS 憑證即將到期

  1. 檢查 expiry：GET /_ssl/certificates
  2. 用同一 CA 產新 cert：elasticsearch-certutil cert --ca elastic-stack-ca.p12 ...
  3. 滾動更新：每節點停 → 換 cert → 啟動（見 CS 11 rolling restart）
  4. 更新 Kibana / clients 的 truststore
  5. 所有 node 重啟後再次檢查 GET /_ssl/certificates

          ta.select();
          try { document.execCommand('copy'); success(); } catch (_) {}
          document.body.removeChild(ta);
        }
      });
      wrap.appendChild(btn);
    });
  }
  if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
  else init();
})();

// Auto-build TOC from h2/h3 in 
    
(function () {
  function init() {
    const toc = document.querySelector('.toc ul');
    if (!toc) return;
    const headings = document.querySelectorAll('main h2, main h3');
    headings.forEach(function (h) {
      if (!h.id) {
        h.id = h.textContent.trim().toLowerCase()
          .replace(/[^\w\u4e00-\u9fff\s-]/g, '')
          .replace(/\s+/g, '-');
      }
      const li = document.createElement('li');
      const a = document.createElement('a');
      a.href = '#' + h.id;
      a.textContent = h.textContent;
      a.className = 'toc-' + h.tagName.toLowerCase();
      li.appendChild(a);
      toc.appendChild(li);
    });
  }
  if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
  else init();
})();

        signalTextColor: isDark ? '#e6f2f8' : '#0f172a',
        labelBoxBkgColor: isDark ? '#1a2530' : '#fef3c7',
        labelBoxBorderColor: isDark ? '#a89060' : '#d97706',
        labelTextColor: isDark ? '#e6f2f8' : '#0f172a',
        loopTextColor: isDark ? '#e6f2f8' : '#0f172a',
        noteBkgColor: isDark ? '#3a4f3a' : '#fef3c7',
        noteBorderColor: isDark ? '#86c598' : '#d97706',
        noteTextColor: isDark ? '#e6f2f8' : '#0f172a',
      },
      flowchart: { useMaxWidth: true, htmlLabels: true, curve: 'basis' },
      sequence: { useMaxWidth: true, showSequenceNumbers: false, mirrorActors: true, bottomMarginAdj: 10 },
    };
  }

  let renderCounter = 0;
  async function renderAll() {
    if (typeof mermaid === 'undefined') return;
    renderCounter++;
    const myCounter = renderCounter;
    mermaid.initialize(getMermaidConfig());
    const blocks = document.querySelectorAll('.mermaid');
    for (const el of blocks) {
      // restore source if already rendered
      if (el.dataset.source) el.innerHTML = el.dataset.source;
      else el.dataset.source = el.innerHTML;
      el.removeAttribute('data-processed');
    }
    if (myCounter !== renderCounter) return;
    try {
      await mermaid.run({ querySelector: '.mermaid' });
    } catch (e) { console.warn('Mermaid render error', e); }
  }

  function init() {
    renderAll();
    window.addEventListener('jhsun:theme-changed', renderAll);
  }

  if (document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
  else init();
})();
