CH 59 · Search Applications 與 Enterprise Search

適合對象 🟡 開發者 ⚡ 進階
閱讀時間 約 25 分鐘
前置知識 CH 36 Query DSL、CH 40 vector search
本章目標 Search Applications API、Search Templates、Query Rules、Behavioral Analytics、與舊版 App Search/Workplace Search 的關係

背景：Enterprise Search 的演進

Elastic 過去有 App Search 與 Workplace Search 兩個獨立產品提供「應用內搜尋」能力。2024-2025 年起 Elastic 把核心功能整併進 Elasticsearch，形成新的 Search Applications + Connectors + Behavioral Analytics 組合。

⚠️ App Search / Workplace Search 已棄用 9.x 起 App Search、Workplace Search 不再接受新部署。新建立的 search 應用應使用 Search Applications API + Connectors + ELSER（CH 40）。

Search Applications 是什麼？

Search Applications 是 9.x 提供的抽象層，把多個 ES indices 包裝成「可搜尋的應用」，前端只透過簡單 API 即可獲得：

  • 跨多 index 搜尋（template 統一）
  • 客戶端 Search 套件（Search UI）
  • 分析事件記錄（behavioral analytics）
  • 權限與限制管理

建立 Search Application

# 1. 建立 search application，關聯多個 index
PUT /_application/search_application/products-search
{
  "indices": [ "products-electronics", "products-fashion", "products-books" ],
  "template": {
    "script": {
      "source": {
        "query": {
          "bool": {
            "must": [
              {
                "multi_match": {
                  "query":  "{{query_string}}",
                  "fields": [ "title^3", "description", "tags^2" ]
                }
              }
            ],
            "filter": [
              {{#category}}
              { "term": { "category": "{{category}}" } }
              {{/category}}
            ]
          }
        },
        "from": "{{from}}",
        "size": "{{size}}"
      },
      "params": {
        "query_string": "*",
        "from":         0,
        "size":         20,
        "category":     ""
      }
    }
  }
}

# 2. 客戶端搜尋（簡化 API）
POST /_application/search_application/products-search/_search
{
  "params": {
    "query_string": "wireless headphones",
    "size":          10,
    "category":      "electronics"
  }
}

Search Templates（Mustache）

Search Application 內部用 Mustache template 把參數轉成完整 ES query：

  • 變數插入：{{query_string}}
  • 條件區塊：{{#category}}...{{/category}} 只在 category 有值時 render
  • 陣列迭代：{{#tags}}...{{/tags}}
  • JSON 注入：{{#toJson}}filters{{/toJson}} 安全注入物件

完整 RAG-ready 範本

{
  "template": {
    "script": {
      "source": {
        "retriever": {
          "rrf": {
            "retrievers": [
              {
                "standard": {
                  "query": {
                    "multi_match": {
                      "query":  "{{q}}",
                      "fields": [ "title^3", "content" ]
                    }
                  }
                }
              },
              {
                "standard": {
                  "query": {
                    "semantic": {
                      "field": "content",
                      "query": "{{q}}"
                    }
                  }
                }
              }
            ],
            "rank_window_size": 50
          }
        },
        "size": "{{size}}",
        "_source": [ "title", "url", "snippet" ]
      },
      "params": {
        "q":    "*",
        "size": 10
      }
    }
  }
}

Query Rules：Pinned 結果與業務規則

常見需求：搜「running shoes」時，把 promo 商品釘在第 1 名。Query Rules 用內建語法達成：

PUT /_query_rules/promotion-rules
{
  "rules": [
    {
      "rule_id": "winter-sale",
      "type":    "pinned",
      "criteria": [
        {
          "type":     "contains",
          "metadata": "query_string",
          "values":   [ "winter", "snow", "ski" ]
        }
      ],
      "actions": {
        "ids": [ "promo-001", "promo-002", "promo-003" ]
      },
      "priority": 1
    },
    {
      "rule_id": "exclude-discontinued",
      "type":    "exclude",
      "criteria": [
        { "type": "always" }
      ],
      "actions": {
        "ids": [ "discontinued-*" ]
      }
    }
  ]
}

# 在 retriever 中使用
POST products/_search
{
  "retriever": {
    "rule": {
      "ruleset_ids":  [ "promotion-rules" ],
      "match_criteria": {
        "query_string": "winter coat"
      },
      "retriever": {
        "standard": {
          "query": { "match": { "name": "winter coat" } }
        }
      }
    }
  }
}

支援 rule type：

  • pinned：釘前面
  • exclude：排除
  • 9.x：promote、bury（依分數調整）

Search UI：JavaScript 前端套件

Elastic 開源的前端 React 元件套件，與 Search Applications 配合：

npm install @elastic/search-ui @elastic/search-ui-elasticsearch-connector

import { SearchProvider, SearchBox, Results, Paging } from "@elastic/search-ui";
import ElasticsearchAPIConnector from "@elastic/search-ui-elasticsearch-connector";

const connector = new ElasticsearchAPIConnector({
  cloud:   { id: "deployment:..." },
  apiKey:  "...",
  index:   "products"
}, {
  query: {
    multi_match: { query: "{searchTerm}", fields: [ "title", "description" ] }
  }
});

提供：search box、facet、result list、pagination、autocomplete、analytics tracking。

Behavioral Analytics

記錄使用者搜尋行為，產生改進 signals：

  • search query（含 zero-result）
  • click position
  • conversion event（購買、訂閱）
# 建立 analytics collection
PUT /_application/analytics/products-analytics

# 前端送事件（用 Search UI 自動或手動）
POST /_application/analytics/products-analytics/event/search
{
  "session": { "id": "user-session-123" },
  "user":    { "id": "u-456" },
  "search":  {
    "query":         "wireless headphones",
    "results":       { "total":  142 },
    "search_application": "products-search"
  }
}

POST /_application/analytics/products-analytics/event/search_click
{
  "session": { "id": "user-session-123" },
  "search":  { "query": "wireless headphones" },
  "document": { "id": "p-789", "index": "products-electronics" },
  "page":    { "url": "/p/789" }
}

事件儲存於 behavioral_analytics-events-* data stream。Kibana 提供 dashboard：

  • Top queries
  • Zero-result queries
  • Click-through rate (CTR)
  • Conversion funnel

用 zero-result 改進

每週 review zero-result queries，補：

  • 同義詞（synonyms）：CH 27 analyzer
  • 新增資料
  • 調整 query rule pinning
  • 增加 vector search 提升 recall

Synonyms API（9.x 改進）

9.x 提供 dedicated Synonyms API，支援動態增刪不需重新建 index：

# 建立 synonyms set
PUT /_synonyms/products-synonyms
{
  "synonyms_set": [
    { "id": "1", "synonyms": "tv, television" },
    { "id": "2", "synonyms": "phone, mobile, smartphone" },
    { "id": "3", "synonyms": "laptop, notebook, ultrabook" }
  ]
}

# 在 analyzer 中使用
PUT my-index
{
  "settings": {
    "analysis": {
      "filter": {
        "synonym_filter": {
          "type":            "synonym_graph",
          "synonyms_set":    "products-synonyms",
          "updateable":      true
        }
      },
      "analyzer": {
        "search_with_synonyms": {
          "tokenizer": "standard",
          "filter":    [ "lowercase", "synonym_filter" ]
        }
      }
    }
  }
}

後續用 API 增刪 synonym 不需 reindex。

Learn-to-Rank（LTR，9.x）

用機器學習根據點擊行為自動學排序模型：

  1. 用 behavioral analytics 收集 click 資料
  2. 訓練 LTR 模型（XGBoost / LightGBM）
  3. 把模型 upload 到 ES（用 eland）
  4. 在 query 中加 rescorer 套用
# 載入 LTR 模型
POST /_ml/trained_models/my-ltr-model
{
  "input": { "field_names": [ "feature1", "feature2", ... ] },
  "inference_config": {
    "learning_to_rank": { "feature_extractors": [ ... ] }
  },
  "definition": { ... }
}

# 在 query 用 rescorer
POST /products/_search
{
  "query": { ... },
  "rescore": {
    "learning_to_rank": {
      "model_id": "my-ltr-model"
    },
    "window_size": 100
  }
}

Search Applications 最佳實踐

  • 抽象層用 Search Application API，前端只接觸 template params
  • 用 Search UI 套件 取代手刻搜尋介面
  • 啟用 Behavioral Analytics，沒資料就無法改進
  • 每週 review zero-result queries，補同義詞/資料
  • 用 retriever + RRF（CH 40）混合 BM25 + 語義
  • 業務需求進 Query Rules，避免硬編在 application
  • 9.x Synonyms API 動態增刪不需 reindex
  • 大量 click 資料後考慮 LTR 自動優化
  • 新部署 不要再用 App Search / Workplace Search
  • API key 權限 限制只能操作 Search Application（CH 22）

本章重點回顧

  • App Search / Workplace Search 在 9.x 棄用，改用 Search Applications + Connectors + ELSER
  • Search Applications 把多 index 包裝成統一搜尋抽象
  • Search Templates 用 Mustache 把參數轉 ES query
  • Query Rules：pinned / exclude，業務規則化（不硬編）
  • Search UI 是 Elastic 開源的 React 套件
  • Behavioral Analytics 收集 search / click / conversion，回饋 LTR
  • 9.x Synonyms API 動態增刪同義詞
  • Learn-to-Rank 用 click 資料訓練排序模型
  • 典型架構：Search UI → Application → Search Applications API → ES（hybrid + LTR）
