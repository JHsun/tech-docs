5 Configuration Synopsis — 設定概要

設定檔預設路徑為 /etc/keepalived/keepalived.conf，由三大區塊組成：

5.1 Global Definitions（全域定義）

global_defs {
    notification_email {
        admin@domain.com
    }
    notification_email_from keepalived@domain.com
    smtp_server           192.168.200.1
    smtp_connect_timeout  30
    lvs_id                LVS_PROD
}
關鍵字說明型別
notification_email接收告警的 Email 清單List
notification_email_from發信人地址（MAIL FROM）Email
smtp_server用於發送通知的 SMTP 伺服器IP/Host
smtp_connect_timeoutSMTP 連線逾時（秒）數字
lvs_idLVS Director 識別名稱字串

5.2 Virtual Server Definitions（虛擬伺服器定義）

virtual_server 192.168.200.15 80 {
    delay_loop            6          # 檢查間隔（秒）
    lb_algo               rr         # 排程演算法
    lb_kind               NAT        # 轉發模式 NAT|DR|TUN
    persistence_timeout   50         # 持續連線逾時（秒）
    protocol              TCP

    sorry_server 192.168.100.100 80  # 所有 RS 皆下線時啟用

    real_server 192.168.100.2 80 {
        weight 2
        HTTP_GET {
            url {
                path   /health
                digest ec90a42b99ea9a2f5ecbe213ac9eba03
            }
            connect_timeout    3
            retry              3
            delay_before_retry 2
        }
    }
}
關鍵字說明
delay_loop兩次健康檢查之間的間隔秒數
lb_algo排程演算法：rr, wrr, lc, wlc, sh, dh, lblc 等
lb_kind轉發模式：NAT / DR / TUN
persistence_timeout同一客戶端在此秒數內的請求固定到同一 RS
sorry_server備援伺服器（所有 RS 皆不可用時接手）
weight後端伺服器權重值
connect_timeoutTCP 連線逾時秒數
retry最大重試次數
delay_before_retry兩次重試之間的等待秒數
digestURL 回傳內容的 MD5 摘要（用 genhash 產生）

5.3 VRRP Instance Definitions（VRRP 實例定義）

vrrp_sync_group VG1 {
    group {
        VI_1
        VI_2
    }
    notify_master "/path/to/master.sh"
    notify_backup "/path/to/backup.sh"
    notify_fault  "/path/to/fault.sh"
}

vrrp_instance VI_1 {
    state              MASTER       # 初始狀態 MASTER|BACKUP
    interface          eth0         # 綁定的網路介面
    virtual_router_id  51           # VRID（0-255，同組須一致）
    priority           150          # 優先權（越大越優先）
    advert_int         1            # 通告間隔（秒）
    smtp_alert                      # 狀態變化時發送郵件

    authentication {
        auth_type PASS             # PASS 或 AH
        auth_pass k@l!ve1
    }

    virtual_ipaddress {              # VIP（上限 20 個）
        192.168.200.10
        192.168.200.11
    }
    virtual_ipaddress_excluded {     # 不計入協議的 VIP（無上限）
        192.168.200.12
    }
}
關鍵字說明
state初始角色（選舉後可能改變）
virtual_router_id同一 VRRP 群組在所有節點上必須一致，且同網段不可重複
priority1–254，數值最大者成為 MASTER
advert_intMASTER 發送通告封包的頻率
auth_typePASS = 明文密碼；AH = IPSEC-AH 認證
vrrp_sync_group同步群組內任一 Instance 切換，其他也跟著切換
notify_*MASTER/BACKUP/FAULT 狀態轉換時執行的腳本

6 Programs Synopsis — 程式參數

6.1 keepalived 主程式常用參數

參數說明
-f, --use-file指定設定檔（預設 /etc/keepalived/keepalived.conf）
-P, --vrrp僅執行 VRRP 子系統（不使用 LVS 時適用）
-C, --check僅執行 Healthcheck 子系統（單 Director 無故障轉移時適用）
-l, --log-console將日誌輸出到 console（預設送 syslog）
-D, --log-detail啟用詳細日誌
-S, --log-facility設定 syslog facility 為 LOG_LOCAL[0-7]
-V, --dont-release-vrrp停止時不移除 VIP 和 VRoute
-I, --dont-release-ipvs停止時不清除 IPVS 規則
-R, --dont-respawn子行程結束後不自動重啟
-n, --dont-fork前景執行（不 daemonize）
-d, --dump-conf傾印設定資料
-x, --snmp啟用 SNMP 子系統

6.2 genhash 工具

genhash 用於產生 HTTP/SSL 健康檢查所需的 MD5 摘要值：

genhash -s 192.168.100.2 -p 80 -u /testurl/test.jsp

將輸出的 MD5 Digest 值複製到設定檔的 digest 欄位即可。

參數說明
-S, --use-ssl使用 SSL 連線
-s, --server目標伺服器 IP
-p, --port目標埠號
-u, --url要計算摘要的 URL 路徑
-V, --use-virtualhost指定 HTTP Host 標頭
-H, --hash選擇雜湊演算法

6.3 啟動與驗證

# 啟動服務
/etc/rc.d/init.d/keepalived.init start

# 查看日誌（確認設定載入）
tail -f /var/log/messages       # RHEL
tail -f /var/log/daemon.log     # Debian

# 使用 -d 參數啟動可在日誌中看到完整的設定傾印

7 IPVS Scheduling Algorithms — 排程演算法

以下為 IPVS 核心支援的排程演算法，在設定檔中以 lb_algo 指定：

Round Robin
lb_algo rr
逐一輪流分配，不考慮伺服器負載或效能差異。最簡單的排程方式。
Weighted Round Robin
lb_algo wrr
依 weight 值加權輪流。weight=4 的伺服器獲得的連線是 weight=1 的 4 倍。
Least Connection
lb_algo lc
將新連線分配給目前活躍連線數最少的伺服器。動態演算法，適合負載變化大的場景。
Weighted Least Connection
lb_algo wlc
加權版最少連線，選擇 Ci/Wi 最小的伺服器（Ci=連線數、Wi=權重）。最常用的演算法。
Locality-Based LC
lb_algo lblc
依目的 IP 做分配，優先導向同一伺服器（快取友善），伺服器過載時才切換。適用於快取叢集。
LBLC with Replication
lb_algo lblcr
同 LBLC，但維護目標 IP → 伺服器集合的映射，支援複寫，適合大型快取叢集。
Destination Hashing
lb_algo dh
依目的 IP 的靜態雜湊表分配。相同目的 IP 永遠導向同一伺服器。
Source Hashing
lb_algo sh
依來源 IP 的靜態雜湊表分配。相同來源 IP 永遠導向同一伺服器（Session 親和性）。
Shortest Expected Delay
lb_algo sed
將連線分配給預期延遲最短的伺服器，計算公式：(Ci + 1) / Wi。
Never Queue
lb_algo nq
雙速模型：有閒置伺服器就直接派送，否則退回 SED 演算法。
⚡ Overflow-Connection (ovf)

所有連線優先送往最高權重伺服器，超過其 weight 值後溢出到下一個。不適用於 UDP（僅追蹤活躍連線）。

選擇建議：一般 Web 服務首選 wlc（加權最少連線）；需要 Session 黏著使用 sh（來源雜湊）；快取叢集使用 lblc/lblcr。

8 IPVS Protocol Support — 協定支援

IPVS 支援以下傳輸層協定的負載均衡：

協定說明
TCP最常用，涵蓋 HTTP、HTTPS、SMTP 等服務
UDPDNS、RADIUS、SIP 等 UDP 服務
SCTP串流控制傳輸協定，電信業常用
ESPIPSec 封裝安全載荷
AHIPSec 認證標頭

應用層輔助器（Application Helpers）

📁 FTP Protocol Helper

處理 FTP 主動/被動模式的連線追蹤，確保資料通道正確轉發。

📞 SIP Persistence Engine

基於 UDP 和 SIP Netfilter 連線追蹤，實現 SIP 會話持續性。

9 Configuring SNMP Support — SNMP 設定

Keepalived 提供 SNMP 子系統，MIB 基礎 OID：.1.3.6.1.4.1.9586.100.5（Debian IANA 空間）。

9.1 啟用步驟

① 安裝 SNMP 套件

yum install net-snmp net-snmp-utils net-snmp-libs

② 編譯時加入 SNMP 支援

./configure --enable-snmp

③ 啟用 AgentX（在 /etc/snmp/snmpd.conf 加入）

master agentx

④ 複製 MIB 檔案

cp KEEPALIVED-MIB /usr/share/snmp/mibs/
# 或放在 ~/.snmp/mibs/

9.2 查詢範例

# 使用 OID 查詢
snmpwalk -v2c -c public localhost .1.3.6.1.4.1.9586.100.5.1.1.0

# 使用 MIB 名稱查詢（需先安裝 MIB）
snmpwalk -v2c -c public localhost KEEPALIVED-MIB::version

9.3 MIB 四大區塊

區塊內容
GlobalKeepalived 版本、Router ID、管理者 Email
VRRP各 VRRP Instance 的名稱、當前狀態、VIP 清單
CheckVirtual Server / Real Server 的統計資訊、排程演算法、連線數
ConformanceMIB 一致性定義（文件待完善）
