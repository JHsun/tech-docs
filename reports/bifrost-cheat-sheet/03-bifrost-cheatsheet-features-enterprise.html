1. Fallbacks（自動容錯）

當主要 Provider 失敗時，Bifrost 自動依序嘗試備援 Provider，直到成功或全部失敗。每次 fallback 視為全新請求，所有 Plugin 重新執行。

觸發 Fallback 的情境
觸發不觸發（回傳原始錯誤）
網路連線問題請求格式錯誤
Provider 錯誤（500/502/503/504）Plugin 阻擋（Governance 違規）
限速（429）Provider 標記為不可重試的錯誤
模型不可用
請求逾時 / 認證失敗
使用方式
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
  "model": "openai/gpt-4o-mini",
  "messages": [{"role":"user","content":"Hello"}],
  "fallbacks": [
    "anthropic/claude-3-5-sonnet-20241022",
    "bedrock/anthropic.claude-3-sonnet-20240229-v1:0"
  ]
}'

# 回應中 extra_fields.provider 顯示實際處理的 Provider
💡 Go SDK：使用 Fallbacks: []schemas.Fallback{{Provider: schemas.Anthropic, Model: "..."}}

2. Load Balance（Key 管理 & 負載平衡）

Bifrost 透過加權隨機分配 API Key，支援模型專屬過濾和自動容錯。

功能說明
加權分配每個 Key 設定 weight，按比例分配流量
模型過濾models: ["gpt-4o"] 限制 Key 可用模型；["*"] 全部
自動容錯Key 失敗時自動切換到其他可用 Key
環境變數"value": "env.OPENAI_KEY" 引用環境變數
Async Inference（非同步推論）

支援非同步推論模式，適用於不需即時回應的批次作業。透過 /v1/batches 端點提交批次任務，Bifrost 追蹤完成狀態。

3. Governance（治理體系）

階層架構
Customer（最高，組織預算）→ Team（部門預算）→ Virtual Key（API 預算 + 限速）→ Provider Config（獨立預算 + 限速）

VK 只能歸屬一個 Team 或一個 Customer（互斥），也可獨立存在。

Virtual Key API
# 建立 VK（含 Provider 路由 + 預算 + 限速）
POST /api/governance/virtual-keys
{
  "name": "marketing-team-vk",
  "provider_configs": [
    {
      "provider": "openai",
      "weight": 0.7,
      "allowed_models": ["gpt-4o", "gpt-4o-mini"],
      "budget": {"max_limit": 500.00, "reset_duration": "1M", "calendar_aligned": true},
      "rate_limit": {
        "token_max_limit": 1000000, "token_reset_duration": "1h",
        "request_max_limit": 1000,  "request_reset_duration": "1h"
      }
    },
    {
      "provider": "anthropic",
      "weight": 0.3,
      "allowed_models": ["claude-3-opus", "claude-3-sonnet"],
      "budget": {"max_limit": 200.00, "reset_duration": "1M"}
    }
  ],
  "team_id": "team-eng-001",
  "budget": {"max_limit": 1000.00, "reset_duration": "1M", "calendar_aligned": true},
  "rate_limit": {
    "token_max_limit": 10000, "token_reset_duration": "1h",
    "request_max_limit": 100,  "request_reset_duration": "1m"
  },
  "key_ids": ["uuid-of-specific-provider-key"],
  "is_active": true
}

# CRUD
GET    /api/governance/virtual-keys
GET    /api/governance/virtual-keys/{id}
PUT    /api/governance/virtual-keys/{id}
DELETE /api/governance/virtual-keys/{id}
Team & Customer API
# Team
POST /api/governance/teams
  {"name":"Engineering","customer_id":"cust-001","budget":{"max_limit":500,"reset_duration":"1M"}}
GET/PUT/DELETE /api/governance/teams/{id}

# Customer
POST /api/governance/customers
  {"name":"Acme Corp","budget":{"max_limit":5000,"reset_duration":"1M"}}
GET/PUT/DELETE /api/governance/customers/{id}
Budget reset_duration 格式
格式說明Calendar Aligned
1d每日UTC 午夜重置
1w每週每週一 00:00 UTC
1M每月每月 1 日
1Y每年每年 1/1
⚠️ 注意：子日級（1h / 30m）不支援 calendar alignment。開啟 calendar_aligned 會重置目前用量。
請求時的 Governance Headers
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "x-bf-vk: vk-engineering-main" \        # 必要：Virtual Key
  -H "x-bf-team: team-eng-001" \              # 選用：追蹤用途
  -H "x-bf-customer: customer-acme-corp" \    # 選用：追蹤用途
  -d '{"model":"gpt-4o-mini","messages":[...]}'
Required Headers（強制 Header）

可設定必要 Header（如 X-Tenant-ID），缺少時回 400。Header 名稱不分大小寫。需啟用 Governance Plugin。

MCP Tool Filtering（VK 層級）

在 Virtual Key 上設定 MCP 工具白名單，嚴格控制每個 VK 可使用的工具。

4. Semantic Caching（語義快取）

利用向量相似度搜尋，對語意相近的請求回傳快取回應，大幅降低 API 成本和延遲。

雙層快取
層方式說明
Exact Hash精確比對完全相同的請求直接命中
Semantic向量相似度語意相近的請求（可調閾值）
所需元件
元件支援
Vector StoreWeaviate Redis/Valkey Qdrant Pinecone
Embedding Provider任何 Bifrost 支援的 Provider（如 OpenAI text-embedding-3-small）
設定（config.json）
{
  "vector_store": {
    "enabled": true,
    "type": "weaviate",
    "config": {"host":"localhost:8080","scheme":"http"}
  },
  "plugins": [{
    "enabled": true,
    "name": "semantic_cache",
    "config": {
      "provider": "openai",
      "embedding_model": "text-embedding-3-small",
      "dimension": 1536,
      "ttl": "5m",                              // 快取存活時間
      "threshold": 0.8,                          // 相似度閾值 (0-1)
      "cleanup_on_shutdown": true,
      "conversation_history_threshold": 3,       // 對話 > N 則跳過快取
      "exclude_system_prompt": false,
      "cache_by_model": true,
      "cache_by_provider": true
    }
  }]
}
Direct Hash Mode（無 Embedding）

只做精確比對，不需 Embedding Provider：設 dimension: 1 且省略 provider。仍需 Vector Store 作儲存後端。

Per-Request 覆寫
# 自訂 TTL 和閾值
curl -H "x-bf-cache-ttl: 10m" \
     -H "x-bf-cache-threshold: 0.9" \
     http://localhost:8080/v1/chat/completions ...

# 強制繞過快取
curl -H "x-bf-cache-type: none" ...

# 只用精確比對
curl -H "x-bf-cache-type: direct" ...

# 儲存但不讀取
curl -H "x-bf-cache-no-store: true" ...

5. Telemetry & Observability

內建 Observability（請求追蹤）

自動捕獲所有 AI 請求/回應，包含輸入、輸出、Token、成本、延遲。非同步寫入，零延遲影響。

# 啟用請求追蹤
PUT /api/config
{
  "client_config": {
    "enable_logging": true,
    "disable_content_logging": false
  }
}
監控整合
整合類型說明
Prometheus開源原生 scraping 或 Push Gateway；追蹤 HTTP + Provider 指標
OpenTelemetry開源OTLP 整合 Grafana / New Relic / Honeycomb
Maxim AI開源自動轉發至 Maxim 平台進行評估與分析
DatadogEnterprise原生 APM traces + LLM Observability
Telemetry（內建 Prometheus 指標）

Bifrost 內建 Prometheus 指標追蹤 HTTP 層級和上游 Provider 指標，無需額外設定。

6. Custom Plugins（自訂插件）

使用 Go 原生 Plugin 系統（.so 共享物件），在請求生命週期中攔截和修改資料。

⚠️ 前提：需先建置 dynamically linked Bifrost binary（預設為靜態連結以簡化部署）。
Plugin Hooks（v1.4.x+）
Hook時機用途
Init(config)初始化載入設定
GetName()—回傳 Plugin 名稱
HTTPTransportPreHookHTTP 請求進入 Bifrost 核心前Header 修改、認證、短路
PreLLMHook到達 Provider 前驗證、快取檢查、請求修改
PostLLMHookProvider 回應後日誌、回應修改、分析
HTTPTransportPostHookHTTP 回應離開 Bifrost 核心後回應包裝
Cleanup()關機釋放資源
建置與使用
# 編譯 Plugin
go build -buildmode=plugin -o myplugin.so main.go

# Plugin 放入 plugins/ 目錄，Bifrost 啟動時自動載入
Plugin Sequencing（執行順序）

PreHook 按設定順序執行，PostHook 則反向執行（LIFO）。任何 PreHook 可短路終止後續處理。

內建 Plugins
Plugin功能
governanceVK / Budget / Rate Limit / Routing
logging請求日誌與分析
semanticcache語義快取
telemetryPrometheus 指標
maximMaxim AI 可觀測性
mocker模擬 Provider 回應（測試用）
otelOpenTelemetry 整合
litellmcompatLiteLLM 相容

⚡ Adaptive Load Balancing

預測式擴展 + 即時健康監控。雙層架構（L1: Provider, L2: Key）。

評分公式：Error(50%) + Latency(20%) + Utilization(5%) - Momentum。

每 5 秒更新權重，30 秒內恢復 90% 懲罰，25% 探索率。

Security 摘要

Bifrost 永不轉發以下 Header：proxy-authorization, cookie, host, content-length, connection, transfer-encoding, x-api-key, x-goog-api-key, x-bf-api-key, x-bf-vk。所有敏感 Key 在 GET 回應中自動遮蔽。
