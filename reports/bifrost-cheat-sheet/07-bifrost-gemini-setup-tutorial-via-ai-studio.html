在 Google AI Studio 建立 Gemini API Key

Gemini API 使用 Google AI Studio 管理 API Key。取得 Key 完全免費，新使用者還會獲得免費額度。

ℹ️
前置條件 只需要一個 Google 帳號。不需要信用卡即可取得免費額度的 API Key。如果需要更高額度，可以之後再啟用 Google Cloud 帳單。
1
前往 Google AI Studio
瀏覽器打開 aistudio.google.com，使用你的 Google 帳號登入。
2
接受服務條款
首次使用會跳出 Generative AI 服務條款，閱讀後點擊「Accept」或「Continue」。接受後 Google 會自動為你建立一個預設的 Google Cloud 專案。
3
進入 API Keys 頁面
在左側選單點選 「Get API key」（取得 API 金鑰），或直接前往：
https://aistudio.google.com/app/apikey
4
建立 API Key
點擊 「Create API key」（建立 API 金鑰）按鈕。
如果有多個 Google Cloud 專案，選擇你要使用的專案（或使用預設專案）。
Key 建立後會顯示在畫面上，格式類似：AIzaSy...（約 39 個字元）。
5
複製並安全保存 API Key
點擊複製按鈕，將 Key 存到安全的地方（密碼管理器或環境變數）。
雖然 Google AI Studio 允許之後再次查看 Key，但建議立即保存。
💡
API Key 重要注意事項 每個 API Key 綁定一個 Google Cloud 專案。免費額度是以專案為單位計算的（不是每個 Key 獨立額度）。同一專案下建立多個 Key，它們共享同一個額度池。API Key 不會自動過期，除非你手動刪除或限制它。

（選用）驗證 API Key 可用

在終端機執行以下指令，確認 Key 能正常使用：

# 將你的 API Key 設為環境變數
export GEMINI_API_KEY="AIzaSy你的金鑰"

# 發送測試請求
curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent" \
  -H "x-goog-api-key: $GEMINI_API_KEY" \
  -H "Content-Type: application/json" \
  -X POST \
  -d '{
    "contents": [{
      "parts": [{"text": "Hello! 請用繁體中文回答。"}]
    }]
  }'

如果收到 JSON 回應且包含生成的文字，代表 API Key 設定成功。

Gemini 免費方案與模型

模型 免費額度 適合場景
gemini-2.5-flash ✅ 有免費額度 高速、低成本、一般用途
gemini-2.5-pro ✅ 有免費額度 複雜推理、高品質回答
gemini-2.0-flash ✅ 有免費額度 上一代快速模型
imagen-4.0-* ✅ 有免費額度 圖片生成

超過免費額度後，需在 Google Cloud Console 啟用帳單才能繼續使用。

在 Bifrost 設定 Gemini Provider

取得 API Key 後，將它設定到 Bifrost 的 Gemini Provider。提供三種設定方式：

方式 A：透過 Web UI（推薦新手）

1
打開 Bifrost 儀表板
瀏覽器前往 http://localhost:8080。
2
前往 Model Providers
左側選單點選 Model Providers（模型供應商）。
3
找到 Google Gemini 並新增 Key
點擊 Gemini 區塊 → Add Key → 輸入名稱（如 gemini-main）和 API Key 值 → Save。

方式 B：透過 API

curl -X POST http://localhost:8080/api/providers \
  -H "Content-Type: application/json" \
  -d '{
    "provider": "gemini",
    "keys": [{
      "name": "gemini-main",
      "value": "AIzaSy你的金鑰",
      "models": ["*"],
      "weight": 1.0
    }]
  }'
⚠️
安全建議 生產環境請使用環境變數引用："value": "env.GEMINI_API_KEY"，並在啟動前設定 export GEMINI_API_KEY=AIzaSy...。避免將真實 Key 寫入設定檔或版本控制。

方式 C：透過 config.json

{
  "providers": {
    "gemini": {
      "keys": [{
        "name": "gemini-main",
        "value": "env.GEMINI_API_KEY",
        "models": ["*"],
        "weight": 1.0
      }]
    }
  },
  "config_store": {
    "enabled": true,
    "type": "sqlite",
    "config": {"path": "./config.db"}
  }
}

多 Key 負載平衡（選用）

如果你有多個 Gemini API Key（例如不同 GCP 專案），可以設定加權分配：

curl -X POST http://localhost:8080/api/providers \
  -H "Content-Type: application/json" \
  -d '{
    "provider": "gemini",
    "keys": [
      {"name": "key-proj-a", "value": "env.GEMINI_KEY_A", "models": ["*"], "weight": 0.7},
      {"name": "key-proj-b", "value": "env.GEMINI_KEY_B", "models": ["*"], "weight": 0.3}
    ]
  }'

限制特定 Key 可用的模型（選用）

# 這個 Key 只能用於 flash 系列模型
"keys": [{
  "name": "flash-only",
  "value": "env.GEMINI_API_KEY",
  "models": ["gemini-2.5-flash", "gemini-2.0-flash"],
  "weight": 1.0
}]
✅

驗證 Gemini 連線成功

列出 Gemini 可用模型

curl http://localhost:8080/v1/models?provider=gemini

應該會回傳 Gemini 的模型列表（gemini-2.5-flash、gemini-2.5-pro 等）。

發送第一個 Gemini 請求

# Chat Completion
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini/gemini-2.5-flash",
    "messages": [{"role": "user", "content": "你好！請用繁體中文介紹你自己。"}]
  }'
✅
成功標誌 收到包含 Gemini 生成文字的 JSON 回應，且 extra_fields.provider 顯示 "gemini"。

測試串流回應

curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini/gemini-2.5-flash",
    "messages": [{"role": "user", "content": "寫一首關於程式設計的短詩"}],
    "stream": true
  }'

常見錯誤排查

錯誤 原因 解決方式
401 Unauthorized API Key 無效或格式錯誤 確認 Key 以 AIza 開頭，無多餘空格
403 Forbidden API Key 沒有 Generative Language API 權限 到 Google Cloud Console 確認 Key 未被限制
429 Rate Limited 超過免費額度或請求速率限制 等待重置或啟用 GCP 帳單
model not found 模型名稱錯誤 使用正確格式如 gemini/gemini-2.5-flash
Provider 設定後模型列表為空 API Key 未正確設定或 Bifrost 無法連線 Google API 確認網路連通性，Key 值是否正確

Gemini 進階設定

Gemini 認證方式

Bifrost 的 Gemini Provider 支援兩種認證方式，會根據端點類型自動選擇：

方式 使用場景 技術細節
Header 標準 Gemini 端點 自動帶 x-goog-api-key: KEY header
Query Param Imagen 模型 / 自訂端點 自動附加 ?key=KEY 到 URL

你不需要手動選擇，Bifrost 會自動根據端點類型決定認證方式。

Gemini 專用參數

Bifrost 會自動轉換 OpenAI 格式的參數到 Gemini 格式。如果需要使用 Gemini 專屬參數，可透過 extra_params 或直接在 request body 中傳入： 

# Gemini 專屬參數（透過 request body 直接傳入）
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini/gemini-2.5-flash",
    "messages": [{"role": "user", "content": "Hello"}],
    "top_k": 40,
    "stop_sequences": ["###"]
  }'
OpenAI 參數 Gemini 轉換結果
max_completion_tokens → maxOutputTokens
temperature → 直接傳遞
top_p → 直接傳遞
stop → stopSequences
response_format → responseMimeType + responseJsonSchema
reasoning.effort → thinkingConfig.thinkingLevel

Thinking / Reasoning（深度思考模式）

curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini/gemini-2.5-flash",
    "messages": [{"role": "user", "content": "解釋量子糾纏"}],
    "reasoning": {
      "effort": "high",
      "max_tokens": 10000
    }
  }'
# Bifrost 自動轉換為 Gemini 的 thinkingConfig：
# {"thinkingConfig": {"includeThoughts": true, "thinkingLevel": "HIGH", "thinkingBudget": 10000}}

多模態（圖片輸入）

curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini/gemini-2.5-flash",
    "messages": [{
      "role": "user",
      "content": [
        {"type": "text", "text": "描述這張圖片"},
        {"type": "image_url", "image_url": {"url": "https://example.com/photo.jpg"}}
      ]
    }]
  }'

圖片生成（Imagen）

curl -X POST http://localhost:8080/v1/images/generations \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini/imagen-4.0-generate-001",
    "prompt": "一隻在月光下的貓",
    "n": 1,
    "size": "1024x1024"
  }'

語音轉文字 / 文字轉語音

TTS（文字轉語音）
POST /v1/audio/speech
{
  "model": "gemini/gemini-2.5-flash",
  "input": "你好世界",
  "voice": "Kore"
}
STT（語音轉文字）
POST /v1/audio/transcriptions
  form: file=@audio.mp3
  form: model=gemini/gemini-2.5-flash

Fallback（容錯）設定

可將 Gemini 作為主要 Provider，其他 Provider 作為備援：

curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini/gemini-2.5-flash",
    "messages": [{"role": "user", "content": "Hello!"}],
    "fallbacks": [
      "openai/gpt-4o-mini",
      "anthropic/claude-sonnet-4-20250514"
    ]
  }'

使用者端整合（使用 Gemini 模型）

使用者只需要知道 Bifrost 位址和 Virtual Key，就能透過統一 API 使用 Gemini（以及你設定的其他所有 Provider）。

Python（OpenAI SDK）

import openai

client = openai.OpenAI(
    base_url="http://your-bifrost:8080/openai",
    api_key="sk-bf-xxxxxxxx"  # Bifrost Virtual Key
)

# 使用 Gemini 模型
response = client.chat.completions.create(
    model="gemini/gemini-2.5-flash",
    messages=[{"role": "user", "content": "你好！"}]
)
print(response.choices[0].message.content)

cURL

curl -X POST http://your-bifrost:8080/v1/chat/completions \
  -H "Authorization: Bearer sk-bf-xxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini/gemini-2.5-flash",
    "messages": [{"role": "user", "content": "你好！"}]
  }'

Google GenAI SDK（原生 Gemini SDK）

Bifrost 也提供 GenAI SDK 相容端點：

# 使用 Bifrost 的 GenAI 整合端點
# Base URL: http://your-bifrost:8080/genai

環境變數（零修改整合）

# 設定後，既有使用 OpenAI SDK 的程式碼自動透過 Bifrost
export OPENAI_BASE_URL=http://your-bifrost:8080/openai
export OPENAI_API_KEY=sk-bf-xxxxxxxx

# 模型名稱改為 gemini/gemini-2.5-flash 即可

完整流程回顧

步驟 做了什麼 在哪裡做
① 建立 API Key 在 Google AI Studio 建立 Gemini API Key aistudio.google.com
② 設定 Provider 將 API Key 加入 Bifrost 的 Gemini Provider Bifrost Web UI / API / config.json
③ 驗證連線 列出模型 + 發送測試請求 curl / Web UI
④ 進階設定 Thinking、多模態、Imagen、Fallback Bifrost API
⑤ 使用者整合 用 Virtual Key + gemini/model 存取 使用者的 SDK / cURL
💡
核心要點 整個流程只需要一個 Gemini API Key（在 Google AI Studio 免費取得），設定到 Bifrost 後，使用者透過 Bifrost 的 Virtual Key 和統一 API 存取，不需要直接接觸 Google 的 Key。Bifrost 自動處理 OpenAI 格式到 Gemini 格式的轉換、認證方式選擇、以及錯誤處理。
