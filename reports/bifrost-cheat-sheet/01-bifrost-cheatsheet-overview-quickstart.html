1. Bifrost 是什麼

Bifrost 是一套以 Go 語言撰寫的高效能 AI 閘道器（Gateway），將 23+ 家 LLM 供應商統一在單一 OpenAI 相容 API 後面，在 5,000 RPS 下僅增加 11 µs 延遲。採用 Apache 2.0 開源授權，由 Maxim AI 維護。

🏗️ 三種部署模式
模式適用場景特色
Gateway (HTTP) 語言無關、微服務、生產環境 內建 Web UI、即時監控、零配置啟動
Go SDK Go 應用直接嵌入 最高效能、原生 Go API、自訂中介軟體
Drop-in 替換 既有應用遷移、零程式碼修改 僅改 base URL，保留原 SDK
🔄 請求流程
Client Request→ FastHTTP 解析→ SDK 整合層→ Middleware Chain→ PreLLMHook→ MCP 工具注入→ Provider Queue→ Key 選擇 (~10ns)→ Provider API→ PostLLMHook→ Response
📊 效能基準
指標t3.medium (2C/4G)t3.xlarge (4C/16G)
Bifrost 額外延遲59 µs11 µs
成功率 @ 5k RPS100%100%
佇列等待時間47 µs1.67 µs
平均延遲（含 Provider）2.12 s1.61 s
Key 選擇時間~10 ns~10 ns
💡 對比：號稱比 LiteLLM 快 50 倍（LiteLLM ~600µs/req），P99 延遲低約 54 倍。

4. 認證（Auth）設定

透過 Web UI：Dashboard → Workspace → Config → Security 開啟密碼保護。

🔑 Virtual Key 傳遞方式
Header格式適用情境
x-bf-vkBifrost 原生任何情境（包含 auth 開啟時）
Authorization: Bearer sk-bf-*OpenAI 風格僅 auth 關閉時
x-api-key: sk-bf-*Anthropic 風格僅 auth 關閉時
x-goog-api-key: sk-bf-*Google 風格僅 auth 關閉時
Auth 關閉時（推論不驗證）
# 方式 A：x-bf-vk header
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "x-bf-vk: <VIRTUAL_KEY>" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o-mini", "messages": [...]}'

# 方式 B：Authorization header（OpenAI 風格）
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Authorization: Bearer <VIRTUAL_KEY>" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o-mini", "messages": [...]}'
Auth 開啟時（推論也驗證）
# 必須同時提供帳密 + Virtual Key
curl -X POST http://localhost:8080/v1/chat/completions \
  -u "your-username:your-password" \
  -H "x-bf-vk: <VIRTUAL_KEY>" \
  -H "Content-Type: application/json" \
  -d '{"model": "openai/gpt-4o", "messages": [...]}'
⚠️ 注意：舊版 Virtual Key（無 sk-bf-* 前綴）僅支援 x-bf-vk header。Session 有效期 30 天。

5. Provider 配置

透過 API 新增 Provider
OpenAI
curl -X POST http://localhost:8080/api/providers \
  -H "Content-Type: application/json" \
  -d '{
  "provider": "openai",
  "keys": [{
    "name": "openai-key-1",
    "value": "env.OPENAI_API_KEY",
    "models": ["*"],
    "weight": 1.0
  }]
}'
Anthropic
curl -X POST http://localhost:8080/api/providers \
  -H "Content-Type: application/json" \
  -d '{
  "provider": "anthropic",
  "keys": [{
    "name": "anthropic-key-1",
    "value": "env.ANTHROPIC_API_KEY",
    "models": ["*"],
    "weight": 1.0
  }]
}'
💡 Key Value 語法："env.VARIABLE_NAME" → 讀取環境變數；"sk-proj-xxx" → 直接寫入金鑰值。所有機密在 GET 時自動遮蔽。
⚖️ 加權負載平衡（70/30 分流）
curl -X POST http://localhost:8080/api/providers \
  -H "Content-Type: application/json" \
  -d '{
  "provider": "openai",
  "keys": [
    {"name": "key-1", "value": "env.KEY_1", "models": ["*"], "weight": 0.7},
    {"name": "key-2", "value": "env.KEY_2", "models": ["*"], "weight": 0.3}
  ]
}'
🎯 模型專屬 Key
"keys": [
  {"name": "standard", "value": "env.KEY",         "models": ["gpt-4o", "gpt-4o-mini"], "weight": 1.0},
  {"name": "premium",  "value": "env.KEY_PREMIUM",  "models": ["o1-preview", "o1-mini"],  "weight": 1.0}
]

設定預設說明
max_retries5最大重試次數
retry_backoff_initial_ms1初始退避毫秒
retry_backoff_max_ms10000最大退避毫秒
concurrency1000並行工作數

default_request_timeout_in_seconds—請求逾時
📤 自訂 Header
類型方式說明
靜態 network_config.extra_headers Provider 層級，永遠附加
動態 請求加 x-bf-eh-* 前綴 轉發時自動去除前綴（x-bf-eh-user-id → user-id）

6. 串流回應（Streaming）

⚠️ 關鍵行為：usage 和 finish_reason 只在最後一個 chunk 才送出。預設串流逾時 30 秒。
Chat Completion 串流
curl http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
  "model": "openai/gpt-4o-mini",
  "messages": [{"role": "user", "content": "Tell me a story"}],
  "stream": true
}'

# SSE 回應格式：
data: {"choices":[{"delta":{"content":"Once"}}],"model":"gpt-4o-mini"}
data: {"choices":[{"delta":{"content":" upon"}}],"model":"gpt-4o-mini"}
data: [DONE]
Responses API 串流（事件式 SSE，無 [DONE]）
event: response.created
data: {"type":"response.created"}

event: response.output_text.delta
data: {"type":"response.output_text.delta","delta": ...}

event: response.completed
data: {"type":"response.completed","response":{...}}
TTS / STT 串流
類型Endpoint關鍵參數
TTSPOST /v1/audio/speech"stream_format": "sse"
STTPOST /v1/audio/transcriptionsstream=true（form-data）

curl http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
  "model": "openai/gpt-4o-mini",
  "messages": [{"role": "user", "content": "What is 15 + 27?"}],
  "tools": [{
    "type": "function",
    "function": {
      "name": "calculator",
      "description": "Basic arithmetic",
      "parameters": {
        "type": "object",
        "properties": {
          "operation": {"type": "string", "enum": ["add","subtract","multiply","divide"]},
          "a": {"type": "number"},
          "b": {"type": "number"}
        },
        "required": ["operation", "a", "b"]
      }
    }
  }],
  "tool_choice": "auto"
}'
tool_choice 選項
值行為
"auto"AI 自行決定（預設）
"none"停用工具
{"type":"function","function":{"name":"xxx"}}強制指定工具

curl http://localhost:8080/api/mcp/client \

  "connection_type": "stdio",
  "stdio_config": {
    "command": ["npx", "@modelcontextprotocol/server-filesystem", "/tmp"],
    "args": []
  }
}'

# 連接類型：stdio（本機）| http（遠端）| sse（串流）

12. Go SDK

go mod init my-bifrost-app
go get github.com/maximhq/bifrost/core
SDK 方法速查
方法用途
bifrost.Init(ctx, config)初始化客戶端
client.Shutdown()優雅關閉
client.ChatCompletionRequest(ctx, req)非串流聊天
client.ChatCompletionStreamRequest(ctx, req)串流聊天
client.TextCompletionStreamRequest(ctx, req)串流文字完成

client.SpeechStreamRequest(ctx, req)TTS 串流
client.TranscriptionStreamRequest(ctx, req)STT 串流
最小範例
package main

import (
    "context"; "fmt"; "os"
    "github.com/maximhq/bifrost/core"
    "github.com/maximhq/bifrost/core/schemas"
)

// 實作 Account 介面（3 個方法）
type MyAccount struct{}
func (a *MyAccount) GetConfiguredProviders() ([]schemas.ModelProvider, error) {
    return []schemas.ModelProvider{schemas.OpenAI}, nil
}
func (a *MyAccount) GetKeysForProvider(ctx *context.Context, p schemas.ModelProvider) ([]schemas.Key, error) {
    return []schemas.Key{{Value: os.Getenv("OPENAI_API_KEY"), Models: []string{}, Weight: 1.0}}, nil
}
func (a *MyAccount) GetConfigForProvider(p schemas.ModelProvider) (*schemas.ProviderConfig, error) {
    return &schemas.ProviderConfig{
        NetworkConfig:            schemas.DefaultNetworkConfig,
        ConcurrencyAndBufferSize: schemas.DefaultConcurrencyAndBufferSize,
    }, nil
}

func main() {
    client, _ := bifrost.Init(context.Background(), schemas.BifrostConfig{Account: &MyAccount{}})
    defer client.Shutdown()

    resp, _ := client.ChatCompletionRequest(
        schemas.NewBifrostContext(context.Background(), schemas.NoDeadline),
        &schemas.BifrostChatRequest{
            Provider: schemas.OpenAI,
            Model:    "gpt-4o-mini",
            Input: []schemas.ChatMessage{{
                Role:    schemas.ChatMessageRoleUser,
                Content: &schemas.ChatMessageContent{ContentStr: schemas.Ptr("Hello!")},
            }},
        },
    )
    fmt.Println(*resp.Choices[0].Message.Content.ContentStr)
}

13. v1.5.0 遷移指南

🚨 10 項破壞性變更（v1.5.0-prerelease1, 2026-04-01）
#變更影響
1空陣列 [] 現在表示「全部拒絕」config / REST API
2allowed_keys → key_idsVK Provider Config
3VK provider_configs 改為預設拒絕（空 = 無 Provider）config / REST API
4WhiteList 驗證：萬用字元不可混用、禁止重複REST API
5weight 改為 nullable（*float64）REST API / Go SDK
6Provider Keys API 分離 — keys 從 provider payload 移出REST API
7enable_litellm_fallbacks → 三個細粒度選項Plugin Config
8Replicate image edits 從 /v1/images/generations 移除REST API
9Ollama / SGL 改為 per-key URL 設定config / REST API
10Go SDK：HuggingFaceKeyConfig 移除等Go SDK
⚠️ 最重要：空陣列語意統一
[]     = 全部拒絕（deny all）— 不允許任何工具/Key/模型
["*"]  = 全部允許（allow all）— 萬用字元
新功能亮點
功能說明
專用 Provider Keys APIKey 改由 /api/providers/{provider}/keys 管理
MCP 工具過濾 Headerx-bf-mcp-include-clients, x-bf-mcp-include-tools
Pricing Override範圍內定價覆寫
MCP Virtual Key 分配Per-tool 存取控制
BlackList 模型欄位新增黑名單支援
並行化模型列表更快啟動速度
