使用 Google Cloud Platform
設定 Bifrost 的 Gemini / Vertex AI

涵蓋兩種路徑：GCP API Key（簡易）與 Service Account JSON（生產環境），以及 Bifrost 的 gemini 與 vertex 兩種 Provider 差異

🗺️

先搞清楚：GCP 有兩種方式存取 Gemini，對應 Bifrost 的兩種 Provider

⚠️
重要觀念：Google AI Studio 的 API Key ≠ GCP 的 API Key ≠ Service Account 它們是三種不同的認證方式，對應 Bifrost 不同的設定方式。本教學專門針對「你使用 Google Cloud Platform（而非 AI Studio）」的場景。

路徑 A — GCP API Key + gemini Provider

在 GCP Console 建立 API Key

GCP 的 API Key 跟 AI Studio 的 API Key 格式相同，但建立的地方不同。如果你是 GCP 使用者，可以直接在 Cloud Console 建立。

方式一：透過 Vertex AI Express Mode（最快）

1
前往 Vertex AI Express Mode
瀏覽器打開 console.cloud.google.com/vertex-ai，選擇你的專案。
2
取得 Express Mode API Key
GCP 會引導你建立或取得一個 API Key。新使用者會獲得 90 天免費試用。既有使用者可連結帳單帳戶直接取得 Key。

方式二：透過 GCP Console 手動建立

1
啟用 Vertex AI API
GCP Console → APIs & Services → Enable APIs → 搜尋 「Vertex AI API」 → 點擊 Enable。
同時啟用 「Generative Language API」（Gemini API 也需要此 API）。
2
建立 API Key
GCP Console → APIs & Services → Credentials → Create Credentials → API Key。建立後會顯示 Key（格式 AIza...）。
3
（建議）限制 API Key
點擊新建的 Key → API restrictions → 選擇 Restrict key → 勾選 Generative Language API → Save。這樣 Key 只能用於 Gemini 相關 API。

設定到 Bifrost（gemini Provider）

curl -X POST http://localhost:8080/api/providers \
  -H "Content-Type: application/json" \
  -d '{
    "provider": "gemini",
    "keys": [{
      "name": "gcp-gemini-key",
      "value": "env.GEMINI_API_KEY",
      "models": ["*"],
      "weight": 1.0
    }]
  }'
# 環境變數
export GEMINI_API_KEY="AIzaSy你的GCP金鑰"

驗證

curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemini/gemini-2.5-flash",
    "messages": [{"role": "user", "content": "你好！"}]
  }'
ℹ️
GCP API Key 與 AI Studio API Key 的差異 技術上格式相同（都是 AIza...），但 GCP 的 Key 綁定你在 Cloud Console 中的專案和帳單帳戶，可透過 Cloud Console 設定更細緻的限制（IP 限制、API 限制等）。AI Studio 的 Key 也是背後綁定 GCP 專案，但管理介面不同。
路徑 B — Service Account + vertex Provider（推薦生產環境）
🏗️

GCP 前置準備

1
確認 GCP 專案和帳單
前往 console.cloud.google.com，選擇或建立一個專案。確認帳單帳戶已連結（Vertex AI 需要帳單）。記下 Project ID（如 my-project-123456）。
2
啟用 Vertex AI API
GCP Console → APIs & Services → Enable APIs and Services → 搜尋 「Vertex AI API」 → 點擊 Enable。
👤

建立 Service Account 並下載 JSON Key

1
進入 IAM & Admin → Service Accounts
GCP Console → 左側選單 → IAM & Admin → Service Accounts。
2
建立 Service Account
點擊 「+ CREATE SERVICE ACCOUNT」：
— Service account name：如 bifrost-vertex
— Service account ID：自動生成（如 bifrost-vertex@my-project.iam.gserviceaccount.com）
— 點擊 Create and Continue
3
授予角色
在 「Grant this service account access to project」 步驟：
— 搜尋並選擇 「Vertex AI User」（roles/aiplatform.user）
— 此角色包含使用 Vertex AI 預測 API 所需的全部權限
— 點擊 Continue → Done
4
建立 JSON Key
在 Service Accounts 列表中點擊剛建立的帳戶 →
上方分頁選 「Keys」 →
Add Key → Create new key →
選擇 JSON → Create
瀏覽器會自動下載一個 .json 檔案。妥善保管此檔案！
🔒
安全警告 Service Account JSON Key 等同於完整的身分認證。不要將它提交到 Git、分享到公開場所、或寫入不安全的位置。建議使用環境變數或 Secret Manager 管理。

{
  "type": "service_account",

在 Bifrost 設定 Vertex AI Provider

Bifrost 的 vertex Provider 使用 vertex_key_config 設定，需要 Project ID、Region、和認證資訊。

認證方式比較

方式設定適合場景
Service Account JSON auth_credentials 填入 JSON 內容 生產環境、遠端伺服器
ADC（Application Default Credentials） auth_credentials 留空 GCP VM / GKE / Cloud Run / 本機開發
GCP API Key value 填入 API Key 快速測試（僅 Gemini 模型）

方式一：Service Account JSON（推薦生產環境）

透過 API

# 先將 JSON Key 內容存為環境變數（一行化）
export VERTEX_CREDENTIALS=$(cat /path/to/service-account-key.json)
export VERTEX_PROJECT_ID="my-project-123456"

# 設定 Vertex Provider
curl -X POST http://localhost:8080/api/providers \
  -H "Content-Type: application/json" \
  -d '{
    "provider": "vertex",
    "keys": [{
      "name": "vertex-sa-key",
      "value": "dummy",
      "models": ["*"],
      "weight": 1.0,
      "vertex_key_config": {
        "project_id": "env.VERTEX_PROJECT_ID",
        "region": "us-central1",
        "auth_credentials": "env.VERTEX_CREDENTIALS"
      }
    }]
  }'

透過 config.json

{
  "providers": {
    "vertex": {
      "keys": [{
        "name": "vertex-sa-key",
        "value": "dummy",
        "models": ["*"],
        "weight": 1.0,
        "vertex_key_config": {
          "project_id": "env.VERTEX_PROJECT_ID",
          "region": "us-central1",
          "auth_credentials": "env.VERTEX_CREDENTIALS"
        }
      }]
    }
  }
}

方式二：ADC（在 GCP 環境中自動認證）

如果 Bifrost 跑在 GCP VM、GKE、Cloud Run 等環境，可以使用 Application Default Credentials，不需要手動管理 Key：

{
  "providers": {
    "vertex": {
      "keys": [{
        "name": "vertex-adc",
        "value": "",
        "models": ["*"],
        "weight": 1.0,
        "vertex_key_config": {
          "project_id": "my-project-123456",
          "region": "us-central1"
        }
      }]
    }
  }
}
// auth_credentials 留空 → Bifrost 自動使用環境的 ADC

本機開發時，先用 gcloud 設定 ADC：

gcloud auth application-default login
# 或指定 Service Account
export GOOGLE_APPLICATION_CREDENTIALS="/path/to/service-account-key.json"

方式三：GCP API Key（僅限 Gemini + Fine-tuned 模型）

{
  "providers": {
    "vertex": {
      "keys": [{
        "name": "vertex-apikey",
        "value": "env.GOOGLE_API_KEY",
        "models": ["*"],
        "weight": 1.0,
        "vertex_key_config": {
          "project_id": "my-project-123456",
          "region": "us-central1"
        }
      }]
    }
  }
}
⚠️
API Key 限制 Vertex AI 的 API Key 認證只支援 Gemini 模型和 Fine-tuned 模型。如果要用 Anthropic Claude on Vertex，必須使用 Service Account 或 ADC。

Region 選擇

Region端點說明
us-central1us-central1-aiplatform.googleapis.com美國中部（最多模型可用）
us-west1us-west1-aiplatform.googleapis.com美國西部
europe-west1europe-west1-aiplatform.googleapis.com歐洲西部
asia-northeast1asia-northeast1-aiplatform.googleapis.com日本東京
globalaiplatform.googleapis.com全域（部分模型可用）

不同 Region 可用的模型不同，建議使用 us-central1 取得最廣泛的模型支援。查詢各 Region 可用模型：GCP 文件。

列出可用模型

測試 Gemini on Vertex

curl -X POST http://localhost:8080/v1/chat/completions \

💡
Vertex AI 的獨特優勢 透過 vertex/ Provider，你可以用同一個 GCP 專案同時存取 Gemini 和 Claude 模型。Bifrost 會根據模型名稱自動判斷路由到 Gemini API 或 Anthropic API。

進階：Fine-tuned 模型與 Deployments

如果你在 Vertex AI 上有自訂 fine-tuned 模型，可以透過 deployments 和 aliases 映射：

{
  "vertex_key_config": {
    "project_id": "my-project-123456",
    "project_number": "123456789012",
    "region": "us-central1",
    "auth_credentials": "env.VERTEX_CREDENTIALS",
    "deployments": {
      "my-gemini-ft": "1234567890",
      "my-custom-claude": "claude-3-5-sonnet-v2@20241022"
    }
  }
}

使用時："model": "vertex/my-gemini-ft"，Bifrost 自動映射到 deployment ID。

ℹ️
Fine-tuned 模型需要 project_number 設定中的 project_number（純數字）與 project_id（含字母的 ID）不同。在 GCP Console 的專案設定頁面可找到。
👩‍💻

使用者端整合

使用者不需要知道後面是 GCP 還是 AI Studio，只需要 Bifrost 位址和 Virtual Key：

# 使用 Vertex 上的 Gemini
curl -X POST http://bifrost:8080/v1/chat/completions \
  -H "Authorization: Bearer sk-bf-xxxxxxxx" \
  -d '{"model": "vertex/gemini-2.5-flash", "messages": [...]}'

# 使用 Vertex 上的 Claude
curl -X POST http://bifrost:8080/v1/chat/completions \
  -H "Authorization: Bearer sk-bf-xxxxxxxx" \
  -d '{"model": "vertex/claude-sonnet-4-20250514", "messages": [...]}'

# Python OpenAI SDK
import openai
client = openai.OpenAI(base_url="http://bifrost:8080/openai", api_key="sk-bf-xxx")
resp = client.chat.completions.create(
    model="vertex/gemini-2.5-flash",
    messages=[{"role": "user", "content": "你好！"}]
)
