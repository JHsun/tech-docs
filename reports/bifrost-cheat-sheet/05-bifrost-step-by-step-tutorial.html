安裝並啟動 Bifrost

Bifrost 支援 NPX 和 Docker 兩種安裝方式，零設定即可啟動。

方式 A：NPX（推薦本機開發）

需要 Node.js 18+

npx -y @maximhq/bifrost
方式 B：Docker（推薦生產環境）
# 基本啟動
docker run -p 8080:8080 maximhq/bifrost

# 持久化設定（重啟不遺失）
docker run -p 8080:8080 \
  -v $(pwd)/data:/app/data \
  maximhq/bifrost
✅
啟動成功的標誌 終端機顯示 Bifrost 已在 http://localhost:8080 運行。打開瀏覽器前往該網址即可看到 Web UI 儀表板。

確認啟動

# 健康檢查
curl http://localhost:8080/health

# 打開 Web UI（macOS）
open http://localhost:8080
🔑

設定線上模型的 API Key

這是最重要的步驟。你需要把各家 AI 供應商（OpenAI、Anthropic、Google 等）的 API Key 設定進 Bifrost，讓它能代理請求。

ℹ️
前置準備 請先到各家供應商的網站取得你的 API Key： OpenAI · Anthropic · Google Gemini · Groq · Mistral

方式 A：透過 Web UI 設定（推薦新手）

1
打開 Web UI
瀏覽器前往 http://localhost:8080，進入 Bifrost 儀表板。
2
前往 Model Providers 頁面
在左側選單點選 「Model Providers」（或「Configurations」），會看到可設定的 Provider 列表。
3
選擇 Provider 並輸入 API Key
點選你要設定的 Provider（如 OpenAI），在 Keys 區域點「Add Key」，輸入名稱和 API Key 值。設定完畢後點擊「Save」。
4
設定更多 Provider（選用）
重複步驟 3，新增 Anthropic、Google Gemini、Groq 等。每個 Provider 可設定多個 Key 做負載平衡。

方式 C：透過 config.json（進階 / GitOps）

將 config.json 放在 app-dir 目錄下（預設 ~/.config/bifrost/）：

{
  "providers": {
    "openai": {
      "keys": [{
        "name": "openai-key",
        "value": "env.OPENAI_API_KEY",
        "models": ["*"],
        "weight": 1.0
      }]
    },
    "anthropic": {
      "keys": [{
        "name": "anthropic-key",
        "value": "env.ANTHROPIC_API_KEY",
        "models": ["*"],
        "weight": 1.0
      }]
    },
    "gemini": {
      "keys": [{
        "name": "gemini-key",
        "value": "env.GEMINI_API_KEY",
        "models": ["*"],
        "weight": 1.0
      }]
    }
  },
  "config_store": {
    "enabled": true,
    "type": "sqlite",
    "config": {"path": "./config.db"}
  }
}
💡
安全提示 使用 "value": "env.OPENAI_API_KEY" 語法可以引用環境變數，避免把真實 Key 寫在設定檔中。在執行前設定環境變數：export OPENAI_API_KEY=sk-proj-xxx

Key 設定欄位說明

欄位說明範例
nameKey 的識別名稱（自訂）"my-openai-key"
valueAPI Key 值或環境變數引用"sk-proj-xxx" 或 "env.OPENAI_API_KEY"
models此 Key 可用的模型（["*"] = 全部）["gpt-4o", "gpt-4o-mini"]
weight負載平衡權重（多 Key 時按比例分配）1.0

保護 Dashboard（設定認證）

在生產環境中，你應該為 Bifrost 儀表板設定密碼保護，防止未授權存取。

1
進入安全設定
Web UI → 左側 Workspace → Config → Security 分頁。
2
開啟密碼保護
啟用 「Password protect the dashboard」開關，輸入管理員帳號和密碼。
3
設定推論呼叫是否需要認證
建議開啟 「Disable authentication on inference calls」，這樣：
— Dashboard 和管理 API 需要帳密登入（受保護）
— 推論 API（/v1/chat/completions 等）不需帳密，使用者只需 Virtual Key 即可
這是最常見的生產設定。
4
儲存
點擊 Save Changes，立即生效，不需重啟。

建立 Virtual Key（給使用者用的金鑰）

Virtual Key 是 Bifrost 的核心治理概念。你不會把真實的 OpenAI / Anthropic API Key 給使用者，而是發給他們一個 Bifrost Virtual Key，透過它控制存取權限、預算上限、可用模型等。

方式 A：透過 Web UI

1
前往 Governance → Virtual Keys
在左側選單找到 Virtual Keys 頁面。
2
建立新的 Virtual Key
點擊「New Virtual Key」，填入名稱、（選用）設定允許的 Provider / 模型 / 預算限制。
3
複製 Virtual Key 值
建立成功後會顯示 sk-bf-xxxxxxxx 格式的 Key。妥善保存！這是使用者連接 Bifrost 的唯一憑證。

方式 B：透過 API

# 建立基本 Virtual Key
curl -X POST http://localhost:8080/api/governance/virtual-keys \
  -H "Content-Type: application/json" \
  -d '{
    "name": "user-alice-vk",
    "is_active": true
  }'

# 建立帶完整治理規則的 Virtual Key
curl -X POST http://localhost:8080/api/governance/virtual-keys \
  -H "Content-Type: application/json" \
  -d '{
    "name": "team-engineering",
    "provider_configs": [
      {
        "provider": "openai",
        "weight": 0.5,
        "allowed_models": ["gpt-4o", "gpt-4o-mini"]
      },
      {
        "provider": "anthropic",
        "weight": 0.5,
        "allowed_models": ["claude-sonnet-4-20250514"]
      }
    ],
    "budget": {
      "max_limit": 100.00,
      "reset_duration": "1M"
    },
    "rate_limit": {
      "request_max_limit": 1000,
      "request_reset_duration": "1h"
    },
    "is_active": true
  }'

回應中會包含 Virtual Key 值（sk-bf-xxxxxxxx），把它發給你的使用者。

Virtual Key 可控制什麼？

功能說明
允許的 Provider / Model限定使用者只能存取特定模型
預算上限月度 / 週度 / 日度的花費上限
限速每小時最大 Token 數 / 請求數
加權路由多 Provider 間的流量分配
啟用 / 停用即時切斷或恢復存取
🧪

使用 Virtual Key 發出請求（驗證完整流程）

模擬使用者角色，用 Virtual Key 發出請求：

# 使用 x-bf-vk header（Bifrost 原生方式）
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "x-bf-vk: sk-bf-xxxxxxxxxxxxxxxx" \
  -d '{
    "model": "openai/gpt-4o-mini",
    "messages": [{"role": "user", "content": "Hello from a Virtual Key user!"}]
  }'

# 或使用 Authorization header（OpenAI SDK 相容方式）
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-bf-xxxxxxxxxxxxxxxx" \
  -d '{
    "model": "anthropic/claude-sonnet-4-20250514",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
✅
如果收到正常回應，恭喜！完整流程驗證成功。 使用者只需要知道 Bifrost 位址和自己的 Virtual Key，就能存取所有你開放的 AI 模型。

使用者端整合指南

以下是你發給使用者的「使用說明」。他們不需要知道背後的 OpenAI / Anthropic Key，只需要 Bifrost 位址和 Virtual Key。

Python（OpenAI SDK）

import openai

client = openai.OpenAI(
    base_url="http://your-bifrost-server:8080/openai",  # Bifrost 位址
    api_key="sk-bf-xxxxxxxxxxxxxxxx"                     # Virtual Key
)

response = client.chat.completions.create(
    model="openai/gpt-4o-mini",   # 或 anthropic/claude-xxx, gemini/xxx
    messages=[{"role": "user", "content": "你好！"}]
)
print(response.choices[0].message.content)

Python（Anthropic SDK）

import anthropic

client = anthropic.Anthropic(
    base_url="http://your-bifrost-server:8080/anthropic",
    api_key="sk-bf-xxxxxxxxxxxxxxxx"
)

message = client.messages.create(
    model="anthropic/claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[{"role": "user", "content": "你好！"}]
)
print(message.content[0].text)

JavaScript / TypeScript

import OpenAI from 'openai';

const client = new OpenAI({
  baseURL: 'http://your-bifrost-server:8080/openai',
  apiKey: 'sk-bf-xxxxxxxxxxxxxxxx',
});

const response = await client.chat.completions.create({
  model: 'openai/gpt-4o-mini',
  messages: [{ role: 'user', content: '你好！' }],
});
console.log(response.choices[0].message.content);

cURL（最基本）

curl -X POST http://your-bifrost-server:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-bf-xxxxxxxxxxxxxxxx" \
  -d '{
    "model": "openai/gpt-4o-mini",
    "messages": [{"role": "user", "content": "你好！"}]
  }'

環境變數設定（讓既有程式碼零修改）

# 設定環境變數後，既有使用 OpenAI SDK 的程式碼不需修改
export OPENAI_BASE_URL=http://your-bifrost-server:8080/openai
export OPENAI_API_KEY=sk-bf-xxxxxxxxxxxxxxxx

# 你的既有 Python 程式碼直接就能透過 Bifrost 運作
python my_existing_app.py

發給使用者的資訊清單

項目值
Bifrost 位址http://your-bifrost-server:8080
OpenAI 相容 Base URLhttp://your-bifrost-server:8080/openai
Anthropic 相容 Base URLhttp://your-bifrost-server:8080/anthropic

進階設定：Fallback 與多模型路由

自動容錯（Fallback）

使用者可以在請求中指定 fallback 模型。主要 Provider 失敗時自動切換：

curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Authorization: Bearer sk-bf-xxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-4o",
    "messages": [{"role": "user", "content": "Hello!"}],
    "fallbacks": [
      "anthropic/claude-sonnet-4-20250514",
      "gemini/gemini-2.5-flash"
    ]
  }'

如果 OpenAI 失敗 → 自動嘗試 Anthropic → 再失敗嘗試 Gemini。使用者端完全無感。

串流回應

# 加上 stream: true 即可
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Authorization: Bearer sk-bf-xxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-4o-mini",
    "messages": [{"role": "user", "content": "寫一首詩"}],
    "stream": true
  }'

完整流程回顧

步驟做了什麼誰做
① 安裝npx -y @maximhq/bifrost 或 Docker管理員
② 設定 API Key透過 Web UI / API / config.json 輸入各家 Provider 的 Key管理員
③ 驗證呼叫 /v1/models 和 /v1/chat/completions 確認設定正確管理員
④ 保護 DashboardSecurity 設定開啟帳密保護 + 推論免認證管理員
⑤ 建立 Virtual Key為每位使用者（或團隊）建立 VK 並設定權限/預算管理員
⑥ 發出請求用 VK 呼叫 /v1/chat/completions 驗證端到端管理員 / 使用者
⑦ 使用者整合設定 base_url + api_key（VK），用既有 SDK 連接使用者
⑧ 監控啟用 Logging，在 Web UI 或 API 查看用量和成本管理員
