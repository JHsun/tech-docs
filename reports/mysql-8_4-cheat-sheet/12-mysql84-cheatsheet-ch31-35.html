Connectors & APIs — 連接器

🟢 官方連接器總覽
連接器語言X DevAPI連線字串格式
Connector/JJava (JDBC)✅jdbc:mysql://host:3306/db?sslMode=REQUIRED
Connector/PythonPython✅connect(host='h', port=3306, user='u', password='p')
Connector/Node.jsNode.js✅（僅 X）mysqlx.getSession('mysqlx://u:p@h:33060')
Connector/C++C++✅xdevapi.h / libmysqlclient
Connector/NET.NET (C#)✅Server=h;Port=3306;Uid=u;Pwd=p;SslMode=Required
Connector/ODBC跨平台 ODBC❌DRIVER={MySQL ODBC 9.x};SERVER=h;DATABASE=db

另有 C API (libmysqlclient) 作為底層函式庫。X Protocol 預設埠 33060，Classic Protocol 預設 3306。

🚨8.4 重大變更：mysql_native_password 預設停用。所有連接器須改用 caching_sha2_password，或在 my.cnf 設 mysql_native_password=ON。Connector/J 8.4.0 新增 OpenTelemetry 支援。
🔶 連線池機制

X DevAPI 透過 Client 物件提供統一連線池：max_size=25、max_idle_time=0（無限）、queue_timeout=0（無限）。Connector/J 須搭配外部池（HikariCP 等）；Connector/NET 預設啟用（Pooling=True，MaxPoolSize=100）。

🔶 HeatWave 概觀

雲端原生 MySQL 服務（獨立產品）。核心：HeatWave Cluster（記憶體內分析加速）、Lakehouse（Object Storage 數百 TB）、AutoML、GenAI（Llama 3.x / Mistral）。部署於 OCI / AWS / Azure。

Telemetry — OpenTelemetry 遙測

🟢 概觀

8.4 首個擁有專屬遙測章節的 LTS 版。Enterprise 功能，整合 OpenTelemetry 推送模型（Traces + Metrics → OTel Collector）。

-- 安裝遙測元件
INSTALL COMPONENT 'file://component_telemetry';
🔶 關鍵遙測變數
變數預設說明
telemetry.trace_enabledON啟用追蹤
telemetry.metrics_enabledON啟用指標
telemetry.query_text_enabledON追蹤含 SQL 文字
telemetry.otel_exporter_otlp_traces_endpointlocalhost:4318/v1/tracesTraces 端點
telemetry.otel_exporter_otlp_metrics_endpointlocalhost:4318/v1/metricsMetrics 端點

約 35 個遙測變數、300+ gauges/counters。追蹤產生 Statement Span（SQL 文字+計時）與 Session Span（processlist_id、user、host）。

🟢 InnoDB 核心術語
術語定義
ACID原子性・一致性・隔離性・持久性
B-treeInnoDB 索引結構，支援精確匹配與範圍查詢
Buffer Pool快取資料與索引的記憶體區域（建議 50-75% RAM）
Clustered Index主鍵索引，組織實際表資料
MVCC多版本並行控制，透過 undo log 維護舊版本
Redo Log崩潰恢復用的順序追加日誌（WAL）
Undo Log回滾與 MVCC 用的修改前資料
Gap Lock鎖定索引記錄間的間隙，防止幻影列
Next-Key LockRecord + Gap Lock，REPEATABLE READ 預設鎖
Doublewrite Buffer防止不完整頁面寫入的安全機制
LSN日誌序號，追蹤 redo log 位置

🔶 8.4 新增保留字 & 關鍵字變更

新增保留字：MANUAL、PARALLEL、QUALIFY、TABLESAMPLE。所有 MASTER_* 關鍵字已移除，替換為 SOURCE_*。查詢完整清單：SELECT * FROM INFORMATION_SCHEMA.KEYWORDS WHERE RESERVED = 1;
