01

General Information — MySQL 8.4 是什麼、有何不同

🟢 核心定位

MySQL 是全球最廣泛使用的開源關聯式資料庫，由 Oracle Corporation 維護，採 GPL 雙授權。以 C/C++ 撰寫的多執行緒伺服器，預設儲存引擎為 InnoDB（ACID 交易、列級鎖定、崩潰復原）。官方發音為「My Ess Que Ell」。

MySQL 8.4 是新版釋出模型下的首個 LTS（長期支援）版本，享有 5 年 Premier + 3 年 Extended Support（共 8 年以上）。同一 8.4.x 系列內不會移除功能或變更資料格式。

🔶 8.4 vs 8.0：InnoDB 預設值變更
變數8.4 新預設8.0 舊預設影響
innodb_adaptive_hash_indexOFFON減少高併發爭用
innodb_change_bufferingnoneall現代 SSD 不需變更緩衝
innodb_flush_method (Linux)O_DIRECTfsync跳過 OS Page Cache
innodb_io_capacity10000200適配 SSD 基準 IOPS
innodb_log_buffer_size64 MiB16 MiB減少 log flush 頻率
innodb_numa_interleaveONOFFNUMA 記憶體交錯
innodb_use_fdatasyncONOFF效能更佳的刷新呼叫
temptable_max_ram3% 系統記憶體1 GiB動態適配
temptable_max_mmap0（關閉）1 GiB避免映射溢出
🔶 認證與複寫語法變更

認證：mysql_native_password 預設停用，default_authentication_plugin 已移除，改由 authentication_policy 控制。預設為 caching_sha2_password。舊客戶端需加 mysql_native_password=ON。

已移除語法（8.4 報錯）替代語法
CHANGE MASTER TOCHANGE REPLICATION SOURCE TO
SHOW MASTER STATUSSHOW BINARY LOG STATUS
SHOW SLAVE STATUSSHOW REPLICA STATUS
START / STOP SLAVESTART / STOP REPLICA
RESET MASTERRESET BINARY LOGS AND GTIDS
🔶 新功能亮點
  • Tagged GTID：UUID:TAG:NUMBER 格式，TAG 最長 8 字元
  • 自動直方圖：ANALYZE TABLE ... UPDATE HISTOGRAM ... AUTO UPDATE
  • Clone Plugin 放寬允許跨 point release 複製
  • Group Replication 預設一致性提升至 BEFORE_ON_PRIMARY_FAILOVER
  • 新權限：FLUSH_PRIVILEGES、OPTIMIZE_LOCAL_TABLE、TRANSACTION_GTID_TAG
  • 新保留字：AUTO、BERNOULLI、GTIDS、LOG、MANUAL、PARALLEL、QUALIFY、TABLESAMPLE 等
⚠️ 若現有 schema 使用上述保留字作為識別符號，升級前必須加反引號（`）包裹。

Installing MySQL — 跨平台安裝指令速查

🟢 Linux — YUM / DNF（RHEL/CentOS/Fedora）
# 1. 安裝 MySQL YUM Repo RPM
sudo yum localinstall mysql84-community-release-el9-1.noarch.rpm

# 2. EL8 須停用系統內建 MySQL 模組
sudo yum module disable mysql

# 3. 安裝 MySQL Server
sudo dnf install mysql-community-server

# 4. 啟動並取得暫時 root 密碼
sudo systemctl start mysqld
sudo grep 'temporary password' /var/log/mysqld.log
🟢 Linux — APT（Debian/Ubuntu）
sudo dpkg -i mysql-apt-config_VERSION_all.deb   # 選 mysql-8.4-lts
sudo apt-get update
sudo apt-get install mysql-server
sudo systemctl start mysql
🟢 Windows

MySQL 8.4 僅支援 64 位元 Windows。MySQL Installer 已於 8.0 後停止，改用 MSI + MySQL Configurator。

:: MSI 靜默安裝
msiexec /i mysql-community-8.4.8.msi /qn /lv install.log

:: ZIP 手動安裝（解壓後）
bin\mysqld --defaults-file=C:\my.ini --initialize --console
bin\mysqld --install "MySQL84"
net start MySQL84
🟢 macOS
# DMG 安裝後，安裝程式會顯示臨時 root 密碼（務必記錄！）

# 啟動 / 停止（Launch Daemon）
sudo launchctl load -F /Library/LaunchDaemons/com.oracle.oss.mysql.mysqld.plist
sudo launchctl unload -F /Library/LaunchDaemons/com.oracle.oss.mysql.mysqld.plist

# Homebrew 替代
brew install mysql && brew services start mysql
🟢 Docker 部署
# 拉取映像
docker pull mysql:8.4

# 啟動容器（含密碼與持久化）
docker run --name=mysql1 \
  -e MYSQL_ROOT_PASSWORD=my-secret-pw \
  -v /my/datadir:/var/lib/mysql \
  -p 3306:3306 -d mysql:8.4

# 進入容器連線
docker exec -it mysql1 mysql -uroot -p
Docker 環境變數說明
MYSQL_ROOT_PASSWORD設定 root 密碼（必要）
MYSQL_RANDOM_ROOT_PASSWORD產生隨機密碼，印至 stdout
MYSQL_DATABASE首次啟動自動建立此資料庫
MYSQL_USER / MYSQL_PASSWORD建立使用者並授權
⚠️ Docker 布林變數陷阱：設為 "false" 或 "0" 仍視為啟用（任何非空字串 = true）。
🔶 原始碼編譯
cmake .. \
  -DCMAKE_INSTALL_PREFIX=/usr/local/mysql \
  -DMYSQL_DATADIR=/usr/local/mysql/data \
  -DWITH_SSL=system \
  -DDEFAULT_CHARSET=utf8mb4 \
  -DDEFAULT_COLLATION=utf8mb4_0900_ai_ci
make -j$(nproc) && make install

需求：CMake、GCC 10+ 或 Clang 12+、OpenSSL、ncurses、GNU make 3.75+。

🟢 安裝後初始化與安全設定
# 建立專用系統帳號
groupadd mysql && useradd -r -g mysql -s /bin/false mysql

# 安全初始化（推薦）— 產生隨機暫時密碼
bin/mysqld --initialize --user=mysql

# 首次登入與密碼設定
mysql -u root -p            # 輸入 error log 中的暫時密碼
ALTER USER 'root'@'localhost' IDENTIFIED BY '新的強密碼';

# 安全設定精靈（推薦所有平台）
mysql_secure_installation
🔶 關鍵組態參數
參數預設值說明
datadir/var/lib/mysql資料目錄
port3306TCP 連接埠
socket/var/lib/mysql/mysql.sockUnix Socket
character_set_serverutf8mb4預設字元集
collation_serverutf8mb4_0900_ai_ci預設排序規則
innodb_data_file_pathibdata1:12M:autoextend系統表空間
secure_file_priv平台相依限制 LOAD DATA / SELECT INTO
⚠️ 常見陷阱：EL8 須先 yum module disable mysql；macOS 不自動建立 my.cnf；資料目錄必須為空才能 --initialize。

Upgrading MySQL — 路徑、步驟與除錯

🟢 支援的升級路徑
來源版本目標支援？方法
8.4.x → 8.4.yLTS 系列內✅就地、邏輯匯出、複寫、Clone
8.0.x → 8.4LTS✅就地、邏輯匯出、複寫
8.1/8.2/8.3 → 8.4Innovation → LTS✅就地、邏輯匯出、複寫
5.7 → 8.4跨大版本❌ 直接須先 5.7→8.0 再 8.0→8.4
8.3 → 9.xInnovation❌ 直接須先 8.3→8.4 再 8.4→9.x
🟢 就地升級步驟
# 1. 前置檢查
mysqlcheck -u root -p --all-databases --check-upgrade

# 2. 確認無未提交 XA 交易
XA RECOVER;

# 3. 正常關閉
SET GLOBAL innodb_fast_shutdown = 1;
mysqladmin -u root -p shutdown

# 4. 替換二進位或升級套件
sudo dnf upgrade mysql-community-server

# 5. 以既有資料目錄啟動新版（自動升級 DD / P_S / sys）
sudo systemctl start mysqld
🔶 邏輯升級步驟
# 1. 匯出所有資料
mysqldump -u root -p --add-drop-table --routines --events \
  --all-databases --force > data-for-upgrade.sql

# 2. 關閉舊版、安裝 8.4、初始化新資料目錄
mysqld --initialize --datadir=/path/to/8.4-datadir

# 3. 啟動新版、匯入
mysql -u root -p --force < data-for-upgrade.sql

# 4. 強制升級系統表
mysqld_safe --upgrade=FORCE &
🔶 升級前必做檢查
  • 最關鍵：執行 MySQL Shell Upgrade Checker
    mysqlsh> util.checkForServerUpgrade('user@host', {"targetVersion":"8.4.0"})
  • 確認無非 InnoDB 分區表
  • 確認外鍵約束名稱 ≤ 64 字元
  • 移除 my.cnf 中已刪除選項（default_authentication_plugin、--ssl 等）
  • 確認保留字衝突（MANUAL、PARALLEL、QUALIFY、TABLESAMPLE）
  • 確保乾淨關閉（非崩潰狀態）
🔶 --upgrade 選項
值行為
AUTO（預設）自動升級所有過期項目
NONE不升級；若 DD 需升級則退出報錯
MINIMAL僅 DD / P_S / I_S（GR 無法啟動）
FORCE強制完整升級，含重建缺失系統表
💡 時區表不會自動升級：mysql_tzinfo_to_sql /usr/share/zoneinfo | mysql -u root -p mysql
🔶 升級失敗排除

升級失敗時伺服器會回滾 DD 變更。回復步驟：移除 redo log → 以舊版重啟 → 修復 → innodb_fast_shutdown=0 關閉 → 重試。常見錯誤：殘留已移除選項（用 mysqld --print-defaults 檢查）、客戶端連結舊函式庫、可載入函式與內建函式同名衝突。

Downgrading MySQL — 規則與限制

🟢 降級路徑速查
🚨 最安全的「降級」永遠是還原升級前的備份。跨版本就地降級不被支援。
降級路徑支援方法限制
8.4.y → 8.4.x（系列內）就地、邏輯匯出、Clone、複寫完整支援
8.4.x → 8.0.y（跨系列）僅邏輯匯出或複寫僅限 rollback 用途
降至 5.7 或更早❌ 完全不支援—
🔶 降級注意事項
  • YUM/APT 套件管理器不支援就地降級
  • 複寫拓撲降級：先降 Source、再降 Replica（與升級相反）
  • 若曾用 8.4 專屬功能（Tagged GTID、自動直方圖等），跨系列降級不可行
  • 確認 mysql_native_password 狀態與目標版本相容
💡 實戰建議：升級前務必執行完整備份（mysqldump、MySQL Shell dump utilities、或 Enterprise Backup），這是最可靠的降級保險。

Tutorial — 連線、查詢與資料操作

🟢 連線與斷線
# 連線
mysql -h hostname -u username -p
mysql -u root -p menagerie    # 直接指定資料庫

# 斷線
mysql> QUIT                    # 或 Unix 按 Ctrl+D
🟢 查詢輸入規則
提示符號含義
mysql>準備接受新查詢
->等待多行查詢的下一行
'>等待單引號字串結束
">等待雙引號字串結束
`>等待反引號識別符結束
/*>等待 /* 註解結束

每條 SQL 以分號結尾。關鍵字不分大小寫。取消輸入中查詢：\c。

🟢 建立資料庫與資料表
CREATE DATABASE menagerie;
USE menagerie;
SELECT DATABASE();            -- 確認目前資料庫

CREATE TABLE pet (
    name    VARCHAR(20),
    owner   VARCHAR(20),
    species VARCHAR(20),
    sex     CHAR(1),
    birth   DATE,
    death   DATE
);

SHOW TABLES;
DESCRIBE pet;
🟢 載入資料
-- 批次載入（Tab 分隔，NULL 以 \N 表示）
LOAD DATA LOCAL INFILE '/path/pet.txt' INTO TABLE pet;

-- Windows 換行
LOAD DATA LOCAL INFILE '/path/pet.txt' INTO TABLE pet
  LINES TERMINATED BY '\r\n';

-- 單筆插入
INSERT INTO pet VALUES
  ('Puffball','Diane','hamster','f','1999-03-30',NULL);
🟢 查詢速查
-- 全部資料
SELECT * FROM pet;

-- 條件篩選
SELECT * FROM pet WHERE species = 'dog' AND sex = 'f';
SELECT * FROM pet WHERE species = 'snake' OR species = 'bird';

-- 去重、排序
SELECT DISTINCT owner FROM pet;
SELECT name, birth FROM pet ORDER BY birth DESC;

-- 日期計算（年齡）
SELECT name, birth, CURDATE(),
       TIMESTAMPDIFF(YEAR, birth, CURDATE()) AS age
FROM pet;

-- NULL 判斷（必須用 IS NULL / IS NOT NULL）
SELECT name FROM pet WHERE death IS NOT NULL;
🟢 模式比對
-- LIKE（_ 匹配單字元，% 匹配任意數量字元）
SELECT * FROM pet WHERE name LIKE 'b%';
SELECT * FROM pet WHERE name LIKE '_____';  -- 恰好 5 字元

-- REGEXP_LIKE（正規表達式）
SELECT * FROM pet WHERE REGEXP_LIKE(name, '^b');
SELECT * FROM pet WHERE REGEXP_LIKE(name, 'fy$');
SELECT * FROM pet WHERE REGEXP_LIKE(name, '^b', 'c'); -- 區分大小寫
🟢 聚合函式與分組
SELECT COUNT(*) FROM pet;
SELECT species, COUNT(*) FROM pet GROUP BY species;
SELECT species, sex, COUNT(*) FROM pet
  WHERE sex IS NOT NULL
  GROUP BY species, sex;
⚠️ ONLY_FULL_GROUP_BY 模式啟用時，SELECT 中非聚合欄位必須出現在 GROUP BY 中。
🟢 多表查詢（JOIN）
-- INNER JOIN
SELECT pet.name, TIMESTAMPDIFF(YEAR, birth, date) AS age, remark
FROM pet INNER JOIN event ON pet.name = event.name
WHERE event.type = 'litter';

-- Self-Join（同表自連接）
SELECT p1.name, p1.sex, p2.name, p2.sex, p1.species
FROM pet AS p1 INNER JOIN pet AS p2
  ON p1.species = p2.species
  AND p1.sex = 'f' AND p2.sex = 'm';
🟢 資訊查詢與批次模式
SELECT DATABASE();         -- 目前資料庫
SHOW DATABASES;            -- 所有資料庫
SHOW TABLES;               -- 目前 DB 的表
DESCRIBE pet;              -- 表結構
SHOW CREATE TABLE pet;    -- 完整建表語句
# 批次執行 SQL 腳本
mysql -u root -p < batch-file.sql
mysql -t < batch-file.sql          # 表格格式輸出

# 在互動模式中
mysql> source /path/to/script.sql;
🔶 進階查詢模式
-- 最大值的完整列
SELECT * FROM shop
WHERE price = (SELECT MAX(price) FROM shop);

-- 每組最大值（相關子查詢）
SELECT article, dealer, price FROM shop s1
WHERE price = (SELECT MAX(s2.price) FROM shop s2
               WHERE s1.article = s2.article);

-- AUTO_INCREMENT
CREATE TABLE animals (
    id MEDIUMINT NOT NULL AUTO_INCREMENT,
    name CHAR(30) NOT NULL,
    PRIMARY KEY (id)
);
SELECT LAST_INSERT_ID();   -- 取得最後插入的自增值
