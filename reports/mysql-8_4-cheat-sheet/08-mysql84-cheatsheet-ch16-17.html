Data Dictionary — 資料字典

🟢 什麼是資料字典

MySQL 8.4 將所有中繼資料（metadata）統一儲存在 InnoDB 交易式表中（mysql.ibd），取代過去的 .frm、.par、.TRN、.TRG、db.opt 檔案。DDL 操作享有 commit/rollback/crash-recovery 保護。

字典表本身隱藏且不可直接存取。透過 INFORMATION_SCHEMA 視圖或 SHOW 語句查詢。

🟢 查看中繼資料
SELECT * FROM INFORMATION_SCHEMA.TABLES WHERE TABLE_SCHEMA = 'mydb';
SELECT * FROM INFORMATION_SCHEMA.COLUMNS WHERE TABLE_NAME = 'orders';
SHOW CREATE TABLE orders\G
SHOW TABLES;
💡INFORMATION_SCHEMA 現在直接讀取字典表的索引，不再建立臨時表或掃描目錄，查詢效能大幅提升。統計資訊快取 information_schema_stats_expiry 預設 86400 秒（24h），設 0 取最新值。
🔶 SDI（序列化字典資訊）

中繼資料以壓縮 JSON 同時寫入 .ibd 檔案，提供冗餘保護。

# 從表空間檔擷取 SDI
ibd2sdi --pretty /var/lib/mysql/mydb/t1.ibd
ibd2sdi --skip-data /var/lib/mysql/ibdata1
🟢 原子式 DDL（Atomic DDL）

字典更新 + 儲存引擎操作 + binlog 寫入合併為單一原子操作。

  • 支援：CREATE/ALTER/DROP TABLE、CREATE/DROP INDEX、CREATE/DROP TABLESPACE、帳戶管理（CREATE/ALTER/DROP USER、GRANT、REVOKE）
  • 不支援：非 InnoDB 引擎的表 DDL、INSTALL/UNINSTALL PLUGIN
⚠️原子 DDL ≠ 交易式 DDL。DDL 仍會隱式提交當前交易，不能包在 START TRANSACTION 區塊內。

InnoDB Storage Engine — InnoDB 儲存引擎

🟢 InnoDB 核心特性
  • ACID 交易：commit / rollback / crash-recovery
  • 列級鎖定 + MVCC：高併發
  • 聚簇索引：按主鍵組織磁碟資料
  • 外鍵：參照完整性
  • 表空間上限 64TB
  • 支援全文索引、空間索引、TDE 加密、壓縮
🟢 InnoDB 架構總覽
┌──────────────── 記憶體結構 ────────────────┐
│ Buffer Pool        │ Change Buffer        │
│ (資料/索引快取)      │ (二級索引延遲寫入)     │
│                    │                      │
│ Adaptive Hash Index │ Log Buffer           │
│ (自動雜湊索引)       │ (Redo Log 緩衝)       │
└──────────────────────────────────────────┘
                      │ 寫入 ↓
┌──────────────── 磁碟結構 ────────────────┐
│ System Tablespace (ibdata1)              │
│ File-Per-Table (.ibd)                    │
│ Redo Log (#ib_redo*)                     │
│ Undo Tablespaces (.ibu)                  │
│ Doublewrite Buffer (.dblwr)              │
└──────────────────────────────────────────┘
🟢 MVCC（多版本並行控制）

每列維護三個隱藏欄位：DB_TRX_ID（6B，交易 ID）、DB_ROLL_PTR（7B，指向 undo log）、DB_ROW_ID（6B，無主鍵時使用）。

Undo log 保存舊版本資料。REPEATABLE READ 下，交易第一次 SELECT 建立快照，之後所有 SELECT 看同一快照。READ COMMITTED 下每次 SELECT 取新快照。

🟢 Buffer Pool（緩衝池）
參數8.4 預設建議
innodb_buffer_pool_size128MB專用伺服器設 50-75% RAM
innodb_buffer_pool_instances動態計算每實例 ≥ 1GB
innodb_buffer_pool_chunk_size128MB僅啟動時可改
-- 線上調整（無需重啟）
SET GLOBAL innodb_buffer_pool_size = 8589934592;  -- 8GB

-- 命中率監控（目標 > 99%）
SHOW GLOBAL STATUS LIKE 'Innodb_buffer_pool_read_requests';
SHOW GLOBAL STATUS LIKE 'Innodb_buffer_pool_reads';

使用改良 LRU 中點插入策略防止全表掃描汙染快取。預熱預設啟用（innodb_buffer_pool_dump_at_shutdown=ON、innodb_buffer_pool_load_at_startup=ON）。

🔶 Change Buffer & Adaptive Hash Index（8.4 重大變更）
參數8.0 預設8.4 預設影響
innodb_change_bufferingallnoneChange Buffer 停用（SSD 不需要）
innodb_adaptive_hash_indexONOFFAHI 停用（減少高併發爭用）
🚨這是 8.4 最重要的預設值變更。寫入密集的二級索引工作負載可能需手動啟用 Change Buffer；高併發查詢場景建議先基準測試再決定是否啟用 AHI。
🟢 Redo Log（重做日誌）
參數8.4 預設說明
innodb_redo_log_capacity100MB取代舊 innodb_log_file_size × innodb_log_files_in_group
innodb_log_buffer_size64MB（8.0 為 16MB）寫入磁碟前的緩衝
innodb_flush_log_at_trx_commit11=每次提交刷盤；2=每秒；0=最快但可能丟
-- 動態調整
SET GLOBAL innodb_redo_log_capacity = 8589934592;  -- 8GB

-- 監控
SELECT * FROM performance_schema.innodb_redo_log_files;

-- ⚠️ 僅限初始載入：停用 redo log
ALTER INSTANCE DISABLE INNODB REDO_LOG;
-- 載入完成後務必重新啟用
ALTER INSTANCE ENABLE INNODB REDO_LOG;

檔案位於 #innodb_redo/，命名 #ib_redoN，共 32 個檔案。

🔶 Undo Log & Doublewrite Buffer
參數8.4 預設說明
innodb_rollback_segments128每 undo 表空間的回滾段數
innodb_undo_log_truncateON自動截斷 undo 表空間
innodb_max_undo_log_size1GB超過觸發截斷
innodb_doublewriteON防止不完整頁面寫入
innodb_doublewrite_pages128（8.0 為 4）更大批次寫入
🟢 索引結構：聚簇 vs 二級

聚簇索引選取順序：① PRIMARY KEY → ② 第一個全 NOT NULL 的 UNIQUE → ③ InnoDB 自動生成隱藏 6-byte 列 ID。

二級索引每條記錄包含主鍵值，查詢時先找二級索引再回聚簇索引。因此主鍵越短，二級索引越小。

覆蓋索引：索引含所有查詢欄位時，無需回表。EXPLAIN Extra 顯示 Using index。

🔶 死鎖處理
參數預設說明
innodb_deadlock_detectON自動偵測，回滾最小交易
innodb_lock_wait_timeout50 秒鎖等待逾時
innodb_print_all_deadlocksOFF設 ON 寫入 error log

最佳實踐：相同順序存取表和列、交易小且短、WHERE 欄位建索引、用 SELECT ... FOR UPDATE 提前取得鎖。

🔶 Online DDL
ALGORITHM說明DML 並發
INSTANT（8.4 預設）僅修改中繼資料完全不阻塞
INPLACE就地執行，可能重建索引可 LOCK=NONE
COPY複製整張表通常需共享/排他鎖
ALTER TABLE t ADD COLUMN email VARCHAR(255),
    ALGORITHM=INSTANT, LOCK=NONE;

INSTANT 支援：新增/刪除欄位、設定預設值、新增虛擬欄位。每表最多 64 個列版本。COMPRESSED 格式不支援 INSTANT。

🔶 TDE（透明資料加密）
-- 建立加密表（需先安裝 Keyring）
CREATE TABLE secure (id INT PRIMARY KEY, ssn VARCHAR(20)) ENCRYPTION='Y';

-- 輪換主金鑰
ALTER INSTANCE ROTATE INNODB MASTER KEY;

-- 加密 redo/undo log
SET PERSIST innodb_redo_log_encrypt = ON;
SET PERSIST innodb_undo_log_encrypt = ON;
⚠️8.4 已移除 keyring_file/keyring_encrypted_file 外掛，改用 keyring 元件（component_keyring_file 等）。
🔶 崩潰恢復與 innodb_force_recovery

正常崩潰恢復只需重啟。流程：Redo log 重播 → 回滾未提交交易 → Change buffer 合併 → Purge。

等級行為可寫？
0正常啟動✅
1忽略損壞頁面✅
2停止背景執行緒✅
3不執行回滾✅
4不合併 change buffer❌ 唯讀
5不掃描 undo log❌ 唯讀
6不重播 redo log❌ 唯讀

從等級 1 開始逐步提升。等級 ≥ 4 可能永久損壞資料。成功匯出後立即重建實例。
