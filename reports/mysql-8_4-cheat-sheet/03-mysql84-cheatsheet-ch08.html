8.1

一般安全性議題

🟢 安全準則總覽
準則說明
不以 OS root 執行 mysqld8.4 預設拒絕以 root 啟動；建立專用 mysql 帳號
不授予非管理員 FILEFILE 可讀寫伺服器檔案，搭配 secure_file_priv
不授予 PROCESS / SUPERSHOW PROCESSLIST 洩漏 SQL 明文
不直接操作 mysql.user使用 GRANT / CREATE USER，避免繞過日誌改寫
MySQL 放在防火牆後方Port 3306 不對不信任主機開放
網路傳輸一律加密使用 SSL/TLS 或 SSH Tunnel
# my.cnf 基本安全設定
[mysqld]
user=mysql
secure_file_priv=/var/lib/mysql-files
local_infile=OFF          # 8.4 預設已 OFF
skip-symbolic-links=ON
🟢 密碼安全 — 安全性排名
  1. mysql_config_editor — 加密儲存於 .mylogin.cnf（最安全）
  2. 互動式提示 — mysql -u user -p（不帶值）
  3. 選項檔案 ~/.my.cnf（權限設 chmod 400）
  4. 命令列 --password=xxx（可被 ps 看到，不安全）
  5. 環境變數 MYSQL_PWD（絕對禁止）
⚠️CREATE USER / ALTER USER 中的明文密碼在日誌中會自動遮蔽。但 --log-raw 會關閉此保護，僅限偵錯使用。
🔶 關鍵安全系統變數
變數預設動態說明
local_infileOFFYes控制 LOAD DATA LOCAL
secure_file_priv平台相依No限制 FILE 操作目錄
skip_name_resolveOFFNo啟用後 Grant 表只能用 IP
skip_networkingOFFNo禁用 TCP/IP（僅 socket）
require_secure_transportOFFYes強制加密連線
8.2

存取控制與帳號管理

🟢 帳號名稱格式

MySQL 帳號採 'user'@'host' 格式。User ≤ 32 字元、Host ≤ 255 字元。省略 host = '%'（任何主機）。

CREATE USER 'app'@'localhost';                -- 僅本機
CREATE USER 'app'@'%';                       -- 任何主機
CREATE USER 'app'@'192.168.1.%';             -- 子網萬用
CREATE USER 'app'@'192.168.1.0/255.255.255.0'; -- 子網遮罩
CREATE USER 'app'@'%.example.com';           -- 網域萬用
💡使用 SELECT CURRENT_USER(); 確認實際匹配的帳號。字面 IP 優先於萬用字元，% 最後匹配。
🟢 CREATE USER 完整語法
CREATE USER [IF NOT EXISTS] 'jeffrey'@'localhost'
  IDENTIFIED BY 'password'
  DEFAULT ROLE role1, role2
  REQUIRE SSL
  WITH MAX_QUERIES_PER_HOUR 100
       MAX_USER_CONNECTIONS 5
  PASSWORD EXPIRE INTERVAL 180 DAY
  PASSWORD HISTORY 5
  PASSWORD REUSE INTERVAL 365 DAY
  FAILED_LOGIN_ATTEMPTS 5
  PASSWORD_LOCK_TIME 2
  ACCOUNT UNLOCK;
🟢 ALTER USER / DROP USER / RENAME USER
-- 修改密碼（需驗證舊密碼）
ALTER USER 'jeffrey'@'localhost'
  IDENTIFIED BY 'new_pw' REPLACE 'old_pw';

-- 修改自己
ALTER USER USER() IDENTIFIED BY 'new_pw' REPLACE 'old_pw';

-- 刪除帳號
DROP USER [IF EXISTS] 'jeffrey'@'localhost';

-- 重新命名
RENAME USER 'old'@'localhost' TO 'new'@'localhost';
🟢 GRANT / REVOKE 語法
-- 授予權限
GRANT SELECT, INSERT, UPDATE ON mydb.* TO 'app'@'%';
GRANT ALL ON mydb.orders TO 'admin'@'localhost';
GRANT SELECT (name, email) ON mydb.users TO 'reader'@'%';
GRANT EXECUTE ON PROCEDURE mydb.sp_report TO 'analyst'@'%';

-- 含 GRANT OPTION（可再授權給他人）
GRANT ALL ON *.* TO 'dba'@'localhost' WITH GRANT OPTION;

-- 撤銷
REVOKE INSERT ON mydb.* FROM 'app'@'%';
REVOKE ALL PRIVILEGES, GRANT OPTION FROM 'app'@'%';

-- 查看權限
SHOW GRANTS FOR 'app'@'%';
SHOW GRANTS FOR CURRENT_USER();
層級ON 語法範例
全域*.*GRANT SELECT ON *.* TO ...
資料庫db.*GRANT ALL ON mydb.* TO ...
資料表db.tblGRANT INSERT ON mydb.orders TO ...
欄位(col1,col2)GRANT SELECT (name) ON mydb.users TO ...
RoutinePROCEDURE/FUNCTIONGRANT EXECUTE ON PROCEDURE ...

🟢 角色管理（Roles）
-- 建立角色
CREATE ROLE 'app_read', 'app_write', 'app_admin';

-- 授予權限給角色
GRANT SELECT ON mydb.* TO 'app_read';
GRANT INSERT, UPDATE, DELETE ON mydb.* TO 'app_write';

-- 將角色授予使用者
GRANT 'app_read', 'app_write' TO 'developer'@'localhost';

-- 設定預設角色（登入自動啟用）
SET DEFAULT ROLE ALL TO 'developer'@'localhost';

-- 會話動態切換
SET ROLE 'app_read';         -- 僅啟用 app_read
SET ROLE ALL;                -- 啟用所有
SET ROLE NONE;               -- 全部停用
SET ROLE DEFAULT;            -- 恢復預設

-- 查看
SELECT CURRENT_ROLE();
-- 全域強制角色
SET PERSIST mandatory_roles = 'app_read';

-- 登入自動啟用所有角色
SET PERSIST activate_all_roles_on_login = ON;
🔶 部分撤銷（Partial Revokes）
SET PERSIST partial_revokes = ON;

-- 全域授予後排除 mysql schema
GRANT SELECT, INSERT, UPDATE ON *.* TO 'analyst'@'%';
REVOKE INSERT, UPDATE ON mysql.* FROM 'analyst'@'%';

SHOW GRANTS FOR 'analyst'@'%';
-- GRANT SELECT, INSERT, UPDATE ON *.* ...
-- REVOKE INSERT, UPDATE ON `mysql`.* ...
🟢 密碼管理
-- 全域過期策略
SET PERSIST default_password_lifetime = 180;  -- 天（0=不過期）

-- 個別帳號
ALTER USER 'u1'@'localhost' PASSWORD EXPIRE INTERVAL 90 DAY;
ALTER USER 'u1'@'localhost' PASSWORD EXPIRE;  -- 立即過期

-- 密碼歷史
SET PERSIST password_history = 6;
SET PERSIST password_reuse_interval = 365;

-- 需驗證舊密碼
SET PERSIST password_require_current = ON;

-- 失敗鎖定
ALTER USER 'u1'@'localhost'
  FAILED_LOGIN_ATTEMPTS 5
  PASSWORD_LOCK_TIME 2;   -- 鎖 2 天
🔶 雙密碼（零停機輪替）
-- Step 1：設新密碼，保留舊密碼
ALTER USER 'appuser'@'%'
  IDENTIFIED BY 'new_pwd' RETAIN CURRENT PASSWORD;

-- Step 2：逐步更新所有應用程式

-- Step 3：丟棄舊密碼
ALTER USER 'appuser'@'%' DISCARD OLD PASSWORD;

需要 APPLICATION_PASSWORD_ADMIN 權限。

🟢 帳號鎖定 & 資源限制
-- 鎖定/解鎖
ALTER USER 'temp'@'localhost' ACCOUNT LOCK;
ALTER USER 'temp'@'localhost' ACCOUNT UNLOCK;

-- 資源限制
CREATE USER 'api'@'%' IDENTIFIED BY 'pw'
  WITH MAX_QUERIES_PER_HOUR 1000
       MAX_UPDATES_PER_HOUR 500
       MAX_CONNECTIONS_PER_HOUR 100
       MAX_USER_CONNECTIONS 10;

-- 移除限制（設 0）
ALTER USER 'api'@'%' WITH MAX_QUERIES_PER_HOUR 0;

-- 重設計數器
FLUSH USER_RESOURCES;
🔶 認證外掛（8.4 重大變更）
外掛8.4 狀態說明
caching_sha2_password✅ 預設SHA-256 + 快取
mysql_native_password⛔ 棄用且預設停用需 mysql_native_password=ON
sha256_password⚠️ 棄用建議遷移至 caching_sha2
authentication_webauthn✅ 可用WebAuthn/FIDO2（MFA 用）
auth_socket✅ 可用Unix socket peer 認證
-- 遷移至 caching_sha2_password
ALTER USER 'legacy'@'%'
  IDENTIFIED WITH caching_sha2_password BY 'new_password';
🚨default_authentication_plugin 已移除，改用 authentication_policy。舊客戶端無法連線時需在 my.cnf 加 mysql_native_password=ON。
🔶 多因子驗證（MFA）
-- 2FA 帳號
CREATE USER 'secure'@'localhost'
  IDENTIFIED WITH caching_sha2_password BY 'pass1'
  AND IDENTIFIED WITH authentication_webauthn;

-- 新增/移除因子
ALTER USER 'u1'@'localhost' ADD 2 FACTOR IDENTIFIED WITH authentication_webauthn;
ALTER USER 'u1'@'localhost' DROP 2 FACTOR;

-- Client 連線
mysql --user=u1 --password1='pass1' --password2='pass2'
8.3

加密連線（SSL/TLS）

🟢 TLS 基礎

MySQL 8.4 僅支援 TLSv1.2 與 TLSv1.3。auto_generate_certs=ON（預設）自動產生自簽憑證。

# Server 端設定
[mysqld]
ssl_ca=ca.pem
ssl_cert=server-cert.pem
ssl_key=server-key.pem
require_secure_transport=ON  # 強制加密
🟢 Client --ssl-mode 選項
模式行為
DISABLED不加密
PREFERRED預設；嘗試加密，失敗退回明文
REQUIRED強制加密，不驗證憑證
VERIFY_CA加密 + 驗證 CA 憑證
VERIFY_IDENTITY加密 + 驗證 CA + 主機名稱（最安全）
# 生產環境建議
mysql --ssl-mode=VERIFY_CA --ssl-ca=ca.pem -u user -p

# 最高安全
mysql --ssl-mode=VERIFY_IDENTITY --ssl-ca=ca.pem \
      --ssl-cert=client-cert.pem --ssl-key=client-key.pem -u user -p
🟢 帳號層級 TLS 需求 & 驗證
CREATE USER 'u1'@'%' IDENTIFIED BY 'pw' REQUIRE SSL;
CREATE USER 'u2'@'%' IDENTIFIED BY 'pw' REQUIRE X509;
CREATE USER 'u3'@'%' IDENTIFIED BY 'pw'
  REQUIRE SUBJECT '/C=TW/O=MyOrg/CN=client'
  AND ISSUER '/C=TW/O=MyOrg/CN=CA';

-- 驗證 SSL 狀態
SHOW SESSION STATUS LIKE 'Ssl_cipher';   -- 非空=已加密
SHOW SESSION STATUS LIKE 'Ssl_version';  -- TLSv1.3
🔶 不停機 TLS 憑證輪替
SET GLOBAL ssl_cert = '/new/server-cert.pem';
SET GLOBAL ssl_key  = '/new/server-key.pem';
ALTER INSTANCE RELOAD TLS;  -- 原子性重載，需 CONNECTION_ADMIN
-- 現有連線不受影響，新連線使用新憑證
🔶 OpenSSL 產生生產憑證
# CA
openssl genrsa 2048 > ca-key.pem
openssl req -new -x509 -nodes -days 3600 -key ca-key.pem -out ca.pem

# Server（CN 需匹配主機名以支援 VERIFY_IDENTITY）
openssl req -newkey rsa:2048 -days 3600 -nodes -keyout server-key.pem -out server-req.pem
openssl x509 -req -in server-req.pem -days 3600 \
  -CA ca.pem -CAkey ca-key.pem -set_serial 01 -out server-cert.pem

# Client
openssl req -newkey rsa:2048 -days 3600 -nodes -keyout client-key.pem -out client-req.pem
openssl x509 -req -in client-req.pem -days 3600 \
  -CA ca.pem -CAkey ca-key.pem -set_serial 02 -out client-cert.pem

# 驗證
openssl verify -CAfile ca.pem server-cert.pem client-cert.pem
⚠️Server 與 Client 的 CN 必須不同。mysql_ssl_rsa_setup 已在 8.4 移除。

安全元件與外掛

🟢 密碼驗證元件（validate_password）
INSTALL COMPONENT 'file://component_validate_password';
SHOW VARIABLES LIKE 'validate_password.%';
SELECT VALIDATE_PASSWORD_STRENGTH('MyP@ss1234');  -- 0-100
等級值檢查項目
LOW0僅長度
MEDIUM1（預設）長度 + 數字 + 大小寫 + 特殊字元
STRONG2MEDIUM + 字典比對
變數預設
validate_password.length8
validate_password.mixed_case_count1
validate_password.number_count1
validate_password.special_char_count1
validate_password.check_user_nameON

🔶 Enterprise Audit & Firewall（企業版）

mask_inner(str, m, n)'0912345678', 3, 2091XXXXX78

-- 建立遮罩檢視
CREATE VIEW v_customer_masked AS
SELECT id, first_name,
       mask_inner(email, 2, LOCATE('@', email)) AS masked_email,
       mask_pan(card_number) AS masked_card
FROM customers;

GRANT SELECT ON mydb.v_customer_masked TO 'analyst'@'%';
