Partitioning — 分區

🟢 分區類型總覽
類型分配方式適用場景
RANGE連續範圍（VALUES LESS THAN）日期/時間序列資料
LIST離散值清單（VALUES IN）地區/分類代碼
HASHMOD(expr, N) 均勻分散無明顯範圍的均勻負載
KEY伺服器內建雜湊（支援非整數）按主鍵自動分散
RANGE COLUMNS多欄位 + 非整數型別DATE/VARCHAR 直接分區
LIST COLUMNS多欄位離散值字串欄位分區

僅 InnoDB 與 NDB 支援分區。最多 8192 個分區（含子分區）。

🟢 RANGE & LIST 分區範例
-- RANGE（以年份分區）
CREATE TABLE orders (
    id INT NOT NULL, order_date DATE NOT NULL,
    PRIMARY KEY (id, order_date)
) PARTITION BY RANGE (YEAR(order_date)) (
    PARTITION p2023 VALUES LESS THAN (2024),
    PARTITION p2024 VALUES LESS THAN (2025),
    PARTITION pmax  VALUES LESS THAN MAXVALUE
);

-- LIST COLUMNS（字串分區）
CREATE TABLE customers (
    id INT, city VARCHAR(30),
    PRIMARY KEY (id, city)
) PARTITION BY LIST COLUMNS(city) (
    PARTITION pTW VALUES IN ('Taipei','Kaohsiung'),
    PARTITION pJP VALUES IN ('Tokyo','Osaka')
);
🚨分區鍵必須包含在所有唯一鍵（含 PK）中——這是最常見的錯誤。分區表不支援外鍵。
🟢 分區管理 DDL
-- 新增（RANGE 只能在最高端）
ALTER TABLE t ADD PARTITION (PARTITION p2026 VALUES LESS THAN (2027));

-- 刪除（資料同步丟棄，極快）
ALTER TABLE t DROP PARTITION p2023;

-- 清空分區
ALTER TABLE t TRUNCATE PARTITION p2024;

-- 重組（合併/拆分，不丟資料）
ALTER TABLE t REORGANIZE PARTITION pmax INTO (
    PARTITION p2026 VALUES LESS THAN (2027),
    PARTITION pmax VALUES LESS THAN MAXVALUE
);

-- 交換分區與普通表
ALTER TABLE t EXCHANGE PARTITION p2023 WITH TABLE t_archive;

-- 明確指定分區查詢
SELECT * FROM orders PARTITION (p2024, p2025) WHERE amount > 1000;

Stored Objects — 預存物件

🟢 Stored Procedure（儲存程序）
DELIMITER //
CREATE PROCEDURE get_total(
    IN  p_cust_id INT,
    OUT p_total   DECIMAL(10,2)
)
BEGIN
    SELECT SUM(amount) INTO p_total
    FROM orders WHERE customer_id = p_cust_id;
END //
DELIMITER ;

CALL get_total(42, @total);
SELECT @total;

參數模式：IN（預設，值傳入）、OUT（程序結束回傳）、INOUT（雙向）。

🟢 Stored Function（儲存函數）
CREATE FUNCTION calc_tax(price DECIMAL(10,2))
    RETURNS DECIMAL(10,2)
    DETERMINISTIC
BEGIN
    RETURN price * 0.05;
END;

SELECT calc_tax(1000);  -- 50.00

必須有 RETURNS + RETURN。參數一律 IN。啟用 binlog 時須宣告 DETERMINISTIC。

🟢 Trigger（觸發器）
CREATE TRIGGER trg_before_ins
    BEFORE INSERT ON orders FOR EACH ROW
BEGIN
    IF NEW.amount < 0 THEN SET NEW.amount = 0; END IF;
    SET NEW.created_at = NOW();
END;
事件OLDNEW
INSERT❌✅ 可讀寫（BEFORE）
UPDATE✅ 唯讀✅ BEFORE 可修改
DELETE✅ 唯讀❌
🟢 Event Scheduler（事件排程器）
-- 8.4 預設 event_scheduler = ON
CREATE EVENT evt_cleanup
    ON SCHEDULE EVERY 1 DAY
    ON COMPLETION PRESERVE
    DO DELETE FROM sessions WHERE expires < NOW();

ALTER EVENT evt_cleanup DISABLE;
DROP EVENT IF EXISTS evt_cleanup;
🔶 流程控制・游標・錯誤處理
-- IF / WHILE / LOOP
IF v > 0 THEN ... ELSEIF v = 0 THEN ... ELSE ... END IF;
WHILE i <= 10 DO ... END WHILE;
lbl: LOOP ... LEAVE lbl; ... ITERATE lbl; ... END LOOP;

-- 游標（唯讀、只能向前）
DECLARE done INT DEFAULT FALSE;
DECLARE cur CURSOR FOR SELECT name FROM t;
DECLARE CONTINUE HANDLER FOR NOT FOUND SET done = TRUE;
OPEN cur; FETCH cur INTO v; CLOSE cur;

-- SIGNAL（主動拋錯）
SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '金額不可為負';

INFORMATION_SCHEMA

🟢 常用中繼資料查詢
-- 表大小
SELECT TABLE_NAME, ENGINE, TABLE_ROWS,
  ROUND(DATA_LENGTH/1048576,2) AS data_mb,
  ROUND(INDEX_LENGTH/1048576,2) AS index_mb
FROM INFORMATION_SCHEMA.TABLES
WHERE TABLE_SCHEMA='mydb' ORDER BY DATA_LENGTH DESC;

-- 欄位資訊
SELECT COLUMN_NAME, COLUMN_TYPE, IS_NULLABLE, COLUMN_DEFAULT, EXTRA
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_SCHEMA='mydb' AND TABLE_NAME='orders';

-- 索引資訊
SELECT INDEX_NAME, COLUMN_NAME, NON_UNIQUE, CARDINALITY, IS_VISIBLE
FROM INFORMATION_SCHEMA.STATISTICS
WHERE TABLE_SCHEMA='mydb' AND TABLE_NAME='orders';

-- 外鍵關係
SELECT CONSTRAINT_NAME, TABLE_NAME, REFERENCED_TABLE_NAME,
       UPDATE_RULE, DELETE_RULE
FROM INFORMATION_SCHEMA.REFERENTIAL_CONSTRAINTS
WHERE CONSTRAINT_SCHEMA='mydb';
💡統計快取 information_schema_stats_expiry 預設 86400 秒（24h）。設 0 取即時值，或執行 ANALYZE TABLE 刷新。
🔶 InnoDB 系統表 & 交易監控
-- 長時間交易
SELECT TRX_ID, TRX_STATE, TRX_QUERY,
  TIMESTAMPDIFF(SECOND, TRX_STARTED, NOW()) AS sec
FROM INFORMATION_SCHEMA.INNODB_TRX
ORDER BY TRX_STARTED;

-- InnoDB 表空間
SELECT NAME, SPACE_TYPE, FILE_SIZE, ENCRYPTION
FROM INFORMATION_SCHEMA.INNODB_TABLESPACES;
⚠️INNODB_LOCKS/INNODB_LOCK_WAITS 已移除，改用 performance_schema.data_locks/data_lock_waits。I_S.PROCESSLIST 已棄用，改用 P_S.processlist。

Performance Schema

🟢 概觀與設定

執行時期監控框架，記憶體內表，重啟後清空。8.4 預設 performance_schema=ON。

-- 核心設定表
SELECT * FROM performance_schema.setup_instruments;  -- 探針
SELECT * FROM performance_schema.setup_consumers;    -- 消費者
SELECT * FROM performance_schema.setup_actors;       -- 帳號過濾

-- 啟用探針/消費者
UPDATE performance_schema.setup_instruments
  SET ENABLED='YES', TIMED='YES' WHERE NAME LIKE 'statement/%';
UPDATE performance_schema.setup_consumers
  SET ENABLED='YES' WHERE NAME LIKE '%history_long%';
🟢 慢查詢分析（語句摘要）
-- Top 10 平均最慢查詢
SELECT LEFT(DIGEST_TEXT, 80) AS query,
  COUNT_STAR AS execs,
  ROUND(AVG_TIMER_WAIT/1e9, 2) AS avg_ms,
  ROUND(SUM_TIMER_WAIT/1e12, 2) AS total_sec,
  SUM_ROWS_EXAMINED, SUM_NO_INDEX_USED
FROM performance_schema.events_statements_summary_by_digest
ORDER BY AVG_TIMER_WAIT DESC LIMIT 10;

8.4 新增欄位：QUANTILE_95、QUANTILE_99（延遲百分位數）、QUERY_SAMPLE_TEXT（原始 SQL 樣本）。

🟢 取代 SHOW PROCESSLIST
SELECT ID, USER, HOST, DB, COMMAND, TIME, STATE, INFO
FROM performance_schema.processlist
WHERE COMMAND != 'Sleep' AND TIME > 5;
🔶 鎖定監控（零額外開銷）
-- 目前所有鎖
SELECT ENGINE_TRANSACTION_ID, OBJECT_NAME, INDEX_NAME,
  LOCK_TYPE, LOCK_MODE, LOCK_STATUS, LOCK_DATA
FROM performance_schema.data_locks;

-- 阻塞分析（誰在等誰）
SELECT w.REQUESTING_ENGINE_TRANSACTION_ID AS waiting,
  w.BLOCKING_ENGINE_TRANSACTION_ID AS blocking
FROM performance_schema.data_lock_waits w;
🔶 複製・記憶體・I/O 監控
-- 複製連線狀態
SELECT * FROM performance_schema.replication_connection_status\G
SELECT * FROM performance_schema.replication_applier_status_by_worker\G

-- 記憶體消耗排名
SELECT EVENT_NAME,
  CURRENT_NUMBER_OF_BYTES_USED/1048576 AS mb
FROM performance_schema.memory_summary_global_by_event_name
ORDER BY CURRENT_NUMBER_OF_BYTES_USED DESC LIMIT 10;

-- I/O 熱點檔案
SELECT FILE_NAME,
  SUM_NUMBER_OF_BYTES_READ/1048576 AS read_mb,
  SUM_NUMBER_OF_BYTES_WRITE/1048576 AS write_mb
FROM performance_schema.file_summary_by_instance
ORDER BY SUM_TIMER_WAIT DESC LIMIT 10;

sys Schema — 人類可讀監控

🟢 核心視圖速查
-- 第 95 百分位慢查詢
SELECT query, exec_count, avg_latency, total_latency
FROM sys.statements_with_runtimes_in_95th_percentile LIMIT 10;

-- 全表掃描查詢
SELECT query, exec_count, no_index_used_pct
FROM sys.statements_with_full_table_scans LIMIT 10;

-- 未使用的索引
SELECT object_schema, object_name, index_name
FROM sys.schema_unused_indexes
WHERE object_schema NOT IN ('mysql','sys');

-- InnoDB 鎖定等待（含 KILL 語句）
SELECT waiting_query, blocking_query, locked_table,
  wait_age, sql_kill_blocking_query
FROM sys.innodb_lock_waits;

-- 記憶體總覽
SELECT * FROM sys.memory_global_total;

-- 主機/使用者摘要
SELECT host, statements, statement_latency FROM sys.host_summary;
SELECT user, statements, current_memory FROM sys.user_summary;

每個視圖都有 x$ 前綴版本（原始數值，供程式消費）。

🔶 內建函數取代 sys 函數（8.4）
sys 函數（棄用）內建替代範例
sys.format_bytes()FORMAT_BYTES()FORMAT_BYTES(1073741824) → '1.00 GiB'
sys.format_time()FORMAT_PICO_TIME()FORMAT_PICO_TIME(1e9) → '1.00 ms'
sys.ps_thread_id()PS_THREAD_ID()PS_THREAD_ID(260)
—PS_CURRENT_THREAD_ID()取得目前執行緒 ID
🔶 診斷程序
-- 啟用/停用探針
CALL sys.ps_setup_enable_instrument('wait/lock/metadata');
CALL sys.ps_setup_disable_instrument('mutex');

-- 重設所有摘要表
CALL sys.ps_truncate_all_tables(FALSE);

-- 全面診斷報告
CALL sys.diagnostics(120, 30, 'current');
