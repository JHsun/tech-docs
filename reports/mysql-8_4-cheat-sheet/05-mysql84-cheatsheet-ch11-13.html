Language Structure — 語法結構

🟢 字串字面值
SELECT 'Hello', "World";            -- 單引號或雙引號（預設模式）
SELECT 'Hello' ' ' 'World';        -- 自動串接 → 'Hello World'
SELECT _utf8mb4'你好';              -- 字元集引介符
SELECT _utf8mb4'café' COLLATE utf8mb4_0900_as_cs;
SELECT N'text';                     -- 等同 _utf8'text'
跳脫意義跳脫意義
\\0NUL (0x00)\\n換行 LF
\\'單引號\\r歸位 CR
\\"雙引號\\t定位 Tab
\\\\反斜線\\ZCtrl+Z
\\%LIKE 字面 %\\_LIKE 字面 _

引號內嵌引號：'' 產生 '、"" 產生 "。啟用 ANSI_QUOTES 後雙引號僅作識別符用。

🟢 數值・日期時間・特殊字面值
-- 數值
42        -- 整數（BIGINT）
3.14      -- 精確值（DECIMAL）
1.5E3     -- 近似值（DOUBLE）

-- 日期時間
DATE '2025-04-01'
TIME '14:30:00'
TIMESTAMP '2025-04-01 14:30:00'

-- 十六進位
X'4D7953514C'      -- 二進位字串
0x4D7953514C      -- 數值語境下為 BIGINT UNSIGNED

-- 位元值
b'11000001'        -- 或 0b11000001

-- 布林
TRUE = 1, FALSE = 0

-- NULL（任何與 NULL 的比較 = NULL）
IS NULL / IS NOT NULL / <=>   -- 安全等號
🟢 識別符：反引號・大小寫・長度
SELECT * FROM `select` WHERE `select`.`column` > 100;  -- 保留字須加反引號
CREATE TABLE `a``b` (id INT);   -- 反引號自身：``
lower_case_table_names行為預設平台
0儲存原樣，比較區分大小寫Linux/Unix
1儲存為小寫，不區分大小寫Windows
2儲存原樣，比較轉小寫macOS

識別符最長 64 字元（別名 256）。欄位名、索引名、Routine 名永遠不區分大小寫。

💡最佳實踐：一律使用小寫名稱確保跨平台可移植性。lower_case_table_names 只能在初始化時設定。
🟢 使用者變數 (@var) & 系統變數 (@@)
-- 使用者變數（Session 作用域）
SET @name = 'MySQL';
SET @count = 0, @total = 100;
SELECT @count := @count + 1;   -- SET 以外須用 :=

-- 系統變數
SELECT @@GLOBAL.max_connections;
SELECT @@SESSION.sql_mode;
SET GLOBAL max_connections = 500;
SET PERSIST max_connections = 500;       -- 寫入 mysqld-auto.cnf
SET PERSIST_ONLY back_log = 100;        -- 僅寫檔，不改執行值
⚠️8.4 棄用：SELECT @a:=...（SET 以外的變數指派）未來可能移除。同一語句中變數讀寫順序未定義。
🔶 運算子優先順序（高→低）
INTERVAL → BINARY, COLLATE → ! → -(一元), ~ → ^
→ *, /, DIV, %, MOD → -, + → <<, >> → & → |
→ =(比較), <=>, >=, >, <=, <, <>, !=, IS, LIKE, REGEXP, IN
→ BETWEEN, CASE → NOT → AND, && → XOR → OR, || → =(指派), :=

|| 預設為邏輯 OR；啟用 PIPES_AS_CONCAT 後變為字串串接。

🟢 註解語法
SELECT 1;   # 井號註解
SELECT 1;   -- 雙橫線（後須空格）
SELECT 1    /* C 風格多行註解 */ + 1;
/*!80400 SELECT '僅 8.4.0+ 執行'; */    -- 版本條件執行
/*+ NO_INDEX(t1) */                     -- 優化器提示

Character Sets & Collations — 字元集與排序規則

🟢 核心概念與 8.4 預設

MySQL 8.4 預設字元集 utf8mb4，預設排序規則 utf8mb4_0900_ai_ci（UCA 9.0.0、accent-insensitive、case-insensitive、NO PAD）。

後綴意義後綴意義
_ai重音不敏感_as重音敏感
_ci大小寫不敏感_cs大小寫敏感
_ks假名敏感（日文）_bin碼點二進位比較
🚨NO PAD vs PAD SPACE：utf8mb4_0900_ai_ci（預設）為 NO PAD，'abc' ≠ 'abc '。舊版 utf8mb4_general_ci 為 PAD SPACE，'abc' = 'abc '。遷移時務必注意！
🟢 字元集層級繼承：Server → DB → Table → Column
-- 資料庫
CREATE DATABASE mydb CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 資料表
CREATE TABLE t1 (...) CHARACTER SET utf8mb4 COLLATE utf8mb4_bin;

-- 欄位（覆蓋表設定）
CREATE TABLE t1 (
    name VARCHAR(100) CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_as_cs,
    bio  TEXT   -- 繼承 Table
) CHARACTER SET latin1;

解析順序：① 同時指定字元集與排序規則 → 用二者；② 僅字元集 → 用其預設排序；③ 僅排序規則 → 用其對應字元集；④ 皆未指定 → 繼承上層。

🟢 連線字元集與 SET NAMES
變數用途
character_set_client客戶端送出語句的字元集
character_set_connection伺服器轉換處理用的字元集
collation_connection字面值比較的排序規則
character_set_results回傳結果的字元集
SET NAMES 'utf8mb4';              -- 三變數統一設定
SET CHARACTER SET 'utf8mb4';     -- connection 設為資料庫字元集（不同！）
🟢 utf8mb4 vs utf8mb3（已棄用）
特性utf8mb3（棄用）utf8mb4（推薦）
每字元最大位元組34
Emoji 支援❌✅
預設排序規則utf8mb3_general_ciutf8mb4_0900_ai_ci
8.4 狀態已棄用預設推薦
-- 遷移 utf8mb3 → utf8mb4
ALTER TABLE t1 CONVERT TO CHARACTER SET utf8mb4;

-- 或逐欄位修改
ALTER TABLE t1 MODIFY col1 VARCHAR(50) CHARACTER SET utf8mb4;
⚠️utf8 在 8.4 中是 utf8mb3 的棄用別名。遷移注意：utf8mb4 每字元 4 位元組，索引前綴從 255 字元降至 191 字元（InnoDB COMPACT/REDUNDANT）。
🔶 排序規則強制性（Coercibility）
等級來源範例
0明確 COLLATEx COLLATE utf8mb4_bin
2欄位column1
3系統常數VERSION()
4字面值'abc'
5數值/時間1000
6NULLNULL

較低等級勝出。等級相同時 Unicode 側勝過非 Unicode 側。

🟢 字元集轉換
SELECT CONVERT('café' USING utf8mb4);
SELECT CAST('café' AS CHAR CHARACTER SET latin1);
SELECT CONVERT(binary_col USING utf8mb4);    -- 二進位→文字

-- 整表轉換
ALTER TABLE t1 CONVERT TO CHARACTER SET utf8mb4;

Data Types — 資料型別

🟢 整數型別
型別儲存有號範圍無號範圍
TINYINT1 B-128 ~ 1270 ~ 255
SMALLINT2 B-32,768 ~ 32,7670 ~ 65,535
MEDIUMINT3 B-8.3M ~ 8.3M0 ~ 16.7M
INT4 B-2.1B ~ 2.1B0 ~ 4.2B
BIGINT8 B-2⁶³ ~ 2⁶³-10 ~ 2⁶⁴-1

BOOL/BOOLEAN = TINYINT(1)。SERIAL = BIGINT UNSIGNED NOT NULL AUTO_INCREMENT UNIQUE。

⚠️8.4 棄用：INT(11) 的顯示寬度與 ZEROFILL。替代：LPAD(id, 4, '0')。
🟢 精確與浮點數值
型別儲存精度說明
DECIMAL(M,D)依 M,D 計算精確M≤65, D≤30。金額首選
FLOAT4 B~7 位小數近似值
DOUBLE8 B~15 位小數近似值
BIT(M)≈(M+7)/8 B—M=1~64 位元
⚠️8.4 棄用：FLOAT(M,D) / DOUBLE(M,D) 語法、FLOAT/DOUBLE 的 UNSIGNED。改用 CHECK (col >= 0)。FLOAT/DOUBLE 的 AUTO_INCREMENT 已移除。
🟢 日期與時間型別
型別儲存範圍格式
DATE3 B1000-01-01 ~ 9999-12-31YYYY-MM-DD
TIME(fsp)3 B + fsp-838:59:59 ~ 838:59:59hh:mm:ss
DATETIME(fsp)5 B + fsp1000-01-01 ~ 9999-12-31YYYY-MM-DD hh:mm:ss
TIMESTAMP(fsp)4 B + fsp1970-01-01 ~ 2038-01-19YYYY-MM-DD hh:mm:ss
YEAR1 B1901 ~ 2155YYYY
fsp（小數秒精度）01-23-45-6
額外位元組0123
🔶 TIMESTAMP vs DATETIME
特性TIMESTAMPDATETIME
儲存轉為 UTC原樣儲存
擷取時自動轉為連線時區原樣返回
範圍僅到 2038到 9999
適用記錄絕對時間點儲存名義日期時間
-- 自動初始化 + 自動更新
ts TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP

-- 含 fsp 時須一致
ts TIMESTAMP(6) DEFAULT CURRENT_TIMESTAMP(6) ON UPDATE CURRENT_TIMESTAMP(6)
💡MySQL fsp 預設為 0（SQL 標準預設 6）。需微秒精度必須明確指定 DATETIME(6)。explicit_defaults_for_timestamp 在 8.4 預設啟用。
🟢 字串型別
型別最大長度儲存特性
CHAR(M)255 字元M × w B固定長度，取出移除尾隨空格
VARCHAR(M)65,535 字元L + 1~2 B變動長度，保留尾隨空格
BINARY(M)255 位元組M B固定，以 0x00 填充
VARBINARY(M)65,535 位元組L + 1~2 B變動長度

VARCHAR 受 65,535 位元組列限制（所有欄位共享）。utf8mb4 下有效最大約 16,383 字元。

🟢 BLOB & TEXT 型別
型別最大長度儲存
TINYBLOB / TINYTEXT255 BL + 1 B
BLOB / TEXT64 KBL + 2 B
MEDIUMBLOB / MEDIUMTEXT16 MBL + 3 B
LONGBLOB / LONGTEXT4 GBL + 4 B

BLOB 為二進位（無字元集），TEXT 為非二進位。BLOB/TEXT 不能有 DEFAULT 值（NULL 除外）。

🟢 ENUM & SET
-- ENUM：最多 65,535 個值；儲存 1-2 B
size ENUM('S', 'M', 'L', 'XL') NOT NULL DEFAULT 'M'
-- ⚠️ 排序依索引號非字母順序

-- SET：最多 64 個成員；儲存 1-8 B（點陣圖）
flags SET('read', 'write', 'execute') NOT NULL DEFAULT ''
🟢 JSON 型別
CREATE TABLE events (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    data JSON
);
INSERT INTO events (data) VALUES ('{"type":"click","x":100}');
類別函式
擷取-> (JSON_EXTRACT)、->> (JSON_UNQUOTE)、JSON_VALUE
修改JSON_SET、JSON_INSERT、JSON_REPLACE、JSON_REMOVE
建立JSON_ARRAY、JSON_OBJECT、JSON_MERGE_PATCH
搜尋JSON_CONTAINS、JSON_SEARCH、MEMBER OF()
表格化JSON_TABLE（展開為關聯式列）
🔶 JSON 索引策略
-- 方法 1：產生欄位 + 索引
ALTER TABLE events ADD COLUMN event_type VARCHAR(50)
    GENERATED ALWAYS AS (data->>'$.type') STORED,
    ADD INDEX idx_type (event_type);

-- 方法 2（8.4 推薦）：函式索引 + JSON_VALUE
CREATE INDEX idx_x ON events ((JSON_VALUE(data, '$.x' RETURNING UNSIGNED)));

-- 方法 3：多值索引（JSON 陣列）
CREATE TABLE customers (
    id BIGINT PRIMARY KEY,
    info JSON,
    INDEX zips ((CAST(info->'$.zipcodes' AS UNSIGNED ARRAY)))
);
🟢 AUTO_INCREMENT
CREATE TABLE t (
    id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT PRIMARY KEY
);
INSERT INTO t VALUES (NULL);  -- 自動產生
INSERT INTO t VALUES (0);     -- 同上（除非 NO_AUTO_VALUE_ON_ZERO）
SELECT LAST_INSERT_ID();       -- 最近自動生成值

InnoDB 預設 innodb_autoinc_lock_mode=2（交錯式），計數器持久化跨重啟。可能有序號間隙（為效能設計）。
