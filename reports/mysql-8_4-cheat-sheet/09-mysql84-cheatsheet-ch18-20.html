Replication — 複寫

🟢 8.4 術語全面更替
已移除（8.0 舊）8.4 替代
CHANGE MASTER TOCHANGE REPLICATION SOURCE TO
SHOW MASTER STATUSSHOW BINARY LOG STATUS
SHOW SLAVE STATUSSHOW REPLICA STATUS
START/STOP SLAVESTART/STOP REPLICA
RESET SLAVERESET REPLICA
RESET MASTERRESET BINARY LOGS AND GTIDS

🚨8.4 中舊命令已完全移除（非僅棄用）。系統變數 slave_* → replica_*，master_* → source_*。
🟢 複寫設定逐步指南（GTID 模式）
-- ═══ Source (my.cnf) ═══
[mysqld]
server-id = 1
log-bin = mysql-bin              -- 8.4 預設已啟用
gtid_mode = ON
enforce_gtid_consistency = ON
innodb_flush_log_at_trx_commit = 1
sync_binlog = 1

-- ═══ Replica (my.cnf) ═══
[mysqld]
server-id = 2                   -- 必須唯一
gtid_mode = ON
enforce_gtid_consistency = ON
-- Step 1：建立複寫帳號（Source 上）
CREATE USER 'repl'@'%' IDENTIFIED BY 'Str0ngP@ss';
GRANT REPLICATION SLAVE ON *.* TO 'repl'@'%';

-- Step 2：設定 Replica
CHANGE REPLICATION SOURCE TO
  SOURCE_HOST      = '192.168.1.100',
  SOURCE_USER      = 'repl',
  SOURCE_PASSWORD  = 'Str0ngP@ss',
  SOURCE_AUTO_POSITION = 1,
  GET_SOURCE_PUBLIC_KEY = 1;

-- Step 3：啟動
START REPLICA;
SHOW REPLICA STATUS\G
-- 確認 Replica_IO_Running=Yes, Replica_SQL_Running=Yes
🔶 CHANGE REPLICATION SOURCE TO 關鍵選項
選項預設說明
SOURCE_AUTO_POSITION0設 1 啟用 GTID 自動定位
SOURCE_CONNECT_RETRY60 秒重連間隔
SOURCE_RETRY_COUNT10⚠️ 8.4 從 86400 改為 10
SOURCE_DELAY0延遲複寫秒數
GET_SOURCE_PUBLIC_KEY0caching_sha2 需設 1
REQUIRE_ROW_FORMAT—強制 ROW 格式
FOR CHANNEL 'name'—多源複寫通道
🟢 GTID 概念

格式：server_uuid:transaction_id，如 3E11FA47-...:23。8.4 新增 Tagged GTID：UUID:TAG:NUMBER。

變數預設說明
gtid_modeOFF必須逐級切換：OFF→OFF_PERMISSIVE→ON_PERMISSIVE→ON
enforce_gtid_consistencyOFFgtid_mode=ON 前必須設 ON
gtid_executed—已提交的所有 GTID 集合（唯讀）
gtid_purged—已提交但從 binlog 清除的 GTID
🔶 半同步複寫（Semisynchronous）
-- Source 端
INSTALL PLUGIN rpl_semi_sync_source SONAME 'semisync_source.so';
SET GLOBAL rpl_semi_sync_source_enabled = 1;

-- Replica 端
INSTALL PLUGIN rpl_semi_sync_replica SONAME 'semisync_replica.so';
SET GLOBAL rpl_semi_sync_replica_enabled = 1;
STOP REPLICA IO_THREAD; START REPLICA IO_THREAD;
等待點行為安全性
AFTER_SYNC（預設）寫 binlog → 等 ACK → 提交引擎無損（Lossless）
AFTER_COMMIT寫 binlog → 提交引擎 → 等 ACK其他 session 可先看到未確認資料

逾時 rpl_semi_sync_source_timeout=10000ms 後降級為非同步。監控：SHOW STATUS LIKE 'Rpl_semi_sync%';

🔶 多源複寫 & 過濾器
-- 多源複寫（最多 256 通道）
CHANGE REPLICATION SOURCE TO
  SOURCE_HOST='source1', SOURCE_AUTO_POSITION=1
  FOR CHANNEL 'ch1';

CHANGE REPLICATION SOURCE TO
  SOURCE_HOST='source2', SOURCE_AUTO_POSITION=1
  FOR CHANNEL 'ch2';

START REPLICA FOR CHANNEL 'ch1';

-- 過濾器
CHANGE REPLICATION FILTER
  REPLICATE_DO_DB = (mydb)
  FOR CHANNEL 'ch1';
🟢 複寫監控
SHOW REPLICA STATUS\G
-- 關鍵欄位：Replica_IO_Running, Replica_SQL_Running,
-- Seconds_Behind_Source, Last_IO_Error, Last_SQL_Error

-- Performance Schema（推薦）
SELECT * FROM performance_schema.replication_connection_status\G
SELECT * FROM performance_schema.replication_applier_status\G
SELECT * FROM performance_schema.replication_applier_status_by_worker\G

Group Replication — 群組複寫

🟢 群組複寫概觀

基於 Paxos 共識協議的虛擬同步複寫。容錯公式 n = 2f + 1（3 個成員容忍 1 個故障，5 個容忍 2 個）。

單主模式（預設）多主模式
寫入僅 PRIMARY所有成員
唯讀SECONDARY 自動 super_read_only=ON無
變數group_replication_single_primary_mode=ON=OFF
限制—不支援 SERIALIZABLE、多層 FK CASCADE 受限
🟢 GR 設定逐步指南
-- ═══ my.cnf（每個成員）═══
[mysqld]
server_id=1
disabled_storage_engines="MyISAM,BLACKHOLE,FEDERATED,ARCHIVE,MEMORY"
gtid_mode=ON
enforce_gtid_consistency=ON
plugin_load_add='group_replication.so'
group_replication_group_name="aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa"
group_replication_start_on_boot=off
group_replication_local_address="s1:33061"
group_replication_group_seeds="s1:33061,s2:33061,s3:33061"
group_replication_bootstrap_group=off
-- 建立複寫帳號（每個成員）
SET SQL_LOG_BIN=0;
CREATE USER 'rpl_user'@'%' IDENTIFIED BY 'Str0ngP@ss';
GRANT REPLICATION SLAVE, CONNECTION_ADMIN, BACKUP_ADMIN,
      GROUP_REPLICATION_STREAM ON *.* TO 'rpl_user'@'%';
SET SQL_LOG_BIN=1;

-- 設定復原通道
CHANGE REPLICATION SOURCE TO
  SOURCE_USER='rpl_user', SOURCE_PASSWORD='Str0ngP@ss'
  FOR CHANNEL 'group_replication_recovery';

-- Bootstrap 第一個成員（僅一次！）
SET GLOBAL group_replication_bootstrap_group=ON;
START GROUP_REPLICATION;
SET GLOBAL group_replication_bootstrap_group=OFF;

-- 其他成員加入
START GROUP_REPLICATION;
🚨絕對不能同時在多個成員上 bootstrap——否則建立兩個同名獨立群組（人工腦裂）。
🟢 GR 監控
-- 成員狀態
SELECT MEMBER_HOST, MEMBER_PORT, MEMBER_STATE, MEMBER_ROLE
FROM performance_schema.replication_group_members;
-- MEMBER_STATE: ONLINE / RECOVERING / ERROR / UNREACHABLE
-- MEMBER_ROLE: PRIMARY / SECONDARY

-- 交易統計
SELECT MEMBER_ID,
  COUNT_TRANSACTIONS_IN_QUEUE,
  COUNT_CONFLICTS_DETECTED,
  COUNT_TRANSACTIONS_REMOTE_APPLIED
FROM performance_schema.replication_group_member_stats\G
🔶 一致性等級（8.4 重大變更）
等級行為
EVENTUAL不等待，可能讀到過時資料
BEFORE_ON_PRIMARY_FAILOVER
（8.4 預設）		新 Primary 阻塞新交易直到積壓完成，防止過時讀
BEFORE讀寫前等待所有先前交易完成
AFTER提交後等待所有成員套用
BEFORE_AND_AFTERBEFORE + AFTER 組合（最強一致性）
💡8.4 預設從 EVENTUAL 升級為 BEFORE_ON_PRIMARY_FAILOVER，從根本消除 failover 後的過時讀取與假性回滾。
🔶 自動容錯轉移 & 流量控制

選舉優先級：① 最低 MySQL 版本 → ② 最高 group_replication_member_weight（預設 50）→ ③ 最低 server_uuid。

-- 手動切換 Primary
SELECT group_replication_set_as_primary('member_uuid');

-- 調整選舉權重
SET GLOBAL group_replication_member_weight = 80;

group_replication_flow_control_modeQUOTAQUOTA=啟用, DISABLED=停用

..._certifier_threshold25000認證佇列觸發門檻
