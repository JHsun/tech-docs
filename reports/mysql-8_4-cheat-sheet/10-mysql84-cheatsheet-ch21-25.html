MySQL Shell（mysqlsh）

🟢 三種語言模式與切換
模式提示符切換API 風格
SQLmysql-sql>\sql—
JavaScriptmysql-js>\jscamelCase
Pythonmysql-py>\pysnake_case
# 啟動方式
mysqlsh                          -- 互動模式（預設 JS）
mysqlsh --sql user@host:3306    -- SQL 模式 + 傳統協定
mysqlsh --py                     -- Python 模式
mysqlsh --mysqlx user@host       -- X Protocol（埠 33060）
mysqlsh --mysql user@host        -- 傳統協定（埠 3306）
🟢 連線方式
// URI 連線
mysqlsh mysqlx://user@host:33060/mydb
mysqlsh mysql://user@host:3306

// 啟動後連線
\connect user@host:3306
\connect --mysqlx user@host

// JS/Python 程式化連線
shell.connect('user@host:3306')
shell.connect({user:'myuser', host:'host', port:3306})
🟢 Shell 工具集（util）
// 平行匯出（多執行緒 + zstd 壓縮）

util.dumpTables("mydb", ["t1"], "/path/dump")

// 平行匯入（支援中斷續傳）

// 表格匯出/匯入

util.importTable("/path/data.csv", {schema:"mydb", table:"t1"})

// 升級相容性檢查

// 8.4 新增：跨實例複製
util.copyInstance(targetURI)
util.copySchemas(["db1"], targetURI)

Document Store（X DevAPI）

🟢 核心概念

透過 X Protocol（埠 33060），MySQL 同時支援 NoSQL 文件操作與 SQL 查詢。Collection 存放 JSON 文件，每個文件必須有 _id 欄位。底層使用 InnoDB，享有完整 ACID 保證。

// 連線 + 取得 Schema / Collection
var session = mysqlx.getSession('user@localhost:33060')
var db = session.getSchema('test')
var coll = db.createCollection('users')  // 建立
var coll = db.getCollection('users')     // 取得既有
db.dropCollection('users')               // 刪除
🟢 CRUD 操作
// add() — 新增文件
coll.add({name:'Alice', age:30}).execute()
coll.add([{name:'Bob'}, {name:'Carol'}]).execute()

// find() — 查詢
coll.find('age > :a').bind('a', 25)
  .fields(['name', 'age'])
  .sort('age DESC').limit(10).execute()

// modify() — 修改（必須有條件）
coll.modify('name = "Alice"').set('age', 31).execute()
coll.modify('_id = "123"').unset('age').execute()
coll.modify('_id = "123"').patch({city:'Taipei'}).execute()

// remove() — 刪除（必須有條件）
coll.remove('age < 18').execute()

// 便捷單文件操作
coll.getOne(id)
coll.replaceOne(id, newDoc)
coll.removeOne(id)
🔶 Collection 索引 & JSON Schema 驗證
// 建立索引
coll.createIndex("age_idx", {
  fields: [{field: "$.age", type: "INT"}]
})
coll.createIndex("name_idx", {
  fields: [{field: "$.name", type: "TEXT(50)"}]
})

// JSON Schema 驗證
db.createCollection("coords", {
  validation: {
    level: "strict",
    schema: {
      type: "object",
      properties: {
        lat: {type:"number", minimum:-90, maximum:90},
        lng: {type:"number", minimum:-180, maximum:180}
      },
      required: ["lat", "lng"]
    }
  }
})

InnoDB Cluster

🟢 架構：GR + Shell + Router
  應用程式 → MySQL Router (6446 RW / 6447 RO)
                    ↓
     ┌──────────────┼──────────────┐
     │ MySQL 1      │ MySQL 2      │ MySQL 3
     │ PRIMARY(R/W) │ SECONDARY    │ SECONDARY
     └──────────────┴──────────────┘
           Group Replication (Paxos)

最少 3 個實例容忍 1 個故障。預設 Single-Primary 模式。

🟢 建立 InnoDB Cluster
// Step 1：檢查實例
dba.checkInstanceConfiguration('icadmin@ic-1:3306')

// Step 2：設定實例（含建立管理帳號）
dba.configureInstance('root@ic-1:3306', {
  clusterAdmin: 'icadmin',
  clusterAdminPassword: 'StrongP@ss!'
})

// Step 3：建立叢集
shell.connect('icadmin@ic-1:3306')
var cluster = dba.createCluster('prodCluster')

// Step 4：新增實例
cluster.addInstance('icadmin@ic-2:3306')
cluster.addInstance('icadmin@ic-3:3306', {recoveryMethod: 'clone'})
🟢 MySQL Router 整合
# Bootstrap（自動偵測叢集組態）
mysqlrouter --bootstrap icadmin@ic-1:3306 --directory /tmp/myrouter
/tmp/myrouter/start.sh
埠協定模式路由
6446ClassicR/W→ PRIMARY
6447ClassicR/O→ SECONDARY（round-robin）
6448X ProtocolR/W→ PRIMARY
6449X ProtocolR/O→ SECONDARY
🟢 叢集監控
cluster.status()              // 狀態總覽
cluster.status({extended:1})  // + GR 詳細資訊
cluster.describe()            // 結構描述
狀態含義
OK上線，可容忍故障
OK_NO_TOLERANCE上線但零容錯
NO_QUORUM失去多數派
🔶 故障復原與模式切換
// 重新加入實例
cluster.rejoinInstance('ic-2:3306')

// 法定人數丟失復原
cluster.forceQuorumUsingPartitionOf('icadmin@ic-1:3306')

// 全面停機後重啟
dba.rebootClusterFromCompleteOutage()

// 模式切換
cluster.switchToMultiPrimaryMode()
cluster.switchToSinglePrimaryMode()
cluster.setPrimaryInstance('ic-2:3306')
🔶 ClusterSet（跨資料中心災難復原）
// 建立 ClusterSet
var cs = cluster.createClusterSet('myClusterSet')

// 建立複製叢集
var cluster2 = cs.createReplicaCluster("dc2-1:3306", "dc2Cluster")
cluster2.addInstance('dc2-2:3306')

// 計畫性切換
cs.setPrimaryCluster('dc2Cluster')

// 緊急故障轉移（可能資料遺失）
cs.forcePrimaryCluster('dc2Cluster')

InnoDB ReplicaSet

🟢 概觀：簡化版非同步複寫
InnoDB ClusterInnoDB ReplicaSet
複製Group Replication（同步）非同步 GTID
自動故障轉移✅❌ 手動
Multi-Primary✅❌
網路要求穩定低延遲可容忍不穩定
寫入效能中高
最少節點32
🟢 建立與管理
// 設定實例
dba.configureReplicaSetInstance('root@rs-1:3306', {clusterAdmin:'rsadmin'})

// 建立 ReplicaSet
shell.connect('rsadmin@rs-1:3306')
var rs = dba.createReplicaSet('myRS')
rs.addInstance('rsadmin@rs-2:3306')

// 監控
rs.status()
rs.status({extended: 1})

// 計畫性切換（所有實例可達）
rs.setPrimaryInstance('rs-2:3306')

// 緊急故障轉移（主節點不可用，可能遺失資料）
rs.forcePrimaryInstance('rs-2:3306')
⚠️forcePrimaryInstance 僅在主節點不可用且無法復原時使用。原主節點被標記為 invalidated，需移除後重新加入。

NDB Cluster（簡介）

🟢 概觀

NDB Cluster 是獨立產品，不包含在標準 MySQL 8.4 中。採用 shared-nothing 架構的分散式記憶體內資料庫，設計目標為 99.999% 可用性與亞秒級故障轉移。

節點類型程式角色
Managementndb_mgmd組態管理、叢集協調
Datandbd/ndbmtd記憶體內資料儲存、自動分片
SQLmysqld --ndbclusterSQL 存取介面
🔶 三大叢集方案比較
InnoDB ClusterReplicaSetNDB Cluster
複製GR（同步共識）非同步 GTID同步二階段提交
自動故障轉移✅❌ 手動✅（< 1 秒）
寫入擴展有限單一主節點自動分片
隔離等級全部全部僅 READ COMMITTED
管理AdminAPIAdminAPIndb_mgm
適用通用高可用讀擴展電信/金融
