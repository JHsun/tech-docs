🟢 選項檔案位置與讀取順序
平台讀取順序（先→後，後者覆蓋前者）
Linux / macOS/etc/my.cnf → /etc/mysql/my.cnf → $MYSQL_HOME/my.cnf → ~/.my.cnf
Windows%WINDIR%\my.ini → C:\my.ini → INSTALLDIR\my.ini → %APPDATA%\MySQL\.mylogin.cnf
# my.cnf 範例結構
[mysqld]                  # 伺服器選項
port            = 3306
datadir         = /var/lib/mysql
innodb_buffer_pool_size = 4G

[client]                  # 所有客戶端程式共用
port            = 3306
socket          = /var/lib/mysql/mysql.sock

[mysql]                   # 僅 mysql CLI
auto-rehash
prompt          = "\\u@\\h [\\d]> "
💡使用 mysqld --print-defaults 可顯示伺服器讀取的所有選項值，除錯時非常有用。
🟢 連線選項速查
選項縮寫說明範例
--host-h伺服器主機-h 192.168.1.10
--user-u使用者名稱-u root
--password-p密碼（後面無空格）-p（互動提示）
--port-PTCP 埠號-P 3307
--socket-SUnix socket 路徑-S /tmp/mysql.sock
--protocol連線協議--protocol=TCP
--ssl-modeSSL 模式--ssl-mode=REQUIRED
--compress-C啟用壓縮-C
--compression-algorithms壓縮演算法zstd,zlib,uncompressed
⚠️-p 與密碼之間不能有空格（-pMyPass ✅、-p MyPass ❌ MyPass 會被解讀為資料庫名）。建議只用 -p 讓系統互動式提示。
🟢 mysql CLI 常用指令
# 連線並指定資料庫
mysql -u root -p -h 127.0.0.1 -P 3306 mydb

# 執行單條 SQL 並退出
mysql -u root -p -e "SHOW DATABASES"

# 匯入 SQL 腳本
mysql -u root -p mydb < script.sql

# 輸出為 Tab 分隔（適合管線處理）
mysql -u root -p -B -N -e "SELECT user,host FROM mysql.user"
互動模式指令說明
\G垂直顯示結果（取代 ;）
\c取消目前查詢
\s / status顯示連線狀態
\T file / tee file將輸出同時寫入檔案
\t / notee停止 tee
source file.sql執行 SQL 腳本
\! cmd / system cmd執行 shell 指令
pager less設定分頁器
nopager取消分頁器
rehash重建自動完成快取
warnings / nowarning顯示/隱藏警告

選項說明

mysqladmin -u root -p ping                 # 檢查存活

mysqladmin -u root -p extended-status      # = SHOW GLOBAL STATUS

mysqladmin -u root -p -i 2 -c 5 status   # 每 2 秒一次，共 5 次
🟢 mysqlcheck — 表維護
mysqlcheck -u root -p --all-databases                 # 檢查全部
mysqlcheck -u root -p --repair mydb mytable            # 修復特定表
mysqlcheck -u root -p --analyze --all-databases       # 更新統計
mysqlcheck -u root -p --optimize mydb                  # 回收碎片
mysqlcheck -u root -p --check-upgrade --all-databases # 升級檢查
🟢 mysqldump — 邏輯備份
mysqldump -u root -p mydb > mydb.sql                   # 單一 DB
mysqldump -u root -p mydb t1 t2 > tables.sql            # 特定表
mysqldump -u root -p --all-databases > full.sql         # 全部
mysqldump -u root -p --no-data mydb > schema.sql        # 僅結構
mysqldump -u root -p --no-create-info mydb > data.sql  # 僅資料

# 還原
mysql -u root -p mydb < mydb.sql
🔶 mysqldump 進階選項
選項說明
--single-transactionInnoDB 一致性快照（不鎖表）— 最重要
--routines / -R包含 stored procedures / functions
--events / -E包含 events
--triggers包含 triggers（預設啟用）
--set-gtid-purged=OFF不含 GTID（匯入非複寫環境）
--source-data=2記錄 binlog 位置（註解形式）
--flush-logs匯出前刷新 binlog
--max-allowed-packet=256M大型 BLOB 必要
--quick逐行擷取（不快取整表至記憶體）
--hex-blob十六進位匯出 BINARY
--column-statistics=0避免 ANALYZE TABLE 權限問題
# 生產環境推薦備份
mysqldump -u root -p \
  --single-transaction \
  --routines --events --triggers \
  --source-data=2 --hex-blob \
  --all-databases | gzip > full_$(date +%F).sql.gz
⚠️8.4 已將 --master-data 改為 --source-data。舊選項仍可用但已棄用。
🔶 mysqlbinlog — 二進位日誌工具
# 查看 binlog
mysqlbinlog binlog.000042

# 按時間範圍
mysqlbinlog --start-datetime="2025-01-15 10:00:00" \
            --stop-datetime="2025-01-15 11:00:00" binlog.000042

# 按位置
mysqlbinlog --start-position=154 --stop-position=1024 binlog.000042

# 重放至伺服器（時間點還原）
mysqlbinlog binlog.000042 binlog.000043 | mysql -u root -p

# 解碼 ROW 事件（8.4 預設 binlog_format=ROW）
mysqlbinlog -v binlog.000042       # -v -v 顯示欄位型別

# 從遠端讀取
mysqlbinlog --read-from-remote-server --host=remote -u repl -p binlog.000042
🔶 mysqlslap 壓力測試 & mysqldumpslow 慢查詢摘要
# 壓力測試（50 併發、200 迭代）
mysqlslap -u root -p \
  --concurrency=50 --iterations=200 \
  --auto-generate-sql --auto-generate-sql-load-type=mixed

# 自訂查詢壓測
mysqlslap -u root -p \
  --query="SELECT * FROM mydb.orders WHERE status='pending'" \
  --concurrency=10 --iterations=100
# 慢查詢摘要（依平均時間排序，前 10 條）
mysqldumpslow -s at -t 10 /var/lib/mysql/slow.log

# -s 排序: c=count, t=time, l=lock, r=rows, at=avg time, al=avg lock

MySQL Server Administration — 伺服器管理

🟢 系統變數 — 查看與設定
-- 查看全域變數
SHOW GLOBAL VARIABLES;
SHOW GLOBAL VARIABLES LIKE 'innodb_buffer%';
SHOW SESSION VARIABLES LIKE 'wait_timeout';

-- Performance Schema 查詢（更靈活）
SELECT * FROM performance_schema.global_variables
WHERE VARIABLE_NAME LIKE '%buffer%';

-- 設定（立即生效，重啟失效）
SET GLOBAL max_connections = 500;
SET SESSION sql_mode = 'STRICT_TRANS_TABLES,NO_ENGINE_SUBSTITUTION';

-- 持久化（寫入 mysqld-auto.cnf，重啟仍有效）
SET PERSIST max_connections = 500;
SET PERSIST_ONLY innodb_buffer_pool_size = 8589934592;  -- 唯讀變數，須重啟

-- 還原持久化
RESET PERSIST max_connections;
RESET PERSIST;  -- 清除全部
💡SET PERSIST 同時變更執行中值並寫入 mysqld-auto.cnf（JSON）。SET PERSIST_ONLY 僅寫檔，適用於唯讀變數。
🔶 關鍵系統變數速查
變數8.4 預設說明調校建議
max_connections151最大連線數注意每連線記憶體消耗
innodb_buffer_pool_size128MInnoDB 緩衝池專用 DB 設 70-80% 實體記憶體
innodb_redo_log_capacity100MRedo log 容量（取代舊 innodb_log_file_size）高寫入設 1-2G
innodb_flush_log_at_trx_commit1交易持久性1=ACID；2=秒級風險；0=最快
innodb_file_per_tableON每表獨立表空間保持 ON
long_query_time10（秒）慢查詢閾值建議 1-2 秒
wait_timeout28800空閒連線超時Web 應用建議 300-600
thread_cache_size-1（自動）執行緒快取8.4 自動調整
table_open_cache4000表快取大型系統可增至 8000+
tmp_table_size16M記憶體暫存表上限搭配 max_heap_table_size
sort_buffer_size256K排序緩衝謹慎調整，每連線分配
join_buffer_size256KJOIN 緩衝謹慎調整，每連線分配
🟢 SQL Mode
SELECT @@GLOBAL.sql_mode;
SET GLOBAL sql_mode = 'STRICT_TRANS_TABLES,NO_ENGINE_SUBSTITUTION,ONLY_FULL_GROUP_BY';
模式說明
STRICT_TRANS_TABLES嚴格模式 — 無效資料產生錯誤
ONLY_FULL_GROUP_BYSELECT 非聚合欄位須在 GROUP BY 中
NO_ZERO_IN_DATE禁止月/日為 0
NO_ZERO_DATE禁止 '0000-00-00'
ERROR_FOR_DIVISION_BY_ZERO除以零產生錯誤
NO_ENGINE_SUBSTITUTION指定引擎不可用時報錯

MySQL 8.4 預設 sql_mode 包含上述所有六項。

🟢 伺服器日誌系統
日誌用途啟用預設位置
Error Log啟動/關閉/嚴重錯誤預設啟用datadir/hostname.err
General Query Log所有 SQL（除錯用）general_log=ONdatadir/hostname.log
Slow Query Log超過閾值的查詢slow_query_log=ONdatadir/*-slow.log
Binary Log資料變更（複寫/PITR）8.4 預設 ONdatadir/binlog.000001
Relay LogReplica 接收的 binlog複寫自動建立datadir/*-relay-bin.*
🔶 Error Log 組件化架構
-- 預設：log_error_services = 'log_filter_internal; log_sink_internal'

-- 啟用 JSON 格式
INSTALL COMPONENT 'file://component_log_sink_json';
SET PERSIST log_error_services =
  'log_filter_internal; log_sink_internal; log_sink_json';

-- 啟用 syslog
INSTALL COMPONENT 'file://component_log_sink_syseventlog';

-- 日誌詳細程度
SET PERSIST log_error_verbosity = 3;  -- 1=Error 2=+Warning 3=+Note
🟢 Slow Query Log 設定
SET GLOBAL slow_query_log = 1;
SET GLOBAL long_query_time = 1;                -- 超過 1 秒記錄
SET GLOBAL log_queries_not_using_indexes = 1;  -- 記錄未用索引的查詢
SET GLOBAL min_examined_row_limit = 1000;
SET GLOBAL slow_query_log_file = '/var/log/mysql/slow.log';

-- 也可輸出到表
SET GLOBAL log_output = 'TABLE';  -- 'FILE' | 'TABLE' | 'FILE,TABLE'
🔶 Binary Log 管理
SHOW BINARY LOG STATUS;           -- 8.4 取代 SHOW MASTER STATUS
SHOW BINARY LOGS;                 -- 列出所有 binlog
SHOW BINLOG EVENTS IN 'binlog.000042' LIMIT 20;

FLUSH BINARY LOGS;                -- 手動輪替

-- 清除舊 binlog
PURGE BINARY LOGS TO 'binlog.000040';
PURGE BINARY LOGS BEFORE '2025-01-01 00:00:00';

-- 自動清除
SET PERSIST binlog_expire_logs_seconds = 604800;  -- 7 天

log_binON啟用 binlog
binlog_formatROWROW / STATEMENT / MIXED
binlog_row_imagefullROW 記錄的欄位範圍
binlog_expire_logs_seconds2592000（30天）自動清除
max_binlog_size1G單檔上限
sync_binlog1每次交易刷盤
binlog_transaction_compressionOFFzstd 壓縮

INSTALL PLUGIN clone SONAME 'mysql_clone.so';

-- 本地複製
CLONE LOCAL DATA DIRECTORY = '/backup/clone-data';

-- 遠端複製（Recipient 執行）
SET GLOBAL clone_valid_donor_list = 'donor:3306';
CLONE INSTANCE FROM 'user'@'donor':3306

-- 監控進度

🔶 單機多實例
# my.cnf 設定
[mysqld_multi]
mysqld     = /usr/bin/mysqld_safe
mysqladmin = /usr/bin/mysqladmin

[mysqld1]
port    = 3306
datadir = /var/lib/mysql1
socket  = /tmp/mysql1.sock

[mysqld2]
port    = 3307
datadir = /var/lib/mysql2
socket  = /tmp/mysql2.sock

# 管理
mysqld_multi start 1
mysqld_multi stop 2
mysqld_multi report

# systemd 方式（推薦）
systemctl start mysqld@instance1
systemctl start mysqld@instance2

每個實例須有獨立 datadir、port、socket、pid-file、log-error。
