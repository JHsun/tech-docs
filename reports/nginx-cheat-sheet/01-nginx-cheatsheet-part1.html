02

啟動、停止與重載 Starting, Stopping, Reloading

⚡ nginx -s 信號速查
指令說明
nginx啟動 nginx
nginx -s stop快速關閉
nginx -s quit優雅關閉（等現有請求完成）
nginx -s reload重新載入設定（零停機）
nginx -s reopen重新開啟日誌檔
nginx -t測試設定檔語法
nginx -T測試並印出完整設定
nginx -V顯示版本與編譯參數
ℹ️ reload 運作原理master 收到 reload → ① 驗證語法 → ② 啟動新 worker → ③ 通知舊 worker 優雅關閉 → ④ 舊 worker 服務完畢後退出。語法錯誤則回滾。
03

信號控制（進階）Signal Control

📡 Master Process 信號
信號等效作用
TERM/INT-s stop快速關閉
QUIT-s quit優雅關閉
HUP-s reload重載設定
USR1-s reopen重開日誌
USR2—在線升級執行檔
WINCH—優雅關閉 worker
# 透過 kill 發送
kill -s QUIT $(cat /var/run/nginx.pid)
🔄 在線升級 (Binary Upgrade)
# 1. 替換執行檔 → 2. 啟動新 master
kill -USR2 $(cat /var/run/nginx.pid)
# 3. 優雅關閉舊 worker
kill -WINCH $(cat /var/run/nginx.pid.oldbin)
# 4. 確認 OK → 關閉舊 master
kill -QUIT $(cat /var/run/nginx.pid.oldbin)
# ❌ 回滾
kill -HUP $(cat /var/run/nginx.pid.oldbin)
kill -QUIT $(cat /var/run/nginx.pid)
📋 日誌輪替
mv /var/log/nginx/access.log /var/log/nginx/access.log.0
kill -USR1 $(cat /var/run/nginx.pid)
gzip /var/log/nginx/access.log.0

設定檔結構 Configuration Structure

🏗️ 層次結構
main (全域)
├── events { }               # 連線處理
├── http { }                  # HTTP
│   ├── upstream name { }    # 上游群組
│   ├── server { }           # 虛擬主機
│   │   └── location / { }  # 路徑匹配
│   └── server { }
├── stream { }                # TCP/UDP
└── mail { }                  # 郵件
📝 簡單指令
worker_processes  auto;
error_log  /var/log/nginx/error.log;
pid        /var/run/nginx.pid;
📦 區塊指令 (Context)
events {
    worker_connections  1024;
}
http {
    server { listen 80; }
}
⚠️ 所有 context 之外的指令屬於 main context。# 為註解。
05

靜態檔案服務 Serving Static Content

📁 基本範例
server {
    listen  80;
    location / {
        root  /data/www;       # /page.html → /data/www/page.html
    }
    location /images/ {
        root  /data;            # /images/cat.png → /data/images/cat.png
    }
}
🆚 root vs alias
指令計算範例
root /data;root + URI/img/a.jpg → /data/img/a.jpg
alias /data/img/;取代前綴/img/a.jpg → /data/img/a.jpg
📂 location 匹配優先順序
語法類型優先級
= /path精確匹配① 最高
^~ /path前綴（停止正則）②
~ regex正則（區分大小寫）③ 依序
~* regex正則（不分大小寫）③ 依序
/path一般前綴④ 最長
/萬用兜底⑤ 最低
ℹ️ 匹配流程先找所有前綴記最長 → 依序查正則 → 命中用正則 → 否則用最長前綴。
06

反向代理入門 Reverse Proxy

🔀 基本反向代理
server {
    listen  80;
    location / {
        proxy_pass  http://localhost:8080;
    }
    location ~ \.(gif|jpg|png)$ {
        root  /data/images;
    }
}
📋 常用 proxy 標頭
proxy_set_header Host              $host;
proxy_set_header X-Real-IP         $remote_addr;
proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
proxy_set_header X-Forwarded-Proto $scheme;
⏱️ 超時設定
指令預設說明
proxy_connect_timeout60s建立連線超時
proxy_read_timeout60s等待回應超時
proxy_send_timeout60s發送請求超時
07

FastCGI 代理 PHP-FPM

🐘 PHP-FPM 典型設定
server {
    listen 80;  server_name example.org;
    root /data/www;

    location / { index index.html index.php; }

    location ~ \.php$ {
        fastcgi_pass   localhost:9000;
        fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
        fastcgi_param  QUERY_STRING     $query_string;
        include        fastcgi_params;
    }

    location ~* \.(gif|jpg|png)$ { expires 30d; }
}
08

請求處理流程 Request Processing

🔍 虛擬主機選擇
步驟說明
①比對 listen 的 IP:port
②用 Host 比對 server_name
③無匹配 → 用 default_server
④選定 server → 匹配 location
# 丟棄未知 Host
server {
    listen 80 default_server;
    server_name "";
    return 444;
}
⚠️ default_server 是 listen port 的屬性，不同 port 可有不同預設。
09

server_name 規則 Server Names

🏷️ 優先順序
優先類型範例
①精確名稱example.org
②前置萬用（最長）*.example.org
③後置萬用（最長）mail.*
④正則（依序首命中）~^www\d+\.example\.net$
✨ 特殊名稱
名稱用途
""匹配無 Host 標頭
_常見 catch-all
$hostname機器主機名
.example.org= example.org + *.example.org
# 正則捕獲群組當變數
server_name ~^(www\.)?(?<domain>.+)$;
root /sites/$domain;
