HTTP 負載平衡 Load Balancing

⚖️ 三種內建演算法
演算法指令說明
Round-Robin預設（無需指令）輪流分配，可搭配 weight
Least Connectedleast_conn;分配到活動連線最少的伺服器
IP Haship_hash;同一 IP 固定到同一伺服器（Session 黏著）
📝 基本設定
http {
    upstream myapp {
        server srv1.example.com;
        server srv2.example.com;
        server srv3.example.com;
    }
    server {
        listen 80;
        location / {
            proxy_pass http://myapp;
        }
    }
}
🏋️ 加權 + least_conn
upstream myapp {
    least_conn;
    server srv1.example.com weight=3;  # 承載 3 倍流量
    server srv2.example.com;
    server srv3.example.com;
}
🔒 Session 黏著 (ip_hash)
upstream myapp {
    ip_hash;
    server srv1.example.com;
    server srv2.example.com;
    server srv3.example.com;
}
ℹ️ ip_hash 使同一用戶端 IP 始終導向同一伺服器，適合需要 session 的應用。

Upstream 伺服器群 ngx_http_upstream_module

🖥️ server 參數速查
參數預設說明
weight=n1權重
max_conns=n0 (無限)最大同時連線
max_fails=n1失敗次數閾值
fail_timeout=t10s失敗計算窗口 + 不可用期間
backup—備用伺服器（僅主伺服器全掛時啟用）
down—標記為永久不可用
slow_start=t—恢復後慢啟動時間 [商業版]
🔧 完整範例
upstream backend {
    server backend1.example.com       weight=5;
    server backend2.example.com:8080;
    server unix:/tmp/backend3;
    server backup1.example.com:8080   backup;
}

server {
    location / {
        proxy_pass http://backend;
    }
}
💪 Keepalive 連線池
upstream backend {
    server backend1.example.com;
    server backend2.example.com;
    keepalive 32;               # 每 worker 保持 32 個閒置連線
    keepalive_requests 1000;   # 單連線最多處理 1000 個請求
    keepalive_timeout 60s;
}

server {
    location / {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Connection "";
    }
}
03

HTTPS 設定 Configuring HTTPS Servers

🔐 基本 HTTPS
server {
    listen              443 ssl;
    server_name         www.example.com;
    ssl_certificate     www.example.com.crt;
    ssl_certificate_key www.example.com.key;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;
}
⚡ 效能優化設定
worker_processes auto;

http {
    ssl_session_cache   shared:SSL:10m;  # 10MB ≈ 40,000 sessions
    ssl_session_timeout 10m;

    server {
        listen              443 ssl;
        keepalive_timeout   70;
        ssl_certificate     www.example.com.crt;
        ssl_certificate_key www.example.com.key;
    }
}
💡 SSL 優化要點① worker_processes = CPU 核心數 · ② 啟用 keepalive 減少握手 · ③ shared session cache 跨 worker 共用 · ④ 增大 session timeout。
📜 憑證鏈組裝
# 伺服器憑證必須在中繼憑證之前
cat www.example.com.crt bundle.crt > www.example.com.chained.crt

ssl_certificate www.example.com.chained.crt;
🔄 HTTP 跳轉 HTTPS
server {
    listen 80;
    server_name example.com;
    return 301 https://$host$request_uri;
}

SSL/TLS 模組速查 ngx_http_ssl_module

📋 常用 SSL 指令
指令預設說明
ssl_certificate—PEM 格式憑證檔案路徑
ssl_certificate_key—私鑰檔案路徑
ssl_protocolsTLSv1.2 TLSv1.3允許的 TLS 版本
ssl_ciphersHIGH:!aNULL:!MD5加密套件
ssl_prefer_server_ciphersoff優先使用伺服器的加密順序
ssl_session_cachenonesession 快取類型與大小
ssl_session_timeout5msession 快取有效期
ssl_dhparam—DH 參數檔案
ssl_staplingoff啟用 OCSP Stapling
ssl_stapling_verifyoff驗證 OCSP 回應
ssl_trusted_certificate—用於驗證 OCSP 的信任 CA
ssl_early_dataoffTLS 1.3 0-RTT 早期資料
🛡️ 生產級 SSL 範本
ssl_protocols       TLSv1.2 TLSv1.3;
ssl_ciphers         ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;

ssl_session_tickets off;
ssl_stapling        on;
ssl_stapling_verify on;
ssl_trusted_certificate /path/to/chain.pem;

add_header Strict-Transport-Security "max-age=63072000" always;

Rewrite 模組 ngx_http_rewrite_module

🔀 核心指令
指令Context說明
rewrite regex replacement [flag]server, location, if用正則重寫 URI
return code [text|URL]server, location, if回傳狀態碼或重導向
if (condition) { }server, location條件判斷
set $var valueserver, location, if設定變數
breakserver, location, if停止執行 rewrite 指令
🚩 rewrite flag
Flag說明
last停止 rewrite，重新搜尋匹配的 location
break停止 rewrite，在當前 location 繼續處理
redirect302 暫時重導向
permanent301 永久重導向
📝 常見範例
# 301 永久重導向
rewrite ^/old-page$ /new-page permanent;

# 強制 www
if ($host !~ ^www\.) {
    return 301 https://www.$host$request_uri;
}

# 去除尾部斜線
rewrite ^/(.*)/$ /$1 permanent;

# URL 美化（/product/123 → /product.php?id=123）
rewrite ^/product/(\d+)$ /product.php?id=$1 last;
🔍 if 條件語法
運算子說明
= !=字串比較
~ ~*正則匹配（分/不分大小寫）
!~ !~*正則不匹配
-f !-f檔案存在/不存在
-d !-d目錄存在/不存在
-e !-e檔案/目錄/連結存在
-x !-x可執行/不可執行
⚠️ if 是邪惡的？在 location 內使用 if 可能產生非預期行為。官方建議：盡量用 try_files 和 return 替代。只有 return 和 rewrite 在 if 中是安全的。

WebSocket 代理 WebSocket Proxying

🔌 基本設定
location /chat/ {
    proxy_pass http://backend;
    proxy_set_header Upgrade    $http_upgrade;
    proxy_set_header Connection "upgrade";
}
🎯 進階：動態判斷 Connection
http {
    map $http_upgrade $connection_upgrade {
        default  upgrade;
        ''       close;
    }

    server {
        location /chat/ {
            proxy_pass http://backend;
            proxy_set_header Upgrade    $http_upgrade;
            proxy_set_header Connection $connection_upgrade;
        }
    }
}
ℹ️ WebSocket 連線預設 60 秒無資料會超時。用 proxy_read_timeout 調大，或讓後端定期發送 ping frame。

核心模組指令精選 ngx_core_module

⚙️ main context 關鍵指令
指令預設說明
worker_processes1Worker 數量，建議 auto
worker_connections512每 worker 最大連線數（含上游）
worker_rlimit_nofile—Worker 可開啟的最大 fd 數
usernobodyWorker 執行身份
pidnginx.pidPID 檔路徑
error_loglogs/error.log error錯誤日誌路徑與級別
include—包含其他設定檔
load_module—動態載入模組
pcre_jitoff啟用 PCRE JIT 加速正則
📝 典型 main context
user                www www;
worker_processes    auto;
worker_rlimit_nofile 65535;
error_log           /var/log/nginx/error.log warn;
pid                 /var/run/nginx.pid;

events {
    use                epoll;
    worker_connections 4096;
    multi_accept       on;
}
💡 最大並發連線數= worker_processes × worker_connections。若做反向代理，每個客戶端請求需要 2 個連線（客戶端 + 上游），所以實際可服務的客戶端數約為此數的一半。
