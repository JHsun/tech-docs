01

njs JavaScript 腳本 Scripting with njs

📜 什麼是 njs？

njs 是 nginx 內建的 JavaScript 子集（基於 ES5/ES6），允許在設定中嵌入腳本邏輯，用於複雜的請求處理、鑑權、內容生成、子請求等。比 Lua 模組更原生，無需第三方依賴。

啟用：需載入動態模組 load_module modules/ngx_http_js_module.so;

📋 HTTP njs 範例

// /etc/nginx/http.js
function hello(r) {
    r.return(200, `Hello from njs!
URI: ${r.uri}
Method: ${r.method}
Args: ${JSON.stringify(r.args)}`);
}

function authorize(r) {
    let token = r.headersIn['Authorization'];
    if (!token || token !== 'Bearer secret123') {
        r.return(401, 'Unauthorized');
        return;
    }
    r.return(200);  // auth_request 成功
}

export default { hello, authorize };
# nginx.conf
load_module modules/ngx_http_js_module.so;

http {
    js_import http from /etc/nginx/http.js;

    server {
        location /hello {
            js_content http.hello;
        }

        location /api/ {
            auth_request /auth;
            proxy_pass http://backend;
        }
        location = /auth {
            internal;
            js_content http.authorize;
        }
    }
}

🔧 njs 常用 API

HTTP 請求物件 r說明
r.uri當前 URI
r.methodHTTP 方法
r.args查詢參數物件
r.headersIn請求標頭
r.headersOut回應標頭
r.variablesnginx 變數存取
r.requestBody請求體
r.return(code, body)返回回應
r.internalRedirect(uri)內部重導
r.subrequest(uri, opts)發送子請求
r.log(msg)記錄日誌
njs 指令說明
js_import匯入 JS 模組
js_content用 JS 函數處理請求並產生回應
js_set用 JS 函數設定 nginx 變數
js_header_filter用 JS 修改回應標頭
js_body_filter用 JS 修改回應體
js_preload_object預載入資料物件
js_shared_dict_zone共享字典（Worker 間共享資料）

📡 Stream njs（TCP/UDP）

// stream.js — 基於 SNI 路由的自訂邏輯
function preread(s) {
    s.on('upload', function(data, flags) {
        // 解析 ClientHello 或自訂協議
        let host = extractSNI(data);
        s.variables.target = host === 'db' ? '10.0.0.1:3306' : '10.0.0.2:3306';
        s.done();
    });
}
export default { preread };
02

動態模組 Dynamic Modules

🔌 載入與管理

# 在 main context 最上方載入
load_module modules/ngx_http_js_module.so;
load_module modules/ngx_http_image_filter_module.so;
load_module modules/ngx_http_geoip_module.so;
load_module modules/ngx_stream_module.so;

動態模組自 1.9.11 起支援。編譯時使用 --with-xxx=dynamic 產生 .so 檔案，放到 modules/ 目錄下。

常見動態模組： njs (js_module)、image_filter、xslt、geoip、perl、mail、stream、http_v2 (舊版)、otel

開發指南摘要 Development Guide

🏗 原始碼編譯參數

./configure \
    --prefix=/etc/nginx \
    --sbin-path=/usr/sbin/nginx \
    --modules-path=/usr/lib64/nginx/modules \
    --conf-path=/etc/nginx/nginx.conf \
    --error-log-path=/var/log/nginx/error.log \
    --http-log-path=/var/log/nginx/access.log \
    --pid-path=/var/run/nginx.pid \
    --with-http_ssl_module \
    --with-http_v2_module \
    --with-http_v3_module \
    --with-http_realip_module \
    --with-http_gzip_static_module \
    --with-http_stub_status_module \
    --with-http_auth_request_module \
    --with-http_sub_module \
    --with-http_image_filter_module=dynamic \
    --with-stream \
    --with-stream_ssl_module \
    --with-stream_ssl_preread_module \
    --with-pcre-jit \
    --with-debug

📝 貢獻指南要點

  • 程式碼風格：C 語言，4 格空白縮排，不使用 Tab
  • Patch 透過郵件列表提交（非 GitHub PR）
  • 編譯環境：GCC / Clang，支援 Linux / FreeBSD / macOS / Windows
  • 測試：使用 Perl 測試框架 Test::Nginx
  • 文件與程式碼一起提交

排障手冊 Troubleshooting

🔍 常見問題排查

症狀可能原因排查方式
502 Bad Gateway後端未啟動 / 超時 / socket 權限檢查 error.log、upstream_response_time、後端存活
504 Gateway Timeout後端處理太慢調大 proxy_read_timeout，檢查後端效能
413 Request Entity Too Large請求體超過限制調大 client_max_body_size
403 Forbidden權限不足 / 無 index 檔檢查檔案權限、SELinux、autoindex
301 重定向迴圈HTTP→HTTPS 和 proxy 衝突檢查 $scheme 和 X-Forwarded-Proto
啟動失敗：地址已使用埠被佔用ss -tlnp | grep :80
設定語法錯誤拼寫 / 缺少分號 / 括號nginx -t
SSL handshake 失敗憑證 / 私鑰不匹配openssl x509 -noout -modulus -in cert 比對
Worker 崩潰Bug / 記憶體不足啟用 core dump、worker_rlimit_core

🛠 除錯工具箱

# 1. 測試設定
nginx -t
nginx -T | grep -i "server_name"

# 2. 查看運行狀態
ps aux | grep nginx
ss -tlnp | grep nginx

# 3. 啟用 debug 日誌
error_log /var/log/nginx/debug.log debug;

# 只對特定 IP 啟用 debug
events {
    debug_connection 192.168.1.100;
}

# 4. 即時觀察日誌
tail -f /var/log/nginx/error.log
tail -f /var/log/nginx/access.log | grep "502\|504\|500"

# 5. 測試連線
curl -I https://example.com
curl -v --resolve example.com:443:127.0.0.1 https://example.com

# 6. SSL 驗證
openssl s_client -connect example.com:443 -servername example.com

# 7. 查看 HTTP/2 和 QUIC
curl --http2 -I https://example.com
curl --http3 -I https://example.com

# 8. 壓力測試
ab -n 10000 -c 100 http://localhost/
wrk -t4 -c200 -d30s http://localhost/

ACME / OpenTelemetry Additional Modules

🔐 ACME 自動憑證

ngx_http_acme_module 讓 nginx 自動向 ACME CA（如 Let's Encrypt）申請和續約 TLS 憑證。

acme_client letsencrypt
    https://acme-v02.api.letsencrypt.org/directory;

server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate     $acme_certificate;
    ssl_certificate_key $acme_certificate_key;
}

📡 OpenTelemetry 可觀測性

ngx_otel_module 為 nginx 提供分散式追蹤（Tracing）和指標（Metrics）匯出。

load_module modules/ngx_otel_module.so;

http {
    otel_exporter {
        endpoint localhost:4317;
    }
    otel_service_name my-nginx;

    server {
        otel_trace on;
        otel_trace_context propagate;
    }
}

⚙ google_perftools & mgmt 模組

模組功能
ngx_google_perftools_module整合 Google Performance Tools 的 CPU Profiler，用於效能分析。透過 google_perftools_profiles 指定輸出檔案。
ngx_mgmt_moduleNGINX Plus 管理模組，支援使用權報告和叢集管理。
mgmt { usage_report endpoint=product.connect.nginx.com; }
