proxy_pass 進階設定 ngx_http_proxy_module

🔀 URI 傳遞規則
設定請求 /api/users轉送至
proxy_pass http://backend;完整 URIhttp://backend/api/users
proxy_pass http://backend/;去掉 location 前綴http://backend/users
proxy_pass http://backend/v2/;替換前綴http://backend/v2/users
⚠️ 尾部的 / 決定是否去掉 location 前綴 — 這是最常見的坑
📋 完整反向代理模板
location /api/ {
    proxy_pass              http://backend;
    proxy_http_version      1.1;        # 後端啟用 keepalive 必須 1.1
    proxy_set_header        Host $host;
    proxy_set_header        X-Real-IP $remote_addr;
    proxy_set_header        X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header        X-Forwarded-Proto $scheme;
    proxy_set_header        Connection ""; # keepalive 需清空

    # 超時設定
    proxy_connect_timeout   5s;
    proxy_send_timeout      60s;
    proxy_read_timeout      60s;

    # 緩衝
    proxy_buffering         on;
    proxy_buffer_size       4k;
    proxy_buffers           8 8k;
}

代理標頭與緩衝 Proxy Headers & Buffering

常用代理標頭
標頭建議值
Host$host
X-Real-IP$remote_addr
X-Forwarded-For$proxy_add_x_forwarded_for
X-Forwarded-Proto$scheme
Connection""（keepalive 時）
緩衝控制
指令說明
proxy_buffering on|off開/關緩衝（SSE 需關閉）
proxy_buffer_size讀取第一部分回應的緩衝
proxy_buffers N size回應 body 緩衝數量 × 大小
proxy_busy_buffers_size忙碌時可用的緩衝大小
proxy_request_buffering是否緩衝客戶端請求 body
💡 SSE/串流場景：proxy_buffering off;

upstream 負載平衡策略 Load Balancing Methods

策略語法說明
加權輪詢（預設）依 weight 依序分配；最常用
最少連線least_conn;選擇當前連線數最少的後端
IP 雜湊ip_hash;同一 IP 固定到同一後端（Session 黏性）
通用雜湊hash $key [consistent];自訂 key 雜湊；consistent 啟用一致性雜湊
隨機random [two [method]];隨機選兩個，再按 method 擇優
最短時間 ⭐least_time header|last_byte;按回應時間選擇（商業版）
📋 實戰範例
# 加權輪詢（預設）
upstream web_pool {
    server 10.0.0.1:8080 weight=5;
    server 10.0.0.2:8080 weight=3;
    server 10.0.0.3:8080 weight=2;
}

# 一致性雜湊（適合快取分佈）
upstream cache_pool {
    hash $request_uri consistent;
    server cache1:6379;
    server cache2:6379;
    server cache3:6379;
}

# IP 黏性（Session 需求）
upstream app_pool {
    ip_hash;
    server 10.0.0.1:3000;
    server 10.0.0.2:3000;
    server 10.0.0.3:3000 down;  # 標記下線不破壞雜湊
}

upstream server 參數 Server Parameters

參數預設說明
weight=N1權重（越大分配越多流量）
max_fails=N1在 fail_timeout 內允許的失敗次數
fail_timeout=Ns10s失敗統計窗口 & 標記不可用的時間
max_conns=N0最大同時連線數（0 = 不限）
backup—備援伺服器（主伺服器全掛才啟用）
down—永久標記為下線
resolve—動態 DNS 解析（商業版，需 zone）
slow_start=Ns—恢復後緩慢增加權重（商業版）

後端長連線 Upstream Keepalive

upstream backend {
    server 10.0.0.1:8080;
    server 10.0.0.2:8080;

    keepalive          32;    # 每 Worker 保留的空閒連線數
    keepalive_requests 1000;  # 單一連線最多處理請求數
    keepalive_timeout  60s;   # 空閒連線超時
}

server {
    location / {
        proxy_pass         http://backend;
        proxy_http_version 1.1;            # 必須！
        proxy_set_header   Connection ""; # 必須清空！
    }
}
⚠️ 後端 keepalive 三件套缺一不可：keepalive N + proxy_http_version 1.1 + Connection ""

proxy_cache 快取 Proxy Cache

💾 完整快取設定
http {
    # 定義快取區域
    proxy_cache_path /var/cache/nginx
        levels=1:2              # 子目錄層級
        keys_zone=my_cache:10m  # 10MB 共享記憶體存放 key
        max_size=10g            # 快取磁碟上限
        inactive=60m            # 60 分鐘無存取則清除
        use_temp_path=off;

    server {
        location / {
            proxy_pass             http://backend;
            proxy_cache            my_cache;
            proxy_cache_key        "$scheme$host$request_uri";
            proxy_cache_valid      200 1h;     # 200 快取 1 小時
            proxy_cache_valid      404 1m;     # 404 快取 1 分鐘
            proxy_cache_use_stale  error timeout updating
                                   http_500 http_502 http_503;
            proxy_cache_background_update on;  # 背景更新
            proxy_cache_lock       on;          # 避免快取穿透

            # 加入快取狀態標頭（除錯用）
            add_header X-Cache-Status $upstream_cache_status;
        }

        # 跳過快取的條件
        location /api/ {
            proxy_pass          http://backend;
            proxy_cache         my_cache;
            proxy_cache_bypass  $http_authorization $cookie_session;
            proxy_no_cache      $http_authorization $cookie_session;
        }
    }
}
$upstream_cache_status 值
值說明
HIT命中快取，直接回傳
MISS未命中，已向後端取得並快取
EXPIRED快取已過期，已重新取得
STALE回傳過期的快取（後端不可用時）
UPDATING背景更新中，回傳舊快取
BYPASS被 proxy_cache_bypass 跳過
REVALIDATED重新驗證（304）後回傳

WebSocket 代理 WebSocket Proxying

# WebSocket 需要 Upgrade hop-by-hop 標頭
map $http_upgrade $connection_upgrade {
    default  upgrade;
    ''       close;
}

server {
    location /ws/ {
        proxy_pass             http://websocket_backend;
        proxy_http_version     1.1;
        proxy_set_header       Upgrade $http_upgrade;
        proxy_set_header       Connection $connection_upgrade;
        proxy_set_header       Host $host;

        # WebSocket 超時（預設 60s 無資料就斷開）
        proxy_read_timeout     3600s;  # 1 小時
        proxy_send_timeout     3600s;
    }
}

gRPC 代理 ngx_http_grpc_module

upstream grpc_servers {
    server 10.0.0.1:50051;
    server 10.0.0.2:50051;
}

server {
    listen 443 ssl http2;  # gRPC 需要 HTTP/2

    ssl_certificate     /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        grpc_pass grpc://grpc_servers;
        # 或 TLS: grpcs://grpc_servers

        grpc_connect_timeout  5s;
        grpc_read_timeout     60s;
        grpc_send_timeout     60s;

        # 錯誤時切換下一台
        grpc_next_upstream    error timeout;
    }
}

被動健康檢查 Passive Health Checks

開源版 nginx 使用被動健康檢查：在實際請求中偵測後端失敗。

upstream backend {
    server 10.0.0.1:8080 max_fails=3 fail_timeout=30s;
    server 10.0.0.2:8080 max_fails=3 fail_timeout=30s;
    server 10.0.0.3:8080 backup;
}

location / {
    proxy_pass http://backend;

    # 定義哪些錯誤觸發切換
    proxy_next_upstream         error timeout http_502 http_503;
    proxy_next_upstream_tries   3;       # 最多嘗試 3 次
    proxy_next_upstream_timeout 10s;     # 總超時 10 秒
}
💡 機制：3 次失敗 → 標記不可用 30 秒 → 30 秒後再嘗試；全掛則啟用 backup
10

容錯切換 proxy_next_upstream

條件值說明
error與後端建立連線 / 傳輸 / 讀取時發生錯誤
timeout連線 / 讀取 / 寫入超時
invalid_header後端回傳無效的回應標頭
http_500後端回傳 500
http_502後端回傳 502
http_503後端回傳 503
http_504後端回傳 504
http_403後端回傳 403
http_404後端回傳 404
non_idempotent允許對 POST 等非冪等請求也重試
off完全不重試
⚠️ 預設只對冪等請求（GET, HEAD 等）重試。若要對 POST 重試需加 non_idempotent（小心重複提交）
