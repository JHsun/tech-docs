🍪 Cookie 與 Header 參數

必須明確使用 Cookie() / Header()，否則會被當成查詢參數。

from fastapi import Cookie, Header

@app.get("/items/")
async def read_items(
    ads_id: Annotated[str | None, Cookie()] = None,
    user_agent: Annotated[str | None, Header()] = None,
    x_token: Annotated[list[str] | None, Header()] = None,
):
    return {"user_agent": user_agent}

Header 特殊行為

特性說明
底線轉連字號user_agent → HTTP User-Agent
停用轉換Header(convert_underscores=False)
重複 Header用 list[str] 型別接收

Cookie / Header 參數模型 (≥ 0.115.0)

class Cookies(BaseModel):
    model_config = {"extra": "forbid"}
    session_id: str
    tracker: str | None = None

@app.get("/")
async def read(cookies: Annotated[Cookies, Cookie()]):
    return cookies
⚡
Swagger UI 限制：文件介面使用 JavaScript，Cookie 不會在點擊「Execute」時被發送。測試 Cookie 請用 curl 或 Postman。

📤 回應模型 — Response Model

回傳型別註解即為回應模型。response_model 裝飾器參數可覆蓋。

class UserIn(BaseModel):
    username: str
    password: str       # 敏感！
    email: str

class UserOut(BaseModel):
    username: str
    email: str          # 不含 password

@app.post("/user/", response_model=UserOut)
async def create_user(user: UserIn) -> Any:
    return user  # password 被 response_model 過濾掉

推薦繼承模式

class BaseUser(BaseModel):
    username: str; email: str

class UserIn(BaseUser):
    password: str

class UserOut(BaseUser):
    pass

class UserInDB(BaseUser):
    hashed_password: str

回應模型重要參數

參數效果
response_model_exclude_unset=True省略未明確設定的欄位
response_model_include={"name"}僅包含指定欄位
response_model_exclude={"tax"}排除指定欄位
response_model_exclude_none=True省略值為 None 的欄位
response_model=None停用回應模型產生
⚠️
response_model 裝飾器參數永遠優先於回傳型別註解。兩者衝突時以 response_model 為準。

🔢 回應狀態碼 — Response Status Code

from fastapi import status

@app.post("/items/", status_code=status.HTTP_201_CREATED)
async def create_item(item: Item):
    return item

常用狀態碼速查

常數數值用途
HTTP_200_OK200成功（GET 預設）
HTTP_201_CREATED201已建立（POST 常用）
HTTP_204_NO_CONTENT204成功但無回應內容（DELETE）
HTTP_301_MOVED_PERMANENTLY301永久重導向
HTTP_304_NOT_MODIFIED304未修改（快取）
HTTP_400_BAD_REQUEST400錯誤請求
HTTP_401_UNAUTHORIZED401未認證
HTTP_403_FORBIDDEN403無權限
HTTP_404_NOT_FOUND404找不到
HTTP_422_UNPROCESSABLE_ENTITY422驗證失敗（FastAPI 預設）
HTTP_500_INTERNAL_SERVER_ERROR500伺服器錯誤
ℹ️
status_code 放在裝飾器上，不是函式內。動態更改：使用 Response 參數模式 response.status_code = 201。

📎 表單資料與檔案上傳 — Form / File / UploadFile

全部需要 pip install python-multipart。Form 繼承自 Body；File 繼承自 Form。

⚠️
HTTP 協議限制：Form/File 不能與 JSON Body 混用。同一個 endpoint 只能選擇一種。
from fastapi import File, Form, UploadFile

@app.post("/files/")
async def create_file(
    file: Annotated[bytes, File()],          # 整個檔案載入記憶體
    fileb: Annotated[UploadFile, File()],     # 大檔案 → 暫存磁碟
    token: Annotated[str, Form()],
):
    return {
        "file_size": len(file),
        "filename": fileb.filename,
    }

UploadFile 屬性與方法

屬性型別說明
.filenamestr原始檔名
.content_typestrMIME 類型
.sizeint檔案大小（bytes）
.headersHeadersmultipart 表頭
.fileSpooledTemporaryFile底層檔案物件
非同步方法說明
await file.read()讀取全部內容
await file.read(size)讀取指定位元組
await file.write(data)寫入資料
await file.seek(0)回到開頭
await file.close()關閉檔案

多檔案上傳

@app.post("/uploadfiles/")
async def create_files(
    files: list[UploadFile]
):
    return {"filenames": [f.filename for f in files]}

表單模型 (≥ 0.113.0)

class LoginForm(BaseModel):
    username: str
    password: str

@app.post("/login/")
async def login(data: Annotated[LoginForm, Form()]):
    return {"user": data.username}

🚨 錯誤處理 — Handling Errors

HTTPException

from fastapi import HTTPException

@app.get("/items/{item_id}")
async def read_item(item_id: str):
    if item_id not in items:
        raise HTTPException(
            status_code=404,
            detail="Item not found",      # 任何可 JSON 化的值
            headers={"X-Error": "custom"},  # 自訂回應標頭
        )
    return items[item_id]

自訂例外處理器

from fastapi.exceptions import RequestValidationError
from starlette.exceptions import HTTPException as StarletteHTTPException

@app.exception_handler(StarletteHTTPException)
async def http_exception_handler(request, exc):
    return PlainTextResponse(str(exc.detail), status_code=exc.status_code)

@app.exception_handler(RequestValidationError)
async def validation_handler(request, exc):
    return JSONResponse(
        status_code=422,
        content={"detail": exc.errors(), "body": exc.body}
    )

自訂例外類別

class UnicornException(Exception):
    def __init__(self, name: str):
        self.name = name

@app.exception_handler(UnicornException)
async def unicorn_handler(request, exc):
    return JSONResponse(
        status_code=418,
        content={"message": f"Oops! {exc.name} did something."}
    )
💡
重用預設處理器：from fastapi.exception_handlers import request_validation_exception_handler，可在自訂處理器中呼叫以保留原始行為再加上額外邏輯。

⚙️ 路徑操作配置

所有配置參數放在裝飾器上：

@app.post(
    "/items/",
    status_code=status.HTTP_201_CREATED,
    tags=["items"],
    summary="建立項目",
    description="建立一個新的項目...",
    response_description="已建立的項目",
    deprecated=False,
)
async def create_item(item: Item):
    """
    建立項目，需要以下參數：

    - **name**: 項目名稱
    - **price**: 價格（必須大於零）

    支援 Markdown 格式！
    \f
    此行以下不會出現在文件中。
    """
    return item

裝飾器參數速查

參數說明
status_code預設回應狀態碼
tags列表或 Enum，分組 API
summary簡短摘要
description覆蓋 docstring
response_description回應說明
deprecated標記為已棄用
responses額外 OpenAPI 回應
callbacks回呼 API 定義
ℹ️
Docstring 支援 Markdown。使用 \f 截斷，之後的內容不出現在 OpenAPI 文件中。

🔄 JSON Encoder 與部分更新

jsonable_encoder()

from fastapi.encoders import jsonable_encoder

# 轉換 Pydantic 模型、datetime 等為 JSON 相容 dict
json_data = jsonable_encoder(item)
# 注意：結果是 dict，不是 JSON 字串！
# 要 JSON 字串需另用 json.dumps()

部分更新 — PATCH 模式（重要！）

@app.patch("/items/{item_id}")
async def update_item(
    item_id: str, item: Item
) -> Item:
    # 1. 取出儲存的資料
    stored = Item(**items[item_id])

    # 2. 只取「有明確傳送」的欄位
    update_data = item.model_dump(exclude_unset=True)

    # 3. 複製並合併更新
    updated = stored.model_copy(update=update_data)

    # 4. 存回
    items[item_id] = jsonable_encoder(updated)
    return updated
⚠️
exclude_unset=True 是關鍵！沒有它，預設值（如 tax=None）會覆蓋已存在的資料。
PUT = 完整替換，PATCH = 部分更新。
