🚀 起步 — First Steps

建立 FastAPI 實例，用裝飾器定義路由，回傳值自動轉為 JSON。

from fastapi import FastAPI

app = FastAPI()

@app.get("/")
async def root():
    return {"message": "Hello World"}

所有 HTTP 方法裝飾器

裝飾器HTTP 方法典型用途
@app.get()GET讀取資料
@app.post()POST建立資料
@app.put()PUT完整更新
@app.patch()PATCH部分更新
@app.delete()DELETE刪除資料
@app.options()OPTIONSCORS 預檢
@app.head()HEAD僅取表頭
@app.trace()TRACE除錯迴路
💡
啟動指令：fastapi dev main.py（開發）或 fastapi run main.py（生產）。
自動文件：/docs（Swagger UI）、/redoc（ReDoc）、/openapi.json（OpenAPI Schema）。
函式可用 async def 或一般 def，FastAPI 會自動處理。

📐 參數辨識規則 — 最重要的對照表

FastAPI 依照以下規則自動判斷參數來源，此表是理解全框架的基礎：

參數特徵被解讀為範例
名稱出現在路徑模板 {param} 中路徑參數 (Path)item_id in "/items/{item_id}"
單一型別（int, str, float, bool）且不在路徑中查詢參數 (Query)q: str
型別為 Pydantic BaseModel 子類別請求主體 (Body)item: Item
有預設值（= 0、= None）非必填 (Optional)q: str = ""
沒有預設值必填 (Required)q: str
⚠️
常見陷阱：str | None 不會讓參數變成非必填 — 真正起作用的是 = None 預設值！
q: str | None → 仍然必填（缺少就報錯）
q: str | None = None → 非必填

🔗 路徑參數 — Path Parameters

在路徑字串中以 {item_id} 語法宣告。型別註解驅動驗證、解析和文件。

基本用法

@app.get("/items/{item_id}")
async def read_item(item_id: int):
    return {"item_id": item_id}

Enum 預定義值

from enum import Enum

class ModelName(str, Enum):
    alexnet = "alexnet"
    resnet = "resnet"

@app.get("/models/{model_name}")
async def get_model(model_name: ModelName):
    return {"model": model_name}

含路徑的參數

# 使用 :path 轉換器來接受含 / 的值
@app.get("/files/{file_path:path}")
async def read_file(file_path: str):
    return {"path": file_path}
⚡
順序很重要：固定路徑如 /users/me 必須宣告在參數化路徑 /users/{user_id} 之前，否則 "me" 會被當成 user_id。
→ 見第 2 張：Path() 數值驗證

🔍 查詢參數 — Query Parameters

不在路徑中的函式參數自動成為查詢參數（URL ?key=value）。

基本用法

@app.get("/items/{item_id}")
async def read_item(
    item_id: str,
    q: str | None = None,  # 非必填
    short: bool = False,     # 有預設值
):
    return {"item_id": item_id, "q": q}

bool 自動轉換

URL 值轉換結果
1、True、true、on、yesTrue
0、False、false、off、noFalse

必填查詢參數

# 沒有預設值 → 必填
@app.get("/items/{item_id}")
async def read_item(
    item_id: str,
    needy: str,  # 缺少會回 422 錯誤
):
    return {"item_id": item_id, "needy": needy}
ℹ️
路徑參數與查詢參數可自由混用，FastAPI 依名稱自動識別來源。
→ 見本張下方：Query() 字串驗證

📦 請求主體 — Request Body

使用 Pydantic BaseModel 定義。自動驗證、序列化與文件產生。

from pydantic import BaseModel

class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None

@app.post("/items/")
async def create_item(item: Item):
    return item.model_dump()

常用 Pydantic 模型方法

方法說明版本
.model_dump()轉為 dictPydantic v2
.model_dump(exclude_unset=True)僅含明確設定的欄位v2
.model_copy(update={...})複製並更新部分欄位v2
.model_json_schema()取得 JSON Schemav2

Body + Path + Query 混用

@app.put("/items/{item_id}")
async def update(
    item_id: int,       # → 路徑參數
    item: Item,          # → 請求主體（BaseModel）
    q: str | None = None, # → 查詢參數
):
    return {"item_id": item_id, **item.model_dump()}
→ 見本張下方：Body 多參數與嵌套模型

✅ 驗證 — Query / Path / Field 參數驗證

推薦使用 Annotated 語法（FastAPI ≥ 0.95.0）。Query、Path、Body、Header、Cookie 都繼承自共同的 Param 類別。

Query() 字串驗證

from typing import Annotated
from fastapi import Query

@app.get("/items/")
async def read_items(
    q: Annotated[str | None, Query(
        min_length=3,
        max_length=50,
        pattern="^fixedquery$",
        title="搜尋字串",
        description="全文搜尋查詢",
        alias="item-query",
        deprecated=True,
    )] = None,
):
    return {"q": q}

查詢參數列表

# URL: ?q=foo&q=bar → ["foo", "bar"]
q: Annotated[list[str] | None, Query()] = None
⚠️
接收列表參數必須明確使用 Query()，否則 FastAPI 會將 list 型別解讀為請求主體！

完整驗證參數對照表

參數型別適用於說明
min_lengthint字串最小長度
max_lengthint字串最大長度
patternstr字串正則表達式
gtnumber數值嚴格大於（不含等於）
genumber數值大於或等於
ltnumber數值嚴格小於
lenumber數值小於或等於
aliasstr全部請求中使用的替代名稱
titlestr全部OpenAPI 標題
descriptionstr全部OpenAPI 描述
deprecatedbool全部標記為已棄用
include_in_schemabool全部是否出現在 OpenAPI
exampleslist全部JSON Schema 範例
openapi_examplesdict全部OpenAPI 範例（含元資料）

Path() 數值驗證

from fastapi import Path

@app.get("/items/{item_id}")
async def read_items(
    item_id: Annotated[int, Path(
        title="項目 ID", ge=1, le=1000
    )],
    size: Annotated[float, Query(gt=0, lt=10.5)],
):
    return {"item_id": item_id}
ℹ️
路徑參數永遠是必填的，不論是否設定預設值。gt=0 表示嚴格 > 0（0.0 會失敗）；ge=0 則允許 0.0。

查詢參數模型 (FastAPI ≥ 0.115.0)

from typing import Literal
from pydantic import BaseModel, Field

class FilterParams(BaseModel):
    model_config = {"extra": "forbid"}  # 拒絕未知參數
    limit: int = Field(100, gt=0, le=100)
    offset: int = Field(0, ge=0)
    order_by: Literal["created_at", "updated_at"] = "created_at"

@app.get("/items/")
async def read_items(
    filter_query: Annotated[FilterParams, Query()]
):
    return filter_query
⚠️
關鍵：沒有 Query() 的 Pydantic 模型會被當成請求主體，不是查詢參數！
另外，Field 要從 pydantic 匯入，不是 fastapi — 這是最常見的匯入錯誤。

🧩 Body 進階 — 多參數 / Fields / 嵌套

多個 Body 參數

多個 Pydantic 模型 → JSON 使用參數名稱作為鍵。

from fastapi import Body

@app.put("/items/{item_id}")
async def update(
    item_id: int,
    item: Item,
    user: User,
    importance: Annotated[int, Body(gt=0)],
):
    ...
# 預期 JSON:
# {"item": {...}, "user": {...}, "importance": 5}

Body(embed=True)

強制單一 Body 模型也使用鍵名包裹：

item: Annotated[Item, Body(embed=True)]
# 預期 JSON: {"item": {...}} 而非 {...}

Field — 模型欄位驗證

from pydantic import BaseModel, Field

class Item(BaseModel):
    name: str
    description: str | None = Field(
        None,
        title="描述",
        max_length=300,
        examples=["A very nice Item"]
    )
    price: float = Field(gt=0, description="必須大於零")
💡
Field 支援與 Query/Path 相同的驗證參數（min_length、gt、ge 等），但用於模型內部的欄位定義。

🪆 嵌套模型與特殊型別

from pydantic import BaseModel, HttpUrl

class Image(BaseModel):
    url: HttpUrl
    name: str

class Item(BaseModel):
    name: str
    tags: set[str] = set()
    images: list[Image] | None = None

特殊集合型別

Python 型別行為
set[str]自動去重：["a","a"] → {"a"}
list[Image]嵌套模型列表
dict[int, float]任意鍵值 dict

純列表 Body

@app.post("/images/")
async def create_images(
    images: list[Image]
):
    return images

方法位置效果

async def update_item(
    item: Annotated[Item, Body(
        openapi_examples={
