CHAPTER 02 · 基礎入門

Hello World & 程式結構

理解 package、import、main 函式，建立符合社群慣例的專案目錄。一份會跑的 Hello World 背後其實藏著 Go 的整個哲學。

關鍵字 · package / import / main / func 新語法 · go.mod ignore (1.25+) 對照 Python · __init__.py + if __name__

最小可執行程式

Go 只有一種可執行程式的寫法：一個名為 main 的 package，裡面有一個名為 main 的無參數、無回傳值函式。

// main.go
package main

import "fmt"

func main() {
    fmt.Println("Hello, Go 1.26!")
}
  1. package main：告訴編譯器「這是一支要產出執行檔的程式」，而不是函式庫。
  2. import：引入套件，路徑是字串。多個 import 用括號包起來（見下一節）。
  3. func main()：程式進入點。不能接收參數、不能有回傳值 — 參數要靠 os.Args 讀取。
  4. 大括號的位置：{ 必須與 func 同行，這是語法強制規定，不是風格偏好。
對照 Python Python 的 if __name__ == "__main__": 是慣例；Go 的 package main + func main() 是語言規則。Go 沒有「腳本模式」，所有程式都必須編譯。

Package 機制

Go 沒有類別（class），模組化的單位是 package。同一目錄下的所有 .go 檔必須屬於同一個 package。

大小寫決定可見性

package mathx

// 首字母大寫 → exported（外部可 import）
func Double(x int) int { return x * 2 }

// 首字母小寫 → unexported（只有同 package 能用）
func helper(x int) int { return x + 1 }

var Pi = 3.14159   // 公開常數
var epsilon = 1e-9  // 內部常數
常見陷阱 Python 習慣用 _ 前綴表示「私有」，但那只是慣例；Go 的大小寫是編譯器強制的，小寫的東西外部絕對 import 不到。

import 寫法

import "fmt"  // 單行

import (              // 多行（推薦，gofmt 會自動排序）
    "fmt"
    "os"
    "strings"

    "github.com/google/uuid"       // 第三方
    "github.com/you/hello/internal/x" // 專案內
)

import (
    f "fmt"             // 取別名：f.Println(...)
    . "math"            // 直接展開（不推薦，會污染命名空間）
    _ "image/png"       // 僅為了 side effect（註冊 decoder）
)

社群慣用目錄結構

Go 沒有強制的專案結構，但社群累積出穩定的慣例。中小型專案用下列結構幾乎都合適。

myapp/
├── go.mod                  # 模組定義
├── go.sum                  # 依賴雜湊鎖定
├── README.md
├── cmd/                    # 可執行檔進入點（每子目錄 = 一支執行檔）
│   ├── myapp/
│   │   └── main.go
│   └── migrate/
│       └── main.go
├── internal/               # 私有程式碼（外部模組無法 import）
│   ├── service/
│   │   └── user.go
│   └── storage/
│       └── postgres.go
├── pkg/                    # 公開函式庫（可被外部模組 import，中小專案常省略）
│   └── validator/
├── api/                    # OpenAPI / proto 定義
├── testdata/               # 測試固件（go 不編譯此目錄）
└── scripts/                # 建置腳本

internal/

Go 編譯器保證：任何 .../internal/... 的套件，只有「祖先目錄」的程式碼可以 import。強制私有，沒得繞。

cmd/

每支 CLI 一個子目錄，子目錄名稱就是產生的執行檔名。跑 go build ./cmd/myapp 產出 myapp。

testdata/

Go 工具鏈硬編碼忽略此目錄，放任何測試用的大檔、JSON、資料庫 dump 都不會被編譯。

pkg/

爭議目錄：Google 自家程式碼不用它，但很多社群專案愛用。小專案可省略，直接把公開 API 放在模組根目錄。

go.mod 的 ignore 指令

Go 1.25 新增 ignore 區塊，讓你明確告訴 go 指令「別碰這些目錄」。適合放 schema、scripts、實驗性 code 等不該被 ./... 掃到的內容。

// go.mod
module github.com/you/myapp

go 1.26

require (
    github.com/spf13/cobra v1.8.1
)

ignore (
    ./testdata          // 原本就會被忽略，寫出來更清楚
    ./experimental      // 未完成的程式碼
    ./docs/examples     // 文件用的獨立片段
)
實用場景 你在 examples/ 目錄放了五支互相衝突的 package main demo — 沒有 ignore 的話 go build ./... 會全部嘗試編譯而失敗。加進 ignore 就能安心。

匯入路徑的心智模型

Go 的 import path 不是檔案系統路徑 — 它是「模組 path + 相對目錄」的組合。

情境import 寫法
標準函式庫"fmt", "net/http", "encoding/json"
第三方套件"github.com/google/uuid"
自家專案（go.mod 是 github.com/you/myapp）"github.com/you/myapp/internal/service"
替身測試（同 package）同目錄下所有 .go 不用 import 就能互相使用
Go 1.25+ 忽略目錄(無法 import；在 go.mod 的 ignore 區塊)
沒有相對匯入！ Python 能寫 from ..utils import foo；Go 完全沒有相對 import，必須用完整的 module path。初學者最常見的錯誤就是寫 import "./utils" — 編譯器會直接拒絕。

完整範例：跨 package 呼叫

一個有主程式、一個 exported 函式、一個 unexported 函式的最小多檔案專案。

FILE · go.mod 
module github.com/you/greeter

go 1.26
FILE · cmd/greeter/main.go 
package main

import (
    "fmt"
    "os"

    "github.com/you/greeter/internal/greet"
)

func main() {
    name := "World"
    if len(os.Args) > 1 {
        name = os.Args[1]
    }
    fmt.Println(greet.Hello(name))
}
FILE · internal/greet/greet.go 
package greet

import "strings"

// Hello 是公開函式（首字母大寫）
func Hello(name string) string {
    return "Hello, " + polish(name) + "!"
}

// polish 是內部輔助函式（首字母小寫）
func polish(s string) string {
    return strings.TrimSpace(strings.Title(s))
}
# 跑起來
go run ./cmd/greeter
# → Hello, World!

go run ./cmd/greeter gopher
# → Hello, Gopher!

本章速記

  1. package main + func main()：可執行檔的固定配方，缺一不可。
  2. 大寫即公開：不用 public/private 關鍵字，看首字母就知道可見性。
  3. 同目錄同 package：Go 強制規則，無法在同目錄放兩個不同 package。
  4. 沒有相對匯入：一律寫完整模組路徑，別想用 ./ 或 ..。
  5. internal/ 是真私有：編譯器層級保證，不只是慣例。
  6. cmd/ 放入口點：一支執行檔對應一個子目錄。
  7. go.mod 的 ignore（1.25+）：處理 examples、scripts、實驗碼的好工具。
  8. import 分三組：標準庫、第三方、專案內，各自空行隔開，gofmt 會處理排序。
