CHAPTER 14 · 並發與進階

Reflection & unsafe

Go 提供兩個「逃生出口」：reflect 在執行期檢視並操作型別，unsafe 直接碰記憶體。兩者都違背靜態型別的精神 — 功能強大但用得不對會痛。

兩個套件 · reflect · unsafe 口號 · Clear is better than clever

為什麼需要反射？

reflect 套件讓程式在執行期能夠：① 檢視任意值的型別、② 取得 struct 的 field 與 tag、③ 呼叫方法、④ 建立/修改任意型別的值。

序列化／反序列化

encoding/json、yaml、xml 都靠 reflect 讀取 struct tag。

ORM 與 validator

GORM、go-validator 等需要在執行期根據 tag 做事。

深度相等比較

reflect.DeepEqual 處理任意結構的比較。

動態調用框架

某些 RPC / 依賴注入框架需要反射來動態路由。

Rob Pike 的三條 reflect 定律 ① 反射會讓程式變慢（比直接 code 慢 10-100 倍）。
② 反射錯誤通常在執行期才爆 — 失去靜態型別的保護。
③ Clear is better than clever — 能不用就不用，尤其是泛型時代之後。

reflect 的三大支柱

import "reflect"

var x float64 = 3.14

// ① reflect.Type — 描述「x 的型別是什麼」
t := reflect.TypeOf(x)
fmt.Println(t)          // float64
fmt.Println(t.Kind())   // float64（reflect.Float64）

// ② reflect.Value — 描述「x 的值是什麼」
v := reflect.ValueOf(x)
fmt.Println(v)          // 3.14
fmt.Println(v.Float())  // 3.14（取出具體值）

// ③ 互相轉換
i := v.Interface()      // 轉回 any
back := i.(float64)      // 再 assert 回來

Type vs Kind

type UserID int64

var id UserID = 42
t := reflect.TypeOf(id)
fmt.Println(t)           // main.UserID  ← 具名型別
fmt.Println(t.Kind())    // int64        ← 底層 kind

// 常見的 Kind
// Invalid、Bool、Int、Int8..Int64、Uint、Uint8..Uint64
// Float32/64、Complex64/128、String、Array、Slice
// Map、Chan、Func、Interface、Ptr、Struct、UnsafePointer

用 reflect 修改值

要透過 reflect 修改值，必須傳指標進去 — 因為值型別只是副本。這是初學者最容易卡住的地方。

var x float64 = 3.14

// ❌ 錯誤：傳 x 進去只是副本
v := reflect.ValueOf(x)
// v.SetFloat(2.71)  ← panic: using unaddressable value

// ✅ 正確：傳 &x，再 Elem() 取指向的值
v := reflect.ValueOf(&x).Elem()
if v.CanSet() {
    v.SetFloat(2.71)
}
fmt.Println(x)    // 2.71

操作 struct 欄位

type User struct {
    Name string `json:"name"`
    Age  int    `json:"age"`
}

u := User{Name: "Alice", Age: 30}
v := reflect.ValueOf(&u).Elem()
t := v.Type()

// 走訪所有欄位
for i := range t.NumField() {
    field := t.Field(i)            // reflect.StructField（型別資訊）
    value := v.Field(i)            // reflect.Value（值）

    jsonTag := field.Tag.Get("json")
    fmt.Printf("%s (%s) = %v [json=%s]\n",
        field.Name, field.Type, value, jsonTag)
}

// 修改特定欄位
v.FieldByName("Age").SetInt(31)
fmt.Println(u.Age)    // 31

用 reflect 呼叫方法

type Greeter struct{}

func (g Greeter) Hello(name string) string {
    return "Hello, " + name
}

g := Greeter{}
v := reflect.ValueOf(g)

// 透過名稱找方法
method := v.MethodByName("Hello")

// 參數需要包裝成 []reflect.Value
args := []reflect.Value{reflect.ValueOf("World")}

// Call 回傳 []reflect.Value
results := method.Call(args)
fmt.Println(results[0].String())   // "Hello, World"
泛型能做的請別用反射 在 Go 1.18 之前，很多容器／工具函式需要 reflect。現在能用泛型的絕對不要用反射 — 型別安全又快得多。反射只剩下「真的不知道型別」的場景（解析 JSON 成 map[string]any、ORM 映射 DB row…）。

reflect.DeepEqual · 最實用的一個

這大概是 90% 的人唯一會主動呼叫的 reflect 函式 — 用來比較任意結構是否「內容相等」。

import "reflect"

a := []int{1, 2, 3}
b := []int{1, 2, 3}

// a == b  ❌ 編譯錯：slice 不支援 ==
reflect.DeepEqual(a, b)    // true

// 複雜結構也能比
u1 := User{Name: "Alice", Tags: []string{"go"}}
u2 := User{Name: "Alice", Tags: []string{"go"}}
reflect.DeepEqual(u1, u2)  // true
效能與正確性 DeepEqual 比 slices.Equal 慢很多；純 slice/map 的比較優先用 slices.Equal / maps.Equal（泛型，Go 1.21+）。
DeepEqual 的語意也有陷阱：nil slice 與空 slice 不相等、不同型別的零值也不相等。測試斷言時更推薦 go-cmp。

unsafe · 關掉型別系統

一般應用永遠不該用 unsafe — 它主要給標準庫、底層基礎設施、以及一些極致效能場景使用。

import "unsafe"

// Sizeof：取得型別的大小（bytes）
unsafe.Sizeof(int64(0))         // 8
unsafe.Sizeof("")               // 16（指標 + 長度）
unsafe.Sizeof([]int{})            // 24（指標 + 長度 + 容量）

// Alignof、Offsetof
type S struct {
    A int32
    B int64
}
unsafe.Alignof(S{})            // 對齊要求
unsafe.Offsetof(S{}.B)         // B 在 struct 中的 offset

// unsafe.Pointer：萬能指標（可在任意指標型別之間轉換）
var x int32 = 42
p := unsafe.Pointer(&x)       // 把 *int32 轉成 unsafe.Pointer
y := (*float32)(p)            // 再 cast 成 *float32
fmt.Println(*y)                 // 以 float32 的方式解讀那塊記憶體

string ↔ []byte 的零拷貝轉換（Go 1.20+）

// 「避免分配」的經典場景
// Go 1.20+ 提供 unsafe.String / unsafe.StringData / unsafe.SliceData

// []byte → string（零拷貝，但結果不能再修改那 []byte！）
b := []byte{'h', 'i'}
s := unsafe.String(&b[0], len(b))   // "hi"

// string → []byte（同樣零拷貝，極度謹慎）
str := "hello"
data := unsafe.Slice(unsafe.StringData(str), len(str))
// data 現在指向 str 的底層 byte，但「絕對不能寫入」 — 字串是不可變的
unsafe 的鐵律 ① 任何使用 unsafe 的程式在未來 Go 版本可能失效。
② GC 不追蹤 unsafe.Pointer 轉換出的指標 — 容易產生懸垂參考。
③ 跨平台可能踩雷（endianness、alignment）。
④ Go 1.14 起 go vet 會對不安全的 unsafe 用法發出警告。
⑤ 如果你看到自己在寫 unsafe，先問：「我真的不能用一般方法嗎？」

實戰模板

TEMPLATE · 使用 reflect 實作簡易 validator 
package validate

import (
    "fmt"
    "reflect"
    "strconv"
    "strings"
)

// 讀取 struct tag 做欄位驗證
func Validate(v any) error {
    rv := reflect.ValueOf(v)
    if rv.Kind() == reflect.Ptr {
        rv = rv.Elem()
    }
    if rv.Kind() != reflect.Struct {
        return fmt.Errorf("expected struct, got %s", rv.Kind())
    }

    rt := rv.Type()
    for i := range rt.NumField() {
        field := rt.Field(i)
        rule := field.Tag.Get("validate")
        if rule == "" {
            continue
        }
        if err := checkRule(rv.Field(i), field.Name, rule); err != nil {
            return err
        }
    }
    return nil
}

func checkRule(v reflect.Value, name, rule string) error {
    for _, r := range strings.Split(rule, ",") {
        switch {
        case r == "required":
            if v.IsZero() {
                return fmt.Errorf("%s: required", name)
            }
        case strings.HasPrefix(r, "min="):
            n, _ := strconv.Atoi(strings.TrimPrefix(r, "min="))
            if v.Kind() == reflect.String && v.Len() < n {
                return fmt.Errorf("%s: min length %d", name, n)
            }
        }
    }
    return nil
}

// 使用
type User struct {
    Name  string `validate:"required,min=2"`
    Email string `validate:"required"`
}

err := Validate(&User{Name: "a"})
// → Name: min length 2

本章速記

  1. reflect 三寶：TypeOf、ValueOf、Interface。
  2. Type 是具名型別，Kind 是底層種類：UserID vs int64。
  3. 要修改值必傳指標：reflect.ValueOf(&x).Elem()。
  4. CanSet 檢查可寫：unaddressable 或未匯出欄位不能改。
  5. Tag 用 field.Tag.Get("json") 讀取。
  6. 反射慢 10-100 倍：熱路徑別用。
  7. 泛型能做的就不要用反射：型別安全優先。
  8. DeepEqual 是最常用的 reflect API。
  9. 測試用 go-cmp：比 DeepEqual 更精準、訊息更友善。
  10. unsafe 是最後手段：想用前先問自己兩次。
  11. unsafe.Sizeof / Alignof / Offsetof 是編譯期常數，沒副作用，可放心用。
  12. Go 1.20+ 的 unsafe.String/Slice：零拷貝字串轉換的現代寫法。
