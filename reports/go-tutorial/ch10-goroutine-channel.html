CHAPTER 10 · 並發與進階

Goroutine & Channel

Go 的招牌。一行 go 就啟動輕量執行緒、用 channel 傳遞資料取代鎖。Rob Pike 的名言：「Don't communicate by sharing memory; share memory by communicating.」

核心關鍵字 · go · chan · select 對照 Python · asyncio / threading

Goroutine · 極輕量的執行緒

Goroutine 不是 OS thread — 它是 Go runtime 排程的「協程」，初始只佔 2KB 堆疊，啟動一百萬個都不會爆。Go 1.25 起 GOMAXPROCS 會自動感知 container 的 CPU quota，雲原生環境下再也不用手動調。

// 把 go 放在函式呼叫前，就這樣
go doSomething()

// 匿名函式也能 go
go func() {
    fmt.Println("hello from goroutine")
}()

// 帶參數的匿名 goroutine（避免閉包陷阱的保險寫法）
for i := range 10 {
    go func(n int) {
        fmt.Println(n)
    }(i)
}
main 結束，全部 goroutine 死 main 函式返回時整個程式退出，其他 goroutine 不管跑到哪裡都會被砍。要讓 main 等它們，請用 sync.WaitGroup、chan 或 context（下一章介紹）。
對照 Python Python 的 threading 受 GIL 限制，真並行要用 multiprocessing；asyncio 則需要整套 async/await 生態。Go 的 goroutine 預設就是並發又並行，程式碼長得和同步版本幾乎一樣。

Channel · 帶型別的管道

Channel 是 goroutine 之間傳值的管道。送值用 ch <- v，收值用 v := <-ch。收送雙方會互相等待，這就是同步。

// 建立 channel（一定要用 make）
ch := make(chan int)           // 無緩衝
bch := make(chan int, 10)       // 緩衝 10

// 送值（若無緩衝，會阻塞等收者）
ch <- 42

// 收值（若空，會阻塞等送者）
v := <-ch

// 安全關閉（通常是送者關閉）
close(ch)

// 收值帶 ok：檢查是否已關閉且排空
v, ok := <-ch
if !ok {
    fmt.Println("channel 已關閉")
}

// range 自動走訪到 channel 關閉為止
for v := range ch {
    fmt.Println(v)
}

無緩衝 vs 有緩衝

項目無緩衝 chan T有緩衝 chan T, N
送方行為阻塞到收方接收未滿：立即放入；已滿：阻塞
收方行為阻塞到送方發送非空：立即取出；為空：阻塞
定位同步點（handshake）佇列（throughput buffer）
常見用途「我到了」的信號生產者消費者、限流

Channel 方向型別

可以在函式簽名中限定 channel 的用法 — 只送或只收。這讓 API 的意圖一目了然，編譯器還會替你檢查違規。

// chan<- int 代表「只能送」
// <-chan int 代表「只能收」

func producer(out chan<- int) {
    defer close(out)
    for i := range 5 {
        out <- i
    }
}

func consumer(in <-chan int) {
    for v := range in {
        fmt.Println(v)
    }
}

func main() {
    ch := make(chan int)
    go producer(ch)   // 雙向 chan 可自動轉成單向
    consumer(ch)
}

select · 多路多工

select 同時監聽多個 channel，哪個先就緒就執行哪個 case。類似 Unix 的 select/poll，是寫並發程式的瑞士刀。

select {
case v := <-ch1:
    fmt.Println("來自 ch1:", v)
case v := <-ch2:
    fmt.Println("來自 ch2:", v)
case ch3 <- 99:
    fmt.Println("送出到 ch3")
case <-time.After(3 * time.Second):
    fmt.Println("timeout")
default:
    fmt.Println("全部都沒準備好，立即離開（非阻塞）")
}
select 的四個關鍵性質 ① 若多個 case 同時就緒，隨機選一個執行（避免飢餓）。
② 有 default 就變成非阻塞。
③ 可以只寫 select {}（空的、永遠阻塞）— 常見於 main 結尾等候信號。
④ case 裡的表達式在 select 觸發前會被求值一次。

timeout 模式

func fetchWithTimeout() (Data, error) {
    result := make(chan Data, 1)
    go func() {
        result <- slowFetch()
    }()

    select {
    case d := <-result:
        return d, nil
    case <-time.After(2 * time.Second):
        return Data{}, errors.New("timeout")
    }
}

四大經典模式

① Worker Pool（工作池）

func worker(id int, jobs <-chan int, results chan<- int) {
    for j := range jobs {
        results <- j * 2
    }
}

func main() {
    jobs := make(chan int, 100)
    results := make(chan int, 100)

    // 啟動 3 個 worker
    for w := range 3 {
        go worker(w, jobs, results)
    }

    // 送工作
    for j := range 9 {
        jobs <- j
    }
    close(jobs)

    // 收結果
    for range 9 {
        fmt.Println(<-results)
    }
}

② Fan-out / Fan-in

// Fan-out：一個 channel 被多個 goroutine 讀
// Fan-in：多個 channel 的資料合併到一個 channel

func fanIn(chs ...<-chan int) <-chan int {
    out := make(chan int)
    var wg sync.WaitGroup

    for _, ch := range chs {
        wg.Add(1)
        go func(c <-chan int) {
            defer wg.Done()
            for v := range c {
                out <- v
            }
        }(ch)
    }

    go func() {
        wg.Wait()
        close(out)
    }()

    return out
}

③ Done channel（取消信號）

// 用關閉 channel 來廣播「停止」信號
done := make(chan struct{})

go func() {
    for {
        select {
        case <-done:
            return       // 收到信號就退出
        default:
            doWork()
        }
    }
}()

// 稍後...
close(done)          // 一次通知所有聽眾

④ Pipeline（串接處理）

func gen(nums ...int) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for _, n := range nums {
            out <- n
        }
    }()
    return out
}

func square(in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for n := range in {
            out <- n * n
        }
    }()
    return out
}

// 串接：gen → square → 最終 range
for v := range square(gen(1, 2, 3, 4)) {
    fmt.Println(v)   // 1, 4, 9, 16
}

四大陷阱（必記）

Goroutine Leak

goroutine 永遠在等 channel，但沒人來送也沒人來收 — 就活該一直佔記憶體。每個 goroutine 都要有「保證退出」的路徑，通常靠 context。

關閉已關閉的 channel

二度 close() 會 panic。
原則：只有送者關 channel，且只關一次。多個送者的話，加一層 coordinator 來負責關。

向已關閉的 channel 送值

同樣 panic。所以：關閉後絕對不能再送。受 context.Done() 保護時要 select 檢查。

nil channel 永久阻塞

讀寫 nil channel 都不會 panic，但會永遠阻塞。這個特性有時可利用：在 select 中把已用完的 channel 設為 nil 就能「關閉該 case」。

偵測 leak 的好朋友 跑 go test -race 開啟 race detector；上線用 net/http/pprof 的 /debug/pprof/goroutine 端點看 goroutine 數量是否持續增長。

實戰模板

TEMPLATE · 並行抓取多個 URL 
package main

import (
    "fmt"
    "io"
    "net/http"
    "sync"
)

type result struct {
    url  string
    size int
    err  error
}

func fetch(url string) result {
    resp, err := http.Get(url)
    if err != nil {
        return result{url: url, err: err}
    }
    defer resp.Body.Close()
    data, err := io.ReadAll(resp.Body)
    return result{url: url, size: len(data), err: err}
}

func main() {
    urls := []string{
        "https://go.dev",
        "https://pkg.go.dev",
        "https://golang.org",
    }

    results := make(chan result, len(urls))
    var wg sync.WaitGroup

    for _, u := range urls {
        wg.Add(1)
        go func(u string) {
            defer wg.Done()
            results <- fetch(u)
        }(u)
    }

    // 另一個 goroutine 等待全部完成後關閉 channel
    go func() {
        wg.Wait()
        close(results)
    }()

    for r := range results {
        if r.err != nil {
            fmt.Printf("%s error: %v\n", r.url, r.err)
        } else {
            fmt.Printf("%s size=%d\n", r.url, r.size)
        }
    }
}

本章速記

  1. go f() 就是 goroutine：別期待它回值，要拿結果請走 channel。
  2. main 結束全員撤退：用 WaitGroup / channel / context 等候。
  3. channel 必 make：nil channel 會永久阻塞（但可利用）。
  4. 無緩衝 = 同步點，有緩衝 = 佇列。
  5. 送方關閉 channel：多送方要加 coordinator。
  6. range chan 會讀到 channel 關閉為止。
  7. v, ok := <-ch：ok 為 false 表示已關閉且排空。
  8. select 隨機挑就緒 case：有 default 就非阻塞。
  9. chan<- T 是只送，<-chan T 是只收：API 設計清楚。
  10. 四大模式：Worker Pool、Fan-out/in、Done channel、Pipeline。
  11. struct{} 是零大小型別：chan struct{} 是語意最清晰的「信號 channel」。
  12. go test -race：抓資料競爭的神器，CI 一定要開。
