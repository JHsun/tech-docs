複合型別 Array / Slice / Map / Struct

Go 的四種資料結構看起來像 Python 的 list / dict / class — 但底層機制（值 vs 引用、容量、零值）差異巨大，是新手最常踩坑的區塊。Go 1.26 讓 new 的用法變得更簡潔。

Array · 固定長度陣列

Go 的 array 長度是型別的一部分：[3]int 和 [4]int 是完全不同的型別，不能互相賦值。實務上 array 用得少，99% 的情境都用 slice。

// 宣告與初始化
var a [3]int                  // [0 0 0]（零值）
b := [3]int{1, 2, 3}
c := [...]int{1, 2, 3, 4}        // ... 讓編譯器推論長度 = 4
d := [5]int{2: 99}           // [0 0 99 0 0]（索引初始化）

// 長度屬於型別
var x [3]int
var y [4]int
// x = y  ❌ 編譯錯誤：type mismatch

// array 是值型別，賦值會完整複製
p := [3]int{1, 2, 3}
q := p                        // q 是 p 的副本
q[0] = 99                     // p 仍然是 {1,2,3}
什麼時候真的要用 array？ 幾乎只在兩個場景：① 對齊底層硬體（例如雜湊摘要 [32]byte），② 想把長度當作型別約束的一部分（例如矩陣固定維度）。一般程式碼都用 slice。

Slice · 動態陣列（Go 的主力）

Slice 是「指向底層 array 的視窗」— 包含三個資訊：指標、長度（len）、容量（cap）。理解這三者關係是掌握 Go 的關鍵。

建立 slice

// 字面量
s := []int{1, 2, 3}            // len=3, cap=3

// make(type, len, cap) — 建立指定長度的 slice
s := make([]int, 5)             // len=5, cap=5, 全 0
s := make([]int, 5, 10)         // len=5, cap=10（預留空間）

// 從 array 或另一個 slice 切片
arr := [5]int{1, 2, 3, 4, 5}
s := arr[1:4]                // [2 3 4]，len=3, cap=4

// nil slice（零值）
var s []int                   // s == nil, len=0, cap=0
// 可以對 nil slice 做 append、len、range — 完全合法！

append 與容量擴張

s := []int{1, 2, 3}         // len=3, cap=3
s = append(s, 4)             // len=4, cap=6（通常 cap 會翻倍）
s = append(s, 5, 6, 7)       // 一次加多個

// 合併兩個 slice（用 ... 展開）
a := []int{1, 2}
b := []int{3, 4}
c := append(a, b...)           // [1 2 3 4]

共享底層陣列的陷阱

// ⚠️ slice 是引用 — 切片操作會共享底層陣列
orig := []int{1, 2, 3, 4, 5}
sub := orig[1:4]             // [2 3 4]
sub[0] = 99
// orig 現在是 [1 99 3 4 5]！

// 要真正複製，用 slices.Clone（Go 1.21+）或 copy()
import "slices"
safe := slices.Clone(orig)       // 深拷貝

// 或傳統寫法
safe := make([]int, len(orig))
copy(safe, orig)
append 的隱藏陷阱 append 若 cap 不夠會分配新的底層陣列；若夠則直接寫入原陣列。這代表 append 可能改變其他共享此底層陣列的 slice — 除錯夢魘。安全做法：拿到 slice 後立刻 slices.Clone。

slices 套件常用函式（Go 1.21+）

函式用途

slices.Contains(s, v)是否包含某值
slices.Index(s, v)找索引，沒找到回 -1
slices.Sort(s)就地排序（限 cmp.Ordered）

slices.Reverse(s)就地反轉

slices.Concat(a, b, c)合併多個 slice（1.22+）

Map · 雜湊表

Map 的語法和 Python dict 相近，但有幾個硬規則：① key 必須可比較（ == 可用），② nil map 不能寫入，③ 走訪順序每次都隨機。

// 建立
var m1 map[string]int          // nil map，只能讀不能寫！
m2 := make(map[string]int)      // 空 map，可讀可寫
m3 := map[string]int{
    "a": 1,
    "b": 2,                      // 最後一個逗號必寫
}

// 讀／寫
m2["x"] = 10
v := m2["x"]                    // 10

// 取不存在的 key：得到零值
z := m2["nope"]                 // 0（int 的零值）— 不會 panic！

// 要區分「零值」和「不存在」，用雙回傳值
v, ok := m2["x"]
if !ok {
    fmt.Println("key 不存在")
}

// 刪除
delete(m2, "x")               // 刪不存在的 key 不會報錯

// 長度
len(m2)

// 走訪（順序隨機！）
for k, v := range m2 {
    fmt.Println(k, v)
}
三大陷阱 ① var m map[string]int 寫入會 panic — 必須 make 或字面量初始化。
② map 不是並發安全的 — 多 goroutine 寫入要用 sync.Map 或 mutex。
③ 不能對 map 的欄位直接取址：&m[k] 編譯錯誤（因為擴容時位址會變）。

Struct · 結構體

Go 的 struct 只有資料、沒有方法（方法是分開定義的，見下一章）。它同時扮演 Python 的 dataclass、namedtuple、甚至部分 class 的角色。

// 定義
type User struct {
    ID       int
    Name     string
    Email    string
    Active   bool
}

// 建立（三種寫法）
var u1 User                     // 零值：{0 "" "" false}

u2 := User{1, "Alice", "a@x", true}  // 位置式（脆弱，不推薦）

u3 := User{                     // 具名式（推薦）
    ID:     1,
    Name:   "Alice",
    Email:  "a@x",
    Active: true,                // 最後逗號必寫
}

// 指標形式（最常用）
u4 := &User{ID: 1, Name: "Bob"}
// u4 是 *User，欄位存取相同：u4.Name（Go 自動解引用）

// 欄位存取
u3.Name = "Alice Chen"

// 比較（所有欄位都可比較時才能用 ==）
if u1 == u2 { ... }

struct tag · 序列化的魔法

type User struct {
    ID    int    `json:"id"`
    Name  string `json:"name" validate:"required,min=2"`
    Email string `json:"email,omitempty"`
    pwd   string `json:"-"`      // 不序列化
}

// tag 是純字串，套件透過 reflect 讀取它

匿名欄位（嵌入，Go 的「繼承」）

type Animal struct {
    Name string
    Age  int
}

type Dog struct {
    Animal          // 匿名嵌入（不是繼承，是組合）
    Breed string
}

d := Dog{
    Animal: Animal{Name: "Rex", Age: 3},
    Breed:  "Labrador",
}
fmt.Println(d.Name)     // "Rex"（提升欄位，自動穿透 Animal）
fmt.Println(d.Animal.Name) // 同樣可用完整路徑
對照 Python Python 的 @dataclass ≈ Go 的 struct；繼承（class Dog(Animal)）在 Go 只能用嵌入。Go 沒有方法覆寫（override），組合優於繼承是硬規則。

make vs new · 兩個容易混淆的內建函式

Go 1.26 擴充了 new 的能力，讓它變得更有用。先把兩者的差異記住：

比較makenew
用途初始化 slice / map / channel配置任意型別並回傳指標
回傳型別本身（T）指向零值的指標（*T）
範例make([]int, 10)new(int) → *int 指向 0
 
// make — 只能用於 slice/map/channel
s := make([]int, 5)       // s 是 []int
m := make(map[string]int) // m 是 map
c := make(chan int, 10)   // c 是 chan

// new — 任何型別，回傳指標
p := new(int)            // p 是 *int，*p == 0
u := new(User)           // u 是 *User，所有欄位為零值

Go 1.26：new 可接受表達式

// ✅ Go 1.26+：new 的參數可以是「表達式」，直接產生指向該值的指標
type Person struct {
    Name string
    Age  *int       // 用指標表示 optional 欄位
}

// 舊寫法（囉唆）
age := 30
p := Person{Name: "Alice", Age: &age}

// Go 1.26 新寫法 — 極度簡潔
p := Person{Name: "Alice", Age: new(30)}

// 對 JSON 的 optional 欄位特別好用
json.Marshal(Person{
    Name: "Bob",
    Age:  new(25),      // 取代 func intPtr(n int) *int { return &n }
})
實用情境 在處理 JSON、Protocol Buffers 或任何用 pointer 表示「可選欄位」的 API 時，new(expression) 省下大量 helper 函式，是 Go 1.26 最貼近日常的改進。

實戰模板

TEMPLATE · 常見集合操作（Go 1.21+ slices 套件） 
import "slices"

// 過濾（Python: [x for x in nums if x > 0]）
func filter[T any](s []T, keep func(T) bool) []T {
    out := make([]T, 0, len(s))
    for _, v := range s {
        if keep(v) {
            out = append(out, v)
        }
    }
    return out
}

// 映射（Python: [f(x) for x in nums]）
func mapFn[T, U any](s []T, fn func(T) U) []U {
    out := make([]U, len(s))
    for i, v := range s {
        out[i] = fn(v)
    }
    return out
}

// 去重
func unique[T comparable](s []T) []T {
    seen := make(map[T]struct{})
    out := make([]T, 0, len(s))
    for _, v := range s {
        if _, ok := seen[v]; !ok {
            seen[v] = struct{}{}
            out = append(out, v)
        }
    }
    return out
}

// 分組（Python: itertools.groupby 或 collections.defaultdict）
func groupBy[T any, K comparable](s []T, key func(T) K) map[K][]T {
    out := make(map[K][]T)
    for _, v := range s {
        k := key(v)
        out[k] = append(out[k], v)
    }
    return out
}

本章速記

  1. Array 長度是型別一部分：實務上很少用，幾乎都用 slice。
  2. Slice = 指標 + len + cap：切片操作共享底層陣列，修改會互相影響。
  3. 要獨立副本用 slices.Clone：或 make + copy。
  4. nil slice 可用：range、len、append 都安全 — 相當於空 slice。
  5. nil map 不可寫：必須 make 或字面量；但可以讀、range、len。
  6. map[key] 回傳零值而非錯誤：要區分「不存在」請用 v, ok := m[k]。
  7. map 走訪順序隨機：需要穩定順序請自己收集 keys 後排序。
  8. struct 用具名初始化：永遠寫 User{Name: "x"} 而非位置式。
  9. struct tag 驅動序列化：`json:"name,omitempty"` 是 Go 生態標配。
  10. 嵌入不是繼承：Go 用組合提供「方法提升」— 後面章節會詳談。
  11. make 給 slice/map/chan，new 給指標。
  12. Go 1.26 的 new(expr) 簡化「想要一個指向某值的指標」的場景。
