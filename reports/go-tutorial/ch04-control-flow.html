CHAPTER 04 · 基礎入門

運算子 & 控制流程

Go 把流程控制削減到最簡：只有一個 for、一個 switch、一個 if，但每個都功能強大。Go 1.22 後 for i := range 10 更讓迴圈達到 Python 的簡潔。

唯一迴圈 · for 無三元運算子 · 一律用 if/else 新語法 · for i := range 10 (1.22+)

運算子總覽

Go 的運算子接近 C，但有幾個明顯差異：沒有三元運算子、沒有 ++i（只有 i++ 且是語句不是表達式）、邏輯比較嚴格到 bool 就是 bool。

類別運算子說明
算術+ - * / %整數除法是整除，餘數用 %
比較== != < <= > >=回傳 bool
邏輯&& || !只能用於 bool，有短路求值
位元& | ^ << >> &^^ 是 XOR，&^ 是「AND NOT」
賦值= := += -= *= /= %= &= |= ^= <<= >>=復合賦值齊全
自增／自減i++ i--只能當語句，不能當表達式
取址／解址&x *p詳見指標章節
通道<-送值與收值，詳見並發章節
三個要特別記的陷阱 ① i++ 不能寫在 x = i++。
② ^ 當一元運算子是「位元 NOT」（其他語言用 ~）。
③ if i = 0 { ... } 編譯失敗 — Go 的 if 條件必須是 bool，不接受賦值語句。

if / else

Go 的 if 最大特色：可在條件前寫一個「init 語句」，讓變數只存在於 if/else 區塊中。這是 Go 處理錯誤的標準手法。

// 基本形式（不用括號，但大括號必寫）
if x > 0 {
    fmt.Println("正")
} else if x == 0 {
    fmt.Println("零")
} else {
    fmt.Println("負")
}

// init 語句（最常見的 Go 慣用法！）
if err := doSomething(); err != nil {
    return err        // err 只存在於這個 if 區塊
}
// 這裡已經看不到 err 變數

// init 也能搭配 else
if v, ok := m["key"]; ok {
    fmt.Println("找到", v)
} else {
    fmt.Println("沒找到")
}
沒有三元運算子 Python 的 a if cond else b 在 Go 沒有對應。多寫一行 if/else 是 Go 刻意的語言哲學 — 可讀性 > 簡潔。想縮短就寫成函式：func ifElse[T any](c bool, a, b T) T { if c { return a }; return b }。

for · 唯一的迴圈

Go 沒有 while、沒有 do-while、沒有 foreach — 只有一個 for，但透過不同形式覆蓋所有情境。

五種 for 寫法

// 1) 經典三段式
for i := 0; i < 10; i++ {
    fmt.Println(i)
}

// 2) while 形式（省略 init 和 post）
i := 0
for i < 10 {
    i++
}

// 3) 無限迴圈
for {
    if done() { break }
}

// 4) range（走訪）— 最常用
for i, v := range slice { ... }      // index, value
for k, v := range myMap { ... }      // key, value
for i, r := range "hello" { ... }    // index, rune
for v := range ch { ... }            // channel 收到的值

// 5) range over int（Go 1.22+，取代 for i := 0; i < N; i++）
for i := range 10 {                // i 從 0 到 9
    fmt.Println(i)
}

for range 5 {                     // 只想跑 5 次，不要索引
    doSomething()
}

range over function（Go 1.23+）

// 函式也能當作 range 的來源（自訂迭代器）
func countDown(yield func(int) bool) {
    for i := 10; i > 0; i-- {
        if !yield(i) { return }   // yield 回傳 false 代表 break
    }
}

func main() {
    for n := range countDown {
        fmt.Println(n)   // 10, 9, 8 ... 1
    }
}
Range 只取一部份 for i := range slice（只一個變數）拿到的是 index。
只要 value 不要 index：for _, v := range slice。

range 的常見陷阱（必讀）

range 有幾個反直覺的行為，即使是有經驗的開發者也會踩坑。Go 1.22 修掉了最嚴重的一個，但其他仍存在。

陷阱 1：v 是副本，不是引用

nums := []int{1, 2, 3}
for _, v := range nums {
    v *= 2             // 改的是副本，不會動到原 slice
}
// nums 依然是 {1, 2, 3}

// 要修改原 slice 必須透過 index
for i := range nums {
    nums[i] *= 2
}

陷阱 2：迴圈變數作用域（Go 1.22 修正）

// Go 1.21 以前：這段 goroutine 會印出三個「3」或其他怪結果
// Go 1.22+ ：每圈有獨立的 i，正確印出 0, 1, 2（順序不一定）
for i := 0; i < 3; i++ {
    go func() {
        fmt.Println(i)
    }()
}

// 如果 go.mod 寫 go 1.22 以上，就享有這個修正
請確認 go.mod 版本 這個修正是由 go.mod 的 go 指令控制的 — 就算用 Go 1.26 編譯，若 go.mod 寫 go 1.21，還是會保留舊的陷阱行為以維持相容性。

switch · 強化版 if

Go 的 switch 比 C 家族好用得多：預設不 fallthrough、case 可以是任意表達式、還能當作 if/else 鏈的替代品。

// 基本形式（不寫 break，預設不會 fallthrough）
switch day {
case "Mon", "Tue", "Wed":    // 多個值用逗號
    fmt.Println("平日")
case "Sat", "Sun":
    fmt.Println("假日")
default:
    fmt.Println("不知道")
}

// 無條件 switch（最常見的 if/else 替代品）
switch {
case score >= 90:
    grade = "A"
case score >= 80:
    grade = "B"
case score >= 70:
    grade = "C"
default:
    grade = "F"
}

// 帶 init（和 if 一樣）
switch name := getName(); name {
case "Alice": ...
case "Bob":   ...
}

// 要 fallthrough 必須明寫
switch n {
case 1:
    fmt.Println("一")
    fallthrough
case 2:
    fmt.Println("二")  // n==1 時也會印這行
}

type switch（型別判斷）

func describe(i any) string {
    switch v := i.(type) {       // 注意 i.(type) 這個特殊語法
    case int:
        return fmt.Sprintf("整數 %d", v)
    case string:
        return fmt.Sprintf("字串 %q", v)
    case bool:
        return fmt.Sprintf("布林 %t", v)
    case nil:
        return "nil"
    default:
        return fmt.Sprintf("未知 %T", v)
    }
}

break / continue / goto / label

Go 支援 labeled statement，這在巢狀迴圈要「跳出多層」時特別有用。

// 一般 break / continue
for i := range 10 {
    if i == 3 { continue }  // 跳過這圈
    if i == 7 { break }     // 離開迴圈
    fmt.Println(i)
}

// labeled break：跳出多層
outer:
for i := range 5 {
    for j := range 5 {
        if i*j > 6 {
            break outer          // 一次跳出兩層
        }
    }
}

// continue 也能用 label
outer2:
for _, row := range grid {
    for _, cell := range row {
        if cell.skip {
            continue outer2    // 跳到外層的下一次迭代
        }
    }
}

// goto（極少用，通常只出現在編譯器產生的程式碼）
if err != nil {
    goto cleanup
}
// ...
cleanup:
    cleanupResources()
goto 有嚴格限制 Go 的 goto 不能跳過變數宣告、不能跳進另一個區塊、不能跳出函式。社群規範幾乎禁止使用，唯一合理場景是「統一的 cleanup」，而這通常 defer 就解決了。

實戰模板

TEMPLATE · 錯誤處理的標準流 
func loadUser(id string) (*User, error) {
    if id == "" {
        return nil, errors.New("empty id")
    }

    data, err := db.Get(id)
    if err != nil {
        return nil, fmt.Errorf("db.Get: %w", err)
    }

    user, err := parse(data)
    if err != nil {
        return nil, fmt.Errorf("parse: %w", err)
    }

    return user, nil
}

// 呼叫端
if user, err := loadUser("42"); err != nil {
    log.Printf("load failed: %v", err)
} else {
    fmt.Println(user.Name)
}
TEMPLATE · 狀態機 
type State int

const (
    StateIdle State = iota
    StateRunning
    StateDone
)

func (s State) transition(event string) State {
    switch s {
    case StateIdle:
        if event == "start" {
            return StateRunning
        }
    case StateRunning:
        switch event {
        case "finish":
            return StateDone
        case "pause":
            return StateIdle
        }
    }
    return s
}

本章速記

  1. 條件不用括號，但大括號必寫：連單行的 if 也不能省大括號。
  2. if + init 是核心慣用法：if err := f(); err != nil { ... } 每天寫幾十次。
  3. for 是唯一迴圈：while 就是 for cond，無限迴圈就是 for {}。
  4. 用 for i := range N（Go 1.22+）取代三段式整數迴圈。
  5. range 的 v 是副本：要改原資料請透過 index 或取址。
  6. switch 不 fallthrough：要穿透必須明寫 fallthrough。
  7. case 可以是任意表達式：空 switch + case cond 就是 if/else 鏈。
  8. type switch：switch v := x.(type) { case int: ... }，是處理 any 的標準工具。
  9. labeled break/continue：巢狀迴圈跳多層的正解。
  10. 沒有三元運算子：習慣了反而覺得程式碼更好讀。
