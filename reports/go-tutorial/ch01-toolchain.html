CHAPTER 01 · 基礎入門

環境 & 工具鏈

安裝 Go 1.26、設定模組、熟悉 go 指令家族與 gopls，打造一套能陪伴你十年的開發環境。

Go 版本 · 1.26.2 (2026-04-07) 關鍵指令 · go run / build / test / mod / fix 對照 Python · venv + pip + pyright

安裝 Go

Go 不像 Python 有多家發行版，唯一來源是官方網站。每次安裝都會覆蓋舊版，多版本共存要用 go install golang.org/dl/goX.Y@latest。

三種主流安裝方式

平台指令備註
macOSbrew install go最快，自動加入 PATH
Linux下載 tar.gz 解壓到 /usr/local官方最推薦，版本最新
Windows.msi 安裝檔 或 winget install GoLang.Go自動設定環境變數
多版本go install golang.org/dl/go1.25@latest再跑 go1.25 download

驗證安裝

# 顯示版本；應該輸出類似 go version go1.26.2 darwin/arm64
go version

# 檢視完整環境變數（GOPATH、GOROOT、GOMODCACHE…）
go env

# 只看某一個變數
go env GOPATH GOMODCACHE GOPROXY
Python 開發者類比 go env ≈ python -c "import sys; print(sys.prefix)" + pip config list。Go 把所有設定集中在一個指令裡，不需要碰 shell rc 檔案。

從 GOPATH 到 Modules

舊文章常提到 $GOPATH/src/github.com/... 的目錄結構 — 那是 2019 年以前的模式。Go 1.16 之後 Modules 是唯一推薦方式，你可以把專案放在任何地方。

建立新專案

# 1) 建目錄並進入
mkdir hello && cd hello

# 2) 初始化模組（module path 通常用 github.com/你/專案名）
go mod init github.com/you/hello

# 3) 新增一支 Go 檔案
cat > main.go <<'EOF'
package main
import "fmt"
func main() { fmt.Println("Hello, Go 1.26!") }
EOF

# 4) 執行
go run .

go.mod 檔案結構

module github.com/you/hello

go 1.26

require (
    github.com/google/uuid v1.6.0
    golang.org/x/sync v0.10.0
)

// 1.25+ 新增：告訴 go 指令忽略這些子目錄
ignore (
    ./testdata
    ./internal/scripts
)
對照 Python go.mod ≈ pyproject.toml + requirements.txt 合一；go.sum ≈ poetry.lock，記錄每個依賴的雜湊值。Go 沒有 venv 概念 — 所有依賴都快取在 $GOMODCACHE，每個專案用 go.mod 隔離版本。

go 指令家族

Go 的 CLI 設計極度克制 — 幾乎所有開發需求都落在下表十個子指令之內。

指令用途Python 對照
go run .編譯並立即執行（不留二進位檔）python main.py
go build編譯出可執行檔pyinstaller
go test ./...跑所有子目錄的測試pytest
go test -race開啟資料競爭偵測器(無直接對應)
go mod tidy整理依賴（加新、砍沒用到的）pip-compile
go get pkg@v1.2.3加入或升級依賴pip install pkg==1.2.3
go fmt ./...自動格式化black .
go vet ./...靜態錯誤分析flake8 / ruff
go fix ./...1.26 重寫：套用現代化建議pyupgrade
go doc fmt.Println離線查文件pydoc
go tool pprof效能分析cProfile
常見陷阱 go run main.go 只編譯單檔，同目錄下其他 .go 會被忽略 — 多檔案專案請固定用 go run .（當前目錄整個 package）。

go fix · 1.26 的明星功能

Go 1.26 把沉睡已久的 go fix 指令完全改寫為「modernizer」框架：用 AST 分析掃出舊寫法，一鍵替換成新語法。這是 AI 時代保持程式碼現代化的關鍵工具。

# 檢視有哪些建議（dry run）
go fix -diff ./...

# 實際套用（會直接修改檔案，請先 commit）
go fix ./...

# 只跑特定 analyzer（例如把 for i:=0;i<N;i++ 改成 for i := range N）
go fix -analyzers=rangeint ./...

rangeint

把 C 風格 for 迴圈改成 for i := range 10（Go 1.22+ 語法）。

slicesdelete

把手寫的 append-trick 換成 slices.Delete。

minmax

把三元判斷式改成內建 min()/max()。

anyconvert

把 interface{} 換成 any（Go 1.18+ 別名）。

gopls · Language Server

所有現代 IDE（VS Code、Neovim、Zed、GoLand）都透過 gopls 提供自動完成、跳轉、重構。你通常不用手動安裝，但要知道怎麼更新。

# 安裝／更新至最新版
go install golang.org/x/tools/gopls@latest

# 檢查版本
gopls version

# VS Code 使用者：安裝 "Go" 官方擴充套件，
# 它會自動偵測並提示安裝 gopls、dlv（debugger）、staticcheck。
推薦配置 VS Code 的 settings.json 加入 "go.useLanguageServer": true 與 "[go]": {"editor.formatOnSave": true, "editor.codeActionsOnSave": {"source.organizeImports": "always"}}，存檔時自動格式化與整理 import。

零基礎專案模板

複製下方指令到終端機，30 秒內擁有一個可跑、可測、可建置的完整 Go 專案。

COPY & RUN 
# 建立目錄骨架
mkdir -p myapp/{cmd/myapp,internal/service} && cd myapp
go mod init github.com/you/myapp

# 主程式
cat > cmd/myapp/main.go <<'EOF'
package main

import (
    "fmt"
    "github.com/you/myapp/internal/service"
)

func main() {
    fmt.Println(service.Greet("Gopher"))
}
EOF

# 業務邏輯（放在 internal/ 代表外部模組無法 import）
cat > internal/service/greet.go <<'EOF'
package service

func Greet(name string) string {
    return "Hello, " + name + "!"
}
EOF

# 測試檔
cat > internal/service/greet_test.go <<'EOF'
package service

import "testing"

func TestGreet(t *testing.T) {
    if got := Greet("World"); got != "Hello, World!" {
        t.Errorf("got %q", got)
    }
}
EOF

# 跑起來
go run ./cmd/myapp
go test ./...
go build -o bin/myapp ./cmd/myapp

本章速記

  1. 安裝：認準 go.dev 官方來源，版本不對先換版本再說。
  2. 一律用 Modules：不要把專案放進 GOPATH，任何位置 go mod init 都能開工。
  3. 十個指令包辦一切：run / build / test / mod / get / fmt / vet / fix / doc / tool。
  4. 設定靠 go.mod，不靠 shell：專案的 Go 版本、依賴、忽略目錄都寫在這。
  5. 每半年升級一次：Go 每年 2 月、8 月各出一個大版本，搭配 go fix 自動現代化程式碼。
  6. 多檔案用 go run .，不要用 go run main.go。
  7. IDE 靠 gopls：是 LSP 不是 IDE 擴充，更新只要 go install @latest。
