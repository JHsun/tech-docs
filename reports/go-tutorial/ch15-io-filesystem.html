CHAPTER 15 · 標準庫與實務

I/O & 檔案系統

Go 的 I/O 設計是「小介面互相組合」的最佳示範 — Reader、Writer 兩個方法撐起整個生態。Go 1.25 推出 os.Root 解決了長年的 path traversal 安全漏洞。

核心套件 · io · os · bufio · io/fs Go 1.25+ · os.Root（安全目錄操作）

io.Reader 與 io.Writer

這兩個介面是 Go 整個 I/O 生態的基石。只要實作了它們，檔案、網路、壓縮、加密…全都能串起來。

// 整個 I/O 生態的基石 — 一個方法
type Reader interface {
    Read(p []byte) (n int, err error)
}

type Writer interface {
    Write(p []byte) (n int, err error)
}

無處不在的 Reader / Writer

實作者它是什麼
*os.File開啟的檔案（同時是 Reader 和 Writer）
*strings.Reader把 string 當 Reader 來讀
*bytes.Buffer記憶體中的 Reader + Writer
net.ConnTCP/UDP 連線
*http.Request.BodyHTTP 請求 body
*http.ResponseWriterHTTP 回應寫入
*gzip.Reader / *gzip.Writer壓縮流
os.Stdin / Stdout / Stderr標準輸入輸出
為什麼這個設計這麼成功？ 函式只依賴 Reader 或 Writer，就能「吃」上面所有東西。寫解析器？傳進 io.Reader，測試用 strings.NewReader，上線用 os.Open，網路用 resp.Body。毫無修改。

讀取檔案的三種方式

① 整個讀進記憶體（小檔案）

import "os"

data, err := os.ReadFile("config.json")
if err != nil {
    return err
}
// data 是 []byte，整個檔案內容
fmt.Println(string(data))

② 開啟後 stream（大檔案）

f, err := os.Open("huge.log")
if err != nil {
    return err
}
defer f.Close()

// 低層：直接 Read
buf := make([]byte, 4096)
for {
    n, err := f.Read(buf)
    if n > 0 {
        process(buf[:n])
    }
    if err == io.EOF { break }
    if err != nil { return err }
}

③ 逐行讀（bufio.Scanner）

import "bufio"

f, _ := os.Open("access.log")
defer f.Close()

scanner := bufio.NewScanner(f)
// 預設最長一行 65536 bytes；長日誌需要自己調大
scanner.Buffer(make([]byte, 0, 64*1024), 1024*1024)

for scanner.Scan() {
    line := scanner.Text()    // 或 .Bytes() 避免拷貝
    process(line)
}
if err := scanner.Err(); err != nil {
    return err
}
Scanner 的行長度陷阱 bufio.Scanner 預設單行上限 65536 bytes，超過會返回 bufio.ErrTooLong。處理長日誌或 JSON Lines 時記得 scanner.Buffer(...) 加大緩衝區。真的可能有超大行，改用 bufio.Reader.ReadString('\n')。

寫入檔案

// 最簡單：一次寫入（覆蓋）
err := os.WriteFile("out.txt", []byte("hello"), 0644)

// 精細控制：OpenFile + flags
f, err := os.OpenFile("log.txt",
    os.O_WRONLY|os.O_CREATE|os.O_APPEND, 0644)
if err != nil {
    return err
}
defer f.Close()

f.WriteString("log line\n")
f.Write([]byte("raw bytes"))
fmt.Fprintf(f, "formatted %d\n", n)

常用 flag 組合

目的flag 組合
覆寫O_WRONLY | O_CREATE | O_TRUNC
附加O_WRONLY | O_CREATE | O_APPEND
必須是新檔（存在則失敗）O_WRONLY | O_CREATE | O_EXCL
讀寫O_RDWR

帶緩衝寫入（高吞吐）

f, _ := os.Create("out.txt")
defer f.Close()

w := bufio.NewWriter(f)
defer w.Flush()               // ⚠️ 一定要 Flush 才會真正寫入

for _, line := range lines {
    w.WriteString(line + "\n")
}
// 函式結束時 Flush 被呼叫 → 緩衝區內容寫入 f
bufio.Writer 的鐵律 defer w.Flush() 要寫！沒 Flush 程式結束時緩衝內容會丟失。更穩的寫法：defer func() { w.Flush(); f.Close() }()，Flush 後才關檔，否則 Flush 錯誤會被忽略。

io 套件的常用工具

import "io"

// io.ReadAll — 一次讀完（取代舊的 ioutil.ReadAll，Go 1.16+）
data, err := io.ReadAll(reader)

// io.Copy — 從 reader 複製到 writer（超常用）
n, err := io.Copy(dst, src)       // 下載檔案、代理請求都靠它

// io.CopyN — 只複製 N bytes
io.CopyN(dst, src, 1024)

// io.LimitReader — 限制最多讀 N bytes
limited := io.LimitReader(reader, 1<<20)  // 最多 1 MiB

// io.TeeReader — 一邊讀一邊寫到 tee（計 hash / debug log 很好用）
h := sha256.New()
tee := io.TeeReader(src, h)
io.Copy(dst, tee)             // 複製時同步計算 hash
fmt.Printf("%x", h.Sum(nil))

// io.MultiReader / io.MultiWriter — 串接多個 Reader / 廣播到多個 Writer
all := io.MultiReader(r1, r2, r3)    // 依序讀完三個
tee := io.MultiWriter(file, os.Stdout) // 一份寫檔、一份印出

// io.Discard — 丟棄一切（適合只想計數）
io.Copy(io.Discard, resp.Body)   // 為了 HTTP keep-alive 還是要把 body 讀完
io.Copy 的威力 上傳、下載、反向代理、檔案搬移 — 幾乎所有「搬 bytes」的需求都是 io.Copy(dst, src)。它會自動使用 WriteTo 或 ReadFrom 最佳化路徑（例如 sendfile），比手寫迴圈高效。

路徑處理：filepath vs path

兩個套件常被搞混：filepath 處理作業系統路徑（Windows 用 \，Unix 用 /），path 處理永遠用 / 的抽象路徑（URL、embed、io/fs）。

import "path/filepath"

filepath.Join("dir", "sub", "file.txt")   // 跨平台正確
filepath.Ext("a/b/c.txt")                 // ".txt"
filepath.Base("a/b/c.txt")                // "c.txt"
filepath.Dir("a/b/c.txt")                 // "a/b"
filepath.Clean("a/./b/../c")               // "a/c"
filepath.Abs("./data")                     // 絕對路徑
filepath.Glob("*.go")                       // 萬用字元比對

// 走訪目錄樹
filepath.WalkDir(".", func(path string, d fs.DirEntry, err error) error {
    if err != nil { return err }
    if d.IsDir() && d.Name() == "node_modules" {
        return filepath.SkipDir  // 不進這個目錄
    }
    fmt.Println(path)
    return nil
})
WalkDir vs Walk filepath.Walk（舊）每個檔案都 stat 一次，慢；filepath.WalkDir（Go 1.16+）只在需要時 stat，大目錄樹下快一個數量級。新程式一律用 WalkDir。

Go 1.25：os.Root · 安全的目錄操作

長久以來 Go 的 path 操作都有 path traversal 風險 — 惡意輸入像 ../../etc/passwd 可以逃出你指定的目錄。Go 1.25 正式 GA 的 os.Root 把「作業範圍限制在某個目錄內」做成系統呼叫層級的保證。

import "os"

// 打開一個「沙箱」根目錄
root, err := os.OpenRoot("/var/lib/myapp")
if err != nil {
    return err
}
defer root.Close()

// 所有操作都限制在 /var/lib/myapp 內
// 即使使用者傳入 "../../etc/passwd" 也會被拒絕
f, err := root.Open(userInput)       // 跨出 root 會回 error
data, err := root.ReadFile("config.json")
err = root.Mkdir("sessions", 0755)
err = root.Remove("old.tmp")
err = root.Rename("a", "b")
什麼時候該用 os.Root？ 任何時候你根據外部輸入建構檔案路徑 — 檔案上傳、模板路徑、使用者上傳的 ZIP 解壓、template include…。這些都是 Zip Slip / Path Traversal CVE 的經典來源。過去靠手動 filepath.Clean + 前綴檢查，容易寫錯；os.Root 讓錯誤不可能發生。

io/fs 生態

import "io/fs"

// io/fs 抽象了「檔案系統」概念
// 最常見的兩個實作：os.DirFS 和 embed.FS

dirfs := os.DirFS("./static")        // 真實目錄當 FS
data, _ := fs.ReadFile(dirfs, "index.html")

// embed（Go 1.16+）— 把檔案打包進 binary
//go:embed templates/*
var tplFS embed.FS

// 同樣的 API
data, _ := fs.ReadFile(tplFS, "templates/home.html")

// http.FS 把 fs.FS 轉成 http.FileSystem
http.Handle("/static/", http.FileServer(http.FS(tplFS)))

臨時檔案與目錄

// 系統臨時目錄（通常 /tmp 或 Windows 的 %TEMP%）
os.TempDir()

// 建立臨時檔（自動取唯一名稱）
f, err := os.CreateTemp("", "upload-*.tmp")
// f.Name() 會是類似 "/tmp/upload-123456.tmp"
defer os.Remove(f.Name())
defer f.Close()

// 建立臨時目錄（完工後整個刪）
dir, err := os.MkdirTemp("", "work-")
defer os.RemoveAll(dir)

// 測試中的慣用法（t.TempDir 會自動在測試結束後清除）
func TestSomething(t *testing.T) {
    dir := t.TempDir()
    // ... 用 dir 做事，完全不用清理
}

實戰模板

TEMPLATE · 安全下載檔案 
func download(url, dest string) error {
    resp, err := http.Get(url)
    if err != nil {
        return fmt.Errorf("get %q: %w", url, err)
    }
    defer resp.Body.Close()

    if resp.StatusCode != http.StatusOK {
        return fmt.Errorf("bad status: %s", resp.Status)
    }

    // 先寫到同目錄下的臨時檔，完成後 rename — atomic 寫入
    dir := filepath.Dir(dest)
    tmp, err := os.CreateTemp(dir, ".download-*.tmp")
    if err != nil {
        return err
    }
    tmpName := tmp.Name()
    defer os.Remove(tmpName)   // 若中途失敗，臨時檔會被清掉

    // 限制大小，避免惡意伺服器灌爆
    limited := io.LimitReader(resp.Body, 100*1024*1024) // 100 MB

    if _, err := io.Copy(tmp, limited); err != nil {
        tmp.Close()
        return fmt.Errorf("write: %w", err)
    }
    if err := tmp.Close(); err != nil {
        return err
    }

    // rename 在同一 filesystem 上是 atomic 的
    return os.Rename(tmpName, dest)
}
TEMPLATE · 安全處理使用者上傳（os.Root） 
func handleUpload(baseDir string, userFilename string, body io.Reader) error {
    // 限制所有操作只能在 baseDir 底下發生
    root, err := os.OpenRoot(baseDir)
    if err != nil {
        return err
    }
    defer root.Close()

    // 即使 userFilename = "../../etc/passwd"，root 也會拒絕
    safeName := filepath.Base(userFilename)   // 雙保險
    f, err := root.Create(safeName)
    if err != nil {
        return err
    }
    defer f.Close()

    limited := io.LimitReader(body, 10*1024*1024) // 10 MB
    _, err = io.Copy(f, limited)
    return err
}

本章速記

  1. io.Reader / Writer 是基石：任何吃 bytes 的 API 都靠它。
  2. 小檔案用 os.ReadFile，大檔案用 stream + bufio。
  3. bufio.Scanner 逐行讀：注意 65536 bytes 的預設上限。
  4. bufio.Writer 必 Flush：用 defer 別忘了。
  5. io.Copy(dst, src) 是搬 bytes 的萬金油。
  6. io.LimitReader 防止惡意輸入吃光記憶體。
  7. filepath 處理 OS 路徑，path 處理 / 路徑。
  8. filepath.WalkDir 取代舊的 Walk。
  9. os.Root（Go 1.25+）：處理外部路徑輸入時必用。
  10. embed.FS / os.DirFS：用同一套 API 操作記憶體或磁碟的檔案樹。
  11. atomic 寫入：臨時檔 + rename 是標準做法。
  12. 測試用 t.TempDir()：自動清理不出錯。
