基本型別 & 變數

Go 是強型別、靜態型別語言，沒有 Python 的「什麼都是物件」。型別系統嚴謹但語法糖（:=、零值、型別推論）讓寫起來一點都不囉唆。

內建型別 · int / float / string / bool / rune / byte 關鍵概念 · 零值、型別推論、常數、iota

四種宣告變數的方式

Go 對「什麼時候能用什麼語法」管得很嚴。理解下列四種寫法的適用情境，比背語法更重要。

// 1) 完整形式：指定型別 + 初值
var age int = 30

// 2) 省略型別（編譯器從右側推論）
var name = "Alice"       // 推論為 string

// 3) 省略初值（使用零值）
var count int            // count == 0
var msg string            // msg == ""

// 4) 短宣告 :=（最常用，但僅限函式內部）
func main() {
    x := 42               // int
    y := 3.14             // float64
    s := "hello"          // string
    ok := true            // bool
}
寫法可用位置場景
var x T = vpackage / func需要同時確立型別與初值
var x = vpackage / func型別明顯不需寫出
var x Tpackage / func先宣告後賦值，使用零值
x := v僅 func 內最常用，簡潔
宣告了就必須用 Go 編譯器規則：函式內宣告的區域變數若沒被讀取，編譯失敗。這是為了避免 bug（Python 最大型別 lint warning 在 Go 是硬性錯誤）。臨時除錯時可用 _ = x 丟棄。

基本型別清單

Go 型別精簡但明確指定位寬，比 Python 的 int（無限精度）更接近 C。

數值型別

型別範圍／精度用途
int / uint平台相關（32 或 64 位元）通用整數，最常用
int8 / int16 / int32 / int64明確位寬序列化、位元運算
uint8 / uint16 / uint32 / uint64無號位元遮罩、byte 操作
byte= uint8 的別名處理原始位元組
rune= int32 的別名Unicode code point
float32 / float64IEEE 754預設用 float64
complex64 / complex128複數科學計算

其他基本型別

型別說明
string不可變的 UTF-8 位元組序列
booltrue 或 false，不能用 0/1 互換
error內建介面型別，表示錯誤
any= interface{}（Go 1.18+ 別名）
對照 Python Python 的 int 無上限；Go 的 int 是 64 位有號整數（64 位平台上），超過會溢位且不會 panic。處理大數要用 math/big 套件。

零值（Zero Value）

Go 沒有 undefined、沒有 null — 任何型別的變數宣告後都有「零值」，這讓程式行為更可預測。

型別零值
數值型別 (int, float, …)0
boolfalse
string"" (空字串，不是 nil)
指標、interface、slice、map、channel、funcnil
array所有元素都是其型別的零值
struct所有欄位都是其型別的零值
善用零值可設計 sync.Mutex、bytes.Buffer 等標準庫型別都刻意設計成「零值直接可用」，不用呼叫什麼 Init()。寫自己的 struct 時也追求這個目標，能省掉很多 constructor。

常數與 iota

Go 的常數在編譯時期決定，且可以「無型別」— 寫 const x = 3 時 x 是「無型別整數常數」，賦值給 int32 或 float64 都不用轉。

const Pi = 3.14159            // 無型別 float 常數
const Name string = "Go"        // 指定型別

const (
    KB = 1 << (10 * (iota + 1))
    MB
    GB
    TB
)
// KB = 1024, MB = 1048576, GB = 1073741824, TB = 1099511627776

iota：列舉神器

type Weekday int

const (
    Sunday Weekday = iota  // 0
    Monday                     // 1（接續上一行的表達式）
    Tuesday                    // 2
    Wednesday                  // 3
    Thursday                   // 4
    Friday                     // 5
    Saturday                   // 6
)

// 跳過某個值：用 _
const (
    _ = iota  // 跳過 0
    Low        // 1
    Medium     // 2
    High       // 3
)

// 位元旗標
type Perm int
const (
    Read Perm = 1 << iota  // 1 (0b001)
    Write                        // 2 (0b010)
    Execute                      // 4 (0b100)
)
iota 的機制 iota 在每個 const (...) 區塊中從 0 開始，每換一行加 1。當某行省略賦值時，會重複上一行的表達式 — 這就是為什麼 Weekday 例子裡只寫第一行即可。

型別轉換（必須顯式）

Go 不做任何隱式轉換，連 int 和 int32 之間都要明寫 — 雖然囉唆，但杜絕了 C 的精度損失 bug。

var i int = 42
var f float64 = float64(i)      // int → float64
var u uint = uint(f)             // float64 → uint

// string 與 byte/rune
s := "hello"
b := []byte(s)                   // string → []byte
r := []rune(s)                   // string → []rune（處理多位元組字元）
s2 := string(b)                  // []byte → string

// string 與 int：要用 strconv！
// 注意：string(65) 是 "A"，不是 "65"，這是常見地雷
import "strconv"
n, err := strconv.Atoi("42")       // string → int
str := strconv.Itoa(42)           // int → "42"
最常見地雷 string(65) 得到 "A"（Unicode 轉字元），不是 "65"。新手 90% 會踩。要得到 "65" 必須用 strconv.Itoa(65)。Go 1.15 之後編譯器會對明顯的 int→string 轉換發出警告。

字串：UTF-8 的真相

Go 的 string 底層是「唯讀位元組陣列」，一般以 UTF-8 編碼儲存。len() 傳回位元組數而非字元數，這點經常讓 Python 出身的人踩雷。

s := "你好, Go!"

len(s)                // 11（"你"、"好" 各佔 3 個 UTF-8 byte）

// 用 range 會按 rune（Unicode code point）走
for i, r := range s {
    fmt.Printf("%d: %c\n", i, r)
}
// 0: 你
// 3: 好   ← 注意 i 是 byte 索引，不是 rune 索引
// 6: ,
// 7:  
// 8: G
// 9: o
// 10: !

// 要計算字元數用 utf8.RuneCountInString
import "unicode/utf8"
utf8.RuneCountInString(s)   // 7

// 字串是不可變的，但可以用 + 拼接（效能差，多段拼接用 strings.Builder）
s1 := "Hello, " + "World!"

實戰模板

TEMPLATE · 列舉與位元旗標 
package main

import "fmt"

type Color int

const (
    Red Color = iota
    Green
    Blue
)

// 讓列舉值好看：實作 String() 方法（下一章會深入）
func (c Color) String() string {
    return [...]string{"Red", "Green", "Blue"}[c]
}

type FileMode uint

const (
    ModeRead FileMode = 1 << iota
    ModeWrite
    ModeExec
)

func main() {
    mode := ModeRead | ModeWrite
    fmt.Printf("%b\n", mode)  // 11

    if mode&ModeRead != 0 {
        fmt.Println("可讀")
    }
}

本章速記

  1. 函式內用 :=，package 層級用 var。
  2. 型別一律顯式轉換：int(f)、float64(i)，沒有隱式轉換。
  3. 零值可直接用：var m map[string]int 能讀但不能寫 — map 的零值是 nil，要用 make。
  4. 宣告未用 → 編譯失敗：臨時可用 _ = x 迴避。
  5. string 是 byte 陣列：len() 是 byte 數，不是字元數。
  6. string(N) 是字元轉換：要「數字轉字串」請用 strconv.Itoa。
  7. iota 是編譯期計數器：專門用來寫列舉和位元旗標。
  8. 常數可無型別：const x = 3 能賦值給任何數值型別，不用轉。
