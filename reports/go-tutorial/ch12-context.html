CHAPTER 12 · 並發與進階

Context ctx

幾乎每個 Go 專案都會看到 ctx context.Context — 它是取消信號、超時、截止時間、請求範圍值的統一載體。不認得它，等於讀不懂 Go 生態。

套件 · context 核心方法 · Done / Err / Deadline / Value 位置 · 函式第一個參數

Context 解決什麼問題？

想像一個 HTTP 請求：需要查資料庫、呼叫多個微服務、執行並發任務。如果使用者放棄了請求（瀏覽器關閉），我們要怎麼通知所有衍生 goroutine 停下來？答案就是 context。

① 取消傳播

從最上層取消後，能沿著函式呼叫鏈一路通知到最深的 goroutine 停止工作。

② 超時控制

「這個請求最多給我 5 秒」— 設定一次，整個呼叫鏈都遵守。

③ 截止時間

比 timeout 更精確：「這個請求必須在 14:00:00 之前完成」。

④ 請求範圍的值

attach 一些值跟著 request 跑：trace ID、使用者身份、locale。

Context 介面本體

介面只有四個方法 — 但支撐了整個 Go 生態。

type Context interface {
    // Done 回傳一個 channel，ctx 被取消或超時會關閉它
    Done() <-chan struct{}

    // Err 告訴你取消的原因：canceled / deadline exceeded
    Err() error

    // Deadline 回傳截止時間（ok 為 false 代表沒設）
    Deadline() (deadline time.Time, ok bool)

    // Value 取出 attach 在 ctx 上的值
    Value(key any) any
}

慣用法：在 select 中監聽 Done

func doWork(ctx context.Context) error {
    for {
        select {
        case <-ctx.Done():
            return ctx.Err()           // 返回 context.Canceled 或 DeadlineExceeded
        case task := <-taskChan:
            process(task)
        }
    }
}

建立 Context 的六種方式

函式用途
context.Background()根 context — main、init、測試的起點
context.TODO()暫時佔位，之後會換成真的 — 傳達「未決定」
context.WithCancel(parent)可手動取消；回傳 ctx + cancel 函式
context.WithTimeout(parent, d)d 時間後自動取消
context.WithDeadline(parent, t)指定時間點取消
context.WithValue(parent, k, v)attach 值（少用、謹慎用）
context.WithCancelCause (Go 1.20+)取消時附上原因的 error
 
// 1) 根 context
ctx := context.Background()

// 2) WithCancel — 手動取消
ctx, cancel := context.WithCancel(ctx)
defer cancel()      // 無論如何都要呼叫！
cancel()             // 需要的時候主動呼叫

// 3) WithTimeout — 最常用
ctx, cancel := context.WithTimeout(ctx, 5 * time.Second)
defer cancel()

// 4) WithDeadline — 明確時間點
ctx, cancel := context.WithDeadline(ctx, time.Now().Add(1*time.Minute))
defer cancel()

// 5) WithValue — 請求範圍的值
ctx = context.WithValue(ctx, "requestID", "abc-123")

// 6) Go 1.20+ WithCancelCause — 取消時帶原因
ctx, cancel := context.WithCancelCause(ctx)
cancel(errors.New("user aborted"))
// 之後 context.Cause(ctx) 可取出那個 error
cancel 永遠要呼叫 不管 WithCancel / WithTimeout / WithDeadline，回傳的 cancel 一定要被呼叫（通常用 defer cancel()）。忘了呼叫會造成 context 樹無法釋放 — linter govet 會警告。

Context 的五條金律

  1. 放在函式的第一個參數：命名就叫 ctx。
    func DoSomething(ctx context.Context, ...)
  2. 不要放進 struct：per-request 的東西就該沿著 call stack 傳。例外是專門為了某個 request 存在的 struct。
  3. 不要傳 nil context：不確定用什麼時，傳 context.TODO()。
  4. WithValue 只用於 request-scoped 的元資料：trace ID、使用者身份、locale — 不要拿它當函式參數傳遞器。
  5. Context 是不可變的：WithXxx 都是建立新的 ctx，不是修改原本的。
WithValue 的 key 陷阱 別用 string 當 key — 容易撞名。標準做法：自訂不匯出的型別。
type ctxKey int; const userKey ctxKey = iota; ctx.Value(userKey)

標準庫的深度整合

Context 不是可選的優雅功能 — Go 標準庫的關鍵 API 都吃 ctx，不用它就丟失了取消能力。

// net/http — 客戶端請求
req, _ := http.NewRequestWithContext(ctx, "GET", url, nil)
resp, err := http.DefaultClient.Do(req)

// HTTP server handler — Request 本身帶 ctx
func handler(w http.ResponseWriter, r *http.Request) {
    ctx := r.Context()        // client 斷線會自動取消
    result, err := doWork(ctx)
    // ...
}

// database/sql
rows, err := db.QueryContext(ctx, "SELECT ...")
_, err := db.ExecContext(ctx, "UPDATE ...")

// os/exec
cmd := exec.CommandContext(ctx, "ls", "-l")

// net (dial 連線)
conn, err := dialer.DialContext(ctx, "tcp", "host:port")
黃金守則 只要某個函式可能阻塞或慢（網路、DB、檔案、子行程），優先找它的 XxxContext 版本。沒有的話視為「不支援取消」— 那你要考慮自己包一層。

Go 1.21+ 的新工具

// context.WithoutCancel — 取出 Value 但斷開取消鏈
// 適用：request 完成後要做 async log / metrics 上傳
detached := context.WithoutCancel(ctx)
go uploadMetrics(detached)    // 即使原 request 取消，這個 goroutine 還會繼續

// context.AfterFunc — 註冊「取消後執行」的回呼
stop := context.AfterFunc(ctx, func() {
    log.Println("ctx was cancelled")
})
defer stop()    // 若還未觸發可以取消註冊

// context.WithDeadlineCause / context.WithTimeoutCause — 帶原因
ctx, cancel := context.WithTimeoutCause(
    ctx, 5*time.Second,
    errors.New("database query too slow"),
)
defer cancel()

// 超時後
ctx.Err()             // context.DeadlineExceeded
context.Cause(ctx)    // "database query too slow"

實戰模板

TEMPLATE · 帶超時的 HTTP handler 
func handler(w http.ResponseWriter, r *http.Request) {
    // 為這個請求加上 3 秒超時（在 client ctx 之上）
    ctx, cancel := context.WithTimeout(r.Context(), 3*time.Second)
    defer cancel()

    result, err := fetchFromDB(ctx)
    if err != nil {
        if errors.Is(err, context.DeadlineExceeded) {
            http.Error(w, "timeout", http.StatusGatewayTimeout)
            return
        }
        http.Error(w, "internal error", 500)
        return
    }
    json.NewEncoder(w).Encode(result)
}

func fetchFromDB(ctx context.Context) ([]Row, error) {
    rows, err := db.QueryContext(ctx, "SELECT * FROM users")
    // ... 如果 ctx 在中途取消，QueryContext 會立刻返回錯誤
}
TEMPLATE · 並發執行多任務 + 任一失敗立即取消 
import "golang.org/x/sync/errgroup"

func fetchAll(ctx context.Context, urls []string) error {
    // errgroup 自動衍生 ctx，任一任務出錯就取消所有兄弟 goroutine
    g, ctx := errgroup.WithContext(ctx)

    for _, url := range urls {
        g.Go(func() error {
            req, err := http.NewRequestWithContext(ctx, "GET", url, nil)
            if err != nil {
                return err
            }
            resp, err := http.DefaultClient.Do(req)
            if err != nil {
                return err
            }
            defer resp.Body.Close()
            // 處理 resp...
            return nil
        })
    }
    return g.Wait()   // 所有完成，或第一個錯誤
}
TEMPLATE · 可取消的自訂 worker 
func worker(ctx context.Context, tasks <-chan Task) {
    for {
        select {
        case <-ctx.Done():
            log.Printf("worker stopping: %v", ctx.Err())
            return
        case task, ok := <-tasks:
            if !ok {
                return
            }
            // 在處理內部也要檢查 ctx
            if err := processWithContext(ctx, task); err != nil {
                log.Printf("task failed: %v", err)
            }
        }
    }
}

本章速記

  1. ctx 永遠是第一個參數：命名就叫 ctx。
  2. 不要把 ctx 放進 struct：它是 per-call 的東西。
  3. 不要傳 nil：用 context.TODO() 當佔位。
  4. cancel 一定要呼叫：defer cancel() 是肌肉記憶。
  5. 在 select 監聽 ctx.Done()：取消傳播的標準寫法。
  6. errors.Is 檢查取消原因：context.Canceled / context.DeadlineExceeded。
  7. WithValue 只用於 request-scoped 元資料：別拿來當參數傳。
  8. Value 的 key 用自訂型別：避免與其他套件撞名。
  9. 標準庫用 XxxContext 版本：HTTP、SQL、exec、net — 都吃 ctx。
  10. errgroup 是 WaitGroup + Context 的組合拳。
  11. Go 1.21+ 工具：WithoutCancel、AfterFunc、WithTimeoutCause。
  12. 記得文件化 ctx 行為：你的函式會因 ctx 取消做什麼？清理嗎？回滾嗎？
