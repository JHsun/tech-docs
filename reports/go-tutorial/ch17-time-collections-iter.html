time 套件基本

Go 的 time.Time 是 struct，包含時間點 + 時區。永遠記得：time.Time 比較要用 Equal，不是 ==（== 會比位置資訊）。

import "time"

// 取得現在
now := time.Now()

// 建立特定時間
t := time.Date(2026, time.April, 17, 14, 30, 0, 0, time.UTC)

// 分量
now.Year()           // 2026
now.Month()          // time.April (= 4)
now.Day()            // 17
now.Weekday()        // time.Friday
now.Hour(), now.Minute(), now.Second()
now.Unix()           // Unix timestamp（秒）
now.UnixMilli()      // 毫秒
now.UnixNano()       // 奈秒

// 比較（NEVER use ==）
t1.Before(t2)
t1.After(t2)
t1.Equal(t2)         // 用這個，會忽略時區差異只比真正的時間點

// 時區
loc, _ := time.LoadLocation("Asia/Taipei")
t := now.In(loc)

Duration · 時間長度

// Duration 是 int64 的別名，單位奈秒
d := 3 * time.Second
d := 500 * time.Millisecond
d := 2*time.Hour + 30*time.Minute

// 常量
time.Nanosecond    // 1
time.Microsecond   // 1000
time.Millisecond   // 1_000_000
time.Second        // 10^9
time.Minute
time.Hour

// 時間運算
t2 := t1.Add(24 * time.Hour)
t3 := t1.AddDate(0, 1, 0)       // +1 month（年月日）
diff := t2.Sub(t1)                // Duration
elapsed := time.Since(start)         // 等同 time.Now().Sub(start)
until := time.Until(deadline)        // deadline.Sub(Now)

// 取得秒數等
d.Seconds()    // float64
d.Minutes()
d.Hours()

// 字串表示
d.String()     // "2h30m0s"
為什麼用 Duration 而不是 int？ time.Sleep(10) 看起來像 10 秒，其實是 10 奈秒。用 Duration 強迫你寫 10 * time.Second，意圖明確、不會出錯。

時間格式化：Go 最「奇特」的設計

Go 不用 %Y-%m-%d，而是用「參考時間」Mon Jan 2 15:04:05 MST 2006（記憶法：1, 2, 3, 4, 5, 6, 7）當樣板。

// 參考時間：Mon Jan 2 15:04:05 MST 2006
//         週一  1月 2日 15時04分05秒 MST 2006年

now.Format("2006-01-02")              // "2026-04-17"
now.Format("15:04:05")                // "14:30:00"
now.Format("2006-01-02 15:04:05")     // "2026-04-17 14:30:00"
now.Format("Mon, 02 Jan 2006")        // "Fri, 17 Apr 2026"

// 內建常用格式常量
now.Format(time.RFC3339)        // "2026-04-17T14:30:00+08:00"（JSON 標準）
now.Format(time.RFC1123)        // HTTP header 格式
now.Format(time.DateTime)       // "2026-04-17 14:30:00"（Go 1.20+）
now.Format(time.DateOnly)       // "2026-04-17"
now.Format(time.TimeOnly)       // "14:30:00"

// 解析
t, err := time.Parse("2006-01-02", "2026-04-17")

// 指定時區解析（解析 local time 必用）
loc, _ := time.LoadLocation("Asia/Taipei")
t, err := time.ParseInLocation("2006-01-02 15:04", "2026-04-17 14:30", loc)
記憶法 01/02 03:04:05 PM '06 -0700（月/日 時:分:秒 PM 年 時區）— 排列成 1 2 3 4 5 6 7。或背 2006-01-02 15:04:05 這一個就夠了。

Timer & Ticker

// time.Sleep — 阻塞式等待
time.Sleep(2 * time.Second)

// time.After — 非阻塞；回傳 channel，N 時間後 fire
select {
case <-taskDone:
    // 任務完成
case <-time.After(5 * time.Second):
    // 超時
}

// time.NewTimer — 可取消的一次性計時器
timer := time.NewTimer(5 * time.Second)
defer timer.Stop()        // 不用時記得 Stop，否則 leak
<-timer.C

// time.NewTicker — 週期性 fire
ticker := time.NewTicker(1 * time.Second)
defer ticker.Stop()

for {
    select {
    case <-ctx.Done():
        return
    case t := <-ticker.C:
        fmt.Println("tick", t)
    }
}

// time.AfterFunc — 在 N 時間後執行函式（自己的 goroutine）
timer := time.AfterFunc(30*time.Second, func() {
    fmt.Println("timed out")
})
// 可用 timer.Stop() 提前取消

slices & maps 套件（Go 1.21+）

泛型 API 取代了老派的型別特定 helper。這兩個套件值得熟記 — 幾乎涵蓋你對 slice/map 會做的所有事。

slices 常用 API

函式作用
slices.Contains(s, v)是否包含
slices.Index(s, v)索引（沒找到 -1）
slices.IndexFunc(s, f)自訂判斷找索引
slices.Sort(s)就地排序（Ordered）
slices.SortFunc(s, cmp)自訂比較排序
slices.SortStableFunc(s, cmp)穩定排序
slices.IsSorted(s)是否已排序
slices.BinarySearch(s, v)二分搜（需已排序）
slices.Clone(s)淺拷貝
slices.Reverse(s)就地反轉
slices.Max(s) / Min(s)最大 / 最小
slices.Equal(a, b)元素相等
slices.Concat(a, b, c)合併（Go 1.22+）
slices.Compact(s)移除相鄰重複
slices.Delete(s, i, j)刪除範圍
slices.Insert(s, i, v...)插入
slices.Collect(seq)從 iterator 收集（Go 1.23+）
 
import (
    "cmp"
    "slices"
)

nums := []int{3, 1, 4, 1, 5, 9, 2, 6}
slices.Sort(nums)                  // [1 1 2 3 4 5 6 9]
i, ok := slices.BinarySearch(nums, 4) // i=4, ok=true

// 自訂型別排序
type User struct{ Name string; Age int }

users := []User{...}
slices.SortFunc(users, func(a, b User) int {
    return cmp.Compare(a.Age, b.Age)   // 年齡升冪
})

// 多欄位排序：cmp.Or（Go 1.22+）
slices.SortFunc(users, func(a, b User) int {
    return cmp.Or(
        cmp.Compare(a.Age, b.Age),
        cmp.Compare(a.Name, b.Name),
    )
})

Go 1.23：iter 套件 · 官方迭代器

Go 1.23 引入「range over function」語法與 iter 套件。這是 Go 十年來最大的語法改動之一 — 終於可以寫自訂、組合、lazy 的迭代器。

兩種 iterator 型別

package iter

// Seq[V]：單值迭代器 — 對應 Python generator
type Seq[V any] func(yield func(V) bool)

// Seq2[K, V]：雙值迭代器 — 對應 Python items()
type Seq2[K, V any] func(yield func(K, V) bool)

寫一個 iterator

import "iter"

// 這個函式「回傳」一個可被 range 的序列
func Countdown(n int) iter.Seq[int] {
    return func(yield func(int) bool) {
        for i := n; i >= 0; i-- {
            if !yield(i) {      // yield 回 false = 消費者 break 了
                return
            }
        }
    }
}

// 使用：range 直接跑
for i := range Countdown(3) {
    fmt.Println(i)          // 3, 2, 1, 0
}

// Seq2：雙值迭代器
func Enumerate[T any](s []T) iter.Seq2[int, T] {
    return func(yield func(int, T) bool) {
        for i, v := range s {
            if !yield(i, v) {
                return
            }
        }
    }
}

for i, v := range Enumerate(fruits) {
    fmt.Println(i, v)
}
對照 Python generator Python 的 yield ≈ Go 的 yield(v) 呼叫。Python 的 for x in gen(): ≈ Go 的 for x := range seq。核心差別是 Go 的 yield 回傳 bool — false 代表消費者 break 了，生產者應該停下來。

搭配 slices / maps 的 iterator

// 新 API
for k := range maps.Keys(m) { ... }
for v := range maps.Values(m) { ... }
for k, v := range maps.All(m) { ... }

// 從 iterator 收集回 slice
keys := slices.Collect(maps.Keys(m))
slices.Sort(keys)

// 從 slice 建立 iterator
for i, v := range slices.All(s) { ... }
for v := range slices.Values(s) { ... }
for v := range slices.Backward(s) { ... }  // 反向走訪

iter 套件的組合工具

// Pull / Pull2 — 把「推」轉成「拉」
// 當你想要兩個 iterator 交錯取值時用
next, stop := iter.Pull(seq)
defer stop()
v, ok := next()      // 每呼叫一次取一個

sort 套件（舊 API）

Go 1.21+ 基本都用 slices.Sort，但舊程式碼還很多 sort.Interface，認得一下。

import "sort"

// 特化的 sort（內建常用型別）
sort.Ints(nums)
sort.Strings(words)
sort.Float64s(prices)

// Slice — 閉包排序
sort.Slice(users, func(i, j int) bool {
    return users[i].Age < users[j].Age
})

// sort.Interface — 完全自訂（底層協定）
type byAge []User
func (a byAge) Len() int           { return len(a) }
func (a byAge) Less(i, j int) bool { return a[i].Age < a[j].Age }
func (a byAge) Swap(i, j int)      { a[i], a[j] = a[j], a[i] }

sort.Sort(byAge(users))

實戰模板

TEMPLATE · 可組合的 iterator 工具庫 
package xiter

import "iter"

// Map 轉換 — 不真的分配 slice，lazy 求值
func Map[T, U any](seq iter.Seq[T], fn func(T) U) iter.Seq[U] {
    return func(yield func(U) bool) {
        for v := range seq {
            if !yield(fn(v)) {
                return
            }
        }
    }
}

// Filter
func Filter[T any](seq iter.Seq[T], keep func(T) bool) iter.Seq[T] {
    return func(yield func(T) bool) {
        for v := range seq {
            if keep(v) && !yield(v) {
                return
            }
        }
    }
}

// Take 只取前 n 個
func Take[T any](seq iter.Seq[T], n int) iter.Seq[T] {
    return func(yield func(T) bool) {
        i := 0
        for v := range seq {
            if i >= n {
                return
            }
            if !yield(v) {
                return
            }
            i++
        }
    }
}

// 使用：Take(Filter(Map(nums, double), isEven), 5)
// 全部 lazy，只在被 range 時才執行
TEMPLATE · Rate Limiter 
import (
    "context"
    "golang.org/x/time/rate"
    "time"
)

func main() {
    // 每秒允許 10 次，burst 20
    limiter := rate.NewLimiter(rate.Every(100*time.Millisecond), 20)

    ctx := context.Background()
    for i := 0; i < 100; i++ {
        if err := limiter.Wait(ctx); err != nil {
            return
        }
        go doWork(i)
    }
}

本章速記

  1. time.Time 比較用 Equal：不要用 ==。
  2. Duration 是 int64 (ns)：永遠寫 5 * time.Second，不寫 5。
  3. 時間格式化用參考時間：2006-01-02 15:04:05。
  4. JSON 用 time.RFC3339：這是跨語言標準。
  5. 解析本地時間用 ParseInLocation：別讓 Go 猜時區。
  6. Timer / Ticker 要 defer Stop：避免 leak。
  7. slices / maps 套件優先：比 sort.Slice 更快且型別安全。
  8. cmp.Compare + cmp.Or：多欄位排序的標準寫法。
  9. iter.Seq / Seq2：Go 1.23+ 的 iterator 型別。
  10. yield 回 false 代表中斷：iterator 要尊重消費者的 break。
  11. slices.Collect(seq)：把 iterator 變回 slice。
  12. iter 生態 lazy by default：Filter/Map 串起來不會分配中間 slice。
