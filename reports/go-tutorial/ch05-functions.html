CHAPTER 05 · 核心語法

函式 Functions

Go 的函式是一等公民：能當變數、能當參數、能當回傳值。多回傳值、defer、closure 是 Go 與其他語言最具辨識度的幾個特色。

關鍵字 · func / return / defer 核心特色 · 多回傳值、具名回傳、變數引數 對照 Python · 沒有 keyword arg，改用 options struct

函式宣告的完整語法

Go 函式語法接近數學風格：func 函式名(參數) 回傳型別。每個參數都要型別，但相同型別可以合併寫。

// 最基本
func add(a int, b int) int {
    return a + b
}

// 相同型別的參數可以合併
func add(a, b int) int {
    return a + b
}

// 無參數、無回傳
func greet() {
    fmt.Println("Hello")
}

// 多種型別的參數
func format(name string, age int, active bool) string {
    return fmt.Sprintf("%s (%d) active=%t", name, age, active)
}
沒有預設值、沒有 keyword argument Python 的 def f(name, age=18, city="Taipei") 在 Go 完全不存在。要模擬這功能請用 Functional Options 模式（本章末尾有範例）或 Config struct。

多回傳值 · Go 的招牌

Go 允許函式同時回傳多個值，這是它處理錯誤的基礎 — 「(結果, error)」的回傳組合出現在幾乎每支 Go 程式。

// 兩個回傳值（超常見）
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("divide by zero")
    }
    return a / b, nil
}

// 呼叫端
result, err := divide(10, 2)
if err != nil {
    return err
}
fmt.Println(result)

// 不要某個回傳值時用 _ 丟棄
result, _ := divide(10, 2)

// 三個以上回傳也可以，但超過 3 個建議改用 struct
func parseCoord(s string) (x, y, z float64, err error) {
    // ...
}

具名回傳值（Named Return Values）

// 在簽章中直接命名回傳值 → 自動宣告為函式內的變數
func split(sum int) (x, y int) {
    x = sum * 4 / 9
    y = sum - x
    return              // 光禿的 return 會回傳 x, y
}

// 搭配 defer 做錯誤包裝（之後章節會大量用到）
func readFile(name string) (data []byte, err error) {
    defer func() {
        if err != nil {
            err = fmt.Errorf("readFile(%s): %w", name, err)
        }
    }()
    // ...
    return os.ReadFile(name)
}
光禿 return 的使用時機 具名回傳 + 光禿 return 只適合短函式（< 10 行）。長函式中濫用會讓人追蹤不到變數從哪裡被賦值。主要用於 defer 包裝 error 的場景。

Variadic · 變數引數

用 ...T 表示「零到多個 T 型別的參數」，在函式內部會得到一個 []T。這是 fmt.Println 能接任意多個參數的秘密。

func sum(nums ...int) int {
    total := 0
    for _, n := range nums {   // nums 是 []int
        total += n
    }
    return total
}

sum()                 // 0
sum(1, 2, 3)          // 6
sum(1, 2, 3, 4, 5)    // 15

// 把既有 slice 展開傳入：加 ...
nums := []int{10, 20, 30}
sum(nums...)           // 60（等同 Python 的 *args）

// variadic 參數必須放最後一個
func log(level string, args ...any) {
    fmt.Printf("[%s] ", level)
    fmt.Println(args...)    // 再把 args 傳給下一個 variadic
}

defer · 延後執行

defer 把語句掛起來，等到外圍函式 return 時才執行。它是 Go 版本的 Python with 敘述 — 保證資源會被釋放。

func readConfig() error {
    f, err := os.Open("config.yaml")
    if err != nil {
        return err
    }
    defer f.Close()      // 不管後面 return 在哪，都會關檔

    // 處理檔案...
    return nil
}

defer 的三大規則

  1. LIFO 順序：多個 defer 以後進先出的順序執行，像堆疊。
  2. 參數立即求值：defer 後面表達式的參數當場被計算，但函式本體要等 return 才跑。
  3. 能修改具名回傳值：透過 closure 存取具名回傳，能實現「AOP」式的攔截。
// 規則 1：LIFO 順序
func main() {
    defer fmt.Println("1")
    defer fmt.Println("2")
    defer fmt.Println("3")
    fmt.Println("main")
}
// 輸出：main → 3 → 2 → 1

// 規則 2：參數立即求值（常見陷阱！）
func main() {
    i := 10
    defer fmt.Println("defer:", i)   // 現在就取 i 的值 10
    i = 20
    fmt.Println("main:", i)
}
// 輸出：main: 20 → defer: 10

// 要延遲求值就用 closure 包起來
defer func() { fmt.Println("defer:", i) }()
// 這樣會印出 20，因為 closure 在 return 時才執行

// 規則 3：修改具名回傳
func f() (result int) {
    defer func() { result *= 2 }()
    return 10   // result = 10, 然後 defer 把它變成 20
}
// 呼叫 f() 得到 20
defer 的慣用情境 檔案關閉、網路連線關閉、mu.Unlock()、資料庫交易 commit/rollback、追蹤函式進出時間。一句話：任何需要「配對」的清理動作。

函式型別 · 一等公民

函式在 Go 中就是一種型別值：可以宣告成變數、放進 map、當作參數傳遞、也能當作回傳值。

// 函式型別可以宣告
type Handler func(req *Request) *Response
type Predicate[T any] func(T) bool

// 函式當變數
var op func(int, int) int
op = func(a, b int) int { return a + b }
fmt.Println(op(3, 4))   // 7

// 函式當參數：filter 的經典實作
func filter(nums []int, keep func(int) bool) []int {
    out := []int{}
    for _, n := range nums {
        if keep(n) {
            out = append(out, n)
        }
    }
    return out
}

evens := filter([]int{1,2,3,4,5}, func(n int) bool {
    return n%2 == 0
})

// 函式當回傳值（返回 closure）
func adder(base int) func(int) int {
    return func(x int) int {
        return base + x
    }
}

add10 := adder(10)
fmt.Println(add10(5))   // 15
fmt.Println(add10(7))   // 17

Closure · 閉包

Closure 是「捕獲了外部變數的匿名函式」。Go 的 closure 捕獲的是變數本身（by reference），不是當下值，這點和 Python 一樣。

// 經典計數器
func counter() func() int {
    count := 0
    return func() int {
        count++          // 每次呼叫都改變同一個 count
        return count
    }
}

c := counter()
fmt.Println(c())   // 1
fmt.Println(c())   // 2
fmt.Println(c())   // 3

// 兩個 counter 彼此獨立
c2 := counter()
fmt.Println(c2())  // 1（不是 4）
並發下要小心 Closure 捕獲的變數是共享的，多個 goroutine 同時呼叫一個 closure 會有 data race。並發計數請用 atomic.Int64（會在 Ch 11 介紹）。

Functional Options · 取代 keyword arg

Go 沒有 keyword argument 也沒有 overload，當一個建構子可能有很多可選參數時，業界標準做法是「Functional Options 模式」。

type Server struct {
    host    string
    port    int
    timeout time.Duration
    logger  *log.Logger
}

// Option 是「會修改 *Server 的函式」
type Option func(*Server)

func WithPort(p int) Option {
    return func(s *Server) { s.port = p }
}
func WithTimeout(d time.Duration) Option {
    return func(s *Server) { s.timeout = d }
}
func WithLogger(l *log.Logger) Option {
    return func(s *Server) { s.logger = l }
}

// 建構子：必填參數在前，options 放最後
func NewServer(host string, opts ...Option) *Server {
    s := &Server{
        host:    host,
        port:    8080,            // 預設值
        timeout: 30 * time.Second,
    }
    for _, opt := range opts {
        opt(s)
    }
    return s
}

// 使用：讀起來像 keyword arg
s := NewServer("localhost",
    WithPort(9000),
    WithTimeout(5*time.Second),
)
Python 對照 Python 的 Server(host, port=8080, timeout=30) 在 Go 寫起來變成 NewServer(host, WithPort(8080), WithTimeout(30*time.Second))。看起來囉唆但極有彈性：新增選項不會破壞現有呼叫點。

本章速記

  1. func 的標準格式：func Name(a, b T1, c T2) (R1, R2, error) { ... }。
  2. 多回傳值：回傳 (result, error) 是 Go 最重要的慣例，99% 會失敗的函式都這樣回傳。
  3. 具名回傳：短函式可用，長函式避免；主要搭配 defer 做錯誤包裝。
  4. variadic 用 ...T：呼叫端要展開 slice 用 slice...。
  5. defer 是 LIFO：最後 defer 的最先執行；參數是立即求值，函式本體才延後。
  6. defer 必配對出現：f.Close()、mu.Unlock()、tx.Rollback() 的搭檔。
  7. 函式是一等公民：當變數、當參數、當回傳值，毫無限制。
  8. Closure 捕獲的是變數本身，不是當下值 — 並發情境要特別小心。
  9. 沒有 keyword argument：用 Functional Options 或 Config struct 模擬。
  10. 沒有函式重載（overload）：同 package 中函式名必須唯一。
