泛型 Generics

Go 1.18 加入泛型，Go 1.26 支援自我引用的 type constraint 與更靈活的推論。比 Java/C# 更克制，比 C++ template 簡單 — Go 的泛型設計重點是「降低重複」而不是「抽象狂歡」。

關鍵語法 · [T any] · ~T · any / comparable 核心套件 · slices · maps · cmp

泛型函式的基本語法

在函式名後加上 [T constraint] 即可。T 是型別參數（慣例大寫單字母），constraint 描述 T 必須滿足什麼條件。

// 最簡單的泛型函式 — any 是最寬鬆的 constraint
func Print[T any](v T) {
    fmt.Println(v)
}

// 呼叫：通常不用明寫型別參數（自動推論）
Print(42)              // T = int
Print("hello")         // T = string
Print[int](42)         // 也能明寫

// 多個型別參數
func Map[T, U any](s []T, fn func(T) U) []U {
    r := make([]U, len(s))
    for i, v := range s {
        r[i] = fn(v)
    }
    return r
}

doubled := Map([]int{1, 2, 3}, func(x int) int { return x * 2 })
strs := Map([]int{1, 2}, strconv.Itoa)
T 只是慣例 你可以取任何名字（K、V、E、Item、Container…）。常見慣例：
T 泛用、K/V map 的 key/value、E 容器元素、S slice 型別。

Constraint · 型別約束

Constraint 本質就是「介面」 — 但介面的意義擴充了：不只是方法集合，還能描述「允許哪些具體型別」。

三種內建 constraint

Constraint允許的型別
any所有型別（= interface{}）
comparable可用 == / != 比較的型別（int、string、bool、指標、array、struct 中全是 comparable 欄位…）
cmp.Ordered (Go 1.21+)可用 < / > 排序的型別（所有數值 + string）
 
// comparable 讓你可以用 == 比較 T 的值
func Contains[T comparable](s []T, v T) bool {
    for _, e := range s {
        if e == v { return true }
    }
    return false
}

// cmp.Ordered 允許排序比較
import "cmp"

func Min[T cmp.Ordered](a, b T) T {
    if a < b { return a }
    return b
}

Min(3, 5)         // 3
Min("apple", "banana")   // "apple"

自訂 constraint

// 型別聯集（type union）：用 | 列出允許的具體型別
type Number interface {
    int | int32 | int64 | float32 | float64
}

func Sum[T Number](s []T) T {
    var total T
    for _, v := range s {
        total += v
    }
    return total
}

// ~T 代表「T 本身，或底層型別是 T 的自訂型別」
type Integer interface {
    ~int | ~int32 | ~int64
}

type UserID int64
// ✗ 沒有 ~ 的話 UserID 不符合 Number（但底層是 int64）
// ✓ 有 ~ 的 Integer 接受 UserID
~ 符號記憶法 ~T 意思是「T 或任何 underlying type 是 T 的型別」。幾乎所有接受具體型別的 constraint 都應該用 ~，這樣使用者能傳入自己的命名型別。

泛型型別（Generic Types）

不只函式能泛型，struct、map、slice alias、channel 也都能。

// 泛型 struct
type Stack[T any] struct {
    items []T
}

func (s *Stack[T]) Push(v T) {
    s.items = append(s.items, v)
}

func (s *Stack[T]) Pop() (T, bool) {
    var zero T
    if len(s.items) == 0 {
        return zero, false
    }
    last := len(s.items) - 1
    v := s.items[last]
    s.items = s.items[:last]
    return v, true
}

// 使用
s := &Stack[int]{}
s.Push(1)
s.Push(2)
v, _ := s.Pop()   // 2

// 多型別參數的 struct
type Pair[K comparable, V any] struct {
    Key   K
    Value V
}

p := Pair[string, int]{Key: "age", Value: 30}
方法不能額外引入型別參數 方法只能用 struct 層級宣告的 T，不能再宣告新的型別參數。例如無法寫 func (s *Stack[T]) Convert[U any]() ...。要多型別參數必須拆成頂層函式。

Go 1.26：自我引用的泛型

Go 1.26 之前，如果你想要「這個型別的方法必須回傳自己」就得繞好幾圈。1.26 完整支援 self-referential type constraint，讓 Clone、Builder、Fluent API 等模式乾淨很多。

// Go 1.26+：T 參數可以「引用自己」的型別
type Cloneable[T any] interface {
    Clone() T
}

// 任何實作 Clone() Self 的型別都符合
func CloneAll[T Cloneable[T]](items []T) []T {
    out := make([]T, len(items))
    for i, v := range items {
        out[i] = v.Clone()
    }
    return out
}

type Node struct{ Val int }

func (n Node) Clone() Node {
    return Node{Val: n.Val}
}

cloned := CloneAll([]Node{{Val: 1}, {Val: 2}})
// 型別推論成功，不用手寫 CloneAll[Node](...)
實務影響 原本在 Go 1.25 以前，這種 self-referential 寫法要嘛編譯失敗、要嘛推論不出來逼你顯式標型別。Go 1.26 讓 cmp.Ordered、builder pattern、immutable data structure 的泛型實作真正可用。

何時該用泛型？何時不該？

Go 官方在 《When To Use Generics》給出明確建議。

該用

① 操作容器型別

slice、map、channel 的通用操作：Filter、Map、Reduce、Keys、Clone…。

② 通用資料結構

Stack、Queue、Tree、LinkedList、LRU Cache…型別不該被寫死。

③ 數學運算

Sum、Average、Abs 這類需要「任何數值型別」的函式。

④ 減少反覆樣板

替代 func(a any) any 這種靠 type assertion 的拙劣寫法。

不該用

× 介面就夠用

若 T 只要求實作某方法，直接用介面更簡潔。func f(r io.Reader) 不需要變成泛型。

× 只為了「或許以後用得到」

YAGNI 原則在 Go 更嚴格：真的有多型別使用場景才寫泛型。

× 型別本身就是介面的意圖

「我想接受任何能格式化的東西」→ 用 fmt.Stringer，不用 [T Stringer]。

× 實作邏輯會因型別不同

若要針對不同型別做不同事，那不是泛型 — 是 interface + 多型。

標準庫中的泛型 API

Go 1.21 起，標準庫大量採用泛型 — 這些是你該先熟悉的套件。

套件主要函式
slicesSort, Contains, Index, Max, Min, Clone, Delete, Insert, Reverse…
mapsKeys, Values, Clone, Copy, DeleteFunc, Equal…
cmpCompare, Less, Or, Ordered (constraint)
sync/atomicPointer[T]
syncOnceValue[T], OnceValues[T1, T2]
 
import (
    "cmp"
    "slices"
    "maps"
)

// slices
nums := []int{3, 1, 4, 1, 5, 9}
slices.Sort(nums)                    // [1, 1, 3, 4, 5, 9]
slices.Contains(nums, 4)             // true
var max = slices.Max(nums)          // 9

// cmp — 用於自訂比較
users := []User{...}
slices.SortFunc(users, func(a, b User) int {
    return cmp.Compare(a.Age, b.Age)
})

// maps
m := map[string]int{"a": 1, "b": 2}
copy := maps.Clone(m)
keys := slices.Collect(maps.Keys(m))   // Go 1.23 的 iter + collect

實戰模板

TEMPLATE · 通用集合操作 
package collect

import "cmp"

// Filter 過濾
func Filter[T any](s []T, keep func(T) bool) []T {
    out := make([]T, 0, len(s))
    for _, v := range s {
        if keep(v) {
            out = append(out, v)
        }
    }
    return out
}

// Map 轉換
func Map[T, U any](s []T, fn func(T) U) []U {
    out := make([]U, len(s))
    for i, v := range s {
        out[i] = fn(v)
    }
    return out
}

// Reduce 摺疊
func Reduce[T, U any](s []T, init U, fn func(U, T) U) U {
    acc := init
    for _, v := range s {
        acc = fn(acc, v)
    }
    return acc
}

// GroupBy 分組
func GroupBy[T any, K comparable](s []T, key func(T) K) map[K][]T {
    out := make(map[K][]T)
    for _, v := range s {
        k := key(v)
        out[k] = append(out[k], v)
    }
    return out
}

// MinBy / MaxBy
func MinBy[T any, U cmp.Ordered](s []T, key func(T) U) T {
    best := s[0]
    bestKey := key(best)
    for _, v := range s[1:] {
        if k := key(v); k < bestKey {
            best, bestKey = v, k
        }
    }
    return best
}
TEMPLATE · 型別安全的 Result 型別 
// 模仿 Rust 的 Result<T, E>（Go 通常不這樣寫，但示範語法）
type Result[T any] struct {
    value T
    err   error
}

func Ok[T any](v T) Result[T]       { return Result[T]{value: v} }
func Err[T any](e error) Result[T]  { return Result[T]{err: e} }

func (r Result[T]) Unwrap() (T, error) {
    return r.value, r.err
}

func (r Result[T]) Map(fn func(T) T) Result[T] {
    if r.err != nil {
        return r
    }
    return Ok(fn(r.value))
}

// 使用
r := Ok(10).Map(func(x int) int { return x * 2 })
v, err := r.Unwrap()   // 20, nil

本章速記

  1. func Name[T constraint](...)：型別參數寫在函式名後。
  2. constraint 就是介面：允許方法集、型別聯集、或兩者混合。
  3. any / comparable 是內建：cmp.Ordered 需要 import。
  4. ~T 允許底層型別：讓使用者的 named type 也能用。
  5. 型別推論聰明：多數情況不用顯式寫 [T]。
  6. struct / type 也能泛型：type Stack[T any] struct { ... }。
  7. 方法不能加新型別參數：只能用 struct 層級定義的。
  8. Go 1.26 支援自我引用：[T Cloneable[T]] 終於能用。
  9. 別為了泛型而泛型：介面夠用就不要泛型。
  10. 先看標準庫：slices / maps / cmp 已涵蓋 80% 需求。
  11. 用 var zero T 取零值：泛型函式中無法寫 nil 或 0。
  12. T 慣例命名：T 通用、K/V map、E 元素、S slice。
