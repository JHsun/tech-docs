CHAPTER 16 · 標準庫與實務

字串 & 編碼

Go 字串是 UTF-8 的 byte 序列、不可變、天生支援多語言。strings / bytes / strconv / encoding/json 構成最常用的文字處理工具鏈。

核心套件 · strings · bytes · strconv · unicode/utf8 · encoding/json

字串是 byte，不是字元

Go 字串是不可變的 []byte 的唯讀視圖，內容通常是 UTF-8 編碼。len 回傳的是 byte 數，不是字元數。這是初學者最常踩的雷。

s := "你好 world"

len(s)              // 12（"你好" 各 3 bytes，空格 1，"world" 5）
len([]rune(s))      // 8（真正的字元數）

// 用 index 會拿到 byte，不是字元！
s[0]                // 228（"你" 的第一個 byte）

// for range 正確地走訪 rune（Unicode code point）
for i, r := range s {
    fmt.Printf("%d: %c\n", i, r)
}
// 0: 你  (i 是 byte index，不是字元 index)
// 3: 好
// 6:   (空格)
// 7: w
// ...
byte / rune / string byte 是 uint8 的別名 — 一個 byte。
rune 是 int32 的別名 — 一個 Unicode code point。
string 是 UTF-8 bytes 的不可變序列。
三者互轉：[]byte(s)、[]rune(s)、string(bs)、string(runes)。
字串拼接效能 迴圈中用 s += x 會重複分配 — 每次拼接都建立新字串。正確做法：strings.Builder。

strings.Builder · 高效拼接

var b strings.Builder
b.Grow(1024)           // 可選：預先分配減少擴容

for _, w := range words {
    b.WriteString(w)
    b.WriteByte(' ')
}
result := b.String()     // 最後取出字串（這步不分配）

// 實測：拼接 1000 個字串
// s += x               ≈ 500 µs（重複分配）
// Builder              ≈ 15  µs（一次分配）
對照 Python Python 的 "".join(list) ≈ Go 的 strings.Join(ss, "")。若字串是迴圈中逐步組出來的，Python 的 io.StringIO ≈ Go 的 strings.Builder。

bytes 套件：與 strings 幾乎鏡像

bytes 的 API 幾乎和 strings 一樣 — 只是操作 []byte。處理 io.Reader 讀進來的原始資料時常用。

import "bytes"

bytes.Contains(data, []byte("error"))
bytes.Split(data, []byte("\n"))
bytes.TrimSpace(data)

// bytes.Buffer — 可讀寫的 byte 緩衝，既是 Reader 也是 Writer
var buf bytes.Buffer
buf.WriteString("hello ")
fmt.Fprintf(&buf, "world %d", 42)
fmt.Println(buf.String())     // "hello world 42"

// 讀取（Buffer 也是 io.Reader）
io.Copy(os.Stdout, &buf)

strconv · 字串與數字的轉換

Go 沒有 Python 的「自動轉型」— string 轉 int 必須明寫。

import "strconv"

// string → 數字
n, err := strconv.Atoi("42")           // int
n, err := strconv.ParseInt("ff", 16, 64)  // 16 進位，int64
f, err := strconv.ParseFloat("3.14", 64)   // float64
b, err := strconv.ParseBool("true")         // true/false/1/0/TRUE…

// 數字 → string
s := strconv.Itoa(42)                     // "42"
s := strconv.FormatInt(255, 16)              // "ff"
s := strconv.FormatFloat(3.14, 'f', 2, 64)  // "3.14"
s := strconv.FormatBool(true)                 // "true"

// 字串加引號（用於產生 JSON-like 輸出）
strconv.Quote("he said \"hi\"")               // "\"he said \\\"hi\\\"\""

fmt 也能做，但 strconv 更快

// 兩種都行，熱路徑用 strconv
fmt.Sprintf("%d", n)    // 慢（要過 reflect）
strconv.Itoa(n)          // 快 3-5 倍

unicode & utf8

import (
    "unicode"
    "unicode/utf8"
)

// unicode — rune 的分類判斷
unicode.IsDigit('3')          // true
unicode.IsLetter('中')        // true（支援全部 Unicode 字母）
unicode.IsSpace(' ')          // true
unicode.IsPunct('!')          // true
unicode.ToLower('A')          // 'a'

// utf8 — UTF-8 編碼的細節
utf8.RuneCountInString("你好")  // 2（正確的字元數）
utf8.ValidString("你好")        // true
utf8.RuneLen('你')                // 3（這個 rune 佔幾個 bytes）

// 逐個 rune 取出（避免轉成 []rune 的記憶體分配）
for i := 0; i < len(s); {
    r, size := utf8.DecodeRuneInString(s[i:])
    fmt.Printf("%c\n", r)
    i += size
}

regexp · 正規表達式

Go 用 RE2 語法（與 Python re 類似但沒有 backreference，換來保證線性時間複雜度）。

import "regexp"

// 編譯：在 package init 或全域變數編譯一次，不要每次呼叫都編
var emailRe = regexp.MustCompile(`\b[\w.-]+@[\w.-]+\.\w+\b`)

// 比對
emailRe.MatchString("a@b.com")                // true

// 搜尋
emailRe.FindString("聯絡 a@b.com 謝謝")         // "a@b.com"
emailRe.FindAllString(text, -1)                 // 全部

// 取代
emailRe.ReplaceAllString(text, "[EMAIL]")

// 分組（具名組用 (?P<name>...)）
re := regexp.MustCompile(`(?P<year>\d{4})-(?P<month>\d{2})`)
m := re.FindStringSubmatch("2026-04")
fmt.Println(m[re.SubexpIndex("year")])        // "2026"
別在熱路徑 Compile regexp.MustCompile 很貴 — 編譯一次就夠，存到全域 var 或 sync.Once。每次 handler 都編譯會讓 TPS 掉一個數量級。

encoding/json · 最常用的編碼套件

Marshal / Unmarshal

import "encoding/json"

type User struct {
    ID       int       `json:"id"`
    Name     string    `json:"name"`
    Email    string    `json:"email,omitempty"`    // 空值不輸出
    Password string    `json:"-"`                    // 完全不編碼
    Age      *int      `json:"age,omitempty"`      // 指標可區分「0 / 未設定」
    Created  time.Time `json:"created_at"`           // 預設 RFC3339
}

// Marshal：struct → []byte
data, err := json.Marshal(u)
data, err := json.MarshalIndent(u, "", "  ")   // pretty print

// Unmarshal：[]byte → struct
var u User
err := json.Unmarshal(data, &u)           // 記得傳指標

動態 JSON（型別未知時）

// 不知道結構時用 map 或 any
var data map[string]any
json.Unmarshal(bs, &data)
// data["key"] 的型別是 any，要 type assertion
if name, ok := data["name"].(string); ok {
    fmt.Println(name)
}

// json.RawMessage — 延遲解析
type Envelope struct {
    Type string          `json:"type"`
    Data json.RawMessage `json:"data"`   // 先保留原始 bytes
}
var env Envelope
json.Unmarshal(bs, &env)
switch env.Type {
case "user":
    var u User
    json.Unmarshal(env.Data, &u)
case "order":
    var o Order
    json.Unmarshal(env.Data, &o)
}
自訂序列化 實作 MarshalJSON() ([]byte, error) 或 UnmarshalJSON([]byte) error 就能完全控制編碼行為 — 例如把 time.Time 輸出為 Unix timestamp、把 enum 型別輸出為字串。

實戰模板

TEMPLATE · 嚴格 JSON 解碼 
// 預設 json.Unmarshal 會「忽略」未知欄位 — 這通常是 bug
// 正確做法：嚴格模式
func strictDecode(r io.Reader, v any) error {
    dec := json.NewDecoder(r)
    dec.DisallowUnknownFields()   // 遇未知欄位直接報錯

    if err := dec.Decode(v); err != nil {
        return err
    }

    // 確認只有一個頂層值（防止 JSON 後面還藏東西）
    if dec.More() {
        return fmt.Errorf("extra JSON content")
    }
    return nil
}
TEMPLATE · 自訂 time.Time 序列化 
type UnixTime time.Time

func (t UnixTime) MarshalJSON() ([]byte, error) {
    return []byte(strconv.FormatInt(time.Time(t).Unix(), 10)), nil
}

func (t *UnixTime) UnmarshalJSON(data []byte) error {
    sec, err := strconv.ParseInt(string(data), 10, 64)
    if err != nil {
        return err
    }
    *t = UnixTime(time.Unix(sec, 0))
    return nil
}

// 使用
type Event struct {
    Time UnixTime `json:"t"`
}
// 輸出：{"t": 1713290400}

本章速記

  1. string 是 byte 序列：len 算 byte，[]rune 才算字元。
  2. for range string 給 rune：index 是 byte offset。
  3. 迴圈拼字串用 strings.Builder：比 += 快一個數量級。
  4. bytes 套件鏡像 strings：操作 []byte 時用。
  5. bytes.Buffer 既是 Reader 也是 Writer：記憶體裡的萬用管道。
  6. strconv 比 fmt 快：熱路徑轉換用 strconv。
  7. regexp.MustCompile 存全域：別在每次呼叫時編譯。
  8. json tag omitempty 空值不輸出，- 完全忽略。
  9. 可選欄位用 *T：指標可區分「零值」與「未設定」。
  10. NewDecoder / NewEncoder 處理 stream；一次性用 Marshal / Unmarshal。
  11. json.RawMessage：延遲解析 + 決定分支的利器。
  12. DisallowUnknownFields：API 嚴格模式。
