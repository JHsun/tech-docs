grafana.ini 完整配置：從 server 到 SMTP 的生產級指南

§ 1. 設定優先序：env > ini > 預設

1.1 三層覆蓋規則

Grafana 設定來源由低到高：

  1. Defaults：程式內建
  2. /usr/share/grafana/conf/defaults.ini：唯讀的官方預設（不要改這個）
  3. /etc/grafana/grafana.ini：您的自訂設定（修改這個）
  4. 環境變數：GF_<SECTION>_<KEY> 最高優先

1.2 環境變數命名規則

規則：GF_ + section 名（大寫底線）+ _ + key 名（大寫底線）

grafana.ini環境變數
[server] → http_portGF_SERVER_HTTP_PORT
[security] → admin_passwordGF_SECURITY_ADMIN_PASSWORD
[auth.github] → client_idGF_AUTH_GITHUB_CLIENT_ID
[users] → allow_sign_upGF_USERS_ALLOW_SIGN_UP

1.3 從檔案讀值（更安全）

密碼、API key 等敏感值不應寫在 ini 或環境變數，用 $__file{} 從檔案讀：

[database]
password = $__file{/etc/grafana/secrets/db-password}

[smtp]
password = $__file{/run/secrets/smtp-password}

K8s 場景：把 secret 掛成 file。

1.4 看「實際生效」的設定

# Web UI（admin role）
Administration → General → Settings

# 或 API
curl -u admin:password http://localhost:3000/api/admin/settings | jq

這個 API 顯示合併後的最終值，包含環境變數覆蓋的部分。Debug 設定問題時超有用。

§ 2. [server] Section

[server]
# 監聽協議
protocol = http              # 或 https / h2 / socket

# 監聽 IP（0.0.0.0 = 全部介面）
http_addr = 0.0.0.0

# 監聽 port
http_port = 3000

# === 重要！對外的完整 URL ===
# 影響：通知連結、OAuth callback、share 連結
domain = grafana.example.com
root_url = https://grafana.example.com
# 若有 sub-path：
# root_url = https://example.com/grafana/
# serve_from_sub_path = true

# HTTPS（直接由 Grafana 處理 TLS）
cert_file = /etc/grafana/ssl/cert.pem
cert_key  = /etc/grafana/ssl/key.pem
# 但通常 production 會用 reverse proxy（NGINX/Caddy）處理 TLS

# Reverse proxy 場景
# 設這個讓 Grafana 信任 X-Forwarded-* headers
enforce_domain = false

# 路由
router_logging = false       # debug 時開
static_root_path = public

# Read / Write timeout
read_timeout = 0             # 0 = 無限制（不建議 production）
⚠️ root_url 是 production 最重要的設定

若 root_url 沒設正確，通知 email、Slack 訊息中的連結會錯，OAuth callback 失敗，share dashboard 跳到錯誤位址。
第一個生產設定就該檢查它。

§ 3. [database] Section

3.1 SQLite（預設）

[database]
type = sqlite3
path = grafana.db          # 相對於 [paths] data
cache_mode = private

# WAL（write-ahead log）模式提升並發
wal = true

3.2 PostgreSQL（推薦 production）

[database]
type = postgres
host = postgres.example.com:5432
name = grafana
user = grafana
password = $__file{/etc/grafana/secrets/db-password}

# SSL 設定
ssl_mode = require           # disable / require / verify-ca / verify-full
ca_cert_path = /etc/grafana/ssl/ca.pem
client_key_path = /etc/grafana/ssl/client-key.pem
client_cert_path = /etc/grafana/ssl/client-cert.pem

# 連線池
max_idle_conn = 2
max_open_conn = 0            # 0 = 無限制
conn_max_lifetime = 14400    # 秒

3.3 MySQL

[database]
type = mysql
host = mysql.example.com:3306
name = grafana
user = grafana
password = $__file{/etc/grafana/secrets/db-password}

ssl_mode = true              # 或 skip-verify / true / false
server_cert_name = mysql.example.com

3.4 PostgreSQL DB 初始化

-- 在 PostgreSQL 上執行
CREATE USER grafana WITH PASSWORD 'StrongPassword';
CREATE DATABASE grafana OWNER grafana;
GRANT ALL PRIVILEGES ON DATABASE grafana TO grafana;

-- Grafana 啟動時會自動建表

3.5 何時必用外部 DB

  • HA 部署（多副本）— SQLite 會破損
  • 需要高可用 backup（PG/MySQL 有完善備份工具）
  • 大量 dashboard / user（> 10000 筆 record）
  • 需要符合企業合規（外部 audit 工具）

§ 4. [security] Section

[security]
# 預設 admin
admin_user = admin
admin_password = $__file{/run/secrets/admin-password}
admin_email = admin@example.com

# Secret key（用於 cookie、session sign）
# 必設！production 一定要改。產生方法：openssl rand -base64 32
secret_key = SW2YcwTIb9zpOOhoPsMm

# Cookie 安全
cookie_secure = true         # HTTPS 才送 cookie
cookie_samesite = lax        # strict / lax / none
disable_gravatar = true      # 不從 gravatar.com 拉頭像（隱私）

# 內容安全策略
content_security_policy = true
content_security_policy_template = """default-src 'self'; ..."""

# 強制 HTTPS（reverse proxy 後不需要）
strict_transport_security = false
strict_transport_security_max_age_seconds = 86400

# 嵌入控制
allow_embedding = false      # 不允許在 iframe 中嵌入

# 密碼策略
disable_brute_force_login_protection = false
login_maximum_inactive_lifetime_duration = 7d   # 7 天沒登入則 logout
login_maximum_lifetime_duration = 30d           # 30 天必重 login

# Encryption（v9+）
# 用於加密儲存在 DB 中的敏感資料（datasource password 等）
encryption_provider = secretKey.v1
data_keys_cache_ttl = 15m
🚨 secret_key 是 Grafana 安全的根

secret_key 用於加密儲存的 datasource password、API key、cookie sign。
絕對不能用預設值 SW2YcwTIb9zpOOhoPsMm（這是 example 不是安全的）。
Production 設定生成法：openssl rand -base64 32。
改了 secret_key 之後，所有舊的加密資料（如 datasource password）需要 re-encrypt（用 grafana-cli encryption rotate-data-encryption-keys）。

§ 5. [users] 與 [auth]

5.1 [users]

[users]
# === 註冊與權限預設 ===
allow_sign_up = false              # production 必設 false
allow_org_create = false           # 一般使用者不能建 org
auto_assign_org = true             # 自動加入 main org
auto_assign_org_id = 1
auto_assign_org_role = Viewer      # 新使用者預設角色

# === 隱藏的權限 ===
viewers_can_edit = false           # Viewer 能不能用 Edit 模式探索
editors_can_admin = false          # Editor 能不能管理 Folder

# === 介面 ===
default_theme = system             # light / dark / system
default_language = en
default_timezone = browser

# === 使用者設定 ===
external_manage_link_url =         # 外部使用者管理 URL（顯示在 profile）
external_manage_link_name =
external_manage_info =

# 隱藏 version footer
hidden_users = ""

5.2 [auth]

[auth]
# 全域控制
disable_login_form = false         # 啟用 SSO 後可設 true
disable_signout_menu = false
oauth_auto_login = false           # 設 true 自動跳到 OAuth provider
oauth_skip_org_role_update_sync = false

# Session
login_cookie_name = grafana_session
login_maximum_inactive_lifetime_duration = 7d
login_maximum_lifetime_duration = 30d
token_rotation_interval_minutes = 10

# Sign out 跳轉
signout_redirect_url =

# === API key 過期 ===
api_key_max_seconds_to_live = -1   # -1 = 不限制（不推薦）。建議設 90d

5.3 [auth.anonymous] 匿名訪問

[auth.anonymous]
enabled = false                    # production 預設 false
org_name = Main Org.
org_role = Viewer
hide_version = true

有些情境會開（如公開的 status dashboard）。Grafana 有「Public Dashboards」（v10+）功能更安全的替代方案。

§ 6. [smtp] Email 設定

Grafana Alerting 的 email receiver 與 invite email 都用 SMTP。

[smtp]
enabled = true
host = smtp.gmail.com:587
user = alertmanager@example.com
password = $__file{/etc/grafana/secrets/smtp-password}

# 寄件者
from_address = grafana@example.com
from_name = Grafana

# TLS
skip_verify = false
startTLS_policy =                  # OpportunisticStartTLS / MandatoryStartTLS / NoStartTLS

# Auth method
auth_type = LOGIN                  # 或 PLAIN, CRAM-MD5

# === 進階 ===
ehlo_identity = grafana.example.com
cert_file =
key_file =

6.1 各家 SMTP 範例

# Gmail（需 App Password）
host = smtp.gmail.com:587
user = your-account@gmail.com
password = xxxx-xxxx-xxxx-xxxx     # App Password

# AWS SES
host = email-smtp.us-east-1.amazonaws.com:587
user = AKIAI...                    # SES SMTP credentials
password = ...

# SendGrid
host = smtp.sendgrid.net:587
user = apikey
password = SG.xxxxxxxx

# 內部 SMTP（無認證）
host = mail-relay.internal:25
user =
password =
skip_verify = true                 # 內部 cert

6.2 測試 SMTP

# UI: Alerting → Contact Points → 建一個 Email contact point → Test
# 或：
curl -X POST http://localhost:3000/api/alerts/test \
  -H "Content-Type: application/json" \
  -u admin:password \
  -d '{...}'

§ 7. [log] 日誌設定

[log]
# 模式：console / file / syslog（可多個 space 分隔）
mode = console file

# Log level：debug / info / warn / error / critical
level = info

# 個別 logger 控制
filters = alerting.notifier:debug rendering:warn

[log.console]
level =
format = console                   # console / text / json

[log.file]
level =
format = text
log_rotate = true
max_lines = 1000000
max_size_shift = 28                # 256 MB
daily_rotate = true
max_days = 7

[log.syslog]
level =
format = text
network =
address =
facility = local7
tag =

7.1 結構化 JSON log（推薦給日誌系統）

[log.console]
format = json

輸出範例：

{"t":"2026-05-02T10:23:45.123Z","level":"info","msg":"User logged in","user":"alice","login":"alice@example.com"}

Loki 等 log 系統可直接解析。

7.2 看 specific subsystem 的 log

啟動時加 --config.target.alerting.notifier=debug 或在 ini：

[log]
filters = alerting:debug oauth:debug ldap:debug

常用的 logger 名稱：alerting / oauth / ldap / rendering / datasource / sql / api。

§ 8. [analytics] 與隱私

[analytics]
# 上傳使用統計給 Grafana Labs（隱私敏感的關掉）
reporting_enabled = false

# 檢查新版本（給 admin 看「有新版本」提示）
check_for_updates = false
check_for_plugin_updates = false

# Application Insights / Google Analytics（給管理介面）
google_analytics_ua_id =
google_tag_manager_id =
rudderstack_write_key =
rudderstack_data_plane_url =

# Feedback links（讓使用者送 feedback）
feedback_links_enabled = true

合規 / 隱私敏感的環境（金融、醫療、政府），建議全部關掉。

§ 9. [unified_alerting] 與 [alerting]

9.1 Grafana 8+ 的 Unified Alerting（預設）

[unified_alerting]
enabled = true                      # 啟用 Grafana-managed alerting

# 評估間隔（rule 不能設更短）
min_interval = 10s

# 在 K8s HA 場景必設，標識當前 instance
ha_listen_address = "${POD_IP}:9094"
ha_advertise_address =
ha_peers =                          # 其他 Grafana 副本：grafana-1:9094,grafana-2:9094
ha_peer_timeout = 15s
ha_gossip_interval = 200ms

# 資料保留
max_attempts = 3
execute_alerts = true

# Notification log
notification_log_retention = 5d

9.2 Legacy Alerting（v7-）

[alerting]
# 已廢棄。新環境不要碰
enabled = false                     # 預設 false（v9+ 完全移除）

若您升級舊 Grafana，會看到「Migrate to Unified Alerting」提示。Ch 40 Grafana Alerting 詳述。

§ 10. Provisioning 設定檔結構

Provisioning 讓您「用設定檔管理 datasource / dashboard / alerting / plugin」，不用點 UI。

10.1 目錄結構

/etc/grafana/provisioning/
├── datasources/
│   ├── prometheus.yml
│   └── loki.yml
├── dashboards/
│   └── default.yml
├── alerting/
│   ├── rules.yml
│   ├── contact-points.yml
│   ├── policies.yml
│   └── templates.yml
├── notifiers/                  # legacy alerting，新版用 alerting/
└── plugins/
    └── apps.yml

10.2 Datasources 範例

# /etc/grafana/provisioning/datasources/all.yml
apiVersion: 1

datasources:
  - name: Prometheus
    type: prometheus
    access: proxy
    url: http://prometheus:9090
    isDefault: true
    editable: false
    jsonData:
      httpMethod: POST
      manageAlerts: true
      prometheusType: Prometheus
      prometheusVersion: 3.11.2
      cacheLevel: High
      timeInterval: 15s

  - name: Loki
    type: loki
    access: proxy
    url: http://loki:3100
    editable: false
    jsonData:
      maxLines: 1000
      derivedFields:
        - datasourceUid: tempo-uid
          matcherRegex: 'trace_id=(\w+)'
          name: TraceID
          url: '$${__value.raw}'

10.3 Dashboards Provider

# /etc/grafana/provisioning/dashboards/default.yml
apiVersion: 1

providers:
  - name: 'default'
    orgId: 1
    folder: ''                     # 根 folder 或指定 folder name
    type: file
    disableDeletion: false
    editable: true
    updateIntervalSeconds: 30      # 多久 rescan dashboard 檔案
    allowUiUpdates: true           # 允許 UI 改後寫回檔案
    options:
      path: /var/lib/grafana/dashboards/default
      foldersFromFilesStructure: true

把 dashboard JSON 放 /var/lib/grafana/dashboards/default/*.json，Grafana 會自動載入。

10.4 Alerting Rules

# /etc/grafana/provisioning/alerting/rules.yml
apiVersion: 1

groups:
  - orgId: 1
    name: production-alerts
    folder: Production
    interval: 1m
    rules:
      - uid: high-error-rate
        title: HighErrorRate
        condition: A
        data:
          - refId: A
            datasourceUid: prometheus-uid
            model:
              expr: 'rate(http_requests_total{status=~"5.."}[5m]) > 0.05'
        # ...

10.5 Contact Points + Notification Policies

# /etc/grafana/provisioning/alerting/contact-points.yml
apiVersion: 1

contactPoints:
  - orgId: 1
    name: slack-alerts
    receivers:
      - uid: slack-1
        type: slack
        settings:
          url: https://hooks.slack.com/services/XXX/YYY/ZZZ
          username: Grafana
          title: '{{ template "default.title" . }}'
          text: '{{ template "default.message" . }}'

11.1 對應的 K8s Secret

  name: grafana-secrets
  namespace: monitoring

  db-password: ChangeMe-DB-Pass
  admin-password: ChangeMe-Admin-Pass
  smtp-password: ChangeMe-SMTP-Pass
  secret-key: GENERATED-WITH-openssl-rand-base64-32

11.2 對應的 PodSpec

spec:
  containers:
    - name: grafana
      image: grafana/grafana:12.4.0
      env:
        - name: POD_IP
          valueFrom:
            fieldRef:
              fieldPath: status.podIP
      volumeMounts:
        - name: config
          mountPath: /etc/grafana
        - name: secrets
          mountPath: /etc/grafana/secrets
          readOnly: true
        - name: data
          mountPath: /var/lib/grafana
  volumes:
    - name: secrets
      secret:
        secretName: grafana-secrets

✅ 本章重點摘要

30 秒回顧
  • 三層覆蓋：defaults.ini → grafana.ini → 環境變數，後者覆蓋前者
  • 環境變數命名：GF_<SECTION>_<KEY>
  • 敏感值用 $__file{} 從檔案讀，不寫在 ini 或 env
  • [server]：root_url 是最重要的設定，影響通知連結與 OAuth callback
  • [database]：HA 必用 PostgreSQL 或 MySQL，SQLite 只給單機
  • [security]：secret_key 必改（用 openssl rand 產生），加密 datasource password 等
  • [users]：production 必設 allow_sign_up = false、allow_org_create = false
  • [auth.anonymous]：production 預設 false
  • [smtp]：給 alert email 與 invite 用，Gmail 要 App Password
  • [log]：format = json 才能被 Loki 等系統解析
  • [analytics]：合規環境全關
  • Provisioning 5 大目錄：datasources / dashboards / alerting / notifiers / plugins

  2. 請寫出對應的環境變數：[auth.github] 的 client_secret。
  3. 解釋為什麼 secret_key 改了之後需要 re-encrypt？舊的 datasource password 會發生什麼事？
  4. 設計一個 Production 的 grafana.ini：用 PostgreSQL 外部 DB、SES 寄信、JSON log、Unified Alerting。
  5. 解釋 Provisioning 的 editable: false 與 allowUiUpdates: true 的差別。各自的使用情境？
