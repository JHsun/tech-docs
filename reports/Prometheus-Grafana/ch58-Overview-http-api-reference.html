§ 1. Prometheus HTTP API

1.1 概覽

Prometheus HTTP API 在 /api/v1/*（v1，穩定）。預設 port 9090。 回應一律 JSON 格式： 

{
  "status": "success",
  "data": { ... }
}

// 錯誤
{
  "status": "error",
  "errorType": "bad_data",
  "error": "invalid query"
}

1.2 認證

Prometheus 預設無認證。Production 部署：

  • Reverse Proxy（nginx / oauth2-proxy）：前端做 auth
  • Basic auth：Prometheus 2.24+ 內建
  • TLS client cert：mTLS
  • K8s NetworkPolicy：限制存取來源
# Basic auth 設定（prometheus.yml 旁的 web-config.yml）
basic_auth_users:
  admin: $2b$12$hNf...   # bcrypt 加密的密碼

# 啟動
prometheus --web.config.file=web-config.yml

# Curl
curl -u admin:password http://localhost:9090/api/v1/query?query=up

§ 2. Prometheus Query API

2.1 Instant Query

GET /api/v1/query

# Parameters
query    : PromQL 查詢字串（必要）
time     : 評估時間（RFC3339 或 Unix timestamp）
timeout  : 查詢逾時（如 60s）

# 範例
curl 'http://localhost:9090/api/v1/query?query=up'

# Response
{
  "status": "success",
  "data": {
    "resultType": "vector",
    "result": [
      {
        "metric": {"__name__": "up", "instance": "localhost:9100", "job": "node"},
        "value": [1714579200.123, "1"]
      }
    ]
  }
}

2.2 Range Query

GET /api/v1/query_range

# Parameters
query : PromQL 查詢
start : 開始時間
end   : 結束時間
step  : 解析度（如 15s）

# 範例：過去 1 小時，每 30 秒一個點
curl 'http://localhost:9090/api/v1/query_range?\
query=rate(http_requests_total[5m])&\
start=2026-05-02T10:00:00Z&\
end=2026-05-02T11:00:00Z&\
step=30s'

# Response
{
  "status": "success",
  "data": {
    "resultType": "matrix",
    "result": [
      {
        "metric": {"job": "api"},
        "values": [
          [1714579200, "12.3"],
          [1714579230, "12.5"],
          ...
        ]
      }
    ]
  }
}

2.3 Series Query

找符合 matcher 的所有 series（不取值）：

GET /api/v1/series

# Parameters
match[] : matcher（可重複）
start   : 時間範圍開始
end     : 結束

# 範例：找所有 5xx 的 series
curl -G http://localhost:9090/api/v1/series \
  --data-urlencode 'match[]=http_requests_total{status=~"5.."}'

# Response
{
  "status": "success",
  "data": [
    {"__name__": "http_requests_total", "status": "500", "service": "api"},
    {"__name__": "http_requests_total", "status": "503", "service": "api"},
    ...
  ]
}

2.4 Labels Query

# 列出所有 label 名稱
GET /api/v1/labels

# 列出特定 label 的所有值
GET /api/v1/label/{label_name}/values

# 範例：列出所有 service
curl http://localhost:9090/api/v1/label/service/values

# Response
{
  "status": "success",
  "data": ["api", "auth", "catalog", "payment"]
}

2.5 POST 查詢（長 query）

Query 太長放 URL 會失敗，用 POST：

POST /api/v1/query
Content-Type: application/x-www-form-urlencoded

query=very_long_promql_query_here&time=...

# Curl
curl -X POST http://localhost:9090/api/v1/query \
  --data-urlencode "query=$(cat long-query.promql)"

§ 4. Prometheus Admin / Status API

4.1 Status Endpoints

Endpoint用途
GET /api/v1/status/buildinfoPrometheus 版本資訊
GET /api/v1/status/runtimeinforuntime 統計（goroutines、memory）
GET /api/v1/status/config當前 config（YAML）
GET /api/v1/status/flags命令列旗標
GET /api/v1/status/tsdbTSDB 統計（series 數、chunk 數）
GET /api/v1/status/walreplayWAL replay 狀態

4.2 Admin Endpoints

需 --web.enable-admin-api 才開：

# Reload config（不用重啟）
POST /-/reload
curl -X POST http://localhost:9090/-/reload

# 觸發 snapshot
POST /api/v1/admin/tsdb/snapshot
curl -X POST http://localhost:9090/api/v1/admin/tsdb/snapshot

# 刪除特定 series
POST /api/v1/admin/tsdb/delete_series?match[]=...
curl -X POST 'http://localhost:9090/api/v1/admin/tsdb/delete_series' \
  --data-urlencode 'match[]={job="abandoned"}'

# 立即清空已刪除的 series
POST /api/v1/admin/tsdb/clean_tombstones

4.3 Health / Liveness

GET /-/healthy   # 立即檢查（給 LB 用）
GET /-/ready     # 檢查是否準備好處理 request

curl http://localhost:9090/-/healthy
# Response: Prometheus is Healthy.

curl http://localhost:9090/-/ready
# Response: Prometheus is Ready.

4.4 Remote Write / Read

POST /api/v1/write    # remote_write 接收
POST /api/v1/read     # remote_read（PromQL 查 backend）

# Body 是 protobuf + snappy 壓縮，不直接用 curl
# 用 mimirtool / promremotewrite 工具測試

§ 5. Alertmanager API v2

5.1 概覽

Alertmanager 用 OpenAPI / Swagger 定義 v2 API。預設 port 9093，前綴 /api/v2/*。

5.2 Alerts

# 列出當前 alert
GET /api/v2/alerts

# Parameters
active     : true | false
silenced   : true | false
inhibited  : true | false
filter     : matcher（如 alertname="X"）

# 範例
curl 'http://localhost:9093/api/v2/alerts?active=true&filter=severity=critical'

# 推送 alert（外部 source）
POST /api/v2/alerts
[{
  "labels": {
    "alertname": "DiskFull",
    "severity": "critical",
    "instance": "host1"
  },
  "annotations": {
    "summary": "Disk full on host1"
  },
  "startsAt": "2026-05-02T10:00:00Z",
  "endsAt": "2026-05-02T11:00:00Z",
  "generatorURL": "http://my-system/alerts/123"
}]

5.3 Silences

# 列出所有 silence
GET /api/v2/silences

# 取得單一 silence
GET /api/v2/silence/{silenceID}

# 建立 silence
POST /api/v2/silences
{
  "matchers": [
    {"name": "alertname", "value": "HighLatency", "isRegex": false},
    {"name": "service", "value": "api", "isRegex": false}
  ],
  "startsAt": "2026-05-02T10:00:00Z",
  "endsAt": "2026-05-02T14:00:00Z",
  "createdBy": "ops@example.com",
  "comment": "Maintenance window"
}

# Response
{ "silenceID": "abc-123-def" }

# 取消 silence
DELETE /api/v2/silence/{silenceID}

5.4 Status

GET /api/v2/status

# Response
{
  "cluster": {
    "name": "alertmanager-0",
    "peers": [...],
    "status": "ready"
  },
  "uptime": "2026-04-30T10:00:00Z",
  "versionInfo": {...},
  "config": {...}
}

5.5 Receivers

# 列出 configured receivers
GET /api/v2/receivers

# Response
[
  {"name": "default-slack"},
  {"name": "pagerduty-critical"},
  {"name": "team-payments"}

§ 6. Grafana API 認證

6.1 三種認證方式

方式用途
API Key（Legacy）9.x 之前用，10+ 漸 deprecated
Service Account Token10+ 推薦，類 K8s ServiceAccount
Basic Authusername:password（admin / dev）

6.2 Service Account 設定

# 建立 service account
POST /api/serviceaccounts
{
  "name": "ci-deploy",
  "role": "Editor"   # Viewer / Editor / Admin
}

# Response
{
  "id": 123,
  "uid": "abc-xyz",
  "name": "ci-deploy",
  "login": "sa-ci-deploy",
  "orgId": 1,
  "isDisabled": false,
  "role": "Editor",
  "tokens": 0
}

# 建立 token
POST /api/serviceaccounts/{id}/tokens
{
  "name": "deploy-token",
  "secondsToLive": 31536000   # 1 年（0 = 永久）
}

# Response
{
  "id": 456,
  "name": "deploy-token",
  "key": "glsa_xxxxxxxxxxxxxxxxxxxxxxx"   # ← 只此次能看
}

6.3 用 Token 呼叫 API

curl -H "Authorization: Bearer glsa_xxxxxxxxxxxxxxxxxxxxxxx" \
  https://grafana.example.com/api/dashboards/home

# 或 Basic auth（不推薦在 production）
curl -u admin:password https://grafana.example.com/api/...

§ 10. Mimir / Loki / Tempo API

10.1 Mimir API

Mimir 大部分 API 與 Prometheus 相容（/api/v1/*），加上 multi-tenant header：

# 查詢（要 X-Scope-OrgID）
curl -H "X-Scope-OrgID: tenant-prod" \
  http://mimir:9009/prometheus/api/v1/query?query=up

# Push metric
curl -X POST -H "X-Scope-OrgID: tenant-prod" \
  -H "Content-Type: application/x-protobuf" \
  -H "Content-Encoding: snappy" \
  --data-binary @data.snappy \
  http://mimir:9009/api/v1/push

# Mimir-specific：rule 管理
GET    /api/v1/rules/{namespace}/{group_name}
POST   /api/v1/rules/{namespace}
DELETE /api/v1/rules/{namespace}/{group_name}

# Cardinality 分析（Mimir-specific）
GET /api/v1/cardinality/label_names
GET /api/v1/cardinality/label_values?label_names[]=service

§ 11. 自動化常見場景

11.1 用 API 部署 dashboard（CI/CD）

#!/bin/bash
# deploy-dashboard.sh

GRAFANA_URL="https://grafana.example.com"
GRAFANA_TOKEN="glsa_..."

# 把 JSON 從 Git 讀進來
DASHBOARD_JSON=$(cat dashboards/api-overview.json)

# Wrap with required structure
PAYLOAD=$(jq -n \
  --argjson db "$DASHBOARD_JSON" \
  '{dashboard: $db, overwrite: true, message: "CI deploy"}')

curl -X POST \
  -H "Authorization: Bearer $GRAFANA_TOKEN" \
  -H "Content-Type: application/json" \
  -d "$PAYLOAD" \
  "$GRAFANA_URL/api/dashboards/db"

11.2 程式化建 silence（maintenance window）

#!/bin/bash
# Silence 4 hours from now

START=$(date -u -d 'now' +"%Y-%m-%dT%H:%M:%SZ")
END=$(date -u -d 'now + 4 hours' +"%Y-%m-%dT%H:%M:%SZ")

curl -X POST http://alertmanager:9093/api/v2/silences \
  -H 'Content-Type: application/json' \
  -d "{
    \"matchers\": [
      {\"name\": \"namespace\", \"value\": \"production\", \"isRegex\": false}
    ],
    \"startsAt\": \"$START\",
    \"endsAt\": \"$END\",
    \"createdBy\": \"ci-bot\",
    \"comment\": \"Database maintenance\"
  }"

11.3 定期 query export（給報表）

#!/usr/bin/env python
import requests
import csv

PROM_URL = "http://prometheus:9090"
QUERY = 'sum by (service) (rate(http_requests_total[5m]))'

resp = requests.get(
    f"{PROM_URL}/api/v1/query",
    params={"query": QUERY}
)
data = resp.json()["data"]["result"]

with open("report.csv", "w") as f:
    writer = csv.writer(f)
    writer.writerow(["service", "rate"])
    for series in data:
        writer.writerow([
            series["metric"]["service"],
            series["value"][1]
        ])

✅ 本章重點摘要

30 秒回顧
  • Prometheus HTTP API 在 /api/v1/*，預設 9090，回應 JSON
  • Query API：instant query / range query / series / labels
  • Range query 必須有 step
  • 長 query 用 POST（不放 URL）
  • Targets / Rules / Alerts API 可程式化監控 Prometheus 自身
  • Admin API（reload / snapshot / delete_series）需 --web.enable-admin-api
  • Health：/-/healthy（liveness）/ /-/ready（readiness）
  • Alertmanager v2 API 在 /api/v2/*，OpenAPI 定義
  • Silence 建立流程：matchers + startsAt/endsAt + createdBy + comment
  • Alertmanager 可接收外部 alert（POST /api/v2/alerts）
  • Grafana 認證：Service Account Token（10+ 推薦），舊 API Key 漸 deprecated
  • Service Account → Token，token 只能看一次
  • Dashboard API：search / get / create / update / delete by uid
  • Datasource API：CRUD + health check + proxy 查詢
  • Mimir 大部分 API 與 Prometheus 相容，加 X-Scope-OrgID header
  • Mimir 獨家：cardinality 分析、rule 管理 API
  • Loki API：/loki/api/v1/*，含 query / push / tail / stats
  • Tempo API：/api/traces/{traceID}、search、TraceQL
  • 常見自動化：CI/CD 部署 dashboard / 程式化 silence / query export / 外部 alert / GitOps
