Grafana OSS 概念、生態與版本選擇

§ 1. Grafana 是什麼？不是什麼？

1.1 一句話定位

Grafana 是一個「資料源無關的視覺化與告警平台」。 它不儲存資料（除了少量設定），而是「查詢」其他系統並把結果視覺化。

1.2 不要把 Grafana 跟以下混淆

常見誤解實際是什麼
「Grafana 是 metric 系統」不對。它查其他系統（如 Prometheus、Mimir、InfluxDB）
「Grafana 取代 Prometheus」不對。兩者是互補關係
「Grafana 只能用 Prometheus」不對。支援 150+ datasource（SQL、Cloud、Logs、Traces 都能）
「Grafana 等於 Dashboard」不對。Dashboard 是其中一部分；alerting、explore、datasource 管理都是核心

1.3 Grafana 主要做什麼？

  1. Visualization：把資料畫成 chart、table、map、heatmap 等
  2. Dashboard：把多個 panel 組合成一個觀察面板
  3. Alerting：基於資料觸發 alert（與 Prometheus alerting 並存）
  4. Exploration：互動式探索資料（Explore 模式、Drilldown）
  5. Datasource Management：統一管理多種資料源的連線
  6. User / Team / Permission：誰能看 / 編輯什麼
  7. Plugins：擴展功能（datasource / visualization / app）

2.1 Grafana Server 內部

元件用途
FrontendReact SPA，dashboard / panel 編輯 UI
BackendGo HTTP server，處理 API、資料源代理、auth
Internal Database儲存設定、dashboard JSON、user、team、permission
Plugin System動態載入 datasource / visualization 插件
Alerting Engine評估 Grafana-managed alert（v8+）

2.2 Internal Database 選擇

DB適用
SQLite（預設）單機部署、PoC、小團隊（< 100 user）
MySQL多 instance HA、企業環境
PostgreSQL同 MySQL，PG 用戶優先

SQLite 雖能用，HA 部署必用 MySQL/PG（多個 Grafana server 共用同個 DB）。

2.3 為什麼 Grafana 不存指標資料

這是個刻意的設計選擇。好處：

  • 單一視角：metrics、logs、traces、SQL 等資料各自由專業系統儲存，Grafana 統一查
  • 解耦：可隨時換 backend（Prom → Mimir、InfluxDB → VictoriaMetrics）不用搬 dashboard
  • 規模化：Grafana 自身輕量；scale 由 backend 負責

3.2 本系列教學焦點

本系列專注 Grafana OSS。所有概念（dashboard、panel、alerting、provisioning）OSS 都有完整支援。 Enterprise 多的功能對學習無影響。Cloud 跟 OSS 操作幾乎一樣，只是不用自架。

3.3 何時該升 Enterprise / Cloud？

  • 需要 SAML / SCIM 等企業 SSO 集成
  • 需要 audit log 滿足合規
  • 需要排程 PDF 報告（給高層）
  • 不想運維（→ Cloud）
  • 需要付費支援

4.2 升級政策

Grafana 主版本一年一次（每年 GrafanaCON 發表），minor 版本每 1-2 個月。 建議：追到 latest minor（例：12.4），主版本可以延後 1-2 個月觀望。

§ 5. Grafana 12 / 13 重要新功能

2025-2026 是 Grafana 的重要轉折，有三個方向值得注意：

5.1 Observability as Code

核心思想：Dashboard、alert、datasource 都該像程式碼一樣管理（version、code review、automated deploy）。

  • Git Sync（GA in 12.4）：Grafana 直接連 GitHub repo，dashboard 變更走 PR 流程
  • 新 API（v1beta1）：Kubernetes-style resource API，未來所有 IaC 都用這套
  • Foundation SDK / Terraform Provider：用 Go / Terraform 管理 dashboard
  • Grafanactl CLI：類似 kubectl，CRUD Grafana resources

這部分將在 Part 4（Ch 42-44）詳述。

5.2 Dynamic Dashboards

讓 dashboard 自身變得「智慧」：

  • Tabs：水平切換不同 context（之前只能垂直 row）
  • Conditional rendering：根據 variable 顯示/隱藏 panel
  • Auto-grid：panel 自動調整大小
  • SQL Expressions：用 SQL 跨資料源 join / transform

5.3 Drilldown 與 AI 助手

  • Drilldown apps（Metrics / Logs / Traces / Profiles 都已 GA）：Code-free 探索
  • AI 助手：自然語言查詢、建立 dashboard、root cause analysis
  • Investigations：跨 signal 串接的調查工具（v12 引入）

§ 6. 與 Prometheus 的搭配模式

6.1 標準搭配

Prometheus（metrics 儲存）  ←→  Grafana（視覺化 + alerting）

這是入門最常見的組合。Prometheus 專注 metric pipeline，Grafana 提供 UI。

6.2 LGTM 全棧

當您不只要 metric，還要 logs + traces + profiling：

領域儲存後端負責 UI
MetricsPrometheus / MimirGrafana
LogsLokiGrafana
TracesTempoGrafana
ProfilingPyroscopeGrafana

這就是所謂 LGTM stack（Loki + Grafana + Tempo + Mimir）。Part 5（Ch 45-50）詳述。

6.3 Alerting 雙系統共存

Ch 27 講過：

  • Prometheus alerting rule + Alertmanager（典型）
  • Grafana Alerting（v8+，多 datasource 友善）
  • 兩者可以共存：Prometheus 寫 metric-based alert，Grafana 寫 cross-datasource alert

6.4 Recording Rule 在哪裡？

  • Prometheus 端：傳統做法，rule file 載入
  • Mimir Ruler：分散式評估（Ch 27）
  • Grafana 12+ Recording Rules UI：在 Grafana UI 中建立 recording rule（v12 GA）

§ 7. Grafana 的「資料模型」

理解 Grafana 怎麼處理資料，能幫您寫出更好的 query 與 panel。

7.1 Data Frame

Grafana 內部把所有資料源回傳的資料統一成 Data Frame（表格格式）： 類似 pandas DataFrame，有 columns 與 rows。 

Time                   | service | qps
2026-05-02T10:00:00Z   | api     | 1234
2026-05-02T10:01:00Z   | api     | 1267
2026-05-02T10:00:00Z   | web     | 567
...

不管 backend 是 Prometheus、SQL、JSON API，最終都變成 Data Frame，由 Grafana 統一處理。

7.2 Query 流程

sequenceDiagram participant U as User participant P as Panel participant DS as Datasource Plugin participant BE as Backend participant GS as Grafana Server participant TX as Transformations participant V as Visualization U->>P: 開啟 dashboard P->>DS: 用 query 配置查詢 DS->>GS: 翻譯成 datasource 原生語言 GS->>BE: HTTP/SQL/... BE-->>GS: 原始資料 GS->>DS: 結果 DS->>P: Data Frame P->>TX: 套用 transformations TX->>V: 視覺化引擎渲染 V->>U: 顯示

7.3 Transformations

在資料到達 visualization 前，可以做 transformations：filter、merge、calculate、rename。 這讓您可在 panel 內做後處理而不修改原始 query。

§ 8. Plugins 生態

8.1 三類 plugin

類型用途範例
Data Source連到外部資料源Prometheus、MySQL、CloudWatch、Snowflake
Panel新的視覺化類型Worldmap、Polystat、Status Map
App整套應用（datasource + panel + dashboard）Kubernetes app、AWS app

8.2 官方 vs 社群 plugin

  • Built-in：預裝（Prometheus、Loki、MySQL、CloudWatch、Tempo 等）
  • Official：Grafana Labs 維護，預設信任
  • Community：社群提供，有審核機制（Grafana Marketplace 列出）

8.3 安裝 plugin

# 用 grafana-cli
grafana-cli plugins install <plugin-id>
sudo systemctl restart grafana-server

# 或在 Helm values.yaml
plugins:
  - grafana-piechart-panel
  - grafana-worldmap-panel

8.4 Angular plugins 的重大警告

🚨 Grafana 12+ 移除 Angular 支援

早期 Grafana plugin 用 Angular。Grafana 11 警告 deprecation，Grafana 12 完全移除。
若您升 Grafana 12 但還用 Angular plugin，那些 panel 無法載入。
升級前請檢查 /admin/plugins 看有無 Angular 標記。

§ 9. License 與商業考量

9.1 AGPL v3

Grafana OSS 自 2021 年從 Apache 2.0 改為 AGPL v3（Affero GPL）。

差別：

  • Apache 2.0：可閉源使用，無強制公開
  • AGPL v3：若您修改 Grafana 並提供 SaaS給別人，必須公開您的修改

9.2 對使用者的實際影響

絕大多數使用者不受影響：

  • 內部用 Grafana 監控自家服務 → ✅ OK，無需公開
  • 修改 Grafana plugin / dashboard → ✅ OK
  • 把修改後的 Grafana 賣給別人當 SaaS → 必須公開修改的程式碼

這個改動主要針對「fork Grafana 賣 SaaS 但不貢獻回上游」的雲廠商（一些大廠 cloud monitoring 產品）。

9.3 Grafana Enterprise 的 license

商業 license，付費取得（per-user / per-seat）。授予不受 AGPL 約束的權利。

✅ 本章重點摘要

30 秒回顧
  • Grafana 是「資料源無關」的視覺化平台，不儲存資料，而是查其他系統
  • 核心元件：Frontend / Backend / Internal DB（SQLite/MySQL/PG）/ Plugin System / Alerting Engine
  • HA 部署必用 MySQL/PG，SQLite 只給單機
  • 三版本：OSS（AGPL v3，免費）/ Enterprise（付費 license + SAML/RBAC/Audit）/ Cloud（SaaS）
  • Grafana 12（2025-04）標誌 Observability as Code 時代：Git Sync、新 API、Foundation SDK、Terraform v1beta1 resources
  • Dynamic Dashboards：tabs / conditional rendering / auto-grid / SQL expressions
  • LGTM stack：Loki（logs）+ Grafana + Tempo（traces）+ Mimir（metrics）
  • 內部用 Data Frame 統一所有 datasource 的回傳格式
  • Plugin 三類：Data Source / Panel / App。Grafana 12 移除 Angular 支援（升級前必查）
  • AGPL v3 對絕大多數使用者無影響，僅針對改機並對外提供 SaaS 的場景
