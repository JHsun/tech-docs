Foundation SDK：用程式碼建 Dashboard 與 Alert

YAML 與 JSON 對於簡單 dashboard 夠用，但管理 100+ dashboard 時，同一個 panel 模板反覆複製貼上會崩潰。 Foundation SDK 讓您用 Go / TypeScript / Python / PHP / Java 寫 dashboard，享有強型別、autocomplete、模組化。 本章是 IaC 進階篇。

§ 1. 為什麼需要 SDK

1.1 JSON / YAML 的痛點

當您管理多 dashboard 時：

  • 重複：每個 service dashboard 都有「QPS / Error / Latency」三個 stat panel，但要 copy 100 次
  • 無 autocomplete：JSON 不知道 panel 有哪些可用欄位
  • 無 type check：寫錯欄位名 deploy 後才發現
  • 難重構：要把所有 dashboard 的「Latency unit 改成 ms」需要逐個改
  • 邏輯難表達：「給每個 service 自動產生 dashboard」需要 templating

1.2 SDK 的解決

  • 強型別：寫錯欄位 compile error
  • Autocomplete：IDE 提示 PanelBuilder 有哪些方法
  • 函式抽象：寫一個 redPanels(service) 給所有 service 用
  • 邏輯：用 for-loop、條件、import
  • Test：能寫 unit test

1.3 SDK 的代價

  • 需要寫程式（不是 YAML）
  • 需要 build 步驟（產生 JSON）
  • JSON 中的細節仍要懂（panel 結構）
  • Public preview 階段，API 可能變動

§ 3. Builder Pattern 概念

3.1 三大 Builder

Builder用途
DashboardBuilder頂層，設定 title / uid / time / refresh / 加 panel/row
PanelBuilder每種 panel 一個（TimeSeriesPanelBuilder / StatPanelBuilder...）
DataqueryBuilder每個 datasource 一個（PrometheusDataqueryBuilder / LokiDataqueryBuilder...）

3.2 Fluent Chain（流暢鏈式）

所有 builder 方法回傳 builder 自己，可以連鎖：

// Go
dashboard.NewDashboardBuilder("My Dashboard").
    Uid("my-dash").
    Tags([]string{"production"}).
    Refresh("30s").
    Time("now-1h", "now").
    WithPanel(...)

3.3 強型別好處

// Go：寫錯 unit IDE 立刻提示
panel.Unit(units.BitsPerSecondSI)     // ✅
panel.Unit("nonexistent_unit")        // ✅ 字串方便但無 check
panel.Unit(units.SecondsThatDoesntExist)  // ❌ compile error

§ 4. Go 實戰

4.1 安裝

mkdir my-dashboards && cd my-dashboards
go mod init my-dashboards
go get github.com/grafana/grafana-foundation-sdk/go

4.2 第一個 Dashboard

// main.go
package main

import (
    "encoding/json"
    "fmt"

    "github.com/grafana/grafana-foundation-sdk/go/common"
    "github.com/grafana/grafana-foundation-sdk/go/dashboard"
    "github.com/grafana/grafana-foundation-sdk/go/prometheus"
    "github.com/grafana/grafana-foundation-sdk/go/timeseries"
    "github.com/grafana/grafana-foundation-sdk/go/units"
)

func main() {
    builder := dashboard.NewDashboardBuilder("API Service Overview").
        Uid("api-overview").
        Tags([]string{"production", "service"}).
        Refresh("30s").
        Time("now-6h", "now").
        Timezone(common.TimeZoneBrowser).
        WithRow(dashboard.NewRowBuilder("HTTP Layer")).
        WithPanel(
            timeseries.NewPanelBuilder().
                Title("Request Rate").
                Unit(units.RequestsPerSecond).
                Min(0).
                WithTarget(
                    prometheus.NewDataqueryBuilder().
                        Expr(`sum by (method) (rate(http_requests_total{job="api"}[$__rate_interval]))`).
                        LegendFormat("{{ method }}"),
                ),
        ).
        WithPanel(
            timeseries.NewPanelBuilder().
                Title("P99 Latency").
                Unit(units.Seconds).
                Min(0).
                WithTarget(
                    prometheus.NewDataqueryBuilder().
                        Expr(`histogram_quantile(0.99, sum by (le) (rate(http_request_duration_seconds_bucket{job="api"}[$__rate_interval])))`).
                        LegendFormat("p99"),
                ),
        )

    d, err := builder.Build()
    if err != nil {
        panic(err)
    }

    out, _ := json.MarshalIndent(d, "", "  ")
    fmt.Println(string(out))
}

5.2 範例

// dashboard.ts
import { DashboardBuilder, RowBuilder, ThresholdsMode } from '@grafana/grafana-foundation-sdk/dashboard';
import { DataqueryBuilder as PromQuery } from '@grafana/grafana-foundation-sdk/prometheus';
import { PanelBuilder as TimeseriesPanel } from '@grafana/grafana-foundation-sdk/timeseries';
import { PanelBuilder as StatPanel } from '@grafana/grafana-foundation-sdk/stat';
import * as units from '@grafana/grafana-foundation-sdk/units';

const dashboard = new DashboardBuilder('API Service Overview')
  .uid('api-overview-ts')
  .tags(['production', 'service', 'typescript'])
  .refresh('30s')
  .time({ from: 'now-6h', to: 'now' })
  .withRow(new RowBuilder('KPI Row'))
  .withPanel(
    new StatPanel()
      .title('QPS')
      .unit(units.RequestsPerSecond)
      .withTarget(
        new PromQuery()
          .expr('sum(rate(http_requests_total{job="api"}[$__rate_interval]))')
      )
  )
  .withPanel(
    new TimeseriesPanel()
      .title('Request Rate by Status')
      .unit(units.RequestsPerSecond)
      .min(0)
      .withTarget(
        new PromQuery()
          .expr('sum by (status) (rate(http_requests_total{job="api"}[$__rate_interval]))')
          .legendFormat('{{ status }}')
      )
  );

console.log(JSON.stringify(dashboard.build(), null, 2));

§ 6. Python 實戰

6.1 安裝

pip install grafana-foundation-sdk

6.2 範例

# dashboard.py
from grafana_foundation_sdk.builders.dashboard import Dashboard, Row
from grafana_foundation_sdk.builders.timeseries import Panel as TimeseriesPanel
from grafana_foundation_sdk.builders.stat import Panel as StatPanel
from grafana_foundation_sdk.builders.prometheus import Dataquery as PromQuery
from grafana_foundation_sdk.cog.encoder import JSONEncoder
from grafana_foundation_sdk.models.common import TimeZoneBrowser

def build_dashboard():
    return (
        Dashboard("API Service Overview")
        .uid("api-overview-py")
        .tags(["production", "service", "python"])
        .refresh("30s")
        .time("now-6h", "now")
        .timezone(TimeZoneBrowser)
        .with_row(Row("HTTP Layer"))
        .with_panel(
            TimeseriesPanel()
            .title("Request Rate")
            .unit("reqps")
            .min_val(0)
            .with_target(
                PromQuery()
                .expr('sum by (method) (rate(http_requests_total{job="api"}[$__rate_interval]))')
                .legend_format("{{ method }}")
            )
        )
        .with_panel(
            StatPanel()
            .title("Active Users")
            .unit("short")
            .with_target(
                PromQuery()
                .expr('sum(active_users{job="api"})')
            )
        )
    )

if __name__ == "__main__":
    dash = build_dashboard().build()
    print(JSONEncoder(sort_keys=True, indent=2).encode(dash))

§ 7. 模板化與重用

7.1 抽象出共用元件

真正的威力：把「RED 三 panel」抽成函式，每個 service 用一行：

// Go: helpers.go
package main

import (
    "fmt"
    "github.com/grafana/grafana-foundation-sdk/go/dashboard"
    "github.com/grafana/grafana-foundation-sdk/go/prometheus"
    "github.com/grafana/grafana-foundation-sdk/go/timeseries"
    "github.com/grafana/grafana-foundation-sdk/go/units"
)

// REDPanels 為一個 service 產生 3 個 RED panel
func REDPanels(service string) []dashboard.PanelBuilder {
    rate := timeseries.NewPanelBuilder().
        Title(fmt.Sprintf("%s · Request Rate", service)).
        Unit(units.RequestsPerSecond).
        Min(0).
        WithTarget(
            prometheus.NewDataqueryBuilder().
                Expr(fmt.Sprintf(`sum(rate(http_requests_total{job="%s"}[$__rate_interval]))`, service)),
        )

    errors := timeseries.NewPanelBuilder().
        Title(fmt.Sprintf("%s · Error Rate", service)).
        Unit(units.PercentUnit).
        Min(0).
        WithTarget(
            prometheus.NewDataqueryBuilder().
                Expr(fmt.Sprintf(
                    `sum(rate(http_requests_total{job="%s",status=~"5.."}[$__rate_interval])) / sum(rate(http_requests_total{job="%s"}[$__rate_interval]))`,
                    service, service)),
        )

    duration := timeseries.NewPanelBuilder().
        Title(fmt.Sprintf("%s · P99 Latency", service)).
        Unit(units.Seconds).
        Min(0).
        WithTarget(
            prometheus.NewDataqueryBuilder().
                Expr(fmt.Sprintf(
                    `histogram_quantile(0.99, sum by (le) (rate(http_request_duration_seconds_bucket{job="%s"}[$__rate_interval])))`,
                    service)),
        )

    return []dashboard.PanelBuilder{rate, errors, duration}
}

7.2 在 dashboard 中使用

func main() {
    services := []string{"api", "web", "auth", "payment"}

    builder := dashboard.NewDashboardBuilder("Multi-Service Overview").
        Uid("multi-overview").
        Refresh("30s")

    for _, svc := range services {
        builder.WithRow(dashboard.NewRowBuilder(svc))
        for _, panel := range REDPanels(svc) {
            builder.WithPanel(panel)
        }
    }

    d, _ := builder.Build()
    // ...
}

40 行的 helper 取代 100+ 個 panel 的重複定義。

7.4 Test Dashboard Code

// dashboard_test.go
func TestREDPanels(t *testing.T) {
    panels := REDPanels("api")

    if len(panels) != 3 {
        t.Errorf("expected 3 panels, got %d", len(panels))
    }

    for _, p := range panels {
        built, err := p.Build()
        if err != nil {
            t.Errorf("panel build failed: %v", err)
        }
        if built.Title == nil || *built.Title == "" {
            t.Error("panel missing title")
        }
    }
}

JSON / YAML 不可能 unit test，SDK 可以。這是大團隊的關鍵。

9.1 Alerting SDK 套件

import "github.com/grafana/grafana-foundation-sdk/go/alerting"

9.2 建立 Alert Rule

rule := alerting.NewAlertRuleBuilder().
    Title("APIErrorRate").
    Condition("C").
    For("5m").
    Labels(map[string]string{
        "severity": "critical",
        "team":     "platform",
    }).
    Annotations(map[string]string{
        "summary":     "API error rate exceeded threshold",
        "runbook_url": "https://wiki/api-errors",
    }).
    Data([]alerting.AlertQuery{
        {
            RefId:         "A",
            DatasourceUid: "prometheus-prod",
            Model: map[string]any{
                "expr": `rate(http_requests_total{status=~"5.."}[5m]) / rate(http_requests_total[5m])`,
            },
        },
        {
            RefId:         "B",
            DatasourceUid: "__expr__",
            Model: map[string]any{
                "type":       "reduce",
                "expression": "A",
                "reducer":    "last",
            },
        },
        {
            RefId:         "C",
            DatasourceUid: "__expr__",
            Model: map[string]any{
                "type":       "threshold",
                "expression": "B",
                "conditions": []map[string]any{
                    {
                        "evaluator": map[string]any{
                            "type":   "gt",
                            "params": []float64{0.05},
                        },
                    },
                },
            },
        },
    }).
    NoDataState(alerting.AlertRuleNoDataStateNoData).
    ExecErrState(alerting.AlertRuleExecErrStateError)

9.3 用 Helper 抽象

// helpers/alerts.go
func ErrorRateAlert(service string, threshold float64) *alerting.AlertRuleBuilder {
    return alerting.NewAlertRuleBuilder().
        Title(fmt.Sprintf("%sErrorRate", strings.Title(service))).
        For("5m").
        Labels(map[string]string{
            "severity": "critical",
            "service":  service,
        }).
        // ... 標準 query / threshold setup
}

// 使用
rules := []alerting.AlertRule{}
for _, svc := range services {
    rule, _ := ErrorRateAlert(svc, 0.05).Build()
    rules = append(rules, rule)
}

§ 10. CI/CD 與部署

10.1 整體 pipeline

  1. 開發者改 Go / TS / Python 程式碼
  2. 推 PR
  3. CI 跑 build + test
  4. 產生 dashboard JSON 檔
  5. 用 grafanactl apply 到 staging Grafana
  6. QA 驗證
  7. Merge 到 main → 部署 production

10.2 GitHub Actions 範例

name: Deploy Foundation SDK Dashboards

on:
  push:
    branches: [main, staging]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version: '1.22'

      - name: Build dashboards
        run: |
          mkdir -p out
          go run ./cmd/build-dashboards/main.go --out=out/

      - name: List generated
        run: ls -la out/

      - name: Install grafanactl
        run: |
          curl -sSL https://github.com/grafana/grafanactl/releases/latest/download/grafanactl_linux_amd64.tar.gz | tar xz
          sudo mv grafanactl /usr/local/bin/

      - name: Deploy to Grafana
        env:
          GRAFANA_TOKEN: ${{ secrets.GRAFANA_TOKEN }}
        run: |
          grafanactl config set-context default \
            --server=https://grafana.example.com \
            --token=$GRAFANA_TOKEN
          grafanactl apply -f out/

10.4 v1beta1 wrapping（給 K8s Operator 用）

// 包裝為 K8s-style resource
type DashboardWrapper struct {
    APIVersion string              `json:"apiVersion"`
    Kind       string              `json:"kind"`
    Metadata   Metadata            `json:"metadata"`
    Spec       dashboard.Dashboard `json:"spec"`
}

func wrapDashboard(d *dashboard.Dashboard) DashboardWrapper {
    return DashboardWrapper{
        APIVersion: "dashboard.grafana.app/v1beta1",
        Kind:       "Dashboard",
        Metadata: Metadata{
            Name: *d.Uid,
        },
        Spec: *d,
    }
}

11.2 Grafonnet 的位置

Grafonnet 是社群維護的 Jsonnet library，比 Foundation SDK 早出現。 現在 Grafana Labs 自己推 Foundation SDK，Grafonnet 仍活躍但新功能可能晚到。

選擇：

  • 已用 Jsonnet 生態（如 Tanka）→ Grafonnet
  • 新 setup → Foundation SDK

✅ 本章重點摘要

30 秒回顧
  • Foundation SDK：用 Go / TypeScript / Python / PHP / Java 寫 Grafana resource
  • 主要支援 Grafana 12+，向下相容 10+
  • 核心優勢：強型別、autocomplete、模板化、可 unit test
  • 仍是 public preview，API 可能 minor 變動
  • 架構：Code → Builder → .Build() → JSON → Deploy

  • Fluent chain：所有方法回傳 builder 自己
  • 真正威力是抽象重用：寫一個 REDPanels(service) 給所有 service 用

  • 支援 Variables / Annotations / Threshold / Field Override
  • Alerting SDK 也支援（AlertRuleBuilder）
  • 部署：API direct / grafanactl apply / Provisioning file / v1beta1 wrap 給 Operator
  • 選型：<10 dashboard 用 YAML、50+ 重複 pattern 用 SDK、Jsonnet 生態用 Grafonnet
