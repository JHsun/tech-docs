Chapter 30 · Grafana 安裝與快速上手

Grafana 安裝與快速上手：從 Docker 到 Helm

實際把 Grafana 跑起來。本章涵蓋 4 種主流安裝方式：Binary、Docker、Docker Compose（含 Prometheus 整合）、Helm（K8s 標準）。 並帶您完成第一個 dashboard 的建立——這是後續所有章節的起點。

§ 1. 安裝方式選擇

方式適用HA難度
Binary + Systemd傳統機房、單機需手動架構低
Docker（單一）本機開發、快速試否最低
Docker Compose整 stack 一起跑（dev/staging）有限低
Helm（單獨）K8s 純 Grafana多副本 + 外部 DB中
kube-prometheus-stackK8s + 完整觀測 stack內建中（業界主流）

1.1 系統需求

規模CPURAMDisk
個人 / PoC1 vCPU512MB5GB

生產（< 100 user）2 vCPU2GB20GB（SQLite）/ 外部 DB
大型（> 100 user）4+ vCPU4GB+外部 DB（MySQL/PG）

§ 2. Binary + Systemd 安裝

2.1 從 .deb / .rpm 套件管理員安裝（推薦）

# Ubuntu / Debian
sudo apt-get install -y software-properties-common
sudo add-apt-repository "deb https://packages.grafana.com/oss/deb stable main"
wget -q -O - https://packages.grafana.com/gpg.key | sudo apt-key add -
sudo apt-get update
sudo apt-get install grafana

# RHEL / CentOS
sudo tee /etc/yum.repos.d/grafana.repo <<EOF
[grafana]
name=grafana
baseurl=https://packages.grafana.com/oss/rpm
repo_gpgcheck=1
enabled=1
gpgcheck=1
gpgkey=https://packages.grafana.com/gpg.key
EOF

sudo yum install grafana

2.2 啟動服務

sudo systemctl daemon-reload
sudo systemctl enable grafana-server
sudo systemctl start grafana-server

# 看狀態
sudo systemctl status grafana-server
sudo journalctl -u grafana-server -f

2.3 預設目錄結構

路徑用途
/usr/share/grafana程式檔、預設 plugin
/etc/grafana/grafana.ini主設定檔（Ch 31 詳述）
/etc/grafana/provisioning/Provisioning 設定（datasources、dashboards）
/var/lib/grafanaInternal DB（SQLite 預設）、plugin、dashboard JSON
/var/log/grafana日誌
/var/lib/grafana/plugins第三方 plugin

2.4 訪問

瀏覽 http://<your-host>:3000，預設帳密 admin / admin，第一次登入會強制改密碼。

§ 3. Docker 單機安裝

3.1 最簡啟動

# Pull image
docker pull grafana/grafana:12.4.0

# Run
docker run -d \
  --name=grafana \
  -p 3000:3000 \
  grafana/grafana:12.4.0

到 http://localhost:3000 用 admin/admin 登入。

⚠️ 用 latest tag 是個壞主意

Production 絕對不要用 grafana/grafana:latest。每次 pull 都可能拉到不同版本， 升級不可控、不可重現。
永遠固定具體版本：grafana/grafana:12.4.0。

3.2 持久化（重要）

# 沒持久化 = container 重啟 dashboard 全沒了
# 用 named volume：
docker volume create grafana-data

docker run -d \
  --name=grafana \
  -p 3000:3000 \
  -v grafana-data:/var/lib/grafana \
  grafana/grafana:12.4.0

3.3 設定環境變數

Grafana 任何設定都能用環境變數覆蓋：格式 GF_<SECTION>_<KEY>。

docker run -d \
  --name=grafana \
  -p 3000:3000 \
  -v grafana-data:/var/lib/grafana \
  -e "GF_SECURITY_ADMIN_USER=admin" \
  -e "GF_SECURITY_ADMIN_PASSWORD=ChangeMe123!" \
  -e "GF_SERVER_ROOT_URL=https://grafana.example.com" \
  -e "GF_USERS_ALLOW_SIGN_UP=false" \
  -e "GF_AUTH_ANONYMOUS_ENABLED=false" \
  -e "GF_INSTALL_PLUGINS=grafana-piechart-panel,grafana-clock-panel" \
  grafana/grafana:12.4.0

對應的 grafana.ini sections：

[security]
admin_user = admin
admin_password = ChangeMe123!

[server]
root_url = https://grafana.example.com

[users]
allow_sign_up = false

[auth.anonymous]
enabled = false

3.4 用設定檔（推薦 Production）

# 把 grafana.ini 掛進去
docker run -d \
  --name=grafana \
  -p 3000:3000 \
  -v grafana-data:/var/lib/grafana \
  -v $(pwd)/grafana.ini:/etc/grafana/grafana.ini:ro \
  -v $(pwd)/provisioning:/etc/grafana/provisioning:ro \
  grafana/grafana:12.4.0

§ 4. Docker Compose（含 Prometheus）

4.1 完整 docker-compose.yml

version: '3.8'

services:
  prometheus:
    image: prom/prometheus:v3.11.2
    container_name: prometheus
    restart: unless-stopped
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml:ro
      - prometheus-data:/prometheus
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.path=/prometheus'
      - '--storage.tsdb.retention.time=30d'
      - '--web.enable-lifecycle'
    ports:
      - "9090:9090"
    networks:
      - monitoring

  node-exporter:
    image: prom/node-exporter:latest
    container_name: node-exporter
    restart: unless-stopped
    pid: host
    volumes:
      - /:/host:ro,rslave
    command:
      - '--path.rootfs=/host'
    ports:
      - "9100:9100"
    networks:
      - monitoring

  grafana:
    image: grafana/grafana:12.4.0
    container_name: grafana
    restart: unless-stopped
    depends_on:
      - prometheus
    environment:
      - GF_SECURITY_ADMIN_USER=admin
      - GF_SECURITY_ADMIN_PASSWORD=ChangeMe123!
      - GF_USERS_ALLOW_SIGN_UP=false
      - GF_INSTALL_PLUGINS=grafana-piechart-panel
    volumes:
      - grafana-data:/var/lib/grafana
      - ./grafana/provisioning:/etc/grafana/provisioning:ro
    ports:
      - "3000:3000"
    networks:
      - monitoring

volumes:
  prometheus-data:
  grafana-data:

networks:
  monitoring:

4.2 配套的 prometheus.yml

global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  - job_name: 'node'
    static_configs:
      - targets: ['node-exporter:9100']

4.3 自動 provisioning Datasource

建立 grafana/provisioning/datasources/prometheus.yml：

apiVersion: 1

datasources:
  - name: Prometheus
    type: prometheus
    access: proxy
    url: http://prometheus:9090
    isDefault: true
    editable: true

啟動後 Grafana 會自動連好 Prometheus，無需手動設定。

4.4 啟動與驗證

# 啟動
docker compose up -d

# 看狀態
docker compose ps
docker compose logs -f grafana

# 停止
docker compose down

# 連同資料一起刪
docker compose down -v

§ 5. Helm 部署到 K8s

5.1 純 Grafana（無 Prometheus）

# 加 Helm repo
helm repo add grafana https://grafana.github.io/helm-charts
helm repo update

# 建專屬 namespace
kubectl create namespace monitoring

# 簡單安裝
helm install grafana grafana/grafana \
  --namespace monitoring

5.2 自訂 values.yaml（生產用）

# grafana-values.yaml
replicas: 2                                   # HA

image:
  repository: grafana/grafana
  tag: "12.4.0"

# Admin 認證（不要用預設！）
admin:
  existingSecret: grafana-admin-secret        # 從 K8s Secret 取
  userKey: admin-user
  passwordKey: admin-password

# 持久化儲存
persistence:
  enabled: true
  type: pvc
  storageClassName: standard
  size: 10Gi
  accessModes:
    - ReadWriteOnce

# 外部 DB（HA 必要）
grafana.ini:
  database:
    type: postgres
    host: postgres.example.com:5432
    name: grafana
    user: grafana
    password: $__file{/etc/secrets/db-password}

# Resource limits
resources:
  requests:
    cpu: 100m
    memory: 256Mi
  limits:
    memory: 1Gi

# Service 設定
service:
  type: ClusterIP

# Ingress
ingress:
  enabled: true
  ingressClassName: nginx
  hosts:
    - grafana.example.com
  tls:
    - secretName: grafana-tls
      hosts:
        - grafana.example.com

# 預先安裝的 plugin
plugins:
  - grafana-piechart-panel
  - grafana-clock-panel

# Datasources（provisioning）
datasources:
  datasources.yaml:
    apiVersion: 1
    datasources:
      - name: Prometheus
        type: prometheus
        url: http://prometheus-server.monitoring.svc.cluster.local:9090
        access: proxy
        isDefault: true

# Dashboards（從 ConfigMap 載入）
dashboardProviders:
  dashboardproviders.yaml:
    apiVersion: 1
    providers:
      - name: 'default'
        orgId: 1
        folder: ''
        type: file
        disableDeletion: false
        editable: true
        options:
          path: /var/lib/grafana/dashboards/default

dashboardsConfigMaps:
  default: grafana-dashboards-default        # 預先建立的 ConfigMap

# Pod Anti-Affinity (HA)
affinity:
  podAntiAffinity:
    preferredDuringSchedulingIgnoredDuringExecution:
      - weight: 100
        podAffinityTerm:
          labelSelector:
            matchExpressions:
              - key: app.kubernetes.io/name
                operator: In
                values: [grafana]
          topologyKey: kubernetes.io/hostname

5.3 安裝

helm install grafana grafana/grafana \
  --namespace monitoring \
  -f grafana-values.yaml

# 升級
helm upgrade grafana grafana/grafana \
  --namespace monitoring \
  -f grafana-values.yaml

# 回滾
helm rollback grafana 1 --namespace monitoring
🚨 SQLite 不適用 HA

多副本 Grafana 使用 SQLite 會導致資料不一致與崩潰。
HA 部署（replicas: 2+）必須使用外部 PostgreSQL 或 MySQL。

§ 6. kube-prometheus-stack

若您要 Prometheus + Alertmanager + Grafana 全套，kube-prometheus-stack 是業界事實標準。 它包含 Grafana 並預設好幾十個 Kubernetes 監控 dashboard。

6.1 安裝

helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo update

helm install kps prometheus-community/kube-prometheus-stack \
  --namespace monitoring \
  --create-namespace \
  --set grafana.adminPassword='ChangeMe123!'

6.2 取得 admin password

kubectl get secret kps-grafana \
  --namespace monitoring \
  -o jsonpath="{.data.admin-password}" | base64 --decode

6.3 訪問 Grafana

# Port-forward 試
kubectl port-forward -n monitoring svc/kps-grafana 3000:80

# 開瀏覽器：http://localhost:3000
# 帳號：admin / 上面拿到的 password

6.4 預載的 dashboard

kube-prometheus-stack 自帶以下 dashboard（在 General/Kubernetes folders）：

  • Node Exporter Full / Nodes / Cluster
  • Kubernetes / Compute Resources / Cluster | Namespace | Pod
  • Kubernetes / Networking / Cluster | Namespace | Pod
  • Kubernetes / Persistent Volumes
  • Prometheus / Overview
  • Alertmanager / Overview

§ 7. 第一次登入與初始設定

7.1 登入流程

  1. 到 http://<host>:3000
  2. 輸入 admin / admin（或您設的密碼）
  3. 強制改密碼（OSS 預設行為）
  4. 進入主畫面

7.2 必做的初始設定

項目位置建議
關閉公開註冊Server admin → Settings 或 grafana.iniusers.allow_sign_up = false
關閉匿名訪問同上auth.anonymous.enabled = false
設 root_urlgrafana.ini server section外部 URL（給 Slack 連結用）
SMTP 設定grafana.ini smtp給 alert 通知用
備份 internal DB—定期備份 /var/lib/grafana/grafana.db

7.3 主介面快速導覽

左側 sidebar 是主導航：

  • Dashboards：所有 dashboard 列表（含 folder）
  • Explore：互動式查資料（debug PromQL 必用）
  • Drilldown：點選式探索 metrics / logs / traces / profiles
  • Alerting：Grafana-managed alert rules
  • Connections：Datasources、外部 plugin、整合
  • Administration：使用者、team、organization、權限、setting

§ 8. 連接 Prometheus 資料源

8.1 透過 UI 加

  1. 左側 sidebar → Connections → Data sources
  2. 點 Add new data source
  3. 選 Prometheus
  4. 填 URL（例 http://prometheus:9090）
  5. 下方的「Type and version」可指定 Prometheus 版本（影響 PromQL 功能 hint）
  6. 點 Save & test 確認連通

8.2 重要設定欄位

欄位意義
URLPrometheus 位址
AccessServer (proxy) — Grafana server 代理；Browser — 直連（已棄用）
AuthBasic auth、TLS、Custom HTTP Headers
Scrape interval讓 Grafana 知道 Prometheus 的 scrape 頻率，影響 step 計算
Query timeout單次 query 上限
Manage alerts via Alerting UI讓 Grafana 從這個 datasource 抓 / 寫 Prometheus alert rule
Cache levelLow / Medium / High（query 結果快取）
Incremental queryingv9+：只查新資料增量，加速 dashboard refresh

8.3 透過 Provisioning（推薦）

# /etc/grafana/provisioning/datasources/prometheus.yml
apiVersion: 1

datasources:
  - name: Prometheus
    type: prometheus
    access: proxy
    url: http://prometheus:9090
    isDefault: true
    editable: false       # ← 鎖住，不讓 UI 改
    jsonData:
      httpMethod: POST    # 大查詢用 POST 避免 URL 太長
      manageAlerts: true
      prometheusType: Prometheus
      prometheusVersion: 3.11.2
      cacheLevel: 'High'
      incrementalQuerying: true
      timeInterval: '15s'

放好後重啟 Grafana 即生效。

§ 9. 建立第一個 Dashboard

9.1 從 Library 匯入（最快）

有現成 dashboard 可直接用，不必從零開始：

  1. 到 grafana.com/grafana/dashboards
  2. 找「Node Exporter Full」（最熱門 ID = 1860）
  3. Grafana UI → Dashboards → New → Import
  4. 填 1860，按 Load
  5. 選 Prometheus datasource，按 Import

常用 dashboard ID：

用途ID
Node Exporter Full1860
Prometheus 2.0 Stats3662
Kubernetes Cluster7249
cAdvisor (containers)14282
NGINX Exporter12708
MySQL Overview7362
Loki Logs Dashboard13186

9.2 從零建一個簡單 dashboard

  1. 左側 → Dashboards → New → New Dashboard
  2. Add visualization → 選 Prometheus datasource
  3. Query 中寫：up → 看所有 target 的 up 狀態
  4. 右側 Visualization 選「Stat」
  5. Title 填 "Targets Up"
  6. Apply（套用）
  7. 頂部 Save dashboard，輸入名稱與 folder

9.3 RED Method 入門 dashboard

對 web 服務最有用的三個指標：Rate、Errors、Duration。

# Panel 1: Rate
sum by (service) (rate(http_requests_total[5m]))

# Panel 2: Errors
sum by (service) (rate(http_requests_total{status=~"5.."}[5m]))
  /
sum by (service) (rate(http_requests_total[5m]))

# Panel 3: Duration p99
histogram_quantile(0.99,
  sum by (service, le) (rate(http_request_duration_seconds_bucket[5m]))
)

9.4 Time Range 控制

右上角的時間選擇器：

  • Quick ranges：Last 5m / 1h / 6h / 24h / 7d
  • Absolute：精確時間
  • Refresh interval：自動 refresh（5s / 30s / 1m...）
  • Time zone：dashboard 上方 settings → time options 設

9.5 Variables（dashboard 變數）

讓一個 dashboard 適配多個 instance / service：

  1. Dashboard settings → Variables → Add
  2. Type: Query
  3. Datasource: Prometheus
  4. Query: label_values(up, instance)
  5. Save
  6. Panel 中用 up{instance="$instance"} 引用

頂部會出現下拉選單，可動態切換 instance。

§ 10. 常見安裝問題

⚠️ 問題 1：Permission denied on /var/lib/grafana

Docker 啟動時報 permission denied。原因：mounted volume 的 owner 不是 grafana user。
解法：
docker run --user 472:472 ...
或讓 Grafana 用 root：-e GF_PATHS_DATA=/var/lib/grafana --user 0（不推薦）

⚠️ 問題 2：忘了 admin 密碼

Binary：sudo grafana-cli admin reset-admin-password newpass
Docker：docker exec -it grafana grafana-cli admin reset-admin-password newpass
K8s：kubectl exec -it grafana-pod -- grafana cli admin reset-admin-password newpass

⚠️ 問題 3：Datasource 無法連通

症狀：Save & test 顯示 connection refused / timeout。
檢查：
• 網路：Grafana 容器能 ping 到 Prometheus 嗎？（同 docker network / K8s service？）
• URL：用 service name（http://prometheus:9090）不是 localhost
• K8s：用 FQDN http://prometheus.monitoring.svc.cluster.local:9090

⚠️ 問題 4：Dashboard 載入很慢

原因：
• Query 太大（一次抓太多 series）
• Datasource 的 step 計算太細（短時間範圍 + interval=1s）
• Prometheus 自身慢（CPU/IO 不夠）
解法：
• Panel 加 $__rate_interval 讓 step 自動調整
• 用 Recording Rule 預先聚合
• 開 datasource 的 incremental querying（v9+）

⚠️ 問題 5：HTTPS / Reverse Proxy 後 Grafana 連結錯誤

症狀：通知 email 中的連結是 http://localhost:3000/...，但實際 URL 是 https://grafana.example.com。
解法：設 GF_SERVER_ROOT_URL=https://grafana.example.com 與 GF_SERVER_SERVE_FROM_SUB_PATH=true（如有 sub-path）。

⚠️ 問題 6：Multi-replica 看到不同的 dashboard

症狀：Helm 設了 replicas=2，刷新時看到不同內容。
原因：兩個 pod 各自用 SQLite，內容不同步。
解法：改用外部 PostgreSQL/MySQL（前述）。
