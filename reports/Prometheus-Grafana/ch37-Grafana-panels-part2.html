§ 1. Heatmap Panel

1.1 Heatmap 是什麼

Heatmap 把資料分布在 X（時間）+ Y（值範圍）的二維平面上，用顏色深淺表示密度。 特別適合視覺化 latency 分布、request size 分布等。

1.2 何時用 Heatmap

適合不適合
Latency 分布隨時間變化單一數值
Request size / response size 分布分類資料
看 outlier 與 mode少於 1 小時的小範圍
找出「雙峰分布」

1.3 為什麼比 percentile line 強

P99 / P95 線只告訴您「某分位數值」，但 heatmap 顯示「整個分布」：

  • 看 P99 線：latency 一直在 500ms — 看似穩定
  • 看 heatmap：實際上 90% 請求在 50ms，10% 突然在 1s — 有 bimodal 問題

1.4 設定 Heatmap from Histogram

Visualization: Heatmap

Query:
  expr: sum by (le) (
    rate(http_request_duration_seconds_bucket[$__rate_interval])
  )
  format: Heatmap          ← 必選！
  legend: '{{ le }}'

Heatmap options:
  - Y Axis:
    Mode: Time series buckets
    Unit: Time/seconds (s)
    Decimals: Auto
  - Cell display:
    Color scheme: Spectral / Reds / Greens
    Color mode: Opacity / Scheme
    Min/Max: Auto / 自訂

1.5 重要：Histogram 必須有 le label

Heatmap 需要 Prometheus histogram 的 _bucket metric 與 le label。 若您的應用沒有用 histogram client library 暴露 metric，畫不出 heatmap。

1.6 Native Histogram（v9+）

Prometheus Native Histogram（Ch 06）讓 heatmap 更精確。Grafana 自動偵測， 若 metric 是 native histogram，會用更高解析度顯示。

§ 2. Histogram Panel

注意：Histogram panel 跟 Heatmap 不同。Histogram = 單一時間點的分布（不含時間維度）。

2.1 何時用 Histogram

  • 「現在這一刻 / 過去 5 分鐘的延遲分布長什麼樣」
  • 不需要看「分布如何隨時間變化」

2.2 設定

Visualization: Histogram

Query: rate(http_request_duration_seconds_bucket[5m])
Format: Time series

Histogram options:
  - Bucket size: Auto / 0.1
  - Bucket offset: 0
  - Combine series: ✅ (合併所有 series)

2.3 Histogram vs Heatmap 對比

面向HistogramHeatmap
維度1D（值分布）2D（時間 × 值分布）
看出「現在分布形狀」「分布如何變化」
典型用例SLO 分析Performance debug

§ 3. State Timeline / Status History

3.1 兩個相關 panel

Panel用途
State Timeline 顯示 state 隨時間變化（每個 series 一橫條，按狀態染色）
Status History 類似 State Timeline，但每個 cell 是固定 interval 的小方塊

3.2 何時用

  • K8s pod status 隨時間變化（Running / Pending / Failed）
  • Service health 歷史（Up / Down）
  • Deployment / job status
  • Synthetic check 結果（每 5 分鐘的 ✅/❌）

3.3 範例：Service Up/Down History

Visualization: State timeline

Query:
  expr: up
  legend: '{{ instance }}'

State timeline options:
  Value mappings:
    1 → text: "Up", color: green
    0 → text: "Down", color: red
  Merge equal consecutive values: ✅
  Show values: Always

3.4 用 Loki 抓 deployment 事件畫 timeline

Datasource: Loki
Query: count_over_time({app="ci-deploy"}[1m])

效果：每分鐘有部署事件 → 紅色 / 綠色染色顯示

§ 4. Logs Panel（Loki 用）

4.1 在 Dashboard 內顯示 logs

當您要在 dashboard 中嵌入「最近的 error log」、「相關事件」等，用 Logs panel。

4.2 基本設定

Visualization: Logs

Query (Loki):
  {namespace="$namespace", level="error"}

Logs options:
  - Time: ✅
  - Unique labels: ✅
  - Common labels: ✅
  - Wrap lines: ✅
  - Prettify JSON: ✅
  - Enable log details: ✅
  - Deduplication: None / Exact / Numbers / Signature
  - Sort order: Descending（最新在上）

4.3 整合 Variables

Query: {namespace="$namespace", app="$app", level=~"$level"}

頂部變數讓使用者切換看不同條件的 logs，
不用每次改 query。

4.4 Derived Fields 連結

Ch 34 已介紹：在 Loki datasource 設 derived field，logs panel 中的 trace_id 就會變成可點擊連結，跳到 Tempo。

§ 5. Geomap Panel

5.1 何時用 Geomap

  • 多 region service 流量視覺化
  • 使用者地理分布
  • CDN 端點 status
  • IoT 設備位置

5.2 基本設定

Visualization: Geomap

Query (PostgreSQL):
SELECT latitude, longitude,
       count(*) AS active_users
FROM user_sessions
WHERE $__timeFilter(created_at)
GROUP BY latitude, longitude;

Geomap options:
  - Map view:
    Initial view: Coordinates / Lookup field / Zoom
    Initial zoom: 3 (世界) / 8 (國家) / 12 (城市)
  - Layers:
    - Type: Markers / Heatmap / Geojson / Photos
    - Field mappings: lat, lon, value

5.3 Layer 類型

Layer用途
Markers每個位置一個點，大小/顏色依值
Heatmap密度視覺化（紅熱）
GeoJSON多邊形（國界、行政區）
Open Street Map底圖
ArcGIS / CARTO商業底圖

5.4 從 IP 自動定位

v9+ 內建 GeoIP lookup，從 IP 自動取得 lat/lon：

Transform: GeoIP lookup
  Source field: ip_address
  Lookup field: city  / country

§ 6. Trend Panel

6.1 Trend Panel 是什麼

v9+ 引入。專門給「非時間 X 軸」的 series 用。例：

  • X = 距離，Y = 訊號強度（不是時間趨勢）
  • X = pod count，Y = response time（scaling test）
  • X = batch size，Y = throughput

6.2 與 Time Series 的區別

Time Series 強制 X 軸是時間。Trend 允許任意 numeric X 軸。 SQL query 回傳兩個 numeric column 時用。

§ 7. Bar Chart 與 XY Chart

7.1 Bar Chart Panel

很多人混淆 Bar Chart 與 Bar Gauge：

Panel用途
Bar Chart 類別 vs 數值（X 是類別，Y 是值）
Bar Gauge 每個項目當前值 vs 範圍
Time Series (Bars style) 時間 vs 值 (柱狀)

7.2 Bar Chart 範例

Query (PG):
SELECT category, SUM(amount) FROM orders
WHERE $__timeFilter(date)
GROUP BY category;

效果：
  electronics  ████████████ $12,345
  books        ████ $4,567
  clothing     ████████ $8,901

7.3 XY Chart Panel

v10+ 引入。Scatter plot：兩個 numeric 維度的散點圖。例：

  • X = response time、Y = error rate（每個 service 一個點）
  • X = CPU usage、Y = memory usage（每個 instance 一個點）

§ 8. Text、News、Dashboard List

8.1 Text Panel

顯示靜態 markdown / HTML，不查 datasource。用於：

  • Dashboard 標題與說明
  • Runbook 連結列表
  • Section 分隔說明
  • 系統公告
Visualization: Text

Mode: Markdown / HTML / Code

Content:
# Service Overview Dashboard

## Owner
- Team: Platform
- On-call: see [PagerDuty](https://...)

## Related Links
- [Runbook](https://...)
- [Service Architecture](https://...)
- [Slack Channel](https://...)

8.2 News Panel

顯示 RSS feed，例：Grafana blog、AWS status、內部部署 RSS。

Visualization: News

URL: https://grafana.com/blog/index.xml
Show image: ✅

8.3 Dashboard List Panel

顯示 dashboard 列表，常用於建立「dashboard hub」：

Visualization: Dashboard list

Show:
  - Starred: ✅ (顯示我 starred 的)
  - Recently viewed: ✅
  - Search: ✅
  - Folders: 選 Production
  - Tags: 'service'

Limits:
  Max items: 10

典型用法：建一個「Production Hub」dashboard，用 Dashboard list 列出所有 production 相關 dashboard，當作「目錄」。

§ 9. Node Graph / Service Graph（trace 用）

9.1 Node Graph

顯示「節點 + 邊」的網路圖，常用於：

  • 分散式 trace 的 service 拓樸
  • Tempo 自動產生的 service map
  • K8s service 依賴圖

9.2 Service Graph (Tempo 自動產生)

Ch 34 提到 Tempo 啟用 metrics generator 後，會把 RED metric 寫回 Prometheus， Grafana 用這些 metric 自動畫服務拓樸圖。 

Tempo datasource → Service Graph tab
（不在 Dashboard panel，而是 Explore 模式中可用）

或 Dashboard 用 Node Graph panel：

Visualization: Node Graph
Datasource: Tempo
Query type: Service Graph

9.3 Node Graph 適用

  • 看「哪個 service 是請求進入點」
  • 看「service 間 latency 與 error rate」
  • 找出依賴最多的 service（影響爆炸半徑）

§ 10. Plugin Panels 速覽

10.1 何時需要 plugin panel

內建 30+ panel 已涵蓋 95% 場景。但仍有特殊需求需要 plugin：

10.2 熱門 Plugin Panels

Plugin用途
Polystat多角形 status grid（小型 service status）
Status MapStatus grid 的另一種視覺
Plotly用 Plotly.js 畫複雜圖（科學視覺化）
Diagram (Flowcharting)用 mxgraph 畫流程圖（已 deprecated）
Boom Table強化版 table（更多 cell 樣式）
Singlestat (legacy)舊版 stat panel（已被 Stat 取代）
Discrete Panel離散狀態 timeline（已被 State Timeline 取代）

10.3 安裝 Plugin Panel

# grafana-cli
grafana-cli plugins install grafana-polystat-panel
sudo systemctl restart grafana-server

# Helm values.yaml
plugins:
  - grafana-polystat-panel
  - natel-plotly-panel
  - briangann-datatable-panel

10.4 Plugin 的選擇原則

⚠️ Plugin Panel 的取捨

慎選 plugin panel：
• 內建優先：能用內建就別用 plugin
• 檢查維護狀態：commits 半年沒更新可能不穩
• Angular plugin 已死：Grafana 12+ 完全不支援 Angular plugin
• Signed status：Grafana 12+ 預設只允許 signed plugin

✅ 本章重點摘要

30 秒回顧
