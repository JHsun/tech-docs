Chapter 36 · Panel 類型上篇

Panel 類型上篇：Time Series、Stat、Gauge、Bar Gauge、Table、Pie

Grafana 內建 30+ panel 類型，每個都有自己的最佳場景。本上篇涵蓋最常用的 6 種： Time Series（90% 使用率）、Stat（KPI 卡片）、Gauge（限制條件視覺化）、Bar Gauge、Table、Pie chart。 每個都附完整範例與配置技巧。

§ 1. Panel 共通設定

1.1 每個 Panel 都有的四個 tab

Tab用途
Query寫 query（PromQL / SQL / etc.）
Transform對 query 結果後處理（filter / merge / calculate）
Alert從 panel 建立 alert（v8+）
Panel options右側 sidebar，所有視覺化設定

1.2 通用 Panel options

  • Title / Description：description 顯示在 panel info icon
  • Transparent background：透明，整合 dashboard 背景
  • Repeat options：對 variable 重複（Ch 35）
  • Links：Panel-level 跳轉（Ch 35）

1.3 Field 概念

Grafana 把 query 結果統一成 Data Frame（Ch 29 提過），每個 column = 一個 field。 Field 可以個別設定 unit、min/max、color、threshold。 

Time              | service | qps   ← 每個 column 是一個 field
2026-05-02 10:00  | api     | 1234
2026-05-02 10:01  | api     | 1267
2026-05-02 10:00  | web     | 567

Field "qps" 可以設：
  unit: req/s
  thresholds: [0=green, 1000=yellow, 5000=red]
  decimals: 0
  color scheme: ...

§ 2. Time Series（最常用）

使用率最高的 panel 類型。畫時間序列折線 / 柱狀圖。取代了舊版 Graph panel（v7-）。

2.1 基本設定

Visualization: Time series

Query A:
  expr: rate(http_requests_total[$__rate_interval])
  legend: '{{ method }} {{ status }}'

2.2 重要視覺選項

Graph styles

  • Lines：折線（預設，最通用）
  • Bars：柱狀
  • Points：散點

Line interpolation

  • Linear：直線連
  • Smooth：曲線（看起來漂亮但 hide 真實 data point）
  • Step before / after：階梯（適合「狀態」類資料）

Fill opacity

0-100。10-25 適合多 series 疊加；100 = 滿填（適合單 series）。

Stack series

  • None：各 series 獨立
  • Normal：堆疊（總和顯示頂部）
  • Percent：每個 series 佔比 0-100%

Connect null values

  • Never：null 處斷線（看出 missing data）
  • Always：強制連起來（hide 問題，少用）
  • Threshold：超過 N 秒的 gap 才斷

2.3 Legend 設定

Legend mode:
  - List (簡單列表)
  - Table (含 calculations)
  - Hidden

Placement: Bottom / Right

Calculations: 當 mode=Table 時，顯示這些統計
  - Last (預設)
  - Mean
  - Max / Min
  - Total
  - Count
  - First / Last *

% mark：取「最後一個非 null 值」（推薦過 Last）

2.4 Tooltip

Mode說明
Single只顯示游標 hover 那條 series
All顯示同 X 的所有 series（debug 友善）
Hidden不顯示

2.5 Axis 設定

Y-Axis:
  Placement: Auto / Left / Right / Hidden
  Label: 自訂 axis 標籤
  Width: Auto / 固定 px
  Soft min / Soft max: 預設範圍但允許資料超出
  Scale: Linear / Logarithmic（基於 base 2 / 10）

# 多軸：適合不同單位疊加（QPS + latency 在同一 graph）
Field override → Axis placement → Right

2.6 Threshold

Thresholds:
  0  → Green   (Base)
  80 → Yellow  (Warning)
  95 → Red     (Critical)

Show thresholds:
  - As lines (橫線)
  - As lines (dashed)
  - As filled regions (整段染色)
  - Off

Threshold 配 Field unit 才能正確顯示百分比 / 秒

2.7 完整 Time Series 範例

HTTP latency p99 panel：

Title: HTTP P99 Latency
Query:
  expr: histogram_quantile(0.99,
    sum by (le, service) (
      rate(http_request_duration_seconds_bucket[$__rate_interval])
    )
  )
  legend: '{{ service }}'

Settings:
  - Graph style: Lines
  - Line width: 2
  - Fill opacity: 10
  - Connect null values: Threshold (10s)
  - Legend: Table, with Last/Mean/Max
  - Y-Axis unit: Time/seconds (s)
  - Thresholds: 0=green, 0.5=yellow, 2=red
  - Show thresholds: As filled regions

§ 3. Stat Panel（KPI 卡片）

Stat panel 顯示單一大數字（可帶 sparkline），是 dashboard 頂部 KPI 列的標配。

3.1 何時用 Stat

  • 「現在 QPS 是多少」
  • 「Error rate 多少 %」
  • 「Available 機器數」
  • 「Last deployment 多久前」

3.2 基本設定

Visualization: Stat

Query:
  expr: sum(rate(http_requests_total[5m]))

Stat options:
  - Color mode: Value / Background / Background gradient / None
  - Graph mode: Area / None    ← 顯示 sparkline
  - Justify mode: Auto / Center
  - Text mode: Value / Value and name / Name / None
  - Wide layout: 高度大時用

Value options:
  - Show: Calculate / All values
  - Calculation: Last * (預設) / Mean / Max / Min / Total
  - Fields: Numeric fields / All fields / Specific fields

3.3 Color Mode 對比

Mode視覺適合
Value數字本身染色淡主題、保守
Background整個 panel 染色狀態指示（紅 = ALARM）
Background gradient底色漸層美化

3.4 Threshold 配 Stat

Stat 沒設 threshold 就只是「藍色一片」沒意義。範例：

Metric: error_rate (%)
Thresholds:
  0   → Green
  0.5 → Yellow
  2   → Red

效果：error_rate = 0.3% → 綠色顯示
       error_rate = 1%   → 黃色
       error_rate = 5%   → 紅色（很顯眼）

3.5 多 series 處理

若 query 回傳多個 series，stat 預設顯示「每個 series 一個 stat」：

Query: rate(up == 0)
  → 回傳每個 instance 的 down 狀態

Stat 顯示：
  [api-1: 1] [api-2: 0] [db-1: 0]    ← 一格一個

Show: Calculate → 變成 [Total: 1]      ← 合併成一個
Calculation: Total

3.6 範例：Service Overview KPI 列

4 個 stat panel 並排：

[QPS]            [Error %]         [P99 (ms)]      [Saturation]
sum(rate())      err/total         histogram_q99    cpu utilization

Threshold:
QPS: 0=green, 1000=yellow, 5000=red
Err: 0=green, 1=yellow, 5=red
P99: 0=green, 500=yellow, 2000=red
Sat: 0=green, 70=yellow, 90=red

§ 4. Gauge Panel

Gauge 像「儀表板」，展示「現值在 min-max 區間的位置」。 視覺強烈，但只適合有明確上下限的 metric。

4.1 何時用 Gauge

適合不適合
CPU usage（0-100%）QPS（沒有自然上限）
Disk usage（0-100%）Latency（理論上無上限）
SLO budget（0-100%）Counter
Connection pool（0-max）

4.2 基本設定

Visualization: Gauge

Query: avg(node_cpu_usage)

Gauge options:
  - Show threshold labels: ✅
  - Show threshold markers: ✅
  - Min: 0
  - Max: 100
  - Unit: percent (0-100)

Thresholds:
  0  → Green
  70 → Yellow
  90 → Red

4.3 Min/Max 設定的重要性

⚠️ Gauge 必設 Min/Max

Gauge 預設 auto min/max，會根據資料自動調整。對 gauge 是災難：
「現值 80」在 max=100 時是「橘色」，在 max=10000 時是「綠色」幾乎滿格。
務必明確設 min/max。

4.4 Gauge 變體：Bar Gauge（更實用）

傳統圓弧 gauge 視覺漂亮但耗版面。Bar Gauge 是「條狀 gauge」， 更節省空間，能在一個 panel 顯示多個 instance 的狀態。

§ 5. Bar Gauge Panel

5.1 適用情境

顯示多個項目的當前值，每個都有同樣的 min-max 範圍。 例：「所有 instance 的 CPU usage」、「各 service 的 SLO budget」。

5.2 基本設定

Visualization: Bar gauge

Query:
  expr: 1 - avg by (instance) (rate(node_cpu_seconds_total{mode="idle"}[5m]))
  legend: '{{ instance }}'

Bar gauge options:
  - Orientation: Horizontal / Vertical / Auto
  - Display mode: Gradient / Retro LCD / Basic
  - Show unfilled area: ✅

Min: 0 / Max: 1
Unit: Percent (0.0-1.0)
Thresholds: 0=green, 0.7=yellow, 0.9=red

5.3 Display Mode 對比

Mode視覺
Gradient整條條漸層染色，視覺平和
Retro LCD分段格子，類似電子計時器
Basic單純的條，依 threshold 整段染色

5.4 Top N 範例

Title: Top 10 services by error rate

Query:
  expr: topk(10,
    sum by (service) (rate(http_requests_total{status=~"5.."}[5m]))
    / sum by (service) (rate(http_requests_total[5m]))
  )

Bar gauge:
  - Orientation: Horizontal
  - Display mode: Gradient
  - Min: 0, Max: 0.1 (10%)
  - Unit: Percent (0-1)
  - Thresholds: 0=green, 0.01=yellow, 0.05=red

§ 6. Table Panel

6.1 何時用 Table

  • 列出多個 instance / service 的多個維度（CPU + memory + disk + uptime）
  • 顯示分類資料（top N error 列表）
  • SQL 查詢結果（業務資料）
  • 狀態 dashboard（每個 service 的 health 列表）

6.2 基本設定

Visualization: Table

Query A:
  expr: up
  format: Table              ← 重要！
  instant: true              ← 取「現值」

Query B:
  expr: rate(node_cpu_seconds_total[5m])
  format: Table
  instant: true

# Transform → Merge: 把 A 與 B 按共同 label 合併

6.3 Table 進階：Cell Type

每個 column 的呈現方式：

Cell Type用途
Auto純文字 / 數字
Colored text文字依 threshold 染色
Colored background整個 cell 染色
Gaugecell 內畫 gauge bar
Sparklinecell 內畫迷你 time series（v9+）
Boolean顯示 ✅/❌
JSON view展開 JSON
Image顯示 URL 為圖片
Data links點擊跳轉

6.4 範例：Service Status Table

Title: Production Services

Query A: up                        (label: service)
Query B: rate(http_5xx_total[5m])  (label: service)
Query C: histogram_quantile(0.99, ...)  (label: service)

Transform 1: Merge by label "service"
Transform 2: Organize fields → 排序與重命名

Table:
  service     | Status | 5xx Rate | P99   | [Action]
  ────────────┼────────┼──────────┼───────┼─────────
  api         | ✅ Up  |   0.05%  |  234  | View
  web         | ✅ Up  |   0.02%  |  178  | View
  payment     | ❌ Down|   N/A    |  N/A  | View

Cell type:
  Status: Boolean (1=✅, 0=❌)
  5xx Rate: Colored background, threshold based
  P99: Colored text
  Action: Data link to detail dashboard

6.5 Pagination

v10+ 支援 pagination，大型 table（1000+ rows）友善：

Panel options → Footer:
  - Show table footer
  - Pagination: ✅
  - Page size: 50

§ 7. Pie Chart Panel

7.1 何時用 Pie（謹慎）

Pie chart 在資料視覺化界是最有爭議的——多數情境 bar chart 更好。但仍有適用場景：

  • 「成本分布」（CPU/Memory/Network 各佔多少）
  • 「請求來源比例」（mobile / web / API）
  • 少於 5 個分類，且分類有明顯比例差

7.2 基本設定

Visualization: Pie chart

Query:
  expr: sum by (status) (rate(http_requests_total[5m]))
  legend: '{{ status }}'

Pie chart options:
  - Pie chart type: Pie / Donut
  - Labels: Name / Value / Percent
  - Tooltip mode: Single / All

Legend:
  - Values: Percent / Value / Both
  - Placement: Bottom / Right

7.3 反面教材

⚠️ Pie chart 常被濫用

不要用 pie chart 顯示：

  • 5+ 個分類（人眼難分辨比例）
  • 各分類比例接近（看不出差異）
  • 時間序列資料（時間維度被丟棄）
  • 需要精確比較的（人眼看「面積」精度差）

大多數情況用 Bar chart 或 Stat panel + 百分比 比較好。

§ 8. Threshold 與 Color Mode

8.1 Threshold 設定

幾乎所有 panel 類型都用 threshold。設定位置：右側 Panel options → Thresholds。

Mode:
  - Absolute: 數值（80 表 80）
  - Percentage: 百分比（80% 表 max-min 的 80%）

Thresholds（從低到高）:
  Base    → Green
  60      → Yellow
  80      → Red

效果：
  value=50 → Green
  value=70 → Yellow
  value=90 → Red

8.2 Color Scheme

影響 panel 內 series 的色彩。Field options → Standard options → Color scheme：

Scheme說明
Single color所有 series 同色
Classic palette預設多色（綠、紅、紫...）
From thresholds (by value)依當前值在 threshold 中位置
Continuous (Green-Yellow-Red)漸層
Continuous (Blue-Yellow-Red)另一種漸層

8.3 Threshold 動態化

用 variable 控制 threshold（v9+）：

Variable: latency_threshold (custom: 100,200,500,1000)

Panel threshold:
  $latency_threshold → Yellow
  ${latency_threshold} * 2 → Red

# 使用者切換 variable，threshold 跟著動

§ 9. Field Override（精細控制）

當 query 回傳多個 field，每個可能要不同的 unit / threshold / display。 Field override 讓您針對特定 field 覆蓋預設。

9.1 何時用 Override

  • 「latency 用秒，error rate 用百分比，request count 用數字」
  • 「特定 series 用紅色顯示」
  • 「'staging' 環境的 series 顯示為虛線」

9.2 設定範例

Panel options → Overrides → Add field override

Field with name matching regex: ^.*p99$
  Standard options → Unit: ms
  Standard options → Color: red
  Graph styles → Line width: 3

Field with value: status="500"
  Color: red

Field with name: rate
  Unit: req/s
  Decimals: 1

9.3 進階範例：多 Y 軸

QPS（左軸）+ Latency（右軸）疊在一個 graph：

Query A: rate(http_requests_total[5m])
  Field name: qps
Query B: histogram_quantile(0.99, ...)
  Field name: latency

Override 1: Field name = qps
  Y-Axis placement: Left
  Unit: req/s

Override 2: Field name = latency
  Y-Axis placement: Right
  Unit: seconds (s)
  Color: red

10.1 三類常見場景的選型

場景建議 panel
「現在 QPS 多少」Stat（含 sparkline）
「過去 24h QPS 趨勢」Time Series
「20 台機器 CPU 比較」Bar Gauge（橫向）
「所有 service health 列表」Table（含 boolean / status）
「Top 10 慢 endpoint」Bar Gauge / Table
「Disk usage 70%」Gauge（單一）/ Bar Gauge（多 disk）
「3 個環境流量比例」Pie chart

10.2 練習：service overview dashboard

把所有學的 panel 用上：

第 1 列（Stat panel × 4）：
  [QPS] [Error %] [P99 ms] [Saturation %]
  顯示「現值」+ sparkline + threshold 染色

第 2 列：
  [Time Series 12wide] Request rate by status (stacked)
  [Time Series 12wide] Latency p50/p95/p99 overlay

第 3 列：
  [Bar Gauge horizontal] Top 10 services by error rate
  [Table] Active alerts (含 boolean status, color background)

第 4 列：
  [Time Series 24wide] Resource usage (CPU + memory 不同 axis)

30 秒回顧
  • 每個 panel 四個 tab：Query / Transform / Alert / Panel options
  • Time Series：90% 使用率，已取代舊 Graph panel。Lines / Bars / Points 三種 graph style
