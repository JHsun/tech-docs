Chapter 20 · Client Libraries

Client Libraries：六大語言完整對照

當您有自己的應用程式碼，直接用 client library 在程式裡埋 metric是最佳做法。 本章涵蓋六大主流語言的官方 client library：Go、Java、Python、Ruby、Rust、.NET。 每個都附完整的初始化、四大 metric 類型範例、與框架整合範例。

1.2 共通的四種 metric 類型

所有 client library 都提供：

  • Counter — 只增加
  • Gauge — 可上可下
  • Histogram — 觀測分布（buckets）
  • Summary — 觀測分布（client-side quantiles，少用）

§ 2. Go：client_golang

2.1 安裝

go get github.com/prometheus/client_golang/prometheus
go get github.com/prometheus/client_golang/prometheus/promauto
go get github.com/prometheus/client_golang/prometheus/promhttp

2.2 完整範例

package main

import (
	"net/http"
	"time"

	"github.com/prometheus/client_golang/prometheus"
	"github.com/prometheus/client_golang/prometheus/promauto"
	"github.com/prometheus/client_golang/prometheus/promhttp"
)

// 推薦：用 promauto 自動註冊
var (
	requestsTotal = promauto.NewCounterVec(
		prometheus.CounterOpts{
			Name: "myapp_http_requests_total",
			Help: "Total HTTP requests by method and path.",
		},
		[]string{"method", "path", "status"},
	)

	activeConnections = promauto.NewGauge(
		prometheus.GaugeOpts{
			Name: "myapp_active_connections",
			Help: "Currently active connections.",
		},
	)

	requestDuration = promauto.NewHistogramVec(
		prometheus.HistogramOpts{
			Name:    "myapp_http_request_duration_seconds",
			Help:    "Request duration in seconds.",
			Buckets: prometheus.DefBuckets, // 0.005, 0.01, 0.025, 0.05, 0.1, 0.25, 0.5, 1, 2.5, 5, 10
			// 啟用 native histogram（v1.20+）
			NativeHistogramBucketFactor:     1.1,
			NativeHistogramMaxBucketNumber:  100,
			NativeHistogramMinResetDuration: time.Hour,
		},
		[]string{"method", "path"},
	)
)

func handler(w http.ResponseWriter, r *http.Request) {
	start := time.Now()
	activeConnections.Inc()
	defer activeConnections.Dec()

	// 業務邏輯
	w.WriteHeader(http.StatusOK)
	w.Write([]byte("OK"))

	// 記錄 metric
	duration := time.Since(start).Seconds()
	requestsTotal.WithLabelValues(r.Method, r.URL.Path, "200").Inc()
	requestDuration.WithLabelValues(r.Method, r.URL.Path).Observe(duration)
}

func main() {
	http.HandleFunc("/api", handler)
	http.Handle("/metrics", promhttp.Handler())
	http.ListenAndServe(":8080", nil)
}

2.3 自動 instrument HTTP handler

用內建 middleware 簡化：

import "github.com/prometheus/client_golang/prometheus/promhttp"

httpHandler := http.HandlerFunc(yourBusinessLogic)
instrumented := promhttp.InstrumentHandlerDuration(requestDuration,
	promhttp.InstrumentHandlerCounter(requestsTotal, httpHandler))

http.Handle("/api", instrumented)

2.4 內建 collector

Go runtime metric（GC、goroutine、memory）自動暴露：

// 使用 promauto 時這些自動註冊（process_*, go_*）
// 若用裸 prometheus.Register，需手動：
prometheus.MustRegister(prometheus.NewBuildInfoCollector())
prometheus.MustRegister(prometheus.NewGoCollector())
prometheus.MustRegister(prometheus.NewProcessCollector(prometheus.ProcessCollectorOpts{}))

3.3 完整範例（client_java 1.x）

import io.prometheus.metrics.core.metrics.Counter;
import io.prometheus.metrics.core.metrics.Gauge;
import io.prometheus.metrics.core.metrics.Histogram;
import io.prometheus.metrics.exporter.httpserver.HTTPServer;
import io.prometheus.metrics.instrumentation.jvm.JvmMetrics;

public class App {
    static final Counter requestsTotal = Counter.builder()
        .name("myapp_http_requests_total")
        .help("Total HTTP requests")
        .labelNames("method", "path", "status")
        .register();

    static final Gauge activeConnections = Gauge.builder()
        .name("myapp_active_connections")
        .help("Active connections")
        .register();

    static final Histogram requestDuration = Histogram.builder()
        .name("myapp_http_request_duration_seconds")
        .help("Request duration")
        .labelNames("method", "path")
        .classicUpperBounds(0.005, 0.01, 0.025, 0.05, 0.1, 0.25, 0.5, 1, 2.5, 5, 10)
        .nativeMaxNumberOfBuckets(100)            // 啟用 native histogram
        .register();

    public static void main(String[] args) throws Exception {
        // 自動註冊 JVM metrics（記憶體、GC、threads...)
        JvmMetrics.builder().register();

        // 啟動 HTTP server 暴露 /metrics
        HTTPServer server = HTTPServer.builder()
            .port(8080)
            .buildAndStart();

        // 模擬業務
        requestsTotal.labelValues("GET", "/api", "200").inc();
        activeConnections.set(5);

        Histogram.Timer timer = requestDuration.labelValues("GET", "/api").startTimer();
        // do business
        timer.observeDuration();
    }
}

3.4 Spring Boot 整合（用 Micrometer）

Spring Boot 標準做法是用 Micrometer（Spring 自家 vendor-agnostic 抽象層）：

// pom.xml
<dependency>
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-registry-prometheus</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
# application.yml
management:
  endpoints:
    web:
      exposure:
        include: prometheus, health
  metrics:
    tags:
      application: myapp

啟動後 /actuator/prometheus 自動暴露所有 Spring Boot 內建 metric + 您自己加的。

§ 4. Python：prometheus_client

4.1 安裝

pip install prometheus-client

4.2 完整範例（Flask）

from flask import Flask, request, Response
from prometheus_client import Counter, Gauge, Histogram, generate_latest, CONTENT_TYPE_LATEST
import time

app = Flask(__name__)

# 定義 metric
REQUESTS = Counter(
    'myapp_http_requests_total',
    'Total HTTP requests',
    ['method', 'path', 'status']
)

ACTIVE = Gauge(
    'myapp_active_connections',
    'Active connections'
)

DURATION = Histogram(
    'myapp_http_request_duration_seconds',
    'Request duration',
    ['method', 'path'],
    buckets=[0.005, 0.01, 0.05, 0.1, 0.5, 1, 5]
)

# Middleware
@app.before_request
def before():
    request._start = time.time()
    ACTIVE.inc()

@app.after_request
def after(response):
    duration = time.time() - request._start
    REQUESTS.labels(request.method, request.path, response.status_code).inc()
    DURATION.labels(request.method, request.path).observe(duration)
    ACTIVE.dec()
    return response

# 暴露 /metrics
@app.route('/metrics')
def metrics():
    return Response(generate_latest(), mimetype=CONTENT_TYPE_LATEST)

# 業務 endpoint
@app.route('/api/users')
def users():
    return {"users": []}

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8080)

4.3 Decorator 簡化

# Counter 自動 +1（exception 時）
@REQUESTS.count_exceptions()
def risky_function():
    ...

# 自動量耗時
@DURATION.time()
def slow_function():
    ...

# 變成 context manager
with DURATION.labels('GET', '/api').time():
    do_business()

4.4 多進程模式（重要！）

⚠️ Python 多 worker（gunicorn / uwsgi）特殊處理

Python prometheus_client 預設假設單進程。 若您用 gunicorn（多 worker）、uwsgi 等多進程框架，每個 worker 持有獨立 counter， Prometheus 抓到的值會根據碰到哪個 worker 跳動。
解法：啟用 multiprocess mode，所有 worker 共用一個目錄寫 metric： 

import os
os.environ['PROMETHEUS_MULTIPROC_DIR'] = '/tmp/prometheus_multiproc'

from prometheus_client import multiprocess, generate_latest, CollectorRegistry

@app.route('/metrics')
def metrics():
    registry = CollectorRegistry()
    multiprocess.MultiProcessCollector(registry)
    return Response(generate_latest(registry), mimetype=CONTENT_TYPE_LATEST)

並在 gunicorn 啟動前清空該目錄。

§ 5. Ruby、Rust、.NET 速覽

5.1 Ruby

# Gemfile
gem 'prometheus-client'

# config.ru
require 'prometheus/middleware/collector'
require 'prometheus/middleware/exporter'

use Prometheus::Middleware::Collector
use Prometheus::Middleware::Exporter

# 自訂 metric
require 'prometheus/client'
prometheus = Prometheus::Client.registry
counter = prometheus.counter(:myapp_requests_total, docstring: 'Total requests', labels: [:method])
counter.increment(labels: { method: 'GET' })

5.2 Rust

# Cargo.toml
[dependencies]
prometheus-client = "0.22"
hyper = "0.14"
use prometheus_client::registry::Registry;
use prometheus_client::metrics::counter::Counter;
use prometheus_client::metrics::histogram::Histogram;
use prometheus_client::encoding::text::encode;

let mut registry = Registry::default();

let requests = Counter::<u64>::default();
registry.register("myapp_requests_total", "Total requests", requests.clone());

requests.inc();

// 暴露
let mut buffer = String::new();
encode(&mut buffer, &registry).unwrap();
println!("{}", buffer);

5.3 .NET（prometheus-net）

// Program.cs (ASP.NET Core)
using Prometheus;

var builder = WebApplication.CreateBuilder(args);
var app = builder.Build();

// 自動 instrument HTTP requests
app.UseHttpMetrics();

// 暴露 /metrics

RequestsTotal.WithLabels("GET", "/api").Inc();

§ 6. 跨語言通用模式

6.1 模式 1：Counter — 計數事件

# Pseudocode
requests_total = Counter('http_requests_total', ..., ['method', 'status'])

# 每個請求結束後
requests_total.labels('GET', '200').inc()

6.2 模式 2：Gauge — 反映瞬時狀態

queue_depth = Gauge('queue_depth', ..., ['queue'])

# 改變狀態時更新
queue_depth.labels('default').set(42)
queue_depth.labels('default').inc()
queue_depth.labels('default').dec()

6.3 模式 3：Histogram — 觀測延遲

duration = Histogram('request_duration_seconds', ..., ['endpoint'],
    buckets=[0.005, 0.01, 0.05, 0.1, 0.5, 1, 5])

# 量耗時
start = time.time()
handle_request()
duration.labels(endpoint).observe(time.time() - start)

6.4 模式 4：Decorator / Context Manager 自動量

# Python decorator
@duration.labels(endpoint='/api').time()
def handle_api():
    ...

# Java try-with-resources
try (Histogram.Timer timer = duration.labelValues("/api").startTimer()) {
    handleApi();
}

# Go defer
timer := prometheus.NewTimer(duration.WithLabelValues("/api"))
defer timer.ObserveDuration()

Native histogram 各語言 client 支援度（2026 現狀）：

語言狀態

Java（client_java 1.x）★★★ 穩定

7.1 Go 啟用方式

requestDuration := promauto.NewHistogramVec(
    prometheus.HistogramOpts{
        Name:    "request_duration_seconds",
        // 不需要 Buckets（如果不要 classic）
        // 或同時提供 classic Buckets，雙模式輸出
        Buckets: prometheus.DefBuckets,

        // 啟用 native histogram
        NativeHistogramBucketFactor:     1.1,    // bucket 增長因子（越小越精確）
        NativeHistogramMaxBucketNumber:  100,    // bucket 數上限
        NativeHistogramMinResetDuration: time.Hour,  // 最小 reset 間隔
    },
    []string{"endpoint"},
)

7.2 同時暴露 classic + native

若您的 Prometheus 還不支援 native histogram，但客戶端先升級了：

# Server 端 scrape config（v2.40+）
scrape_configs:
  - job_name: 'app'
    # 不啟用 native ⇒ 只看 classic _bucket
    static_configs:
      - targets: ['app:8080']

# 啟用後
    scrape_protocols:
      - PrometheusProto    # 才能解析 native histogram

§ 8. Exemplars 與 OTel 整合

8.1 Exemplar 是什麼

Exemplar 是「綁在 metric sample 上的 trace ID」，讓 dashboard 點開 metric 圖表能直接跳到對應 trace。 這是 metrics ↔ traces 串接的關鍵。

8.2 Go 範例

import "github.com/prometheus/client_golang/prometheus"

// HistogramVec 支援 ObserveWithExemplar
duration.WithLabelValues("/api").(prometheus.ExemplarObserver).ObserveWithExemplar(
    elapsed.Seconds(),
    prometheus.Labels{
        "trace_id": traceID,
        "span_id":  spanID,
    },
)

8.3 暴露 Exemplar 的格式

Exemplar 在 OpenMetrics 格式中以 # {trace_id="abc"} 0.123 表示：

app_request_duration_seconds_bucket{le="0.5"} 1234 # {trace_id="abc123"} 0.345 1714521600
app_request_duration_seconds_bucket{le="1.0"} 5678

Prometheus 用 --enable-feature=exemplar-storage 啟用 exemplar 儲存， Grafana 在 panel 點圖時會顯示 trace ID 連結。

8.4 與 OpenTelemetry 共存

若您用 OTel SDK 同時做 traces 與 metrics，OTel SDK 自動處理 exemplar。 OTel SDK 的 metric 也能用 OTLP 推到 Prometheus（v2.47+ 支援 OTLP receiver，見 Ch 04）。

§ 9. 常見陷阱

⚠️ 陷阱 1：在 hot path 用 string format 算 label

requests.WithLabelValues(fmt.Sprintf("user-%d", id)) 在每個請求都建立 string， 還會產生百萬條 series。
解法：label 值用固定 enum，動態值放 logs/traces。

⚠️ 陷阱 2：在請求路徑用 metric initialization

每個 request 都 NewCounter = 重新註冊 = 報錯或 metric duplicate。
Counter / Gauge 等應該在啟動時初始化一次，後續用同一個 instance。

⚠️ 陷阱 3：Counter _total 後綴沒加

OpenMetrics 規範要求 counter 必須以 _total 結尾。 Java client_java 1.x 會自動拒絕沒有 _total 的 counter。

⚠️ 陷阱 4：在 Python 用多 worker 沒設 multiprocess mode

Counter 在不同 worker 各自累積，Prometheus 抓到的值會跳動。 多 worker 環境必設 PROMETHEUS_MULTIPROC_DIR。

⚠️ 陷阱 5：Histogram bucket 太多 / 太少

預設 10 個 bucket 對 99% 場景夠用。
要 finer 用 native histogram，不要把 classic bucket 加到 50+ 個。

⚠️ 陷阱 6：用 Summary 跨 instance 平均

Summary 算的是 client-side quantile，多 instance 數學上不能聚合。 除非單機監控，都用 Histogram。
