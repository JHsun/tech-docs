Chapter 26 · Alertmanager Templates

Alertmanager Templates 完整教學

寫一個能讓人凌晨三點看懂的告警通知，是 SRE 的軟實力。 Alertmanager 用 Go template 系統渲染通知內容——學會它能把「Alert is firing」這種垃圾訊息 轉成包含 severity、affected pods、runbook、dashboard 連結的精美卡片。

§ 1. Template 系統運作原理

Alertmanager 收到 Prometheus 送來的 alert 後，要把它們組合並渲染成「給人看的訊息」。 這個渲染由 Go template 完成，使用 Go 標準庫的 text/template 與 html/template。

1.1 何時呼叫 template

每一個 receiver 的每一個欄位都可能是 template，例如：

slack_configs:
  - channel: '#alerts-{{ .CommonLabels.team }}'      # ← template
    title: '{{ template "slack.title" . }}'           # ← 命名 template
    text: |                                           # ← 內聯多行 template
      {{ range .Alerts }}
      *{{ .Labels.alertname }}*
      {{ .Annotations.summary }}
      {{ end }}

1.2 兩種寫法

寫法適用
內聯（inline）簡單一兩行的 template，直接寫在 alertmanager.yml
外部檔案（external）多行複雜 template，寫在 .tmpl 檔，用 {{ template "name" . }} 引用

1.3 引用外部 template

# alertmanager.yml
templates:
  - '/etc/alertmanager/templates/*.tmpl'    # 載入這些 .tmpl 檔

receivers:
  - name: 'slack'
    slack_configs:
      - channel: '#alerts'
        title: '{{ template "slack.title" . }}'
        text:  '{{ template "slack.text"  . }}'
# /etc/alertmanager/templates/slack.tmpl
{{ define "slack.title" }}
[{{ .Status | toUpper }}] {{ .CommonLabels.alertname }}
{{ end }}

{{ define "slack.text" }}
{{ range .Alerts }}
Severity: {{ .Labels.severity }}
Summary: {{ .Annotations.summary }}
{{ end }}
{{ end }}

§ 2. Template Data 完整結構

每次 template 渲染時，Alertmanager 傳進來的資料叫做 Data，結構如下：

type Data struct {
    Receiver         string         // 當前 receiver 名稱
    Status           string         // "firing" 或 "resolved"
    Alerts           Alerts         // 此通知包含的所有 alert
    GroupLabels      KV             // 用於分組的 label
    CommonLabels     KV             // 所有 alert 都共有的 label
    CommonAnnotations KV            // 所有 alert 都共有的 annotation
    ExternalURL      string         // Alertmanager 的外部 URL
}

2.1 Alert 物件結構

type Alert struct {
    Status       string     // "firing" 或 "resolved"
    Labels       KV         // alert 的所有 label
    Annotations  KV         // alert 的所有 annotation
    StartsAt     time.Time  // 開始時間
    EndsAt       time.Time  // 結束時間（仍 firing 時為 future）
    GeneratorURL string     // Prometheus 端的查詢連結
    Fingerprint  string     // alert 唯一 hash
}

2.2 Alerts 集合的便利方法

// 從 .Alerts 可以取出 firing / resolved 子集
{{ .Alerts.Firing | len }}        // 正在 firing 的數量
{{ .Alerts.Resolved | len }}      // 已 resolved 的數量

// 對所有 alert 迭代
{{ range .Alerts }}
  ...
{{ end }}

// 只對 firing 迭代
{{ range .Alerts.Firing }}
  ...
{{ end }}

2.3 KV 的用法

KV（key-value pairs）有幾個有用的方法：

// 直接存取（最常用）
{{ .CommonLabels.severity }}      // 取出 severity
{{ .Labels.alertname }}            // 取出 alertname

// 排序後迭代（很實用！）
{{ range .CommonLabels.SortedPairs }}
  {{ .Name }}: {{ .Value }}
{{ end }}

// 取得名稱列表
{{ .CommonLabels.Names }}

// 取得值列表
{{ .CommonLabels.Values }}

// 排除某些 label
{{ .CommonLabels.Remove (list "alertname" "severity") }}

2.4 一個 Data 範例

以下是 webhook 收到的 JSON，可幫助理解結構：

{
  "receiver": "slack",
  "status": "firing",
  "alerts": [
    {
      "status": "firing",
      "labels": {
        "alertname": "HighErrorRate",
        "severity": "critical",
        "service": "api",
        "instance": "api-1:9090"
      },
      "annotations": {
        "summary": "API error rate > 5%",
        "runbook_url": "https://wiki/api-errors"
      },
      "startsAt": "2026-05-02T10:23:45Z",
      "endsAt": "0001-01-01T00:00:00Z",
      "generatorURL": "https://prometheus/graph?...",
      "fingerprint": "abc123"
    }
  ],
  "groupLabels": {"alertname": "HighErrorRate"},
  "commonLabels": {
    "alertname": "HighErrorRate",
    "severity": "critical",
    "service": "api"
  },
  "commonAnnotations": {"summary": "API error rate > 5%"},
  "externalURL": "https://alertmanager.example.com"
}

§ 3. Go Template 基礎語法

3.1 基本輸出

// 印出值
{{ .CommonLabels.severity }}
{{ .Status }}

// 函式調用
{{ len .Alerts }}
{{ upper .Status }}

// Pipeline（左到右串接）
{{ .Status | toUpper }}
{{ .CommonLabels.severity | upper | printf "Sev: %s" }}

3.2 條件判斷

// 簡單 if
{{ if eq .Status "firing" }}
  🔥 ALERT FIRING
{{ end }}

// if / else
{{ if eq .Status "firing" }}
  🔥
{{ else }}
  ✅
{{ end }}

// if / else if / else
{{ if eq .CommonLabels.severity "critical" }}
  CRITICAL
{{ else if eq .CommonLabels.severity "warning" }}
  WARNING
{{ else }}
  INFO
{{ end }}

// 比較運算
{{ if gt (len .Alerts) 5 }}
  Too many alerts!
{{ end }}

3.3 迴圈

// 迭代 alert list
{{ range .Alerts }}
  - {{ .Labels.alertname }}: {{ .Annotations.summary }}
{{ end }}

// 取得迭代索引
{{ range $i, $alert := .Alerts }}
  Alert #{{ $i }}: {{ $alert.Labels.alertname }}
{{ end }}

// 迭代 KV map（要先 SortedPairs）
{{ range .CommonLabels.SortedPairs }}
  {{ .Name }}={{ .Value }}
{{ end }}

// range 裡 . 變了，要存原 context 用 $
{{ $root := . }}
{{ range .Alerts }}
  Receiver: {{ $root.Receiver }}     // 從 root 取
  Alert: {{ .Labels.alertname }}     // 從當前 alert 取
{{ end }}

3.4 變數宣告

// 宣告變數
{{ $count := len .Alerts }}
There are {{ $count }} alerts.

{{ $crit_color := "#FF0000" }}
{{ $warn_color := "#FFAA00" }}
{{ if eq .CommonLabels.severity "critical" }}
  Color: {{ $crit_color }}
{{ end }}

3.5 比較運算子

運算子意義
eq等於
ne不等於
lt / le小於 / 小於等於
gt / ge大於 / 大於等於
and / or / not邏輯

§ 4. 內建函式與字串處理

Alertmanager 在 Go template 之上加了 DefaultFuncs，常用的：

4.1 字串函式

函式用途範例
toUpper轉大寫{{ .Status | toUpper }}
toLower轉小寫{{ "ABC" | toLower }}
title標題化（首字大寫）{{ "hello world" | title }}
trimSpace去前後空白{{ .X | trimSpace }}
join用分隔符串接{{ .CommonLabels.Values | join ", " }}
matchregex 匹配{{ if match "^api" .Labels.service }}...{{ end }}
reReplaceAllregex 替換{{ reReplaceAll "/users/\\d+" "/users/:id" .Labels.path }}
safeHtml標記為「安全 HTML」（不 escape）用於 email html template
stringSlice把多個字串組成 slice{{ stringSlice "a" "b" "c" }}

4.2 排序與切片

// SortedPairs 是最常用的（讓輸出穩定）
{{ range .CommonLabels.SortedPairs }}
  {{ .Name }}: {{ .Value }}
{{ end }}

// Remove 排除某些 label
{{ range (.CommonLabels.Remove (stringSlice "alertname")).SortedPairs }}
  {{ .Name }}: {{ .Value }}
{{ end }}

4.3 數字格式化（用 printf）

{{ printf "%.2f" 3.14159 }}                       // 3.14
{{ printf "%d alerts" (len .Alerts) }}            // 5 alerts
{{ printf "%-10s %s" .Labels.severity .Annotations.summary }}

4.4 自訂簡單轉換

// 例：把 severity 轉成 emoji
{{ define "severity.emoji" }}
{{- if eq . "critical" }}🔴
{{- else if eq . "warning" }}🟡
{{- else }}🔵
{{- end -}}
{{ end }}

// 使用
{{ template "severity.emoji" .CommonLabels.severity }} {{ .CommonLabels.alertname }}

§ 5. 外部 template 檔案管理

5.1 推薦的目錄結構

/etc/alertmanager/
├── alertmanager.yml
└── templates/
    ├── shared.tmpl          # 共用元件（severity emoji, color 等）
    ├── slack.tmpl
    ├── email.tmpl
    ├── pagerduty.tmpl
    └── webhook.tmpl

5.2 引用方式

# alertmanager.yml
templates:
  - '/etc/alertmanager/templates/*.tmpl'

所有 .tmpl 檔被合併到單一 namespace。命名要小心衝突。

5.3 共用元件範例（shared.tmpl）

{{/* 共用：依 severity 取 emoji */}}
{{ define "shared.severityEmoji" }}
{{- if eq . "critical" }}🔴{{- else if eq . "warning" }}🟡{{- else if eq . "info" }}ℹ️{{- else }}❓{{- end -}}
{{ end }}

{{/* 共用：依 status 取 emoji */}}
{{ define "shared.statusEmoji" }}
{{- if eq . "firing" }}🔥{{- else }}✅{{- end -}}
{{ end }}

{{/* 共用：依 severity 取顏色 hex */}}
{{ define "shared.color" }}
{{- if eq . "critical" }}#E01E5A
{{- else if eq . "warning" }}#ECB22E
{{- else if eq . "info" }}#36C5F0
{{- else }}#999999{{- end -}}
{{ end }}

{{/* 共用：產生 instance + service 的人類可讀名 */}}
{{ define "shared.contextLine" }}
{{- if .Labels.service -}}
{{ .Labels.service }}
{{- end -}}
{{- if .Labels.instance -}}
 @ {{ .Labels.instance }}
{{- end -}}
{{ end }}

之後 slack/email 等都能引用這些共用 component。

§ 8. PagerDuty / Webhook templates

8.1 PagerDuty 簡化模板

PagerDuty 的 details 欄位是個 KV map，每個 value 都是 template：

pagerduty_configs:
  - routing_key: ''
    description: '{{ template "pagerduty.description" . }}'
    severity: '{{ if eq .CommonLabels.severity "critical" }}critical{{ else }}warning{{ end }}'
    details:
      firing_count: '{{ .Alerts.Firing | len }}'
      resolved_count: '{{ .Alerts.Resolved | len }}'
      service: '{{ .CommonLabels.service }}'
      cluster: '{{ .CommonLabels.cluster }}'
      runbook: '{{ (index .Alerts 0).Annotations.runbook_url }}'
      summary: '{{ (index .Alerts 0).Annotations.summary }}'
{{ define "pagerduty.description" -}}
[{{ .CommonLabels.severity | toUpper }}] {{ .CommonLabels.alertname }}{{ if .CommonLabels.service }} on {{ .CommonLabels.service }}{{ end }} ({{ .Alerts | len }} alert{{ if gt (len .Alerts) 1 }}s{{ end }})
{{- end }}

8.2 Webhook payload 自訂

Webhook receiver 不能自訂 payload（永遠送固定的 Alertmanager Data 結構）。 若要自訂格式，需在中介層（自家 webhook server）轉換。

但可在呼叫的 URL 用 template：

webhook_configs:
  - url: 'https://incidents.example.com/api/v1/team/{{ .CommonLabels.team }}/alerts'
    send_resolved: true
    http_config:
      bearer_token_file: /etc/alertmanager/secrets/webhook-token

§ 9. Template 開發與測試流程

9.1 用 amtool 渲染 template

# 渲染指定 template
amtool template render \
  --template.glob='/etc/alertmanager/templates/*.tmpl' \
  --template.text='{{ template "slack.title" . }}' \
  --template.data='{
    "Receiver": "slack",
    "Status": "firing",
    "Alerts": [{
      "Status": "firing",
      "Labels": {"alertname": "TestAlert", "severity": "critical"},
      "Annotations": {"summary": "Test summary"},
      "StartsAt": "2026-05-02T10:00:00Z",
      "EndsAt": "0001-01-01T00:00:00Z",
      "GeneratorURL": "https://prometheus/test"
    }],
    "GroupLabels": {"alertname": "TestAlert"},
    "CommonLabels": {"alertname": "TestAlert", "severity": "critical"},
    "CommonAnnotations": {"summary": "Test summary"},
    "ExternalURL": "https://alertmanager.example.com"
  }'

9.2 用真實 alert 測試

POST 一個測試 alert 到 Alertmanager，看實際 Slack/Email 渲染：

  -d '[{
    "labels": {
      "alertname": "TemplateTestAlert",
      "severity": "warning",
      "service": "api",
      "instance": "api-1:9090",
      "team": "platform"
    },
    "annotations": {
      "summary": "Testing the new Slack template",
      "description": "This is just for template testing",
      "runbook_url": "https://wiki/test"
    },
    "startsAt": "2026-05-02T10:00:00Z"
  }]'

9.3 Template 開發 Loop

  1. 修改 .tmpl 檔
  2. amtool check-config 驗證語法（包含 template 解析）
  3. amtool template render 看渲染結果
  4. POST 真 alert → 看 receiver 端實際呈現
  5. POST /-/reload 重新載入

§ 10. 常見陷阱

⚠️ 陷阱 1：template 名稱衝突

所有 .tmpl 檔合併到單一 namespace。 若您在 slack.tmpl 與 email.tmpl 都 define "title"， 後載入的會覆蓋前者，導致一個 receiver 用了另一個的 template。
解法：用前綴命名（slack.title / email.title）。

⚠️ 陷阱 2：access nil pointer

{{ .Annotations.summary }} 在 annotation 不存在時不會壞掉（Go template 容錯）， 但 {{ .Annotations.summary | toUpper }} 會失敗（nil 不能 pipeline）。
解法：用 default 或 if： 

{{ if .Annotations.summary }}{{ .Annotations.summary }}{{ else }}No summary{{ end }}
⚠️ 陷阱 3：HTML escape 問題

HTML template 自動 escape <、> 等字元。 若您要插入已經是 HTML 的內容，會看到 &lt;br&gt;。
解法：用 safeHtml：{{ .X | safeHtml }}。
但小心 XSS——只用在已驗證的內容上。

⚠️ 陷阱 4：忘記 send_resolved 或 status 區分

Template 收到 resolved alert 時，.Status = "resolved"。 若 template 永遠當 firing 寫，resolved 通知會看起來很怪（仍寫「ALERT」）。
解法：在 title / color 都用 if eq .Status "firing" 區分。

⚠️ 陷阱 5：時間格式

Go 的時間格式很特殊，必須用「2006-01-02 15:04:05」這個固定參考時間， 不是 YYYY-MM-DD HH:mm:ss。
常用格式：
• "2006-01-02 15:04:05 MST" → 2026-05-02 10:23:45 UTC
• "Jan 02 15:04:05" → May 02 10:23:45
• "2006-01-02T15:04:05Z07:00" → ISO 8601

⚠️ 陷阱 6：忘記 trim 空白

Go template 的 {{ }} 會留下空白行。用 {{- ... -}} 去除前後空白： 

{{ if .X }}             // 留下空行
content
{{ end }}

{{- if .X -}}           // 緊貼
content
{{- end -}}
