APPENDIX E · 推薦資源

Books, Blogs, Podcasts, Conferences
讀完本書之後 — 持續學習的路標

本書的目的是建立基礎。但 reverse proxy 工程師持續學習是必須的 — 技術每 5 年大改一次。本附錄給你讀完本書之後的延伸資源：書單、blog、podcast、conference、community。不求面面俱到，求真實有用。

Epilogue 已給出 5 條進階閱讀路線（協定底層 / 分散式系統 / 性能 Observability / Security / 軟體工程文化）。本附錄是更具體的延伸：明確的書名、blog URL、podcast 名稱、conference 連結。建議：選 1-2 個 blog 加 RSS、訂閱 1 個 podcast、每年看 1-2 場 conference talk、長期 commit 學習。

A · 基礎書單（給入門到中階）

讀完本書、想再深化基礎的下一站：

  • 《High Performance Browser Networking》 — Ilya Grigorik（O'Reilly 2013，2nd ed. ongoing online free）。HTTP/2、TLS、WebRTC 等的頂級解釋，免費讀於 hpbn.co
  • 《Computer Networking: A Top-Down Approach》 — Kurose & Ross（8th ed. 2020）。經典大學教材，TCP/IP、DNS、CDN 都從上往下解
  • 《HTTP/2 in Action》 — Barry Pollard（Manning 2019）。雖然 2019，但 HTTP/2 的核心沒變，是最易讀的 H2 入門
  • 《Linux Performance: Out of the Box》 — Brendan Gregg（系列文章 + workshop）— 進入 performance engineering 的第一站
  • 《Nginx HTTP Server》 — Clément Nedelcu（Packt 多版本）— Nginx 配置實務

B · 進階書單（給已經 production-ready 工程師）

  • 《Designing Data-Intensive Applications》 — Martin Kleppmann（O'Reilly 2017）。必讀。對分散式系統、資料一致性、共識、多 region 架構的最完整當代著作。下一版未定（作者寫 2nd ed. 中）
  • 《Database Internals》 — Alex Petrov（O'Reilly 2019）。Storage engine、B-Tree vs LSM、replication、distributed transactions 深度解
  • 《Systems Performance, 2nd Edition》 — Brendan Gregg（Pearson 2020）。USE method 起源、CPU / memory / disk / network 全方位 performance engineering
  • 《BPF Performance Tools》 — Brendan Gregg（Pearson 2019）。bpftrace、bcc tools 完整指南，eBPF 工程師必讀
  • 《Observability Engineering》 — Charity Majors, Liz Fong-Jones, George Miranda（O'Reilly 2022）。Honeycomb 視角的 observability 2.0 範式
  • 《Real-World Cryptography》 — David Wong（Manning 2021）。比學術 crypto 書平易、給工程師的實用密碼學
  • 《Cloud Native Patterns》 — Cornelia Davis（Manning 2019）。K8s 時代的應用設計模式

C · 經典必讀（不會過時的智慧）

  • 《TCP/IP Illustrated, Volume 1》 — W. Richard Stevens（1994）。30 年前的書，但 TCP/IP 仍如此。Stevens 的清晰寫作至今未被超越
  • 《The Mythical Man-Month》 — Fred Brooks（1975, anniversary ed. 1995）。軟體工程的永恆真理，Brooks's Law 出處
  • 《The Pragmatic Programmer, 20th Anniversary Ed.》 — Hunt & Thomas（2019）。1999 經典更新，工程實務原則
  • 《Refactoring, 2nd Edition》 — Martin Fowler（Addison-Wesley 2018）。程式碼演進的最權威著作
  • 《The Phoenix Project》 + 《The Unicorn Project》 — Gene Kim et al.（IT Revolution Press 2013/2019）。DevOps 文化用小說形式講出來，比技術書更有說服力
  • 《Site Reliability Engineering》 + 《SRE Workbook》 + 《Building Secure and Reliable Systems》 — Google 三本 SRE 書（O'Reilly 2016/2018/2020）。免費讀於 sre.google
  • 《Accelerate》 — Forsgren, Humble, Kim（IT Revolution Press 2018）。DORA metrics 起源、用研究數據證明 DevOps 實務

D · 2024-2026 新書（前沿議題）

  • 《Platform Engineering on Kubernetes》 — Mauricio Salatino（Manning 2024）。Platform engineering 範式的當代著作
  • 《Building Microservices, 2nd Edition》 — Sam Newman（O'Reilly 2021）。微服務從炒作到成熟之後的反思
  • 《eBPF: Programming the Linux Kernel》 — Liz Rice（O'Reilly 2023）。Cilium 範式作者寫的 eBPF 入門
  • 《Hands-On Kubernetes Validating Admission Policies》 — Niraj Sutariya（2024）— K8s 進階治理
  • 《Generative AI on Kubernetes》 — Sébastien Goasguen（Manning 2025 MEAP）— LLM workload K8s 部署
  • 《Staff Engineer》 + 《An Elegant Puzzle》 — Will Larson（Stripe 前 staff eng）。Senior+ 工程師職涯指南
  • 《The Manager's Path》 — Camille Fournier（O'Reilly 2017，仍是當代主流）。從 senior eng 到 CTO 的職涯
  • 《Observability Engineering, 2nd Edition》（rumored 2026）— 跟著 OTel 演進的更新

G · Conference（線上 / 實體均可）

  • KubeCon + CloudNativeCon（CNCF，每年 3 場）— K8s + cloud native 生態
  • QCon（每年多場全球）— senior+ 工程師導向，廣議題
  • PromCon（CNCF）— Prometheus 專門
  • Strange Loop（2009-2023）— 已停辦但 archive 是最佳資源（distributed system、PL theory）

給未來的你

這本書寫於 2026 年。你讀到這頁時可能是 2027、2030、甚至 2035。技術會變，但學習的方式不太會變：

  1. 選 1-2 個 blog 訂閱 RSS：每天看 5-10 分鐘，累積成質變
  2. 每週 1 個 podcast：通勤、跑步時聽，把死時間變學習時間
  3. 每年 2-3 場 conference talk：YouTube archive 完全免費、不需請假
  4. 每年 4-6 本書：選經典 + 1-2 本新書、深度勝於數量
  5. 加入 1-2 個社群：CNCF Slack、Platform Eng Slack、HN — 不只看，也回應
  6. 寫部落格 / 演講：教學是最好的學習，公開分享逼你想清楚

本書 70 章 + 後記 + 5 附錄到此結束。真正的學習從你關上這本書、打開下一本書、看下一個 talk、寫下一行 config 開始。

2026 工程師的學習法則
  1. 底層原理長期投資：HTTP / TCP / queueing theory / cache — 永不過時
  2. 具體工具短期投資：Nginx → Envoy → 下個工具，每 5 年換一次
  3. 持續曝光社群：blog / podcast / conference — 提早看到 trend
  4. 實作勝於閱讀：100 篇 blog 不如 deploy 1 個 production reverse proxy
  5. 分享勝於私藏：寫部落格 / 開源 / 演講 — 教學是最好的學習
