APPENDIX A · 術語表

Glossary
繁中 + English 對照 · 200+ 核心術語

本附錄按 8 個主題分類，列出 reverse proxy 工程師日常使用的核心術語。每個項目含英文原文、繁中對照、簡明定義（1-2 句）、以及主要章節參照（如 Ch.50 §50.3）。當你讀其他英文資料卡關，這裡查得到。

A · 協定與網路

HTTP · 超文本傳輸協定
HyperText Transfer Protocol。Web 的核心應用層協定。版本：HTTP/1.1（1997）、HTTP/2（2015，RFC 7540 → 9113）、HTTP/3（2022，RFC 9114）。Ch.7-9。
HTTP/2
Multiplexed binary framing 協定，建於 TCP+TLS。引入 stream、HPACK 壓縮 header、server push（後棄用）。Ch.8。
HTTP/3
建於 QUIC（UDP-based）的 HTTP 版本。不再受 TCP head-of-line blocking 影響、0-RTT、connection migration。Ch.9、Ch.48。
QUIC
Quick UDP Internet Connections。RFC 9000（2021）。Google 2012 起設計，IETF 後標準化。HTTP/3 的傳輸層。
TCP · 傳輸控制協定
Transmission Control Protocol（RFC 793, 1981）。可靠的、有順序的、面向 connection 的傳輸層協定。
UDP
User Datagram Protocol（RFC 768, 1980）。無連線、不保證順序、低延遲。QUIC 與 DNS 的底層。
TLS
Transport Layer Security。版本：1.2（RFC 5246, 2008）、1.3（RFC 8446, 2018）。建於 TCP（HTTP/1.1 / 2）或內建於 QUIC（HTTP/3）。
WebSocket
RFC 6455。Full-duplex 雙向通訊建於 HTTP upgrade。Ch.12。
SSE · Server-Sent Events
單向 server → client 串流 over HTTP。LLM streaming 主流。Ch.13、Ch.42。
gRPC
Google 的 RPC 框架建於 HTTP/2。Protobuf 序列化、雙向 streaming。Ch.14。
MASQUE
Multiplexed Application Substrate over QUIC Encryption。RFC 9298（CONNECT-UDP）+ draft（CONNECT-IP）。VPN over QUIC。Ch.47。
HPACK / QPACK
HTTP/2 的 HPACK（RFC 7541）與 HTTP/3 的 QPACK（RFC 9204）— header 壓縮算法。
0-RTT
Zero Round-Trip Time。TLS 1.3 / QUIC 在 resumption 時不需 handshake，立刻發 application data。有 replay 風險。
SNI · Server Name Indication
TLS extension（RFC 6066）讓 client 告知 server 想連的 hostname，allowing 同 IP 多 cert。ESNI / ECH 是加密版。
ECH · Encrypted Client Hello
RFC 9180+。加密 SNI 防止 ISP / 中介者監聽 hostname。Cloudflare 已實作。
ALPN · Application-Layer Protocol Negotiation
TLS extension（RFC 7301）。在 handshake 中協商使用 HTTP/1.1、H2 或 H3。

B · Proxy 與 Load Balancing

Forward Proxy · 正向代理
代表 client 發送 request 到 server。典型：企業 egress、debug、隱私。Ch.2、Part III。
Reverse Proxy · 反向代理
代表 server 接收 client request，後 route 給 backend。典型：load balancer、API gateway、CDN edge。Ch.3、Part IV。
Transparent Proxy
Client 不知道 proxy 存在的代理。常用於 ISP、企業 monitoring。
Load Balancer · 負載均衡器
分散流量到多個 backend instance。L4（TCP）或 L7（HTTP-aware）。
L4 LB / L7 LB
L4：TCP/UDP 層、不解析 application protocol（HAProxy、AWS NLB）。L7：HTTP-aware、能 route by URL/header（Nginx、Envoy、AWS ALB）。Ch.5。
Round Robin
輪詢演算法，請求依序分配到 backend。最簡單，但不考慮 backend 負載。
Least Connections
把新請求送到當前連線數最少的 backend。適合 long-lived connection 場景（WebSocket、gRPC）。
Consistent Hashing
把 request key（如 user_id）hash 到 backend，加減 backend 時最小化 reroute。Cache 場景關鍵。
Sticky Session
同一 client 的 request 持續送到同一 backend。Cookie-based 或 IP-based。Ch.31。
Health Check
定期 probe backend 確認健康。Active（主動 probe）vs Passive（觀察 request 失敗）。Ch.30。
Connection Pooling
Reverse proxy 維持到 backend 的 long-lived connection 池，避免每個 request 重建。Ch.32。
Circuit Breaker
Backend 失敗超過 threshold 時，proxy 暫停 forward request 給它，給時間恢復。模仿電路熔絲。Ch.33。
Retry with Backoff
請求失敗後重試、間隔指數成長 + jitter，避免 thundering herd。
Anycast
同 IP 從多個 PoP 廣播，網路自動 route 到最近 PoP。Cloudflare、AWS Global Accelerator 用。Ch.67 §67.4。
GeoDNS
DNS 依 resolver 地理位置回傳不同 IP。Route 53、Cloudflare DNS、NS1。
GSLB · Global Server Load Balancing
跨地理 region 的智慧 load balancing，考慮地理 + 健康 + 負載。F5 GSLB、Citrix ADC。
Service Mesh
每個 service 旁邊 sidecar proxy（Envoy / Linkerd），處理 service-to-service 通訊：mTLS、retry、tracing。Istio、Linkerd、Cilium。
Sidecar Pattern
每個 application pod 旁邊 deploy 一個 helper container（如 Envoy）處理橫切議題。
API Gateway
API 層的 reverse proxy + 認證 + rate limit + transformation + analytics。Kong、AWS API Gateway、Apigee、APISIX。Part V。
Ingress / Egress
進入 / 流出系統邊界的流量。K8s Ingress 是 cluster 入口、egress 是流出。

C · TLS 與安全

Cipher Suite
TLS 用的加密算法組合：key exchange + authentication + symmetric cipher + MAC。如 TLS_AES_128_GCM_SHA256（TLS 1.3）。
Forward Secrecy · 完整前向保密
長期 key 洩漏不會危及過往連線（每個 session 用 ephemeral key）。TLS 1.3 預設。
mTLS · Mutual TLS
雙向 TLS — server 認證 client、client 也認證 server。Service mesh 內部標準。
X.509 Certificate
數位憑證標準（ITU-T X.509）。包含公鑰、subject、CA 簽名等。
CA · Certificate Authority
憑證發行機構，被信任簽發 certificate。商業 CA（DigiCert、Sectigo）+ 免費（Let's Encrypt、ZeroSSL）。
OCSP / OCSP Stapling
Online Certificate Status Protocol 檢查 cert 是否被 revoke。OCSP stapling 由 server 預先取得 OCSP 回應 + 附在 handshake，省去 client 額外查詢。
HSTS · HTTP Strict Transport Security
RFC 6797。Header 告訴 browser 永遠用 HTTPS。
CORS · Cross-Origin Resource Sharing
Browser 安全機制控制跨 origin 請求。Preflight（OPTIONS）+ Access-Control-* headers。
CSRF · Cross-Site Request Forgery
攻擊：誘騙 browser 用使用者 cookie 發送惡意 request。防：CSRF token、SameSite cookie。
XSS · Cross-Site Scripting
攻擊：注入惡意 JS 到網頁。防：CSP header、輸出 escape。
OWASP Top 10
Open Web Application Security Project 每年更新的 web 安全 10 大威脅。API Top 10 是另一個列表。
WAF · Web Application Firewall
L7 firewall 解析 HTTP，擋 SQL injection / XSS / 已知攻擊。Cloudflare WAF、ModSecurity、AWS WAF。
Rate Limit
限制 client 一段時間內 request 數。算法：fixed window、sliding window、token bucket、leaky bucket。Ch.35。
Token Bucket
Rate limit 算法。bucket 持續補 token，每 request 消耗一個；空了就拒。
JWT · JSON Web Token
RFC 7519。三段 base64 編碼（header.payload.signature）的 token。Stateless 認證。
OAuth 2.0
RFC 6749。授權框架。Flow：authorization code、client credentials、device code（RFC 8628）等。
OIDC · OpenID Connect
建於 OAuth 2.0 之上的身份層。額外的 ID token 含 user identity。
Zero Trust
「從不信任、永遠驗證」。每個 request 都認證 + 授權，沒有 trusted 內網。Google BeyondCorp 起源。Ch.52。
Post-Quantum Cryptography（PQC）
抗量子電腦的密碼學。NIST 2024 標準化 ML-KEM（Kyber）、ML-DSA、SLH-DSA。Ch.49。
X25519MLKEM768
Cloudflare 2024 部署的 hybrid key exchange — X25519（傳統）+ ML-KEM-768（後量子）。

D · 性能與容量

Latency
請求-回應的時間。Average、p50（median）、p95、p99、p99.9 — 越尾越關鍵。
p99 / p99.9 Latency · Tail Latency
99% 或 99.9% 的 request 完成時間 ≤ X。比 average 更反映真實使用者體驗。Ch.66 §66.6。
Throughput / RPS
每秒請求數（Requests Per Second）。系統處理速率。
Concurrency
同時處理的請求數。Little's Law：concurrency = arrival_rate × latency。Ch.66 §66.1。
Little's Law
L = λW。系統內 request 數 = 流入速率 × 處理時間（John Little, 1961, MIT）。
USL · Universal Scalability Law
C(N) = N / (1 + α(N-1) + βN(N-1))。Neil Gunther 1993 提出，描述水平 scale 受 contention + coherence 限制。Ch.66 §66.3。
Saturation Point
系統達到 SLO 上限的最大 RPS。超過此值 latency 急遽上升。
Headroom
系統閒置容量。業界共識 production 跑 50-70% saturation。Ch.66 §66.5。
N+1 / N+2 Redundancy
N 個 instance 處理 peak 流量 + 1 / 2 個多餘，任一 / 二 fail 不影響 SLO。
Cache Hit Ratio
Cache 命中比例。Reverse proxy / CDN 關鍵指標。> 80% 是常見目標。
Cache Stampede
熱門 cache key TTL 過期後，所有 request 同時打 origin。解：single-flight、lazy refresh、jitter。
Hedged Request
同一 request 發給兩個 backend，先回的算數。Google 用以降 tail latency。
Open-loop / Closed-loop
Open-loop：流量速率固定（真實場景）。Closed-loop：流量由 server 速度決定（wrk 預設）。Load test 必須 open-loop。Ch.60 §60.2。
Forecasting
預測未來流量。Linear regression、ARIMA、Prophet（Meta 2017）、Neural network。Ch.66 §66.4。

E · Observability 與 Debug

Observability
從外部能推斷系統內部狀態。三大 pillar：metrics、logs、traces。Ch.57-58。
OpenTelemetry（OTel）
CNCF 標準化 observability 框架（OpenTracing 2016 + OpenCensus 2017 合併）。Ch.57。
OTLP · OpenTelemetry Protocol
OTel 傳輸協定。gRPC（4317）/ HTTP Protobuf（4318）/ HTTP JSON。
Trace / Span
Trace：跨服務的 request 路徑。Span：單一操作（DB query、HTTP call）。Span 串成樹狀組成 trace。
Distributed Tracing
追蹤跨服務的請求路徑。W3C Trace Context（traceparent header）跨工具標準。
Sampling
採樣 trace 以降成本。Head-based（前段決定）vs Tail-based（後段，能看 outcome）。Ch.57 §57.7。
Cardinality
Metric label 不同值組合數。高 cardinality（如 user_id 當 label）爆 observability 成本。Ch.58 §58.7。
Histogram / Exponential Histogram
Metric 類型描述分布。Exponential histogram（OTel 2024）自動 bucket，省記憶體。
Profiling / Continuous Profiling
持續 sample stack trace 看 CPU 在做什麼。Pyroscope（Grafana 2023 收購）、Parca。Ch.59 §59.8。
Flame Graph
Brendan Gregg 發明的 stack trace 視覺化。X 軸是 sample 數、Y 軸是 stack depth。
USE Method
Brendan Gregg 提出。對每個 resource 看 Utilization / Saturation / Errors。Ch.59 §59.1。
RED Method
對 service 看 Rate / Errors / Duration。USE 的 service 版本。
tcpdump / tshark / Wireshark
Packet capture 工具。tcpdump CLI、tshark 協定解析、Wireshark GUI。Ch.59 §59.2。
strace / bpftrace
Strace：syscall 追蹤（會拖慢）。bpftrace：eBPF-based、production-safe。Ch.59 §59.6。
eBPF
extended Berkeley Packet Filter。Linux kernel 內安全執行用戶 code 的機制。Cilium、Pixie、Pyroscope 用。Ch.50 §50.4。
OBI · OpenTelemetry eBPF Instrumentation
原 Beyla（Grafana）2025 捐 OTel 改名。eBPF zero-code instrumentation。Ch.57 §57.5。

F · SRE 與可靠性

SRE · Site Reliability Engineering
Google 2003 Ben Treynor 發明。「軟體工程師設計 ops team」。Ch.62。
SLI · Service Level Indicator
對使用者體驗的具體量化（availability、latency、quality）。Ch.62 §62.4。
SLO · Service Level Objective
SLI 的內部目標值（如 99.9% availability）。團隊承諾、決定 release pace。
SLA · Service Level Agreement
對外合約，違反有 penalty。SLO < SLA（給 buffer）。
Error Budget
100% - SLO。允許的不可靠程度。Ch.63。
Burn Rate
Error budget 消耗速度。MWMBR alerting：14.4× / 6× / 1× 是 Google 推薦數值。Ch.63 §63.3。
MWMBR · Multi-Window Multi-Burn-Rate Alerting
用多個窗口（1h / 6h / 3d）+ 多個 burn rate 觸發 alert，避免 false alarm。
Toil
重複、手動、可自動化、與系統量線性 scale 的工作。Google SRE 規定 toil < 50%。
MTTD / MTTA / MTTR / MTBF
Mean Time To Detect / Acknowledge / Resolve / Between Failures。Ch.64 §64.2。
Incident Commander（IC）
Incident response 角色：協調、決策。不 hands-on debug。Ch.64 §64.4。
Blameless Postmortem
事後分析從「個人責任」轉為「系統責任」。John Allspaw 2012 推廣。Ch.65。
Just Culture
Sidney Dekker 範式。區分 human error / at-risk / reckless 三類行為。
Contributing Factors
取代 single root cause 的概念 — 列出多個促成因素，每個對應 action item。
RTO / RPO
Recovery Time / Point Objective。災難恢復的兩個核心 metric。Ch.67 §67.1。
Active-Active / Active-Passive
Multi-region 模式。Active-Active 所有 region 同時服務、Active-Passive 主備。Ch.67 §67.3。
Split-Brain
網路分割後雙 leader。解：consensus protocol（Raft、Paxos、etcd、Consul、ZooKeeper）+ 奇數節點。Ch.67 §67.6。
Chaos Engineering
主動製造故障找系統弱點。Netflix 2010 Chaos Monkey 起源。Ch.68。
Game Day
計畫好的 chaos session — team 集合、製造故障、模擬 incident response。Ch.68 §68.6。
Page Budget
每週 max page 數（incident.io 2026 推廣）。超過 freeze feature work。Ch.69 §69.4。
Alert Fatigue
過多 alert 導致工程師失去敏感度。62% on-call 工程師回報影響工作品質（PagerDuty 2026）。

G · API 與開發

OpenAPI
API 描述標準。版本：3.0（2017）、3.1（2021，對齊 JSON Schema）、3.2（2025/09，streaming + QUERY method）。Ch.53。
AsyncAPI
事件驅動 API 描述標準。版本 3.0（2023/12）。Kafka / MQTT / WebSocket / SSE binding。Ch.54。
JSON Schema 2020-12
JSON 結構描述標準。OpenAPI 3.1 完整對齊。
RFC 7807 / 9457 · Problem Details
HTTP API error response 標準格式。application/problem+json。Ch.56 §56.4。
RFC 8594 Sunset Header
API deprecation 通知標準。配合 RFC 9745 Deprecation header（2025/03）。
SDK · Software Development Kit
給開發者用的程式庫。2026 主流 SDK generator：Stainless、Speakeasy、Fern、OpenAPI Generator、Kiota。Ch.55。
llms.txt
給 AI agent 看的 markdown API 文件。Jeremy Howard 2024 提出。Ch.55 §55.5。
TTFC · Time-to-First-Call
從看到 API 文件到第一個成功 200 OK 的時間。DX 核心 KPI。Stripe 標桿 5 分鐘。Ch.56 §56.1。
IDP · Internal Developer Platform
Platform team 提供給 developer 的 self-service abstraction。Backstage、Port、Cortex、Humanitec。Ch.61。
Backstage
Spotify 2016 內部建構、2020 開源、2025 CNCF graduate。OSS IDP 主流。
GitOps
Git 是 single source of truth、controller 自動 reconcile。Weaveworks 2017 提出。ArgoCD / Flux。Ch.61 §61.4。
Progressive Delivery
自動化 canary deployment。Argo Rollouts、Flagger。流量逐步切換、metric 異常自動 rollback。

H · 雲端與 Kubernetes

Kubernetes（K8s）
Container orchestration platform。CNCF graduated。2014 由 Google 開源（Borg 範式）。
Pod / Service / Deployment
K8s 核心抽象。Pod：一或多個 container 的最小單位。Service：穩定網路端點。Deployment：管理 pod replica。
Ingress / Gateway API
Ingress 是 v1 對外 routing API。Gateway API（2022 GA）是 Ingress 的後繼，支援 multi-protocol、role-oriented。Ch.50。
HPA / VPA
Horizontal / Vertical Pod Autoscaler。HPA scale pod 數、VPA scale 單個 pod resource。
Helm
K8s package manager。Chart 是 package 格式。
CRD · Custom Resource Definition
K8s 擴充自訂資源類型。Operator pattern 基礎。
Cilium
eBPF-based K8s networking + observability + security。CNCF graduated。
Istio / Linkerd
主流 service mesh。Istio 功能豐富但複雜、Linkerd 輕量易用。
Envoy
Lyft 2016 開源的 L7 proxy。Service mesh data plane 標準（Istio / Linkerd 都用 Envoy 變種）。
CNCF · Cloud Native Computing Foundation
Linux Foundation 旗下，治理 K8s、Envoy、Prometheus、OpenTelemetry 等 100+ 雲原生專案。
FinOps
雲端成本治理範式。Linux Foundation 2020 成立 FinOps Foundation。Ch.70 §70.5。
Spot / Preemptible Instance
雲端短期可用 instance（AWS Spot、GCP Preemptible），價 60-90% 折扣，被收回時無預警。適合 stateless workload。
Reserved Instance / Savings Plans
1-3 年合約折扣 30-72%。AWS RI 限定 instance type、Savings Plans 跨 type 更靈活。
NAT Gateway
AWS 跨 VPC 出口。$0.045/GB processed — 大流量隱形殺手。替代：VPC Endpoint、IPv6。Ch.70 §70.2。
如何使用本附錄

200+ 術語覆蓋本書 70 章涵蓋的所有核心概念。讀其他英文資料卡關時：(1) 在這裡查中文對照、(2) 看簡明定義快速理解、(3) 跳到對應章節深入。每個術語都有章節參照方便延伸閱讀。如果你發現某個術語沒列出，或定義不夠清楚 — 那是作者的失誤，不是你的問題。
