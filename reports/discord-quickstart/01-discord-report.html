▎ 第一部分

一般使用者操作

傳送、編輯、刪除、回覆、轉發、釘選、Reaction、Thread、搜尋、附件——使用者在 Discord 中與訊息互動的所有方式。

傳送訊息的所有方式與字數限制

💬

純文字訊息

免費用戶上限 2,000 字元，Nitro 訂閱者可達 4,000 字元。超過上限時自動轉為 .txt 附件，可直接展開閱讀。

📎

圖片、影片與檔案

透過「+」按鈕或拖曳上傳。多張圖片以馬賽克拼貼呈現。支援 PNG、JPG、GIF、MP4、WebM、MOV、PDF 等幾乎所有格式。

🎤

語音訊息

長按麥克風圖示錄製，最長 20 分鐘。錄製後可預聽再傳送，聊天中顯示音訊波形與時長。

😀

Emoji、Sticker 與 GIF

單獨傳送的 Emoji 以 jumbo 尺寸顯示（最多 27 個維持大尺寸）。跨伺服器使用自訂 Emoji/Sticker 需 Nitro。GIF 透過內建 Tenor/GIPHY 搜尋。

📊

Polls（投票）

問題上限 300 字元，最多 10 個選項（每項 55 字元），投票期限最長 1 週。投票不匿名，所有人可見投票者身份。

🔇

靜默訊息

訊息開頭輸入 @silent，訊息會傳送但不產生推播或桌面通知。訊息上會顯示一個帶「z」的鈴鐺圖示。

編輯訊息：無時間限制，但歷史不可見

使用者可隨時編輯自己的訊息，沒有時間限制。桌面版可按上方向鍵快速編輯最後一則訊息。編輯後訊息旁出現小字「(edited)」，滑鼠懸停可查看最後編輯時間。

⚠ 關鍵限制

Discord 不提供編輯歷史，其他使用者只能看到最新版本。附件無法在編輯中新增或替換，僅能在桌面版刪除現有附件。第三方管理 Bot（如 Dyno、Carl-bot）可記錄編輯歷史，但這非平台原生功能。

刪除、回覆、轉發與釘選

功能操作方式限制與注意事項
刪除 右鍵 → Delete（Shift+點刪除可跳過確認） 刪除後永久消失，無法復原。刪除他人訊息需 Manage Messages 權限
回覆 (Reply) 懸停訊息 → 點回覆箭頭 預設 ping 被回覆者，可切換 @ON/@OFF。僅限同頻道，不支援跨頻道
轉發 (Forward) 右鍵 → Forward（2024 年 10 月上線） 最多 5 個目的地。顯示「Forwarded」標籤但不顯示原作者。內容為靜態快照，原訊息編輯不同步
釘選 (Pin) 右鍵 → Pin Message 每個頻道最多 50 則釘選訊息。需 Manage Messages 權限

Reaction 與 Thread

每則訊息最多 20 種不同的 Emoji Reaction，每種的反應人數不限。Super Reactions 為 Nitro 專屬，呈現動畫爆發效果。

Thread 從頻道訊息分支出來，名稱上限 100 字元。Auto-archive 選項包括 1 小時、24 小時、3 天、7 天。每個 Thread 最多 1,000 名成員，可獨立設定 Slowmode。Private Thread 需伺服器 Level 2 Boost。

搜尋語法

篩選器用法
from:username來自特定使用者
mentions:username提及特定使用者
has:link / image / file / video / sound / poll包含特定內容類型
in:#channel在特定頻道中
before:YYYY-MM-DD指定日期之前
after:YYYY-MM-DD指定日期之後
during:YYYY-MM-DD指定日期當天

搜尋使用 token-based 匹配，不支援部分字串比對。結果可依 Relevance 或 Recency 排序（僅桌面版）。

方案每檔上限
免費10 MB（2024 年 9 月起自 25 MB 下調）
Nitro Basic50 MB
Nitro500 MB

🚨 重要警告：MESSAGE_CONTENT Privileged Intent

未啟用 MESSAGE_CONTENT Privileged Intent 的 Bot，在 MESSAGE_CREATE 事件中 content、embeds、attachments、components 會收到空值。例外：Bot 自己的訊息、DM 訊息、@mention Bot 的訊息。已驗證 Bot（100+ guilds）需向 Discord 申請核准才能啟用。

訊息相關 API Endpoint

操作方法與路徑關鍵限制
建立訊息 POST /channels/{id}/messages 需 SEND_MESSAGES。至少提供 content / embeds / sticker_ids / components / files / poll 之一。最大 25 MiB
編輯訊息 PATCH /channels/{id}/messages/{id} 原作者可改 content/embeds/flags/components。他人僅能用 MANAGE_MESSAGES 改 flags
刪除單則 DELETE /channels/{id}/messages/{id} 刪他人訊息需 MANAGE_MESSAGES
批次刪除 POST /channels/{id}/messages/bulk-delete 2–100 個 ID，不能刪超過 14 天的訊息。需 MANAGE_MESSAGES
取得列表 GET /channels/{id}/messages 支援 before/after/around（互斥）+ limit（1–100，預設 50）
取得單則 GET /channels/{id}/messages/{id} 需 VIEW_CHANNEL + READ_MESSAGE_HISTORY
跨頻發布 POST /channels/{id}/messages/{id}/crosspost 每頻道每小時最多 10 次

Reaction API

方法路徑說明

Webhook 與 Bot 訊息的差異

Webhook 端點 POST /webhooks/{id}/{token} 不需 Bot Token。可在每則訊息覆寫 username 與 avatar_url。allowed_mentions 預設只解析 ["users"]（Bot 預設解析全部）。加上 wait=true 參數可讓 API 回傳完整訊息物件。

BUTTON2六種樣式：Primary / Secondary / Success / Danger / Link / Premium
STRING_SELECT3文字下拉選單，最多 25 個選項
TEXT_INPUT4文字輸入（僅限 Modal）

📋 Rate Limit 最佳實踐

全域限制 50 請求/秒。傳送訊息限制為每頻道每 5 秒 5 則。永遠不要硬編碼 Rate Limit 數值，而是解析回應 Header（X-RateLimit-Remaining、X-RateLimit-Reset-After 等）並實作 exponential backoff。收到 429 時使用 retry_after 欄位。無效請求上限為 10 分鐘內 10,000 次。

Ephemeral 訊息透過 flag 1 << 6（64）設定，僅觸發者可見。初始回應須在 3 秒內完成，Token 有效期 15 分鐘。Followup 訊息透過 POST /webhooks/{app_id}/{token} 發送。

▎ 第三部分

伺服器管理與權限

訊息相關權限、Slowmode、AutoMod、Audit Log、公告頻道、Forum 頻道與安全設定的完整管理指南。

訊息相關權限清單

權限API 名稱效果
Send MessagesSEND_MESSAGES傳送訊息。拒絕時隱性拒絕 MENTION_EVERYONE、SEND_TTS 等
Send in ThreadsSEND_MESSAGES_IN_THREADS在 Thread 中傳送（獨立於 SEND_MESSAGES）
Manage MessagesMANAGE_MESSAGES刪除他人訊息、釘選、管理 Reaction、發布公告
Manage ThreadsMANAGE_THREADS重命名/刪除/關閉 Thread、設定 Thread Slowmode
Read HistoryREAD_MESSAGE_HISTORY關閉後只能看即時新訊息
Attach FilesATTACH_FILES上傳檔案與媒體
Embed LinksEMBED_LINKS連結自動生成嵌入預覽
Add ReactionsADD_REACTIONS新增 Reaction（關閉仍可點擊已有的）
External EmojiUSE_EXTERNAL_EMOJIS使用其他伺服器的 Emoji
Mention @everyoneMENTION_EVERYONE使用 @everyone 與 @here
Voice MessagesSEND_VOICE_MESSAGES傳送語音訊息
App CommandsUSE_APPLICATION_COMMANDS使用斜線指令

權限優先順序：使用者專屬覆寫 → 頻道角色覆寫 → @everyone 頻道覆寫 → 伺服器角色 → @everyone 伺服器。Administrator 可繞過所有限制。被 Timeout 的成員暫時失去除 VIEW_CHANNEL 和 READ_MESSAGE_HISTORY 以外的所有權限。

Slowmode 設定

限制使用者傳送訊息的頻率，範圍 5 秒到 6 小時（API 支援 0–21,600 秒任意整數）。UI 預設值包括 5s/10s/15s/30s/1m/2m/5m/10m/15m/30m/1h/2h/6h。

⚠ 2026 年 2 月重大變更

Discord 已將 Slowmode 繞過整合為單一的「Bypass Slowmode」權限。舊版 Manage Messages / Manage Channel 不再自動繞過 Slowmode，管理員需重新檢視角色設定。

AutoMod 自動審核

AutoMod 在訊息發布前自動偵測並處置，涵蓋文字頻道、Thread、Voice Text 與 Forum。

規則類型上限說明
KEYWORD6 條自訂關鍵字最多 1,000 個（每個 60 字元），regex 最多 10 條（260 字元，Rust 風格）
KEYWORD_PRESET1 條Discord 內建詞庫：Profanity / Sexual Content / Slurs
SPAM1 條機器學習偵測垃圾訊息
MENTION_SPAM1 條mention_total_limit 最高 50 個唯一提及
MEMBER_PROFILE1 條檢查成員名稱

處置動作包括 Block Message（可附帶最多 150 字元自訂訊息）、Send Alert Message（記錄到指定頻道）、Timeout（最長 4 週）。每條規則可豁免最多 20 個角色與 50 個頻道。

Audit Log

記錄的訊息操作包括 MESSAGE_DELETE(72)、MESSAGE_BULK_DELETE(73)、MESSAGE_PIN(74)、MESSAGE_UNPIN(75)、AUTO_MODERATION_BLOCK_MESSAGE(143) 等。所有條目保留 45 天。

📋 注意

Audit Log 不記錄使用者自行刪除的訊息，僅記錄被他人（或 Bot）刪除的情況。要完整記錄需依賴第三方 Bot。

公告頻道與 Forum 頻道

Announcement Channel 僅限 Community 伺服器。訊息可「Publish」後推送至所有追蹤伺服器，限制為每頻道每小時 10 次發布。

Forum Channel 每篇貼文即一個 Thread，建立時必須包含標題與初始訊息。最多 20 個標籤，每篇可選 5 個。

Mention 語法

類型語法
使用者<@USER_ID>
角色<@&ROLE_ID>
頻道<#CHANNEL_ID>
斜線指令</command:COMMAND_ID>

API 的 allowed_mentions 物件控制通知行為，parse 陣列可包含 "users"/"roles"/"everyone"。空的 parse: [] 完全抑制所有 Mention 通知。

自訂 Emoji 與連結預覽

靜態 Emoji：<:name:ID>，動態：<a:name:ID>。CDN 位址為 https://cdn.discordapp.com/emojis/{id}.png（動態用 .gif）。

Discord 透過 OpenGraph（og:title、og:description、og:image）與 Twitter Card metadata 自動生成連結預覽。YouTube、Spotify、Twitter 等享有強化嵌入。抑制方式：角括號包住 URL、設定 SUPPRESS_EMBEDS flag、或在使用者設定中關閉。
