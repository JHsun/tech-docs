應用程式骨架 Writing a Streams App

Java
Properties props = new Properties();
props.put(StreamsConfig.APPLICATION_ID_CONFIG, "my-app");
props.put(StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
props.put(StreamsConfig.DEFAULT_KEY_SERDE_CLASS_CONFIG, Serdes.String().getClass());
props.put(StreamsConfig.DEFAULT_VALUE_SERDE_CLASS_CONFIG, Serdes.String().getClass());

StreamsBuilder builder = new StreamsBuilder();
builder.stream("input").filter((k, v) -> v.length() > 5).to("output");

KafkaStreams streams = new KafkaStreams(builder.build(), props);
Runtime.getRuntime().addShutdownHook(new Thread(streams::close));
streams.start();

生命週期：start() → 處理中 → close()。cleanUp() 清除本地 State（在 start 前或 close 後呼叫）。

操作會 Re-partition？程式碼範例

map✅stream.map((k,v) -> KeyValue.pair(v, k))
mapValues❌stream.mapValues(v -> v.toUpperCase())

selectKey✅stream.selectKey((k,v) -> v.substring(0,3))
peek❌stream.peek((k,v) -> log.info(k))
merge❌stream1.merge(stream2)
repartition✅stream.repartition(Repartitioned.numberOfPartitions(10))

split（Branch）❌見下方

Branch / Split

Java
Map<String, KStream<String, Long>> branches = stream
    .split(Named.as("B-"))
    .branch((k,v) -> k.startsWith("A"), Branched.as("A"))
    .branch((k,v) -> k.startsWith("B"), Branched.as("B"))
    .defaultBranch(Branched.as("other"));
// branches.get("B-A"), branches.get("B-B"), branches.get("B-other")
03

DSL：Aggregation 與 Window Stateful Transforms

Java — Aggregation
// count
grouped.count();

// reduce
grouped.reduce((agg, newVal) -> agg + newVal);

// aggregate（自訂初始值 + 累加器）
grouped.aggregate(
    () -> 0L,
    (key, val, agg) -> agg + val.length(),
    Materialized.<String, Long, KeyValueStore<Bytes, byte[]>>as("store")
        .withValueSerde(Serdes.Long()));

// KGroupedTable aggregate（需 adder + subtractor）
groupedTable.aggregate(() -> 0L,
    (key, newVal, agg) -> agg + newVal,   // adder
    (key, oldVal, agg) -> agg - oldVal,   // subtractor
    Materialized.as("table-agg"));
Java — Windowed Aggregation
// Tumbling 5 分鐘
grouped.windowedBy(TimeWindows.ofSizeWithNoGrace(Duration.ofMinutes(5))).count();

// Hopping 5 分鐘視窗、1 分鐘步進
grouped.windowedBy(TimeWindows.ofSizeWithNoGrace(Duration.ofMinutes(5)).advanceBy(Duration.ofMinutes(1))).count();

// Session 30 分鐘不活動間隔
grouped.windowedBy(SessionWindows.ofInactivityGapWithNoGrace(Duration.ofMinutes(30))).count();

Suppress — 僅發送最終結果

Java
windowedCounts.suppress(Suppressed.untilWindowCloses(BufferConfig.unbounded()));

DSL：Join 程式碼 Join Examples

KStream–KStream（Windowed）

Java
left.join(right,
    (lv, rv) -> "left=" + lv + ",right=" + rv,
    JoinWindows.ofTimeDifferenceWithNoGrace(Duration.ofMinutes(5)),
    StreamJoined.with(keySerde, lSerde, rSerde));

KStream–KTable（Table Lookup）

Java
stream.join(table, (sv, tv) -> sv + "/" + tv);

KStream–GlobalKTable（不需 Co-partitioning）

Java
stream.join(globalTable,
    (streamKey, streamVal) -> streamKey,  // Key Mapper
    (sv, gv) -> sv + "/" + gv);

KTable–KTable Foreign-Key Join

Java
leftTable.join(rightTable,
    (leftVal) -> leftVal.getForeignKey(),  // FK Extractor
    (lv, rv) -> "joined");

Processor API 低階 API

Java
Topology topology = new Topology();
topology.addSource("Source", "input-topic")
    .addProcessor("Process", () -> new MyProcessor(), "Source")
    .addStateStore(storeBuilder, "Process")
    .addSink("Sink", "output-topic", "Process");

Processor 介面

方法說明
init(ProcessorContext)初始化。取得 State Store、註冊 Punctuator
process(Record)處理每筆 Record
close()清理（不要關閉 State Store）

Punctuator

類型觸發方式
STREAM_TIME由資料 Timestamp 驅動
WALL_CLOCK_TIME由實際時鐘驅動

Naming 與 Serde Naming Operators & Serialization

Naming（避免 Topology 變更導致不相容）

Java
builder.stream("input", Consumed.as("source"))
    .filter((k,v) -> v != null, Named.as("filter_null"))
    .to("output", Produced.as("sink"));

內建 Serde

測試 TopologyTestDriver

TopologyTestDriver driver = new TopologyTestDriver(topology, props);

TestInputTopic<String,Long> input = driver.createInputTopic(
    "input", strSer, longSer);

Interactive Queries 查詢狀態

Java
// 查詢本地 State Store
ReadOnlyKeyValueStore<String,Long> store =
    streams.store("store", QueryableStoreTypes.keyValueStore());
store.get("hello");
store.range("a", "z");
store.all();

// Window Store
ReadOnlyWindowStore<String,Long> wStore =
    streams.store("wStore", QueryableStoreTypes.windowStore());

// 遠端狀態探索（需設定 application.server）

記憶體管理 Memory Management

機制設定說明
Record Cachestatestore.cache.max.bytes=10485760壓縮中間更新再寫入 Store。設 0 停用。LRU 淘汰。
RocksDBrocksdb.config.setter自訂 Block Cache、Write Buffer Manager。建議共享單一 LRUCache
Commit 頻率commit.interval.msat-least-once=30s；exactly-once=100ms。影響 Cache Flush 頻率
💡 最佳實踐跨所有 RocksDB 實例共享單一 Cache + WriteBufferManager。使用 jemalloc。明確關閉 Iterator。預設每個 Store 的 Block Cache 為 50MB。

Application Reset Tool 重設工具

bash
$ bin/kafka-streams-application-reset.sh \
    --application-id my-app \
    --bootstrap-server localhost:9092 \
    --input-topics input-topic \
    --dry-run
選項說明
--input-topics重設到 earliest
--intermediate-topics跳到 end
--to-earliest / --to-latest重設位置
--to-datetime重設到指定時間
--shift-by <±N>相對位移
--dry-run預覽不執行
--force強制移除 Consumer Group 成員

執行後：透過 streams.cleanUp() 或 rm -rf /tmp/kafka-streams/<app-id> 清除本地狀態。

Streams Rebalance Protocol KIP-1071，4.2 GA NEW

將 Task 分配從 Client 端移至 Broker 端，應用註冊為 Streams Group（非 Consumer Group），實現持續增量式 Rebalance。

面向Classic ProtocolStreams Protocol（4.2）
Task 分配Client 端（選出 Leader）Broker 端（Coordinator）
Group 類型Consumer GroupStreams Group
Heartbeat RPCJoinGroup / SyncGroupStreamsGroupHeartbeat
Internal Topic 建立Client 端Broker 端
Rebalance 方式全域同步屏障持續增量

啟用方式

bash — Broker 端
$ bin/kafka-features.sh --bootstrap-server localhost:9092 upgrade --feature streams.version=1
Java — Client 端
props.put("group.protocol", "streams"); // 預設為 "classic"
⚠️ 4.2.0 注意事項（KAFKA-20254）避免將既有 Classic Group 遷移為 Streams Group。新建的 Streams Group 不受影響。

Broker 端設定

設定說明
group.coordinator.rebalance.protocols需包含 streams
group.streams.session.timeout.msSession 逾時（預設 45s）
group.streams.heartbeat.interval.msHeartbeat 間隔（預設 5s）

kafka-streams-groups.sh 4.2 新工具 NEW

專門管理 Streams Group 的 CLI 工具，取代 Reset Tool 對 Streams Group 的功能。

bash
# 列出所有 Streams Group
$ bin/kafka-streams-groups.sh --bootstrap-server localhost:9092 --list

# 查看 Group 狀態、成員、Offset/Lag
$ bin/kafka-streams-groups.sh ... --describe --group my-app --state --members --verbose
$ bin/kafka-streams-groups.sh ... --describe --group my-app --offsets

# 預覽 Offset 重設
$ bin/kafka-streams-groups.sh ... --group my-app \
    --reset-offsets --all-input-topics --to-earliest --dry-run

# 執行 Offset 重設到指定時間
$ bin/kafka-streams-groups.sh ... --group my-app \
    --reset-offsets --all-input-topics --to-datetime 2025-01-31T23:57:00.000 --execute

# 刪除 Group（含 Internal Topic）
$ bin/kafka-streams-groups.sh ... --delete --group my-app \
    --internal-topics my-app-repartition-0,my-app-changelog

Streams Group 狀態：Empty → Not Ready → Assigning → Reconciling → Stable → Dead

✅ 最佳實踐操作前先用 --dry-run 預覽。確保應用實例已停止且 Group 為 Empty 狀態再執行修改操作。
