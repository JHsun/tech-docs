架構與核心概念 Overview

Kafka Connect 是用於在 Kafka 與外部系統間可擴展、可靠地串流資料的框架。

Source Connector

從外部系統（DB、檔案、MQ）讀取資料 → 寫入 Kafka Topic

Sink Connector

從 Kafka Topic 讀取 → 寫入外部系統（HDFS、ES、S3）。需設定 topics 或 topics.regex

Worker

執行 Connector 與 Task 的程序。Distributed 模式下形成叢集，透過 Kafka Group Protocol 協調

Task

實際複製資料的執行單位。每個 Connector 最多建立 tasks.max 個 Task

Converter

在 Connect 內部格式與 Kafka 序列化格式間轉換（JSON、Avro、Protobuf 等），與 Connector 解耦

Transform（SMT）

輕量級逐筆訊息轉換，套用在 Connector Pipeline 中

REST API 完整端點 預設 Port 8083

叢集與 Connector 管理

方法路徑說明
GET/叢集資訊（版本、Kafka Cluster ID）
GET/connectors列出所有 Connector
POST/connectors建立 Connector（可設 initial_state: RUNNING/PAUSED/STOPPED）
GET/connectors/{name}取得 Connector 資訊
GET/connectors/{name}/config取得設定
PUT/connectors/{name}/config建立或更新設定
PATCH/connectors/{name}/config部分更新（null 值移除 Key）
GET/connectors/{name}/status查看 Connector 與 Task 狀態
PUT/connectors/{name}/pause暫停（資源保留）
PUT/connectors/{name}/resume恢復
PUT/connectors/{name}/stop停止（Task 關閉、釋放資源，3.5+）
POST/connectors/{name}/restart重啟（可加 ?includeTasks=true&onlyFailed=true）
DELETE/connectors/{name}刪除

Task 管理

方法路徑說明
GET/connectors/{name}/tasks列出 Task 及其設定
GET/connectors/{name}/tasks/{id}/statusTask 狀態
POST/connectors/{name}/tasks/{id}/restart重啟特定 Task

Offset 管理（KIP-875，Connector 須為 STOPPED 狀態）

方法路徑說明
GET/connectors/{name}/offsets取得目前 Offset
PATCH/connectors/{name}/offsets修改 Offset
DELETE/connectors/{name}/offsets重設所有 Offset

Plugin 與管理

方法路徑說明
GET/connector-plugins列出已安裝 Plugin
PUT/connector-plugins/{type}/config/validate驗證設定
GET/PUT/admin/loggers/{name}取得/設定 Log Level

Exactly-Once 語義 Exactly-Once Delivery

Source Connector（3.3+）

Worker 設定 exactly.once.source.support=enabled。Connector 實作 exactlyOnceSupport() 回傳 SUPPORTED。可自訂 Transaction 邊界。

Sink Connector

Consumer 端設定 consumer.override.isolation.level=read_committed 即可。搭配 Transactional Producer 使用。

💡 升級路徑既有叢集：先設 exactly.once.source.support=preparing 滾動重啟 → 再改為 enabled 滾動重啟。

Connect 資料模型

12 種 Schema 類型：INT8、INT16、INT32、INT64、FLOAT32、FLOAT64、BOOLEAN、STRING、BYTES、ARRAY、MAP、STRUCT

Java — Schema + Struct 範例
Schema schema = SchemaBuilder.struct().name("User")
    .field("name", Schema.STRING_SCHEMA)
    .field("age", Schema.INT32_SCHEMA)
    .build();
Struct struct = new Struct(schema).put("name", "Alice").put("age", 30);
