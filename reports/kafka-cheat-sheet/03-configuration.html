Broker Configs 叢集基礎

Kafka 4.2 為 KRaft-only，五個必要設定：node.id、process.roles、controller.quorum.bootstrap.servers、controller.listener.names、log.dirs。

必要設定

設定預設說明
node.id必填KRaft 模式下的唯一節點識別碼，取代舊版 broker.id
process.roles必填broker / controller / broker,controller
controller.quorum.bootstrap.servers必填Controller Quorum 連線位址
controller.listener.names必填Controller 使用的 Listener 名稱
log.dirs/tmp/kafka-logs資料目錄，多個用逗號分隔（提升 I/O 平行度）

網路與監聽

設定預設說明
listenersPLAINTEXT://:9092Listener URI 清單。用 0.0.0.0 綁定所有介面
advertised.listenersnull（用 listeners）對外公開位址。雲端/容器必設
listener.security.protocol.map每個 Listener 預設 PLAINTEXTListener 名稱 → 協定映射
num.network.threads3每個 Listener 的網路 I/O 執行緒
num.io.threads8請求處理（含磁碟 I/O）執行緒

Log 與保留策略

設定預設說明
log.retention.hours168（7 天）保留時間（優先順序：ms > minutes > hours）
log.retention.msnull毫秒精度保留時間，-1 = 永久
log.retention.bytes-1（無限制）每個 Partition 的最大 Log 大小
log.segment.bytes1 GiB單一 Segment 檔案大小。最小值 4.0 起為 1 MB
log.cleanup.policydeletedelete、compact、delete,compact
num.recovery.threads.per.data.dir2啟動時 Log 復原執行緒（4.0 從 1 改為 2）

Replication 與可靠性

設定預設說明
num.partitions1自動建立 Topic 的預設 Partition 數
default.replication.factor1預設 Replication Factor。生產環境設 3
min.insync.replicas1acks=all 所需的最小 ISR 數。生產環境設 2
unclean.leader.election.enablefalse允許落後 Replica 成為 Leader（有資料遺失風險）
auto.create.topics.enabletrue自動建立 Topic。生產環境設 false
message.max.bytes~1 MB最大 Record Batch 大小，需與 Producer 端 max.request.size 協調
compression.typeproducerBroker 端壓縮覆寫：uncompressed、zstd、lz4、snappy、gzip、producer

內部 Topic Replication

設定預設說明
offsets.topic.replication.factor3__consumer_offsets 的 RF
transaction.state.log.replication.factor3__transaction_state 的 RF
transaction.state.log.min.isr2Transaction 狀態 Topic 的最小 ISR
✅ 生產環境黃金三角
default.replication.factor=3 + min.insync.replicas=2 + Producer 端 acks=all — 確保高可用與資料持久性。

Group Configs 四種 Group Protocol

Kafka 4.2 支援四種 Group Protocol：classic、consumer（KIP-848）、share（KIP-932）、streams（KIP-1071）。
Broker 端透過 group.coordinator.rebalance.protocols 控制啟用哪些，預設值為 classic,consumer,streams。

Classic Group

設定預設說明
group.min.session.timeout.ms6000Consumer Session 逾時下限
group.max.session.timeout.ms1800000（30 min）Consumer Session 逾時上限
group.initial.rebalance.delay.ms3000新 Group 首次 Rebalance 延遲
group.max.sizeInteger.MAX每個 Group 的最大 Consumer 數

Consumer Group（KIP-848，4.0 GA）

設定預設說明
group.consumer.session.timeout.ms45000Broker 端控制的 Session 逾時
group.consumer.heartbeat.interval.ms5000Broker 端控制的 Heartbeat 間隔
group.consumer.migration.policybidirectionalClassic ↔ Consumer Protocol 遷移策略

Share Group（KIP-932，4.2 正式可用）

設定預設範圍說明
group.share.delivery.count.limit5[2, 10]最大投遞次數
group.share.record.lock.duration.ms30000[1s, 1hr]Acquisition Lock 逾時
group.share.max.size200[1, 1000]最大 Share Group 成員數
group.share.partition.max.record.locks2000[100, 10000]每個 Share-Partition 最大鎖定數
group.share.session.timeout.ms45000—Session 逾時
group.share.heartbeat.interval.ms5000—Heartbeat 間隔

Streams Group（KIP-1071，4.2 GA）

設定預設說明
group.streams.session.timeout.ms45000Streams Group Session 逾時
group.streams.heartbeat.interval.ms5000Heartbeat 間隔

Producer Configs 吞吐、延遲與可靠性調校

設定預設說明
acksall0=不等待，1=僅 Leader，all=完整 ISR 確認
retriesInteger.MAX建議改用 delivery.timeout.ms 控制重試總時間
delivery.timeout.ms120000（2 min）發送到確認的總上限時間。須 ≥ request.timeout.ms + linger.ms
batch.size16384（16 KB）每個 Partition 的最大 Batch 大小。加大可提升吞吐
linger.ms5 4.0 變更人為延遲以利 Batching（舊版預設 0）
buffer.memory32 MBProducer 總緩衝區，滿時阻塞後拋例外
max.block.ms60000Send / Metadata 的最大阻塞時間
compression.typenonenone、gzip、snappy、lz4、zstd
enable.idempotencetruePartition 內 Exactly-Once。須 acks=all, inflight ≤ 5
transactional.idnull設定後啟用 Transaction，隱含 Idempotence
max.in.flight.requests.per.connection5未確認請求上限。Idempotent 模式須 ≤ 5
max.request.size1 MB單次請求最大大小，需與 Broker message.max.bytes 協調
request.timeout.ms30000單次請求逾時
💡 吞吐量調校建議
加大 batch.size（如 65536）、提高 linger.ms（如 20~100）、使用 compression.type=lz4 或 zstd。
⚠️ 最大持久性設定
acks=all + enable.idempotence=true + Broker 端 min.insync.replicas=2

Tiered Storage Configs 冷熱分層

Tiered Storage 將熱資料（Local）與冷資料（Remote）分離。需在 Broker 端啟用，再逐 Topic 開啟。

Broker 層級

設定預設說明
remote.log.storage.system.enablefalseTiered Storage 主開關
remote.log.storage.manager.class.namenullRemoteStorageManager 實作類別
remote.log.metadata.manager.class.nameTopicBased...RemoteLogMetadataManager 實作
remote.log.manager.copier.thread.pool.size10Segment 複製到 Remote 的執行緒
remote.log.reader.threads10Remote 讀取執行緒
log.local.retention.ms-2（用 log.retention.ms）本地保留時間。須 ≤ log.retention.ms
log.local.retention.bytes-2（用 log.retention.bytes）本地保留大小。須 ≤ log.retention.bytes

Topic 層級

設定預設說明
remote.storage.enablefalse逐 Topic 開啟 Tiered Storage
local.retention.ms-2覆寫 Topic 的本地保留時間
local.retention.bytes-2覆寫 Topic 的本地保留大小
⚠️ 4.2 棄用
remote.log.manager.thread.pool.size 已棄用，改用更細粒度的 remote.log.manager.follower.thread.pool.size（預設 2）。

Configuration Providers 外部化機敏設定

透過 ConfigProvider 介面從外部來源載入機敏資料（密碼、Token 等），避免明文寫在設定檔中。

語法

server.properties
# 1. 定義 Provider 別名與類別
config.providers=file,env
config.providers.file.class=org.apache.kafka.common.config.provider.FileConfigProvider
config.providers.env.class=org.apache.kafka.common.config.provider.EnvVarConfigProvider

# 2. 限制允許路徑（安全性）
config.providers.file.param.allowed.paths=/etc/kafka/secrets

# 3. 引用值：${別名:路徑:key}
ssl.keystore.password=${file:/etc/kafka/secrets/creds.properties:keystore.password}
sasl.jaas.config=${env:KAFKA_JAAS_CONFIG}

三種內建 Provider

FileConfigProvider

從 Properties 檔案讀取。語法：${alias:/path/file.properties:key}
安全限制：allowed.paths 參數

DirectoryConfigProvider

目錄中每個檔案名為 Key、內容為 Value。適合 Kubernetes Secrets。
語法：${alias:/path/dir:filename}

EnvVarConfigProvider

讀取環境變數。語法：${alias:ENV_VAR_NAME}
安全限制：allowlist.pattern（Regex）

💡 適用範圍
所有 Provider 均可在 Broker、Client、Connect、Streams、MirrorMaker 中使用。不信任環境中請設定 -Dorg.apache.kafka.automatic.config.providers=none 限制自動載入。
