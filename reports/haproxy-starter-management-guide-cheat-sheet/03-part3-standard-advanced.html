§ 3.4.1

Sample Fetch & Converters

🔬 Sample Fetch = 從各處擷取資料

Sample 來源 Converter 鏈 最終用途 ┌─ constants (int,str,IP,bin) ┌─ 算術/邏輯運算 ┌─ ACL 條件判斷 ├─ process (date,env,state,counters) ├─ IP mask 分群 ├─ Stickiness ├─ variables (session/req/resp) ───▶ ├─ 編碼(URL/b64/hex/JSON/hash) ──▶ ├─ 日誌 ├─ client conn (addr,port,counters) ├─ 字串(substr/field/regex/case) ├─ HTTP Header 注入 ├─ SSL session (proto,cipher,SNI,certs) ├─ 日期格式轉換 ├─ 路由決策 ├─ buffer (payload@offset,RDP,TLS SNI) ├─ stick-table 查詢 └─ 回應產生 └─ HTTP (method,URI,hdr,cookie,body) └─ map 鍵值對應 → 任意數量的 converter 可串聯使用

§ 3.4.2

Maps — 快速鍵值對應

🗺️ Maps 運作原理

啟動時載入兩欄檔案 → B-tree 索引 → 查詢輸入 sample → 匹配第一欄 → 回傳第二欄值（或預設值）。

核心優勢

  • 支援 longest match（網路位址）
  • 數十萬筆仍極快（B-tree）
  • 可從 CLI 即時更新
  • 可從 action 用 sample 動態寫入

典型用途

  • IP → 國家代碼（地理定位）
  • IP → AS Number
  • URL → Backend 名稱
  • User-Agent → 裝置類型
# 範例：用 map 做 geolocation 路由 http-request set-header X-Country %[src,map_ip(/etc/haproxy/geo.map,--)] use_backend be_%[req.hdr(Host),lower,map(/etc/haproxy/domains.map,default)]
§ 3.4.3

ACL & 條件

🔑 ACL 組成四元素

① Sample Fetch ──▶ ② Converter 鏈（可選） ──▶ ③ Pattern 清單 ──▶ ④ 比對方法 範例： acl is_api hdr(Host) -i api.example.com # 字串比對（不分大小寫） acl is_cdn path_beg /static /images /css # 前綴比對 acl abuse src_http_req_rate() gt 100 # stick-table 速率比對

比對方法約 13 種：IP mask、整數範圍、子字串、正則、前綴、後綴、域名等。

組合方式：if is_api !is_cdn（AND）、if is_api || is_cdn（OR）、if !abuse（NOT）

💡 ACL 與 Map 共用同一底層
內部結構、比對方法、效能完全相同。差異只在 ACL 回傳 found/not found，Map 回傳值。
§ 3.4.4

Content Switching — 內容路由

🔀 用 ACL 把流量導到不同 Backend

# 範例：虛擬主機 + 路徑路由 frontend http-in bind *:80 acl host_api hdr(host) -i api.example.com acl path_static path_beg /static use_backend be_api if host_api use_backend be_static if path_static default_backend be_web # 動態 content switching — sample 直接變成 backend 名稱（免 ACL） use_backend be_%[req.hdr(Host),lower,map(/etc/haproxy/hosts.map,be_default)]

已報告可正常運作於 300,000 個 backend 的環境。

§ 3.4.5

Stick-tables — 共享儲存引擎

🗄️ Stick-table 不只做 Stickiness

Stick-table 是 HAProxy 內建的高速 in-memory key-value store，可儲存多種統計資料。

Key 類型追蹤統計進階用途
ip
integer
string
binary		 conn_cur 併發連線
conn_rate 連線速率
http_req_rate 請求速率
http_err_rate 錯誤率
bytes_in/out_rate 頻寬
gpc0/gpc1 通用計數器 		▸ DDoS 偵測：http_req_rate > 閾值 → 封鎖/限速
▸ QoS 調整：依客戶等級分配優先順序
▸ 暴力破解防護：追蹤登入失敗
▸ 回訪者辨識：gpc 標記已知用戶

最多 8 個平行 key、最多 3 個同時追蹤 key（來自不同 table）。

透過 peers 實現主動-主動複寫（含 reload 時新舊行程間複寫）。

⚠️ 未設 peers → reload 時資料全丟
即使單節點也建議設定 peers section — 確保 reload 期間 stick-table 資料不遺失。
§ 3.4.6

Formatted Strings

📝 動態字串產生引擎

原名 "log-format"，但用途遠超日誌：header 內容、回應資料、模板、自訂日誌行。

  • 嵌入變數和 sample fetch 表達式
  • 結果轉字串時可調整編碼（如加引號）
  • ~50 種快捷標籤涵蓋常用日誌欄位
# 範例 http-response set-header X-Request-ID %[unique-id] log-format "%ci:%cp [%tr] %ft %b/%s %TR/%Tw/%Tc/%Tr/%Ta %ST %B"
§ 3.4.7

HTTP 改寫 & 重導

✏️ 改寫 ≠ 重導

Rewriting 改寫

Client 與 Server 看到不同內容（URL、header）。Client 不知道。

http-request set-path /v2%[path] http-request set-header X-Forwarded-Proto https http-response del-header Server

Redirecting 重導

回傳 3xx 請 client 訪問新 URL。Client 看得到。

http-request redirect scheme https code 301 http-request redirect location /new%[capture.req.uri] http-request redirect prefix /app if !{ path_beg /app }

return 指令可直接回傳完全自訂回應（狀態碼、header、body 模板），所有操作皆支援 ACL 條件。

§ 3.4.8

Server Protection — 七層防禦

層防禦說明
1請求完整性僅轉發完整有效請求。Slowloris 對 HAProxy 幾乎無影響
2緩衝卸載請求存到完整才連 server → server 連線短暫 → 節省資源
3連線/請求限制Per-server maxconn；超額排隊、可重派、可中斷。防止「reload 風暴」
4Slow-start保護重啟中的 server（JIT 編譯、cache 預熱期間）
5協定層防護寬鬆解析修正無害偏差；拒絕安全關鍵問題（如重複 Content-Length 值不同）
6攻擊回應回傳 400、TCP reset、或 tarpit（長延遲假錯誤 → 使攻擊成本極高）
7資料外洩防護記錄/阻擋可疑回應（如可快取回應含 cookie → session 分享風險）
§ 3.5

進階功能

🖥️ §3.5.1 CLI 管理 — 不改設定檔即可操作

Server 管理

  • 變更 server 位址/權重/狀態
  • 新增/移除 server（動態）
  • 停用/啟用前端
  • 終止指定 session

  • dump/清除 stick-table（可按 key 過濾）

四大推薦工具：

socat — CLI 連線 halog — 日誌分析（1-2 GB/sec） tcpdump — 網路追蹤 strace — 系統層除錯

🐧 §3.5.2 Linux 特定最佳化

Network Namespace（跨容器閘道） TCP FastOpen TCP User Timeout TCP Splicing（kernel 側零拷貝轉發） defer-accept tcp-smart-connect

HAProxy 維護自己的單調時鐘，補償系統時鐘漂移（NUMA / 虛擬化環境曾觀察到 30 秒跳躍）。

📜 §3.5.3 Lua 腳本

嵌入 Lua 處理：複雜請求/回應操作、路由決策、統計處理、平行連線到其他伺服器。

典型用例：自訂認證系統

🔍 §3.5.4 Tracing

透過 CLI 即時開啟各子系統追蹤。詳細度從每請求 1 行到 ~500 行。

支援 filter + 自動 capture on/off/pause — 等特定事件發生再記錄。

版本、授權 & 取得方式

📦 四大取得來源

來源特點
haproxy.org（官方社群）原始碼；釋出週期較慢
GitHubMirror + 開發分支；供貢獻者
OS 發行版長期維護但修補可能落後

驗證版本：haproxy -v（簡要）/ haproxy -vv（詳細 — 回報 bug 時必附）。

CH 4-5

互補產品 & 替代方案

產品與 HAProxy 的關係
Apache可做反向代理但效能/功能不及 HAProxy。適合放在 HAProxy 後面做靜態/動態內容
Nginx可做反向代理和基本 LB。HAProxy 在 L7 LB、stick-table、ACL 進階功能上更強。可互補
Varnish進階快取的首選。與 HAProxy 完美整合（HAProxy 做 SSL + LB，Varnish 做 cache）
IPVSLinux kernel L4 LB → 與 HAProxy（L7）完美互補
💡 經典組合架構
IPVS（L4） → HAProxy（L7 + SSL） → Varnish（Cache） → Nginx/Apache（Web Server）
