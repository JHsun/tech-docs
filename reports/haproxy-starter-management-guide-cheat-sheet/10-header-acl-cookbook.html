X-Forwarded-* & XFF 防偽造

🛡️ XFF 防偽造模式（必備安全設定）

# ⚠️ 錯誤：僅用 option forwardfor → 客戶端可注入假 XFF # option forwardfor ← 這會 append，不會替換客戶端送來的值 # ✅ 正確：先刪除再設定（防止偽造） frontend ft_web bind *:443 ssl crt /etc/haproxy/certs/site.pem # 步驟 1：刪除客戶端可能注入的 header http-request del-header X-Forwarded-For http-request del-header X-Real-IP http-request del-header X-Forwarded-Proto http-request del-header X-Forwarded-Host # 步驟 2：設定可信值 http-request set-header X-Forwarded-For %[src] http-request set-header X-Real-IP %[src] http-request set-header X-Forwarded-Proto %[ssl_fc,iif(https,http)] http-request set-header X-Forwarded-Host %[req.hdr(Host)] http-request set-header X-Forwarded-Port %[dst_port] default_backend bk_web
🚨 option forwardfor 的陷阱
option forwardfor 只會 append，不會替換。攻擊者送 X-Forwarded-For: 10.0.0.1 → 後端看到 10.0.0.1, 真實IP → 若後端取第一個值 = 被偽造。

📋 多層代理的 XFF 處理

# 若 HAProxy 前方有可信代理（如 CDN），保留其 XFF 並 append frontend ft_behind_cdn bind *:80 acl from_cdn src 198.51.100.0/24 203.0.113.0/24 # CDN IP 範圍 # 來自 CDN：信任其 XFF，append 自己的 http-request set-header X-Forwarded-For %[req.hdr(X-Forwarded-For)],%[src] if from_cdn # 不來自 CDN：覆寫（防偽造） http-request del-header X-Forwarded-For unless from_cdn http-request set-header X-Forwarded-For %[src] unless from_cdn
02

安全 Header 套裝（直接複製貼上）

# ============================================================ # 安全 Header 套裝 — 加在 frontend 或 backend # ============================================================ # HSTS：強制 HTTPS（2 年 + 包含子域 + preload） http-response set-header Strict-Transport-Security "max-age=63072000; includeSubDomains; preload" # 防止 Clickjacking http-response set-header X-Frame-Options "SAMEORIGIN" # 防止 MIME type sniffing http-response set-header X-Content-Type-Options "nosniff" # Referrer 控制 http-response set-header Referrer-Policy "strict-origin-when-cross-origin" # 權限控制（依需求調整） http-response set-header Permissions-Policy "camera=(), microphone=(), geolocation=()" # CSP（依應用調整 — 以下為嚴格範例） http-response set-header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; font-src 'self'; frame-ancestors 'self'" # 移除已棄用的 X-XSS-Protection（現代瀏覽器不需要） http-response del-header X-XSS-Protection # 移除洩漏伺服器資訊的 header http-response del-header Server http-response del-header X-Powered-By http-response del-header X-AspNet-Version
⚠️ CSP 須依應用調整
過嚴的 CSP 會破壞功能（如 inline script、CDN 資源）。建議先用 Content-Security-Policy-Report-Only 測試，確認無誤再切換為強制模式。
03

CORS 處理

🌐 單一來源 CORS（最簡單）

frontend ft_api bind *:443 ssl crt /etc/haproxy/certs/api.pem mode http # 處理 preflight OPTIONS acl is_options method OPTIONS http-request return status 204 \ hdr Access-Control-Allow-Origin "https://app.example.com" \ hdr Access-Control-Allow-Methods "GET,POST,PUT,DELETE,OPTIONS" \ hdr Access-Control-Allow-Headers "Authorization,Content-Type" \ hdr Access-Control-Max-Age "86400" \ if is_options # 正常請求加 CORS header http-response set-header Access-Control-Allow-Origin "https://app.example.com" http-response set-header Access-Control-Allow-Credentials "true" default_backend bk_api

🔀 多來源 CORS（用 map 動態匹配）

# /etc/haproxy/cors-origins.lst（一行一個允許的 origin） # https://app1.example.com # https://app2.example.com # https://staging.example.com frontend ft_api bind *:443 ssl crt /etc/haproxy/certs/api.pem # 檢查 Origin 是否在白名單 acl is_cors_origin req.hdr(Origin) -f /etc/haproxy/cors-origins.lst acl is_options method OPTIONS # Preflight：動態回傳請求的 Origin（而非 *） http-request return status 204 \ hdr Access-Control-Allow-Origin %[req.hdr(Origin)] \ hdr Access-Control-Allow-Methods "GET,POST,PUT,DELETE,OPTIONS" \ hdr Access-Control-Allow-Headers "Authorization,Content-Type" \ hdr Access-Control-Allow-Credentials "true" \ hdr Access-Control-Max-Age "86400" \ hdr Vary "Origin" \ if is_options is_cors_origin # 正常回應 http-response set-header Access-Control-Allow-Origin %[req.hdr(Origin)] if is_cors_origin http-response set-header Access-Control-Allow-Credentials "true" if is_cors_origin http-response set-header Vary "Origin" if is_cors_origin # 未在白名單的 OPTIONS → 403 http-request deny deny_status 403 if is_options !is_cors_origin default_backend bk_api
💡 為什麼不用 Access-Control-Allow-Origin: *？
* 不能與 Access-Control-Allow-Credentials: true 同時使用。若需要帶 cookie 的跨域請求，必須回傳精確的 origin 值。

URL 改寫模式

指令功能範例
set-path替換路徑（保留 query string）http-request set-path /v2%[path]
set-uri替換完整 URI（含 query）http-request set-uri %[url,regsub(^/old,/new)]
replace-path正則替換路徑http-request replace-path ^/api/v1(.*) /api/v2\1
replace-uri正則替換完整 URIhttp-request replace-uri ^/blog/(.*) /articles/\1
set-query替換 query stringhttp-request set-query %[query]&source=haproxy

📋 常用改寫範例

# 路徑前綴移除（strip /api prefix） http-request replace-path ^/api(/.*)$ \1 # 路徑前綴添加 http-request set-path /app%[path] # 尾部斜線正規化（加上 /） http-request redirect code 301 location %[url]/ if { path_reg ^/[^.]*[^/]$ } # 移除重複斜線 http-request replace-path ^(.*)//+(.*) \1/\2 # 小寫化路徑 http-request set-path %[path,lower] # 重導 www → non-www http-request redirect prefix https://example.com code 301 if { hdr(host) -i www.example.com }

Content Switching 模式

# ===== Host-based 虛擬主機 ===== frontend ft_web bind *:443 ssl crt /etc/haproxy/certs/ use_backend bk_api if { hdr(host) -i api.example.com } use_backend bk_admin if { hdr(host) -i admin.example.com } default_backend bk_web # ===== Path-based 路由 ===== use_backend bk_static if { path_beg /static /images /css /js } use_backend bk_api if { path_beg /api /graphql } use_backend bk_ws if { hdr(Upgrade) -i websocket } # ===== Method-based 路由 ===== use_backend bk_read if { method GET HEAD OPTIONS } use_backend bk_write if { method POST PUT DELETE PATCH } # ===== 動態 backend（用 map，免 ACL）===== use_backend %[req.hdr(Host),lower,map(/etc/haproxy/hosts.map,bk_default)] # /etc/haproxy/hosts.map： # api.example.com bk_api # admin.example.com bk_admin # www.example.com bk_web

棄用語法遷移對照表

🔄 1.x → 2.x+ 語法遷移

舊語法（已棄用）新語法說明
reqadd X-Custom:\ valhttp-request add-header X-Custom val新增 request header
rspadd X-Custom:\ valhttp-response add-header X-Custom val新增 response header
reqdel ^X-Bad:http-request del-header X-Bad刪除 request header
rspdel ^Server:http-response del-header Server刪除 response header
reqrep ^(.*) /new\1http-request replace-path ...URL 正則替換
rsprep ^(Set-Cookie:.*) \1;Securehttp-response replace-header Set-Cookie ...Response header 正則替換
redirect prefixhttp-request redirect prefix重導（功能不變）
block if ...http-request deny if ...拒絕請求
⚠️ 舊語法仍能用但會產生警告
HAProxy 3.x 啟用 -dW（zero-warning）時，舊語法會阻止啟動。建議全面遷移。
