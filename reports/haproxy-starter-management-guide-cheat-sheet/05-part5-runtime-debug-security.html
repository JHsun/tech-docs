CH 9

Stats Socket 設定 & 連線

🔌 設定 Runtime API 接口

global stats socket /var/run/haproxy.sock mode 600 level admin stats socket ipv4@127.0.0.1:9999 level operator stats timeout 2m

連線方式：

# 互動模式 socat /var/run/haproxy.sock readline # 腳本模式（單一指令） echo "show info" | socat stdio /var/run/haproxy.sock # 多行輸入 socat /var/run/haproxy.sock - <<'EOF' show stat show info EOF
⚠️ 安全等級
level admin：完整控制（變更 server 狀態、清除 stats）。level operator：唯讀 + 有限操作。level user：唯讀。生產環境用 UNIX socket + mode 600，絕不公開 TCP。

SSL 憑證管理指令（25+）

🔒 憑證生命週期

new ssl cert <path> 建立空憑證插槽 │ set ssl cert <path> <<payload 上傳新憑證內容（PEM） │ commit ssl cert <path> 套用到所有使用該憑證的 bind │ （若需取消） abort ssl cert <path> 撤銷尚未 commit 的變更

相同模式適用於 CA 檔 (ssl ca-file)、CRL 檔 (ssl crl-file)。

# 完整更新憑證範例 echo -e "set ssl cert /etc/ssl/site.pem <<\n$(cat new-cert.pem)\n" | \ socat stdio /var/run/haproxy.sock echo "commit ssl cert /etc/ssl/site.pem" | socat stdio /var/run/haproxy.sock # 查詢憑證資訊 echo "show ssl cert /etc/ssl/site.pem" | socat stdio /var/run/haproxy.sock # CRT-list 管理 echo "show ssl crt-list /etc/ssl/crt-list.txt" | socat stdio /var/run/haproxy.sock echo "add ssl crt-list /etc/ssl/crt-list.txt /etc/ssl/newsite.pem" | socat stdio /var/run/haproxy.sock

Prometheus Exporter & Stats-file

📈 Prometheus（內建）

frontend stats bind *:8404 http-request use-service \ prometheus-exporter \ if { path /metrics }

輸出 150+ 指標。編譯時加 USE_PROMEX=1。

💾 Stats-file（計數器持久化）

# 1. 設定中加 GUID # 2. dump echo "dump stats-file" | \ socat stdio haproxy.sock \ > /etc/haproxy/stats-file # 3. global 加入 stats-file /etc/haproxy/stats-file

Reload 後計數器從檔案初始化。

CH 11

常見陷阱（Well-known Traps）

陷阱症狀修正
VIP 綁定失敗 重開機時 HAProxy 先於 keepalived 啟動 → 無法綁定 VIP → 啟動失敗 sysctl -w net.ipv4.ip_nonlocal_bind=1
幽靈舊行程 SO_REUSEPORT 允許多行程監聽同 port。未停止的舊行程接收部分連線 → 設定變更「不生效」 netstat -lntp 找到並殺掉 rogue process
CLI ACL 不持久 用 add acl 新增的項目 reload 後消失 同步更新磁碟檔案
多行程 source port 衝突 多行程用同一 source port range → 高負載下隨機失敗 改用 nbthread，棄用 nbproc

除錯方法論

🔬 三工具延遲診斷法

同時使用三個工具，對比時間戳找瓶頸： ① tcpdump（本地端） tcpdump -nvvttSs0 -i eth0 port 80 → 看資料何時到達網路介面 ② strace strace -tts200 -p <haproxy_pid> → 看 HAProxy 何時處理資料 ⚠️ 必須加 -dS 停用 splice() 否則看不到轉發資料 ③ tcpdump（遠端） → 看資料何時到達/離開 server 端

🔎 常見延遲模式

症狀原因修正
資料收到但未送出Output buffer 滿（接收端太慢）檢查 server 或 client 端瓶頸
40ms 或 200ms 延遲Nagle 演算法（idle 流量）option http-no-delay
本地送出但遠端未收到WAN 壅塞、LAN flow control、或虛擬化延遲（VM 中幾乎一定是此原因）檢查 hypervisor
TCP SACK segment偵測到封包遺失檢查網路品質

📊 netstat -i 介面診斷

計數器增長意義
Rx-Ovr系統資源不足以接收封包
Rx-Drp封包在 network stack 中遺失（應用太慢）
Tx-Drp輸出佇列滿（出站鏈路飽和）
CH 13

安全強化

🛡️ 安全架構原則

HAProxy 設計為「以 root 啟動，以非 root 運行」。啟動需 root 的原因：調整 FD 限制、綁定特權 port（<1024）、設定 chroot、降權到非特權 UID。運行時需 root 的場景極少（出站特權 source port、transparent bind — 大多數用戶不需要）。

🔧 建議安全設定

global chroot /var/empty # 空目錄 chmod 0 user haproxy group haproxy stats socket /var/run/haproxy.sock uid hatop gid hatop mode 600 # 建立 chroot 目錄 mkdir /var/empty && chmod 0 /var/empty

🐧 Linux Capabilities（§13.1）

Capability用途
CAP_NET_BIND_SERVICE綁定特權 port（<1024）免 root
CAP_SYS_CHROOT執行 chroot() 免 root
CAP_SETUID / CAP_SETGID降權
CAP_NET_RAWTransparent proxying（TPROXY）
CAP_NET_ADMIN特定網路操作
# 設定 capabilities（免 root 綁定特權 port） setcap 'cap_net_bind_service=+ep' /usr/sbin/haproxy

🔒 額外強化要點

  • Stats socket 永遠用 UNIX socket + mode 600，絕不公開 TCP
  • 日誌可能含機密資訊 — 與未授權存取日誌分開
  • harden.reject-privileged-ports.tcp / .quic — 拒絕來自特權 source port 的連線
  • set-dumpable — 降權後重新啟用 core dump（除錯用）
