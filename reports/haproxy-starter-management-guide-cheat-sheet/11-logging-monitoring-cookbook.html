五大計時欄位圖解

Client HAProxy Server ──TCP connect──▶ ──request──▶ ──queue──▶ ──TCP connect──▶ ──response──▶ ──data──▶ │ │ │ │ │ │ ├────Tq─────┤ │ │ │ │ │ 請求接收 ├────Tw────┤ │ │ │ │ │ 佇列等待 ├────Tc───┤ │ │ │ │ │ TCP 建立 ├──────Tr──────┤ │ │ │ │ │ 回應 header │ │ ├─────────────┴───────────┴───────────┴──────────────┴──Ta──────────┤ │ 總時間 │
欄位起點終點異常值含義
Tq連線接受完整 request 收到高 = 慢客戶端或大 body
Twrequest 完成分配到 server高 = server 過載，佇列中等待
Tc開始連 serverTCP 建立完成高 = 網路延遲或 server 無回應
Trrequest 送出response header 收到高 = server 思考時間（最重要的後端效能指標）
Ta連線接受最後一個 byte 傳完≈ Tq+Tw+Tc+Tr + data 傳輸
💡 快速診斷
Tq 高 → 客戶端問題。Tw > 0 → server 全忙（檢查 maxconn）。Tc 高 → 網路或 server 健康。Tr 高 → 後端應用慢。Ta ≫ Tq+Tw+Tc+Tr → 回應 body 傳輸慢。
03

終止狀態碼（Termination State Code）

🔤 四個字元的含義

格式：XYZW — X=會話關閉原因, Y=會話狀態, Z=持久化 cookie, W=持久化 cookie 處理

📋 第一字元 X：會話關閉原因（最常查）

碼含義常見場景
CClient 先關閉用戶取消、瀏覽器關閉
SServer 先關閉Server crash、主動斷開
PProxy 主動關閉HAProxy deny/tarpit/redirect
LLocal 處理Stats 頁面、CLI、cache 命中
RResource 不足記憶體不足、maxconn 達上限
IInternal 錯誤HAProxy bug（應回報）
DDenied被 ACL 或 TCP request rule 拒絕
cClient timeouttimeout client 到期
sServer timeouttimeout server 到期
-正常完成一切正常

📋 第二字元 Y：會話狀態（中斷發生在哪個階段）

碼階段說明
RRequest接收 request 期間
QQueue等待 server 佇列中
CConnect連線 server 期間
HResponse Header等待 server 回應 header
DData傳輸回應 body 期間
LLast傳輸最後資料
TTarpittarpit 等待期間
-完成正常結束

🔥 最常見組合解讀

碼含義對策
----✅ 正常完成無需動作
CD--Client 在資料傳輸中斷開正常（用戶取消下載等）
CR--Client 在送 request 時斷開正常（瀏覽器取消）
SC--Server 在連線後立即關閉⚠️ 檢查 server 健康
sH--Server timeout（等 response header）⚠️ server 回應太慢，調 timeout server
sD--Server timeout（傳資料中）⚠️ 長時間傳輸超時
PR--HAProxy deny 在 request 階段ACL 規則阻擋
PH--HAProxy deny 在等 response 階段http-response deny
LR--本地處理（stats/CLI/redirect）正常
cR--Client 送 request timeouttimeout http-request 太短或 Slowloris

JSON 日誌

🆕 HAProxy 3.0+ 原生 JSON 編碼（推薦）

defaults mode http log global option httplog log-format '{"time":"%Tl","client":"%ci:%cp","frontend":"%f","backend":"%b","server":"%s","status":%ST,"bytes":%B,"timers":{"tq":%Tq,"tw":%Tw,"tc":%Tc,"tr":%Tr,"ta":%Ta},"method":"%HM","uri":"%{+Q,+E}HU","termination":"%tsc","retries":%rc,"srv_queue":%sq,"bk_queue":%bq}'
💡 3.0+ 的 %{+E} 自動跳脫
%{+E} 自動對特殊字元做 JSON 安全跳脫。%{+Q} 加雙引號。兩者組合 %{+Q,+E} 最安全。

⚠️ HAProxy 2.x 手動 JSON（注意跳脫）

# 2.x 需手動建構 JSON + 防止用戶輸入破壞格式 log-format '{"time":"%Tl","client":"%ci","fe":"%f","be":"%b","srv":"%s","st":%ST,"b":%B,"tq":%Tq,"tw":%Tw,"tc":%Tc,"tr":%Tr,"ta":%Ta,"method":"%HM","uri":"%[capture.req.uri,json(utf8s)]","tsc":"%tsc"}' # json(utf8s) converter：將非 UTF-8 和特殊字元安全跳脫 # 不用它 → 用戶在 URL 中放 " 或 \ 會破壞 JSON
⚠️ 預設日誌長度僅 1024 字元
JSON 日誌容易超限。設定 log 127.0.0.1:514 local0 len 8192 或更大值。

日誌過濾與取樣

# 不記錄健康檢查和空連線 option dontlognull # 不記錄正常請求，僅記錄異常（大幅減少日誌量） option dontlog-normal # ACL 調整 log level（降噪+提高異常可見度） http-request set-log-level silent if { path_beg /healthz } http-request set-log-level warning if { status ge 500 } # 日誌取樣（僅記錄 1/10 的正常流量） http-request set-log-level silent unless { rand(100) lt 10 } || { status ge 400 } # 每個 frontend/backend 獨立日誌輸出（多租戶） frontend ft_tenant_a log 127.0.0.1:514 local2 log-tag tenant-a frontend ft_tenant_b log 127.0.0.1:514 local3 log-tag tenant-b

Prometheus 告警指標

📊 設定

# 編譯時需 USE_PROMEX=1 frontend stats bind *:8404 http-request use-service prometheus-exporter if { path /metrics } # 僅限內網存取 http-request deny unless { src 10.0.0.0/8 }

🔔 關鍵告警指標

指標危險閾值含義
haproxy_process_idle_time_percent< 30%CPU 即將飽和
haproxy_backend_current_queue> 0 持續後端容量不足
haproxy_server_status非 1Server DOWN
haproxy_backend_http_responses_total{code="5xx"}突增後端錯誤
haproxy_backend_response_time_average_seconds> SLA回應時間超標
haproxy_frontend_denied_connections_total突增攻擊或設定問題
haproxy_backend_retry_warnings_total突增後端不穩定
haproxy_process_current_connections接近 maxconn容量瓶頸

🖥️ CLI 快速查看（免 Prometheus）

# CPU 使用率（最重要） echo "show info" | socat stdio $SOCK | grep Idle_pct # 連線概況 echo "show info" | socat stdio $SOCK | grep -E "^(CurrConns|MaxConn|ConnRate|SessRate)" # 各 backend 狀態總覽 echo "show stat" | socat stdio $SOCK | cut -d, -f1,2,18,37 | column -t -s, # SSL 計數器 echo "show info" | socat stdio $SOCK | grep -i ssl
07

halog 日誌分析工具

⚡ 處理速度 1–2 GB/秒

# 依回應時間排序 URL（找最慢的） halog -srv < /var/log/haproxy.log | sort -k3 -rn | head -20 # 依請求次數排序 URL（找最熱門的） halog -ut < /var/log/haproxy.log | sort -k2 -rn | head -20 # 依來源 IP 排序（找最活躍的客戶端） halog -ic < /var/log/haproxy.log | sort -k2 -rn | head -20 # 僅篩選 5xx 錯誤 halog -hs 500-599 < /var/log/haproxy.log # 僅篩選回應時間 > 5 秒的請求 halog -s 5000 < /var/log/haproxy.log # 篩選特定終止碼 halog -tc sH < /var/log/haproxy.log # server timeout halog -tc SC < /var/log/haproxy.log # server close halog -tc PR < /var/log/haproxy.log # proxy deny

自訂錯誤頁

📄 errorfile（靜態檔案）

# 檔案須含完整 HTTP 回應 # /etc/haproxy/errors/503.http HTTP/1.1 503 Service Unavailable Content-Type: text/html Cache-Control: no-cache <html><body> <h1>維護中</h1> </body></html>
defaults errorfile 400 /etc/haproxy/errors/400.http errorfile 403 /etc/haproxy/errors/403.http errorfile 408 /etc/haproxy/errors/408.http errorfile 500 /etc/haproxy/errors/500.http errorfile 502 /etc/haproxy/errors/502.http errorfile 503 /etc/haproxy/errors/503.http errorfile 504 /etc/haproxy/errors/504.http

⚡ http-request return（動態）

# 內聯字串 http-request return \ status 503 \ content-type "text/html" \ string "<h1>維護中</h1>" \ if { path_beg /api } # 從檔案載入 body http-request return \ status 503 \ content-type "text/html" \ file /etc/haproxy/maint.html # JSON 回應 http-request return \ status 429 \ content-type "application/json" \ string '{"error":"too_many"}' # 含自訂 header http-request return \ status 503 \ hdr Retry-After "300" \ content-type "text/plain" \ string "Back in 5 min"
