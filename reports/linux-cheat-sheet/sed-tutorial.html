01

什麼是 sed？

sed（Stream Editor，串流編輯器）是 Unix/Linux 系統中最經典的文字處理工具之一。它能逐行讀取輸入資料，依照指定的編輯指令進行轉換，再將結果輸出。由於不需要開啟互動式編輯器，sed 特別適合用在腳本自動化、批次處理、日誌分析等場景。

核心概念：sed 不會直接修改原始檔案（除非加上 -i 選項），它預設將處理結果輸出到標準輸出（stdout），原始檔案保持不變。

sed 的工作流程

每一次循環中，sed 會執行以下步驟：

  1. 從輸入讀取一行文字到「模式空間」（Pattern Space）
  2. 對模式空間中的內容執行所有指令
  3. 將模式空間的內容輸出到標準輸出
  4. 清空模式空間，讀取下一行，重複以上步驟
02

基本語法結構

基本語法
sed [選項] '指令' 輸入檔案

# 也可以搭配管線使用
指令 | sed [選項] '指令'

其中「指令」的格式為：

指令格式
[位址]命令

# 範例：
'3'd            # 刪除第 3 行
'/error/'p       # 列印含有 error 的行
'1,5's/old/new/ # 在第 1~5 行替換 old 為 new
03

常用選項（Options）

選項說明範例
-n 安靜模式，抑制自動輸出，僅輸出經 p 指令明確要求的行 sed -n '3p' file
-e 指定多個編輯指令（可串接多個 -e） sed -e 's/a/A/' -e 's/b/B/' file
-i 直接修改原始檔案（就地編輯） sed -i 's/old/new/g' file
-i.bak 就地編輯並建立備份檔案（副檔名為 .bak） sed -i.bak 's/old/new/g' file
-r / -E 啟用擴展正規表示式（ERE），免除部分跳脫字元 sed -E 's/(foo|bar)/baz/g' file
-f 從檔案讀取 sed 指令 sed -f script.sed file
macOS 注意：macOS 預設使用 BSD 版 sed，-i 後面必須加空字串 -i ''，否則會報錯。GNU sed（Linux）則可以直接寫 -i。
04

替換指令（s）— 最常用的功能

替換指令是 sed 最核心的功能，語法如下：

替換語法
s/搜尋模式/替換文字/旗標

替換旗標（Flags）

旗標說明
g全域替換（替換該行所有匹配項，而非僅第一個）
數字只替換第 N 個匹配項，如 2 替換第二個
p列印替換成功的行
w 檔案將替換成功的行寫入指定檔案
i / I忽略大小寫匹配（GNU sed 擴展）

基本替換範例

範例：基本替換
# 將每行第一個 "cat" 替換為 "dog"
echo "I have a cat and a cat" | sed 's/cat/dog/'
OUTPUT
I have a dog and a cat

# 全域替換（加上 g 旗標）
echo "I have a cat and a cat" | sed 's/cat/dog/g'
OUTPUT
I have a dog and a dog

# 只替換第 2 個匹配項
echo "aaa bbb aaa bbb aaa" | sed 's/aaa/XXX/2'
OUTPUT
aaa bbb XXX bbb aaa

使用不同分隔符

當搜尋模式中包含 / 時（如路徑），可以改用其他字元作為分隔符：

範例：自訂分隔符
# 使用 | 作為分隔符（替換檔案路徑）
sed 's|/usr/local/bin|/opt/bin|g' config.txt

# 使用 # 作為分隔符
sed 's#http://#https://#g' urls.txt

# 使用 @ 作為分隔符
sed 's@oldpath@newpath@g' file.txt

反向參照（Back Reference）

使用 \( 和 \) 捕捉群組，再用 \1、\2 等引用：

範例：反向參照
# 將 "John Smith" 轉為 "Smith, John"
echo "John Smith" | sed 's/\(.*\) \(.*\)/\2, \1/'
OUTPUT
Smith, John

# 使用 ERE（-E），括號不需跳脫
echo "John Smith" | sed -E 's/(.*) (.*)/\2, \1/'
OUTPUT
Smith, John

# 在每個單字前後加上引號
echo "hello world" | sed -E 's/([a-z]+)/"\1"/g'
OUTPUT
"hello" "world"

# & 代表整個匹配的內容
echo "1234" | sed 's/[0-9]*/[&]/'
OUTPUT
[1234]

刪除指令（d）

範例：刪除操作
# 刪除第 3 行
sed '3d' file.txt

# 刪除第 2 到第 5 行
sed '2,5d' file.txt

# 刪除最後一行
sed '$d' file.txt

# 刪除空白行
sed '/^$/d' file.txt

# 刪除含有 "DEBUG" 的行
sed '/DEBUG/d' log.txt

# 刪除以 # 開頭的註解行
sed '/^#/d' config.txt

# 刪除以 # 開頭的行和空白行
sed -e '/^#/d' -e '/^$/d' config.txt

# 刪除第 1 行到第一個空白行
sed '1,/^$/d' file.txt
06

插入、附加與變更（i / a / c）

指令說明
i\在指定行之前插入文字
a\在指定行之後附加文字
c\將指定行整行取代為新文字
範例：插入、附加、變更
# 在第 1 行之前插入一行文字
sed '1i\#!/bin/bash' script.sh

# 在第 3 行之後附加一行文字
sed '3a\# 這是新加入的註解' file.txt

# 在最後一行之後附加文字
sed '$a\END OF FILE' file.txt

# 將含有 "old_config" 的行整行替換
sed '/old_config/c\new_config=true' config.txt

# 在每行匹配 "TODO" 的行之前插入空行
sed '/TODO/i\\' file.txt
07

列印指令（p）與行號（=）

範例：列印操作
# 列印第 5 行（搭配 -n 避免重複輸出）
sed -n '5p' file.txt

# 列印第 10 到第 20 行
sed -n '10,20p' file.txt

# 列印含有 "error" 的行（類似 grep）
sed -n '/error/p' log.txt

# 忽略大小寫列印含 "error" 的行
sed -n '/error/Ip' log.txt

# 列印第 1 行到匹配 "END" 的行
sed -n '1,/END/p' file.txt

# 顯示行號
sed -n '/pattern/=' file.txt

# 同時顯示行號和內容
sed -n '/pattern/{=;p}' file.txt
08

位址定位（Addressing）

位址用來指定 sed 指令要作用在哪些行上。

位址格式說明範例
n第 n 行sed '5d' — 刪除第 5 行
$最後一行sed '$d' — 刪除最後一行
n,m第 n 到第 m 行sed '2,5d' — 刪除 2~5 行
n~step從第 n 行開始，每隔 step 行sed '1~2d' — 刪除奇數行
/pattern/匹配正規表示式的行sed '/^#/d' — 刪除 # 開頭的行
/p1/,/p2/從匹配 p1 到匹配 p2 的範圍sed '/START/,/END/d'
n,/pattern/從第 n 行到匹配 pattern 的行sed '1,/^$/d'
!反向選取（不匹配的行）sed '/keep/!d' — 只保留含 keep 的行
範例：位址定位
# 只在第 1 行進行替換
sed '1s/hello/HELLO/' file.txt

# 從第 3 行到最後一行替換
sed '3,$s/foo/bar/g' file.txt

# 每隔一行列印（列印偶數行）
sed -n '2~2p' file.txt

# 反選：刪除「不含」error 的所有行（保留含 error 的行）
sed '/error/!d' log.txt

# 在 BEGIN 和 END 之間的行進行替換
sed '/BEGIN/,/END/s/old/new/g' file.txt
09

sed 中的正規表示式

基本正規表示式（BRE）

符號說明範例
.匹配任意單一字元s/h.t/hat/
*前一字元重複 0 次或多次s/ab*/a/
^行首/^#/ 匹配 # 開頭的行
$行尾/;$/ 匹配 ; 結尾的行
[abc]字元集合，匹配 a、b 或 cs/[aeiou]//g
[^abc]否定字元集合s/[^0-9]//g
\( \)捕捉群組s/\(.*\)/[\1]/
\1 \2反向參照第 N 個群組見替換章節範例
\{n\}前一字元恰好重複 n 次/a\{3\}/ 匹配 aaa
\{n,m\}前一字元重複 n 到 m 次/a\{2,4\}/

擴展正規表示式（ERE）— 使用 -E 選項

ERE 與 BRE 功能相同，但括號和量詞不需要跳脫，寫起來更簡潔：

ERE 符號等同 BRE說明
+\+前一字元至少出現 1 次
?\?前一字元出現 0 或 1 次
()\( \)捕捉群組
{n,m}\{n,m\}重複次數範圍
|\|或（alternation）
範例：BRE vs ERE
# BRE：需要跳脫
sed 's/\(foo\|bar\)/baz/g' file.txt

# ERE：更簡潔（推薦）
sed -E 's/(foo|bar)/baz/g' file.txt

進階技巧

多指令組合

範例：多指令
# 方法一：使用 -e
sed -e 's/foo/bar/g' -e 's/baz/qux/g' file.txt

# 方法二：使用分號
sed 's/foo/bar/g; s/baz/qux/g' file.txt

# 方法三：使用大括號對同一行執行多個動作
sed '/pattern/{ s/a/A/g; s/b/B/g; }' file.txt

Hold Space（保持空間）

除了模式空間外，sed 還有一個「保持空間」（Hold Space），可以暫存資料：

指令說明
h將模式空間內容複製到保持空間（覆蓋）
H將模式空間內容附加到保持空間
g將保持空間內容複製到模式空間（覆蓋）
G將保持空間內容附加到模式空間
x交換模式空間與保持空間
範例：反轉檔案行序（模擬 tac）
# 利用 Hold Space 反轉所有行
sed -n '1!G; h; $p' file.txt

多行處理（N / P / D）

指令說明
N讀取下一行並附加到模式空間（以 \n 分隔）
P列印模式空間中第一個 \n 之前的內容
D刪除模式空間中第一個 \n 之前的內容，重新開始循環
範例：合併相鄰行
# 將每兩行合併為一行，用空格分隔
sed 'N; s/\n/ /' file.txt

# 刪除跨行的連續空白行（僅保留一個空行）
sed '/^$/{ N; /^\n$/d; }' file.txt

分支與標籤（b / t / :label）

範例：標籤與迴圈
# 持續替換直到沒有匹配（模擬迴圈）
# 將連續空格壓縮為單一空格
sed ':loop; s/  / /; t loop' file.txt

# 解釋：
# :loop   → 定義標籤 "loop"
# s/  / / → 將兩個空格替換為一個
# t loop  → 如果替換成功，跳回 loop 繼續執行

讀取與寫入檔案（r / w）

範例：讀寫檔案
# 在第 3 行之後插入 header.txt 的內容
sed '3r header.txt' file.txt

# 在匹配 "INSERT_HERE" 的行之後插入檔案內容
sed '/INSERT_HERE/r template.txt' file.txt

# 將含有 "ERROR" 的行寫入新檔案
sed -n '/ERROR/w errors.txt' log.txt

轉換指令（y）

y 指令進行逐字元對應轉換，類似 tr 命令：

範例：字元轉換
# 將小寫 abc 轉為大寫 ABC
echo "apple banana cherry" | sed 'y/abc/ABC/'
OUTPUT
Apple BAnAnA Cherry

實戰應用範例

日誌與系統管理

日誌處理
# 提取特定時間範圍的日誌
sed -n '/2024-01-15 10:00/,/2024-01-15 11:00/p' app.log

# 移除日誌中的 ANSI 色彩碼
sed -E 's/\x1b\[[0-9;]*m//g' colored.log

# 統一 IP 位址格式（隱碼處理）
sed -E 's/[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}/xxx.xxx.xxx.xxx/g' access.log

設定檔處理

設定檔修改
# 修改設定值
sed -i 's/^port=.*/port=8080/' config.ini

# 取消某行的註解（移除行首 #）
sed -i '/^#.*listen_port/s/^#//' nginx.conf

# 加上註解（在行首加 #）
sed -i '/dangerous_option/s/^/#/' config.txt

# 在 [section] 區塊後加入新的設定
sed -i '/^\[database\]/a\host=localhost' config.ini

程式碼與文件處理

程式碼處理
# 移除行尾空白
sed -i 's/[[:space:]]*$//' *.py

# 將 Tab 替換為 4 個空格
sed -i 's/\t/    /g' code.py

# 在每行前加上行號
sed '=' file.txt | sed 'N; s/\n/\t/'

# 移除 HTML 標籤
sed -E 's/<[^>]+>//g' page.html

# CSV：提取第 2 欄
sed -E 's/^[^,]*,([^,]*).*/\1/' data.csv

批次檔案重新命名（搭配其他命令）

批次操作
# 批次重新命名：將空格替換為底線
for f in *; do
  new=$(echo "$f" | sed 's/ /_/g')
  [ "$f" != "$new" ] && mv "$f" "$new"
done

# 批次替換多個檔案中的字串
find . -name "*.txt" -exec sed -i 's/old_api/new_api/g' {} +

# 使用 sed 產生 SQL INSERT 語句
sed -E "s/^(.*),(.*),(.*)\$/INSERT INTO users VALUES ('\1','\2','\3');/" data.csv

常用單行指令集錦

One-liners
# 列印前 N 行（模擬 head）
sed '10q'

# 列印最後一行（模擬 tail -1）
sed -n '$p'

# 移除所有空白行
sed '/^[[:space:]]*$/d'

# 移除行首空白
sed 's/^[[:space:]]*//'

# 移除行尾空白
sed 's/[[:space:]]*$//'

# 移除行首和行尾空白（trim）
sed 's/^[[:space:]]*//; s/[[:space:]]*$//'

# 在每行前加上前綴
sed 's/^/PREFIX: /'

# 在每行後加上後綴
sed 's/$/ :SUFFIX/'

# 將 DOS 換行（\r\n）轉為 Unix 換行（\n）
sed 's/\r$//'

# 雙倍行距（每行後插入空行）
sed 'G'

# 壓縮連續空行為單一空行
sed '/^$/N;/^\n$/d'
安全提醒：使用 sed -i 直接修改檔案前，建議先用 sed -i.bak 建立備份，或先不加 -i 預覽輸出結果，確認無誤後再執行就地修改。
