CHAPTER 01

認識 tmux 與安裝

tmux 解決什麼問題？

想像這個場景：你 SSH 到遠端伺服器跑一個 30 分鐘的編譯，中途網路斷線——所有進度歸零。或是你想同時看 log、編輯程式、監控系統，卻得不斷切換終端視窗。tmux（terminal multiplexer）正是為這些痛點而生。

它讓你：

  • Detach / Attach：把工作階段「放進背景」，稍後從任何地方重新連回，就像從未離開。
  • 分割畫面：一個終端窗口可以切成多個 pane，同時顯示不同指令輸出。
  • 多工階段：為每個專案保留一組視窗與佈局，隨時切換。
  • 遠端穩定：SSH 斷線不會殺掉在 tmux 裡跑的行程。

核心概念：三層結構

tmux 的物件有層次，理解這張圖勝過一百次操作：

Server  (tmux 背景守護程序，每位使用者一個)
│
├── Session A    # 工作階段：一組具名的工作空間
│   ├── Window 0 # 視窗：像瀏覽器分頁
│   │   ├── Pane 0   # 窗格：視窗內被分割出的終端
│   │   └── Pane 1
│   └── Window 1
│       └── Pane 0
│
└── Session B
    └── Window 0
        └── Pane 0
Server
第一次執行 tmux 時自動啟動的背景程序，所有 session 屬於它。
Session
具名的工作階段（例：work、dotfiles），可 detach／attach。
Window
session 內的「分頁」，每個 window 佔滿整個終端畫面。
Pane
window 內的「窗格」，一個 window 可以分割成多個 pane。

macOS（Homebrew）

$ brew install tmux

Ubuntu / Debian

$ sudo apt update
$ sudo apt install tmux

CHAPTER 02

基礎操作與 Prefix Key

Prefix Key 是什麼？

tmux 的所有按鍵操作都有一個共通起手式：先按 prefix key，再按命令鍵。預設的 prefix 是 Ctrl+b（本指南後續以 C-b 表示）。例如想要新開 window，流程是：

  1. 按 Ctrl+b（放開）
  2. 按 c（create window 的縮寫）

這種兩段式設計避免了與 shell 快捷鍵衝突。許多使用者會把 prefix 改成 Ctrl+a（較好按），我們會在第 7 章示範。

表示法約定
本指南中 C-b 代表 Ctrl+b，M-a 代表 Meta/Alt+a。C-b → c 代表「先按 prefix 再按 c」。

兩種操作模式

按鍵模式（key binding）

最常用的方式：C-b 後接一個單鍵，如 c、d、"。速度快、適合日常。

指令模式（command prompt）

按 C-b → : 進入指令提示列，可打完整指令，例如：

:new-session -s mywork
:rename-window logs
:kill-pane

所有按鍵都是某個指令的綁定。想知道 C-b → c 對應哪個指令？用 list-keys。

Detach / Attach：tmux 的靈魂

這是 tmux 最迷人的功能：

  1. 在 tmux 裡工作一陣子，跑著 npm run dev、tail -f 等。
  2. 按 C-b → d（detach），畫面回到純 shell，但 tmux 仍在背景跑。
  3. 關掉終端、斷開 SSH、重開電腦都不影響（前提是 server 未被殺）。
  4. 重新 SSH 回來，輸入 tmux attach（或 tmux a），一切原樣回來。

CHAPTER 03

Session 工作階段管理

為什麼要用具名 session？

預設 tmux 建立的 session 叫 0、1、2……，一多就分不清誰是誰。好習慣是為每個專案建立具名 session：dotfiles、frontend、infra。這樣未來 tmux ls 一目了然，tmux attach -t frontend 也不用記編號。

建立、列出、切換

# 建立具名 session 並進入
$ tmux new -s frontend

# 建立 session 但不進入（背景建立）
$ tmux new -d -s backend

# 列出所有 session
$ tmux ls
frontend: 2 windows (created Mon Dec  1 10:21:33 2025)
backend:  1 windows (created Mon Dec  1 11:00:01 2025)

# 接入特定 session
$ tmux attach -t frontend    # 或簡寫 tmux a -t frontend

# 接入時若目前已在另一個 client，不踢掉原 client
$ tmux attach -t frontend -d  # -d 會 detach 其他 client

在 tmux 內切換 session

已經在 tmux 裡，想跳到另一個 session 不用先 detach——按 C-b → s 開啟tree 模式，上下選、Enter 切入。

或用指令：

:switch-client -t backend
:switch-client -n        # 下一個 session
:switch-client -p        # 上一個 session
:switch-client -l        # 前一個（last）session

重命名與刪除

# 按鍵：
C-b $            # 重命名目前 session

# 指令：
:rename-session new-name
:kill-session -t backend        # 刪除特定 session
:kill-session -a -t frontend    # 刪除「除了 frontend 以外」所有 session

detach-on-destroy：session 消失時

當你所在的 session 被關掉（例如最後一個 window 關閉），tmux 的行為由 detach-on-destroy 選項決定：

on
（預設）client 會 detach 回 shell
off
切換到另一個仍存在的 session
no-detached
若有其他已 detach 的 session 就切過去，否則 detach
previous / next
切到前／後一個 session
3.6 行為調整
tmux 3.6 修正了 detach-on-destroy 的行為，在 off / previous 等模式下，會選擇「最新的」session（符合 man page 描述），先前版本有時會跳到非預期的 session。

實用工作流：one-session-per-project

# 啟動時若 session 存在就進入，否則建立
$ tmux new -A -s work
#      ↑ -A = attach if exists, else create

# 包裝成 alias 放進 .zshrc / .bashrc：
alias tw='tmux new -A -s work'
alias td='tmux new -A -s dotfiles'

CHAPTER 04

Window 視窗管理

Window 像瀏覽器分頁

一個 session 裡可以有任意多個 window，每個 window 佔滿整個終端畫面，底部狀態列顯示所有 window 清單： 

[work] 0:editor  1:server*  2:logs-  3:git        "hostname" 14:25

符號意義：

*
目前的 window
-
上一個使用的 window
!
有活動通知（monitor-activity）
#
有銀鈴通知（bell）
Z
某個 pane 已 zoom

建立與切換

# 按鍵
C-b c            # 建立新 window，焦點跳過去
C-b 0–9         # 跳到該編號 window
C-b n / p        # 下一個 / 上一個
C-b l            # 跳到前一個使用的 window（last）
C-b w            # Tree 模式選 window
C-b f            # 依內容搜尋 window

# 指令
:new-window           # 同 C-b c
:new-window -n editor -c ~/projects   # 命名 + 指定起始目錄
:select-window -t 3   # 切到編號 3
:select-window -t :editor  # 用名稱切

重命名、移動、關閉

C-b ,            # 重命名目前 window
C-b &            # 關閉目前 window（會確認）
C-b .            # 更改 window 編號（Move window index）

:swap-window -s 1 -t 3    # 交換 window 1 與 3
:move-window -t 5       # 把目前 window 移到編號 5
:move-window -s work:1 -t main:9   # 跨 session 移動

Tree 模式搜尋與操作

按 C-b → w 進入 tree 模式，以視覺化方式瀏覽所有 session / window / pane。3.6 在 tree 模式有幾個新按鍵：

3.6 Tree 模式新按鍵
  • / 向前搜尋，n 下一個；N 反向搜尋（3.6 新增）
  • Shift+↑/↓ 移動 window 順序（3.6 新增）
  • x 關閉選中項（session / window / pane），X 強制關閉
  • Enter 切過去

開新 window 時自動設定

常用技巧：開 window 時直接指定名字、目錄、執行指令。

:new-window -n logs -c /var/log 'tail -f syslog'
# -n 名字  -c 起始目錄  最後是要執行的指令

CHAPTER 05

Pane 分割與窗格操作

分割：一個 window 變多個 pane

pane 是 tmux 最強的功能之一——把一個 window 切成多格，同時看 log、跑 server、寫程式。

C-b %            # 垂直分割（vertical split = 左右兩塊）
C-b "            # 水平分割（horizontal split = 上下兩塊）
名稱的陷阱
split-window -h 的 -h 是 horizontal，但它做的是左右分割（在水平方向上並排）。這反直覺，記口訣：-h 左右、-v 上下。按鍵 % 與 " 則符合視覺直覺（百分號像垂直線、引號像水平線）。

切換與導航

C-b o            # 循環到下一個 pane
C-b ;            # 跳到前一個使用的 pane
C-b ↑↓←→         # 按方向移動焦點
C-b q            # 顯示 pane 編號（顯示期間按數字直接跳）
C-b q 2          # 跳到編號 2 的 pane

調整 pane 大小

C-b C-↑↓←→       # 按住 Ctrl 連續微調（1 格）
C-b M-↑↓←→       # 按住 Alt 大幅調整（5 格）

:resize-pane -U 10      # 向上擴 10 格
:resize-pane -D 5       # 向下
:resize-pane -L 5       # 向左
:resize-pane -R 5       # 向右
:resize-pane -x 80 -y 24   # 絕對大小

Zoom：暫時放大一個 pane

C-b → z 把當前 pane 放到全螢幕，狀態列會出現 Z。再按一次還原。非常適合臨時專心看某個 pane 的內容。

移動、交換、合併 pane

C-b {            # 把目前 pane 與前一個交換
C-b }            # 與後一個交換
C-b !            # 把目前 pane 拆成獨立 window（break-pane）

:join-pane -s 2:0 -t 1    # 把 window 2 的第 0 個 pane 併入 window 1
:join-pane -s :editor     # 把 editor window 的 pane 併到目前 window
:break-pane              # 把目前 pane 變成新 window
:swap-pane -s 0 -t 2     # 交換指定 pane

佈局（Layout）：五種預設排列

按 C-b → Space 循環預設佈局，或用指令 :select-layout 指定：

even-horizontal
所有 pane 左右等寬
even-vertical
所有 pane 上下等高
main-horizontal
上方一個大 pane，下方小 pane 等分
main-vertical
左側一個大 pane，右側小 pane 等分
tiled
平均鋪滿整個畫面（格狀）

同步輸入（synchronize-panes）

需要同時對多台伺服器下相同指令？開幾個 pane 各 SSH 到不同主機，然後：

:setw synchronize-panes on      # 開啟同步
:setw synchronize-panes off     # 關閉

之後在任何一個 pane 打字，所有 pane 都會收到相同輸入。操作完務必記得關掉，否則很危險。

3.6 新特性：原生 Pane 捲軸

3.6 亮點

tmux 3.6 新增 pane-scrollbars 選項，讓 pane 右側顯示真正的捲軸（而不是只能進 copy mode 用鍵盤捲動）。

set -g pane-scrollbars on
set -g pane-scrollbars-style 'fg=colour8,bg=colour0'

CHAPTER 06

Copy Mode 複製模式與緩衝區

什麼是 Copy Mode？

在 tmux 內，一般情況下無法用滑鼠滾輪往上捲（沒啟用滑鼠時）。要回看輸出，必須進入 copy mode。這個模式提供類似 vim 或 emacs 的導航鍵，可以瀏覽歷史、選取、複製文字到 tmux 的 buffer，再貼到別處。

進出 Copy Mode

C-b [            # 進入 copy mode
q                # 離開 copy mode
Esc              # （vi 模式）離開選取但仍在 copy mode

C-b ]            # 貼上最近一次複製的內容

3.6 改進

TPM 與必裝外掛

什麼是 TPM？

Tmux Plugin Manager（tmux-plugins/tpm）是最流行的 tmux 外掛管理器。它讓安裝、更新、移除外掛變得像 vim 的 vim-plug 一樣簡單，只要寫一行 config 宣告，按一個快捷鍵就裝好。

安裝 TPM

$ git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm

然後在 ~/.tmux.conf 最底部加入：

# === TPM 外掛區塊 ===
set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'tmux-plugins/tmux-sensible'

# 你想要的其他外掛寫在這邊...

# 初始化 TPM（必須是設定檔的最後一行）
run '~/.tmux/plugins/tpm/tpm'

然後在 tmux 內按 C-b → I（大寫）安裝。

TPM 快捷鍵

C-b I
安裝宣告的外掛（Install）
C-b U
更新外掛（Update）
C-b M-u
移除未列在 config 的外掛（clean）

必裝外掛清單

tmux-sensible（強烈推薦）

一組「所有人都應該這樣設」的預設值，包含合理的 escape-time、history-limit、聚焦事件等。裝了就忘了它。

set -g @plugin 'tmux-plugins/tmux-sensible'

tmux-resurrect（救命級）

可以手動儲存與還原整個 tmux 環境——所有 session、window、pane、工作目錄，甚至可還原 vim / neovim 的開啟檔案。電腦重開機不再是災難。

set -g @plugin 'tmux-plugins/tmux-resurrect'

# 還原 vim/neovim 編輯中的檔案
set -g @resurrect-strategy-nvim 'session'

set -g @continuum-save-interval '15'

進階技巧 與腳本化工作流

send-keys：遠端送鍵到其他 pane

可以從一個 pane 對另一個 pane 發送按鍵，常用於自動化啟動。

# 對目前 pane 發送 "ls" + Enter
:send-keys 'ls' Enter

# 對 session work 的 window 1 的 pane 2 發送
:send-keys -t work:1.2 'npm run dev' Enter

# 送 Ctrl-C（中斷行程）
:send-keys -t 1 C-c

if-shell 與 run-shell

# if-shell：依 shell 條件決定執行哪條指令
if-shell '[ -f ~/.tmux.local ]' 'source-file ~/.tmux.local'

# run-shell：執行外部指令（通常用於外掛）
run-shell '~/scripts/my-tmux-helper.sh'

Hooks：事件觸發指令

tmux 可以在特定事件發生時自動執行指令。

# Window 關閉時自動補齊編號
set-hook -g window-linked 'run "tmux list-windows"'

# Session 建立時做事
set-hook -g session-created 'run "~/.tmux/on-session-create.sh"'

# 3.6 新增:指令執行錯誤時
set-hook -g command-error 'display "⚠ tmux command failed"'

# 常用 hook 事件
# pane-focus-in / pane-focus-out
# session-created / session-closed
# window-linked / window-unlinked
# client-attached / client-detached
# command-error (3.6 新增)

display-menu 與 display-popup

tmux 3.0+ 支援彈出式選單與視窗。3.6 又增加了幾個有用的旗標。

display-menu 彈出選單

bind C-m display-menu -T "#[align=centre] 主選單 " -x P -y P \
  "New Window"       n "new-window" \
  "Split Horizontal" s "split-window -v" \
  "Split Vertical"   v "split-window -h" \
  "" \
  "Kill Pane"        x "kill-pane" \
  "Kill Window"      X "kill-window"
3.6 新增旗標
  • display-menu -M：強制開啟滑鼠支援（即便未全域啟用 mouse）
  • display-menu -O：放掉滑鼠鍵後不自動關閉選單
  • display-menu -x / -y 支援 format 展開（可動態定位）

display-popup 彈出視窗

# 在中央開一個 80×24 的 popup 跑 lazygit
:display-popup -w 80% -h 80% -E 'lazygit'

# -E 代表命令結束後自動關閉 popup
# -d 指定起始目錄
# -w -h 可用百分比或絕對數字

# 綁定熱鍵：C-b g 開 lazygit
bind g display-popup -E -w 90% -h 90% 'lazygit'

自動化啟動佈局

常見做法：寫一個 shell 腳本，幾秒鐘內開好整個工作環境。

#!/usr/bin/env bash
# 啟動一個開發環境：backend + frontend + logs

SESSION="dev"

# 若 session 已存在就直接接入
tmux has-session -t $SESSION 2>/dev/null
if [ $? != 0 ]; then
  tmux new-session -d -s $SESSION -n editor -c ~/code
  tmux send-keys -t $SESSION:editor 'nvim .' Enter

  tmux new-window -t $SESSION -n server -c ~/code/backend
  tmux send-keys -t $SESSION:server 'npm run dev' Enter

  tmux new-window -t $SESSION -n web -c ~/code/frontend
  tmux split-window -h -t $SESSION:web
  tmux send-keys -t $SESSION:web.1 'npm run dev' Enter

CHAPTER 11

疑難排解 與效能調校

問題 1：顏色不對 / Vim 配色怪怪的

症狀：在 tmux 外看 vim 配色正常，進 tmux 就變色系錯亂。原因通常是 $TERM 與 truecolor 設定。

# 推薦組合（tmux 3.2+）
set -g default-terminal "tmux-256color"
set -ga terminal-overrides ",*256col*:Tc"
set -ga terminal-overrides ",xterm-256color:RGB"

檢查：

$ echo $TERM                 # tmux 內應為 tmux-256color
$ tmux info | grep -i rgb    # 應看到 RGB/Tc 相關 capability

問題 2：按 Esc 有延遲（Vim 特別明顯）

tmux 把 Esc 視為多鍵序列起點，會等一段時間看後續輸入。解法：

set -sg escape-time 10
# 3.6 預設已改為 10ms，如果已升級可省略

問題 3：SSH Agent 斷線後失效

tmux session 裡的 $SSH_AUTH_SOCK 指向舊的 socket，重新 SSH 進來後對不上。解法是讓它固定指向一個符號連結：

# 放在 ~/.ssh/rc
if [ ! -z "$SSH_AUTH_SOCK" ] && [ "$SSH_AUTH_SOCK" != "$HOME/.ssh/ssh_auth_sock" ]; then
    ln -sf "$SSH_AUTH_SOCK" "$HOME/.ssh/ssh_auth_sock"
fi
# 告訴 tmux 繼承這個變數
set -g update-environment "SSH_AUTH_SOCK SSH_CONNECTION DISPLAY"
# 進入 shell 時指向固定路徑
export SSH_AUTH_SOCK="$HOME/.ssh/ssh_auth_sock"
3.6 改進
update-environment 在 3.6 支援 fnmatch 萬用字元，可以用 SSH_* 一次包含所有 SSH_ 開頭的變數。

問題 4：nested tmux（多層 tmux）

SSH 到遠端主機發現裡面也有 tmux，按 prefix 會被外層攔截。解法是送兩次 prefix：

C-a C-a c    # 第一個 C-a 被外層吃，第二個送給內層

或改變嵌套時的 prefix 切換模式：

# F12 切換外層 prefix（讓內層獨佔）
bind -T root F12 \
  set prefix None \; \
  set key-table off \; \
  set status-style "fg=default,bg=colour235" \; \
  refresh-client -S

bind -T off F12 \
  set -u prefix \; \
  set -u key-table \; \
  set -u status-style \; \
  refresh-client -S

除錯：-v / -vv 模式

$ tmux -vv new -s debug
# 會在當前目錄產生：
# - tmux-server-NNNN.log
# - tmux-client-NNNN.log
# - tmux-out-NNNN.log

Server 協定版本不符

3.6 升級提示
tmux 3.6 升級了協定版本。若升級後看到 protocol version mismatch 錯誤，代表舊版 server 還在跑。解法：先 tmux kill-server（會關掉所有現有 session），再用新版重啟。

常用診斷指令

:show-options -g            # 列出所有 global 選項
:show-window-options -g     # 列出所有 window 選項
:list-keys                  # 列出所有按鍵綁定
:list-keys -T copy-mode-vi  # 列出 copy mode vi 的按鍵
:show-messages              # 顯示最近訊息 / 錯誤
:info                       # 顯示 server 資訊（等同 tmux info）

CHAPTER 12

tmux 3.6 新特性總覽

本章集中整理從 3.5a 到 3.6 的所有重要變更，供既有使用者快速盤點是否升級、設定檔是否需要調整。tmux 3.6 於 2025-11-26 釋出 3.6 正式版，2025-12-05 發布 bugfix 版本 3.6a。

亮點功能

① 原生 pane 捲軸

最有感的視覺改進。以前要看 pane 歷史必須進 copy mode，現在直接顯示右側捲軸。

set -g pane-scrollbars on
set -g pane-scrollbars-style 'fg=colour8,bg=default'
set -g pane-scrollbars-position right   # or left

② Mode 2031 自動主題偵測

tmux 自動偵測終端是暗色還是亮色主題，配合新 format 變數 #{client_theme} 動態調整 UI。macOS 切換系統外觀時，tmux 會即時響應。

③ escape-time 預設縮短到 10ms

以前預設 500ms 造成 vim 使用者按 Esc 有明顯延遲，幾乎所有教學都會叫你調低。3.6 終於改到合理值。

④ 3.6 新預設 ? 摘要

以前 C-b → ? 是一長串 list-keys 輸出，現在顯示可讀的摘要。同時新增 C-b → /：按完後再按任何鍵，會顯示那個鍵的註解說明。

設定與語法改進

  • bind -N "note"：只改註解不綁指令。
  • unbind -q：隱藏「未綁定」錯誤訊息。
  • set-option -F、source-file -F：參數可含 format 展開。
  • update-environment 支援 fnmatch 萬用字元（如 SSH_*）。
  • color 寫法（美式）被接受，與 colour 通用。
  • set-buffer -w / load-buffer -w：送到 OSC 52 系統剪貼簿。

Copy Mode 與搜尋

  • 新增 selection-mode 指令，明確切換字元 / 列 / 矩形選取。
  • Tree 模式新增 N（反向搜尋）、Shift+↑/↓（移動 window）。
  • 新增 searchpresent、searchmatch format 變數。

Menu / Popup 增強

  • display-menu -M：強制開啟滑鼠支援。
  • display-menu -O：滑鼠釋放不自動關閉。
  • display-menu -x -y 支援 format 展開。
  • display-popup 若在 popup 裡再叫，會修改目前 popup（而非新開）。

Hooks 與事件

  • 新增 command-error hook（指令失敗時觸發）。
  • allow-set-title 選項：可禁止應用程式修改 pane 標題。

Format 系統增強

  • Boolean 表達式 &&、|| 支援多參數。
  • 新增 ! / !!（not / not-not）運算子。
  • #? 條件表達式支援 else if。
  • 新增 n: modifier 取得字串長度。
  • 新增 w modifier 取得字串顯示寬度（處理寬字元）。

終端相容性

  • 支援 DECSLRM / DECFRA（iTerm2 偵測後自動啟用，水平分割大幅加速）。
  • iTerm2 DSR 1337 取終端版本。
  • 改善 Unicode 處理：regional indicator、emoji modifier、Hangul filler。
  • Alternate screen 下保存並還原圖片。

其他改進

  • Clock 模式新增秒數顯示選項。
  • default-client-command：可改變 tmux 無參數啟動時的預設行為。
  • 慢終端偵測：輸出過多時會先丟棄並重繪，避免卡死。
  • 新增多個 session/window/pane 相關 format 變數。

破壞性變更

注意事項
  • 協定版本 bump：升級後需 tmux kill-server 重啟，否則舊 client 無法連。
  • escape-time 預設變更：若你之前手動設大值（例如 300ms）來避開問題，現在可以移除。
  • Any key 綁定：root table 綁定現在也會在 copy mode 生效（除非被覆蓋）。
  • config 執行環境：config file 執行時不再綁定到第一個 client（恢復早期行為）。

是否升級？

一般用戶
強烈建議升級。escape-time、scrollbar、主題自動偵測都是日常有感的改善。
設定檔複雜的進階用戶
建議先在測試環境跑一次 config，確認沒有用到被修改的行為。
伺服器生產環境
若套件庫沒有 3.6，可維持 3.5a 亦無大礙；新功能多為個人生產力改進而非伺服器關鍵。

# 作者：tmux 快速學習指南
# 適用：macOS / Linux / WSL
# ======================================================= 

# ──────────── Prefix ────────────
set -g prefix C-a
unbind C-b
bind C-a send-prefix

# ──────────── 基礎行為 ────────────

setw -g pane-base-index 1
set -g renumber-windows on
set -g mouse on
setw -g mode-keys vi
set -sg escape-time 10
set -g history-limit 50000
set -g focus-events on
setw -g monitor-activity on
set -g visual-activity off

set -ga terminal-overrides ",xterm-256color:RGB"

bind | split-window -h -c "#{pane_current_path}"
bind - split-window -v -c "#{pane_current_path}"
