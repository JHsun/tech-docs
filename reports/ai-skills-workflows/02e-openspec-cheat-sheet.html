project.md → config.yaml 遷移

openspec/project.md 不會被自動刪除（可能包含使用者內容）。需手動處理。

遷移步驟

1. 檢視 project.md 內容
2. 將有用的部分搬到 config.yaml 的 context: 欄位
3. 確認無用後手動刪除 project.md

關鍵差異

舊版 project.md新版 config.yaml context
被動式——AI 可能讀、可能不讀主動注入到每個規劃請求中
格式自由，內容可能冗長建議精簡：技術堆疊、關鍵約束、慣例

AI 輔助遷移提示語

我正在從 OpenSpec 舊版 project.md 遷移到新版 config.yaml 格式。
以下是我目前的 project.md：[貼上內容]

請幫我建立 config.yaml：
1. 精簡的 context: 欄位（技術堆疊、關鍵約束、慣例）
2. 如有 artifact 特定的內容，放入 rules: 欄位
刪掉通用的內容，盡可能精簡。

執行遷移

# 方法 1：重新初始化（新增工具或重新配置）
openspec init

# 方法 2：僅更新檔案（既有工具）
openspec update

# CI / 非互動式
openspec init --force
init 和 update 都會自動偵測 legacy 檔案並引導清理。新安裝預設 core profile；遷移安裝保留先前的 workflows，寫入 custom profile。

遷移後的目錄結構

project/
├── openspec/
│   ├── specs/              # 不變
│   ├── changes/            # 不變
│   │   └── archive/        # 不變
│   └── config.yaml         # 新增：專案設定
├── .claude/
│   └── skills/             # 新增：OPSX 技能
│       ├── openspec-propose/
│       ├── openspec-explore/
│       ├── openspec-apply-change/
│       └── openspec-archive-change/
└── CLAUDE.md               # 保留（OpenSpec 標記已移除）

遷移後啟用擴展指令

openspec config profile    # 選擇 expanded workflows
openspec update            # 套用到專案

疑難排解

問題解決方式
CI / 非互動式環境openspec init --force
IDE 未偵測到新指令重新啟動 IDE（技能在啟動時載入）
rules: 欄位不生效確認 key 與 schema artifact ID 匹配
project.md 仍存在設計如此——手動檢視、遷移、刪除
想預覽偵測結果執行 init 並在清理提示時選「否」

日常開發流程

# === 安裝與初始化 ===
npm install -g @fission-ai/openspec@latest
cd your-project && openspec init

# === Core 工作流（4 個指令）===
/opsx:explore                    # 思考與探索
/opsx:propose add-feature        # 規劃一切
/opsx:apply                      # 實作
/opsx:archive                    # 歸檔

# === Expanded 工作流（需啟用）===
openspec config profile          # 啟用
openspec update                  # 套用
/opsx:new add-feature            # 建立骨架
/opsx:continue                   # 逐步建立文件
/opsx:ff                         # 或一次全建
/opsx:apply                      # 實作
/opsx:verify                     # 驗證
/opsx:archive                    # 歸檔

# === 管理指令 ===
openspec list                    # 列出變更
openspec show <name>             # 查看詳情
openspec validate <name>         # 驗證結構
openspec view                    # 互動式儀表板

# === 升級 ===
npm install -g @fission-ai/openspec@latest
openspec update                  # 在每個專案中執行

推薦的 AI 模型

規劃與實作皆推薦：Opus 4.5 和 GPT 5.2
