專案設定指令

openspec init [path]

在專案中初始化 OpenSpec，建立資料夾結構並配置 AI 工具整合。

旗標說明範例
--tools <list>指定工具：all、none、或逗號分隔--tools claude,cursor
--profile <n>覆蓋 profile：core 或 custom--profile core
--force跳過提示，自動清理 legacy 檔案
openspec init                          # 互動式
openspec init ./my-project             # 指定目錄
openspec init --tools claude,cursor    # 非互動式
openspec init --tools all --force      # 所有工具，跳過提示

openspec update [path]

升級 CLI 後，重新產生 AI 工具設定檔。使用當前全域 profile、workflows、delivery 模式。

旗標說明
--force即使已是最新版也強制更新
openspec update            # 標準更新
openspec update --force    # 強制重新產生

openspec config

查看和修改全域 OpenSpec 設定。

子指令說明
config list顯示當前設定（如與專案不同步會提醒）
config profile互動式切換 profile / delivery / workflows
config edit用編輯器開啟設定檔
openspec config list              # 查看設定
openspec config profile           # 修改 profile
# profile 子選項：
#   Change delivery only → skills / commands / both
#   Change workflows only → 勾選/取消 workflow
#   Change both
全域設定存放在 ~/.config/openspec/config.json（或 $XDG_CONFIG_HOME/openspec/config.json）
查閱與驗證指令

openspec list

列出專案中的變更或規格。

旗標說明預設
--specs列出 specs 而非 changesfalse
--changes明確指定列出 changestrue
--sort <order>排序：recent 或 namerecent
--jsonJSON 輸出（供腳本使用）false
openspec list                  # 列出所有進行中的變更
openspec list --specs          # 列出所有 specs
openspec list --json           # JSON 格式

openspec show [item]

顯示變更或 spec 的詳細資訊。

旗標說明
--type spec指定顯示 spec（消歧義）
--jsonJSON 輸出
--no-interactive跳過選擇提示
openspec show                          # 互動式選擇
openspec show add-dark-mode            # 特定變更
openspec show auth --type spec         # 特定 spec
openspec show add-dark-mode --json     # JSON

openspec validate [change-name]

驗證變更和 specs 的結構問題。

旗標說明預設
--all驗證所有（changes + specs）false
--changes驗證所有 changesfalse
--jsonJSON 輸出（CI 用）false
--strict警告視為錯誤false
--concurrency <n>平行驗證數量6
--no-interactive停用 spinner（CI/pre-commit）false
openspec validate                                    # 互動式
openspec validate add-dark-mode                      # 特定變更
openspec validate --all --json                       # 全部 JSON
openspec validate --all --strict --concurrency 12    # 嚴格模式

openspec view

開啟互動式終端儀表板，瀏覽 specs 和 changes。無旗標。

openspec view
歸檔與 Schema 指令

openspec archive [change-name]

旗標說明
--yes跳過確認提示（CI / 腳本用）
--skip-specs歸檔但不合併 specs（工具性變更）
openspec archive                                # 互動式
openspec archive add-dark-mode --yes            # 非互動式
openspec archive update-ci-config --skip-specs  # 跳過 spec 合併

openspec schema 子指令

子指令說明
schema init <name>建立新的自訂 schema（互動式）
schema fork <source> <name>複製既有 schema 進行自訂
schema validate [name]驗證 schema 結構（省略 name 驗證全部）
schema which <name>顯示解析來源（--all 列出全部）
openspec schema fork spec-driven my-workflow
openspec schema init rapid \
  --description "快速迭代" \
  --artifacts "proposal,tasks" \
  --default
openspec schema validate my-workflow
openspec schema which spec-driven
# → spec-driven resolves from: package

openspec new change <name>

旗標說明
--schema <name>指定使用的 schema
--description <text>可選描述文字

旗標說明

openspec templates

openspec feedback

openspec feedback "功能需求" \
  --body "詳細描述..."
