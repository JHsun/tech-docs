📦 安裝指南

一鍵安裝（交互式）
npx get-shit-done-cc@latest

安裝器會引導選擇 Runtime（支援多選）→ Location（Global 全域 / Local 本地專案）。
適用 Mac、Windows、Linux。需 Node.js ≥ 20

非交互式安裝（所有 Runtime）
Runtime全域安裝指令安裝路徑
Claude Code--claude --global~/.claude/
OpenCode--opencode --global~/.config/opencode/
Gemini CLI--gemini --global~/.gemini/
Codex--codex --global~/.codex/
Copilot--copilot --global~/.github/
Cursor--cursor --global~/.cursor/
Windsurf--windsurf --global~/.windsurf/
Antigravity--antigravity --global~/.antigravity/
Kilo--kilo --global~/.kilo/
Augment--augment --global~/.augment/
Trae--trae --global~/.trae/
Cline--cline --global~/.cline/
全部安裝--all --global以上全部

所有指令前綴皆為 npx get-shit-done-cc。本地安裝將 --global 替換為 --local。

驗證安裝
Runtime驗證指令
Claude Code / Gemini / Copilot / Antigravity / Kilo / Augment / Trae / Cline/gsd:help
OpenCode/gsd-help
Codex$gsd-help
權限設定

推薦：跳過權限確認（GSD 設計為無摩擦自動化，每個 phase 需確認 50+ 次操作會嚴重影響體驗）

claude --dangerously-skip-permissions
▸ 或使用細粒度權限 JSON 
{
  "permissions": {
    "allow": [
      "Bash(date:*)", "Bash(echo:*)", "Bash(cat:*)",
      "Bash(ls:*)", "Bash(mkdir:*)", "Bash(wc:*)",
      "Bash(head:*)", "Bash(tail:*)", "Bash(sort:*)",
      "Bash(grep:*)", "Bash(tr:*)",
      "Bash(git add:*)", "Bash(git commit:*)",
      "Bash(git status:*)", "Bash(git log:*)",
      "Bash(git diff:*)", "Bash(git tag:*)"
    ]
  }
}

存放於 .claude/settings.json

更新 ＆ 卸載
# 更新到最新版
npx get-shit-done-cc@latest

# 卸載（範例：Claude Code 全域）
npx get-shit-done-cc --claude --global --uninstall
💡 Docker / 容器環境：若 ~ 路徑無法展開，先設定 CLAUDE_CONFIG_DIR=/home/youruser/.claude 再安裝。

📋 核心步驟詳解

1

初始化專案 /gsd:new-project

系統動作說明
Questions深入提問直到完全理解你的想法（目標、約束、技術偏好、邊界情況）
Research生成 4 個並行研究代理調查領域（可選但推薦）
Requirements分離 v1（必做）、v2（之後做）和 Out of Scope
Roadmap將需求映射為可執行的阶段序列

產生檔案： PROJECT.md REQUIREMENTS.md ROADMAP.md STATE.md config.json research/

棕地專案（已有程式碼）：先跑 /gsd:map-codebase → 再 /gsd:new-project，問題會聚焦於你要新增什麼。
從文件自動初始化：/gsd:new-project --auto @prd.md
2

討論階段 /gsd:discuss-phase N

在研究或規劃之前鎖定你的實作偏好。系統會分析階段並識別灰色區域。

功能類型討論重點
視覺功能布局、密度、交互方式、空狀態處理
API / CLI回應格式、命令標誌、錯誤處理、詳細度
內容系統結構、語調、深度、呈現流程
組織任務分組標準、命名規則、重複處理

產生檔案：CONTEXT.md（含 Decisions / Deferred Ideas / Claude's Discretion）

兩種討論模式：
discuss（預設） 針對灰色區域提出開放式問題
assumptions 先讀程式碼 → 給出假設 → 只要求糾正錯誤
切換：/gsd:settings → workflow.discuss_mode
3

規劃階段 /gsd:plan-phase N

生成 2–3 個原子計畫，每個都夠小以在單一上下文窗口內完成執行。

內部流程：
① 4 個並行研究員（技術棧 / 功能 / 架構 / 陷阱）→ RESEARCH.md
② 規劃器讀取全部上下文 → 產出 XX-YY-PLAN.md（XML 結構化）
③ Plan Checker 驗證循環（最多 3 次）→ PASS 或迭代修改

產生檔案： {phase}-RESEARCH.md {phase}-{N}-PLAN.md

標誌： --no-research 跳過研究 ・ --reviews 載入程式碼審查發現

4

執行階段 /gsd:execute-phase N

核心機制：計畫按依賴關係分組為波次（Waves），波次內並行、波次間順序。每個執行器獲得全新 200K 上下文。

WAVE 1 並行
Plan 01 User Model
Plan 02 Product Model
→
WAVE 2 並行
Plan 03 Orders API
Plan 04 Cart API
→
WAVE 3
Plan 05 Checkout UI

獨立計畫 → 同一 Wave 並行 ｜ 有依賴 → 下一 Wave 等待 ｜ 檔案衝突 → 同一計畫或順序

產生檔案： XX-YY-SUMMARY.md VERIFICATION.md
Git 提交：每個 task 完成後立即原子提交，格式如 feat(08-02): implement password hashing

5

驗證工作 /gsd:verify-work N

① 提取可測試交付物（「你現在應該能做什麼」）
② 逐一走查（「你能用信箱登入嗎？」→ 是 / 否）
③ 自動診斷失敗 → 生成調試代理找根因 → 建立修復計畫

產生檔案：VERIFICATION.md（成功/失敗結果）・若有問題產出修復計畫

6

發布 → 完成 → 下一里程碑

/gsd:ship N                  # 建立 PR
/gsd:ui-review N             # 6 柱視覺審計（前端可選）
/gsd:audit-milestone         # 檢查完成度
/gsd:complete-milestone      # 歸檔 + 打標籤
/gsd:new-milestone [name]    # 開始下一版本週期

⚡ Quick Mode 快速模式

/gsd:quick — 用於不需完整規劃的即時任務（bug 修復、小功能、設定調整）， 仍享有 GSD 保障（原子提交、狀態追蹤）。 

/gsd:quick
> "Add dark mode toggle to settings"
可組合標誌
標誌功能
--discuss先收集上下文偏好
--research生成研究代理調查實作方法、套件選項、潛在問題
--full啟用計畫檢查（最多 2 次）+ 執行後驗證

可組合使用：/gsd:quick --discuss --research --full = 討論 + 研究 + 計畫檢查 + 驗證

另有：/gsd:fast <text> — 更輕量，完全跳過規劃直接執行（拼寫修正、微調設定等瑣碎任務）。

🚀 典型使用流程速查

# ═══ 全新專案完整週期 ═══
claude --dangerously-skip-permissions
/gsd:new-project            # 回答問題 → 配置 → 批准路線圖
/clear
/gsd:discuss-phase 1        # 鎖定偏好
/gsd:ui-phase 1             # 設計契約（前端階段）
/gsd:plan-phase 1           # 研究 + 計畫 + 驗證
/gsd:execute-phase 1        # 並行執行
/gsd:verify-work 1          # 人工驗收
/gsd:ship 1                 # 建立 PR
/clear
/gsd:next                   # 自動偵測下一步
...重複每個階段...
/gsd:complete-milestone     # 歸檔 + 打標籤

# ═══ 從已有文件初始化 ═══
/gsd:new-project --auto @prd.md

# ═══ 棕地專案（已有程式碼）═══
/gsd:scan                   # 輕量程式碼庫掃描（--focus tech/arch/quality/concerns）
/gsd:map-codebase           # 深度分析（4 個並行代理）
/gsd:new-project

# ═══ 自主全自動執行 ═══
/gsd:autonomous             # 自動執行所有剩餘 phase（--from N --to N --interactive）

# ═══ 快速修復 ═══
/gsd:quick > "Fix login button on mobile Safari"

# ═══ 恢復會話 ═══
/gsd:resume-work   # 或 /gsd:progress
