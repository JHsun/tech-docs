⌨️ 完整指令參考

分類標籤： 核心 導航 階段管理 UI 設計 代碼品質 Backlog 工作流 工具

🔶 核心工作流程
指令用途常用標誌
/gsd-new-project完整初始化：提問→研究→需求→路線圖--auto @file
/gsd-discuss-phase [N]鎖定實作偏好--batch --analyze --auto
/gsd-plan-phase [N]研究 + 計畫 + 驗證--no-research --reviews
/gsd-execute-phase <N>波次並行執行所有計畫—
/gsd-verify-work [N]人工驗收 + 自動診斷—
/gsd-ship [N]建立 PR--draft
/gsd-next自動偵測狀態並執行下一步—
/gsd-fast <text>瑣碎任務直接執行（跳過規劃）—
/gsd-quick快速任務（含 GSD 保障）--discuss --research --full
/gsd-audit-milestone驗證里程碑完成度—
/gsd-complete-milestone歸檔 + 打標籤—
/gsd-new-milestone [name]開始下一版本週期—
/gsd-forensics [desc]工作流失敗事後診斷—
/gsd-milestone-summary已完成里程碑摘要（入職用）—
/gsd-autonomous自動執行所有剩餘階段（完全自主模式）—
/gsd-explore <idea>蘇格拉底式構想探索（互動腦力激盪）—
/gsd-import匯入外部計畫（含一致性驗證）—
🔵 導航指令
指令用途
/gsd-progress我在哪裡？下一步是什麼？
/gsd-resume-work從上次會話恢復完整上下文
/gsd-pause-work保存交接（寫入 HANDOFF.json）
/gsd-session-report生成會話摘要報告
/gsd-help顯示所有指令
/gsd-update更新 GSD 並預覽變更日誌
/gsd-manager互動式指令中心（多階段管理）
🟢 階段管理
指令用途
/gsd-add-phase追加新階段到路線圖
/gsd-insert-phase [N]插入緊急工作（十進制編號）
/gsd-remove-phase [N]移除未來階段並重新編號
/gsd-list-phase-assumptions [N]預覽 Claude 的預期方法（僅輸出）
/gsd-plan-milestone-gaps為審計差距建立補充階段
/gsd-research-phase [N]僅深度研究（不規劃）
/gsd-validate-phase [N]追溯驗證審計
/gsd-map-codebase [area]分析現有程式碼（並行代理）
/gsd-analyze-dependencies分析階段間依賴關係與執行順序
/gsd-cleanup歸檔已累積的階段目錄（清理 .planning/）
🔴 UI 設計
指令用途產出
/gsd-ui-phase [N]生成設計契約（間距/色彩/字型/文案）UI-SPEC.md
/gsd-ui-review [N]追溯 6 柱視覺審計UI-REVIEW.md

6 柱評分（1-4 分）：Copywriting・Visuals・Color・Typography・Spacing・Experience Design

/gsd-audit-fix自主審計到修復的完整管道

⚙️ 設定配置

配置存於 .planning/config.json。透過 /gsd-new-project 建立，或 /gsd-settings 修改。規則：未設定 = 啟用。

核心設定
設定選項預設說明
modeinteractive / yolointeractive每步確認 vs 自動批准
granularitycoarse / standard / finestandard階段粒度（3-5 / 5-8 / 8-12 階段）
模型配置（Model Profiles）

控制每個代理使用哪個 Claude 模型。用 /gsd-set-profile <name> 快速切換。

🏆 quality

規劃 = Opus
執行 = Opus
驗證 = Sonnet
適合配額充足、工作關鍵

⚖️ balanced 預設

規劃 = Opus
執行 = Sonnet
驗證 = Sonnet
僅架構決策用 Opus

💰 budget

規劃 = Sonnet
執行 = Sonnet
驗證 = Haiku
高量或非關鍵工作

🔗 inherit

所有代理 = 當前會話模型
適合非 Anthropic 提供者
（OpenRouter、本地模型）

工作流開關（Workflow Agents）
設定預設說明
workflow.research✅ true計畫前的領域調查
workflow.plan_check✅ true計畫驗證循環（最多 3 次）
workflow.verifier✅ true執行後自動驗證
workflow.nyquist_validation✅ true測試覆蓋率映射到需求
workflow.ui_phase✅ true前端階段生成 UI 設計契約
workflow.discuss_modestandardstandard（開放式）/ assumptions（假設式）
workflow.skip_discuss❌ false自主模式下跳過討論
workflow.research_before_questions❌ false先研究再討論
workflow.auto_advance❌ false自動串接 discuss→plan→execute
workflow.text_mode❌ false純文字模式（無 TUI 選單，適合遠端）
Git 分支策略
策略建立分支適用場景
none 預設不建立，提交到當前分支單人開發、簡單專案
phase每個 execute-phase 建立分支每階段程式碼審查
milestone首次 execute-phase 建立一個分支每版本一個 PR

分支名稱範本：gsd/phase-{phase}-{slug}（可在 config.json 中自訂）
