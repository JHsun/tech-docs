OPSX 斜線指令完整參考（commands.md）

Core Profile 指令（預設安裝）

指令說明作用
/opsx:propose <name>一步完成提案建立變更資料夾 + 一次產生所有規劃文件（proposal、specs、design、tasks）
/opsx:explore思考與探索讀取程式碼、繪製架構圖、問問題。絕不寫程式碼
/opsx:apply [name]實作任務依 tasks.md 的勾選框逐一實作，完成後打勾
/opsx:archive [name]歸檔變更合併 delta specs → 移至 archive/ → 更新真實來源

Expanded Workflow 指令（需啟用）

指令說明作用
/opsx:new <name>建立變更骨架僅建立資料夾 + .openspec.yaml 中繼資料
/opsx:continue [name]逐步建立文件依依賴鏈建立下一個 artifact（一次一個）
/opsx:ff [name]快轉一次建立所有尚未完成的規劃文件
/opsx:verify [name]驗證實作檢查完成度、正確性、一致性（不阻擋歸檔）
/opsx:sync [name]同步規格將 delta specs 合併到主規格，但不歸檔
/opsx:bulk-archive批次歸檔一次歸檔多個已完成的變更，偵測衝突
/opsx:onboard引導式教學約 15 分鐘的互動式教程，使用真實程式碼庫
# 啟用擴展指令
openspec config profile    # 選擇 expanded 或自訂 workflows
openspec update            # 套用到專案

工作流模式（workflows.md）

模式 1：快速變更（預設）

/opsx:propose→ /opsx:apply→ /opsx:archive

或搭配 expanded：

/opsx:new→ /opsx:ff→ /opsx:apply→ /opsx:archive

適用：中小功能、bug 修復、需求明確的變更。

模式 2：探索優先

/opsx:explore→ /opsx:new→ /opsx:continue→ ×N /opsx:apply

適用：效能優化、除錯、架構決策、需求不明確時。先調查再正式化。

模式 3：逐步推進

/opsx:new→ /opsx:continue→ /opsx:continue→ /opsx:continue→ /opsx:apply

適用：複雜功能，需要逐一審查每個 artifact 再繼續。Brownfield 專案推薦此模式。

模式 4：並行切換

變更 A：/opsx:new → /opsx:ff → /opsx:apply（進行中）
                                      │
                                 context switch
                                      │
變更 B：/opsx:new → /opsx:ff ──────► /opsx:apply
                                      │
                                  回到變更 A
                                      │
你：/opsx:apply add-dark-mode         ← AI 自動接續上次進度

適用：並行工作流、緊急插件、團隊協作。

模式 5：批次歸檔

你：/opsx:bulk-archive
AI：Found 3 completed changes:
    - add-dark-mode (8/8 tasks)
    - fix-login-bug (3/3 tasks)
    - update-footer (2/2 tasks)
    ⚠ add-dark-mode and update-footer both touch specs/ui/
    Will merge in chronological order.
    Archive all 3 changes?
你：Yes
AI：✓ Archived all 3. Specs merged.

自動偵測 spec 衝突，依建立時間順序合併。

工作流選擇指南

場景推薦模式核心指令
需求清楚、小功能快速變更propose → apply → archive
需求不明、需調查探索優先explore → new → continue...
複雜功能、需審查逐步推進new → continue ×N → apply
被緊急任務打斷並行切換開新 change，稍後 apply <name>
多個變更已完成批次歸檔bulk-archive
Brownfield 既有專案逐步推進new → continue ×N → apply
